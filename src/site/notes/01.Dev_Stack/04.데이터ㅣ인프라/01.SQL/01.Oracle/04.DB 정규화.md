## 📂 데이터베이스 정규화 정리 (1NF - 3NF)

정규화는 데이터 중복을 최소화하고 구조를 단순화하여 데이터 무결성을 유지하기 위한 필수 과정입니다.

### 1. 제1정규형 (1NF): 원자값 확보 (ATOMIC VALUES)

* **규칙:** 모든 컬럼은 더 이상 쪼갤 수 없는 **단일 값(ATOMIC VALUE)**만 가져야 합니다.
* **해결 방법:** 하나의 칸에 여러 데이터(예: '산책, 목욕')가 들어있다면, 행을 분리하거나 별도의 테이블로 구성합니다.
* **목적:** 데이터 검색 및 가공의 효율성을 높이고 반복 그룹을 제거합니다.

### 2. 제2정규형 (2NF): 부분 함수 종속성 제거

* **규칙:** 기본키가 복합키(여러 컬럼의 조합)일 때, 모든 일반 컬럼은 기본키의 **전체**에만 종속되어야 합니다.
* **핵심:** 기본키의 일부에만 의존하는 컬럼(부분 함수 종속)이 있다면 테이블을 분리합니다.
* **수식:**  인 상황에서  가 성립한다면, 이는 2NF 위반이므로 분리 대상입니다.

### 3. 제3정규형 (3NF): 이행적 함수 종속성 제거

* **규칙:** 기본키가 아닌 컬럼들 간에 종속 관계가 존재해서는 안 됩니다.
* **핵심:** "A가 B를 결정하고, B가 C를 결정하는" 징검다리 구조()를 끊어냅니다.
* **예시:** '게시판' 테이블에 `유저 아이디`와 `유저 별명`가 함께 있다면, 종류는 번호에 종속되므로 별도의 '별명' 테이블로 분리해야 합니다.

---

### 📊 정규화 단계 요약표

| 단계              | 핵심 키워드          | 주요 해결 내용             |
| --------------- | --------------- | -------------------- |
| **제1정규형 (1NF)** | **원자값**         | 중복 속성 제거 및 단일 값 저장   |
| **제2정규형 (2NF)** | **부분 함수 종속 제거** | 복합키의 일부에만 의존하는 속성 분리 |
| **제3정규형 (3NF)** | **이행 함수 종속 제거** | 기본키 이외의 속성 간 종속성 제거  |
| **BCNF**        | **결정자/후보키**     | 모든 결정자가 후보키가 되도록 강제  |

---

### 💡 실무 적용 전략 및 주의사항

* **과도한 정규화 경계:** 테이블을 너무 잘게 쪼개면 조회 시 다수의 **JOIN**이 발생하여 시스템 성능이 저하될 수 있습니다.
* **역정규화 (DENORMALIZATION):** 읽기 성능이 극도로 중요하거나 통계 데이터 산출이 빈번한 경우, 의도적으로 중복을 허용하기도 합니다.
* **설계 순서:** 반드시 **제3정규형까지 철저히 설계**한 후, 실제 운영 환경의 성능 테스트 결과에 따라 필요한 부분만 선별적으로 합치는 것이 정석입니다.

> **정규화의 황금률 (THE GOLDEN RULE)**
> "모든 일반 컬럼은 기본키(THE KEY)에, 기본키 전체(THE WHOLE KEY)에, 그리고 오직 기본키(NOTHING BUT THE KEY)에만 의존해야 한다."

# 📘 Oracle 사용자 계정 생성 및 권한 부여

## Summary 📝

> [!summary] Summary 📝
> 
> - Oracle에서 사용자 계정을 생성하고 권한을 부여하는 전체 절차를 정리함.
> - 접두어 없이 계정 생성하는 방법, 관리자와 사용자 계정의 역할 구분, `GRANT` 권한별 실제 사용 예시 포함.

---

## 목적 🎯

- Oracle 데이터베이스에서 새로운 사용자 계정을 생성하고 필요한 권한을 부여하는 방법을 이해함.
- 관리자 계정과 일반 사용자 계정의 역할을 구분하고, `GRANT` 구문을 통해 권한을 적절히 부여하는 실무 적용 능력 확보.

---

## 핵심 개념 💡

### ✅ 계정 생성 관련 SQL 구문
- 예시 참조
> Oracle 12c 이상에서는 기본적으로 `C##` 접두어가 필요하며, 이를 생략하려면 `_ORACLE_SCRIPT` 설정이 필요함.

---

### ✅ 계정 유형 비교

| 계정 유형                  | 역할                     | 권한 범위             |
| ---------------------- | ---------------------- | ----------------- |
| **관리자 계정 (예: SYSTEM)** | DB 생성, 관리, 전체 권한 보유    | 모든 객체 및 사용자 관리 가능 |
| **사용자 계정**             | 질의, 갱신, 보고서 작성 등 업무 수행 | 필요한 최소한의 권한만 부여   |

> 사용자 계정은 보안과 안정성을 위해 **업무에 필요한 최소 권한만 부여**하는 것이 원칙임.

---

### ✅ GRANT 권한 유형 분류

| 권한 유형        | 설명                          | 예시                                              |
| ------------ | --------------------------- | ----------------------------------------------- |
| **시스템 권한**   | 데이터베이스 수준에서 기능을 사용할 수 있는 권한 | `CREATE TABLE`, `CREATE USER`, `DROP ANY TABLE` |
| **객체 권한**    | 특정 테이블, 뷰, 시퀀스 등에 대한 접근 권한  | `SELECT`, `INSERT`, `UPDATE`, `DELETE`          |
| **역할(Role)** | 여러 권한을 묶어둔 그룹               | `CONNECT`, `RESOURCE`, `DBA`                    |

> 시스템 권한은 전체 DB에 영향을 주며, 객체 권한은 특정 객체에만 적용됨.  
> 역할(Role)은 권한을 묶어서 관리할 수 있는 편리한 방식임.

---

## 특징 ✨

### ✅ GRANT 구문 구조

```sql
GRANT 권한 [, 권한2, ...] TO 사용자 [, 사용자2, ...];
```

- 여러 권한을 동시에 부여 가능
- `WITH GRANT OPTION`을 추가하면 해당 사용자가 다른 사용자에게 권한을 다시 부여할 수 있음

---

## 팁과 주의점 ⚠️

- `_ORACLE_SCRIPT`는 **공식 문서에 명시되지 않은 내부 설정**으로, 사용 시 보안 정책에 따라 제한될 수 있음
	- `ALTER SESSION SET "_ORACLE_SCRIPT" = TRUE;`로 C## 없이 계정 생성 가능(비권장)
- `GRANT` 구문은 **객체가 존재해야** 객체 권한 부여 가능
- `WITH GRANT OPTION`은 **권한 확산 위험**이 있으므로 신중하게 사용
- `REVOKE` 구문으로 권한 회수 가능
- `CONNECT`, `RESOURCE`는 Oracle에서 기본 제공하는 역할(Role)로, 일반적인 작업에 충분함
- `UNLIMITED TABLESPACE`는 디스크 사용 제한을 없애므로 **관리자 승인 필요**

---

## 예시 📌

### ✅ 계정 생성 기본 양식

```sql
-- 사용자 계정 생성
CREATE USER CHUN IDENTIFIED BY CHUN;
-- 기본 권한 부여
GRANT CONNECT, RESOURCE TO CHUN;
-- 테이블스페이스 무제한 권한 부여
GRANT UNLIMITED TABLESPACE TO CHUN;
```

---

### ✅ 시스템 권한 부여

```sql
-- 테이블 생성 권한 부여
GRANT CREATE TABLE TO CHUN;

-- 사용자 생성 권한 부여 (관리자만 가능)
GRANT CREATE USER TO CHUN;
```

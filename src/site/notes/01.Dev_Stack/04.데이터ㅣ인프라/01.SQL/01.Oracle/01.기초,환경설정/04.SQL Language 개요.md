# 📘 SQL LANGUAGE

## Summary 📝

> [!summary] Summary 📝
> 
> - SQL은 데이터베이스를 다루는 언어로, **DDL, DML, DCL, TCL** 네 가지로 구분
> - 각 언어는 **구조 정의, 데이터 조작, 권한 제어, 트랜잭션 제어**라는 서로 다른 역할 담당
> - 일부는 서로 연결되어 동작하며, 특히 DML과 TCL은 밀접한 관계

---

## 목적 🎯

- 데이터베이스의 **구조 정의**, **데이터 조작**, **권한 관리**, **트랜잭션 제어**를 명확히 구분하여 이해
- SQL 명령어의 성격과 적용 범위를 구분

---

## 핵심 개념 💡

| 분류      | 풀네임                                       | 대표 명령어                                               | 다루는 영역         | 특징                         |
| ------- | ----------------------------------------- | ---------------------------------------------------- | -------------- | -------------------------- |
| **DDL** | Data Definition Language (데이터 정의 언어)      | `CREATE`, `ALTER`, `DROP`                            | 데이터베이스 구조(스키마) | 테이블, 뷰, 인덱스 같은 틀을 정의·변경·삭제 |
| **DML** | Data Manipulation Language (데이터 조작 언어)    | `SELECT`, `INSERT`, `UPDATE`, `DELETE`               | 테이블 안의 데이터     | 데이터를 조회·추가·수정·삭제           |
| **DCL** | Data Control Language (데이터 제어 언어)         | `GRANT`, `REVOKE`                                    | 사용자 권한         | 객체 접근 권한을 부여·회수            |
| **TCL** | Transaction Control Language (트랜잭션 제어 언어) | `COMMIT`, `ROLLBACK`, `SAVEPOINT`, `SET TRANSACTION` | 트랜잭션(작업 단위)    | 데이터 변경을 확정하거나 취소           |

---

## 특징 ✨

- **DDL**: 구조 변경 시 자동 COMMIT 발생 → TCL과 연결
- **DML**: 데이터 변경 후 TCL로 확정(COMMIT) 또는 취소(ROLLBACK) 필요
- **DCL**: 독립적이지만, 결국 DDL/DML로 만든 객체와 데이터에 대한 접근 권한 제어
- **TCL**: DML과 직접 연결, DDL은 자동 COMMIT로 인해 간접 연결

---

## 팁과 주의점 ⚠️

- `NOT NULL` 같은 제약조건은 DDL 단계에서 정의
- DML 실행 후 COMMIT/ROLLBACK을 하지 않으면 트랜잭션이 열린 상태 유지
- DDL 실행 시 자동 COMMIT 발생 → 되돌릴 수 없음

## 예시 📌

### ✅ DDL

```sql
-- 테이블 생성
CREATE TABLE MEMBER (
    ID NUMBER PRIMARY KEY,
    NAME VARCHAR2(20)
);

-- 컬럼 추가
ALTER TABLE MEMBER 
	ADD EMAIL VARCHAR2(50);

-- 테이블 삭제
DROP TABLE MEMBER;
```

### ✅ DML

```sql
-- 데이터 삽입
INSERT INTO MEMBER VALUES (1, '홍길동', 'hong@test.com');

-- 데이터 수정
UPDATE MEMBER SET NAME = '김철수' WHERE ID = 1;

-- 데이터 삭제
DELETE FROM MEMBER WHERE ID = 1;

-- 데이터 조회
SELECT * FROM MEMBER;
```

### ✅ DCL

```sql
-- SELECT 권한 부여
GRANT SELECT ON MEMBER TO USER1;

-- 권한 회수
REVOKE SELECT ON MEMBER FROM USER1;
```

### ✅ TCL

```sql
-- 데이터 삽입
INSERT INTO MEMBER VALUES (2, '이영희', 'lee@test.com');

-- 트랜잭션 확정
COMMIT;

-- 데이터 수정
UPDATE MEMBER SET NAME = '박민수' WHERE ID = 2;

-- 저장 지점 설정
SAVEPOINT sp1;

-- 롤백 (sp1 이후 작업 취소)
ROLLBACK TO sp1;
```

## 📊 흐름도

```
[DDL] ──▶ 구조 정의 (테이블, 뷰, 인덱스)
   │
   ▼
[DML] ──▶ 데이터 조작 (INSERT, UPDATE, DELETE, SELECT)
   │
   ▼
[TCL] ──▶ 트랜잭션 제어 (COMMIT, ROLLBACK)
   │
   ▼
[DCL] ──▶ 권한 제어 (GRANT, REVOKE)
```

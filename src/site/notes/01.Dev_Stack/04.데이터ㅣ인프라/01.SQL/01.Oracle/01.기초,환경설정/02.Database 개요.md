## 핵심 개념 💡

### ✅ Data / Database / 정보

|구분|정의|예시|
|---|---|---|
|**Data**|관찰 결과로 나타난 정량적·정성적 실제 값|`에베레스트 높이: 8848m`|
|**정보**|데이터를 기반으로 의미를 부여한 것|`에베레스트는 세계에서 가장 높은 산이다`|
|**Database**|조직에 필요한 정보를 여러 응용 시스템에서 공용할 수 있도록 논리적으로 연관된 데이터를 모아 중복 최소화 후 구조적으로 통합·저장한 것|기업 고객 DB|

---

### ✅ Database 정의

1. **공용 데이터 (Shared Data)**: 공동 사용 데이터
2. **통합 데이터 (Integrated Data)**: 중복 최소화, 불일치 제거
3. **저장 데이터 (Stored Data)**: 저장장치에 저장된 데이터
4. **운영 데이터 (Operational Data)**: 조직 목적에 사용되는 데이터

---

### ✅ Database 특징

- **실시간 접근성**: 요청 시 즉시 결과 제공
- **계속적 변화**: 시간에 따라 값이 변동
- **동시 공유**: 여러 사용자·업무에서 동시 사용 가능

---

### ✅ DBMS (Database Management System)

- 데이터베이스 전용 관리 프로그램
- **기능**
    - 데이터 추출 (조회)
    - 데이터 조작 (삽입, 수정, 삭제)
    - 데이터 정의 (구조 정의 및 변경)
    - 데이터 제어 (사용자 관리, 접근 제어, 백업, 회복, 동시성 제어)

---

### ✅ DBMS 종류와 특징

|DBMS|제조사|운영체제|용도|
|---|---|---|---|
|**Access**|MS|윈도우|개인용|
|**SQL Server**|MS|윈도우|기업용|
|**Oracle**|Oracle|윈도우, 유닉스, 리눅스|대용량 DB|
|**MySQL**|Oracle|윈도우, 유닉스, 리눅스|소용량 DB|
|**DB2**|IBM|유닉스|대용량 DB|
|**SQLite**|오픈소스|모바일 OS|모바일 전용 DB|

---

### ✅ DBMS 사용 이점

1. 데이터 독립화 → 응용 프로그램과 데이터 분리
2. 데이터 중복 최소화 및 무결성 보장
3. 데이터 보안 향상 → 권한 제어, 암호화 가능
4. 관리 편의성 향상 → 백업·복구 용이

---

### ✅ DBMS 변천 과정

|시대|데이터 모델|특징|
|---|---|---|
|1960년대 이전|파일시스템|단순 파일 저장|
|1970년대|계층 모델|트리 구조|
|1980년대|네트워크 모델|복잡한 관계 표현|
|1990년대|관계형 모델|2차원 테이블|
|2000년대|객체 모델|객체 지향|
|2010년대|객체-관계 모델|혼합형|

---

### ✅ Database 유형

1. **계층형**: 트리 구조, 상위 계층부터 검색
2. **네트워크형**: 하위 데이터 간 관계 정의 가능, 복잡
3. **관계형**: 2차원 테이블 구조, 중복 최소화, 적응력 우수

---

## 특징 ✨

- 데이터베이스는 **공용성, 통합성, 저장성, 운영성**을 특징으로 함
- DBMS는 **데이터 독립성, 무결성, 보안성, 관리 편의성**을 제공함
- 데이터베이스 유형은 **계층형, 네트워크형, 관계형**으로 발전

---

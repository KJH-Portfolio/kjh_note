# Oracle 데이터베이스 주요 자료형(Data Type) 정리

> [!summary]
> Oracle 데이터베이스는 데이터의 효율적인 저장과 무결성을 위해 다양한 자료형을 제공합니다. 가변 길이 문자형인 **VARCHAR2**, 고정 소수점을 지원하는 **NUMBER**, 정밀한 시간을 기록하는 **TIMESTAMP** 등이 핵심 자료형입니다.

## 1. 문자 자료형 (Character Types)

문자 데이터를 저장하며, 고정 길이와 가변 길이 여부에 따라 구분됩니다.

| 자료형 | 설명 | 최대 크기 | 특징 |
| --- | --- | --- | --- |
| **CHAR** | 고정 길이 문자 | 2,000 Byte | 지정된 길이보다 짧은 데이터 입력 시 나머지 공간을 공백으로 채움 |
| **VARCHAR2** | 가변 길이 문자 | 4,000 Byte | 입력된 실제 데이터 크기만큼만 공간 점유 (가장 많이 사용됨) |
| **NCHAR** | 고정 길이 유니코드 | 2,000 Byte | 다국어(한글, 한자 등) 저장 시 유리 |
| **NVARCHAR2** | 가변 길이 유니코드 | 4,000 Byte | 다국어 가변 길이 데이터 저장 |

## 2. 숫자 자료형 (Number Types)

숫자 자료형은 하나로 통합되어 있으며, 정수와 실수를 모두 표현합니다.

### NUMBER(p, s) 형식

* **p (Precision):** 전체 유효 자릿수 (1~38)
* **s (Scale):** 소수점 이하 자릿수 (-84~127)

> [!EXAMPLE] NUMBER 활용 예시
> * `NUMBER(5)` : 최대 5자리 정수 (예: 99999)
> * `NUMBER(5, 2)` : 전체 5자리 중 소수점 이하 2자리 (예: 123.45)
> * `NUMBER` : 자릿수 제한 없이 저장 가능
> 
> 

## 3. 날짜 및 시간 자료형 (Date & Time Types)

날짜와 시각 정보를 관리하는 자료형입니다.

| 자료형                          | 설명             | 포함 정보                     |
| ---------------------------- | -------------- | ------------------------- |
| **DATE**                     | 기본적인 날짜와 시간    | 년, 월, 일, 시, 분, 초          |
| **TIMESTAMP**                | 정밀한 날짜와 시간     | DATE 정보 + 밀리초(초 이하 단위) 포함 |
| **TIMESTAMP WITH TIME ZONE** | 지역 시간대가 포함된 시간 | 특정 지역(Timezone) 정보 포함     |

## 4. 대용량 객체 자료형 (LOB Types)

텍스트 파일, 이미지, 동영상 등 일반적인 자료형의 크기를 초과하는 대용량 데이터를 저장합니다.

* **CLOB (Character LOB):** 대용량 텍스트 데이터 저장 (최대 4GB)
* **BLOB (Binary LOB):** 이미지, 동영상, 사운드 등 바이너리 데이터 저장 (최대 4GB)
* **BFILE:** 운영체제의 물리적인 파일 형태로 저장

---

## 5. 실습 예시 (DDL)

실제 테이블 생성 시 각 자료형을 적용하는 방법입니다.

```sql
CREATE TABLE member_profile (
    user_id     VARCHAR2(20),     -- 가변 길이 ID
    user_name   NVARCHAR2(10),    -- 유니코드 이름
    age         NUMBER(3),        -- 3자리 정수 나열
    height      NUMBER(5, 2),     -- 전체 5자리 중 소수점 2자리
    bio         CLOB,             -- 대용량 자기소개
    created_at  DATE DEFAULT SYSDATE -- 가입일 (기본값 현재 시각)
);

```

---

* **수정 사항:** 단순 나열보다는 속성(크기, 특징 등)을 한눈에 비교할 수 있도록 **마크다운 표(Table)**를 적극 활용하여 재구성했습니다.
* **제안 사항:** 자료형 외에 각 컬럼에 제약 조건(NOT NULL, PRIMARY KEY 등)을 설정하는 방법에 대해서도 이어서 정리해 보시면 데이터 설계 역량 강화에 큰 도움이 될 것입니다.

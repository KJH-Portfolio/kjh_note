# 📘 Oracle SQL SEQUENCE

## Summary 📝

> [!summary] Summary 📝
> 
> - **SEQUENCE**: 중복 없이 순차적인 숫자 생성 객체
> - **핵심 구문**: NEXTVAL, CURRVAL, CREATE/ALTER/DROP SEQUENCE
> - **옵션**: START WITH, INCREMENT BY, MAX/MINVALUE, CYCLE, CACHE
> - **사용 목적**: 기본키 자동 채번, 트랜잭션 안전한 일련번호 부여

---

## 목적 🎯

- **기본키 자동 생성 목적**
- **중복 방지 목적**
- **고성능 채번 목적**(CACHE 활용)
- **트랜잭션 일관성 유지 목적**

---

## 핵심 개념 💡

### ✅ 정의와 동작

- **정의:** 데이터베이스가 관리하는 숫자 발생 객체
- **동작:** `NEXTVAL`로 증가값만큼 이동 후 반환, `CURRVAL`로 현재 세션의 마지막 `NEXTVAL` 값 반환
- **세션 특성:** `CURRVAL`은 세션 내 최초 `NEXTVAL` 호출 이후에만 사용 가능
- $ 최신 문법으로 GENERATED AS IDENTITY 방식도 존재
  간결하지만 수동 제어가 힘든 최신 문법

### ✅ 기초 문법

```sql
-- 생성: 기본 형태
CREATE SEQUENCE 시퀀스명
  START WITH 1          -- 시작값
  INCREMENT BY 1        -- 증가 단위
  MAXVALUE 99999999999   -- 최대값
  MINVALUE 1            -- 최소값
  CYCLE                 -- 최대/최소 도달 시 순환 허용
  CACHE 20;             -- 캐시 크기 (성능 최적화)

-- 사용: 번호 발생
SELECT 시퀀스명.NEXTVAL FROM DUAL;  -- 증가 후 값 반환
SELECT 시퀀스명.CURRVAL FROM DUAL;  -- 현재 세션의 마지막 값

-- 변경: 일부 속성만 변경 가능
ALTER SEQUENCE 시퀀스명
  INCREMENT BY 10
  MAXVALUE 1000
  NOCACHE;

-- 삭제
DROP SEQUENCE 시퀀스명;
```

### ✅ 옵션 비교표

| 옵션                    | 의미    | 주요 효과                    |
| --------------------- | ----- | ------------------------ |
| **START WITH**        | 시작값   | 최초 `NEXTVAL` 반환값 지정      |
| **INCREMENT BY**      | 증감 단위 | 양수 증감, 음수 설정 시 감소 시퀀스    |
| **MAXVALUE/MINVALUE** | 상·하한  | 범위 통제, 초과/미만 시 오류 또는 순환  |
| **CYCLE/NOCYCLE**     | 순환 여부 | 경계 도달 시 시작점으로 순환 여부      |
| **CACHE/NOCACHE**     | 캐시 사용 | 성능 최적화(캐시), 안정성(NOCACHE) |

---

## 특징 ✨

### ✅ 동작 및 트랜잭션 특성

- **원자성 보장:** `NEXTVAL` 호출은 다른 세션과 충돌 없이 고유 값 보장
- **롤백 비반영:** `NEXTVAL`로 소비된 값은 트랜잭션 롤백 시 되돌리지 않음
- **세션 독립성:** 각 세션의 `CURRVAL`은 자체 세션의 `NEXTVAL` 이력에만 의존

### ✅ 성능·운영 특성

- **캐시 사용 시 성능 향상:** 다빈도 채번 환경에서 `CACHE`로 래치 경합 감소
- **NOCACHE 시 안정성:** 예기치 않은 인스턴스 다운 시 번호 건너뜀 최소화
- **감소 시퀀스 지원:** `INCREMENT BY` 음수 설정으로 역순 번호 발생 가능

### ✅ 제한 및 주의

- **START WITH 변경 불가:** `ALTER`로 수정 불가, **재생성 필요**
- **ORDER 보장 부재:** 멀티 세션에서 발생 순서의 전역적 정렬 보장 불가
- **범위 초과 시 처리:** `NOCYCLE`은 오류, `CYCLE`은 경계 넘어 순환

---

## 팁과 주의점 ⚠️

- PK 자동 채번 관례: `INSERT` 시 `시퀀스.NEXTVAL` 직접 사용 권장
- CURRVAL 사용 조건: 최초 `NEXTVAL` 호출 이후 세션 내에서만 사용 가능
- MAX/MIN 설정 실무값: 실무에서는 충분히 큰 `MAXVALUE`로 설정하여 초과 방지
- CACHE 선택 기준: 고빈도 삽입 테이블은 `CACHE` 권장, 금융·정산 등 번호 연속성이 중요한 경우 `NOCACHE` 고려
- 감소 시퀀스 경계 관리: 음수 증가 시 `MINVALUE`와 `NOCYCLE` 조합 시 오류 가능성 주의
- **시퀀스의 함수화** : 외부에서 NEXTVAL에 접근할때 용이할 수 있음

## 예시 📌

### ✅ 기본키 자동 채번 흐름

```sql
-- 시퀀스 생성
CREATE SEQUENCE SEQ_MEMBER_ID
  START WITH 1000
  INCREMENT BY 1
  NOCYCLE
  CACHE 50;

-- 테이블 생성
CREATE TABLE MEMBER (
  ID NUMBER PRIMARY KEY,
  NAME VARCHAR2(50) NOT NULL
);

-- 데이터 삽입: 시퀀스 사용
INSERT INTO MEMBER (ID, NAME)
VALUES (SEQ_MEMBER_ID.NEXTVAL, '홍길동');

-- 확인: 현재 세션의 마지막 값
SELECT SEQ_MEMBER_ID.CURRVAL FROM DUAL;  -- 마지막 발생 값 확인
```

### ✅ 감소 시퀀스 설정 및 사용

```sql
-- 감소 시퀀스
CREATE SEQUENCE SEQ_DESC
  START WITH 10
  INCREMENT BY -2
  MINVALUE 0
  NOCYCLE
  NOCACHE;

-- 사용 흐름
SELECT SEQ_DESC.NEXTVAL FROM DUAL; -- 10
SELECT SEQ_DESC.NEXTVAL FROM DUAL; -- 8
SELECT SEQ_DESC.NEXTVAL FROM DUAL; -- 6
```

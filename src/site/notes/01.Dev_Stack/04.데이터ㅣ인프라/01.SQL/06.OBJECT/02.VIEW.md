# 📘 Oracle SQL VIEW 

## Summary 📝

> [!summary] Summary 📝
> 
> - **VIEW**: SELECT 구문을 저장하는 가상 테이블 객체
> - **목적**: 복잡한 조회 구문 단순화, 재사용성 확보, 보안 강화
> - **특징**: 실제 데이터 미보유, 실행 시 SELECT 구문 수행
> - **옵션**: OR REPLACE, FORCE, WITH READ ONLY
> - **DML**: 특정 조건 충족 시 원본 테이블에 반영, 대부분은 조회 전용 사용 권장

---

## 목적 🎯

- 복잡한 **JOIN/조건문**을 캡슐화하여 재사용
- **보안**: 민감 컬럼 제외, 필요한 컬럼만 노출
- **스키마 변경 최소화**: 내부 구조 변경 시에도 동일 인터페이스 제공

---

## 핵심 개념 💡

### ✅ VIEW 생성 방식

|구문|설명|
|---|---|
|`CREATE VIEW 뷰명 AS ...`|신규 생성|
|`CREATE OR REPLACE VIEW 뷰명 AS ...`|기존 뷰 갱신 또는 신규 생성|
|`CREATE FORCE VIEW 뷰명 AS ...`|참조 객체 없어도 강제 생성|

### ✅ 별칭

- 함수·연산식 포함 컬럼은 반드시 별칭 지정
- 뷰 정의 시 `(컬럼1, 컬럼2, …)` 형태로 일괄 별칭 지정 가능

### ✅ DML 가능 여부

- **가능**: 단일 테이블 기반, 모든 NOT NULL 컬럼 포함, 함수·집계·DISTINCT 미포함
- **불가**: JOIN, GROUP BY, DISTINCT, 함수/연산식 포함, 정의되지 않은 NOT NULL 컬럼 존재
- **차단**: `WITH READ ONLY` 옵션

---

## 특징 ✨

- **실제 데이터 없음**: SELECT 구문만 저장
- **실행 시점 평가**: 뷰 호출 시 SELECT 실행
- **보안성**: 특정 컬럼만 노출 가능
- **재사용성**: 자주 쓰는 쿼리 표준화

---

## 팁과 주의점 ⚠️

- **ORDER BY**는 뷰 내부에서 보장되지 않음 → 최종 SELECT에서 정렬
- **별칭 필수**: 함수·연산식 컬럼은 반드시 지정
- **DML 지양**: 원본 테이블에 직접 반영되므로 주의
- **USER_VIEWS** 활용: 뷰 정의 확인 가능

## 예시 📌

### ✅ VIEW 생성 및 활용

```sql
-- 권한 부여 (시스템 계정)
GRANT CREATE VIEW TO KH;

-- VIEW 생성 (KH 계정)
CREATE VIEW VW_EMPLOYEE AS
SELECT
    E.EMP_ID,
    E.EMP_NAME,
    D.DEPT_TITLE,
    E.SALARY,
    N.NATIONAL_NAME,
    J.JOB_NAME
FROM EMPLOYEE E
LEFT JOIN DEPARTMENT D ON E.DEPT_CODE = D.DEPT_ID
LEFT JOIN JOB J USING (JOB_CODE)
LEFT JOIN LOCATION L ON L.LOCATION_ID = E.LOCAL_CODE
LEFT JOIN NATIONAL N USING (NATIONAL_CODE);

-- 조회
SELECT * FROM VW_EMPLOYEE WHERE NATIONAL_NAME = '한국';
```

### ✅ VIEW 갱신 (BONUS 컬럼 추가)

```sql
CREATE OR REPLACE VIEW VW_EMPLOYEE AS
SELECT
    E.EMP_ID,
    E.EMP_NAME,
    D.DEPT_TITLE,
    E.SALARY,
    E.BONUS,           -- 추가
    N.NATIONAL_NAME,
    J.JOB_NAME
FROM EMPLOYEE E
LEFT JOIN DEPARTMENT D ON E.DEPT_CODE = D.DEPT_ID
LEFT JOIN JOB J USING (JOB_CODE)
LEFT JOIN LOCATION L ON L.LOCATION_ID = E.LOCAL_CODE
LEFT JOIN NATIONAL N USING (NATIONAL_CODE);

-- 러시아 근무자 보너스 조회
SELECT EMP_ID, EMP_NAME, JOB_NAME, BONUS
FROM VW_EMPLOYEE
WHERE NATIONAL_NAME = '러시아';
```

### ✅ 함수 포함 시 별칭 필수

```sql
CREATE OR REPLACE VIEW VW_EMP_JOB AS
SELECT
    E.EMP_ID,
    E.EMP_NAME,
    J.JOB_NAME,
    DECODE(SUBSTR(E.EMP_NO, 8, 1), '1', '남', '2', '여') AS 성별,
    EXTRACT(YEAR FROM SYSDATE) - EXTRACT(YEAR FROM E.HIRE_DATE) AS 근무기간
FROM EMPLOYEE E
JOIN JOB J USING (JOB_CODE);

SELECT * FROM VW_EMP_JOB;
```

### ✅ VIEW에 대한 DML

```sql
-- 단순 뷰 생성
CREATE VIEW VW_JOB AS
SELECT * FROM JOB;

-- INSERT → 원본 JOB 반영
INSERT INTO VW_JOB VALUES ('J8', '인턴');

-- UPDATE → 원본 JOB 반영
UPDATE VW_JOB
SET JOB_NAME = '알바'
WHERE JOB_CODE = 'J8';

-- DELETE → 원본 JOB 반영
DELETE FROM VW_JOB
WHERE JOB_CODE = 'J8';
```

### ✅ READ ONLY 옵션

```sql
CREATE OR REPLACE VIEW VW_READONLY AS
SELECT *
FROM EMPLOYEE
WITH READ ONLY;

-- DML 시도 → 오류 발생
DELETE FROM VW_READONLY; -- 실행 불가
```

# 📘 오라클 TCL (COMMIT, ROLLBACK, SAVEPOINT)

## Summary 📝

> [!summary] Summary 📝
> 
> - **TCL(Transaction Control Language)** 은 트랜잭션 단위로 데이터 변경을 제어하는 언어
> - 주요 명령어: **COMMIT, ROLLBACK, SAVEPOINT, ROLLBACK TO**
> - DML(INSERT, UPDATE, DELETE) 실행 시 트랜잭션이 발생하며, COMMIT 전까지는 확정되지 않음
> - DDL(CREATE, ALTER, DROP)은 실행 시 자동 COMMIT 발생
> - 트랜잭션은 **ACID 특성(원자성, 일관성, 고립성, 지속성)**을 보장

---

## 목적 🎯

- 데이터 변경을 **트랜잭션 단위**로 묶어 안정적으로 관리
- 변경 사항을 **확정(COMMIT)** 하거나 **취소(*ROLBACK*)** 하여 데이터 무결성 보장
- 특정 시점으로 되돌아갈 수 있는 **SAVEPOINT** 기능 제공
- 다중 사용자 환경에서 **세션 단위 격리**를 통해 데이터 일관성 유지

---

## 핵심 개념 💡

|명령어|설명|특징|
|---|---|---|
|**COMMIT**|현재 트랜잭션의 변경 사항을 DB에 영구 반영|확정 후 트랜잭션 비워짐|
|**ROLLBACK**|현재 트랜잭션의 변경 사항을 취소하고 마지막 COMMIT 시점으로 되돌림|미확정 변경 사항 삭제|
|**SAVEPOINT**|트랜잭션 내 특정 시점을 임시 저장|부분 취소 지점 설정 가능|
|**ROLLBACK TO**|지정한 SAVEPOINT까지만 되돌림|전체 취소가 아닌 부분 취소|
|**DDL 자동 COMMIT**|CREATE, ALTER, DROP 실행 시 자동 COMMIT 발생|이전 DML 변경 사항도 강제 확정|

---

## 특징 ✨

- **DML 구문**(INSERT, UPDATE, DELETE)만 트랜잭션 대상
- **DDL 구문** 실행 시 자동 COMMIT 발생 → 이전 변경 사항도 확정됨
- **세션 단위 격리**: COMMIT 전 변경 사항은 현재 세션에서만 보임, 다른 세션에서는 확인 불가
- **SAVEPOINT**는 COMMIT 시 모두 소멸

---

## 팁과 주의점 ⚠️

- DDL 실행 전 DML 작업이 있었다면 반드시 **COMMIT 또는 ROLLBACK**으로 정리 후 진행
- COMMIT 이후에는 ROLLBACK 불가 (이미 확정됨)
- SAVEPOINT는 **트랜잭션 내에서만 유효**
- 실습 시 **세션 A vs 세션 B**를 나눠서 COMMIT 전후 가시성 차이를 확인하면 이해가 쉬움

---

## 예시 📌

### ✅ COMMIT과 ROLLBACK

```sql
-- EMP_01 테이블 생성
CREATE TABLE EMP_01 AS
SELECT EMP_ID, EMP_NAME, DEPT_TITLE
FROM EMPLOYEE
LEFT JOIN DEPARTMENT ON (DEPT_CODE = DEPT_ID);

-- 데이터 변경 (아직 COMMIT 안 함)
DELETE FROM EMP_01 WHERE EMP_ID = '901';
UPDATE EMP_01 SET DEPT_TITLE = '인사부' WHERE EMP_ID = '214';
INSERT INTO EMP_01 VALUES('999','김구구','운영관리부');

-- 변경 취소
ROLLBACK;  -- 마지막 COMMIT 시점으로 복구
```

### ✅ COMMIT 이후 ROLLBACK

```sql
UPDATE EMP_01 SET DEPT_TITLE = '인사부' WHERE EMP_ID = '214';
INSERT INTO EMP_01 VALUES('999','김구구','운영관리부');
COMMIT;  -- 확정

ROLLBACK; -- 이미 확정된 데이터는 되돌릴 수 없음
```

### ✅ SAVEPOINT와 ROLLBACK TO

```sql
-- 현재 행 수: 25
DELETE FROM EMP_01 WHERE EMP_ID IN (900,901,999); -- 23행
SAVEPOINT SP1;  -- 저장 지점 설정

DELETE FROM EMP_01 WHERE EMP_ID > 300; -- 1행

ROLLBACK;        -- 전체 취소 → 다시 25행
ROLLBACK TO SP1; -- SP1 시점으로 복구 → 23행
```

### ✅ 세션 간 가시성 차이

```sql
-- 세션 A
INSERT INTO EMP_01 VALUES('1000','세션A','개발부');
-- 아직 COMMIT 안 함

-- 세션 B
SELECT * FROM EMP_01 WHERE EMP_ID = '1000';
-- 결과 없음 (세션 A에서 COMMIT 전이므로 보이지 않음)

-- 세션 A
COMMIT;

-- 세션 B
SELECT * FROM EMP_01 WHERE EMP_ID = '1000';
-- 이제 조회 가능
```

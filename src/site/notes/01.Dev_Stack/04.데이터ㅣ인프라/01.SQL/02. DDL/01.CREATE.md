# 📘 DDL - CREATE (테이블 생성)

## Summary 📝

> [!summary] 핵심 요약
>
>   * **CREATE TABLE**: 데이터 저장소인 테이블의 구조(컬럼, 자료형, 제약조건) 정의
>   * **핵심 요소**:
>       * **자료형**: 문자(CHAR, VARCHAR2), 숫자(NUMBER), 날짜(DATE)
>       * **제약조건**: PK, FK, UK, NOT NULL, CHECK (데이터 무결성 보장)
>       * **DEFAULT**: 데이터 입력 생략 시 자동 입력값
>   * **CTAS**: `SELECT` 문을 활용하여 기존 테이블 복제 생성 (제약조건 미복사 주의)
>   * **COMMENT**: 컬럼 및 테이블에 대한 설명(주석) 등록

## 1\. 기본 생성 문법 🛠️

```sql
CREATE TABLE 테이블명 (
    컬럼명 자료형 [DEFAULT 기본값] [제약조건],
    컬럼명 자료형... ,
    [CONSTRAINT 제약조건명 제약조건타입 (대상컬럼)]
);
```

### 1.1 자료형 (Data Types)

| 자료형 | 설명 | 특징 | 예시 |
| :--- | :--- | :--- | :--- |
| **CHAR(n)** | 고정 길이 문자 | 지정된 길이만큼 공간 차지 (남는 공간 공백) | `CHAR(10)` |
| **VARCHAR2(n)** | 가변 길이 문자 | 실제 입력된 데이터 길이만큼만 공간 차지 | `VARCHAR2(50)` |
| **NUMBER(p, s)** | 숫자 | p: 전체 자리수, s: 소수점 이하 자리수 | `NUMBER(10,2)` |
| **DATE** | 날짜 | 연, 월, 일, 시, 분, 초 저장 | `DATE` |

### 1.2 제약조건 (Constraints)

데이터의 정확성과 신뢰성(무결성)을 지키기 위한 규칙입니다.

| 제약조건 | 키워드 | 설명 | 특징 |
| :--- | :--- | :--- | :--- |
| **Primary Key** | `PRIMARY KEY` | 행을 식별하는 고유 키 | **NOT NULL + UNIQUE**, 테이블당 1개 |
| **Foreign Key** | `FOREIGN KEY` | 다른 테이블 참조 | 참조 무결성 보장, 부모 키 값만 허용 |
| **Unique** | `UNIQUE` | 중복 값 금지 | NULL 허용 (여러 행 가능) |
| **Not Null** | `NOT NULL` | 필수 입력 항목 | 컬럼 레벨에서만 정의 가능 |
| **Check** | `CHECK` | 값의 범위/조건 제한 | 조건식(`WHERE`절 유사) 사용 |

> [!tip] FOREIGN KEY 삭제 옵션
> 부모 테이블의 데이터가 삭제될 때 자식 데이터를 어떻게 처리할지 정의합니다.
>
>   * `ON DELETE CASCADE`: 부모 데이터 삭제 시 자식 데이터도 **함께 삭제**
>   * `ON DELETE SET NULL`: 부모 데이터 삭제 시 자식 데이터의 FK 값을 **NULL로 변경**
>   * (생략 시): 자식 데이터가 있으면 부모 데이터 **삭제 불가** (기본값)

### 1.3 주석 (COMMENT)

```sql
COMMENT ON TABLE 테이블명 IS '테이블 설명';
COMMENT ON COLUMN 테이블명.컬럼명 IS '컬럼 설명';
```

## 2\. CTAS (Create Table As Select) 📋

쿼리(SELECT) 결과를 기반으로 테이블을 생성합니다. (데이터 백업, 테스트용)

  * **문법**: `CREATE TABLE 새테이블명 AS SELECT ...`
  * **특징**:
      * 데이터와 컬럼 구조는 복사됨.
      * **제약조건은 `NOT NULL`만 복사됨** (PK, FK, DEFAULT 등은 복사 안 됨).
      * 함수/수식 사용 시 반드시 **별칭(Alias)** 필요.
      * `WHERE 1=0` 조건 사용 시 데이터 없이 **구조만 복사**.

## 3\. 종합 예시 (Example) 📌

### ✅ 시나리오: 부서 및 사원 관리 시스템

**1. 부서(DEPT) 테이블 생성** (부모 테이블)

```sql
CREATE TABLE DEPT (
    DEPT_ID     NUMBER          CONSTRAINT PK_DEPT PRIMARY KEY, -- 기본키 설정
    DEPT_NAME   VARCHAR2(30)    NOT NULL,                       -- 필수 입력
    LOC         VARCHAR2(30)    DEFAULT 'SEOUL'                 -- 기본값 설정
);

COMMENT ON TABLE DEPT IS '부서 정보 테이블';
```

**2. 사원(EMP) 테이블 생성** (자식 테이블)

```sql
CREATE TABLE EMP (
    EMP_ID      NUMBER          CONSTRAINT PK_EMP PRIMARY KEY,
    EMP_NAME    VARCHAR2(30)    NOT NULL,
    EMAIL       VARCHAR2(50),
    GENDER      CHAR(1)         CONSTRAINT CK_GENDER CHECK (GENDER IN ('M', 'F')), -- 값 제한
    SALARY      NUMBER          DEFAULT 0,
    DEPT_ID     NUMBER,         -- 외래키로 사용할 컬럼
    
    -- 테이블 레벨 제약조건 정의
    CONSTRAINT UK_EMAIL UNIQUE (EMAIL), -- 이메일 중복 불가
    CONSTRAINT FK_EMP_DEPT FOREIGN KEY (DEPT_ID) 
        REFERENCES DEPT(DEPT_ID) 
        ON DELETE SET NULL      -- 부서 삭제 시 사원의 부서정보를 NULL로 변경
);
```

**3. CTAS 활용 (보너스 포함 연봉 테이블 생성)**

```sql
-- 사원 테이블에서 이름과 계산된 연봉을 가져와 새 테이블 생성
CREATE TABLE EMP_SALARY_INFO AS
SELECT 
    EMP_NAME,
    SALARY * 12 + NVL(BONUS, 0) AS ANNUAL_SALARY -- 산술식은 별칭(Alias) 필수
FROM EMP
WHERE DEPT_ID = 10;
```

-----

**수정 사항:**

  * `ALTER`, `DROP` 내용을 제외하고 `CREATE`에만 집중하여 재구성했습니다.
  * **자료형**과 **제약조건**을 표로 정리하여 가독성을 높였습니다.

**제안 사항:**

  * 테이블 생성 후에는 반드시 `DESC 테이블명;` (SQL Plus) 또는 DB 툴의 정보를 통해 구조가 의도대로 생성되었는지 확인하는 습관을 들이는 것이 좋습니다.

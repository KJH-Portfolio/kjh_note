# 📘 DDL - 부록 (데이터 딕셔너리)

## Summary 📝

> [!summary] 핵심 요약
>
>   * **데이터 딕셔너리**: 데이터베이스의 자원을 관리하기 위한 **시스템 테이블(메타데이터)**.
>   * **USER\_...**: 현재 접속한 사용자가 소유한 객체 정보 조회.
>   * **주요 뷰(View)**:
>       * `USER_TABLES`: 테이블 정보 (테이블명, 생성일 등)
>       * `USER_TAB_COLUMNS`: 컬럼 정보 (컬럼명, 자료형, NULL 여부)
>       * `USER_CONSTRAINTS`: 제약조건 정보 (제약조건명, 타입, 상태)
>       * `USER_CONS_COLUMNS`: 제약조건이 걸린 컬럼 매핑 정보

-----

## 1\. 주요 딕셔너리 뷰 🔍

사용자가 생성한 DDL의 결과를 확인할 때 사용합니다.

| 뷰 이름 (View Name) | 설명 | 주요 확인 컬럼 |
| :--- | :--- | :--- |
| **USER\_TABLES** | 본인 소유의 **테이블** 목록 | `TABLE_NAME`, `TABLESPACE_NAME` |
| **USER\_TAB\_COLUMNS** | 테이블별 **컬럼** 상세 정보 | `COLUMN_NAME`, `DATA_TYPE`, `DATA_LENGTH`, `NULLABLE` |
| **USER\_CONSTRAINTS** | **제약조건**의 정의 확인 | `CONSTRAINT_NAME`, `CONSTRAINT_TYPE`, `STATUS` |
| **USER\_CONS\_COLUMNS** | 제약조건이 할당된 **컬럼** 확인 | `COLUMN_NAME`, `POSITION` |

### ✅ 제약조건 타입 코드 (CONSTRAINT\_TYPE)

`USER_CONSTRAINTS` 조회 시 나오는 약어의 의미입니다.

  * **P**: Primary Key
  * **R**: Foreign Key (Referential)
  * **U**: Unique Key
  * **C**: Check, Not Null
      * *(참고: Not Null은 별도 타입 없이 Check 조건으로 관리됨)*

-----

## 2\. 조회 예시 (Example) 📌

### ✅ 테이블 및 컬럼 확인

```sql
-- 1. 내가 가진 모든 테이블 목록 조회
SELECT TABLE_NAME, STATUS 
FROM USER_TABLES;

-- 2. 특정 테이블(EMP)의 컬럼 정보 상세 조회
SELECT COLUMN_NAME, DATA_TYPE, DATA_LENGTH, NULLABLE
FROM USER_TAB_COLUMNS
WHERE TABLE_NAME = 'EMP'; -- 테이블명은 반드시 대문자로 검색
```

### ✅ 제약조건 확인 (가장 중요)

제약조건이 제대로 걸렸는지, FK가 어느 테이블을 참조하는지 확인할 때 유용합니다.

```sql
SELECT 
    A.CONSTRAINT_NAME,      -- 제약조건 이름
    A.CONSTRAINT_TYPE,      -- 타입 (P, R, U, C)
    A.TABLE_NAME,           -- 테이블명
    B.COLUMN_NAME,          -- 적용된 컬럼명
    A.SEARCH_CONDITION      -- CHECK 조건 내용 (NOT NULL 포함)
FROM USER_CONSTRAINTS A
JOIN USER_CONS_COLUMNS B
  ON A.CONSTRAINT_NAME = B.CONSTRAINT_NAME
WHERE A.TABLE_NAME = 'EMP'; -- 테이블명 대문자
```

## 3\. 주석(Comment) 확인 💬

DDL 작성 시 입력한 주석(`COMMENT ON...`)을 확인합니다.

| 뷰 이름 | 설명 |
| :--- | :--- |
| **USER\_TAB\_COMMENTS** | 테이블에 달린 주석 확인 |
| **USER\_COL\_COMMENTS** | 컬럼에 달린 주석 확인 |

```sql
-- EMP 테이블의 컬럼별 설명(주석) 조회
SELECT COLUMN_NAME, COMMENTS
FROM USER_COL_COMMENTS
WHERE TABLE_NAME = 'EMP';
```

**피드백:**

  * 놓쳤던 **데이터 딕셔너리** 부분을 별도 노트로 분리하여 완성했습니다.
  * DDL 작업(`CREATE`, `ALTER`) 후에는 반드시 이 딕셔너리 쿼리들을 통해 **내가 의도한 대로 구조가 잡혔는지 검증**하는 습관을 들이시는 것이 좋습니다.

이제 보내주신 원본 텍스트의 모든 기술적 요소(DDL 3대장 + 메타데이터 조회)가 정리되었습니다. 추가로 궁금한 점이 있으신가요?

컬럼 및 제약조건의 추가·수정·삭제 문법을 **표**로 정리하여 가독성을 높였습니다.

# 📘 DDL - ALTER (테이블 수정)

## Summary 📝

> [!summary] 핵심 요약
>
>   * **ALTER TABLE**: 기존 테이블의 구조(컬럼, 제약조건, 이름)를 변경
>   * **주요 기능**:
>       * **컬럼(Column)**: 추가(ADD), 수정(MODIFY), 삭제(DROP), 이름 변경(RENAME)
>       * **제약조건(Constraint)**: 추가(ADD), 삭제(DROP), 비활성화(DISABLE)
>       * **테이블(Table)**: 테이블 이름 변경(RENAME TO)
>   * **주의사항**: 컬럼 타입을 변경하거나 크기를 줄일 때, 이미 데이터가 존재하면 실패할 수 있음.

-----

## 1\. 컬럼(Column) 관리 🏗️

테이블의 속성(컬럼)을 변경하는 작업입니다.

### 1.1 컬럼 작업 문법

| 작업 유형 | 설명 | 문법 (Syntax) | 비고 |
| :--- | :--- | :--- | :--- |
| **추가** | 새로운 컬럼 추가 | `ADD (컬럼명 자료형 [DEFAULT 값])` | 마지막 컬럼 뒤에 추가됨 |
| **수정** | 자료형, 크기, 기본값 변경 | `MODIFY (컬럼명 자료형 [DEFAULT 값])` | 데이터 있을 시 타입 변경 주의 |
| **삭제** | 컬럼 및 데이터 삭제 | `DROP COLUMN 컬럼명` | **복구 불가능**, 시스템 부하 큼 |
| **이름 변경** | 컬럼명 변경 | `RENAME COLUMN 기존명 TO 새이름` | |

> [\!warning] 컬럼 수정(MODIFY) 시 주의사항
>
>   * **크기 축소**: 해당 컬럼에 저장된 데이터 중 변경하려는 크기보다 큰 값이 있으면 **오류 발생**.
>   * **타입 변경**: 데이터가 **모두 NULL**일 때만 자유롭게 변경 가능 (예: CHAR → NUMBER). 데이터가 있다면 호환되는 타입(VARCHAR → CHAR)만 가능.

-----

## 2\. 제약조건(Constraint) 관리 🛡️

데이터 무결성 규칙을 추가하거나 제거합니다.

### 2.1 제약조건 추가 (ADD)

기존 테이블에 제약조건을 새로 부여합니다.

| 제약조건 유형 | 문법 | 예시 |
| :--- | :--- | :--- |
| **PK / FK / UK / CHECK** | `ADD CONSTRAINT 제약명 타입 (컬럼)` | `ADD CONSTRAINT PK_EMP PRIMARY KEY (EMP_ID)` |
| **NOT NULL** | **`MODIFY`** 구문 사용 | `MODIFY (컬럼명 CONSTRAINT 제약명 NOT NULL)` |

> [\!tip] NOT NULL의 특이성
> `NOT NULL`은 컬럼의 **속성**을 변경하는 개념이 강하므로 `ADD CONSTRAINT`가 아닌 **`MODIFY`** 명령어를 사용해야 합니다.

### 2.2 제약조건 삭제 (DROP)

| 작업 | 문법 | 옵션 |
| :--- | :--- | :--- |
| **일반 삭제** | `DROP CONSTRAINT 제약명` | |
| **PK 삭제** | `DROP PRIMARY KEY` | 제약명 몰라도 삭제 가능 |
| **CASCADE** | `DROP ... CASCADE` | 해당 키를 참조하는 FK까지 함께 삭제 |

-----

## 3\. 테이블 관리 및 기타 📦

### 3.1 이름 변경 (RENAME)

  * **테이블 이름 변경**: `RENAME 기존테이블명 TO 새테이블명;`
  * 또는 `ALTER TABLE 기존테이블명 RENAME TO 새테이블명;`

### 3.2 제약조건 비활성화 (ENABLE / DISABLE)

제약조건을 삭제하지 않고 일시적으로 멈출 때 사용합니다. (대량 데이터 입력 시 유용)

  * **비활성화**: `ALTER TABLE 테이블명 DISABLE CONSTRAINT 제약명;`
  * **활성화**: `ALTER TABLE 테이블명 ENABLE CONSTRAINT 제약명;`

-----

## 4\. 예시 (Example) 📌

### ✅ 시나리오: 프로젝트 진행에 따른 테이블 구조 변경

**1. 초기 상태: 사원 테이블**

```sql
CREATE TABLE EMP_PROJ (
    EMP_ID NUMBER,
    NAME VARCHAR2(10)
);
```

**2. 컬럼 추가 및 수정**

```sql
-- 1. 이메일 컬럼 추가 (기본값 설정)
ALTER TABLE EMP_PROJ ADD (EMAIL VARCHAR2(50) DEFAULT 'NO_EMAIL');

-- 2. 이름 컬럼 크기 증가 (10 -> 20)
ALTER TABLE EMP_PROJ MODIFY (NAME VARCHAR2(20));

-- 3. 잘못 만든 컬럼 삭제
ALTER TABLE EMP_PROJ DROP COLUMN EMAIL;
```

**3. 제약조건 추가 및 삭제**

```sql
-- 1. 기본키(PK) 추가
ALTER TABLE EMP_PROJ ADD CONSTRAINT PK_EMP_PROJ PRIMARY KEY (EMP_ID);

-- 2. 필수 입력(NOT NULL) 조건 추가 (MODIFY 사용!)
ALTER TABLE EMP_PROJ MODIFY (NAME CONSTRAINT NN_EMP_NAME NOT NULL);

-- 3. 제약조건 삭제 (제약조건 이름을 알 때)
ALTER TABLE EMP_PROJ DROP CONSTRAINT NN_EMP_NAME;
```

**4. 테이블 이름 변경**

```sql
ALTER TABLE EMP_PROJ RENAME TO EMPLOYEE_PROJECT_V2;
```

-----

**수정 사항:**

  * `ALTER` 명령어의 핵심인 **컬럼**과 **제약조건** 관리를 명확히 구분했습니다.
  * 초보자가 가장 많이 실수하는 `NOT NULL` 추가 문법(`MODIFY` 사용)을 별도 팁으로 강조했습니다.

**제안 사항:**

  * 실무에서는 컬럼 삭제(`DROP COLUMN`)가 시스템 부하를 많이 주므로, 사용량이 적은 시간대에 수행하거나 `SET UNUSED` 옵션(논리적 삭제)을 먼저 사용하는 방법을 별도로 학습하면 좋습니다.

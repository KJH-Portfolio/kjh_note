# 📘 오라클 DCL

## Summary 📝

> [!summary] Summary 📝
> 
> - 오라클에서 시스템 권한과 객체 권한의 부여·회수 절차 정리
> - 다른 스키마 객체 접근, 트랜잭션 반영 흐름 설명
> - ROLE 활용 및 권한 조회 구문 제시
> - 권한 부족 오류 원인과 해결책 명시

---

## 목적 🎯

- 데이터베이스에서 사용자 객체에 대한 접근 권한 제어
- 보안과 무결성을 유지

---

## 핵심 개념 💡

### ✅ 권한 구분 및 대상

- **시스템 권한:** 계정 수준 작업 허용 권한 집합
    - **CREATE TABLE,....
    - **UNLIMITED TABLESPACE** 등
- **객체 권한:** 특정 객체에 대한 조작 허용 권한 집합
    - **SELECT**, **INSERT**, **UPDATE**, **DELETE** (TABLE/VIEW),
    - **SELECT**(SEQUENCE) - 단순한 객체라서
- **ROLE:** 권한 묶음 관리 단위
    - **CONNECT**(접속 권한)
    - **RESOURCE**(개발 권한 묶음)

#### 권한 비교표

| 구분         | 적용 대상     | 비고          |
| ---------- | --------- | ----------- |
| **시스템 권한** | 계정        | 작업 가능 범위 설정 |
| **객체 권한**  | 테이블/뷰/시퀀스 | 객체 단위 권한 부여 |
| **ROLE**   | 계정        | 권한 묶음 부여    |

### ✅ 기본 문법
```SQL
GRANT SELECT ON 계정명.테이블 TO 사용자명
REVOKE SELECT ON 계정명.테이블명 TO 사용자명

GRANT CONNECT TO 사용자명 --사용자에 역할 부여. 테이블 단위로 할 수 없음
```

## 특징 ✨

### ✅ 트랜잭션 반영 특성

- **DML 트랜잭션:** COMMIT 이전 변경 사항은 **현재 세션에서만 보임**
- **격리 수준:** 기본 READ COMMITTED 사용으로 타 세션에서는 커밋된 데이터만 노출
- **DDL 자동 커밋:** DDL 실행 시 암묵적 COMMIT 발생

### ✅ 객체 권한 독립성

- **INSERT와 SELECT등.. 권한들은 서로 독립** → 하나만 줘서는 다른 기능이 따라오지 않는다.
- **스키마 단위 GRANT는 불가능** → 테이블마다 따로 권한을 줘야 한다.
	- `GRANT SELECT ON KH.* TO TEST;` 미지원

---

## 팁과 주의점 ⚠️

- **권한 부족 오류 해석:** ORA-01031(권한 부족), ORA-01950(테이블스페이스 권한 없음) 원인 분리 판단
- **테이블스페이스 권한:** 데이터 삽입 실패 시 USERS 테이블스페이스 사용 권한 확인 요망
- **동의어 활용:** 타 스키마 객체 접근 시 스키마 접두사 회피 목적의 SYNONYM(동의어,별칭) 생성 권장
	- `CREATE SYNONYM 동의어 FOR 계정명.테이블명;`
- **최소 권한 원칙:** 필요 권한만 부여하여 보안 유지
- **ROLE 오남용 주의:** ROLE 권한 구성이 환경마다 상이, 실제 권한 구성 확인 필요

## 예시 📌

### ✅ 계정 생성 및 시스템 권한 부여 흐름

```sql
-- C## 접두사 없이 계정 생성 허용 설정(테스트 환경 전용 권장)
ALTER SESSION SET "_ORACLE_SCRIPT" = TRUE;

-- 1. 계정 생성
CREATE USER TEST IDENTIFIED BY TEST;
-- 실패 예상: ORA-01045 (CREATE SESSION 권한 부재로 로그온 거절)

-- 2. 접속 권한 부여
GRANT CREATE SESSION TO TEST;

-- 3. 테이블 생성 권한 부여
GRANT CREATE TABLE TO TEST;

-- 4. 테이블스페이스 권한 부여 (데이터 저장 공간 권한)
GRANT UNLIMITED TABLESPACE TO TEST;

-- 5. 뷰 생성 권한 부여
GRANT CREATE VIEW TO TEST;
```

### ✅ 객체 생성 및 권한 부족 오류 해결 흐름

```sql
-- TEST 사용자로 로그인 후 테이블 생성
CREATE TABLE TEST (
  NO   NUMBER,
  NAME VARCHAR2(20)
);
-- 권한 부여 전: ORA-01031 발생 가능 (CREATE TABLE 권한 부재)
-- 권한 부여 후 성공

-- 데이터 삽입
INSERT INTO TEST VALUES (1, '김일번');
-- 권한 부여 전: ORA-01950 발생 가능 (USERS 테이블스페이스 권한 부재)
-- UNLIMITED TABLESPACE 부여 후 성공

-- 뷰 생성
CREATE VIEW VW_TEST AS
  SELECT * FROM TEST;
-- 권한 부여 전: ORA-01031 (CREATE VIEW 권한 부재)
-- 권한 부여 후 성공

-- 뷰 조회
SELECT * FROM VW_TEST;
```

### ✅ 타 스키마 객체 접근 및 트랜잭션 반영 흐름

```sql
-- SYSTEM(또는 KH 객체 소유자)에서 TEST에 객체 권한 부여
GRANT SELECT ON KH.EMPLOYEE TO TEST;                 -- 조회 권한 부여
GRANT INSERT, SELECT ON KH.DEPT_COPY2 TO TEST;       -- 삽입/조회 권한 부여

-- TEST 사용자에서 타 스키마 테이블 접근
SELECT * FROM KH.EMPLOYEE;                           -- 권한 부여 후 조회 가능

-- 트랜잭션 테스트: 삽입 후 COMMIT 전 가시성
INSERT INTO KH.DEPT_COPY2 VALUES ('미국');           -- 현재 세션 내 삽입 반영
SELECT * FROM KH.DEPT_COPY2;                         -- 현재 세션에서 확인 가능
-- 다른 세션/계정에서는 COMMIT 전까지 변경사항 미가시성

COMMIT;                                              -- 트랜잭션 확정, 타 세션에서 조회 가능
```

### ✅ ROLE 권한 조회 흐름

```sql
-- ROLE에 포함된 시스템 권한 조회
SELECT *
FROM DBA_SYS_PRIVS
WHERE GRANTEE IN ('CONNECT', 'RESOURCE')
ORDER BY 1;

-- 주의: ROLE에 포함된 권한 구성은 DB 버전/정책에 따라 상이 가능
```

### ✅ 권한 회수(REVOKE) 흐름

```sql
-- TEST 사용자로부터 테이블 생성 권한 회수
REVOKE CREATE TABLE FROM TEST;

-- 회수 후 동작 확인
CREATE TABLE NEWTABLE (
  NO   NUMBER,
  NAME VARCHAR2(15)
);
-- ORA-01031: 권한이 불충분합니다
```

# 📘 PL/SQL 프로시저

## Summary 📝

> [!summary] Summary 📝
> 
> - **정의:** 데이터베이스에 저장해 호출하는 PL/SQL 프로그램 단위
> - **목적:** 로직 재사용, 성능 향상, 보안 강화, 유지보수 단순화
> - **핵심:** IN/OUT 매개변수, 예외처리, 트랜잭션 제어, 컴파일-저장-실행 흐름
> - **예시:** 데이터 삭제, 부서 평균 급여 및 인원수 반환

---

## 목적 🎯

- **재사용성 확보:** 반복 로직의 표준화 및 호출 기반 실행
- **성능 최적화:** 서버 측 컴파일·저장으로 실행 효율 향상
- **보안 제어:** 테이블 직접 접근 차단, 프로시저 실행 권한 부여 중심 제어
- **유지보수 단순화:** 로직 수정 시 호출부 변경 최소화

---

## 핵심 개념 💡

### ✅ 기본 문법

```sql
CREATE [OR REPLACE] PROCEDURE 프로시저명 [(매개변수)]
IS | AS
    -- 선언부 (변수, 상수 등)
BEGIN
    -- 실행부 (SQL, 제어문 등)
EXCEPTION
    -- 예외 처리부 (선택)
END;
/
```

### ✅ 매개변수 모드

- **IN:** 입력값 전달, 내부 수정 불가 권장
- **OUT:** 결과값 반환, 호출자 변수에 값 반영
- **IN OUT:** 입력받아 수정 후 반환, 상태 갱신 절차에 사용

#### 비교표

|구분|목적|데이터 흐름|사용 사례|
|---|---|---|---|
|프로시저|로직 실행|IN/OUT 매개변수로 입·출력|DML 수행, 배치 처리, 보고서 생성|
|함수|값 반환|RETURN 단일 값|계산식, SELECT 내 호출, 트랜스폼|

---

## 특징 ✨

#### ✅ 장점

- 처리 속도가 빠름
- 대량 데이터 처리에 효율적임
    - DB에서 직접 처리 시 애플리케이션 단에서 처리하는 것보다 효율적
    - 대용량 데이터를 애플리케이션으로 전송할 때 발생하는 네트워크 비용 절감 가능
#### ⚠️ 단점

- DB 자원을 직접 사용하기 때문에 서버에 부하를 줄 수 있음
- 유지보수 시 애플리케이션 소스코드와 DB 내 프로시저 코드를 동시에 관리해야 하므로 관리가 복잡해질 수 있음
#### ✅ 기타 특징
- **컴파일-저장-실행:** 데이터베이스에 저장되어 이름으로 실행
- **권한 제어:** EXECUTE 권한 부여로 안전한 접근 경로 확보
- **에러 처리 일원화:** 예외 메시지 표준화로 운영 안정성 확보
- **스키마 결합:** %TYPE, %ROWTYPE으로 스키마 변경 대응성 강화
- **가독성:** 매개변수 정렬, SELECT/INTO 구문 정렬, 주석 활용

---

## 팁과 주의점 ⚠️

- **트랜잭션 경계:** DML 포함 시 COMMIT/ROLLBACK 정책 명확화
- **예외 구체화:** ZERO_DIVIDE, NO_DATA_FOUND, TOO_MANY_ROWS 등 개별 처리
- **OUT 변수 초기화:** 예외 발생 시 OUT 값 명시 초기화
- **성능:** 대량 처리 시 BULK COLLECT, FORALL 고려
- **보안:** 직접 테이블 권한 제한, 프로시저 EXECUTE 권한 중심 운영

## 예시 📌

### ✅ 단순 삭제 프로시저

```sql
-- 목적: 테스트 테이블 전체 삭제 및 커밋 수행
CREATE OR REPLACE PROCEDURE DELETE_DATA
IS
BEGIN
    DELETE FROM PRO_TEST; -- 전체 행 삭제
    COMMIT;               -- 트랜잭션 반영
EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('오류: ' || SQLERRM);
END;
/

-- 실행
EXEC DELETE_DATA;

-- 결과 확인
SELECT COUNT(*) AS CNT FROM PRO_TEST;
```

### ✅ 부서 평균 급여 및 인원수 반환

```sql
-- 목적: 부서 평균 급여와 인원수 반환
CREATE OR REPLACE PROCEDURE DEPT_AVG (
    P_DEPT_CODE IN  EMP_PRO_TEST.DEPT_CODE%TYPE, -- 입력: 부서코드
    O_AVG_SAL   OUT NUMBER,                      -- 출력: 평균급여
    O_COUNT     OUT NUMBER                       -- 출력: 인원수
)
IS
BEGIN
    SELECT AVG(SALARY), COUNT(*)
      INTO O_AVG_SAL, O_COUNT
      FROM EMP_PRO_TEST
     WHERE DEPT_CODE = P_DEPT_CODE;

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        O_AVG_SAL := NULL;
        O_COUNT   := 0;
        DBMS_OUTPUT.PUT_LINE('결과 없음');
END;
/

-- 호출
DECLARE
    v_avg_sal NUMBER;
    v_count   NUMBER;
BEGIN
    DEPT_AVG('&부서코드', v_avg_sal, v_count);

    DBMS_OUTPUT.PUT_LINE(
        '평균 급여 : ' || ROUND(v_avg_sal, -3) || '원 / 인원수 : ' || v_count || '명'
    );
END;
/
```

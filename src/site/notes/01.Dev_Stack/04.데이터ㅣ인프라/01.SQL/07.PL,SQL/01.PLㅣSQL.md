# 📘 Oracle PL/SQL 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - **PL/SQL**: Oracle DB에 내장된 절차형 언어
> - **구조**: 선언부(DECLARE) → 실행부(BEGIN) → 예외처리부(EXCEPTION) → 종료부(END)
> - **특징**: 변수/상수 선언, 조건문·반복문·예외처리 지원, 다수 SQL문 일괄 실행 가능
> - **변수 유형**: 일반 타입, 레퍼런스 타입(%TYPE), ROW타입(%ROWTYPE)

---

## 목적 🎯

- SQL 단점 보완(조건·반복·예외 처리 가능)
- 다수 SQL문을 하나의 블록으로 실행
- 변수 활용으로 코드 재사용성 및 가독성 향상

---

## 핵심 개념 💡

### ✅ PL/SQL 블록 구조

|구문|설명|
|---|---|
|DECLARE|변수·상수 선언|
|BEGIN|실행부, SQL 및 제어문 작성|
|EXCEPTION|예외 발생 시 처리|
|END|블록 종료 (`/`로 실행)|

### ✅ 변수 선언 유형

1. **일반 타입 변수**
    
    ```sql
    변수명 [CONSTANT] 자료형 [:= 값];
    ```
    
2. **레퍼런스 타입 변수 (%TYPE)**
    
    - 특정 테이블 컬럼의 자료형 참조
    
    ```sql
    변수명 테이블명.컬럼명%TYPE;
    ```
    
3. **ROW타입 변수 (%ROWTYPE)**
    
    - 테이블 한 행 전체를 담는 변수
    
    ```sql
    변수명 테이블명%ROWTYPE;
    ```
### ✅ PL/SQL 제어문 종류

- **조건 제어문**
    - `IF ... THEN ... END IF;`
    - `IF ... ELSIF ... ELSE ... END IF;`
    - `CASE (WHEN...THEN 반복) END CASE;`
    - 
- **반복 제어문**
    - `LOOP ... END LOOP;`
	    - 실행식, 탈출 조건으로 구성
    - `WHILE ... LOOP ... END LOOP;`
	    - 반복조건, 실행식으로 구성
    - `FOR 변수 IN 범위지정 LOOP ... END LOOP;`
	    - 반복할 범위(횟수) 지정, 실행식 으로 구성
	```SQL
	-- 1) BASIC LOOP문
	LOOP
		반복 실행 구문;
		-- 반복 탈출 구문
		-- 1) IF 조건식 THEN EXIT; END IF;
		-- 2) EXIT WHEN 조건식;
	END LOOP;
	
	
	-- 2) FOR LOOP문
	FOR 변수 IN 초기값..최종값
	LOOP
		반복 실행 구문;
	END LOOP;
	```
- **예외 처리문**
    
    - `EXCEPTION WHEN 예외명 THEN ...`

| 예외 이름           | 설명                     |
| --------------- | ---------------------- |
| `ZERO_DIVIDE`   | 0으로 나누었을 때 발생          |
| `NO_DATA_FOUND` | SELECT 결과가 없을 때 발생     |
| `TOO_MANY_ROWS` | SELECT가 여러 행을 반환할 때 발생 |
| `OTHERS`        | 위에 명시되지 않은 모든 예외 처리    |

## 특징 ✨

- `SET SERVEROUTPUT ON;`를 켜야 화면에 출력 내용을 보여준다
- **DBMS_OUTPUT.PUT_LINE**으로 출력 확인
- **&변수명**: 사용자 입력값 바인딩
- **SELECT … INTO**: 조회 결과를 변수에 대입
- **VIEW + %ROWTYPE**: 뷰 기반으로도 행 단위 변수 선언 가능

---

## 팁과 주의점 ⚠️

- `/`는 PL/SQL 블록 종료 표시, 뒤에 주석 불가
- `CURRVAL`처럼 **SELECT INTO는 반드시 1행만 반환**해야 함 → 다중행 반환 시 오류 발생
- 사용자 입력 시 문자열은 `'&값'` 형태로 감싸야 함
- ROWTYPE 사용 시 컬럼명 접근은 `변수명.컬럼명`
#### IF 제어문 문법
```sql
IF 조건식 THEN
    실행구문;
ELSIF 조건식 THEN
    실행구문;
ELSE
    실행구문;
END IF;
```

## 예시 📌

### ✅ 기본 출력

```sql
SET SERVEROUTPUT ON;

BEGIN
    DBMS_OUTPUT.PUT_LINE('HELLO ORACLE');
END;
/
```

---

### ✅ 일반 타입 변수

```sql
DECLARE
    EID NUMBER;
    ENAME VARCHAR2(20);
    PI CONSTANT NUMBER := 3.14;
BEGIN
    EID := 999;
    ENAME := '김구구';
    DBMS_OUTPUT.PUT_LINE('EID : '||EID);
    DBMS_OUTPUT.PUT_LINE('ENAME : '||ENAME);
END;
/
```

---

### ✅ 사용자 입력

```sql
DECLARE
    EID NUMBER;
    ENAME VARCHAR2(20);
BEGIN
    EID := &번호;
    ENAME := '&이름';
    DBMS_OUTPUT.PUT_LINE('EID : '||EID);
    DBMS_OUTPUT.PUT_LINE('ENAME : '||ENAME);
END;
/
```

---

### ✅ 레퍼런스 타입 변수 (%TYPE)

```sql
DECLARE
    EID EMPLOYEE.EMP_ID%TYPE;
    ENAME EMPLOYEE.EMP_NAME%TYPE;
    SALARY EMPLOYEE.SALARY%TYPE;
BEGIN
    SELECT EMP_ID, EMP_NAME, SALARY
    INTO EID, ENAME, SALARY
    FROM EMPLOYEE
    WHERE EMP_ID = 200;

    DBMS_OUTPUT.PUT_LINE('사번 : '||EID);
    DBMS_OUTPUT.PUT_LINE('사원명 : '||ENAME);
    DBMS_OUTPUT.PUT_LINE('급여 : '||SALARY);
END;
/
```

---

### ✅ 실습: 사번 입력받아 사원정보 출력

```sql
DECLARE
    EID EMPLOYEE.EMP_ID%TYPE;
    ENAME EMPLOYEE.EMP_NAME%TYPE;
    JCODE EMPLOYEE.JOB_CODE%TYPE;
    DTITLE DEPARTMENT.DEPT_TITLE%TYPE;
    SAL EMPLOYEE.SALARY%TYPE;
BEGIN
    SELECT EMP_ID, EMP_NAME, JOB_CODE, DEPT_TITLE, SALARY
    INTO EID, ENAME, JCODE, DTITLE, SAL
    FROM EMPLOYEE E
    JOIN DEPARTMENT D ON (E.DEPT_CODE = D.DEPT_ID)
    WHERE EMP_ID = &사번;

    DBMS_OUTPUT.PUT_LINE('사번 : '||EID);
    DBMS_OUTPUT.PUT_LINE('사원명 : '||ENAME);
    DBMS_OUTPUT.PUT_LINE('직업코드 : '||JCODE);
    DBMS_OUTPUT.PUT_LINE('부서이름 : '||DTITLE);
    DBMS_OUTPUT.PUT_LINE('월급 : '||SAL);
END;
/
```

---

### ✅ ROW타입 변수 (%ROWTYPE)

```sql
DECLARE
    E EMPLOYEE%ROWTYPE;
BEGIN
    SELECT *
    INTO E
    FROM EMPLOYEE
    WHERE EMP_ID = '&사번';

    DBMS_OUTPUT.PUT_LINE('사원명 : '||E.EMP_NAME);
    DBMS_OUTPUT.PUT_LINE('전화번호 : '||E.PHONE);
    DBMS_OUTPUT.PUT_LINE('주민번호 : '||E.EMP_NO);
END;
/
```

---

### ✅ VIEW + ROWTYPE 활용

```sql
CREATE OR REPLACE VIEW VW_PLSQL AS
SELECT EMP_NAME, DEPT_TITLE, JOB_NAME
FROM EMPLOYEE E
LEFT JOIN JOB J ON (E.JOB_CODE = J.JOB_CODE)
LEFT JOIN DEPARTMENT D ON (E.DEPT_CODE = D.DEPT_ID);

DECLARE
    PL VW_PLSQL%ROWTYPE;
BEGIN
    SELECT *
    INTO PL
    FROM VW_PLSQL
    WHERE EMP_NAME = (
        SELECT EMP_NAME
        FROM EMPLOYEE
        WHERE EMP_ID = '&사번'
    );

    DBMS_OUTPUT.PUT_LINE('사원명 : '||PL.EMP_NAME);
    DBMS_OUTPUT.PUT_LINE('부서명 : '||PL.DEPT_TITLE);
    DBMS_OUTPUT.PUT_LINE('직급명 : '||PL.JOB_NAME);
END;
/
```

## 개선점 🔧

- **예외처리부(EXCEPTION)** 예시 추가 필요 (NO_DATA_FOUND, TOO_MANY_ROWS 등)
- **반복문(FOR, WHILE, LOOP)** 활용 예시 보강 필요
- **조건문(IF)** 예시 포함 시 학습 효과 상승

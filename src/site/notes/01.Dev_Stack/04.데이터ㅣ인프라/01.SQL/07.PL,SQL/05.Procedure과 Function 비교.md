# Procedure과 Funtion 차이점
#### ✅ 함수(Function)가 유리한 경우

- **단일 값 반환이 목적일 때**  
    예: 세금 계산, 문자열 변환, 특정 값 집계
- **SQL 구문 안에서 직접 호출해야 할 때**  
    예:
    
    ```sql
    SELECT EMP_NAME, CALC_TAX(SALARY) FROM EMPLOYEE;
    ```
    
- **순수 계산/변환 로직**을 캡슐화할 때  
    → DB 상태를 바꾸지 않고, 입력값 → 결과값 변환

---

#### ✅ 프로시저(Procedure)가 유리한 경우

- **여러 작업을 묶어서 실행할 때**  
    예: 여러 테이블에 INSERT/UPDATE/DELETE 수행
- **트랜잭션 제어가 필요할 때**  
    예: 대량 데이터 처리 후 COMMIT/ROLLBACK
- **여러 개의 OUT 값 반환이 필요할 때**  
    예: 부서 평균 급여 + 인원수 같이 다중 결과 반환
- **보안/권한 제어**  
    → 테이블 직접 접근 대신 프로시저 실행 권한만 부여

---

# 📌 정리

|구분|함수(Function)|프로시저(Procedure)|
|---|---|---|
|반환|단일 값 (RETURN)|여러 값 (OUT 매개변수) 가능|
|호출 위치|SQL 구문 안, PL/SQL 블록|PL/SQL 블록, EXEC 명령|
|주 용도|계산, 변환, 집계|DML, 배치 처리, 트랜잭션|
|특징|순수 함수 권장, 부작용 최소화|상태 변경 가능, 예외 처리 중심|

- **“값을 계산해서 돌려줘야 한다” → 함수(Function)**
- **“여러 작업을 실행하고 상태를 바꿔야 한다” → 프로시저(Procedure)**

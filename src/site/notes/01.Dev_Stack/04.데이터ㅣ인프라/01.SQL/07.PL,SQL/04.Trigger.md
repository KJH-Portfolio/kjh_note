# 📘 PL/SQL 트리거 (Trigger)

## Summary 📝

> [!summary] Summary 📝
> 
> - **정의:** 특정 이벤트(INSERT, UPDATE, DELETE, DDL, DB 이벤트 등)가 발생할 때 **자동으로 실행되는 PL/SQL 프로그램 단위**
> - **목적:** 데이터 무결성 유지, 자동 로깅, 제약 조건 보완, 업무 규칙 자동화
> - **핵심:** 사용자가 직접 호출하지 않고, **DB 이벤트에 의해 암묵적으로 실행**
> - **예시:** 급여 변경 시 로그 기록, 삭제 시 관련 데이터 자동 정리

---

## 목적 🎯

- **자동화:** 데이터 변경 시 별도 호출 없이 자동 실행
- **무결성 유지:** 복잡한 제약 조건을 보완하여 데이터 일관성 확보
- **감사/로깅:** 변경 이력 자동 기록
- **업무 규칙 반영:** 특정 컬럼 자동 갱신(예: 수정일자, 변경자)

---

## 핵심 개념 💡

### ✅ 기본 문법

```sql
CREATE [OR REPLACE] TRIGGER 트리거명
{BEFORE | AFTER | INSTEAD OF} 
	{INSERT | UPDATE | DELETE} [OF 컬럼명]  ON 테이블명 
[FOR EACH ROW]
[WHEN (조건)]
DECLARE
    -- 선언부 (변수, 상수 등)
BEGIN
    -- 실행부 (자동 실행될 로직)
EXCEPTION
    -- 예외 처리부 (선택)
END;
/
```

### ✅ 트리거 유형

- **DML 트리거:** INSERT, UPDATE, DELETE 시 실행
- **DDL 트리거:** CREATE, ALTER, DROP 같은 DDL 실행 시 실행
- **DB 이벤트 트리거:** LOGON, LOGOFF, STARTUP, SHUTDOWN 시 실행

---

## 특징 ✨

#### ✅ 장점

- 데이터 변경 시 자동 실행 → 개발자 개입 최소화
- 무결성, 보안, 로깅 등 다양한 자동화 가능
- 복잡한 제약 조건을 구현 가능

#### ⚠️ 단점

- 남용 시 성능 저하 (모든 DML마다 실행되므로)
- 디버깅이 어려움 (암묵적으로 실행되기 때문)
- 잘못 작성하면 무한 루프 발생 가능 (트리거 안에서 같은 테이블 갱신 시)

#### ✅ 기타 특징

- **자동 실행:** 사용자가 직접 호출하지 않음
- **:OLD / :NEW 가상 레코드:** 변경 전/후 값 참조 가능
- **행 단위/문 단위:** `FOR EACH ROW` 여부에 따라 실행 단위 달라짐

---

## 팁과 주의점 ⚠️

- **성능 고려:** 꼭 필요한 경우에만 작성
- **무한 루프 방지:** 같은 테이블을 다시 갱신하지 않도록 주의
- **예외 처리:** 트리거 내부 오류는 전체 DML을 실패시킬 수 있음
- **로깅 용도:** 변경 이력 기록 시 유용

---

## 📌 정리

- **프로시저**: 내가 직접 실행하는 작업 단위
- **함수**: 값을 계산해서 반환하는 단위
- **트리거**: DB 이벤트가 발생하면 자동 실행되는 숨은 프로시저

---

## Procedure / Function / Trigger 차이점

|구분|프로시저|함수|트리거|
|---|---|---|---|
|실행 방식|사용자가 직접 호출|SQL/PLSQL 내 호출|**DB 이벤트 발생 시 자동 실행**|
|반환|OUT 매개변수로 여러 값|RETURN 단일 값|반환 없음|
|주 용도|작업 실행, 배치 처리|계산, 변환, 집계|무결성 유지, 자동 로깅|
|호출 위치|EXEC, PL/SQL 블록|SELECT, WHERE 등|INSERT/UPDATE/DELETE, DDL, DB 이벤트|

---

## 예시 📌

### ✅ 급여 변경 시 로그 기록 트리거

```sql
CREATE OR REPLACE TRIGGER EMP_SALARY_AUDIT
AFTER UPDATE OF SALARY ON EMPLOYEE
FOR EACH ROW
BEGIN
    INSERT INTO EMP_SALARY_LOG(EMP_ID, OLD_SAL, NEW_SAL, CHANGE_DATE)
    VALUES(:OLD.EMP_ID, :OLD.SALARY, :NEW.SALARY, SYSDATE);
END;
/
```

- `:OLD` → 변경 전 값
- `:NEW` → 변경 후 값
- 실행: `UPDATE EMPLOYEE SET SALARY = SALARY*1.1 WHERE EMP_ID=200;`  
    → 자동으로 로그 테이블에 기록됨

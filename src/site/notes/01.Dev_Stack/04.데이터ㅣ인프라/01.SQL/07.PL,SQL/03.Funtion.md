# 📘 PL/SQL 함수 (Function)

## Summary 📝

> [!summary] Summary 📝
> 
> - **정의:** 특정 연산을 수행하고 **단일 값을 반환**하는 PL/SQL 프로그램 단위
> - **목적:** 계산, 변환, 집계 등 결과값을 반환하는 로직 캡슐화
> - **핵심:** `RETURN` 절 필수, SELECT/표현식 내 호출 가능, IN 매개변수 중심
> - **예시:** 세금 계산, 급여 등급 산출, 문자열 변환

---

## 목적 🎯

- **재사용성 확보:** 반복되는 계산 로직을 함수로 정의해 호출
- **표현식 활용:** SELECT, WHERE, ORDER BY 등 SQL 구문 안에서 직접 사용 가능
- **캡슐화:** 복잡한 계산을 함수로 감싸 단순화
- **유지보수 용이:** 로직 변경 시 함수만 수정하면 됨

---

## 핵심 개념 💡

### ✅ 기본 문법

```sql
CREATE [OR REPLACE] FUNCTION 함수명 [(매개변수)]
RETURN 반환타입
IS | AS
    -- 선언부 (변수, 상수 등)
BEGIN
    -- 실행부 (계산, SQL 등)
    RETURN 반환값;
EXCEPTION
    -- 예외 처리부 (선택)
END;
/
```

### ✅ 매개변수 모드

- 함수는 일반적으로 **IN 매개변수**만 사용 (OUT/IN OUT은 권장되지 않음)
- 반드시 **RETURN 절**로 단일 값을 반환해야 함

---

## 특징 ✨

#### ✅ 장점

- SQL 구문 안에서 직접 호출 가능 (`SELECT`, `WHERE`, `ORDER BY` 등)
- 복잡한 계산 로직을 단순화
- 재사용성 및 가독성 향상

#### ⚠️ 단점

- 반드시 **하나의 값만 반환** 가능
- DML(INSERT/UPDATE/DELETE) 포함 시 제약이 많음 (순수 함수 권장)
- 과도한 함수 호출은 성능 저하 유발 가능

#### ✅ 기타 특징

- **컴파일-저장-실행:** DB에 저장되어 이름으로 호출
- **권한 제어:** EXECUTE 권한 필요
- **스키마 결합:** %TYPE, %ROWTYPE 활용 가능
- **표현식 통합:** SQL 문장 안에서 자유롭게 사용

---

## 팁과 주의점 ⚠️

- **RETURN 필수:** 모든 실행 경로에서 RETURN 값이 있어야 함
- **순수 함수 권장:** SELECT 안에서 호출 시 부작용 없는 함수가 이상적
- **예외 처리:** NO_DATA_FOUND, ZERO_DIVIDE 등 상황별 처리
- **성능 고려:** 대량 데이터 조회 시 함수 남용 주의
- SQL 구문을 작성할때 **프로시저의 NEXTVAL을 뽑아내는 함수를 만들어** 접근 용이하게 사용하는 경우도 있음

## 예시 📌

### ✅ 기본 구조 예시
```sql
CREATE OR REPLACE FUNCTION MYFUNC(STR VARCHAR2) 
--VARCHAR2의 바이트 크기는 전달받은 곳에서 작성 

RETURN VARCHAR2
--반환할 데이터 타입을 지정
IS  --선언부 
    RESULT VARCHAR2(100); --변수 선언
BEGIN
    DBMS_OUTPUT.PUT_LINE('STR : '||STR);
    RESULT := STR||'입니다';
    RETURN RESULT; --반환값     
END;
/
```

```sql
-- 함수 호출
SELECT MYFUNC('저는 학생')
FROM DUAL;
```



---

### ✅ 함수 호출 예시

```sql
-- SQL 구문 안에서 직접 호출
SELECT EMP_NAME,
       SALARY,
       CALC_TAX(SALARY) AS TAX
  FROM EMP_PRO_TEST;

-- PL/SQL 블록에서 호출
DECLARE
    v_tax NUMBER;
BEGIN
    v_tax := CALC_TAX(3500000);
    DBMS_OUTPUT.PUT_LINE('세금: ' || v_tax);
END;
/
```

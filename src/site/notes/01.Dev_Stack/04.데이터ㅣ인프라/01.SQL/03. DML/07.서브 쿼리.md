## 📘 서브쿼리(Subquery) — 최적화 개선판

## Summary 📝

> [!summary] Summary 📝
> 
> - 서브쿼리는 SQL문 안에 포함된 보조 SELECT 구문
> - 결과 행·열 개수에 따라 유형 분류
> - 연산자 사용 가능 여부는 결과 유형에 따라 달라짐
> - 최적화 시 JOIN, EXISTS, WITH 절, 인덱스 활용 고려

---

## 목적 🎯

- SQL문 내에서 **보조 데이터 조회** 및 **조건 필터링** 수행 목적
- 복잡한 쿼리를 **단순화**하고 **성능 최적화**를 통한 효율적 데이터 처리 목적

---

## 핵심 개념 💡

### ✅ 서브쿼리 유형 분류

| 유형      | 결과 형태      | 사용 연산자/특징                       |
| ------- | ---------- | ------------------------------- |
| 단일행 단일열 | 1행 1열      | 일반 비교 연산자 (=, <>, >, <, >=, <=) |
| 다중행 단일열 | 여러 행, 1열   | `IN`, `ANY(SOME)`, `ALL`        |
| 단일행 다중열 | 1행, 여러 열   | 다중 컬럼 비교 가능                     |
| 다중행 다중열 | 여러 행, 여러 열 | 주로 `EXISTS`, `IN`과 함께 사용        |
### ✅ WITH 절을 활용한 서브쿼리
- 최적화된 방법
- 서브쿼리를 외부에 선언해서 참조,공유 하는 방식
```SQL
WITH 서브쿼리 AS(
	SELECT 컬럼
	FROM 테이블)
SELECT
FROM 서브쿼리;
```

## 특징 ✨

### ✅ 연산자 활용

- **단일행 단일열:** 일반 비교 연산자 사용
- **다중행 단일열:**
    - `IN`: 결과 중 하나라도 일치
    - `> ANY`: 결과 중 하나보다 크면 참
    - `< ANY`: 결과 중 하나보다 작으면 참
    - `> ALL`: 모든 결과보다 크면 참
    - `< ALL`: 모든 결과보다 작으면 참
```SQL
SELECT * FROM EMPLOYEE
WHERE SALARY > ANY (SELECT SALARY FROM EMPLOYEE WHERE DEPT_CODE = 'D1');

10  > ANY (3,5,7) -- 참
```

### ✅ WITH 절

- 긴 서브쿼리를 **사전 선언**하여 테이블처럼 사용
- 가독성과 최적화에 유리

### ✅ 자주 사용하는 함수

- `ROWNUM`: 행 번호 반환 (정렬 후 사용 권장)
- `RANK() OVER(정렬기준)`: 순위 반환, 동일 값은 동일 순위, 건너뜀
- `DENSE_RANK() OVER(정렬기준)`: 순위 반환, 동일 값은 동일 순위, 건너뛰지 않음

---

## 팁과 주의점 ⚠️

- **EXISTS**는 조건 충족 시 즉시 TRUE 반환 → 대용량 데이터에서 효율적
- **IN**은 전체 결과 집합을 메모리에 로드 → 소규모 데이터에 적합
- **JOIN 권장:** 동일 결과라면 JOIN이 일반적으로 더 빠름
- **WITH 절 활용:** 복잡한 서브쿼리 재사용 가능, 실행 계획 단순화
- **인덱스(RANK) 활용:** WHERE 조건 컬럼에 인덱스 생성 시 성능 향상
- **SELECT * 지양:** 필요한 컬럼만 조회하여 불필요한 I/O 감소
- **ROWNUM 주의:** 반드시 정렬 후 사용해야 의미 있는 결과 확보
- **실행 계획 확인:** `EXPLAIN PLAN`으로 실제 성능 검증 필수

## 예시 📌

### ✅ **단일행 단일열 서브쿼리**

```sql
-- 동등 비교 가능
WHERE 월급 = (SELECT MAX(월급) FROM 사원);
```

---

### ✅ **다중행 단일열 서브쿼리**

```sql
-- 일반 동등비교 불가(행이 여러개일 수도 있기에)
SELECT 사번, 월급
FROM 사원
WHERE 월급 IN (SELECT 월급 FROM 사원 WHERE 부서번호 = 10);

-- 동일 의미 (ANY 사용)
SELECT 사번, 월급
FROM 사원
WHERE 월급 = ANY (SELECT 월급 FROM 사원 WHERE 부서번호 = 10);
```

---

### ✅ **다중행 다중열 서브쿼리**

```sql
-- 부서별 최고/최저 월급을 받는 사원 찾기
SELECT 사번, 부서번호, 월급
FROM 사원
WHERE (부서번호, 월급) IN (
    SELECT 부서번호, MAX(월급)
    FROM 사원
    GROUP BY 부서번호
    UNION
    SELECT 부서번호, MIN(월급)
    FROM 사원
    GROUP BY 부서번호
);
```

---

### ✅ **EXISTS vs IN 비교**

```sql
-- IN 사용 (소규모 데이터에 적합)
SELECT 사번, 이름
FROM 사원 e
WHERE e.부서번호 IN (
    SELECT d.부서번호
    FROM 부서 d
    WHERE d.지역번호 = 1700
);

-- EXISTS 사용 (대규모 데이터에 적합)
SELECT 사번, 이름
FROM 사원 e
WHERE EXISTS (
    SELECT 1
    FROM 부서 d
    WHERE d.부서번호 = e.부서번호
      AND d.지역번호 = 1700
);
```

---

### ✅ **WITH 절을 활용한 서브쿼리 최적화**

```sql
-- WITH 절로 고액 연봉자 중 평균 초과자 찾기
WITH 고액사원 AS (
    SELECT 사번, 월급
    FROM 사원
    WHERE 월급 > 5000
)
SELECT 사번, 월급
FROM 고액사원
WHERE 월급 > (SELECT AVG(월급) FROM 고액사원);
```

### ✅인덱스 부여하기
- `ROWNUM`  활용 
```SQL
WITH SUB AS(
	SELECT 
		EMP_ID
		,EMP_NAME
	FROM EMPLOYEE
	ORDER BY EMP_ID
)
SELECT 
	EMP_ID
	,EMP_NAME
	,ROWNUM
FROM SUB
WHERE ROWNUM = 1;
```
- `RANK()` 활용
```SQL
WITH SUB AS(
	SELECT
		RANK()OVER(ORDER BY EMP_ID)
		,EMP_NAME
	FROM EMPLOYEE
)
SELECT *
FROM SUB
WHERE RANK =1;
```

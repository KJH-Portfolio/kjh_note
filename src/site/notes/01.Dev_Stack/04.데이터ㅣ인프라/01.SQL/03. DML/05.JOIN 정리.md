# 📘 JOIN 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - 관계형 데이터베이스에서 여러 테이블을 연관 컬럼으로 결합하여 하나의 결과 집합으로 조회하는 구문 정리
> - ANSI 표준 JOIN과 오라클 전용 JOIN 비교, 내부·외부·교차·비등가·자체·다중 조인 개념과 예시 정리

---

## 목적 🎯

- 중복 최소화된 테이블 간의 관계를 맺어 필요한 데이터를 통합 조회하는 목적
- 연관된 컬럼을 기준으로 결합하여 의미 있는 결과 집합 생성 목적

---

## 핵심 개념 💡

### ✅ JOIN 분류 개요

- **등가/내부 조인:** 연결 컬럼 값이 일치하는 행만 결합 **INNER**
- **외부 조인:** 기준 테이블의 불일치 행까지 포함하여 결합 **OUTER**
- **교차 조인:** 모든 행 조합 반환(카테시안 곱) **CROSS**
- **비등가 조인:** 비교 연산자 활용, 범위 매칭 기반 결합 **BETWEEN .. AND..**
- **자체 조인:** 동일 테이블 간 결합(별칭 필수)
- **다중 조인:** 3개 이상 테이블 결합(조인 순서 중요)

### ✅ ANSI vs 오라클 전용 JOIN 비교

|항목|ANSI JOIN 구문|오라클 전용 JOIN 구문|
|---|---|---|
|표준 여부|국제 표준|오라클 전용|
|OUTER JOIN 표현|LEFT/RIGHT/FULL JOIN|(+) 기호 사용|
|가독성/유지보수|높음|낮음(복잡해질 수 있음)|
|권장 여부|현재 권장|구버전 호환용으로 제한적 사용|

## 특징 ✨

### ✅ 내부(등가) 조인 INNER

- **조건:** 동등비교 연산자 = 사용
- **결과:** 연결 컬럼이 일치하는 행만 반환
- **구문 선택:** ANSI JOIN USING/ON, 오라클 WHERE 결합 지원

### ✅ 외부 조인 OUTER

- **LEFT OUTER JOIN:** 왼쪽(기준) 테이블 모든 행 포함
- **RIGHT OUTER JOIN:** 오른쪽(기준) 테이블 모든 행 포함
- **FULL OUTER JOIN:** 양쪽 모든 행 포함(오라클 전용 구문으로는 불가)
- **오라클 :** 기준 반대 테이블의 연결 컬럼 뒤에 (+) 표기

### ✅ 교차 조인(카테시안 곱) CROSS

- **결과:** 모든 행 조합 반환, 결과 행수 N×M
- **주의:** 조건 누락 시 과도한 결과로 과부하 위험

### ✅ 비등가 조인 BETWEEN AND

- **조건:** >, <, >=, <=, BETWEEN … AND … 등 범위 조건 사용
- **결과:** 값 일치가 아닌 범위 포함 기준으로 결합

### ✅ 자체 조인

- **조건:** 같은 테이블을 서로 다른 별칭으로 결합
- **필수:** 컬럼 구분을 위해 테이블별칭 사용

### ✅ 다중 조인

- **조건:** 3개 이상 테이블 결합
- **중요:** 조인 순서와 조건 배치 중요, 필터 시점 최적화 필요

---

## 팁과 주의점 ⚠️

- **컬럼명 충돌 주의:** 동일 컬럼명 존재 시 테이블명/별칭 명시 필요
- **USING vs ON:** USING은 컬럼명이 동일할 때만, 컬럼명이 다르면 ON 사용
- **필터 위치:** WHERE(조인 후 필터)와 ON(조인 조건) 분리하여 의도 명확화
	- ON() 안에 **어러개의 조건**이 들어는 방식으로도 사용 가능.
- **카테시안 곱 주의:** 조건 누락 방지, 불필요한 CROSS JOIN 금지
- **오라클 (+) 사용:** OUTER JOIN 방향 정확히 인지하여 반대 테이블 컬럼에 (+) 표기
- **FULL OUTER JOIN:** 오라클 전용 구문으로 표현 불가, ANSI 표준 구문 사용 필요
- **ANSI JOIN 권장**, 유지보수와 확장성 강화

## 예시 📌

## ✅INNER JOIN

```sql
SELECT 
    E.EMP_NAME AS 사원이름,
    D.DEPT_TITLE AS 부서이름
FROM EMPLOYEE E
JOIN DEPARTMENT D
  ON E.DEPT_CODE = D.DEPT_ID;
```

---

## ✅ 오라클에서 OUTER JOIN

```sql
SELECT 
    E.EMP_NAME AS 사원이름,
    D.DEPT_TITLE AS 부서이름
FROM EMPLOYEE E, DEPARTMENT D
WHERE E.DEPT_CODE = D.DEPT_ID(+);
```

---

## ✅ ANSI LEFT/RIGHT/FULL OUTER JOIN

```sql
-- LEFT JOIN
SELECT 
    E.EMP_NAME AS 사원이름,
    D.DEPT_TITLE AS 부서이름
FROM EMPLOYEE E
LEFT JOIN DEPARTMENT D
  ON E.DEPT_CODE = D.DEPT_ID;

-- RIGHT JOIN
SELECT 
    E.EMP_NAME AS 사원이름,
    D.DEPT_TITLE AS 부서이름
FROM EMPLOYEE E
RIGHT JOIN DEPARTMENT D
  ON E.DEPT_CODE = D.DEPT_ID;

-- FULL JOIN
SELECT 
    E.EMP_NAME AS 사원이름,
    D.DEPT_TITLE AS 부서이름
FROM EMPLOYEE E
FULL OUTER JOIN DEPARTMENT D
  ON E.DEPT_CODE = D.DEPT_ID;
```

## ✅ 교차 조인

```sql
SELECT 
    A.COL AS 항목A,
    B.COL AS 항목B
FROM TABLE_A A
CROSS JOIN TABLE_B B;
```

---

## ✅ 비등가 조인 (BETWEEN)

```sql
SELECT 
    E.EMP_NAME AS 사원이름,
    G.GRADE_NAME AS 급여등급
FROM EMPLOYEE E
JOIN SAL_GRADE G
  ON E.SAL BETWEEN G.MIN_SAL AND G.MAX_SAL;
```

---

## ✅ 자체 조인

```sql
SELECT 
    E.EMP_NAME AS 사원이름,
    M.EMP_NAME AS 상사이름
FROM EMPLOYEE E
JOIN EMPLOYEE M
  ON E.MGR_ID = M.EMP_ID;
```

---

## ✅ 다중 조인

```sql
SELECT 
    E.EMP_NAME AS 사원이름,
    D.DEPT_TITLE AS 부서이름,
    J.JOB_TITLE AS 직무이름
FROM EMPLOYEE E
JOIN DEPARTMENT D
  ON E.DEPT_CODE = D.DEPT_ID
JOIN JOB J
  ON E.JOB_CODE = J.JOB_ID;
```

---

## ✅ JOIN ON에 조건 넣기

```sql
SELECT
    S.STUDENT_NAME AS 학생이름,
    NVL(P.PROFESSOR_NAME, '지도교수 미지정') AS 지도교수이름
FROM TB_STUDENT S
LEFT JOIN TB_DEPARTMENT D 
  ON S.DEPARTMENT_NO = D.DEPARTMENT_NO AND D.DEPARTMENT_NAME = '서반어학과'
LEFT JOIN TB_PROFESSOR P 
  ON S.COACH_PROFESSOR_NO = P.PROFESSOR_NO;
```

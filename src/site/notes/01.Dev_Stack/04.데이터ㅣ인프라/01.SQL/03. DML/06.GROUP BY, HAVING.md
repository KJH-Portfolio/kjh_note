# 📘  GROUP BY, HAVING

## Summary 📝

> [!summary] Summary 📝
> 
> - GROUP BY와 HAVING 절, 집계 함수(SUM, AVG, COUNT, MAX, MIN) 정리
> - SELECT 실행 순서 포함

---

## 목적 🎯

- 데이터를 그룹 단위로 묶어 집계하고 조건을 적용하는 방법 학습

---

## 핵심 개념 💡

### ✅ GROUP BY

- 특정 컬럼 기준으로 행을 그룹화
- 그룹 함수와 함께 사용

```sql
SELECT DEPT_CODE, AVG(SALARY)
FROM EMPLOYEE
GROUP BY DEPT_CODE;
```

---

### ✅ HAVING

- 그룹 조건 지정 (그룹 함수 결과에 조건 적용)

```sql
SELECT DEPT_CODE, AVG(SALARY)
FROM EMPLOYEE
GROUP BY DEPT_CODE
HAVING AVG(SALARY) >= 3000;
```

---

### ✅ 집계 함수 종류

|함수|설명|NULL 처리|
|---|---|---|
|`SUM(컬럼)`|합계|NULL 제외|
|`AVG(컬럼)`|평균|NULL 제외|
|`MAX(컬럼)`|최댓값|NULL 제외|
|`MIN(컬럼)`|최솟값|NULL 제외|
|`COUNT(*)`|전체 행 개수|NULL 포함|
|`COUNT(컬럼)`|NULL 제외한 행 개수|NULL 제외|
|`COUNT(DISTINCT 컬럼)`|중복 제거 후 개수|NULL 제외|

---

## 특징 ✨

- WHERE는 행 단위 조건, HAVING은 그룹 조건
- GROUP BY 없는 집계 함수 → 전체 테이블 기준
- GROUP BY 있는 집계 함수 → 그룹 단위 집계

---

## 팁과 주의점 ⚠️

- SELECT 절에 일반 컬럼과 집계 함수를 함께 쓰려면 반드시 GROUP BY 필요
	- 집계함수만 사용할때는 문제 없음
- WHERE 절에는 집계 함수 사용 불가 → HAVING 사용
- COUNT(*)는 NULL 포함, COUNT(컬럼)은 NULL 제외

---

## 예시 📌

```sql
-- 부서별 평균 급여와 인원 수
SELECT
    DEPT_CODE,
    AVG(SALARY) AS 평균급여,
    COUNT(*) AS 인원수
FROM EMPLOYEE
GROUP BY DEPT_CODE
HAVING AVG(SALARY) >= 3000
ORDER BY 평균급여 DESC;
```

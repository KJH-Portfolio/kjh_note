# 계층형 쿼리 (Hierarchical Query)

> [!summary]
> **계층형 쿼리**는 테이블에 저장된 데이터를 계층적인 트리(Tree) 구조로 조회하기 위해 사용하는 SQL 쿼리입니다. 조직도, 카테고리, 답변형 게시판 등 부모와 자식 간의 관계가 명확한 데이터를 다룰 때 사용합니다.

## 1. 주요 용어 정리

계층형 구조를 이해하기 위해 필요한 기본 용어들입니다.

| 용어                      | 설명                                      |
| ----------------------- | --------------------------------------- |
| **루트 노드 (Root Node)**   | 계층 구조의 최상위에 있는 노드 (부모가 없는 노드)           |
| **부모 노드 (Parent Node)** | 하위에 자식 노드를 가지고 있는 노드                    |
| **자식 노드 (Child Node)**  | 상위에 부모 노드를 가지고 있는 노드                    |
| **리프 노드 (Leaf Node)**   | 최하위에 위치하여 더 이상 자식이 없는 노드                |
| **레벨 (Level)**          | 루트 노드(Level 1)부터 시작하여 아래로 내려갈수록 증가하는 단계 |

---

## 2. Oracle 계층형 쿼리 문법

Oracle에서는 `START WITH`와 `CONNECT BY` 절을 사용하여 직관적으로 계층 구조를 표현합니다.

### 핵심 키워드

* **START WITH**: 계층 구조가 시작될 루트 행을 지정합니다.
* **CONNECT BY PRIOR**: 부모와 자식 노드 간의 관계를 정의합니다.
* **PRIOR**: `PRIOR 자식 = 부모` (순방향), `PRIOR 부모 = 자식` (역방향) 형태로 방향을 결정합니다.
* **ORDER SIBLINGS BY**: 계층 구조를 유지하면서 동일 레벨(형제) 내에서만 정렬을 수행합니다.

### 가상 컬럼 및 함수

* **LEVEL**: 현재 노드의 깊이를 나타냅니다.
* **SYS_CONNECT_BY_PATH(컬럼, 구분자)**: 루트부터 현재 노드까지의 경로를 출력합니다.
* **CONNECT_BY_ISLEAF**: 해당 노드가 리프 노드이면 1, 아니면 0을 반환합니다.

## 3. ANSI SQL 계층형 쿼리 (Recursive CTE)

Oracle 외의 DB(MySQL 8.0+, PostgreSQL, SQL Server 등)에서는 **Recursive CTE**를 사용하여 계층 구조를 구현합니다.

1. **Anchor Member**: 비반복 부분 (루트 데이터를 가져오는 쿼리)
2. **Recursive Member**: 반복 부분 (부모와 자식을 조인하여 확장하는 쿼리)
3. **UNION ALL**: 두 부분을 결합

## 4. 예시 (Example)

### 조직도 조회 예시 (Oracle 방식)

상사(MGR)와 사원(EMPNO)의 관계를 가진 직원 테이블에서 조직도를 조회하는 경우입니다.

```sql
SELECT 
    LEVEL "계층깊이(가상테이블)",
    ENAME AS "직원명",
    JOB "직책",
    MGR "관리자 번호"
FROM EMP
START WITH MGR IS NULL -- 사장이 최상위(루트)
CONNECT BY PRIOR EMPNO = MGR -- 현재 사원번호가 다음 행의 상사번호인 경우 연결
ORDER SIBLINGS BY ENAME;

```

### 카테고리 경로 출력 예시 (ANSI 방식)

```sql
WITH RECURSIVE CategoryTree AS (
    -- 1. 루트 카테고리 선택
    SELECT id, name, parent_id, name AS path
    FROM categories
    WHERE parent_id IS NULL
    
    UNION ALL
    
    -- 2. 자식 카테고리를 반복적으로 조인
    SELECT c.id, c.name, c.parent_id, CONCAT(ct.path, ' > ', c.name)
    FROM categories c
    INNER JOIN CategoryTree ct ON c.parent_id = ct.id
)
SELECT * FROM CategoryTree;

```

## 5. 주의사항 및 팁

* **무한 루프(Cycle) 방지**: 데이터 구조상 자식이 다시 부모가 되는 순환 구조가 있다면 쿼리가 멈추지 않습니다. Oracle에서는 `NOCYCLE` 키워드를 사용하여 이를 방지할 수 있습니다.
* **성능 최적화**: 계층형 쿼리는 조인이 반복적으로 발생하므로, 연결 조건이 되는 컬럼(부모 ID, 자식 ID)에 **인덱스**가 설정되어 있어야 성능 저하를 막을 수 있습니다.

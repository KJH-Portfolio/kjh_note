# 📘 DML - INSERT (데이터 삽입)

## Summary 📝

> [!summary] 핵심 요약
>
>   * **INSERT**: 테이블에 행(Row) 추가.
>   * **유형**: 전체 컬럼 삽입, 지정 컬럼 삽입, 서브쿼리 삽입.
>   * **INSERT ALL**: 하나의 쿼리로 여러 테이블에 동시 삽입 (Oracle).
>   * **오류 주의**: 값의 개수와 자료형이 일치해야 함.

## 1\. 기본 삽입 문법 📥

### 1.1 전체 vs 지정 컬럼

| 유형 | 문법 | 특징 및 주의사항 |
| :--- | :--- | :--- |
| **전체 컬럼** | `INSERT INTO 테이블 VALUES (값1, ...);` | 컬럼 순서/개수 완벽 일치 필수.<br>값 부족 시: `NOT ENOUGH VALUES`<br>값 초과 시: `TOO MANY VALUES` |
| **지정 컬럼** | `INSERT INTO 테이블 (A, B) VALUES (값A, 값B);` | 생략된 컬럼은 `NULL` 또는 `DEFAULT` 값 자동 입력.<br>`NOT NULL` 컬럼 생략 시 오류 발생. |

### 1.2 서브쿼리 INSERT

`VALUES` 대신 `SELECT` 결과를 대량으로 삽입합니다.

```sql
INSERT INTO 테이블명 (컬럼1, 컬럼2)
SELECT 컬럼A, 컬럼B FROM 원본테이블 WHERE 조건;
```

## 2\. INSERT ALL (다중 테이블 삽입) 🔀

서브쿼리 결과를 한 번에 여러 테이블에 나누어 넣습니다.

### 2.1 Unconditional (단순)

조건 없이 지정된 모든 테이블에 데이터를 넣습니다.

```sql
INSERT ALL
  INTO 테이블A VALUES (컬럼1, 컬럼2)
  INTO 테이블B VALUES (컬럼1, 컬럼3)
SELECT 컬럼1, 컬럼2, 컬럼3 FROM 원본테이블;
```

### 2.2 Conditional (조건부)

`WHEN` 조건을 사용하여 분기 처리합니다.

```sql
INSERT ALL
  WHEN SALARY >= 5000 THEN
    INTO HIGH_SALARY_EMP VALUES (EMP_ID, SALARY)
  WHEN SALARY < 5000 THEN
    INTO LOW_SALARY_EMP VALUES (EMP_ID, SALARY)
SELECT EMP_ID, SALARY FROM EMPLOYEE;
```

# 📘 DML - SELECT (데이터 조회)

## Summary 📝

> [!summary] 핵심 요약
>
>   * **SELECT**: 데이터베이스 조회 구문.
>   * **작성 순서**: SELECT → FROM → JOIN → WHERE → GROUP BY → HAVING → ORDER BY
>   * **실행 순서**: **FROM/JOIN** → **WHERE** → **GROUP BY** → **HAVING** → **SELECT** → **ORDER BY**
>   * **별칭(Alias)**: 컬럼명이나 연산식에 이름 부여 (SELECT, ORDER BY에서 사용 가능).
>   * **기능**: 산술 연산, 연결(`||`), 중복 제거(`DISTINCT`), 리터럴 사용.

## 1\. 문법 구조 및 순서 🏗️

요청하신 **JOIN 절**을 포함한 전체 작성 구조입니다.

```sql
SELECT [DISTINCT] 컬럼명 [AS 별칭], ...
  FROM 테이블명
 [JOIN 연결테이블명 ON 연결조건]
 [WHERE 필터링조건]
 [GROUP BY 그룹화컬럼]
 [HAVING 그룹조건]
 [ORDER BY 정렬기준 [ASC|DESC] [NULLS FIRST|LAST]];
```

### 1.1 실행 순서 (Parsing Order) 🚀

SQL 엔진이 해석하는 순서입니다. (별칭 사용 가능 여부의 기준)

1. **FROM / JOIN**: 테이블 가져오기 및 연결
2. **WHERE**: 행(Row) 필터링 (**별칭 사용 불가**)
3. **GROUP BY**: 그룹핑
4. **HAVING**: 그룹 필터링
5. **SELECT**: 데이터 추출 및 별칭(Alias) 할당
6. **ORDER BY**: 정렬 (**별칭 및 컬럼 순번 사용 가능**)

## 2\. 주요 기능 및 연산자 🧮

### 2.1 연산자 우선순위

| 순위 | 연산자 | 설명 |
| :---: | :--- | :--- |
| **0** | `()` | 괄호 |
| **1** | `+ - * /` | 산술 연산자 |
| **2** | `||` | 연결 연산자 (문자열 결합) |
| **3** | `= != <> > <` | 비교 연산자 |
| **4** | `IS NULL, LIKE, IN` | SQL 비교 연산자 |
| **5** | `BETWEEN AND` | 범위 연산자 |
| **6** | `NOT` | 논리 부정 |
| **7** | `AND` | 논리 곱 |
| **8** | `OR` | 논리 합 |

### 2.2 정렬 (ORDER BY)

  * **컬럼명**: `ORDER BY SALARY DESC`
  * **별칭**: `ORDER BY 연봉 DESC`
  * **컬럼 순번**: `ORDER BY 2 ASC` (SELECT 절의 2번째 컬럼 기준)
  * **임의의 방식대로 정렬**(CASE WHEN... THEN... ELSE... END ...)
  ```sql
  SELECT MEMBER_NAME
  FROM MEMBERS
  ORDER BY
	CASE
		WHEN MEMBER_STATUS = '승인대기' THEN 1
		WHEN MEMBER_STATUS = '승인' THEN 2
		ELSE 3
	END ASC;

  ```

## 예시 📌

```sql
SELECT 
    E.EMP_NAME AS 이름, 
    D.DEPT_TITLE AS 부서명, 
    E.SALARY * 12 AS 연봉
FROM EMPLOYEE E
JOIN DEPARTMENT D ON E.DEPT_CODE = D.DEPT_ID
WHERE E.SALARY >= 3000000
ORDER BY 3 DESC; -- 3번째 컬럼(연봉) 기준으로 내림차순 정렬
```

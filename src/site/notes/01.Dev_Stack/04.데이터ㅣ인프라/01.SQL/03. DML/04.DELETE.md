# 📘 DML - DELETE (데이터 삭제)

## Summary 📝

> [!summary] 핵심 요약
>
>   * **DELETE**: 행(Row) 단위 데이터 삭제.
>   * **WHERE**: 삭제할 대상 지정 (생략 시 **전체 삭제**).
>   * **복구**: DML이므로 `ROLLBACK` 가능.
>   * **효율성**: 대량/전체 삭제 시 `TRUNCATE`가 더 빠름.

## 1\. 문법 🗑️

```sql
DELETE FROM 테이블명
 [WHERE 조건];
```

## 2\. 삭제 시 고려사항 🚧

| 구분 | DELETE | TRUNCATE (비교) |
| :--- | :--- | :--- |
| **동작** | 조건에 맞는 행 삭제 (로그 기록함) | 테이블 전체 데이터 초기화 (로그 기록 안 함) |
| **속도** | 느림 | 빠름 |
| **복구** | `ROLLBACK` 가능 | 불가능 (Auto Commit) |
| **제약조건** | 자식 테이블이 참조 중이면 삭제 실패<br>(ON DELETE 옵션 확인 필요) | 참조 중이면 실행 불가 |

## 예시 📌

```sql
-- 특정 사원 삭제
DELETE FROM EMP WHERE EMP_ID = 1001;

-- 전체 삭제 (주의)
DELETE FROM EMP; 
```

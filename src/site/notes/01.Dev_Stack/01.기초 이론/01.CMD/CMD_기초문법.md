# CMD 주요 명령어 및 구문 가이드
> [!summary] 핵심 요약
> 
> - Windows 명령 프롬프트(CMD) 환경에서의 시스템 제어 및 파일 관리를 위한 기본 구문 안내
>     
> - 목적: GUI를 통하지 않는 빠른 시스템 제어, 자동화 스크립트(Batch) 작성 기반 마련
>     
> - 핵심: ==경로 이동(`cd`)== 및 ==파일 목록 확인(`dir`)== 숙달
>     
## 1. 기본 탐색 및 경로 제어
- **경로 지정 규칙**
    - 공백이 포함된 폴더명이나 파일명은 반드시 `"{경로}"`와 같이 ==큰따옴표로 감싸서 입력==함.
    - 대소문자를 구분하지 않으나, 가독성을 위해 표준 명칭 사용 권장.

| **명령어** | **기능 요약**                 | **비고**                        |
| ------- | ------------------------- | ----------------------------- |
| `cd`    | 현재 디렉토리 위치 변경             | `cd ..` 입력 시 상위 폴더 이동         |
| `dir`   | 현재 디렉토리의 파일 및 하위 폴더 목록 출력 | `/w`(와이드), `/p`(페이지 단위) 옵션 존재 |
| `cls`   | 명령 프롬프트 화면의 모든 텍스트 삭제     | 화면 정리용                        |
| `exit`  | CMD 창 종료                  | -                             |

```plaintext
Directory Navigation Structure
 ├─ C:\ (Root)
 │   └─ Users
 │       └─ Admin (Current)
 │           ├─ Documents
 │           └─ Downloads
 └─ D:\ (External Drive)
     └─ =d: 입력 시 드라이브 즉시 변경
```
## 2. 파일 및 디렉토리 관리
- **파일 조작 주의사항**
    - `del` 명령어 사용 시 ==복구 불가능==하므로 삭제 전 대상 재확인 필수.
    - 와일드카드(`*`) 활용 시 대량의 파일이 의도치 않게 조작될 위험 존재.

|**명령어**|**기능 요약**|**주요 옵션**|
|---|---|---|
|`mkdir` (md)|새 디렉토리(폴더) 생성|-|
|`rmdir` (rd)|디렉토리 삭제|`/s` 포함 시 하위 내용 전체 삭제|
|`copy`|파일 복사|-|
|`move`|파일 이동 또는 이름 변경|-|
|`ren`|파일/폴더 이름 변경|-|
|`del`|파일 삭제|`/q` (확인 메시지 생략)|

```dos
// 예시: 실무 파일 관리 프로세스
// 1. temp 폴더 생성
mkdir C:\temp

// 2. 현재 경로의 모든 .log 파일을 temp로 이동
move *.log C:\temp

// 3. temp 폴더 내의 모든 파일 강제 삭제
del /q C:\temp\*.*
```
## 3. 시스템 정보 및 네트워크 확인
- **네트워크 진단**
    - 연결성 테스트 시 가장 먼저 수행하는 기본 단계.

|**명령어**|**기능 요약**|**활용 예시**|
|---|---|---|
|`ipconfig`|네트워크 어댑터 정보(IP 주소 등) 확인|`/all` (상세 정보)|
|`ping`|대상 서버와의 연결 상태 및 응답 시간 확인|`ping 8.8.8.8`|
|`systeminfo`|OS 버전, RAM, 프로세서 등 시스템 사양 출력|-|
|`tasklist`|현재 실행 중인 프로세스 목록 출력|-|
|`taskkill`|실행 중인 프로세스 강제 종료|`/f` (강제), `/im` (이미지 이름)|

## 4. 파이프 및 리다이렉션 (특수 연산자)
- **데이터 흐름 제어**
    - 결과값을 파일로 저장하거나 다른 명령어의 입력값으로 전달.

|**기호**|**기능**|**설명**|
|---|---|---|
|`>`|출력 리다이렉션 (덮어쓰기)|명령어 결과를 파일로 저장|
|`>>`|출력 리다이렉션 (추가하기)|기존 파일 내용 끝에 결과 추가|
|`\|`|파이프 (Pipe)|앞 명령어의 출력을 뒤 명령어의 입력으로 전달|
|`&`|순차 실행|앞 명령어 실행 후 즉시 뒤 명령어 실행|

```plaintext
Data Flow Logic
 [Command A] --(Output)--> | Pipe | --(Input)--> [Command B]
  (예: tasklist | find "chrome")
```

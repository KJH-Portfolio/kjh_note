# 📂 컨텍스트 루트 (Context Root) 가져오기

> **핵심 요약**
> \*\*"현재 프로젝트의 기본 경로"\*\*를 동적으로 가져오는 방법입니다. (예: `/spring`, `/myapp`)
> 이미지, CSS, JS 파일 로딩이나 페이지 이동(Link) 시 **404 에러를 방지하기 위해 필수**입니다.

-----


## 1️⃣ Java (Controller/Service)에서 사용

| 메서드 | 반환 타입 | 설명 |
| :--- | :--- | :--- |
| **`request.getContextPath()`** | `String` | 현재 프로젝트의 루트 경로(Context Path)를 반환합니다. |

### 📝 예제 코드

```java
// Controller 메서드 내부
public String method(HttpServletRequest request) {
    
    // 1. 경로 가져오기
    String contextPath = request.getContextPath();
    System.out.println("현재 프로젝트 경로: " + contextPath); // 결과 예시: /spring
    
    // 2. 활용 예시 (리다이렉트 등)
    // 참고: Spring에서 return "redirect:/..." 사용 시 자동으로 붙여주기도 함
    return "redirect:" + contextPath + "/list.bo"; 
}
```

-----

## 2️⃣ JSP (View 화면)에서 사용 [⭐ 가장 중요]

JSP에서는 \*\*EL(Expression Language)\*\*을 사용하는 것이 표준이며 가장 깔끔합니다.

| 문법 | 설명 |
| :--- | :--- |
| **`${pageContext.request.contextPath}`** | `request.getContextPath()`의 JSP EL 표현식입니다. |

### 💡 실무 권장 패턴 (`c:set` 활용)

매번 긴 EL 표현식을 쓰기 번거로우므로, 보통 `header.jsp`나 `menubar.jsp` 상단에 변수로 선언해두고 재사용합니다.

```ejs
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<c:set var="contextRoot" value="${pageContext.request.contextPath}" />

<a href="${contextRoot}/list.bo">게시판 가기</a>

<img src="${contextRoot}/resources/images/logo.png">
<link rel="stylesheet" href="${contextRoot}/resources/css/style.css">
<script src="${contextRoot}/resources/js/main.js"></script>
```

-----

## 3️⃣ JavaScript (JS 파일)에서 사용

외부 JS 파일(`.js`)은 **서버의 정보(Session, Request 등)를 직접 읽을 수 없습니다.** 따라서 JSP에서 변수를 만들어 넘겨주는 **"다리(Bridge)"** 역할이 필요합니다.

### ✅ 해결 방법: 전역 변수 할당

**STEP 1. JSP 파일 (데이터 주입)**
JS 파일이 로드되기 **전**에, 스크립트 태그를 열어 전역 변수를 선언합니다.

```ejs
<script>
    // JSP가 실행될 때 서버 값을 JS 변수(전역)에 담아줍니다.
    const contextPath = "${pageContext.request.contextPath}";
</script>

<script src="${pageContext.request.contextPath}/resources/js/myScript.js"></script>
```

**STEP 2. JS 파일 (사용)**
이제 외부 JS 파일에서도 `contextPath` 변수를 사용할 수 있습니다.

```javascript
/* resources/js/myScript.js */

function goDetail(bno) {
    // JSP에서 선언한 전역 변수 사용 가능
    location.href = contextPath + "/detail.bo?bno=" + bno;
}
```

## 🚀 요약: 상황별 Best Practice

이 3가지만 구분해서 사용하면, 나중에 **프로젝트 이름이 바뀌거나 서버 설정이 변경되어도 경로가 깨지지 않습니다.**

| 사용 위치 | 추천 코드 (Syntax) | 비고 |
| :--- | :--- | :--- |
| **Java** | `request.getContextPath()` | `HttpServletRequest` 필요 |
| **JSP** | `${pageContext.request.contextPath}` | `<c:set>`으로 변수화 추천 |
| **JavaScript** | `contextPath` (변수) | JSP에서 미리 전역 변수로 선언 필요 |

# Spring MVC 파일 업로드 및 다운로드 처리

> [!summary] 핵심 요약
> **Spring MVC**에서 파일을 처리하기 위해서는 **MultipartResolver** 설정과 `multipart/form-data` 인코딩이 필수적입니다.
> 클라이언트가 전송한 바이너리 데이터는 컨트롤러에서 **MultipartFile** 객체로 주입받아 처리하며, 파일명 중복 방지를 위한 **Renaming** 정책과 서버의 물리적 경로(**RealPath**) 확보가 구현의 핵심입니다.

## 1\. 사전 설정 및 이론 (Prerequisites)

파일 업로드를 구현하기 위해 반드시 선행되어야 할 이론적 배경과 설정입니다.

### 1-1. 필수 라이브러리 및 설정

  * **Dependency (`pom.xml`):** 파일 업로드를 처리하기 위해 아래 두 가지 라이브러리가 필요합니다.
      * `commons-fileupload`: 파일 업로드 기능 구현 라이브러리.
      * `commons-io`: 입출력 처리를 위한 유틸리티 라이브러리.
  * **MultipartResolver 등록:** `servlet-context.xml`에 빈(Bean)으로 등록해야 합니다. HTTP의 Multipart 요청을 파싱하여 컨트롤러에 `MultipartFile` 객체로 전달하는 역할을 합니다.
  * **Form 태그 설정:** `<form>` 태그의 `enctype` 속성을 반드시 `multipart/form-data`로 지정해야 바이너리 데이터 전송이 가능합니다.
  * `accept="image/*" required multiple`로 파일 형식과 다중IO인지 설정
  * 서버 우클릭후 설정에서 `Serve modules without publishing` 체크하기 - 팀 프로젝트를 할 때 파일도 같이 공유하는 설정(필요하지 않다면x)

### 1-2. 핵심 인터페이스: MultipartFile

Spring에서 업로드된 파일을 다루기 위해 제공하는 인터페이스입니다. 컨트롤러의 매개변수로 선언하여 사용합니다.

| 메서드명                        | 설명                              | 비고            |
| :-------------------------- | :------------------------------ | :------------ |
| **`getOriginalFilename()`** | 업로드한 파일의 원본 이름 반환               | 확장자 포함        |
| **`transferTo(File dest)`** | 업로드된 파일을 지정된 경로의 파일(`dest`)로 저장 | **실제 저장 수행**  |
| **`isEmpty()`**             | 파일이 업로드되지 않았는지 확인               | `true`: 파일 없음 |
| **`getSize()`**             | 파일의 크기(Byte) 반환                 | 용량 제한 체크 시 사용 |

## 2\. 서버 경로 처리 (ServletContext)

웹 애플리케이션은 OS 상의 실제 경로와 웹상에서의 경로가 다릅니다. 파일 저장을 위해서는 **물리적인 절대 경로**가 필요합니다.

  * **`session.getServletContext()`**
      * 현재 실행 중인 웹 애플리케이션(Context)의 전역 정보에 접근합니다.
  * **`.getRealPath("resources/uploadFiles/")`**
      * 웹 경로(`/resources/...`)를 서버의 **물리적 절대 경로**(예: `C:\workspace\.metadata\...\webapp\resources\...`)로 변환합니다.
      * 파일 입출력(`File` 객체 생성) 시에는 반드시 이 절대 경로를 사용해야 합니다.

## 3\. 파일 업로드 구현 로직

### 3-1. 컨트롤러 처리 흐름 (`@PostMapping`)

1. **매개변수 수신:** `MultipartFile uploadFile` 객체를 통해 파일 데이터를 받습니다. (파일이 없어도 객체는 생성되므로 `null`이 아님)
2. **유효성 검사:** `uploadFile.getOriginalFilename()`이 빈 문자열인지 확인하여 실제 파일 첨부 여부를 판단합니다.
3. **파일 저장 (Renaming):** 중복 방지를 위해 파일명을 변경하여 서버에 저장합니다 (`saveFile` 메서드 위임).
4. **DB 데이터 처리:** 원본 파일명(`OriginName`)과 변경된 파일명(`ChangeName`)을 `Board` 객체에 세팅(Set) 후 Service로 전달합니다.

### 3-2. 파일명 변경 전략 (Renaming Policy)

사용자가 올린 파일명이 중복되어 덮어쓰기 되는 것을 방지하기 위해 고유한 이름으로 변경합니다.

  * **조합 방식:** `생성시간(년월일시분초)` + `랜덤값(5자리)` + `원본확장자`
  * **코드 예시:** `20231215093000` + `12345` + `.jpg`

## 4\. 파일 다운로드 및 삭제

### 4-1. 파일 다운로드 (View)

HTML5의 `download` 속성을 사용하여 간단하게 구현할 수 있습니다.

```html
<a href="${contextRoot}${detail.changeName}" download="${detail.originName}">
    ${detail.originName}
</a>
```

### 4-2. 파일 삭제 로직

DB 데이터 삭제뿐만 아니라, **서버에 저장된 실제 파일**도 삭제해야 합니다.

1. DB에서 해당 게시글(`bno`)의 파일 정보를 조회합니다.
2. `getRealPath`로 저장 경로를 구합니다.
3. `new File(경로 + 파일명).delete()` 메서드로 실제 파일을 삭제합니다.

-----
## 5. 여러 파일 등록/조회
- **List 처리** : `List<MultipartFile>`로 여러 파일 한번에 받기
- **시퀀스 선발급** : 시퀀스를 미리 조회해서 게시글과 첨부파일에 각각 넣어주기
- **트랜잭션 처리** : 모두 성공했을때만 commit 처리
## 6. 코드 예시
#### 파일 입출력

```java
@Controller
public class BoardController {

    // [페이지 이동] 글 작성 폼
    @GetMapping("/insert.bo")
    public String boardEnroll() {
        return "board/boardEnrollForm";
    }

    // [요청 처리] 글 작성 및 파일 업로드
    @PostMapping("/insert.bo")
    public String boardInsert(Board b, 
                              MultipartFile uploadFile, 
                              HttpSession session) {
        
        // 1. 첨부파일 유무 확인 (파일명 존재 여부로 판단)
        // MultipartFile 객체는 파일이 없어도 생성되므로 null 체크가 아닌 파일명 체크 필요
        if (!uploadFile.getOriginalFilename().equals("")) {
            
            // 2. 파일 저장 및 이름 변경 (Helper 메서드 호출)
            String changeName = saveFile(session, uploadFile);
            
            // 3. DB 저장을 위한 VO 객체 세팅
            b.setOriginName(uploadFile.getOriginalFilename());
            b.setChangeName("/resources/uploadFiles/" + changeName); // 웹 접근 경로 포함
        }

        // 4. Service 호출
        int result = service.boardInsert(b);

        if (result > 0) {
            session.setAttribute("alertMsg", "게시글 등록 성공!");
        } else {
            session.setAttribute("alertMsg", "게시글 등록 실패!");
        }
        
        return "redirect:list.bo";
    }

    // [Helper] 실제 파일 저장 및 리네임 처리 메서드
    public String saveFile(HttpSession session, MultipartFile uploadFile) {
        
        // 1. 저장 경로 획득 (물리적 절대 경로)
        String savePath = session.getServletContext().getRealPath("resources/uploadFiles/");
        
        // 2. 원본 파일명 및 확장자 추출
        String originName = uploadFile.getOriginalFilename();
        String ext = originName.substring(originName.lastIndexOf("."));
        
        // 3. 리네임 규칙 생성 (시간 + 랜덤수)
        String currentTime = new SimpleDateFormat("yyyyMMddHHmmss").format(new Date());
        int ranNum = (int)(Math.random() * 90000 + 10000);
        
        // 4. 변경된 파일명 조합
        String changeName = currentTime + ranNum + ext;

        // 5. 서버에 파일 저장 (transferTo)
        try {
            uploadFile.transferTo(new File(savePath + changeName));
        } catch (IllegalStateException | IOException e) {
            e.printStackTrace();
        }
        
        return changeName;
    }
    
    // [기능] 파일 삭제 (게시글 삭제 시 호출 가정)
    public void deleteFile(String bno, HttpSession session) {
        // 1. 기존 게시글 정보 조회
        Board b = service.boardDetail(bno);
        
        // 2. 경로 및 파일명 추출
        String savePath = session.getServletContext().getRealPath("resources/uploadFiles/");
        String changeName = b.getChangeName();
        // DB에 경로가 포함되어 있다면 파일명만 추출
        String fileName = changeName.substring(changeName.lastIndexOf("/") + 1);
        
        // 3. 실제 파일 삭제 수행
        File file = new File(savePath + fileName);
        if(file.exists()) {
            file.delete();
        }
    }
}
```

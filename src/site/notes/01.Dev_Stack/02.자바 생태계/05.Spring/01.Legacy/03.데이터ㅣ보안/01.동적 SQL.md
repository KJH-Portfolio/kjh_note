# MyBatis 고급 매핑: 동적 SQL & SelectKey

> [!summary] 핵심 요약
>
>   * **동적 SQL:** 실행 시점의 파라미터 조건에 따라 SQL 구문을 동적으로 변경하는 기능입니다. JSTL이나 EL이 아닌 **MyBatis 고유 태그**(`if`, `choose` 등)를 사용합니다.
>   * **SelectKey:** `INSERT` 수행 전후에 PK값(Sequence)을 조회하여, 자바 객체(VO)에 즉시 바인딩할 때 사용합니다.

## 1\. 동적 SQL (Dynamic SQL)

정적 쿼리 하나로 처리하기 힘든 **다중 검색 조건**이나 **선택적 업데이트** 등을 처리할 때 사용합니다.

### 1-1. `<choose>, <when>, <otherwise>`

Java의 `switch-case` 문과 유사하게 동작합니다. 여러 조건 중 하나만 실행됩니다.

  * **문법 특징:** EL(`${}`)이 아닌 OGNL 표기법을 사용하므로 `test` 속성 안에서는 `#{}` 없이 변수명을 바로 사용합니다.
  * **주의:** 문자열 비교 시 `'값'` (작은따옴표) 형식을 준수해야 합니다.

```xml
<select id="searchListCount" resultType="_int">
    SELECT COUNT(*)
    FROM BOARD
    <choose>
        <when test="condition == 'title'">
            WHERE BOARD_TITLE LIKE '%' || #{keyword} || '%'
        </when>
        <when test="condition == 'writer'">
            WHERE BOARD_WRITER = #{keyword}
        </when>
        <otherwise>
            WHERE BOARD_CONTENT LIKE '%' || #{keyword} || '%'
        </otherwise>
    </choose>
    AND STATUS = 'Y'
</select>
```

> [!TIP] Oracle 연결 연산자
> Oracle에서는 문자열 연결 시 `||`를 사용합니다. (MySQL은 `CONCAT` 함수 사용)
>
>   * `'%' || #{keyword} || '%'`  → `'%키워드%'`
#### 와일드카드 문법

| **기호**  | **의미**         | **예시**      | **설명**                         |
| ------- | -------------- | ----------- | ------------------------------ |
| **`%`** | **0개 이상의 문자**  | `LIKE '김%'` | '김', '김철수', '김태희' 모두 매칭        |
| **`_`** | **정확히 1개의 문자** | `LIKE '김_'` | '김현', '김철' (2글자만 가능, '김철수' 불가) |

## 2. SelectKey (기본키 및 시퀀스 제어)

`INSERT` 쿼리 실행 시, 자동 생성되는 키(Sequence) 값을 **쿼리 실행 전(BEFORE)이나 후(AFTER)** 에 가져와서 파라미터 객체(VO)에 세팅해 줍니다.

### 2-1. 주요 속성

| 속성 | 설명 |
| :--- | :--- |
| **`keyProperty`** | 조회한 값을 저장할 **VO 객체의 필드명** (예: `boardNo`, `refBno`) |
| **`resultType`** | 조회 결과의 데이터 타입 (예: `int`, `string`) |
| **`order`** | **`BEFORE`**: 메인 쿼리 실행 **전**에 조회<br>**`AFTER`**: 메인 쿼리 실행 **후**에 조회 |

### 2-2. 적용 예시 (첨부파일 등록)

첨부파일 등록 시 현재 게시글의 번호(`CURRVAL`)를 알아내어 `refBno`에 세팅하는 로직입니다.

```xml
<insert id="insertAttachmentList" parameterType="Attachment">
    <selectKey keyProperty="refBno" resultType="int" order="BEFORE">
        SELECT SEQ_BNO.CURRVAL FROM DUAL
    </selectKey>

    INSERT INTO ATTACHMENT (
        FILE_NO, 
        REF_BNO, 
        ORIGIN_NAME, 
        CHANGE_NAME
    )
    VALUES (
        SEQ_FNO.NEXTVAL, 
        #{refBno}, 
        #{originName}, 
        #{changeName}
    )
</insert>
```

> [!WARNING] SelectKey 사용 시 주의사항
>
>   * **`SEQ.CURRVAL` + `BEFORE` 조합:** 이 조합은 반드시 **같은 트랜잭션(Session) 내에서 게시글이 먼저 INSERT 된 직후**에만 유효합니다. 그렇지 않으면 "세션에 정의되지 않음" 오류나 엉뚱한 번호를 가져올 수 있습니다.
>   * **일반적인 패턴:** 부모(게시글) 등록 시에는 `SEQ.NEXTVAL` + `BEFORE`를 사용하여 ID를 생성하고, 자식(파일) 등록 시에는 자바(Service) 단에서 ID를 넘겨받는 방식이 더 안전합니다.

# Spring MVC 핵심 컴포넌트 및 어노테이션 상세 가이드
> [!summary] 핵심 요약
> 
> - 범용 객체부터 구체적인 계층(Web, Business, Data) 흐름에 따른 어노테이션 논리적 재배치
>     
> - 목적: 스프링 빈(Bean) 라이프사이클 및 계층형 아키텍처의 직관적 이해와 가독성 향상
>     
> - 핵심: ==어노테이션의 레이어(Layer)별 역할 분리== 및 ==데이터 바인딩 전략==
>     
## 계층별 빈(Bean) 등록 어노테이션
> [!info] 범용에서 구체적인 레이어(Layer) 흐름으로의 전개
- **@Component** (가장 기본)
    - 모든 스프링 빈 등록 어노테이션의 뿌리
    - ==특정 계층에 속하지 않는 일반적인 클래스==를 빈으로 등록 시 사용
- **@Controller / @RestController** (Web 계층)
    - 사용자 요청을 가장 먼저 받는 관문 역할
    - 뷰(JSP) 반환 제어(`@Controller`)
    - JSON/XML 형태의 응답 데이터 반환(`@RestController : @Controller + @ResponseBody`)
- **@Service** (Business 계층)
    - 비즈니스 로직이 수행되는 중간 다리 역할
    - ==트랜잭션(@Transactional) 단위 설정==의 주요 계층
- **@Repository** (Data 계층)
    - 데이터베이스에 접근하는 마지막 단계 (DAO 연동)
    - DB 벤더별 예외를 ==스프링의 공통 예외로 추상화하여 변환==하는 특징 보유
- **@Configuration & @Bean** (설정 계층)
    - 클래스 단위 자동 스캔이 아닌, ==개발자가 수동으로 빈을 등록==하는 특수 케이스
    - 주로 외부 라이브러리 객체 등록 및 전역 설정에 활용
    - `@Configuration`안에서만 `@Bean`을 사용할 수 있으며 해당 클래스를 싱글톤 패턴의 프록시 객체로 만듬
```java
@Configuration
public class SecurityConfig {

    @Bean
    public BCryptPasswordEncoder passwordEncoder() {
        // 내가 만든 클래스는 아니지만, 스프링 컨테이너에 등록해서 
        // 어디서든 @Autowired나 생성자 주입으로 쓰고 싶을 때!
        return new BCryptPasswordEncoder();
    }
}
```
## 제어 및 데이터 통신용 어노테이션
> [!tip] 역할군에 따른 기능별 그룹화
- **의존성 주입 (DI)**
    - **@Autowired**: 타입(Type) 일치 기반 ==객체 자동 주입== (생성자/필드/세터 활용)
    - **@Value**: 설정 파일(`.properties`, `.yml`)의 특정 값을 변수에 주입
- **요청 및 라우팅 매핑**
    - **@RequestMapping**: 특정 URL 패턴과 컨트롤러 메서드 매핑 (클래스 레벨 전역 경로 설정 포함)
    - **@GetMapping / @PostMapping**: HTTP Method(GET/POST) 전용 매핑
- **데이터 입출력 처리**
    - **@RequestBody**: HTTP 요청 본문(JSON 등)을 ==자바 객체로 역직렬화==하여 수신
    - **@ResponseBody**: 메서드 반환값을 HTTP 응답 본문(Body)으로 직접 전송 (JSON 직렬화)
    - **@PathVariable**: URL 경로상에 포함된 동적 변수값 추출
- **기타 제어**
    - **@Transactional**: 선언적 트랜잭션 관리 (정상 종료 시 커밋, 예외 발생 시 롤백)
    - **@Scheduler**: 주기적 작업 실행 예약 ([[3. Resource/03.Dev_Stack/02.자바 생태계/05.Spring/01.Legacy/05.고급 기능ㅣ기타/01.Scheduler ㅣ Cron|01.Scheduler ㅣ Cron]])
```java
@RestController
@RequestMapping("/api/v1")
public class RestSampleController {
    
    @GetMapping("/user/{id}")
    public User getUser(@PathVariable("id") String id) {
        return userService.findById(id);
    }

    @PostMapping("/save")
    public String saveUser(@RequestBody User user) {
        userService.save(user);
        return "success";
    }
}
```

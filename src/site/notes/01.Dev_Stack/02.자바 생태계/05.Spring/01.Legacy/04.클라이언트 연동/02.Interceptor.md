# 🚀 Spring Interceptor (인터셉터)
> [!summary] 요약
> **Interceptor**는 `DispatcherServlet`과 `Controller` 사이에서 요청(Request)을 **가로채는(Intercept)** 역할을 수행함.
>
>   * **주목적**: 로그인 체크(권한 관리), 로그 기록, 공통 트랜잭션 처리 등.
>   * **구현**: `HandlerInterceptor` 인터페이스 구현 + `servlet-context.xml`에 등록.
## 1. 개념 및 동작 원리
### A. 필터(Filter) vs 인터셉터(Interceptor) 비교
두 기술 모두 요청을 가로채지만 **실행 시점과 관할 영역**이 다름.

| 구분 | 필터 (Filter) | 인터셉터 (Interceptor) |
| :--- | :--- | :--- |
| **관할 영역** | **Web Container** (Tomcat) | **Spring Container** |
| **작동 시점** | DispatcherServlet **진입 전** | DispatcherServlet **이후**, Controller **진입 전/후** |
| **주요 용도** | 인코딩 변환, XSS 방어 (전역 보안) | 인증(로그인), 권한 체크, API 로깅 (비즈니스 밀접) |
| **조작 객체** | `ServletRequest`, `ServletResponse` | `HttpServletRequest`, `ModelAndView` |
### B. 3단계 간섭 시점 (Lifecycle)
`HandlerInterceptor` 인터페이스의 메서드 오버라이딩을 통해 제어함.
1. **`preHandle` (요청 처리 전)**
      * **시점**: 컨트롤러 실행 직전.
      * **역할**: 로그인 여부 확인 등 **진입 차단(Gatekeeper)**.
      * **리턴**: `true` (진행), `false` (중단).
2. **`postHandle` (요청 처리 후)**
      * **시점**: 컨트롤러 수행 후, 뷰(JSP) 렌더링 전.
      * **역할**: 컨트롤러가 넘겨준 데이터(`ModelAndView`) 가공.
3. **`afterCompletion` (뷰 렌더링 후)**
      * **시점**: 모든 작업 완료 후 (사용자에게 응답 직전).
      * **역할**: 리소스 정리, 예외 로그 처리.

## 2\. Java 구현 (HandlerInterceptor)
실제 인터셉터 클래스를 작성하는 코드임. (제공해주신 `TestInterceptor` 예시)
```java
public class TestInterceptor implements HandlerInterceptor {

    // 1. 전처리 (Controller 실행 전)
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("==== [1] 요청 처리 전 (preHandle) ====");
        
        // [실습 로직] 로그인 체크 예시
        HttpSession session = request.getSession();
        if(session.getAttribute("loginMember") == null) {
            session.setAttribute("msg", "로그인 후 이용 가능합니다.");
            response.sendRedirect(request.getContextPath() + "/login.me");
            return false; // 컨트롤러 실행 차단
        }
        
        return true; // 정상 진행
    }

    // 2. 후처리 (Controller 실행 후, View 렌더링 전)
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("==== [2] 요청 처리 후 (postHandle) ====");
        
        // View 데이터 조작 가능 (예: 에러 발생 시 뷰 페이지 변경)
        if(modelAndView != null && modelAndView.getModel().get("list") == null) {
            // modelAndView.setViewName("common/errorPage");
        }
    }

    // 3. 완료 처리 (View 렌더링 끝난 후)
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        System.out.println("==== [3] 최종 완료 (afterCompletion) ====");
        
        // 예외 발생 여부 확인 가능
        if (ex != null) {
            System.out.println("Exception 발생: " + ex.getMessage());
        }
    }
}
```

## 3\. 설정 및 등록 (servlet-context.xml)
작성한 인터셉터 클래스를 스프링이 인식하고, **어떤 URL**에 적용할지 설정하는 파일임.
### A. 인터셉터 매핑 설정 (`<interceptors>`)
`<interceptor>` 태그 내부에 **적용할 경로**와 **제외할 경로**, \*\*실행할 빈(Class)\*\*을 정의함.
```xml
<interceptors>
    <interceptor>
        <mapping path="/*.me"/>
        <exclude-mapping path="/enrollForm.me"/>
        <beans:bean class="com.kh.spring.common.interceptor.LoginInterceptor"/>
    </interceptor>
</interceptors>
```
### 부록 : servlet-context 기타 설정
- % (`servlet-context.xml`)에 주로 포함되는 설정들

| 설정 항목                 | 설명 및 주요 속성                                                                     |
| :-------------------- | :----------------------------------------------------------------------------- |
| **ViewResolver**      | Controller가 반환한 문자열을 실제 JSP 경로로 변환 (`prefix: /WEB-INF/views/`, `suffix: .jsp`) |
| **Resources**         | CSS, JS, 이미지 등 정적 파일 경로 매핑 (`/resources/**`)                                   |
| **MultipartResolver** | **파일 업로드** 처리를 위한 설정 (ID: `multipartResolver` 필수)                              |
> [!TIP] 파일 업로드 설정 값 상세
>
>   * **`defaultEncoding`**: 파일명 깨짐 방지를 위한 인코딩 (`UTF-8`).
>   * **`maxUploadSize`**: 전체 요청의 최대 업로드 크기 (약 100MB).
>   * **`maxInMemorySize`**: 디스크에 쓰기 전 메모리에 저장할 최대 크기.
```xml
<beans:bean class="org.springframework.web.multipart.commons.CommonsMultipartResolver" id="multipartResolver">
    <beans:property name="defaultEncoding" value="UTF-8"/>
    <beans:property name="maxUploadSize" value="100000000"/> </beans:bean>
```

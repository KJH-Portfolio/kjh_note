# Spring WebSocket 채팅 서버 구현 (ChatServer)
> [!summary] 핵심 요약
> 
> ==TextWebSocketHandler==를 상속받아 구현한 실시간 다중 접속 채팅 서버 로직.
> 
> `Map` 자료구조를 활용한 접속 유저 세션 관리 및 실시간 접속자 목록 동기화 처리.
> 
> Gson 객체를 활용한 JSON 파싱과 일반 채팅 및 귓속말 목적지에 따른 메시지 라우팅 구현.
## 세션 관리 로직
- **유저 세션 저장소**
    - `Map<String, WebSocketSession>` 구조를 활용하여 아이디와 세션 정보 매핑 및 저장.
    - `Collections.synchronizedMap`을 적용하여 스레드 안전성(Thread-safety) 확보.
```java
//Map<id,session> 생성해 접속유저들의 정보 담기
private Map<String,WebSocketSession> users = Collections.synchronizedMap(new HashMap<>());
```
## 소켓 연결 처리 (afterConnectionEstablished)
- **접속 유저 등록**
    - WebSocketSession의 속성(Attributes)에서 `loginMember` 객체 추출.
    - 유저 ID를 Key로, Session을 Value로 Map에 등록.
- **접속자 목록 동기화**
    - Map의 `keySet()`을 활용해 현재 접속 중인 유저 목록(Set) 추출.
    - `Map<String, Set<String>>` 형태로 가공 후 Gson을 통해 JSON 문자열로 변환.
    - 변환된 데이터를 `TextMessage` 객체에 담아 모든 접속 유저에게 브로드캐스팅.
```java
//접속시
@Override
public void afterConnectionEstablished(WebSocketSession session) throws Exception {
    
    //접속 유저 등록
    Member loginMember = (Member)session.getAttributes().get("loginMember");
    users.put(loginMember.getUserId(), session);
    
    //userList 데이터 가공
    Set<String> userList = users.keySet(); 
    Map<String,Set<String>> userListMap = new HashMap<>();
    userListMap.put("userList", userList);
    
    //TextMessage 에 JsonString 형태로 데이터 담기
    TextMessage tm = new TextMessage(new Gson().toJson(userListMap));

    //TextMessage 송신 
    for(String user : userList) {
        users.get(user).sendMessage(tm);
    }
}
```
## 메시지 송수신 처리 (handleTextMessage)
- **메시지 파싱 및 데이터 가공**
    - Gson을 활용해 수신된 페이로드(Payload)를 `MessageVO` 객체로 역직렬화.
    - `SimpleDateFormat`을 사용해 현재 시간을 추출하고 `MessageVO`에 시간 데이터 세팅.
    - 가공 완료된 객체를 다시 JSON 형태의 `TextMessage`로 재변환.
- **메시지 라우팅 (일반/귓속말)**
    - 일반 채팅(`type == 1`): Map에 저장된 전체 유저 세션을 순회하며 메시지 일괄 전송.
    - 귓속말(`type != 1`): Map에서 수신자(`otherId`) 검색 후 해당 단일 세션에만 메시지 전송.
    - 수신자 미접속 예외 처리: 대상이 Map에 없을 경우, 발신자 세션에만 안내 메시지 반환.
```java
//메시지를 받았을때 작동
@Override
protected void handleTextMessage(WebSocketSession session, TextMessage message) throws Exception {
    
    //메시지의 원래 형태로 파싱 (MessabeVO)
    Gson gson = new Gson();
    MessageVO mvo = gson.fromJson(message.getPayload(), MessageVO.class);
    //데이터 가공(시간 추가)
    String time = new SimpleDateFormat("HH:mm:ss").format(new Date());
    mvo.setTime(time);
    message = new TextMessage(gson.toJson(mvo));
    
    //일반채팅
    if(mvo.getType()==1) {
        for(Entry<String, WebSocketSession> e : users.entrySet()) {
            e.getValue().sendMessage(message);
        }
    //귓속말
    }else {
        if(users.containsKey(mvo.getOtherId())) {
            users.get(mvo.getOtherId()).sendMessage(message);
        }else {
            mvo.setMessage("상대방이 접속을 종료하였습니다.");
            message = new TextMessage(gson.toJson(mvo));
            session.sendMessage(message);
        }
    }
}
```
## 소켓 종료 처리 (afterConnectionClosed)
- **세션 제거 및 목록 갱신**
    - 접속을 종료한 유저의 ID를 조회하여 Map에서 해당 엔트리(Entry) 삭제.
    - 갱신된 최신 유저 목록을 다시 JSON으로 가공하여 남은 유저들에게 일괄 전송 처리.
```java
//접속 종료시
@Override
public void afterConnectionClosed(WebSocketSession session, CloseStatus status) throws Exception {
    
    //종료한 사용자 삭제
    Member loginMember = (Member)session.getAttributes().get("loginMember");
    users.remove(loginMember.getUserId());
    
    //사용자가 줄어들었으니 목록 갱신후 메시지 전달 
    Set<String> userList = users.keySet();
    
    Map<String,Set<String>> userListMap = new HashMap<>();
    userListMap.put("userList", userList);
    TextMessage tm = new TextMessage(new Gson().toJson(userListMap));

    for(String user : userList) {
        users.get(user).sendMessage(tm);
    }
}
```

### 💡 Note Context
- **수정점:** 가독성 향상을 위해 단일 코드 블록을 해체하고, 각 논리적 흐름(세션 관리, 연결, 송수신, 종료)에 맞추어 설명 하단에 분할 배치 처리.
- **보완점:** `MessageVO` 클래스의 필드 구조 및 `HandshakeInterceptor`를 통한 `loginMember` 세션 주입 로직에 대한 추가 노트 작성 필요.

# Spring Legacy WebSocket 설정 (XML 방식)
> [!summary] 요약
> 
> **Spring Framework** 레거시 환경에서의 **WebSocket** 활성화를 위한 **pom.xml** 의존성 추가 및 **servlet-context.xml** 기반의 핸들러 매핑과 인터셉터 설정 절차.
## 라이브러리 추가
- **spring-websocket**
    - 웹소켓 기능 구현을 위한 핵심 Maven 라이브러리.
```xml
// Maven 의존성 설정
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-websocket</artifactId>
    <version>${org.springframework-version}</version>
</dependency>
```
## Servlet-Context 설정 (XML)
- **Bean 등록**
    - 웹소켓 서버 역할을 수행할 핸들러 클래스의 객체 생성.
- **Handler 매핑**
    - 특정 URL 요청에 대응하는 핸들러 연결 및 CORS 허용 범위 지정.
### 주요 설정 태그 구성

|**태그(Tag)**|**설명**|
|---|---|
|**`beans:bean`**|웹소켓 요청 처리 서버 클래스(Handler)의 빈 등록|
|**`websocket:handlers`**|핸들러 그룹 정의 및 `allowed-origins="*"`를 통한 CORS 문제 방지|
|**`websocket:mapping`**|요청 경로(`path`)와 처리 객체(`handler`) 간의 매핑|
|**`websocket:handshake-interceptors`**|HTTP Session 데이터를 웹소켓 Session으로 전환하기 위한 가로채기 설정|

## 프로토콜 및 주요 개념
- **접근 프로토콜의 변화**
    - 실시간 양방향 통신을 위한 전용 프로토콜 사용.
    - 기존 HTTP: `http://localhost:8080/...`
    - 웹소켓: `ws://localhost:8080/...`
- **Handshake Interceptor**
    - 로그인 사용자 정보 등 기존 웹 세션의 정보를 웹소켓 연결 시점에 복사하여 전달하는 핵심 구성 요소.
## 설정 예시 (servlet-context.xml)
```xml
// servlet-context.xml 웹소켓 구성 예시
<?xml version="1.0" encoding="UTF-8"?>
<beans:beans xmlns="http://www.springframework.org/schema/mvc"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:beans="http://www.springframework.org/schema/beans"
    xmlns:websocket="http://www.springframework.org/schema/websocket"
    xsi:schemaLocation="http://www.springframework.org/schema/mvc 
                        https://www.springframework.org/schema/mvc/spring-mvc.xsd
                        http://www.springframework.org/schema/beans 
                        https://www.springframework.org/schema/beans/spring-beans.xsd
                        http://www.springframework.org/schema/websocket 
                        http://www.springframework.org/schema/websocket/spring-websocket.xsd">

    // 핸들러 빈 등록
    <beans:bean id="basic" class="com.kh.spring.websocket.server.BasicServer"/>
    <beans:bean id="group" class="com.kh.spring.websocket.server.GroupServer"/>
    <beans:bean id="member" class="com.kh.spring.websocket.server.MemberServer"/>

    // 웹소켓 핸들러 및 인터셉터 설정
    <websocket:handlers allowed-origins="*">
        <websocket:mapping handler="basic" path="/basic"/>
        <websocket:mapping handler="group" path="/group"/>
        <websocket:mapping handler="member" path="/member"/>
        <websocket:handshake-interceptors>
            <beans:bean class="org.springframework.web.socket.server.support.HttpSessionHandshakeInterceptor"/>
        </websocket:handshake-interceptors>
    </websocket:handlers>

</beans:beans>
```

### 노트 수정/추가/삭제 사항
- XML 주석 처리 방식을 규칙에 따라 `//`로 변경 적용.
- 불필요한 서술형 어미(~함, ~임 등)를 모두 제거하고 순수 명사형으로 교체.
- 가독성을 위해 주요 설정 태그 표의 레이아웃 조정.
### 노트 수정/추가/삭제 제안
- 각 핸들러 클래스(`BasicServer`, `GroupServer` 등)의 Java 소스 코드 구현 예시 추가 권장.
- 클라이언트 측 JavaScript(`new WebSocket()`) 연결 예시 코드 보완 제안.
### 노트 관련 사항 질문
- 웹소켓 핸들러 구현 시 상속받는 `TextWebSocketHandler`에 대한 상세 노트를 작성하시겠습니까?
- SockJS 사용을 위한 추가 설정 및 라이브러리 관련 노트를 구성하시겠습니까?

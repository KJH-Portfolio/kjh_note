# 스프링 스케줄러 (Spring Scheduler) 설정 및 사용법

> [!summary] 요약
> 스프링에서 주기적인 작업을 자동화하거나 특정 시점에 작업을 예약하는 방법입니다.
>
> 1.  **정적 스케줄링 (`@Scheduled`)**: 설정된 주기(Cron, Delay)에 따라 **반복 실행** (예: 매일 0시 백업).
> 2.  **동적 스케줄링 (`TaskScheduler`)**: 비즈니스 로직 수행 후 **특정 시간 뒤 실행**하거나 **동적으로 주기를 변경**할 때 사용.

## 1\. 환경 설정 (Configuration)

스케줄러가 작동하려면 **설정 파일 등록**이 선행되어야 합니다. 프로젝트 환경(Legacy vs Boot)에 따라 설정합니다.

### 1-1. Legacy Project (XML 설정)

`root-context.xml`에 `task` 네임스페이스를 추가하고 빈을 등록합니다.

**1) Namespaces 추가**
`root-context.xml` 하단 **Namespaces** 탭에서 `task` 체크 또는 아래 코드 추가:

```xml
xmlns:task="http://www.springframework.org/schema/task"
xsi:schemaLocation="http://www.springframework.org/schema/task http://www.springframework.org/schema/task/spring-task-4.3.xsd"
```

**2) 스케줄러 활성화 및 빈 등록**

```xml
<task:annotation-driven/>

<task:scheduler id="taskScheduler" pool-size="10" />

<context:component-scan base-package="com.kh.spring.common.scheduler" />
```

### 1-2. Spring Boot Project (Java 설정)

스프링 부트 환경이라면 메인 클래스에 어노테이션 하나만 추가하면 됩니다.

```java
@EnableScheduling // 이 어노테이션 하나로 설정 완료
@SpringBootApplication
public class Application { 
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

## 2\. 정적 스케줄링 구현 (@Scheduled)

  * **위치**: `@Service` 또는 `@Component` 클래스 메서드 위
  * **특징**: 서버 시작 후 계속 반복됨

### 2-1. 주요 속성 (ms 단위)

| 속성 | 설명 | 예시 |
| :--- | :--- | :--- |
| `initialDelay` | **서버 시작 후** 설정 시간 경과 뒤 첫 실행 | `initialDelay=1000` (1초 뒤 시작) |
| `fixedDelay` | **이전 작업 종료 시점**부터 설정 시간 간격으로 실행 | `fixedDelay=2000` (2초 간격) |
| `cron` | **Cron 표현식**을 사용하여 정교한 시간 설정 | `cron="0 0 * * * *"` (매시 정각) |

### 2-2. Cron 표현식 규칙

  * **구조**: `초 분 시 일 월 요일 (년도)` (총 6\~7필드)

| 순서 | 필드명 | 허용 값 | 특수문자 |
| :---: | :--- | :--- | :--- |
| 1 | **초** | 0 \~ 59 | `*` `-` `/` |
| 2 | **분** | 0 \~ 59 | `*` `-` `/` |
| 3 | **시** | 0 \~ 23 | `*` `-` `/` |
| 4 | **일** | 1 \~ 31 | `*` `-` `/` `?` `L` `W` |
| 5 | **월** | 1 \~ 12 | `*` `-` `/` |
| 6 | **요일** | 1 \~ 7 (1:일 ... 7:토) | `*` `-` `/` `?` `L` `#` |

> [!note] 특수문자 설명
>
>   * `*`: 모든 값 / `?`: 설정 없음 (일/요일 필드) / `/`: 반복 단위 (`0/5`=5초마다)
>   * `L`: 마지막 날 / `W`: 가장 가까운 평일 / `#`: 주차 (`2#2`=둘째 주 월요일)

### 2-3. 정적 스케줄러 예시 코드

**Delay 방식 및 Cron 방식 혼용**

```java
@Service
public class SchedulerTest {
    @Autowired
    private BoardService bs;

    // 1. 단순 반복: 시작 1초 후, 2초 간격 실행
    @Scheduled(initialDelay=1000, fixedDelay=2000)
    public void simpleTest() {
        System.out.println("2초마다 실행됨");
    }
    
    // 2. Cron: 매 30초마다 DB 업데이트
    @Scheduled(cron = "*/30 * * * * *")
    public void schedulerUpdate() {
        int result = bs.boardUpdateStatusN();
        if(result > 0) System.out.println("게시글 상태 변경 완료");
    }
}
```

## 3\. 동적 스케줄링 구현 (TaskScheduler)

  * **개념**: 사용자가 원하는 시점(메서드 호출 시)에 작업을 예약하는 방식입니다.
  * **준비**: `TaskScheduler` 빈(Bean)을 주입받아 사용합니다.
  * **메서드**:
    1. `schedule(Runnable task, Date startTime)`: 특정 시간에 **1회** 실행
    2. `schedule(Runnable task, Trigger trigger)`: 조건에 따라 **주기적** 실행

### 3-1. 구현 코드 (TaskTest.java)

**기본 예약 및 Trigger 사용**

```java
package com.kh.spring.common.scheduler.template;

import java.util.Date;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.scheduling.TaskScheduler;
import org.springframework.scheduling.support.CronTrigger;
import org.springframework.stereotype.Component;
import com.kh.spring.board.model.service.BoardService;

@Component
public class TaskTest {

    @Autowired
    private BoardService service;
    
    // root-context.xml에서 등록한 <task:scheduler id="taskScheduler"/> 주입
    @Autowired
    private TaskScheduler taskScheduler;

    /**
     * 1. 기본 사용법 (Date 객체 사용)
     * 메서드가 호출된 시점으로부터 5초 뒤에 1회 실행
     */
    public void taskTest() {
        taskScheduler.schedule(new Runnable(){
            @Override
            public void run() {
                System.out.println("taskTest 동작! (5초 경과)");
            }
        }, new Date(System.currentTimeMillis() + 5000));
    }
    
    /**
     * 2. 실무 활용 (Date 객체 사용)
     * 게시글 상세보기 후 10초 뒤 상태 변경
     */
    public void updateStatusN(int bno) {
        taskScheduler.schedule(new Runnable() {
            @Override
            public void run() {
                System.out.println("예약된 작업 실행: 게시글 " + bno + "번 비활성화");
                int result = service.boardUpdateN(bno);
            }
        }, new Date(System.currentTimeMillis() + 10000));
    }

    /**
     * 3. CronTrigger 사용
     * 동적 스케줄러에서도 Cron 표현식을 이용해 주기를 설정할 수 있음
     */
    public void scTrigger() {
        taskScheduler.schedule(new Runnable() {
            @Override
            public void run() {
                System.out.println("트리거를 이용하여 주기설정하기 (매초 실행)");
            }
        }, new CronTrigger("* * * * * *")); // 매초 실행
    }
    
    /**
     * 4. 사용자 정의 Trigger 사용
     * Trigger 인터페이스를 구현한 클래스를 통해 복잡한 실행 주기를 제어
     */
    public void scCustomTrigger() {
        taskScheduler.schedule(new Runnable() {
            @Override
            public void run() {
                System.out.println("사용자 정의 트리거 확인");
            }
        }, new CustomTrigger());
    }
}
```

### 3-2. 사용자 정의 트리거 (CustomTrigger) 구현

`CustomTrigger`를 사용하려면 `Trigger` 인터페이스를 구현해야 합니다.

**CustomTrigger.java 예시**

```java
import java.util.Date;
import org.springframework.scheduling.Trigger;
import org.springframework.scheduling.TriggerContext;

public class CustomTrigger implements Trigger {
    @Override
    public Date nextExecutionTime(TriggerContext triggerContext) {
        // 이전 실행의 완료 시점 가져오기 (첫 실행이면 null)
        Date lastCompletionTime = triggerContext.lastCompletionTime();
        
        // 첫 실행이거나 아직 실행된 적이 없으면 현재 시간 반환
        if (lastCompletionTime == null) {
            return new Date();
        }
        
        // 이전 작업 완료 후 1초(1000ms) 뒤에 다음 작업 실행 (동적 주기 설정 로직)
        return new Date(lastCompletionTime.getTime() + 1000);
    }
}
```

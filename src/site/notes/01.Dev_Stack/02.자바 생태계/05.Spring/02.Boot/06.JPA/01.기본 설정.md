# Spring Boot Data JPA Maven 환경 구성 가이드
> [!summary] 핵심 요약
> 
> Maven(`pom.xml`) 빌드 도구 기반의 Spring Data JPA 의존성 라이브러리 구성.
> 
> `application.properties`를 통한 데이터베이스 연결 정보 및 하이버네이트 동작 제어 설정.
> 
> 영속성 계층(Persistence Layer) 가동을 위한 인프라 설정의 표준화된 절차.
## 1. Maven 의존성 관리 (pom.xml)
- **Spring Data JPA 스타터**
    - `spring-boot-starter-data-jpa` 라이브러리 포함.
    - 하이버네이트(Hibernate), Spring Data, Spring ORM 등의 핵심 모듈 자동 로드.
- **데이터베이스 드라이버 (JDBC Driver)**
    - 실제 연결할 DB(H2, MySQL, PostgreSQL 등)의 커넥터 정의.
    - 실행 시점에만 필요하므로 `<scope>runtime</scope>` 권장.
- **Lombok (선택 사항)**
    - 엔티티의 Getter/Setter 등 반복 코드(Boilerplate) 제거를 위한 도구.
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>

<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <optional>true</optional>
</dependency>
```
## 2. 애플리케이션 환경 설정 (application.properties)
- **데이터소스(DataSource) 설정**
    - `spring.datasource.url`: 데이터베이스 접속 주소 명시.
    - `spring.datasource.driver-class-name`: 특정 DB 벤더의 JDBC 드라이버 클래스 지정.
- **JPA/하이버네이트 전략 설정**
    - `spring.jpa.hibernate.ddl-auto`: 스키마 자동 생성 전략(create, update, validate, none) 수립.
    - `spring.jpa.show-sql`: 콘솔 내 SQL 실행 로그 출력 여부 결정.
    - `spring.jpa.properties.hibernate.format_sql`: 쿼리 가독성을 위한 포맷팅 적용.
```properties
# 1. Database Connection (H2 예시)
spring.datasource.url=jdbc:h2:mem:testdb
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# 2. JPA/Hibernate Configuration
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
spring.jpa.properties.hibernate.use_sql_comments=true

# 3. H2 Console (Web UI 확인용)
spring.h2.console.enabled=true
```
## 3. 영속성 계층 핵심 컴포넌트 구조
- **도메인 엔티티 (Entity)**
    - `@Entity` 어노테이션을 사용하여 자바 객체와 DB 테이블 매핑.
    - `@Id` 및 `@GeneratedValue`를 통한 기본키(PK) 전략 수립.
- **리포지토리 인터페이스 (Repository)**
    - `JpaRepository<T, ID>`를 상속받아 기본적인 CRUD 및 페이징 로직 확보.
    - [[쿼리 메서드]] 및 [[JPQL]]을 활용한 데이터 접근 로직 구현.

# JPA 엔티티 심화 매핑 및 아키텍처 가이드
> [!summary] 핵심 요약
> 
> 객체지향 패러다임과 관계형 데이터베이스 모델의 차이를 극복하기 위한 심화 매핑 전략.
> 
> 연관관계의 주인(Owner) 개념을 통한 데이터 무결성 보장 및 성능 최적화를 위한 지연 로딩(Lazy Loading) 적용.
> 
> 상속 구조 및 공통 엔티티 속성의 효율적 관리를 위한 고도화된 아키텍처 설계 지침.
## 연관관계 매핑 (Relationship Mapping)
- **연관관계의 주인과 mappedBy**
    - 객체의 양방향 참조 시 외래 키(FK)를 관리하는 주체 설정.
    - **주인(Owner)**: `@JoinColumn`을 사용하여 실제 DB 컬럼과 매핑되는 쪽.
    - **주인이 아닌 쪽**: `mappedBy` 속성을 통해 읽기 전용으로 설정 및 소유권 포기 명시.
- **다대일(@ManyToOne) 및 일대다(@OneToMany)**
    - 실무에서 가장 빈번하게 사용되는 기본 연관관계 구조.
    - 가급적 다대일 단방향 매핑을 우선 고려하고 필요시 양방향 추가.
- **지연 로딩(Lazy Loading) 전략**
    - `fetch = FetchType.LAZY` 설정을 통한 프록시 객체 활용.
    - 불필요한 SQL 실행 방지 및 N+1 문제 발생 가능성 차단.
    - Repository 에서 페치 조인으로 N+1 문제 해결 필요
- **영속성 전이(Cascade) 및 고아 객체**
    - `cascade = CascadeType.ALL`: 부모 엔티티의 상태 변화를 자식에게 전파.
    - `orphanRemoval = true`: 부모와의 연관관계가 끊어진 자식 엔티티를 자동 삭제.
```java
@Entity
public class Member {
    @Id @GeneratedValue
    private Long id;

    @ManyToOne(fetch = FetchType.LAZY) // 지연 로딩 권장
    @JoinColumn(name = "team_id")      // 외래 키 매핑
    private Team team;
}

@Entity
public class Team {
    @Id @GeneratedValue
    private Long id;

    @OneToMany(mappedBy = "team", cascade = CascadeType.ALL) // 주인이 아님을 명시
    private List<Member> members = new ArrayList<>();
}
```
## 상속 관계 매핑 (Inheritance Mapping)
- **상속 전략 선택 (@Inheritance)**
    - **JOINED**: 공통 속성은 부모 테이블에, 개별 속성은 자식 테이블에 분산 저장 (정규화 전략).
    - **SINGLE_TABLE**: 모든 속성을 하나의 테이블에 통합 관리 (성능 최적화, DTYPE 필수).
    - **TABLE_PER_CLASS**: 구현 클래스마다 각각의 테이블 생성 (비권장).
- **구분 컬럼 (@DiscriminatorColumn)**
    - 부모 테이블에서 자식 엔티티의 종류를 구분하기 위한 컬럼 설정.
## 공통 매핑 정보 관리
- **MappedSuperclass**
    - 등록일, 수정일 등 공통적으로 사용되는 매핑 정보 상속용 추상 클래스.
    - 실제 테이블로 생성되지 않으며 자식 엔티티에게 매핑 정보만 제공.
- **JPA Auditing**
    - `@EntityListeners(AuditingEntityListener.class)`를 통한 자동 생성/수정 시간 추적.
```java
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseEntity {
    @CreatedDate
    private LocalDateTime createdAt;

    @LastModifiedDate
    private LocalDateTime updatedAt;
}
```
해당 기능을 쓰려면 스프링 부트 설정 클래스(보통 메인 클래스) 위에 **`@EnableJpaAuditing`**이라는 어노테이션을 반드시 붙여줘야 감시 시스템이 가동됩니다.
```java
@EnableJpaAuditing // 이게 있어야 감시자가 일을 시작함!
@SpringBootApplication
public class JpaApplication { ... }
```
## 임베디드 타입 (Value Object)
- **@Embeddable & @Embedded**
    - 새로운 값 타입을 정의하여 엔티티의 응집력 향상.
    - 주소(Address), 기간(Period) 등 반복되는 필드 그룹을 하나의 객체로 모듈화.

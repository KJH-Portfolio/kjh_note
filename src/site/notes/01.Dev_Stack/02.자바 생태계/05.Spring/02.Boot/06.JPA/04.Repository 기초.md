# Spring Data JPA 레포지토리 핵심 가이드
> [!summary] 핵심 요약
> 
> 스프링 데이터 JPA(Spring Data JPA) 환경에서 데이터베이스 접근을 추상화하는 핵심 인터페이스. `JpaRepository` 상속만으로 기본적인 CRUD 연산 및 페이징 처리 자동 지원. 메서드 이름 규칙 및 `@Query`를 통한 유연한 데이터 조작 환경 제공.
## JPA 레포지토리 기본 구조
- **인터페이스 상속**
    - `JpaRepository<엔티티 타입, 식별자 타입>` 인터페이스 상속을 통한 구현체 자동 생성
    - 스프링 컨테이너가 런타임에 프록시 객체를 주입하므로 `@Repository` 어노테이션 생략 가능
    - `PagingAndSortingRepository`, `CrudRepository`의 기능을 모두 포함
- **기본 제공 메서드 (CRUD)**
    - `findById(id)`: 식별자(PK)를 통한 단건 엔티티 조회 (Optional 반환)
    - `findAll()`: 해당 엔티티의 전체 목록 조회
    - `delete(entity)`: 전달된 엔티티 정보 삭제
    - `count()`: 전체 엔티티 개수 반환
    - (비권장)`save(entity)`: 새로운 엔티티 저장 또는 기존 엔티티 병합

```java
// 기본 JpaRepository 상속 예시
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.Optional;

public interface MemberRepository extends JpaRepository<Member, Long> {
    // JpaRepository 상속만으로 기본적인 CRUD 메서드 사용 가능
}
```
## 쿼리 메서드 (Query Methods)
- **메서드 이름 규칙(Naming Convention) 기반 쿼리 생성**
    - 인터페이스 내 메서드 이름을 분석하여 **JPQL** 쿼리를 자동 생성 및 실행
    - `findBy`로 시작하여 조회 조건 및 속성명 결합
- **주요 키워드 및 패턴**
    - `And` / `Or`: 여러 속성을 조합한 조건 검색 (`findByUsernameAndAge`)
    - `GreaterThan` / `LessThan`: 대소 비교 연산 (`findByAgeGreaterThan`)
    - `Like` / `Containing`: 특정 문자열 포함 여부 검색 (`findByUsernameContaining`)
    - `OrderBy`: 결과 정렬 조건 추가 (`findByAgeOrderByNameDesc`)
```java
// 쿼리 메서드 활용 예시
import java.util.List;

public interface MemberRepository extends JpaRepository<Member, Long> {
    
    // 1. 단순 조건 조회 (SELECT * FROM member WHERE username = ?)
    List<Member> findByUsername(String username);
    
    // 2. 다중 조건 조회 (SELECT * FROM member WHERE username = ? AND age > ?)
    List<Member> findByUsernameAndAgeGreaterThan(String username, int age);
    
    // 3. 정렬 포함 조회 (SELECT * FROM member WHERE role = ? ORDER BY created_at DESC)
    List<Member> findByRoleOrderByCreatedAtDesc(String role);
}
```
## 사용자 정의 쿼리 (@Query)
- **복잡한 쿼리 직접 작성**
    - 메서드 이름 규칙만으로 해결하기 힘든 복잡한 조인, 통계성 쿼리에 적용
    - 메서드 위에 `@Query` 어노테이션을 선언하여 JPQL(Java Persistence Query Language) 직접 매핑
    - 파라미터 바인딩은 `@Param` 어노테이션 활용
- (비권장) 네이티브 SQL (Native Query) 지원
    - `nativeQuery = true` 속성 부여 시 특정 DB에 종속적인 순수 SQL 실행 가능
    - 가급적 JPQL 사용을 권장하며, 성능 최적화 등 불가피한 경우에만 제한적 사용
```java
// @Query 어노테이션 활용 예시
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;

public interface MemberRepository extends JpaRepository<Member, Long> {
    
    // 1. JPQL을 이용한 직접 쿼리 작성 (엔티티 객체 대상)
    @Query("SELECT m FROM Member m WHERE m.username = :name AND m.age = :age")
    List<Member> findUserCustom(@Param("name") String username, @Param("age") int age);
    
    // 2. 특정 필드만 DTO로 조회 (new 명령어 사용)
    @Query("SELECT new com.example.dto.MemberDto(m.id, m.username) FROM Member m")
    List<MemberDto> findMemberDto();
    
    // 3. **비권장** 네이티브 쿼리 사용 (실제 데이터베이스 테이블과 컬럼명 기준)
    @Query(value = "SELECT * FROM member_table WHERE email LIKE %:domain%", nativeQuery = true)
    List<Member> findUsersByEmailDomain(@Param("domain") String domain);
}
```

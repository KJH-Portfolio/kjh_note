# Swagger (스웨거)
> [!summary] 요약
> 
> REST API의 설계, 빌드, 문서화 및 소비를 돕는 오픈소스 소프트웨어 프레임워크 및 사양의 통칭.
## 개요 및 정의
- **API 문서 자동화**
    - 코드 변경 시 API 문서를 실시간으로 동기화 및 갱신하는 기능
- **인터랙티브 UI**
    - 별도의 클라이언트(Postman 등) 없이 브라우저 내에서 API 엔드포인트 직접 호출 및 테스트 가능
- **표준 규격 (OpenAPI Specification)**
    - RESTful 서비스의 구조를 설명하기 위한 업계 표준 인터페이스 정의 언어(IDL) 기반
## 주요 구성 요소
- **Swagger UI**
    - OAS 기반으로 시각화된 API 문서를 생성하고 사용자 테스트 환경을 제공하는 웹 인터페이스
- **Swagger Editor**
    - 브라우저 기반의 편집기를 통해 API 스펙을 YAML 또는 JSON 형식으로 설계하고 실시간으로 검토
- **Swagger Codegen**
    - 정의된 API 스펙에 맞춰 다양한 언어의 클라이언트 SDK 및 서버 스텁(Stub) 코드 자동 생성
## 주요 어노테이션 (Java/Spring 기준)
- **@Tag**
    - API 그룹화 및 설명 추가를 위한 컨트롤러 단위 설정
- **@Operation**
    - 특정 API 엔드포인트의 목적 및 상세 동작 설명
- **@Parameter**
    - 호출 파라미터의 속성(필수 여부, 타입 등) 정의
- **@ApiResponse**
    - HTTP 응답 상태 코드 및 메시지 상세 기술
```java
// Swagger API 설정 예시
@Tag(name = "User API", description = "사용자 관리 API")
@RestController
@RequestMapping("/api/users")
public class UserController {

    @Operation(summary = "사용자 조회", description = "ID를 통한 개별 사용자 정보 반환")
    @GetMapping("/{id}")
    public UserResponse getUser(@Parameter(description = "사용자 ID") @PathVariable Long id) {
        return userService.findById(id);
    }
}
```
## 장점 및 효과
- **문서 유지보수 효율성**
    - 수동 문서 작성 비용 절감 및 최신 상태 유지 보장
- **협업 프로세스 개선**
    - 프론트엔드와 백엔드 간 명확한 인터페이스 공유로 커뮤니케이션 오류 방지
- **개발 생산성 향상**
    - API 호출을 위한 예시 데이터 확인 및 즉각적인 응답 결과 검증
```yaml
# OpenAPI Specification (OAS) 3.0 예시
openapi: 3.0.0
info:
  title: Sample API
  version: 1.0.0
paths:
  /users:
    get:
      summary: 전체 사용자 목록 조회
      responses:
        '200':
          description: 성공
```
---
- **수정/추가 사항**: Swagger v2(Springfox)와 v3(Springdoc)의 설정 차이점 및 라이브러리 의존성 정보 추가 가능
- **삭제 사항**: 불필요한 서술형 어미 일체 배제 완료
- **제안**: Spring Boot 프로젝트에서 Swagger를 적용하기 위한 `SecurityConfig` 설정법이나 `application.yml` 설정 옵션에 대한 정리가 필요합니까?
- **질문**: 특정 프로그래밍 언어나 프레임워크(Node.js, Python 등) 환경에서의 Swagger 설정 예시가 추가로 필요합니까?

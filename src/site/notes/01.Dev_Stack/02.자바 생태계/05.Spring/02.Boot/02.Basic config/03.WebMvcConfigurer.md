# WebMvcConfigurer 현대적 설정 및 정적 자원 최적화 명세(Interceptor 포함)
> [!summary] Java 8 Default Method 기반의 선택적 오버라이딩 방식 Spring MVC 설정 체계. **Interceptor**, **CORS**, **Static Resource** 등 웹 계층 인프라의 통합 관리 및 최적화 명세.
---
## 현대적 구현 체계 및 원칙
> [!info] 인터페이스 중심 설정 구조
- **Default Interface 활용**
    - Java 8 인터페이스 내 `default` 메서드 구현을 통한 불필요한 코드 작성 방지 및 유연한 선택적 재정의 지원.
- **WebMvcConfigurerAdapter 폐기**
    - Java 8 이상 환경에서의 어댑터 클래스 사용 중단 및 `WebMvcConfigurer` 인터페이스 직접 구현 방식 채택.
- **@Configuration 기반 빈 등록**
    - 자바 설정 클래스 명시를 통한 스프링 컨테이너 관리 및 컴포넌트 스캔 자동화 처리.
```java
@Configuration // 스프링 설정 클래스 선언
public class WebMvcConfig implements WebMvcConfigurer { // MVC 설정 인터페이스 구현 시작
    // 필요한 메서드만 선택적으로 오버라이딩 수행
}
```

## HandlerInterceptor 메커니즘 및 자원 매핑 구조
> [!note] 요청 처리 단계별 실행 시점 및 자원 접근 흐름
- **preHandle**
    - 컨트롤러 실행 직전 호출. 로그인 여부, 권한 검증 등 전처리 로직 수행 및 진행 여부 결정.
- **postHandle**
    - 컨트롤러 작업 완료 후 뷰 렌더링 전 호출. 추가적인 모델 데이터 조작 및 가공 기능 제공.
- **afterCompletion**
    - 모든 작업 완료 후 호출. 예외 발생 여부와 관계없는 리소스 반납 및 실행 시간 로깅 수행.
- **ResourceHandler 동작**
    - 특정 URL 패턴과 물리적 파일 경로 매핑을 통한 정적 컨텐츠(이미지, JS, CSS)의 효율적 서빙.

## 인터셉터, CORS 및 정적 자원 상세 설정
> [!tip] 웹 보안 및 리소스 관리 최적화 정책
- **addInterceptors**(중요!)
	- 용도: 특정 URL 패턴에 대해 로그인 체크나 권한 검사를 수행하는 **인터셉터**를 등록합니다.
	- 예시: 모든 요청에 대해 로그를 남기거나, 인증되지 않은 사용자를 차단할 때 사용합니다.
- **addCorsMappings**(중요!)
	- 용도: **CORS(Cross-Origin Resource Sharing)** 설정을 합니다.
	- 예시: 특정 도메인(예: React나 Vue 앱)에서 오는 API 요청을 허용하도록 설정합니다.
- **addViewControllers**(중요!)
	- 용도: 컨트롤러 클래스 없이 특정 URL을 특정 뷰(HTML)로 바로 연결할 때 사용합니다.
	- 예시: 단순히 `/login` 요청이 오면 `login.html`을 보여주고 싶을 때 아주 간편합니다.
- **addResourceHandlers**(중요!)
	- 용도: 정적 리소스(이미지, JS, CSS 등)의 경로를 설정합니다.
	- 예시: `/images/**`로 들어오는 요청을 서버의 실제 디렉토리와 연결해 줍니다.
- **configureMessageConverters**
	- 용도: HTTP 요청 본문을 객체로 바꾸거나, 객체를 응답 본문으로 바꿀 때 사용하는 **메시지 컨버터**를 설정합니다.

```java
@Configuration // 설정 정보 빈 등록
@RequiredArgsConstructor // final 필드 생성자 자동 주입
public class WebMvcConfig implements WebMvcConfigurer { // 설정 구현체 정의

    private final LoginInterceptor loginInterceptor; // 로그인 인터셉터 주입

    @Override // 인터셉터 설정 재정의
    public void addInterceptors(InterceptorRegistry registry) { // 등록 프로세스 시작
        registry.addInterceptor(loginInterceptor) // 인터셉터 객체 추가
                .addPathPatterns("/api/v1/**") // 적용 범위 지정
                .excludePathPatterns("/api/v1/auth/**", "/static/**") // 제외 범위 지정
                .order(1); // 실행 우선순위 할당
    } // 메서드 종료

    @Override // CORS 정책 설정 재정의
    public void addCorsMappings(CorsRegistry registry) { // 매핑 프로세스 시작
        registry.addMapping("/api/**") // 적용 대상 URL 패턴
                .allowedOrigins("http://localhost:3000") // 허용 도메인 주소
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS") // 허용 HTTP 메서드
                .allowedHeaders("*") // 모든 헤더 허용
                .allowCredentials(true) // 인증 정보 포함 허용
                .maxAge(3600); // 브라우저 캐싱 시간 설정(초)
    } // 메서드 종료

    @Override // 정적 자원 핸들러 설정 재정의
    public void addResourceHandlers(ResourceHandlerRegistry registry) { // 리소스 등록 시작
        registry.addResourceHandler("/images/**") // 접근 가능한 URL 패턴 정의
                .addResourceLocations("file:///C:/upload/images/") // 실제 물리적 파일 경로 매핑
                .setCachePeriod(3600) // 정적 자원 캐싱 기간 설정(초)
                .resourceChain(true); // 리소스 체이닝 최적화 활성화
    } // 메서드 종료
}
```

## 인터셉터 구현 및 프로세스 요약
### 1. LoginInterceptor 구현
- **보안 검증 로직**
    - `HandlerInterceptor` 구현 및 `preHandle`을 활용한 JWT 토큰 유효성 검사.
```java
@Component // 컴포넌트 스캔 대상 등록
public class LoginInterceptor implements HandlerInterceptor { // 인터셉터 정의 시작

    @Override // 핸들러 실행 전 처리 로직
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception { // 로직 시작
        String token = request.getHeader("Authorization"); // 인증 헤더 추출
        if (token == null || !token.startsWith("Bearer ")) { // 유효성 검증
            response.setStatus(401); // 미인증 코드 반환
            response.getWriter().write("Login Required!"); // 에러 메시지 출력
            return false; // 컨트롤러 진입 차단
        } // 조건문 종료
        return true; // 요청 정상 진행
    } // 메서드 종료
}
```
### 2. 처리 프로세스 요약
- **경로 매칭 및 인터셉트**
    - 클라이언트 요청 감지 시 등록된 `PathPatterns` 기반 필터링 수행.
- **권한 확인 및 자원 서빙**
    - 인터셉터 통과 시 컨트롤러 호출 또는 리소스 핸들러를 통한 정적 파일 응답.
- **최종 응답 및 사후 처리**
    - 뷰 렌더링 및 `afterCompletion` 로직 실행 후 클라이언트 최종 응답 전송.

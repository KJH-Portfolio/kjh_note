# Spring Boot AsyncConfig 비동기 처리 구현
> [!summary] 핵심 요약
> 
> Spring Boot 환경에서 `@EnableAsync`와 `ThreadPoolTaskExecutor`를 활용한 비동기 스레드 풀 설정 및 `@Async` 어노테이션 기반의 메서드 구현 방법과 제약 사항 정리.
## 비동기 설정 (Configuration)
- **@EnableAsync**
    - Spring Boot의 비동기 처리 기능 활성화.
    - `@Configuration` 클래스에 선언하여 프록시 기반 비동기 처리 지원.
- **ThreadPoolTaskExecutor**
    - 비동기 작업 수행을 위한 커스텀 스레드 풀 정의.
    - **CorePoolSize:** 기본적으로 대기하는 스레드 개수.
    - **MaxPoolSize:** 동시 동작 가능한 최대 스레드 개수.
    - **QueueCapacity:** MaxPoolSize 초과 시 대기하는 큐(Queue) 크기.
```java
@Configuration
@EnableAsync
public class AsyncConfig {

    @Bean(name = "taskExecutor")
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);      // 기본 스레드 수
        executor.setMaxPoolSize(10);      // 최대 스레드 수
        executor.setQueueCapacity(50);    // 대기열 크기
        executor.setThreadNamePrefix("Async-Executor-");
        executor.initialize();
        return executor;
    }
}
```
## 비동기 메서드 구현 (@Async)
- **메서드 선언**
    - 실행 메서드 상단에 `@Async("taskExecutor")` 명시.
    - 설정된 Executor(별도 스레드)를 통해 해당 로직 실행.
- **반환 타입 (Return Type)**
    - **Void:** 리턴 값이 없는 단순 작업(Fire & Forget)에 사용.
    - **CompletableFuture:** 작업 결과값 반환 또는 연산 체이닝 필요 시 사용.
```java
@Service
public class AsyncService {

    // 1. 리턴값이 없는 경우
    @Async("taskExecutor")
    public void runAsyncTask(String message) {
        System.out.println("비동기 작업 시작: " + message);
    }

    // 2. 리턴값이 필요한 경우
    @Async("taskExecutor")
    public CompletableFuture<String> runAsyncTaskWithReturn() {
        return CompletableFuture.completedFuture("작업 완료");
    }
}
```
## 주의사항 및 제약 (Constraints)
- **접근 제어자 (Access Modifier)**
    - 프록시 생성을 위해 반드시 **Public 메서드** 선언 필수.
    - Private 메서드 적용 불가.
- **자가 호출 금지 (Self-Invocation)**
    - 동일 클래스 내부 메서드에서 `@Async` 메서드 호출 시 비동기 미작동.
    - 프록시를 거치지 않고 내부 직접 호출(Direct Call)로 처리되기 때문 (별도 Bean 분리 필요).

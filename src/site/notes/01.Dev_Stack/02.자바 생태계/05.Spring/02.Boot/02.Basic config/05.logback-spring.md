# 스프링 부트(Spring Boot) Logback 로깅 시스템 통합 가이드
> [!summary]
> 
> 스프링 부트 기본 로깅 프레임워크인 **Logback**의 계층적 로깅 레벨, **상세 패턴(Conversion Word)** 구성, 환경별 분기 처리 및 **Lombok** 연동을 통한 실무 최적화 전략의 구조적 정리.
---
## 로깅 기본 개념 및 계층 구조
- **기본 엔진**
    - `spring-boot-starter-logging` 의존성 기본 내장.
    - **Logback** 프레임워크 기반의 강력한 기본 설정 제공.
- **로깅 레벨 우선순위 (상세도순)**
    - **TRACE**: 가장 상세한 정보 기록 (개발 단계).
    - **DEBUG**: 디버깅을 위한 변수 및 흐름 확인.
    - **INFO**: 시스템 정상 작동 확인용 표준 레벨 (기본값).
    - **WARN**: 잠재적 위험 및 경고 사항.
    - **ERROR**: 예외 발생 및 시스템 오류 기록.
- **설정 파일 우선순위**
    - `logback-spring.xml` 사용 권장 (스프링 프로파일 확장 기능 지원).
    - `application.yml` 또는 `.properties`를 통한 간이 설정 병행 가능.
---
## 상세 로그 패턴 (Conversion Pattern)
- **주요 패턴 키워드 정의**

|**키워드**|**설명**|**예시**|
|---|---|---|
|`%d`|로그 발생 시간 (날짜 및 시각)|`%d{yyyy-MM-dd HH:mm:ss.SSS}`|
|`%-5level`|로그 레벨 (5칸 확보 및 왼쪽 정렬)|`INFO` , `DEBUG`|
|`${PID:-}`|현재 OS 프로세스 ID|`12345`|
|`%thread`|로그 발생 스레드명|`http-nio-8080-exec-1`|
|`%logger{36}`|로거 이름 (최대 36자 축약)|`c.m.p.controller.MyController`|
|`%msg%n`|로그 메시지 내용 및 줄바꿈|`데이터 조회 성공\n`|
|`%highlight`|로그 레벨별 색상 차등 적용|(콘솔 전용)|

## 단계별 설정 구성 및 적용
### application.properties 기반 간이 설정
- 복잡한 XML 구성 없이 핵심 로깅 정책(레벨, 경로, 파일 크기) 정의.
```properties
// 1. 패키지별 로깅 레벨 지정
logging.level.root=info
logging.level.com.myproject=debug

// 2. 콘솔 출력 색상 및 저장 경로 설정
spring.output.ansi.enabled=always
logging.file.path=logs
logging.file.name=logs/spring_boot_app.log

// 3. 파일 롤링 정책 (최대 크기 및 보관 기간)
logging.logback.rollingpolicy.max-file-size=10MB
logging.logback.rollingpolicy.max-history=7
```
### logback-spring.xml 기반 환경별 분기 설정
- **Appender**: 로그 출력 대상(Console, File) 및 패턴 지정.
- **SpringProfile**: 개발(dev) 및 운영(prod) 환경별 로깅 전략 차별화.
```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    // 공통 패턴 변수 정의
    <property name="CONSOLE_LOG_PATTERN" value="%d{yyyy-MM-dd HH:mm:ss.SSS} %highlight(%-5level) %magenta(${PID:-}) --- [%15thread] %cyan(%-40logger{36}) : %msg%n"/>
    <property name="FILE_LOG_PATTERN" value="%d{yyyy-MM-dd HH:mm:ss.SSS} %-5level ${PID:-} --- [%thread] %-40logger{36} : %msg%n"/>

    // 개발 환경 설정: 콘솔 출력 위주
    <springProfile name="dev">
        <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
            <encoder><pattern>${CONSOLE_LOG_PATTERN}</pattern></encoder>
        </appender>
        <root level="DEBUG"><appender-ref ref="CONSOLE"/></root>
    </springProfile>

    // 운영 환경 설정: 파일 저장 및 롤링 정책 적용
    <springProfile name="prod">
        <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
            <file>logs/prod-app.log</file>
            <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
                <fileNamePattern>logs/prod-app.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
                <maxFileSize>100MB</maxFileSize>
                <maxHistory>30</maxHistory>
            </rollingPolicy>
            <encoder><pattern>${FILE_LOG_PATTERN}</pattern></encoder>
        </appender>
        <root level="INFO"><appender-ref ref="FILE"/></root>
    </springProfile>
</configuration>
```

## 실무 최적화 및 코드 구현
- **Lombok @Slf4j 활용**
    - 로거 객체 선언 코드 자동화 및 가독성 향상.
- **치환 문법 ({}) 사용**
    - 문자열 더하기 연산 배제를 통한 성능 최적화.
    - 로그 레벨 미충족 시 메시지 구성 비용 미발생.
```java
import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.annotation.RestController;

@Slf4j
@RestController
public class LogController {

    public void logExample() {
        // 상황별 로그 레벨 활용
        log.debug("디버깅 데이터: {}", data); // 개발 단계 확인
        log.info("사용자 요청 완료 - ID: {}", userId); // 정상 흐름 기록
        log.error("데이터베이스 연결 실패: {}", exception.getMessage()); // 오류 추적

        // 문자열 연산 지양 (치환 문법 권장)
        log.info("ID: " + id + ", Name: " + name); // 지양 (성능 저하)
        log.info("ID: {}, Name: {}", id, name); // 지향 (효율적)
    }
}
```

# Spring Boot Scheduler 설정 및 TaskScheduler 이론 명세

> [!summary]
> **@EnableScheduling** 기반의 작업 자동화 및 **ThreadPoolTaskScheduler**를 활용한 멀티 스레드 환경 구축, 그리고 기존 레거시 XML 설정 방식의 **Java Config 완전 대체성**에 관한 기술 명세 요약.

## 스케줄링 핵심 이론 및 구성 요소 (Scheduling Theory)

> [!info] 주기적 작업 자동화 및 스레드 풀 관리

* **스케줄링(Scheduling)**: 특정 시간 또는 일정한 간격에 맞추어 비즈니스 로직을 자동 실행하는 메커니즘.
* **@EnableScheduling**: 스프링 컨테이너 내 스케줄러 기능 활성화 및 `@Scheduled` 어노테이션 탐색 지시.
* **ThreadPoolTaskScheduler**: 멀티 스레드 기반의 태스크 실행기.
* **병렬 처리**: 기본 단일 스레드 방식의 병목 현상 방지 및 다중 작업 동시 수행 지원.
* **자원 효율성**: 스레드 풀(Thread Pool) 관리로 인한 시스템 자원 생성/소멸 비용 최적화.



---

## 설정 방식의 변천 및 대체성 (Substitutability)

> [!important] XML 설정에서 Java 기반 설정으로의 완전한 전환

* **XML 설정 대체**: 기존 Spring Legacy의 `root-context.xml` 내 `<task:scheduler>`, `<task:executor>` 태그 설정을 **@Configuration** 및 **@Bean** 방식으로 완전 대체.
* **유연성 및 확장성**: XML 방식의 정적 설정을 탈피하여 자바 코드를 통한 동적 조건별 스레드 풀 사이즈 조절 및 세부 옵션 제어 가능.
* **타입 안정성 확보**: 문자열 기반의 XML 설정과 달리 컴파일 시점에 설정 오류 확인 및 IDE의 자동 완성 기능 활용 가능.
* **Spring Boot 최적화**: 별도의 외부 파일 관리 없이 애플리케이션 컨텍스트 내에서 스케줄링 라이프사이클 통합 관리.

## 코드 분석 및 보완 사항 (Code Review & Enhancement)

> [!tip] 운영 환경 안정성 향상을 위한 권장 설정

```java
@Configuration
@EnableScheduling //스케줄링 기능 명시 어노테이션
public class SchedulerConfig {

	//시스템 기록(에러) 기록
    private static final Logger log = LoggerFactory.getLogger(SchedulerConfig.class);

    // application.properties에서 설정값 주입 (스레드 기본값 10)
    @Value("${scheduler.pool.size:10}")
    private int poolSize;

    @Bean
    public TaskScheduler taskScheduler() {
	    //스레드 풀 방식으로 여러 작업 동시 처리하게 해주는 객체
        ThreadPoolTaskScheduler scheduler = new ThreadPoolTaskScheduler();
        
        // 1. 스레드 풀 사이즈 설정 (외부 설정값 반영)
        scheduler.setPoolSize(poolSize);
        
        // 2. 스레드 이름 접두사 설정 (모니터링 가시성 확보)
        scheduler.setThreadNamePrefix("KH-Scheduler-");
        
        // 3. 에러 핸들러 등록 (예외 발생 시 전체 중단 방지 및 로그 기록)
        scheduler.setErrorHandler(err -> {
            log.error("스케줄러 작업 중 예외 발생: {}", err.getMessage());
        });
        
        // 4. 스레드 풀 초기화(설정 값을 바탕으로 생성,준비)
        scheduler.initialize();

        return scheduler;
    }
}
```
#### 사용 코드
```java
@Slf4j // log를 사용가능한 어노테이션
@Component // 1. 스프링이 이 클래스를 관리하도록 등록
public class MyBatchService {

    // 사용법 1: 고정된 간격으로 실행 (fixedRate)
    // 5000ms = 5초마다 무조건 실행
    @Scheduled(fixedRate = 5000)
    public void runEveryFiveSeconds() {
        // 아까 설정한 'KH-Scheduler-X' 이름이 찍히는지 확인해 보세요!
        log.info("5초마다 실행됨! 현재 스레드: {}", Thread.currentThread().getName());
    }

    // 사용법 2: 특정 시간에 실행 (Cron 표현식)
    // "초 분 시 일 월 요일" 순서 -> 매일 오후 2시 30분 0초에 실행
    @Scheduled(cron = "0 30 14 * * *")
    public void runAtSpecificTime() {
        log.info("점심 먹고 졸릴 시간, 오후 2시 30분입니다!");
    }
}
```

# Spring Boot + React STOMP 심화 가이드 (이론 및 상세 주석 포함)
> [!summary] 요약
> 
> 초심자를 위한 STOMP 프로토콜의 이론적 배경 설명과 Oracle DB 저장, JWT 인증, 비동기 처리가 포함된 상세 주석 코드 가이드.
---
## 1. 핵심 이론 개념 (Theoretical Concepts)
### STOMP (Simple Text Oriented Messaging Protocol)
WebSocket은 단순히 양방향 통신 통로(Pipe)만 열어주는 기술. 데이터의 형식(Format)은 정의하지 않음.
STOMP는 WebSocket 위에서 동작하는 **텍스트 기반 메시징 프로토콜**.
HTTP와 유사하게 `Header`와 `Body` 구조를 가지며, **Pub/Sub(발행/구독)** 모델을 사용.
- **Pub/Sub 모델**
    - **Publisher (발행자):** 특정 토픽(Topic)에 메시지를 보냄.
    - **Subscriber (구독자):** 해당 토픽을 듣고 있는(Listening) 클라이언트.
    - **Broker (중개자):** 발행된 메시지를 구독자들에게 전달해주는 역할 (Spring 내장 브로커 사용).
### STOMP 통신 흐름
1. **Handshake:** HTTP 요청을 통해 WebSocket 연결 수립.
2. **Connect:** STOMP 프로토콜 연결 요청 (이때 인증 헤더 전송).
3. **Subscribe:** 클라이언트가 특정 경로(`/sub/...`) 구독.
4. **Send:** 클라이언트가 메시지 전송(`/pub/...`).
5. **Message:** 서버가 구독자들에게 메시지 브로드캐스팅.

## 2. Backend 구현 (Spring Boot)
### 비동기 처리 설정 (Async Configuration)
채팅 시스템은 "실시간성"이 최우선. DB 저장과 같은 무거운 작업이 메시지 전송을 막으면 안 됨.
따라서 DB 저장은 별도의 스레드(Thread)에서 돌도록 `@Async` 설정.
```java
package com.example.chat.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.EnableAsync;

// [비동기 설정]
// @Async 어노테이션이 붙은 메서드를 별도의 스레드에서 실행하게 함.
// 이를 통해 채팅 전송(빠름)과 DB 저장(느림)을 분리하여 렉(Lag)을 방지.
@EnableAsync
@Configuration
public class AsyncConfig {
    // 필요 시 ThreadPoolTaskExecutor 빈(Bean) 정의를 통해 스레드 풀 튜닝 가능
}
```
### WebSocket 및 보안 설정 (Config & Interceptor)
가장 중요한 설정 파일. 연결 엔드포인트와 메시지 라우팅 규칙 정의.
```java
package com.example.chat.config;

import lombok.RequiredArgsConstructor;
import org.springframework.context.annotation.Configuration;
import org.springframework.messaging.simp.config.ChannelRegistration;
import org.springframework.messaging.simp.config.MessageBrokerRegistry;
import org.springframework.web.socket.config.annotation.*;

@Configuration
@EnableWebSocketMessageBroker // STOMP 사용 선언
@RequiredArgsConstructor
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    private final StompHandler stompHandler; // JWT 검증을 위한 인터셉터

    /**
     * 1. 연결 엔드포인트 설정 (Handshake)
     */
    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        // 클라이언트가 연결할 주소: ws://localhost:8080/ws-stomp
        registry.addEndpoint("/ws-stomp")
                .setAllowedOriginPatterns("*") // CORS 설정: 모든 도메인 허용 (배포 시 프론트 주소로 변경 권장)
    }

    /**
     * 2. 메시지 브로커 설정 (Routing)
     */
    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        // [구독 주소] 서버 -> 클라이언트
        // "/sub"로 시작하는 주소는 내장 브로커(SimpleBroker)가 처리하여 구독자에게 바로 전송
        registry.enableSimpleBroker("/sub");

        // [발행 주소] 클라이언트 -> 서버
        // "/pub"로 시작하는 주소는 @Controller의 @MessageMapping 메서드로 라우팅됨
        registry.setApplicationDestinationPrefixes("/pub");
    }

    /**
     * 3. 인바운드 채널 인터셉터 등록 (Security)
     * 메시지가 서버로 들어올 때(Inbound) 가로채서 검증 수행
     */
    @Override
    public void configureClientInboundChannel(ChannelRegistration registration) {
        registration.interceptors(stompHandler);
    }
}
```
### JWT 검증 핸들러 (StompHandler)
HTTP 요청이 아닌 TCP/WebSocket 레벨에서의 메시지 프레임 감청 및 인증.
```java
package com.example.chat.handler;

import lombok.RequiredArgsConstructor;
import org.springframework.messaging.Message;
import org.springframework.messaging.MessageChannel;
import org.springframework.messaging.simp.stomp.StompCommand;
import org.springframework.messaging.simp.stomp.StompHeaderAccessor;
import org.springframework.messaging.support.ChannelInterceptor;
import org.springframework.stereotype.Component;

@Component
@RequiredArgsConstructor
public class StompHandler implements ChannelInterceptor {

    private final JwtTokenProvider jwtTokenProvider; // 기존의 JWT 검증 로직 활용

    // 메시지가 채널로 전송되기 "직전"에 실행됨
    @Override
    public Message<?> preSend(Message<?> message, MessageChannel channel) {
        // 메시지에서 STOMP 헤더 정보를 추출
        StompHeaderAccessor accessor = StompHeaderAccessor.wrap(message);

        // [검증 시점] 모든 메시지가 아니라, 최초 연결(CONNECT) 시점에만 토큰 확인
        if (StompCommand.CONNECT == accessor.getCommand()) { //StompCommand.CONNECT(연결요청 상태 자체상수)
            // 헤더에서 "Authorization" 값 추출 (NativeHeader 사용 필수)
            String jwt = accessor.getFirstNativeHeader("Authorization");

            // 토큰이 없거나 유효하지 않으면 예외 발생 -> 연결 거부
            // "Bearer " 접두사 제거 로직 등은 상황에 맞게 추가
            if (jwt == null || !jwtTokenProvider.validateToken(jwt)) {
                throw new IllegalArgumentException("유효하지 않은 JWT 토큰입니다.");
            }
            
            // 필요 시 accessor.setUser(authentication) 등을 통해 인증 정보 저장 가능
        }
        return message;
    }
}
```
### 채팅 컨트롤러 및 비동기 서비스
**Controller**는 "교통 정리(Routing)", **Service**는 "일꾼(Business Logic)".
```java
@Controller
@RequiredArgsConstructor
public class ChatController {

    private final SimpMessagingTemplate template; // 메시지 전송 도구
    private final ChatService chatService;

    // 클라이언트가 "/pub/chat/message"로 보낸 메시지를 받음
    @MessageMapping("/chat/message")
    public void message(ChatMessageDto message, Principal principal) {
        // [보안] 프론트에서 보낸 sender를 믿지 않고, JWT 인증 정보(Principal)에서 ID 추출
        String validSender = principal.getName();
        message.setSender(validSender);

        // 1. [실시간 전송] 대기 시간 없이 즉시 구독자들에게 전송
        // 목적지: /sub/chat/room/{roomId}
        template.convertAndSend("/sub/chat/room/" + message.getRoomId(), message);

        // 2. [비동기 저장] DB 저장은 백그라운드에서 처리
        chatService.saveMessageAsync(message);
    }
}

@Service
@RequiredArgsConstructor
public class ChatService {
    
    private final ChatRepository chatRepository;

    // [비동기 메서드]
    // 호출자는 기다리지 않고 즉시 리턴됨. 별도 스레드가 DB Insert 수행.
    @Async 
    public void saveMessageAsync(ChatMessageDto dto) {
        ChatLog entity = ChatLog.builder()
                .roomId(dto.getRoomId())
                .sender(dto.getSender())
                .message(dto.getMessage())
                .build();
        
        // Oracle DB에 저장 (JPA 사용 시 트랜잭션 주의)
        chatRepository.save(entity); 
    }
}
```

## 3. Frontend 구현 (React)
### 상세 주석이 포함된 컴포넌트
`useRef`와 `useEffect`의 역할을 이해하는 것이 핵심.
```javascript
import React, { useEffect, useRef, useState } from 'react';
import * as StompJs from '@stomp/stompjs'; // STOMP 프로토콜 클라이언트
import * as SockJS from 'sockjs-client';   // WebSocket 호환성 라이브러리
import axios from 'axios';

const ChatRoom = () => {
  // [상태 관리]
  const [chatList, setChatList] = useState([]); // 화면에 표시될 채팅 목록
  const [msg, setMsg] = useState('');           // 입력창 메시지
  
  // [Client 객체 유지]
  // useState가 아닌 useRef를 쓰는 이유: 
  // 리렌더링이 되어도 소켓 연결 객체가 초기화되지 않고 유지되어야 하기 때문.
  const client = useRef({});

  const roomId = "ROOM_001"; 
  const token = localStorage.getItem("accessToken"); // 로그인 시 저장한 JWT

  // [생명주기 관리] 컴포넌트가 마운트될 때 한 번 실행
  useEffect(() => {
    // 1. 기존 채팅 내역 불러오기 (REST API)
    fetchHistory();
    
    // 2. 소켓 연결 시작
    connect();

    // 3. 컴포넌트 언마운트(종료) 시 연결 끊기 (메모리 누수 방지)
    return () => disconnect();
  }, []);

  const fetchHistory = async () => {
     // DB에 저장된 내역을 HTTP GET으로 가져와서 chatList 초기화
     const res = await axios.get(`/api/chat/${roomId}`);
     setChatList(res.data);
  }

  const connect = () => {
    // STOMP 클라이언트 생성
    client.current = new StompJs.Client({
      // SockJS Factory 사용: ws:// 프로토콜 대신 http:// 주소 사용 (SockJS가 내부적으로 처리)
      webSocketFactory: () => new SockJS("http://localhost:8080/ws-stomp"),
      
      // [인증 헤더] 핸드쉐이크 및 CONNECT 프레임 전송 시 JWT 포함
      connectHeaders: {
        Authorization: token, 
      },
      
      // [연결 성공 시 콜백]
      onConnect: () => {
        console.log("STOMP 연결 성공");
        
        // [구독] 서버가 "/sub/chat/room/1"로 보내는 메시지를 듣기 시작
        client.current.subscribe(`/sub/chat/room/${roomId}`, (message) => {
          // 메시지가 오면 JSON 파싱 후 리스트에 추가
          const newChat = JSON.parse(message.body);
          setChatList(prev => [...prev, newChat]);
        });
      },
      
      // [에러 처리]
      onStompError: (frame) => {
        console.error("브로커 에러:", frame.headers['message']);
      },
    });

    // 클라이언트 활성화 (실제 연결 시도)
    client.current.activate();
  };

  const sendMessage = () => {
    if (!msg) return; // 빈 메시지 방지

    // [발행] 서버의 "/pub/chat/message"로 메시지 전송
    client.current.publish({
      destination: "/pub/chat/message",
      body: JSON.stringify({
        roomId: roomId,
        message: msg,
        type: 'TALK' // 메시지 타입 (입장/퇴장/대화)
      }),
    });
    
    setMsg(''); // 입력창 비우기
  };
  
  const disconnect = () => {
    // 연결이 되어있다면 끊기
    if (client.current) {
      client.current.deactivate();
    }
  }

  return (
    <div>
      {/* UI 렌더링 부분 */}
      <div className="chat-box">
        {chatList.map((chat, idx) => (
            <div key={idx}>[{chat.sender}] {chat.message}</div>
        ))}
      </div>
      <input value={msg} onChange={e => setMsg(e.target.value)} />
      <button onClick={sendMessage}>보내기</button>
    </div>
  );
};

export default ChatRoom;
```

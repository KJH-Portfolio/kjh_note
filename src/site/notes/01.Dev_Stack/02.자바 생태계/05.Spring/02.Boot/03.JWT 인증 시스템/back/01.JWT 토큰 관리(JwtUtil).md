# Spring Boot JwtUtil 구현 및 구성
> [!summary] 핵심 요약
> 
> - Spring Boot 환경에서의 JWT(JSON Web Token) 생성, 검증, 정보 추출 유틸리티 클래스 구현
>     
> - 목적: 세션 방식을 대체하여 서버 부하를 줄이고 시스템 확장성 및 안전한 인증 정보 전달 확보
>     
> - 핵심: `JwtUtil` 클래스의 4대 핵심 로직(생성, 검출, 검증, 만료 확인) 숙달 및 ==비밀키(Secret Key) 보안 관리==
>     
## JWT (JSON Web Token) 개요
- **JWT**: 사용자 인증 정보를 안전하게 전달하기 위한 웹 표준 통신 방식
- **도입 의도**: 세션 상태를 서버에 저장하지 않는 Stateless 아키텍처 구현

|**구성 요소**|**설명**|
|---|---|
|**Header** (헤더)|토큰의 타입(JWT) 및 사용된 암호화 알고리즘(예: HS256) 명시|
|**Payload** (페이로드)|사용자 식별자(ID), 권한 등 실제 전송할 데이터인 ==Claim(클레임)== 포함|
|**Signature** (서명)|헤더와 페이로드를 조합한 후 비밀키로 암호화하여 데이터 무결성 및 위변조 방지 보장|

## 클래스 설정 및 키 생성
- **@Component**: Spring 컨테이너가 싱글톤 객체로 관리하도록 빈(Bean) 등록
- **@Value**: `application.properties` 또는 `.yml` 파일에 정의된 환경 변수 주입
- **보안 주의**: 비밀키(`secret`)는 소스 코드에 하드코딩하지 않고 ==반드시 외부 환경변수로 분리==할 것
```java
@Component
public class JwtUtil {

	//비밀키
    @Value("${jwt.secret}")
    private String secret;
    
	//키 유효기간
    @Value("${jwt.expiration}")
    private long expiration;

	//키 발급 로직
    private SecretKey getSignKey() {
        // 문자열 기반 비밀키를 바이트 배열로 변환하여 HMAC-SHA 알고리즘용 키 객체 생성
        return Keys.hmacShaKeyFor(secret.getBytes());
    }
}
```
## 핵심 처리 로직 (Core Methods)
### 토큰 생성 (`generateToken`)
- **목적**: 사용자 식별 데이터(`userId`)를 기반으로 새로운 토큰 발급
- **구성**: 현재 시간 기준 발급일(`IssuedAt`)과 만료일(`Expiration`) 설정 후 비밀키로 서명(`signWith`)
```java
public String generateToken(String userId) {
    Date now = new Date();
    Date expiryDate = new Date(now.getTime() + expiration);

        return Jwts.builder()
                .setSubject(userId) // 토큰의 제목(사용자 ID) 설정
                .setIssuedAt(now) // 토큰 발행 시간 설정
                .setExpiration(expiryDate) // 토큰 만료 시간 설정
                .signWith(getSignKey()) // 생성한 SecretKey로 서명 수행
                .compact(); // 최종적으로 압축하여 문자열로 반환
}
```
### 토큰 유효성 검증 (`validateToken`)
- **목적**: 서명 일치 여부, 토큰 형식 준수, 만료 여부를 통합적으로 확인
- **안전성**: `JwtException` 예외 처리를 통해 ==모든 비정상/위변조 토큰 접근 차단==
```java
public boolean validateToken(String token) {
    try {
        Jwts.parserBuilder()
            .setSigningKey(getSignKey())
            .build()                     // 1. 토큰 해독기 생성
            .parseClaimsJws(token);      // 2. 해당 토큰 해독기로 서명 및 유효성 검증
        return true;
    } catch (JwtException e) {
        // 서명 불일치, 만료, 형식 오류 등 발생 시 false 반환
        return false;
    }
}
```
### 페이로드 정보 추출 (`getUserIdFromToken`)
- **목적**: 검증된 토큰에서 `Subject`로 등록했던 사용자 ID(`userId`) 획득
- **전제 조건**: 반드시 `validateToken`을 통해 ==유효성이 검증된 토큰에 한해서만 수행==할 것
```java
public String getUserIdFromToken(String token) {
    Claims claims = Jwts.parserBuilder()
                        .setSigningKey(getSignKey())
                        .build()
                        .parseClaimsJws(token)
                        .getBody();      // 페이로드(Claims) 추출
    return claims.getSubject();          // 페이로드에 등록된 userId 반환
}
```
### 토큰 만료 전용 확인 (`isTokenExpired`)
- **목적**: 토큰의 페이로드에 명시된 만료 시점(`Expiration`)과 현재 서버 시간 비교
- **활용**: 리프레시 토큰(Refresh Token) 발급 여부 판단 등의 세밀한 로직에 사용
```java
public boolean isTokenExpired(String token) {
    try {
        Claims claims = Jwts.parserBuilder()
                            .setSigningKey(getSignKey())
                            .build()                      //토큰 해독기
                            .parseClaimsJws(token)
                            .getBody();
        return claims.getExpiration().before(new Date()); // 현재 시간 이전이면 만료(true)
    } catch (JwtException e) {
        return false;
    }
}
```

# WebSocket JWT 인증 및 핸드셰이크 인터셉터 구현
> [!summary] 요약: 웹소켓 연결 수립 과정에서 클라이언트의 JWT 토큰 검증 및 보안 컨텍스트 데이터 전이 수행.
---
## 클래스 선언 및 핵심 의존성
- **JwtHandShakeInterceptor**
	- 웹소켓 핸드셰이크 요청 가로채기 및 사용자 인증 인터셉터 클래스.
- **HandshakeInterceptor**
	- 웹소켓 연결 전후의 로직 제어를 위한 스프링 표준 인터페이스.
- **JwtUtil**
	- 토큰 파싱, 유효성 검증 및 클레임 데이터 추출 유틸리티.
```java
package com.kh.spring.common.interceptor;

import ...

@Component
public class JwtHandShakeInterceptor implements HandshakeInterceptor {

    @Autowired
    private JwtUtil jwtUtil; // JWT 검증 및 데이터 추출을 위한 컴포넌트 주입
```

## beforeHandshake 상세 로직
- **요청 객체 처리**
	- HTTP 요청(`ServerHttpRequest`)의 서블릿 API 접근을 위한 `ServletServerHttpRequest` 변환.
- **토큰 획득 및 검증**
	- 쿼리 스트링(`?token=...`) 기반의 인증 토큰 식별 및 존재 유무 판단.
- **보안 컨텍스트 속성 등록**
	- `attributes` 맵 내 사용자 ID 저장 및 `WebSocketSession` 데이터 공유 활성화.
- **연결 제어 정책**
	- 유효 토큰 확인 시 핸드셰이크 승인 및 미인증 요청 차단 정책 적용.
```java
    // 1. 웹소켓 연결 시도 시 핸들러 도달 전 실행되는 사전 인증 메서드
    @Override
    public boolean beforeHandshake(ServerHttpRequest request, ServerHttpResponse response, WebSocketHandler wsHandler,
                                   Map<String, Object> attributes) throws Exception {

        // 2. HTTP 서블릿 기반 요청 여부 확인 및 다운캐스팅 진행
        if (request instanceof ServletServerHttpRequest) {
            ServletServerHttpRequest servletRequest = (ServletServerHttpRequest) request;

            // 3. 쿼리 파라미터 영역에서 "token" 명칭의 토큰값 획득
            // // 프론트엔드 연결 주소 예시: ws://server/chat?token=jwt_token_string
            String token = servletRequest.getServletRequest().getParameter("token");

            // 4. 토큰 실재 여부 및 유효성 검증 프로세스
            if (token != null) {
                // // JwtUtil 사용: 토큰 복호화 및 페이로드 내 사용자 ID(Subject) 추출
                String userId = jwtUtil.getUserIdFromToken(token);
                
                // 5. 웹소켓 세션 속성(Attributes)에 사용자 식별 정보 바인딩
                // // 해당 데이터는 WebSocketHandler의 session.getAttributes()에서 참조 가능
                attributes.put("userId", userId);
                
                // 6. 연결 허용: 다음 단계인 웹소켓 핸들러로 제어권 이양
                return true;
            }
        }

        // 7. 연결 거부: 토큰 미포함 또는 인증 실패 시 핸드셰이크 중단 처리
        return false;
    }
```

## afterHandshake 및 마무리
- **연결 후속 처리**
	- 핸드셰이크 완료 및 프로토콜 업그레이드 성공 이후의 추가 조작 구역.
- **예외 관리**
	- 핸드셰이크 과정 중 발생한 예외 상황 기록 및 로그 분석 지원.
```java
    // 1. 핸드셰이크 프로세스 완료 및 연결 수립 후 실행 구역
    @Override
    public void afterHandshake(ServerHttpRequest request, ServerHttpResponse response, WebSocketHandler wsHandler,
                               Exception exception) {
        // // 연결 성공 후의 추가 로직이나 특정 예외 발생 시의 리소스 정리 수행 구역
    }
}
```

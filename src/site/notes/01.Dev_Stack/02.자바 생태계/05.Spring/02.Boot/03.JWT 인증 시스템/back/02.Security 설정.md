# Spring Security 설정 및 JWT 필터 체인 통합
> [!summary] 요약: Spring Security 6.x 환경에서의 무상태(Stateless) 보안 정책 수립, CORS 허용 범위 정의 및 JWT 인증 필터의 우선 순위 배치 설정.
---
## 보안 필터 체인 구성
- **CSRF 및 세션 관리 정책**
	- **CSRF 비활성화**: JWT 토큰을 사용하여 요청을 인증하므로 서버 측 세션 상태를 유지하지 않는 특성에 따른 보안 설정 해제.
	- **Stateless 세션**: `SessionCreationPolicy.STATELESS` 설정을 통한 서버 내 세션 생성 및 참조 원천 차단.
- **인가(Authorization) 규칙 정의**
	- **무허가 접근 경로**: 회원가입(`/member/register`), 로그인(`/member/login`), 중복 체크, 정적 리소스 등 인증 미요구 경로 지정.
	- **인증 요구 경로**: 지정된 경로 외 모든 HTTP 요청에 대한 유효한 인증 객체 보유 필수화.
- **필터 순서 및 통합**
	- **addFilterBefore**: 표준 아이디/비밀번호 인증 필터 수행 전, 커스텀 **JwtAuthenticationFilter**를 배치하여 토큰 검증 우선 수행.
---
## CORS 및 암호화 인프라 설정
- **교차 출처 자원 공유(CORS)**
	- **허용 출처**: 리액트(React) 프론트엔드 서버 주소(`http://localhost:3000`)에 대한 명시적 허용.
	- **허용 메서드**: GET, POST, PUT, DELETE, OPTIONS 등 REST API 필수 메서드 개방.
	- **자격 증명 허용**: `setAllowCredentials(true)` 설정을 통한 클라이언트 측 쿠키 및 인증 헤더 수신 허용.
- **비밀번호 보안**
	- **BCryptPasswordEncoder**: 강력한 해시 알고리즘인 BCrypt 기반의 단방향 암호화 빈(Bean) 등록.

## SecurityConfig 전체 소스 코드
```java
import ...

@Configuration
@EnableWebSecurity // Spring Security 활성화 및 웹 보안 설정 클래스 정의 _ 모든 HTTP 요청을 감시
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtAuthenticationFilter;

    // 생성자 주입을 통한 JwtAuthenticationFilter 의존성 확보
    public SecurityConfig(JwtAuthenticationFilter jwtAuthenticationFilter) {
        this.jwtAuthenticationFilter = jwtAuthenticationFilter;
    }

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            // 1. CORS 설정 적용: 별도 정의한 corsConfigurationSource 사용
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            
            // 2. CSRF 보호 비활성화: REST API 서버는 stateless하므로 csrf 공격 위험이 적음
            .csrf(csrf -> csrf.disable())
            
            // 3. 세션 관리 정책: 세션을 생성하거나 사용하지 않도록 STATELESS 설정
            .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            
            // 4. 요청별 인가 설정
            .authorizeHttpRequests(auth -> auth
                // 회원 관련 기능 및 파일 업로드 경로는 인증 없이 접근 허용
                .requestMatchers("/member/login", "/member/register", "/member/checkId/**", "/spring/upload/**").permitAll()
                // 그 외 모든 경로는 Spring Security 컨텍스트 내 인증 정보 필요
                .anyRequest().authenticated()
            )
            
            // 5. 필터 체인 커스텀: UsernamePasswordAuthenticationFilter 실행 직전에 JWT 필터 수행
            // - 유효한 토큰이 확인되면 SecurityContext에 인증 정보가 설정되어 다음 필터 통과
            .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    // CORS 설정을 위한 빈 등록 (프론트엔드-백엔드 간 자원 공유 허용)
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();
        
        // 허용할 원격 출처(React 서버) 설정
        config.setAllowedOrigins(List.of("http://localhost:3000"));
        // 허용할 HTTP 메서드 목록 설정
        config.setAllowedMethods(List.of("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        // 모든 HTTP 헤더 수용 설정
        config.setAllowedHeaders(List.of("*"));
        // 쿠키 및 인증 헤더를 포함한 요청 허용 설정
        config.setAllowCredentials(true);

        // 모든 경로(/**)에 대해 위 설정 적용
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);
        return source;
    }
    
    // 사용자의 비밀번호 암호화를 수행할 인코더 빈 등록
    @Bean
    public PasswordEncoder passwordEncoder() {
        // BCrypt 알고리즘을 사용한 강력한 해싱 제공
        return new BCryptPasswordEncoder();
    }
}
```

### 노트 내용 수정/추가/삭제 사항 안내
- **추가**: `HttpSecurity` 설정 내 각 메서드 체이닝에 대한 동작 원리 주석 강화.
- **추가**: CORS 설정의 보안적 의미(Credentials 허용 등)에 대한 설명 보완.
### 노트 내용 수정/추가/삭제 제안
- **제안**: 개발 환경과 운영 환경의 CORS 주소를 분리하기 위한 `@Value` 기반 프로퍼티 주입 권장.
- **제안**: 특정 권한(Role)에 따른 접근 제한이 필요한 경우 `.requestMatchers().hasRole()` 설정 추가 검토.

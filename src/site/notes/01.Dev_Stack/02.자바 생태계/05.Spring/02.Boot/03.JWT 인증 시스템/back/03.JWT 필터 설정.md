# JwtAuthenticationFilter 기반 Spring Security 인증 파이프라인 (Early Return 적용)
> [!summary] 핵심 요약
> 
> - HTTP 요청 헤더의 JWT 토큰 검증 및 Spring Security 컨텍스트 내 인증 정보 등록을 수행하는 필터 계층
>     
> - 목적: 클라이언트의 무상태(Stateless) JWT 요청을 시큐리티 인증 체계와 연결하여 API 보안 보장
>     
> - 핵심: ==OncePerRequestFilter== 상속을 통한 단일 실행 보장 및 ==Early Return 패턴==을 적용한 가독성 높은 필터 로직 구현
>     
## 주요 구성 요소 및 역할 분리
- **필터 계층 역할**
    - 스프링 시큐리티의 필터 체인에 개입하여 토큰 유효성을 평가하는 핵심 컴포넌트이며, 각 객체의 성격에 따라 '시큐리티 인프라'와 '인증/토큰 관리 도구'로 역할을 구분함
- **Spring Security 인프라 요소 (Core Infrastructure)**
    - **OncePerRequestFilter**: 동일한 HTTP 요청 내에서 ==단 한 번의 필터 실행==을 보장하는 스프링 시큐리티 기반 추상 클래스
    - **SecurityContextHolder**: 최종적으로 인증된 사용자의 Principal 정보 및 권한(Authorities)을 보관하는 ==전역 컨텍스트 저장소==
- **인증 및 토큰 관리 도구 (Auth & Token Management)**
    - **JwtUtil**: 토큰 문자열 파싱, 사용자 식별자(ID) 추출, 서명 유효성 및 만료 시간 검증 로직을 수행하는 ==커스텀 유틸리티==
    - **UserDetailsService**: 데이터베이스 등 저장소로부터 실제 사용자 정보를 조회하여 시큐리티 전용 `UserDetails` 객체를 반환하는 ==인증 데이터 제공 인터페이스==
## JWT 인증 프로세스 흐름도
```plaintext
JWT Authentication Flow (Early Return Pattern)
 ├─ 1. Header Check
 │   ├─ 없음 or 'Bearer ' 아님 -> [다음 필터 호출 후 종료 (return)]
 │   └─ 정상 -> [진행]
 ├─ 2. Token Parsing
 │   ├─ 파싱 실패(예외) -> [로그 출력 -> 다음 필터 호출 후 종료 (return)]
 │   └─ 파싱 성공(userId 획득) -> [진행]
 ├─ 3. Context & Validity Check
 │   ├─ 이미 인증됨 or 유효하지 않은 토큰 -> [다음 필터 호출 후 종료 (return)]
 │   └─ 정상 -> [진행]
 └─ 4. Authentication Registration
     ├─ UserDetailsService로 DB 사용자 조회 (UserDetails)
     ├─ UsernamePasswordAuthenticationToken 생성
     └─ SecurityContextHolder 등록 -> [다음 필터 호출 후 종료]
```
## 필터 로직 단계별 분해 (Early Return 리팩토링)
- **전체 흐름 요약**
    - 기존의 중첩된 `if`문(Depth)을 제거하고, ==조건에 맞지 않으면 즉시 필터 체인을 호출하고 종료(`return`)==하는 평탄화된 로직 적용
    - 주의: 필터 로직을 조기 종료할 때도 다음 필터로 요청이 넘어가도록 반드시 `filterChain.doFilter`를 선행 호출해야 함
### 1. HTTP 헤더 추출 및 예외 조기 종료 (Early Return)
- **동작 원리**
    - 헤더가 없거나 ==Bearer ==(공백 포함 필수)로 시작하지 않으면 해당 필터의 역할을 즉시 종료함
```java
// 1. HTTP 요청 헤더 추출 및 검증
String authHeader = request.getHeader("Authorization");

// [Early Return] 헤더가 없거나 규칙에 맞지 않으면 다음 필터로 넘기고 로직 종료
if (authHeader == null || !authHeader.startsWith("Bearer ")) {
    filterChain.doFilter(request, response);
    return;
}
```
### 2. Bearer 토큰 분리 및 사용자 ID 추출
- **동작 원리**
    - 접두사 7자리를 잘라내어 순수 JWT 문자열만 추출
    - 파싱 실패 시 서버가 다운되지 않도록 예외를 잡고 다음 필터로 제어권을 넘김
```java
// 2. 토큰 문자열 획득 및 사용자 ID 파싱
String token = authHeader.substring(7); // "Bearer " 제거
String userId = null;

try {
    userId = jwtUtil.getUserIdFromToken(token);
} catch (Exception e) {
    logger.error("JWT 토큰 파싱 실패: " + e.getMessage());
    // [Early Return] 파싱 실패 시 인증 불가 상태로 다음 필터 진행
    filterChain.doFilter(request, response);
    return;
}
```
### 3. 인증 상태 및 유효성 사전 검증
- **동작 원리**
    - 식별자가 없거나, ==이미 스레드에 인증 객체가 존재==하거나, ==토큰이 유효하지 않은 경우== 즉시 종료 처리 (불필요한 DB 조회 방지)
```java
// 3. 사용자 식별자, 기존 인증 여부 및 토큰 유효성 검증
if (userId == null || SecurityContextHolder.getContext().getAuthentication() != null || !jwtUtil.validateToken(token)) {
    // [Early Return] 조건 미충족 시 더 이상 진행하지 않음
    filterChain.doFilter(request, response);
    return;
}
```
### 4. UserDetails 로드 및 인증 객체 생성
- **동작 원리**
    - 모든 예외 케이스를 통과한 '안전한' 상태에서 DB 조회를 수행함
    - `UsernamePasswordAuthenticationToken` 생성 시 ==비밀번호는 null==로 전달
```java
// 4. 안전한 상태에서 DB 내 사용자 정보 로드 및 인증 객체 생성
UserDetails userDetails = this.userDetailsService.loadUserByUsername(userId);

UsernamePasswordAuthenticationToken authToken = new UsernamePasswordAuthenticationToken(
        userDetails,
        null,
        userDetails.getAuthorities()
);
```
### 5. 세부 정보 결합 및 SecurityContext 등록
- **동작 원리**
    - 인증 객체에 원본 HTTP 요청의 세부 정보 추가 후 `SecurityContextHolder`에 세팅
    - 최종적으로 인증 처리가 완료되었으므로 체인을 이어감
```java
// 5. 요청 기반 상세 데이터 설정 및 전역 컨텍스트 등록
authToken.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
SecurityContextHolder.getContext().setAuthentication(authToken);

// 6. 정상 처리 완료 후 다음 필터로 제어권 넘김
filterChain.doFilter(request, response);
```
## CustomUserDetailsService 연결 로직
- **CustomUserDetailsService 구현**
    - 필터에서 호출하여 DB의 사용자 엔티티를 시큐리티가 이해할 수 있는 `UserDetails`로 변환 담당
```java
@Service
@RequiredArgsConstructor
public class CustomUserDetailsService implements UserDetailsService {

    private final MemberRepository memberRepository;

    @Override
    public UserDetails loadUserByUsername(String userId) throws UsernameNotFoundException {
        // 1. DB 조회 및 예외 처리
        Member member = memberRepository.findById(userId)
                .orElseThrow(() -> new UsernameNotFoundException("사용자를 찾을 수 없습니다: " + userId));

        // 2. UserDetails 객체 빌드 및 반환
        return User.builder()
                .username(member.getUserId())
                .password(member.getPassword()) 
                .roles(member.getRole())        
                .build();
    }
}
```

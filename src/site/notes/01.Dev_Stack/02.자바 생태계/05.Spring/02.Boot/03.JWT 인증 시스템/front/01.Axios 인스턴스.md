# Axios 인스턴스 설정 및 인터셉터 관리
> [!summary]
> 
> **Axios 라이브러리**를 기반으로 공통 API 통신 환경을 구축하고, **요청(Request) 및 응답(Response) 인터셉터**를 통해 JWT 토큰의 자동 주입과 전역적인 HTTP 에러 상태 관리 시스템의 구축.
---
## Axios 인스턴스 생성 및 기본 환경 설정
- **Base Configuration**
    - **baseURL**: 모든 API 요청의 기본 접두사 경로(`/spring`) 설정.
    - **timeout**: 서버 응답 대기 최대 시간(10,000ms) 지정.
    - **headers**: 데이터 전송 표준 규격(`application/json`) 정의.
---
## 요청 인터셉터 (Request Interceptor): 인증 처리
- **JWT 토큰 자동 주입 로직**
    - **로컬 스토리지 조회**: 요청 직전 `localStorage` 내 저장된 인증 토큰(`token`) 확인.
    - **Authorization 헤더 구성**: 토큰 존재 시 `Bearer` 타입의 인증 정보 자동 추가.
    - **요청 전파**: 수정된 `config` 객체 반환을 통한 실제 서버 요청 진행.
---
## 응답 인터셉터 (Response Interceptor): 에러 대응
- **HTTP 상태 코드별 전역 처리**
    - **401 (Unauthorized)**: 토큰 만료 시 로컬 인증 정보(`token`, `user`) 삭제 및 로그아웃 강제화.
    - **403 (Forbidden)**: 접근 권한 부족에 대한 보안 에러 메시지 출력.
    - **404 (Not Found)**: 존재하지 않는 리소스 요청에 대한 예외 처리.
    - **500 (Server Error)**: 서버 내부 로직 오류에 대한 관리자 안내 메시지 출력.
    - **네트워크 오류**: 서버 미응답 또는 연결 끊김 현상에 대한 예외 로직.
---
## axios.js 상세 주석 코드 전문
```javascript
// Axios 라이브러리 임포트
import axios from "axios";

// 공통 설정을 포함한 커스텀 Axios 인스턴스 생성
const api = axios.create({
    // 백엔드 서버 API 기본 경로 설정
    baseURL : '/spring',
    // 요청 후 응답까지의 최대 대기 시간 (10초)
    timeout : 10000,
    // 기본 요청 헤더: 데이터 형식을 JSON으로 고정
    headers : {
        'Content-Type' : 'application/json'
    }
});

// 요청 인터셉터 설정: 서버로 요청이 발송되기 전 실행
api.interceptors.request.use(
    (config) => {
        // 브라우저 로컬 스토리지에서 저장된 인증 토큰 추출
        const token = localStorage.getItem('token');
        // 토큰이 존재할 경우 요청 헤더의 Authorization 항목에 Bearer 토큰 주입
        if(token){
            config.headers.Authorization = `Bearer ${token}`;
        }
        // 설정이 완료된 config 객체 반환 (요청 진행)
        return config;
    },
    (error) => {
        // 요청 과정에서 발생한 에러를 프로미스 거절로 반환
        return Promise.reject(error);
    }
);

// 응답 인터셉터 설정: 서버 응답 수신 직후 처리 로직
api.interceptors.response.use(
    (response) => {
        // 성공적인 응답(2xx 범위)은 그대로 반환
        return response;
    },
    (error) => {
        // 에러 객체에서 서버 응답 데이터 추출
        const {response} = error;

        // 서버 응답이 존재하는 경우 상태 코드별 분기 처리
        if(response){
            switch(response.status) {
                // 401: 인증 실패 또는 토큰 만료 시 세션 데이터 정리
                case 401 : 
                    localStorage.removeItem('token');
                    localStorage.removeItem('user');
                    //로그인 페이지 강제 리다이렉트 처리 가능(로그아웃)
                    window.location.href = '/login';
                break;
                // 403: 접근 권한이 없는 리소스 요청 시 에러 출력
                case 403 : 
                    console.error('접근권한이 없습니다.');
                break;
                // 404: 요청 경로가 잘못되었거나 리소스가 존재하지 않음
                case 404 : 
                    console.error('요청한 리소스를 찾을 수 없습니다.');
                break;
                // 500: 서버 내부 로직 예외 발생 시 에러 출력
                case 500 : 
                    console.error('서버 오류가 발생했습니다.');
                break;
                // 기타 정의되지 않은 HTTP 에러 처리
                default : 
                    console.error('알 수 없는 오류가 발생했습니다.');
            }
        } else {
            // 서버 접속 불가 등 물리적인 네트워크 오류 처리
            console.error('네트워크 오류가 발생했습니다.');
        }

        // 컴포넌트 내 catch 구문에서 에러를 인지할 수 있도록 Reject 처리
        return Promise.reject(error);
    }
);

// 설정된 api 인스턴스 외부 공유
export default api;
```

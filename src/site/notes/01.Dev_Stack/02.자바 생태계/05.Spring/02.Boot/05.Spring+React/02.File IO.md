# 스프링 부트 및 리액트 기반 파일 I/O 시스템 설계 및 구현
> [!summary] 요약
> 
> 스프링 부트 FileUtil 컴포넌트 내 @Value 어노테이션 기반 경로 주입 및 리액트 FormData 연동을 통한 파일 업로드/다운로드 로직 설계.
---
## 백엔드 설계 (Spring Boot)
### 환경 설정 (Configuration)
- **설정 파일 정의** (`application.properties`)
    - 파일 저장 경로 변수 선언.
    - 멀티파트(Multipart) 파일 사이즈 제한 설정.
```properties
# 파일 업로드 경로 작성 - 별도의 파일 경로를 작성하여 해당 위치에 업로드 처리할 것
# 윈도우/맥 환경 모두 호환되도로 ${user.home} 이용
file.upload.path=${user.home}/uploads/

# Spring Servlet Multipart 설정
spring.servlet.multipart.max-file-size=10MB
spring.servlet.multipart.max-request-size=10MB
```
### WebMvcConfig 설정
- **정적 리소스 매핑**
    - `WebMvcConfigurer` 인터페이스 구현.
    - `addResourceHandlers` 오버라이딩을 통한 외부 경로 매핑.
    - 브라우저에서 `/images/파일명`으로 로컬 파일 직접 접근 허용.
- **경로 설정 주의사항**
    - `file:///` 프로토콜 접두사 필수 사용 (운영체제에 따른 경로 구분자 확인 필요).
```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Value("${file.upload.path}")
    private String uploadPath;

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        // /images/로 시작하는 요청을 로컬 업로드 경로로 매핑
        registry.addResourceHandler("/images/**")
                .addResourceLocations("file:///" + uploadPath);
    }
}
```
### FileUtil 클래스 구현
- **컴포넌트 등록**
    - `@Component` 어노테이션 사용으로 빈(Bean) 등록.
- **경로 주입 (핵심 요구사항)**
    - `@Value` 어노테이션을 통한 `application.yml` 설정 값 주입.
- **기능 메서드**
    - **파일 저장**: `MultipartFile` 객체 수신 및 `UUID` 활용 파일명 중복 방지 처리.
    - **파일 로드**: 파일명 기반 `Path` 객체 생성 및 `Resource` 반환.
```java
@Component
public class FileUtil {

    // 설정 파일의 경로 주입
    @Value("${file.upload.path}")
    private String filePath;

    // 파일 업로드 처리
    public String uploadFile(MultipartFile file) throws IOException {
        if (file.isEmpty()) {
            return null;
        }

        String originalFileName = file.getOriginalFilename();
        String uuid = UUID.randomUUID().toString();
        String savedFileName = uuid + "_" + originalFileName;

        File targetFile = new File(filePath + savedFileName);
        
        // 디렉토리 존재 여부 확인 및 생성
        if (!targetFile.getParentFile().exists()) {
            targetFile.getParentFile().mkdirs();
        }
		
		//저장
        file.transferTo(targetFile);

        return savedFileName;
    }
    
    // 파일 전체 경로 반환 (다운로드 시 사용)
    public String getFullPath(String fileName) {
        return filePath + fileName;
    }
}
```
### 컨트롤러 (Controller) 구성
- **API 엔드포인트 정의**
    - `POST` 요청 처리 및 `ResponseEntity` 반환.
- **FileUtil 의존성 주입**
    - 생성자 주입 방식을 통한 `FileUtil` 사용.
```java
@RestController
@RequestMapping("/api/files")
@RequiredArgsConstructor
public class FileController {

    private final FileUtil fileUtil;

    @PostMapping("/upload")
    public ResponseEntity<String> upload(@RequestParam("file") MultipartFile file) {
        try {
            String savedFileName = fileUtil.uploadFile(file);
            return ResponseEntity.ok(savedFileName);
        } catch (IOException e) {
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body("Fail");
        }
    }
}
```

## 프론트엔드 설계 (React)
### 파일 업로드 (Upload)
- **상태 관리**
    - `useState` 활용 선택 파일 관리.
- **FormData 활용**
    - `multipart/form-data` 전송을 위한 `FormData` 객체 생성.
    - `append` 메서드로 파일 객체 추가.
- **API 호출**
    - `axios` 활용 `POST` 요청 전송.
    - 헤더 설정: `Content-Type`: `multipart/form-data`.
```javascript
import React, { useState } from 'react';
import axios from 'axios';

const FileUpload = () => {
  const [file, setFile] = useState(null);
  
  //파일폼 체인지 핸들러
  const handleFileChange = (e) => {
    setFile(e.target.files[0]);
  };
  //파일 업로드 핸들러
  const handleUpload = async () => {
    if (!file) return;

    const formData = new FormData();
    formData.append('file', file);

    try {
      const response = await axios.post('/api/files/upload', formData, {
        headers: {
          'Content-Type': 'multipart/form-data',
        },
      });
    } catch (error) {
      console.error('Upload Error:', error);
    }
  };

  return (
    <div>
      <input type="file" onChange={handleFileChange} />
      <button onClick={handleUpload}>Upload</button>
    </div>
  );
};

export default FileUpload;
```
### 파일 다운로드 (Download)
- **Blob 처리**
    - `responseType: 'blob'` 설정 필수.
- **가상 링크 생성**
    - `window.URL.createObjectURL(new Blob([data]))` 활용 URL 생성.
    - `<a>` 태그 생성 및 `download` 속성 설정 후 클릭 이벤트 트리거.
```javascript
const handleDownload = async (fileName) => {
  try {
    const response = await axios.get(`/api/files/download/${fileName}`, {
      responseType: 'blob', // 중요 설정
    });

    const url = window.URL.createObjectURL(new Blob([response.data]));
    const link = document.createElement('a');
    link.href = url;
    link.setAttribute('download', fileName);
    document.body.appendChild(link);
    link.click();
    link.remove();
  } catch (error) {
    console.error('Download Error:', error);
  }
};
```
### 이미지 뷰어 (View) (추가)
- **정적 리소스 접근**
    - `WebMvcConfig`에서 설정한 경로 패턴(`/images/**`) 활용.
    - 서버 도메인 + 매핑 경로 + 파일명 조합으로 `src` 속성 바인딩.
```javascript
const ImageViewer = ({ fileName }) => {
  // 로컬 개발 환경 기준 URL 예시
  const imageUrl = `http://localhost:8080/images/${fileName}`;

  return (
    <div>
      {fileName && (
        <img 
            src={imageUrl} 
            alt="Uploaded Content" 
            style={{ maxWidth: '100%', height: 'auto' }} 
        />
      )}
    </div>
  );
};
```

# Spring Boot-React 연동 REST API 설계 및 데이터 관리 전략

> [!summary]
> **ResponseEntity**를 통한 HTTP 응답의 명시적 제어, **DTO** 기반의 계층 간 효율적 데이터 전송, **MyBatis 동적 SQL** 및 페이징 처리를 활용한 리액트-스프링 부트 간 표준 통신 규격 정리.

---

## RESTful API 인터페이스 원칙

> [!abstract] 리액트 등 클라이언트와의 일관된 통신 규격 확립을 위한 설계 기반

* **자원(Resource)의 명사화**
	* URI 내 동작(Verb) 배제 및 자원(Noun) 위주의 경로 구성.
* **HTTP Method 활용**
	* GET(조회), POST(생성), PUT(수정), DELETE(삭제)를 통한 행위 정의.
* **표현(Representation)**
	* 자원의 JSON(JavaScript Object Notation) 형태 반환을 통한 웹 표준 준수.
* **무상태성(Stateless)**
	* 서버 측 클라이언트 상태 정보 저장 지양 및 각 요청의 독립적 처리.
* **Uniform Interface**
	* HTTP 표준 상태 코드 사용을 통한 통신 가독성 및 일관성 확보.

## MVC 계층별 주요 어노테이션 및 역할

> [!info] 데이터 흐름 제어를 위한 계층별 책임 분리

* **Presentation Layer (@RestController)**
	* 데이터(JSON/XML) 반환 전용 컨트롤러 및 View 반환 로직 배제.
* **Routing (@RequestMapping)**
	* 클래스 레벨의 공통 경로 지정을 통한 엔드포인트 구조화.
* **Business Layer (@Service)**
	* 트랜잭션(@Transactional) 경계 설정 및 핵심 비즈니스 로직 수행.
* **Persistence Layer (@Mapper / @Repository)**
	* MyBatis 인터페이스 매핑 및 데이터베이스 접근 전담.

## ResponseEntity 기반 HTTP 응답 상태 관리

> [!tip] 리액트 프론트엔드에서의 예외 처리 및 분기 로직 지원을 위한 상태 제어

* **ResponseEntity 객체 정의**
	* HTTP 응답의 세 요소(**상태 코드, 헤더, 바디**)를 직접 제어하는 Spring 프레임워크 클래스.
	* `ResponseEntity<?>`값을 리턴함

* **성공(2xx) 상태 코드**:
	* **200 OK**: 요청의 정상 처리 및 데이터 반환 성공.
	* **201 Created**: 리소스 생성 성공 및 생성된 위치(URI) 정보 제공.
	* **204 No Content**: 요청 수행 성공 및 반환 데이터 부재(삭제 등).
* **클라이언트 오류(4xx) 상태 코드**:
	* **400 Bad Request**: 요청 파라미터 누락 또는 구문 오류 발생.
	* **401 Unauthorized**: 인증 정보 미비에 따른 접근 거부.
	* **403 Forbidden**: 리소스 접근 권한 부족.
	* **404 Not Found**: 요청한 리소스의 부재 확인.
	* **409 Conflict**: 서버의 현재 상태와 요청 간의 충돌 발생(데이터 중복 등).
* **서버 오류(5xx) 상태 코드**:
	* **500 Internal Server Error**: 서버 내부 로직 처리 중 미처리 예외 발생.

## 통합 구현 상세 (Full-Stack Flow)

### 1. Data Transfer Object (DTO) 및 페이징 모델

* **BoardDTO**: 클라이언트 요청 및 응답을 위한 데이터 전송 객체.
* **BoardListDTO**: 리액트 페이징 처리를 위한 목록 데이터 및 페이지 정보(PageInfo) 통합 객체.

```java
// BoardDTO.java (요청/응답 통합 모델)
@NoArgsConstructor
@AllArgsConstructor
@Data
@Builder
public class BoardDTO {
    private int boardNo;
    private String boardTitle;
    private String boardContent;
    private int version; // 낙관적 락을 위한 버전 관리
}

// BoardListDTO.java (React 응답 전용)
@NoArgsConstructor
@AllArgsConstructor
@Data
@Builder
public class BoardListDTO {
    private List<BoardDTO> list;
    private PageInfo pi;
}
```

### 2. Controller 레이어 (API 엔드포인트)

* **@PathVariable**: 주소 경로에 포함된 값을 변수로 추출.
* **@RequestBody**: 요청 본문의 JSON 데이터를 자바 객체로 역직렬화.
* **@RequestParam(defaultValue)**: 쿼리 스트링 데이터 수신 및 누락 시 기본값 적용.

```java
@Slf4j
@RestController
@RequestMapping("/api/v1/board") 
public class BoardController {

    @Autowired
    private BoardService boardService;

    @PutMapping("/{no}") 
    public ResponseEntity<String> updateBoard(@PathVariable("no") int no, @RequestBody BoardDTO dto) {
        dto.setBoardNo(no);
        int result = boardService.updateBoard(dto);
        
        return result > 0 ? 
               ResponseEntity.ok("수정 성공") : 
               ResponseEntity.status(HttpStatus.CONFLICT).body("데이터 충돌 발생");
    }

    @GetMapping
    public ResponseEntity<BoardListDTO> getBoardList(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size) {
        return ResponseEntity.ok(boardService.selectBoardList(page, size));
    }
}

```


---
### 3. Service 레이어 (비즈니스 로직 및 트랜잭션)

* **@Transactional**: 선언적 트랜잭션 처리를 통한 데이터 정합성 보장.
* **Pagination**: 공통 페이징 유틸리티를 활용한 페이지 정보 산출.

```java
@Service
public class BoardServiceImpl implements BoardService {
    @Autowired
    private BoardMapper boardMapper;

    @Override
    @Transactional 
    public int updateBoard(BoardDTO dto) {
        return boardMapper.updateBoard(dto);
    }

    @Override
    public BoardListDTO selectBoardList(int page, int size) {
        int totalCount = boardMapper.selectTotalCount();
        PageInfo pi = Pagination.getPageInfo(totalCount, page, 5, size);
        List<BoardDTO> list = boardMapper.selectBoardList(pi);
        return new BoardListDTO(list, pi);
    }
}

```

### 4. Mapper 인터페이스 및 XML (Persistence Layer)

* **@Mapper**: MyBatis 인터페이스 매핑 기능을 통한 SQL 호출 구조화.
* **동적 쿼리**: `<set>` 및 `<if>` 태그를 이용한 선택적 필드 업데이트.

```java
@Mapper 
public interface BoardMapper {
    int updateBoard(BoardDTO dto);
    int selectTotalCount();
    List<BoardDTO> selectBoardList(PageInfo pi);
}

```

```xml
<update id="updateBoard" parameterType="BoardDTO">
    UPDATE board
    <set>
        <if test="boardTitle != null and boardTitle != ''">
            board_title = #{boardTitle},
        </if>
        <if test="boardContent != null">
            board_content = #{boardContent},
        </if>
        board_modify_date = NOW(),
        version = version + 1
    </set>
    WHERE board_no = #{boardNo}
    AND version = #{version} 
</update>

```

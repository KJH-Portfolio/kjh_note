# 자바 스트림(java.io) 입출력 아키텍처
> [!summary] 핵심 요약
> 
> - **개념**: 파일, 네트워크, 메모리 등 다양한 외부 및 내부 자원과 데이터를 주고받기 위한 단방향 데이터 통로
>     
> - **왜?**: 입출력 대상 물리 매체(하드디스크, 램, 랜카드)가 달라도 애플리케이션에서는 일관된 API로 데이터를 조작하기 위함
>     
> - **언제?**: 외부 파일 읽기/쓰기, 네트워크 소켓 통신, 애플리케이션 메모리 내부의 임시 데이터 처리 시 사용
>     
> - **핵심**: 데이터의 종류(바이트/문자)에 따른 올바른 클래스 선택과 성능 향상을 위한 ==보조 스트림(Buffer)의 활용==
>     
## 스트림의 기본 분류 및 특징
- **바이트 기반 스트림 (Byte Stream)**
    - 1바이트 단위로 데이터를 처리하며 이미지, 영상, 실행 파일 등 이진(Binary) 데이터 처리에 필수적
    - `read()` 반환값 및 `write(int)` 연산 시 데이터의 하위 8비트 동작에 대한 명확한 이해 필요
- **문자 기반 스트림 (Character Stream)**
    - 2바이트(유니코드) 단위로 처리하며 순수 텍스트 데이터 처리에 특화
    - 자원(OS)에 따라 문자 표현 방식이 다르므로 ==정확한 인코딩(Encoding) 명시==가 필수적
```plaintext
[ Stream Classification ]

   단위             입력(Input)                 출력(Output)
 ┌──────┐        ┌─────────────────────┐     ┌──────────────────────┐
 │ Byte │ ─────▶ │ InputStream         │ ──▶ │ OutputStream         │
 ├──────┤        ├─────────────────────┤     ├──────────────────────┤
 │ Char │ ─────▶ │ Reader              │ ──▶ │ Writer               │
 └──────┘        └─────────────────────┘     └──────────────────────┘
```

|**분류**|**입력 최상위 클래스**|**출력 최상위 클래스**|**처리 단위**|**예시 하위 클래스**|
|---|---|---|---|---|
|**바이트 기반**|`InputStream`|`OutputStream`|바이트 (8-bit)|`FileInputStream`, `DataOutputStream`|
|**문자 기반**|`Reader`|`Writer`|문자 (Unicode)|`FileReader`, `BufferedReader`|

## 대상(Target)에 따른 스트림의 종류
- **파일 입출력 (File I/O)**
    - 하드 디스크의 파일 시스템과 직접 연결되어 데이터를 읽고 씀 (`FileInputStream`, `FileWriter` 등)
- **메모리 입출력 (Memory I/O)**
    - 데이터 소스나 목적지로 외부 장치 대신 애플리케이션 내부의 `byte[]` 배열을 사용
    - 별도의 외부 하드웨어(디스크, 네트워크 I/O) 연결 없이 ==RAM 내부에서만 동작==하므로 처리 속도가 매우 빠름
    - **어떻게?**: 네트워크 패킷 조립, 임시 데이터 암호화 등 빠르게 메모리 버퍼를 조작해야 할 때 `ByteArrayInputStream`과 `ByteArrayOutputStream` 활용
```java
import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;

public class MemoryStreamExample {
    public static void main(String[] args) {
        byte[] inSrc = {0, 1, 2, 3, 4}; // 메모리 내부 원본 배열
        byte[] outSrc = null;

        // 1. 메모리 기반 입출력 스트림 생성
        try (ByteArrayInputStream bais = new ByteArrayInputStream(inSrc);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            int data;
            // 2. RAM 내부 배열에서 읽어와 다른 배열 통로로 출력
            while ((data = bais.read()) != -1) {
                baos.write(data); 
            }
            
            // 3. 출력 스트림에 모인 데이터를 새로운 byte 배열로 반환
            outSrc = baos.toByteArray();
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
## 데이터 처리 계층 (기반 vs 보조)
- **기반 스트림 (Base Stream)**
    - 실제 데이터 소스(파일, 메모리 배열 등)와 직접 연결되어 입출력을 수행하는 1차 통로
- **어댑터 스트림 (Adapter Stream)**
    - 바이트 스트림을 문자 스트림으로 이어주는 연결 고리 역할
    - 외부 시스템과의 인코딩(UTF-8 등) 형식을 맞추기 위해 필수적으로 사용
- **보조 스트림 (Filter Stream)**
    - 기반 스트림을 감싸서 처리 속도(버퍼링)나 기능(원시 타입 처리)을 확장
    - 단독으로 입출력 대상을 지정할 수 없음
```plaintext
[ Stream Wrapper Pattern ]

  Raw Data (Byte)            Adapter (Char)            Buffer (Char)
 ┌───────────────┐         ┌───────────────┐         ┌───────────────┐
 │FileInputStream│ ──────▶ │InputStrmReader│ ──────▶ │BufferedReader │
 └──────┬────────┘         └──────┬────────┘         └───────┬───────┘
        │                         │                          │
  파일에서 바이트 읽기     문자열로 인코딩(UTF-8)      메모리 버퍼링(속도 향상)
```
- **어떻게?**: 파일 텍스트를 읽을 때 `FileInputStream`을 어댑터로 감싸 인코딩을 지정하고, 최종적으로 `BufferedReader`로 감싸 성능 최적화 및 줄 단위 읽기 수행
```java
import java.io.BufferedReader;
import java.io.FileInputStream;
import java.io.InputStreamReader;
import java.io.IOException;
import java.nio.charset.StandardCharsets;

public class StreamLayerExample {
    public static void main(String[] args) {
        // 기반 -> 어댑터 -> 보조 스트림 래핑 패턴
        try (BufferedReader br = new BufferedReader(
                 new InputStreamReader(
                     new FileInputStream("sample.txt"), StandardCharsets.UTF_8))) {
            
            String line;
            while ((line = br.readLine()) != null) { // 버퍼를 활용해 한 줄씩 빠르게 읽기
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

|**구분**|**역할**|**대표 클래스 예시**|
|---|---|---|
|**기반 스트림**|실제 데이터 소스/대상과 연결되어 직접 입출력 수행|`FileInputStream`, `ByteArrayOutputStream`|
|**어댑터**|바이트 데이터를 지정된 인코딩에 맞춰 문자로 변환|`InputStreamReader`, `OutputStreamWriter`|
|**보조 스트림**|스트림을 래핑하여 버퍼링, 자료형 처리 등 성능 및 기능 확장|`BufferedReader`, `DataInputStream`|

## 주요 클래스별 핵심 메소드 스펙
### 1. 바이트 스트림 주요 메소드 (`InputStream` / `OutputStream`)

|**메소드**|**시그니처**|**동작 원리 및 설명**|
|---|---|---|
|**1바이트 읽기/쓰기**|`int read()` / `void write(int b)`|1바이트를 읽거나(반환 0~255, 끝이면 -1) 씀(하위 8비트만 기록)|
|**배열 읽기/쓰기**|`int read(byte[] b)` / `void write(byte[] b)`|배열 크기만큼 묶어서 처리하여 I/O 성능 향상|
|**건너뛰기**|`long skip(long n)`|입력 스트림에서 `n` 바이트를 무시하고 진행|
|**가용량 확인**|`int available()`|스트림 블로킹(대기) 없이 즉시 읽을 수 있는 잔여 바이트 수 반환|

### 2. 문자 스트림 주요 메소드 (`Reader` / `Writer`)

|**메소드**|**시그니처**|**동작 원리 및 설명**|
|---|---|---|
|**문자 읽기/쓰기**|`int read()` / `void write(int c)`|1문자(유니코드)를 읽거나(반환 0~65535, 끝이면 -1) 씀(하위 16비트 기록)|
|**문자열 쓰기**|`void write(String str)`|문자열 데이터 전체를 스트림으로 한 번에 출력|
|**라인 읽기**|`String readLine()`|`BufferedReader` 전용. 개행 문자를 기준으로 한 줄을 통째로 읽어 반환|
|**줄바꿈**|`void newLine()`|`BufferedWriter` 전용. 운영체제(OS)에 맞는 안전한 줄바꿈 문자 삽입|

## 💡 필수 주의 사항
- **자원 해제**: 사용이 끝난 스트림은 OS 자원 누수 방지를 위해 반드시 `close()` 해야 하며, 이를 보장하기 위해 ==try-with-resources 구문 적용==이 표준
- **버퍼 강제 전송**: 출력 스트림(`OutputStream`, `Writer`) 내부의 버퍼가 가득 차지 않았더라도 데이터를 즉시 전송해야 할 경우 ==`flush()`를 명시적으로 호출==해야 함
- **단방향성**: 스트림은 기본적으로 읽기 전용이거나 쓰기 전용의 단방향이므로, 동시 작업이 필요할 경우 입력과 출력 스트림 인스턴스를 각각 분리하여 생성해야 함

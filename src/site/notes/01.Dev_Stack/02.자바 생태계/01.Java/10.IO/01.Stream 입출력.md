# 📘 자바 스트림(Stream) 입출력

## Summary 📝

> [!summary] Summary 📝
> 
> - 자바 스트림 계층과 사용 원칙 정리, 바이트 기반과 문자 기반의 구분 및 어댑터 역할 명확화
> - 핵심 메소드 가독성 개선 및 기반 스트림 목록 보강
> - 인코딩 명시·보조 스트림 활용·try-with-resources 권장 사항 포함

---

## 목적 🎯

- 파일·네트워크·메모리 등 다양한 입출력 대상과의 데이터 교환을 위한 스트림 구조 이해 목적
- 스트림 클래스 계층, 주요 메소드의 직관적 정리 및 안전한 사용 패턴 숙지 목적

---

## 핵심 개념 💡

### ✅ 스트림 분류표 (핵심 속성)

|분류|입력 최상위 클래스|출력 최상위 클래스|처리 단위|예시 하위 클래스|
|---|---|---|---|---|
|바이트 기반|`InputStream`|`OutputStream`|바이트 단위|`FileInputStream`; `BufferedInputStream`; `DataOutputStream`|
|문자 기반|`Reader`|`Writer`|문자 단위 (유니코드)|`FileReader`; `BufferedReader`; `OutputStreamWriter`|

- **설명**: 문자 스트림(`Reader`/`Writer`)과 바이트 스트림(`InputStream`/`OutputStream`)은 별개 계층 구분. 바이트↔문자 변환이 필요한 경우 `InputStreamReader`/`OutputStreamWriter` 어댑터 사용 권장.

---

### ✅ 기반 스트림 vs 보조 스트림 (정의 표)

|구분|역할|예시|
|---|---|---|
|기반 스트림|실제 데이터 소스/대상과 연결되어 직접 입출력 수행|`FileInputStream`; `FileOutputStream`; `FileReader`; `FileWriter`|
|보조 스트림|성능 및 기능 확장(버퍼링·자료형 처리·필터링) 목적, 기반 스트림 래핑|`BufferedInputStream`; `BufferedReader`; `DataInputStream`; `DataOutputStream`|
|어댑터|바이트↔문자 변환(인코딩·디코딩) 역할|`InputStreamReader`; `OutputStreamWriter`|

---

### ✅ 주요 클래스별 핵심 메소드 (가독성 향상 표)

#### InputStream 계열 (바이트 입력)

|메소드|시그니처|설명|
|---|---|---|
|**read 한 바이트**|`int read()`|한 바이트 읽음, 반환값 0~255 또는 EOF이면 -1|
|**배열 읽기**|`int read(byte[] b)`|최대 b.length 만큼 읽어 배열에 저장, 실제 읽은 바이트 수 반환 또는 -1|
|**부분 배열 읽기**|`int read(byte[] b, int off, int len)`|b[off]부터 len 바이트 읽음, 읽은 수 반환|
|**건너뛰기**|`long skip(long n)`|n 바이트 건너뛰기, 실제 건너뛴 바이트 수 반환|
|**사용 가능 바이트 수**|`int available()`|즉시 읽을 수 있는 바이트 수 예상값 반환|
|**닫기**|`void close()`|스트림 닫아 자원 해제|

#### OutputStream 계열 (바이트 출력)

|메소드|시그니처|설명|
|---|---|---|
|**한 바이트 쓰기**|`void write(int b)`|인자의 하위 8비트만 기록|
|**배열 전체 쓰기**|`void write(byte[] b)`|배열 전체를 출력|
|**부분 배열 쓰기**|`void write(byte[] b, int off, int len)`|b[off]부터 len 바이트 출력|
|**버퍼 비우기**|`void flush()`|버퍼가 있다면 강제 전송|
|**닫기**|`void close()`|스트림 닫고 자원 해제(대부분 flush 포함)|

#### Reader 계열 (문자 입력)

|메소드|시그니처|설명|
|---|---|---|
|**한 문자 읽기**|`int read()`|한 문자 읽음, 반환값 0~65535 또는 EOF이면 -1|
|**문자 배열 읽기**|`int read(char[] c)`|최대 c.length 만큼 읽어 저장, 실제 읽은 문자 수 반환|
|**부분 배열 읽기**|`int read(char[] c, int off, int len)`|c[off]부터 len 문자 읽음, 읽은 수 반환|
|**라인 읽기**|`String readLine()` (BufferedReader)|한 줄 읽어 문자열 반환, EOF면 null|
|**닫기**|`void close()`|스트림 닫아 자원 해제|

#### Writer 계열 (문자 출력)

|메소드|시그니처|설명|
|---|---|---|
|**한 문자 쓰기**|`void write(int c)`|인자의 하위 16비트(문자) 기록|
|**문자 배열 쓰기**|`void write(char[] c)`|배열 전체 출력|
|**부분 배열 쓰기**|`void write(char[] c, int off, int len)`|c[off]부터 len 문자 출력|
|**문자열 쓰기**|`void write(String str)`|문자열 전체 출력|
|**부분 문자열 쓰기**|`void write(String str, int off, int len)`|str의 일부 출력|
|**줄바꿈 편의**|`void newLine()` (BufferedWriter 권장)|플랫폼에 맞는 줄바꿈 추가|
|**버퍼 비우기**|`void flush()`|버퍼의 내용 강제 전송|
|**닫기**|`void close()`|스트림 닫아 자원 해제|

## 특징 ✨

### ✅ 바이트 스트림 특징

- 바이트 단위 처리로 이진 데이터 처리에 적합함.
- `read()` 반환값 및 `write(int)`의 하위 8비트 동작 명확성 필요함.

### ✅ 문자 스트림 특징

- 문자(유니코드) 단위 처리로 텍스트 처리에 적합함.
- 인코딩 의존성으로 인코딩 명시 필요함.

### ✅ 보조 스트림과 어댑터 특징

- 버퍼링(`Buffered*`)으로 I/O 성능 개선 가능함.
- 자료형 직접 입출력(`Data*`)으로 원시 타입 처리 용이함.
- 어댑터(`InputStreamReader`/`OutputStreamWriter`)로 인코딩 지정 후 안전한 텍스트 입출력 가능함.

---

## 팁과 주의점 ⚠️

- 텍스트 처리 시 항상 인코딩 명시(예: `StandardCharsets.UTF_8`) 권장.
- 스트림은 단방향 객체이므로 동시 읽기·쓰기 필요 시 입력·출력 스트림 각각 생성.
- 보조 스트림 사용으로 성능·기능 향상 권장.
- 리소스 누수 방지 목적의 try-with-resources 사용 권장.
- `flush()`는 즉시 전송 보장 목적, `close()`는 내부적으로 `flush()`를 호출하는 구현이 존재함.

## 예시 📌

### ✅ 예시 1: UTF-8 텍스트 파일 읽기 (동작 흐름 중심)

- 동작 흐름: 파일 열기 → FileInputStream 생성 → InputStreamReader(UTF-8)으로 변환 → BufferedReader로 버퍼링 → 한 문자씩 읽어 출력 → 자동 닫기

```java
import java.io.BufferedReader;
import java.io.FileInputStream;
import java.io.InputStreamReader;
import java.io.IOException;
import java.nio.charset.StandardCharsets;

public class ReadExample {
    public static void main(String[] args) {
        // try-with-resources로 스트림 자동 종료 보장
        try (BufferedReader br = new BufferedReader(
                 new InputStreamReader(new FileInputStream("sample.txt"), StandardCharsets.UTF_8))) {
            int ch;
            while ((ch = br.read()) != -1) {                 // 한 문자씩 읽음, -1이면 EOF
                System.out.print((char) ch);                 // 읽은 문자 출력
            }
        } catch (IOException e) {
            e.printStackTrace();                              // 예외 발생 시 스택트레이스 출력
        }
    }
}
```

### ✅ 예시 2: UTF-8 텍스트 파일 쓰기 (동작 흐름 중심)

- 동작 흐름: FileOutputStream 생성 → OutputStreamWriter(UTF-8)으로 문자→바이트 변환 → BufferedWriter로 버퍼링 → 문자열 기록 → flush → 자동 닫기

```java
import java.io.BufferedWriter;
import java.io.OutputStreamWriter;
import java.io.FileOutputStream;
import java.io.IOException;
import java.nio.charset.StandardCharsets;

public class WriteExample {
    public static void main(String[] args) {
        try (BufferedWriter writer = new BufferedWriter(
                 new OutputStreamWriter(new FileOutputStream("output.txt"), StandardCharsets.UTF_8))) {
            writer.write("안녕하세요! 자바 스트림 예제입니다."); // 문자열 기록
            writer.newLine();                                // 플랫폼별 줄바꿈 추가
            writer.flush();                                  // 버퍼 강제 비움(필요 시)
        } catch (IOException e) {
            e.printStackTrace();                              // 예외 처리
        }
    }
}
```

### ✅ 예시 3: 바이너리 파일 읽기 (버퍼 사용, 동작 흐름 중심)

- 동작 흐름: FileInputStream 생성 → BufferedInputStream으로 버퍼링 → 바이트 배열 단위로 반복 읽기 → 처리 → 자동 닫기

```java
import java.io.FileInputStream;
import java.io.BufferedInputStream;
import java.io.IOException;

public class BinaryReadExample {
    public static void main(String[] args) {
        try (BufferedInputStream bis = new BufferedInputStream(new FileInputStream("data.bin"))) {
            byte[] buffer = new byte[4096];                   // 읽기 버퍼 할당
            int read;
            while ((read = bis.read(buffer)) != -1) {         // 읽은 바이트 수 확인
                // buffer[0..read-1] 범위에 대해 처리 수행
            }
        } catch (IOException e) {
            e.printStackTrace();                              // 예외 처리
        }
    }
}
```

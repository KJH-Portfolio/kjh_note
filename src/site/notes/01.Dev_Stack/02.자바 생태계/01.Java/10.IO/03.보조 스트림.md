# 📘 보조 스트림 (Auxiliary Stream)

## Summary 📝

> [!summary] Summary 📝
> 
> - 보조 스트림은 기반 스트림에 기능을 추가하여 입출력의 성능과 편의성을 향상시킴
> - 문자 변환(어댑터), 버퍼링, 기본 타입 입출력, 객체 직렬화 등 기능별 보조 스트림 존재함
> - 보조 스트림은 단독 입출력 불가, 반드시 기반 스트림을 래핑하여 사용함

---

## 목적 🎯

- 기반 스트림의 기능 확장과 안전한 입출력 구현 방법 제시
- 문자 인코딩·버퍼링·원시 타입 저장·객체 직렬화 사용 시 유의사항 및 올바른 코드 패턴 제공

---

## 핵심 개념 💡

### ✅ 문자 변환 보조 스트림

| 클래스명                 | 역할                              |
| -------------------- | ------------------------------- |
| `InputStreamReader`  | 바이트 스트림을 문자 스트림으로 변환, 인코딩 지정 가능 |
| `OutputStreamWriter` | 문자 스트림을 바이트 스트림으로 변환, 인코딩 지정 가능 |


- 인코딩 미지정 시 플랫폼 기본 인코딩 의존으로 문자 깨짐 발생 위험.
- 생성자에는 Charset 객체 사용 권장 (`StandardCharsets.UTF_8`)으로 예외 및 오타 방지.

---

### ✅ 버퍼링 보조 스트림

| 클래스명                                           | 역할                                       |
| ---------------------------------------------- | ---------------------------------------- |
| `BufferedReader` / `BufferedWriter`            | 문자 스트림 버퍼링 및 `readLine()`/`newLine()` 제공 |
| `BufferedInputStream` / `BufferedOutputStream` | 바이트 스트림 버퍼링 제공                           |

- 버퍼링으로 시스템 호출 횟수 감소 및 처리 속도 향상.
- `flush()`로 출력 버퍼 강제 비움 필요성 존재.

---

### ✅ 기본 타입 입출력 보조 스트림

| 클래스명               | 역할                         |
| ------------------ | -------------------------- |
| `DataInputStream`  | 바이트 스트림 위에 원시 타입 읽기 API 제공 |
| `DataOutputStream` | 바이트 스트림 위에 원시 타입 쓰기 API 제공 |

- `writeInt`/`readInt` 등은 **빅엔디안** 형식으로 입출력함.
- `writeUTF`/`readUTF`는 Java의 **modified UTF-8** 형식 사용 및 길이 제한(최대 65535 바이트) 존재.
- 읽기·쓰기 순서 및 자료형 일치 필수.

---

### ✅ 객체 입출력 보조 스트림

| 클래스명                 | 역할                    |
| -------------------- | --------------------- |
| `ObjectOutputStream` | 객체 직렬화하여 바이트 스트림으로 출력 |
| `ObjectInputStream`  | 직렬화된 바이트를 객체로 복원      |

- 직렬화 대상 클래스는 `Serializable` 구현 필요.
- 클래스 변경 시 역직렬화 실패 가능성 존재. `serialVersionUID` 명시 권장.
- 파일에 여러 번 객체를 이어서 쓸 경우 `ObjectOutputStream` 헤더 중복 문제 발생하므로 주의 필요.

---

## 특징 ✨

- 보조 스트림은 기반 스트림을 래핑해야만 동작함.
- 보조 스트림 조합으로 유연한 입출력 파이프라인 구성 가능(예: FileInputStream → BufferedInputStream → DataInputStream).
- 올바른 자원 해제를 위해 try-with-resources 사용 권장.
- 일부 메서드(`readLine()`, `newLine()`, `readUTF()`)의 동작·제약 조건 명확화 필요.

---

## 팁과 주의점 ⚠️

- 인코딩은 반드시 명시. 문자열 파일 처리 시 StandardCharsets 사용 권장.
- `DataOutputStream.writeUTF`/`DataInputStream.readUTF`는 상호 호환 전용이며 일반 텍스트 파일 대체용으로 사용 금지.
- `ObjectOutputStream`으로 파일에 여러 번 객체를 추가할 때는 기존 파일에 헤더를 재작성하지 않도록 처리 필요(append 모드에서 헤더 생략 등).
- 보조 스트림은 성능 향상이 목적이므로, 불필요한 중복 래핑 지양.
- 스트림 조합 시 읽기·쓰기 순서 일치 및 예외 처리 의무.

## 예시 📌

### ✅ 예시: InputStreamReader 사용
```java
import java.io.FileInputStream;
import java.io.InputStreamReader;
import java.io.BufferedReader;
import java.io.IOException;
import java.nio.charset.StandardCharsets;

public class InputStreamReaderExample {
    public static void main(String[] args) {
        // FileInputStream을 InputStreamReader로 감싸 문자 인코딩을 명시하고 BufferedReader로 버퍼링
        try (BufferedReader br = new BufferedReader(
                 new InputStreamReader(new FileInputStream("sample.txt"), StandardCharsets.UTF_8))) {
            String line;
            while ((line = br.readLine()) != null) {   // readLine은 BufferedReader에만 존재
                System.out.println(line);               // 한 줄 단위 출력
            }
        } catch (IOException e) {
            e.printStackTrace();                        // 예외 처리
        }
    }
}
```

### ✅ 예시: OutputStreamWriter 사용 
```java
import java.io.FileOutputStream;
import java.io.OutputStreamWriter;
import java.io.BufferedWriter;
import java.io.IOException;
import java.nio.charset.StandardCharsets;

public class OutputStreamWriterExample {
    public static void main(String[] args) {
        // OutputStreamWriter로 문자→바이트 변환, BufferedWriter로 성능 개선
        try (BufferedWriter bw = new BufferedWriter(
                 new OutputStreamWriter(new FileOutputStream("output.txt"), StandardCharsets.UTF_8))) {
            bw.write("문자 변환 스트림 예제"); // 문자열 기록
            bw.newLine();                     // 플랫폼 독립적 줄바꿈
            // close()에서 자동 flush 발생하므로 명시 flush는 선택적
        } catch (IOException e) {
            e.printStackTrace();               // 예외 처리
        }
    }
}
```

### ✅ 예시: BufferedReader/BufferedWriter 파일 복사

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;

public class CopyFileExample {
    public static void main(String[] args) {
        // FileReader/FileWriter는 기본 인코딩 사용. 인코딩 명시 필요 시 InputStreamReader/OutputStreamWriter 사용 권장
        try (BufferedReader br = new BufferedReader(new FileReader("source.txt"));
             BufferedWriter bw = new BufferedWriter(new FileWriter("copy.txt"))) {
            String line;
            while ((line = br.readLine()) != null) { // readLine은 null이면 EOF
                bw.write(line);                       // 한 줄 기록
                bw.newLine();                         // 줄바꿈 추가
            }
            bw.flush();                              // 안전을 위해 flush 호출 가능
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### ✅ 예시: DataOutputStream / DataInputStream


```java
import java.io.DataOutputStream;
import java.io.DataInputStream;
import java.io.FileOutputStream;
import java.io.FileInputStream;
import java.io.IOException;

public class DataStreamExample {
    public static void main(String[] args) {
        // 쓰기: DataOutputStream은 바이트스트림을 래핑하여 원시 타입을 빅엔디안으로 저장
        try (DataOutputStream dos = new DataOutputStream(new FileOutputStream("data.bin"))) {
            dos.writeInt(42);            // 4바이트 정수 기록
            dos.writeDouble(3.14);       // 8바이트 실수 기록
            dos.writeUTF("Hello");       // modified UTF-8로 길이 포함 기록(최대 65535 바이트)
        } catch (IOException e) {
            e.printStackTrace();
        }

        // 읽기: 저장된 순서와 자료형이 동일해야 정확히 복원 가능
        try (DataInputStream dis = new DataInputStream(new FileInputStream("data.bin"))) {
            int num = dis.readInt();            // writeInt와 순서 일치
            double pi = dis.readDouble();       // writeDouble와 순서 일치
            String msg = dis.readUTF();         // writeUTF로 기록된 문자열 복원
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### ✅ 예시: ObjectOutputStream / ObjectInputStream 

```java
import java.io.Serializable;
import java.io.ObjectOutputStream;
import java.io.ObjectInputStream;
import java.io.FileOutputStream;
import java.io.FileInputStream;
import java.io.IOException;
import java.io.File;

// 직렬화 대상 클래스: Serializable 구현 및 serialVersionUID 명시
public class Book implements Serializable {
    private static final long serialVersionUID = 1L;
    private String title;
    private String author;
    private int price;
    // 생성자 및 getter/setter 생략
}

// 객체 저장 예시: 새 파일에 객체 쓰기
try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("book.dat"))) {
    Book b = new Book("자바의 정석", "남궁성", 30000);
    oos.writeObject(b); // 객체 직렬화 및 기록
} catch (IOException e) {
    e.printStackTrace();
}

// 객체 복원 예시
try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream("book.dat"))) {
    Book b = (Book) ois.readObject(); // ClassNotFoundException 처리 필요
} catch (IOException | ClassNotFoundException e) {
    e.printStackTrace();
}
```

- 추가 주의: 같은 파일에 여러 번 ObjectOutputStream으로 append 시 매번 스트림 헤더가 기록되어 복원 불가 현상 발생. append 필요 시 첫 번째 기록 이후에는 `new ObjectOutputStream(OutputStream)` 대신 헤더 생성을 막는 커스텀 스트림 구현 또는 `oos.reset()`과 같은 방법 사용 권장.
- 보안 주의: 역직렬화 과정에서 악의적 페이로드 실행 위험 존재. 신뢰할 수 없는 데이터에 대한 역직렬화 금지 권장.

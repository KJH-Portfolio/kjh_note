# 📘 File 클래스

## Summary 📝

> [!summary] Summary 📝
> 
> - `File` 클래스는 파일·디렉토리의 생성·삭제·조회 등 메타데이터 조작 목적
> - 실제 파일 데이터의 입출력은 스트림(`FileInputStream`/`FileOutputStream` 등) 또는 NIO(`Path`/`Files`) 사용 권장
> - 보조 스트림을 통한 문자 변환·버퍼링·기본 타입·객체 직렬화 기능 확장 가능
> - 예제는 자원 안전성(try-with-resources), 인코딩 명시(StandardCharsets), 경로 처리(File.separator/Paths) 반영

---

## 목적 🎯

- 파일·디렉토리의 생성·삭제·탐색·속성 확인 기능 제공 목적
- 바이트·문자 단위의 파일 입출력 구현 안내 목적
- 보조 스트림 및 직렬화 사용 시 안전한 패턴 및 주의사항 전달 목적

## 핵심 개념 💡

### ✅ `File` 클래스 주요 메서드 및 동작

|리턴 타입|메서드|정확한 동작 설명|
|---|---|---|
|`boolean`|`createNewFile()`|파일이 없으면 생성 후 `true` 반환, 파일이 이미 있으면 `false` 반환, I/O 오류 시 `IOException` 발생|
|`boolean`|`mkdir()`|즉시 부모 디렉토리가 존재해야 성공, 성공 시 `true` 반환|
|`boolean`|`mkdirs()`|필요한 부모까지 모두 생성 시도, 성공 시 `true` 반환|
|`boolean`|`delete()`|파일 또는 비어있는 디렉토리 삭제 시도, 성공 시 `true` 반환|
|`boolean`|`exists()`|경로 존재 여부 확인|
|`boolean`|`canRead()` / `canWrite()`|접근 권한 여부 확인|
|`boolean`|`isFile()` / `isDirectory()`|파일/디렉토리 여부 판별|
|`String`|`getName()` / `getParent()`|파일명 또는 상위 경로 반환 (`getParent()`는 null 가능)|
|`long`|`length()`|파일 크기(바이트) 반환, 디렉토리일 경우 플랫폼·JVM별 동작(일반적으로 0)|
|`String[]`|`list()`|디렉토리 내 이름 배열 반환, 대상이 디렉토리가 아니거나 접근 불가 시 `null` 반환|
|`File[]`|`listFiles()`|디렉토리 내 `File` 배열 반환, 대상이 디렉토리가 아니거나 접근 불가 시 `null` 반환|

- **경로 구분자**: 하드코딩된 `\\` 또는 `/` 대신 `File.separator` 또는 `Paths.get(...)` 사용 권장.

---

### ✅ 파일 기반 스트림 비교 (요약)

|클래스명|상속|처리 단위|주요 용도|파일 존재 시 기본 동작|
|---|---|---|---|---|
|`FileInputStream`|`InputStream`|바이트|이진·텍스트 파일 읽기|존재하지 않으면 `FileNotFoundException` 발생|
|`FileOutputStream`|`OutputStream`|바이트|이진·텍스트 파일 쓰기|기본 생성자: 덮어쓰기, `new FileOutputStream(file, true)`: 이어쓰기|
|`FileReader`|`Reader`|문자|텍스트 파일 읽기(플랫폼 기본 인코딩)|존재하지 않으면 `FileNotFoundException` 발생|
|`FileWriter`|`Writer`|문자|텍스트 파일 쓰기(플랫폼 기본 인코딩)|기본 생성자: 덮어쓰기, append 옵션 존재|

- **인코딩 주의**: `FileReader`/`FileWriter`는 플랫폼 기본 인코딩 사용. 명시적 인코딩 필요 시 `InputStreamReader`/`OutputStreamWriter`와 `StandardCharsets` 사용 필요.

---

### ✅ 보조 스트림 및 직렬화 핵심 포인트

|기능|클래스|주의사항|
|---|---|---|
|문자 변환|`InputStreamReader`/`OutputStreamWriter`|Charset 명시 권장(`StandardCharsets.UTF_8`)|
|버퍼링|`BufferedReader`/`BufferedWriter` / `BufferedInputStream`/`BufferedOutputStream`|I/O 호출 축소 목적, `flush()` 사용 고려|
|기본 타입 입출력|`DataInputStream`/`DataOutputStream`|`writeXXX`/`readXXX` 순서·자료형 일치 필수, `writeUTF`는 modified UTF-8 및 최대 65535 바이트 제약|
|객체 직렬화|`ObjectOutputStream`/`ObjectInputStream`|`Serializable` 구현·`serialVersionUID` 명시 권장, append 시 헤더 중복 문제 및 역직렬화 보안 위험 존재|

---

## 특징 ✨

- `File` 객체는 파일 시스템 상의 경로와 속성 표현 목적, 데이터 읽기·쓰기는 별도의 스트림 또는 NIO API 필요
- `createNewFile()` 등은 상태·예외 구분 필요(반환값 + 예외)
- `list()`/`listFiles()`는 접근 권한·정상 디렉토리 여부에 따라 `null` 반환 가능
- 보조 스트림 조합으로 유연한 입출력 파이프라인 구성 가능(예: FileInputStream → BufferedInputStream → DataInputStream)
- 객체 직렬화는 편리하지만 헤더·버전·보안 이슈 존재

---

## 팁과 주의점 ⚠️

- 경로 처리 시 플랫폼 독립성 확보 목적의 `File.separator`·`Paths.get()` 사용 권장
- 텍스트 입출력 시 인코딩 명시(예: `StandardCharsets.UTF_8`) 필수
- 모든 스트림은 try-with-resources로 안전하게 닫기 권장
- `FileOutputStream`의 기본 동작은 덮어쓰기이며 이어쓰기 필요 시 `append=true` 옵션 사용
- `DataOutputStream.writeUTF`는 일반 텍스트 파일과 호환되지 않음(특수 형식)
- `ObjectOutputStream`로 파일에 이어쓰기 시 헤더 중복 문제 주의(헤더 생략 커스텀 구현 또는 첫 쓰기만 ObjectOutputStream 사용)
- 역직렬화는 신뢰 가능한 데이터에 한정 실행 권장(allowlist/검증 필요)

## 예시 📌

### ✅ 예시1 — 파일 생성(createNewFile) 및 존재 확인(exists, isFile)

```java
import java.io.File;
import java.io.IOException;

public class FileCreateExistsExample {
    public static void main(String[] args) {
        File file = new File("data" + File.separator + "example.txt"); // 플랫폼 독립적 경로

        // 디렉토리 보장
        File parent = file.getParentFile();
        if (parent != null && !parent.exists()) {
            boolean dirsCreated = parent.mkdirs(); // 필요시 부모 디렉토리 생성
            System.out.println("디렉토리 생성 여부: " + dirsCreated);
        }

        // 파일 생성: 이미 존재하면 false 반환
        try {
            boolean created = file.createNewFile(); // IOException 발생 가능
            System.out.println("파일 생성 여부: " + created);
        } catch (IOException e) {
            e.printStackTrace();
        }

        // 존재 및 타입 확인
        System.out.println("존재: " + file.exists());
        System.out.println("파일 여부: " + file.isFile());
    }
}
```

### ✅ 예시2 — 디렉토리 생성(mkdir / mkdirs) 및 삭제(delete)

```java
import java.io.File;

public class DirectoryCreateDeleteExample {
    public static void main(String[] args) {
        File singleDir = new File("tmp" + File.separator + "single");
        File nestedDir = new File("tmp" + File.separator + "parent" + File.separator + "child");

        // 부모가 있어야 성공하는 단일 디렉토리 생성
        boolean mkdirResult = singleDir.mkdir();
        System.out.println("mkdir 결과: " + mkdirResult);

        // 부모까지 모두 생성하는 mkdirs
        boolean mkdirsResult = nestedDir.mkdirs();
        System.out.println("mkdirs 결과: " + mkdirsResult);

        // 디렉토리 삭제: 비어있어야 삭제 성공
        boolean deleteSingle = singleDir.delete();
        System.out.println("single 삭제: " + deleteSingle);

        // 재귀 삭제 필요 시 사용자 구현 필요(단일 delete는 비어있을 때만 성공)
        boolean deleteNested = nestedDir.delete();
        System.out.println("nested 삭제: " + deleteNested);
    }
}
```

### ✅ 예시3 — 파일 삭제 및 renameTo(이름 변경/이동)

```java
import java.io.File;

public class FileDeleteRenameExample {
    public static void main(String[] args) {
        File src = new File("data" + File.separator + "old.txt");
        File dest = new File("data" + File.separator + "new_name.txt");

        // 파일 삭제
        if (src.exists()) {
            boolean deleted = src.delete(); // 실패 시 보통 권한 문제 또는 사용 중
            System.out.println("삭제 여부: " + deleted);
        } else {
            System.out.println("삭제할 파일이 존재하지 않음");
        }

        // 파일 이름 변경(같은 파일시스템 내 이동)
        File fileToMove = new File("data" + File.separator + "move_me.txt");
        boolean renamed = fileToMove.renameTo(dest); // 실패 시 false 반환
        System.out.println("이름 변경(이동) 성공 여부: " + renamed);
    }
}
```

### ✅ 예시4 — 접근 권한 확인 및 변경(canRead, canWrite, setReadable, setWritable)

```java
import java.io.File;

public class FilePermissionExample {
    public static void main(String[] args) {
        File file = new File("data" + File.separator + "perm.txt");

        // 읽기/쓰기 가능 여부 확인
        System.out.println("읽기 가능: " + file.canRead());
        System.out.println("쓰기 가능: " + file.canWrite());

        // 권한 변경 시도 (운영체제 정책에 따라 실패 가능)
        boolean readableSet = file.setReadable(true, false); // 모든 사용자에게 읽기 허용
        boolean writableSet = file.setWritable(false, false); // 모든 사용자 쓰기 금지
        System.out.println("읽기 권한 설정 성공: " + readableSet);
        System.out.println("쓰기 권한 설정 성공: " + writableSet);
    }
}
```

### ✅ 예시5 — 디렉토리 목록 조회(list, listFiles) 및 파일 속성(length, getName, getParent)

```java
import java.io.File;

public class FileListAndPropertiesExample {
    public static void main(String[] args) {
        File dir = new File("data");

        // 이름 목록 반환 (디렉토리가 아니면 null)
        String[] names = dir.list();
        if (names != null) {
            for (String name : names) {
                System.out.println("이름: " + name);
            }
        } else {
            System.out.println("디렉토리 아님 또는 접근 불가");
        }

        // File 객체 목록 반환
        File[] files = dir.listFiles();
        if (files != null) {
            for (File f : files) {
                System.out.printf("이름: %s, 크기: %d, 파일인지: %b, 부모: %s%n",
                                  f.getName(), f.length(), f.isFile(), f.getParent());
            }
        }

        // 특정 파일 속성
        File sample = new File(dir, "example.txt");
        System.out.println("파일명: " + sample.getName());
        System.out.println("상위 경로: " + sample.getParent());
        System.out.println("파일 크기(바이트): " + sample.length());
    }
}
```

### ✅ 예시6 — 파일 삭제 재귀 구현(비어있지 않은 디렉토리 삭제용 유틸)

```java
import java.io.File;

public class FileUtils {
    // 재귀적으로 파일/디렉토리 삭제. 성공 시 true 반환
    public static boolean deleteRecursive(File target) {
        if (target == null || !target.exists()) return false;
        if (target.isDirectory()) {
            File[] children = target.listFiles();
            if (children != null) {
                for (File child : children) {
                    deleteRecursive(child);
                }
            }
        }
        return target.delete(); // 파일이거나 비어있는 디렉토리는 삭제 성공
    }

    // 샘플 사용법
    public static void main(String[] args) {
        File toDelete = new File("tmp" + File.separator + "to_remove");
        boolean result = deleteRecursive(toDelete);
        System.out.println("재귀 삭제 결과: " + result);
    }
}
```

### ✅ 예시7: 객체 직렬화

```java
import java.io.Serializable;
import java.io.ObjectOutputStream;
import java.io.ObjectInputStream;
import java.io.FileOutputStream;
import java.io.FileInputStream;
import java.io.IOException;
import java.io.File;

// 직렬화 대상 클래스
public class Book implements Serializable {
    private static final long serialVersionUID = 1L; // 명시 권장
    private String title;
    private String author;
    private int price;
    // 생성자, getter/setter 생략
}

// 새 파일에 객체 저장(헤더 정상 기록)
try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("book.dat"))) {
    Book b = new Book("자바의 정석", "남궁성", 30000);
    oos.writeObject(b);
} catch (IOException e) {
    e.printStackTrace();
}

// 복원
try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream("book.dat"))) {
    Book b = (Book) ois.readObject(); // ClassNotFoundException 처리 필요
} catch (IOException | ClassNotFoundException e) {
    e.printStackTrace();
}

// append로 여러 객체를 같은 파일에 기록해야 하는 경우
// 첫 번째 기록은 ObjectOutputStream으로 수행하고, 그 이후에는 헤더 생성을 막는 커스텀 OutputStream 사용 권장
```

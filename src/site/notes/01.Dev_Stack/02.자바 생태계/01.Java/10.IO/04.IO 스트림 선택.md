# 📘 기반 스트림 vs 보조 스트림 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - **기반 스트림**은 실제 데이터의 출발지/도착지와 직접 연결되는 통로
> - **보조 스트림**은 기반 스트림을 감싸 기능과 성능을 향상시키는 역할
> - 보조 스트림은 단독 사용 불가, 반드시 기반 스트림과 함께 사용
> - 선택 기준은 데이터 성격(문자/바이트), 처리 단위(라인/객체), 성능 요구사항

---

## 목적 🎯

- 기반 스트림과 보조 스트림의 개념 구분
- 상황별 스트림 선택 기준 이해
- 파일, 네트워크, 객체 입출력에서의 활용 방식 학습

---

## 핵심 개념 💡

### ✅ 기반 스트림 (Base Stream)

- 정의: 실제 데이터의 출발지/도착지(파일, 네트워크 소켓, 키보드 입력 등)와 직접 연결되는 스트림
- 역할: 데이터의 **통로**
- 종류
    - 바이트 기반: `InputStream`, `OutputStream` 계열
    - 문자 기반: `Reader`, `Writer` 계열

**예시**

```java
FileInputStream fis = new FileInputStream("data.txt");   // 파일에서 바이트 읽기
FileReader fr = new FileReader("data.txt");              // 파일에서 문자 읽기
Socket socket = new Socket("127.0.0.1", 3000);
InputStream is = socket.getInputStream();                // 네트워크 입력
```

---

### ✅ 보조 스트림 (Filter/Wrapper Stream)

- 정의: 기반 스트림을 감싸 기능을 향상시키는 스트림
- 역할: 버퍼링, 형변환, 편의 기능 제공
- 특징: 단독 사용 불가, 반드시 기반 스트림과 연결

**예시**

```java
BufferedReader br = new BufferedReader(new FileReader("data.txt")); // 버퍼링 + readLine()
PrintWriter pw = new PrintWriter(new FileWriter("log.txt"));        // println(), printf() 지원
DataInputStream dis = new DataInputStream(new FileInputStream("data.bin")); // 기본형 데이터 읽기
```

## 특징 ✨

### ✅ 상황별 선택 가이드
### 📄 1. 텍스트 파일 읽기

| 구성 요소      | 클래스                 | 설명                    |
| ---------- | ------------------- | --------------------- |
| **기반 스트림** | `FileReader`        | 파일에서 문자 단위로 읽음        |
| **보조 스트림** | `BufferedReader`    | 버퍼링 + `readLine()` 지원 |
| **이유**     | 한 줄씩 효율적으로 읽기 위해 사용 |                       |

---

### 📝 2. 텍스트 파일 쓰기

| 구성 요소      | 클래스                               | 설명                                               |
| ---------- | --------------------------------- | ------------------------------------------------ |
| **기반 스트림** | `FileWriter`                      | 파일에 문자 단위로 씀                                     |
| **보조 스트림** | `PrintWriter` 또는 `BufferedWriter` | `println()`, `printf()` 또는 `write()`,`newLine()` |
| **이유**     | 편리한 출력 메서드 또는 성능 향상 목적            |                                                  |

#### 🆚 `PrintWriter` vs `BufferedWriter`

|항목|`PrintWriter`|`BufferedWriter`|
|---|---|---|
|**주요 기능**|편리한 출력 메서드 제공 (`print`, `println`, `printf`)|버퍼링을 통한 성능 향상|
|**자동 flush**|가능 (`new PrintWriter(..., true)`)|없음 (직접 `flush()` 호출 필요)|
|**출력 대상**|문자 기반 출력 (파일, 콘솔, 네트워크 등)|문자 기반 출력 (파일, 네트워크 등)|
|**주요 메서드**|`print()`, `println()`, `printf()`|`write()`, `newLine()`, `flush()`|
|**사용 목적**|**편의성 중심**: 포맷팅, 줄바꿈 등 쉽게 출력|**성능 중심**: 많은 양의 문자 출력 시 효율적|
|**예외 처리**|내부에서 `IOException`을 숨김|`IOException`을 직접 처리해야 함|

---

### 📌미니 예제 비교

#### ✅ `PrintWriter` 예시

```java
PrintWriter pw = new PrintWriter(new FileWriter("output.txt"), true);
pw.println("Hello, world!");
pw.printf("Score: %d%n", 95);
```

- `println()`과 `printf()` 사용 가능
- `true`로 설정하면 자동 flush됨

#### ✅ `BufferedWriter` 예시

```java
BufferedWriter bw = new BufferedWriter(new FileWriter("output.txt"));
bw.write("Hello, world!");
bw.newLine();
bw.write("Score: 95");
bw.flush(); // 직접 flush 필요
```

- `write()`와 `newLine()` 사용
- `flush()`를 직접 호출해야 출력됨
#### 🧠 언제 어떤 걸 써야 할까?

- **간단한 텍스트 출력 + 포맷팅 필요** → `PrintWriter`
- **대량의 문자 출력 + 성능 중요** → `BufferedWriter`
- `BufferedWriter`를 `PrintWriter`로 감싸서 포맷팅과 성능을 동시에 잡는 것도 가능

---

### 📦 3. 바이너리 파일 입출력

| 구성 요소      | 클래스                                           | 설명             |
| ---------- | --------------------------------------------- | -------------- |
| **기반 스트림** | `FileInputStream`, `FileOutputStream`         | 바이트 단위로 파일 입출력 |
| **보조 스트림** | `BufferedInputStream`, `BufferedOutputStream` | 버퍼링으로 성능 향상    |
| **이유**     | 대용량 파일 처리 시 효율성 증가                            |                |

---

### 🌐 4. 네트워크 채팅 (문자 기반)

| 구성 요소        | 클래스                                                   | 설명                   |
| ------------ | ----------------------------------------------------- | -------------------- |
| **기반 스트림**   | `socket.getInputStream()`, `socket.getOutputStream()` | 바이트 스트림              |
| **보조 스트림 1** | `InputStreamReader`, `OutputStreamWriter`             | 바이트 → 문자 / 문자 -> 바이트 |
| **보조 스트림 2** | `BufferedReader`, `PrintWriter`                       | 한 줄 단위 처리 및 편의 메서드   |
| **이유**       | 문자 기반 통신, 줄 단위 송수신, 자동 flush 등 편의성 확보                 |                      |

---

### 🔢 5. 기본형 데이터 저장

| 구성 요소      | 클래스                             | 설명                                     |
| ---------- | ------------------------------- | -------------------------------------- |
| **기반 스트림** | `FileOutputStream`              | 바이트 단위 출력                              |
| **보조 스트림** | `DataOutputStream`              | `writeInt()`, `writeDouble()` 등 기본형 지원 |
| **이유**     | 숫자, 실수 등 기본형 데이터를 이진 형식으로 저장 가능 |                                        |

---

### 🧱 6. 객체 저장/복원 (직렬화)

|구성 요소|클래스|설명|
|---|---|---|
|**기반 스트림**|`FileOutputStream`, `FileInputStream`|바이트 단위 입출력|
|**보조 스트림**|`ObjectOutputStream`, `ObjectInputStream`|객체 단위 직렬화/역직렬화|
|**이유**|객체를 통째로 저장하거나 복원할 수 있음 (`Serializable` 필요)|

---
## 팁과 주의점 ⚠️

- 보조 스트림은 반드시 기반 스트림과 함께 사용해야 함
- 문자 데이터는 `Reader`/`Writer`, 바이너리 데이터는 `InputStream`/`OutputStream` 계열 선택
- 대용량 처리 시 `Buffered` 계열 사용으로 성능 향상
- 네트워크 통신 시 `flush()` 호출로 즉시 전송 보장

---

## 예시 📌

### ✅ 텍스트 파일 읽기

```java
BufferedReader br = new BufferedReader(new FileReader("memo.txt"));
String line;
while ((line = br.readLine()) != null) {
    System.out.println(line); // 한 줄 단위 출력
}
br.close();
```

---

### ✅ 텍스트 파일 쓰기

```java
PrintWriter pw = new PrintWriter(new FileWriter("log.txt"));
pw.println("로그 기록 시작");
pw.printf("현재 시간: %tT%n", new java.util.Date());
pw.close();
```

---

### ✅ 네트워크 채팅

```java
BufferedReader br = new BufferedReader(new InputStreamReader(socket.getInputStream()));
PrintWriter pw = new PrintWriter(socket.getOutputStream(), true);

String msg = br.readLine();          // 클라이언트 메시지 읽기
pw.println("서버 응답: " + msg);      // 클라이언트에게 답장
```


#### 🔄 흐름 정리: 문자열을 읽는 과정
1. `socket.getInputStream()` → 네트워크에서 들어오는 **바이트 스트림**을 가져옴 (기반 스트림)
    
2. `InputStreamReader(...)` → 바이트를 **문자(char)**로 변환하는 **보조 스트림** → 문자 인코딩(`UTF-8`, `EUC-KR` 등)에 따라 바이트를 해석함
3. `BufferedReader(...)` → 문자 스트림을 **버퍼링**해서 효율적으로 읽음 → `readLine()` 같은 편리한 메서드 제공
#### 🔄 흐름 정리: 문자열을 쓰는 과정
1. `socket.getOutputStream()` → 네트워크로 데이터를 보내는 **바이트 스트림**을 가져옴 (기반 스트림) → 이 스트림은 바이트 단위로만 데이터를 전송할 수 있음
2. `OutputStreamWriter(...)` _(내부적으로 사용됨)_ → **문자(char)**를 **바이트**로 변환하는 **보조 스트림** → 문자 인코딩(`UTF-8`, `EUC-KR` 등)에 따라 문자를 바이트로 인코딩함 → `PrintWriter`는 내부적으로 이 과정을 자동으로 처리함
3. `PrintWriter(...)` → 문자열을 편리하게 출력할 수 있는 **보조 스트림** → `print()`, `println()`, `printf()` 등 다양한 출력 메서드 제공 → `true` 설정 시 `println()` 호출 시마다 **자동 flush**되어 즉시 전송됨
---

## ✅ 최종 정리

- **기반 스트림**: 실제 데이터 통로
- **보조 스트림**: 기반 스트림을 감싸 기능/성능 향상
- **선택 기준**: 데이터 성격, 처리 단위, 성능 요구사항

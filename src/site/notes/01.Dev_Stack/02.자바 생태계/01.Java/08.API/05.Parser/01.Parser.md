# 📘 Java Parser 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - **Parser**: Java에서 데이터를 특정 형식(문자열, 파일 등)에서 읽어 구조화된 객체 또는 데이터로 변환하는 도구 또는 클래스.
> - 다양한 데이터 형식(JSON, XML, 문자열 등)을 처리하며, 애플리케이션에서 데이터 분석 및 처리에 사용.

---

## 목적 🎯

- 외부 데이터(문자열, 파일, 네트워크 입력)를 구조화된 형태로 변환하여 프로그램 내에서 사용 가능하도록 함.
- 데이터 검증, 변환, 처리 작업을 통해 입력 데이터를 의미 있는 객체로 매핑.
- API 응답, 설정 파일, 사용자 입력 등을 처리하여 애플리케이션 로직에 활용.

---

## 핵심 개념 💡

- **정의**: Parser는 입력 데이터를 분석(parse)하여 프로그램이 이해할 수 있는 구조(예: 객체, 트리, 맵)로 변환하는 역할 수행.
- **종류**:
    - **문자열 파서**: 문자열을 구분자나 패턴으로 분리 (예: String.split, Scanner).
    - **구문 분석 파서**: 코드나 데이터를 문법 규칙에 따라 분석 (예: Java 코드 파싱).
    - **데이터 형식 파서**: JSON, XML, CSV 등 특정 형식 처리 (예: JsonParser, SAXParser).
- **작동 원리**: 입력 데이터를 토큰으로 분리 → 문법 검사 → 구조화된 출력 생성.
- **라이브러리**: Java 표준 라이브러리(java.util, javax.xml.parsers)와 서드파티 라이브러리(Gson, Jackson) 활용.

#### 비교표: 주요 Java Parser 도구

|파서 종류|예시 클래스/라이브러리|용도|특징|
|---|---|---|---|
|**문자열 파서**|String.split, Scanner|문자열 분리 및 처리|간단, 정규 표현식 지원|
|**JSON 파서**|JsonParser(Jackson), Gson|JSON 데이터를 객체로 변환|객체 매핑, 유연성 높음|
|**XML 파서**|SAXParser, DOMParser|XML 문서 분석|스트림 기반(SAX) 또는 트리 기반(DOM)|
|**구문 분석 파서**|JavaParser(서드파티)|Java 소스 코드 분석|AST(추상 구문 트리) 생성|

## 특징 ✨

- ✅ **유연성**: 다양한 데이터 형식(JSON, XML, CSV, 사용자 정의 문자열 등) 처리 가능.
- ✅ **확장성**: 서드파티 라이브러리(Gson, Jackson)로 복잡한 데이터 구조 지원.
- ✅ **에러 처리**: 잘못된 입력에 대해 예외 처리 및 검증 기능 제공.
- ✅ **효율성**: 스트림 기반 파싱(SAX) 또는 메모리 기반 파싱(DOM) 선택 가능.
- ✅ **사용 사례**: API 데이터 처리, 설정 파일 로드, 로그 분석, 코드 분석 도구.

---

## 팁과 주의점 ⚠️

- ✅ **데이터 형식 확인**: 입력 데이터 형식이 일치하는지 사전 검증 권장.
- ✅ **성능 고려**: 대량 데이터 처리 시 스트림 기반 파서(SAX, JsonParser) 사용으로 메모리 절약.
- ⚠️ **예외 처리**: 잘못된 입력에 대비해 try-catch로 예외 관리 필수.
- ⚠️ **라이브러리 선택**: 표준 라이브러리(javax.xml.parsers)는 간단한 작업에 적합, 복잡한 경우 Jackson/Gson 권장.
- ✅ **재사용성**: 자주 사용하는 파서 로직은 유틸리티 클래스로 캡슐화 권장.

## 예시 📌

### ✅ JSON 데이터 파싱 (Jackson JsonParser)

```java
import com.fasterxml.jackson.core.JsonParser;
import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;

public class JsonParserExample {
    public static void main(String[] args) {
        String json = "{\"name\":\"John\", \"age\":30}";
        ObjectMapper mapper = new ObjectMapper();

        try {
            // JSON 문자열을 JsonNode로 파싱
            JsonNode node = mapper.readTree(json);
            
            // 필드 값 추출
            String name = node.get("name").asText();
            int age = node.get("age").asInt();
            
            // 결과 출력
            System.out.println("Name: " + name); // 출력: Name: John
            System.out.println("Age: " + age);   // 출력: Age: 30
        } catch (Exception e) {
            System.err.println("파싱 오류: " + e.getMessage());
        }
    }
}
```

**실행 흐름**:

1. Jackson의 ObjectMapper로 JSON 문자열 입력.
2. readTree 메서드로 JSON을 JsonNode 객체로 파싱.
3. 필드(name, age)를 추출하여 변수에 저장.
4. 예외 처리로 잘못된 JSON 입력 대비.

### ✅ XML 데이터 파싱 (SAXParser)

```java
import javax.xml.parsers.SAXParser;
import javax.xml.parsers.SAXParserFactory;
import org.xml.sax.Attributes;
import org.xml.sax.helpers.DefaultHandler;

public class SAXParserExample {
    public static void main(String[] args) {
        try {
            // SAXParserFactory 생성
            SAXParserFactory factory = SAXParserFactory.newInstance();
            SAXParser saxParser = factory.newSAXParser();

            // XML 파싱 핸들러 정의
            DefaultHandler handler = new DefaultHandler() {
                boolean isName = false;

                @Override
                public void startElement(String uri, String localName, String qName, Attributes attributes) {
                    if (qName.equalsIgnoreCase("name")) {
                        isName = true; // name 태그 시작
                    }
                }

                @Override
                public void characters(char[] ch, int start, int length) {
                    if (isName) {
                        System.out.println("Name: " + new String(ch, start, length));
                        isName = false; // name 태그 처리 완료
                    }
                }
            };

            // XML 문자열 파싱
            String xml = "<person><name>John</name><age>30</age></person>";
            saxParser.parse(new java.io.ByteArrayInputStream(xml.getBytes()), handler);
        } catch (Exception e) {
            System.err.println("파싱 오류: " + e.getMessage());
        }
    }
}
```

**실행 흐름**:

1. SAXParserFactory로 SAXParser 생성.
2. DefaultHandler를 확장하여 XML 요소(name) 처리 로직 구현.
3. XML 문자열을 바이트 스트림으로 변환 후 파싱.
4. name 태그의 값을 출력 (예: "Name: John").

# 📘 Java CompactNumberFormat 정리

## Summary 📝
> [!summary] Summary 📝
> - `CompactNumberFormat`: 큰 숫자를 간결한 형식(예: 1K, 1M, 1만)으로 포맷팅하는 클래스.  
> - 로케일별 표현과 스타일(SHORT, LONG) 지원으로 사용자 친화적 숫자 표시 제공.

---

## 목적 🎯
- 숫자를 간결하고 읽기 쉬운 형태로 변환하여 UI/UX 개선.  
- 다국어 애플리케이션에서 로케일별 숫자 단위(예: 천, K, million) 표현.  
- 보고서, 통계 데이터 등에서 가독성 높은 숫자 표시.

---

## 핵심 개념 💡
- **클래스**: `java.text.CompactNumberFormat` (Java 12 이상).  
- **기능**: 큰 숫자를 로케일별 단위로 간소화 (예: 1000 → 1K, 10000 → 1만).  
- **스타일**: 
  - `SHORT`: 간결한 표현 (예: 1K, 1M).  
  - `LONG`: 설명적 표현 (예: 1천, 1백만).  
- **로케일**: 지역별 접미사와 숫자 형식 지원 (예: 한국어-만, 영어-K).  
- **커스터마이징**: 소수점 자리 수, 접미사 배열 사용자 정의 가능.

#### 비교표: CompactNumberFormat vs. NumberFormat
| 항목               | CompactNumberFormat                     | NumberFormat                     |
|--------------------|-----------------------------------------|----------------------------------|
| **용도**           | 간결한 숫자 표현 (예: 1K, 1M)          | 일반 숫자 포맷팅 (예: 1,000)    |
| **스타일**         | SHORT, LONG                            | 일반, 통화, 퍼센트              |
| **로케일 지원**    | O (지역별 접미사)                      | O (지역별 형식)                 |
| **커스터마이징**   | 접미사, 소수점 설정 가능               | 패턴, 기호 설정 가능            |
| **Java 버전**      | Java 12 이상                           | Java 1.1 이상                   |

---

## 특징 ✨
- ✅ **간결성**: 큰 숫자를 짧고 직관적인 형태로 변환 (예: 1000000 → 1M).  
- ✅ **로케일 지원**: 한국어(만, 억), 영어(K, M), 기타 지역별 표현 제공.  
- ✅ **유연성**: 소수점 자리, 접미사 등 사용자 정의 가능.  
- ✅ **상속 구조**: `NumberFormat` 상속으로 기존 포맷팅 기능 확장.  
- ✅ **사용 사례**: 소셜 미디어 조회 수, 앱 통계, 금융 데이터 표시 등.

---

## 팁과 주의점 ⚠️
- ✅ **로케일 테스트**: 지역별 접미사 차이로 인해 로케일별 출력 확인 필수.  
- ✅ **소수점 조정**: `setMaximumFractionDigits`로 소수점 자리 수 설정.  
- ⚠️ **스레드 안전**: 다중 스레드 환경에서 동기화 필요.  
- ⚠️ **Java 버전**: Java 12 이상에서만 사용 가능, 하위 호환성 주의.  
- ✅ **인스턴스 재사용**: 성능 최적화를 위해 객체 재사용 권장.

## 예시 📌
### ✅ 한국어 로케일에서 숫자 포맷팅
```java
import java.text.CompactNumberFormat;
import java.text.DecimalFormatSymbols;
import java.text.NumberFormat;
import java.util.Locale;

public class CompactNumberFormatExample {
    public static void main(String[] args) {
        // 한국어 로케일로 SHORT 스타일 설정
        CompactNumberFormat cnf = new CompactNumberFormat(
            NumberFormat.getNumberInstance(Locale.KOREA).toPattern(), // 기본 숫자 패턴
            DecimalFormatSymbols.getInstance(Locale.KOREA),           // 로케일별 기호 (쉼표, 소수점 등)
            new String[] { "", "천", "백만", "십억", "조" }          // 사용자 정의 접미사 배열
        );

        // 소수점 최대 자리 수 설정
        cnf.setMaximumFractionDigits(1);

        // 숫자 포맷팅 출력
        System.out.println(cnf.format(1000));      // 출력: 1천
        System.out.println(cnf.format(1500000));   // 출력: 1.5백만
        System.out.println(cnf.format(2000000000)); // 출력: 20억
    }
}
```
**실행 흐름**:  
1. `CompactNumberFormat` 인스턴스 생성, 한국어 로케일과 사용자 정의 접미사 지정.  
2. 소수점 자리 수 설정 (`setMaximumFractionDigits`).  
3. `format` 메서드로 숫자를 로케일에 맞춘 간결한 문자열로 변환.  
4. 출력 결과는 한국어 접미사(천, 백만, 십억) 적용.

### ✅ 영어 로케일에서 SHORT와 LONG 스타일 비교
```java
import java.text.CompactNumberFormat;
import java.text.NumberFormat;
import java.util.Locale;

public class CompactNumberFormatStyleExample {
    public static void main(String[] args) {
        // 영어 로케일, SHORT 스타일
        CompactNumberFormat cnfShort = (CompactNumberFormat) NumberFormat.getCompactNumberInstance(
            Locale.US, NumberFormat.Style.SHORT);
        // 영어 로케일, LONG 스타일
        CompactNumberFormat cnfLong = (CompactNumberFormat) NumberFormat.getCompactNumberInstance(
            Locale.US, NumberFormat.Style.LONG);

        // 숫자 포맷팅 출력
        System.out.println("SHORT: " + cnfShort.format(1000000)); // 출력: 1M
        System.out.println("LONG: " + cnfLong.format(1000000));   // 출력: 1 million
    }
}
```
**실행 흐름**:  
1. `getCompactNumberInstance`로 영어 로케일의 `SHORT`와 `LONG` 스타일 인스턴스 생성.  
2. 동일한 숫자(1000000)를 각 스타일로 포맷팅.  
3. `SHORT`는 간결한 "1M", `LONG`은 "1 million" 출력.

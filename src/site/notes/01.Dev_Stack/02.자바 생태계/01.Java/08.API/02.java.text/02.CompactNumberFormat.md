# Java CompactNumberFormat 핵심 정리
> [!summary] 핵심 요약
> 
> - **개념**: 큰 숫자를 사람이 읽기 쉬운 간결한 단위(예: 1K, 1M, 1만)로 축약하여 포맷팅하는 기능
>     
> - **왜?**: 화면 공간이 제한된 UI에서 조회수, 통계, 금액 등의 가독성을 극대화하기 위함
>     
> - **언제?**: ==Java 12 이상== 환경의 대시보드, 소셜 미디어 지표, 리포트 출력 계층
>     
> - **핵심**: `Locale`에 따라 서구권의 1,000 단위(K, M)와 동아시아권의 10,000 단위(만, 억) 자동 변환 지원
>     
## 핵심 원리 및 스타일 비교
- **동작 방식**
    - `NumberFormat` 클래스를 상속받아 구현된 특수 목적의 포맷터
    - `NumberFormat.getCompactNumberInstance()` 팩토리 메서드를 통해 인스턴스 획득
    - 소수점 최대/최소 자릿수(`setMaximumFractionDigits` 등) 설정으로 정밀도 조절 가능
- **지원 스타일 (`NumberFormat.Style`)**
    - `SHORT`: 공간 절약을 위한 최적의 축약형 (예: 1K, 1만)
    - `LONG`: 단위를 풀어서 설명하는 형태 (예: 1 thousand, 1만)

|**비교 항목**|**CompactNumberFormat**|**일반 NumberFormat**|
|---|---|---|
|**주요 목적**|큰 숫자의 직관적 축약 표현|정확한 자릿수 및 기호(,) 표현|
|**출력 예시**|`1K`, `1.5M`, `2만`|`1,000`, `1,500,000`, `20,000`|
|**스타일 종류**|`SHORT`, `LONG`|Number, Currency, Percent|
|**단위 기준**|로케일 종속적 (10^3 또는 10^4)|고정된 천 단위 구분자(,)|
|**요구 버전**|Java 12 이상|Java 1.1 이상|

## 구현 방식 및 주의사항
- **로케일 기반 자동 변환**
    - 서구권(`Locale.US`): 1,000 단위로 축약 (`K` -> `M` -> `B`)
    - 한국어(`Locale.KOREA`): 10,000 단위로 축약 (`만` -> `억` -> `조`)
    - **어떻게?**: 개발자가 수동으로 배열을 선언할 필요 없이 자바 표준 API에 위임
- **멀티스레드 환경 주의**
    - 부모 클래스인 `NumberFormat`과 마찬가지로 ==상태를 가지므로 Thread-Safe하지 않음==
    - 다중 스레드 환경에서는 `ThreadLocal`로 감싸거나 필요 시점마다 새로 생성해야 함
```java
import java.text.NumberFormat;
import java.util.Locale;

public class CompactFormatStandardExample {
    public static void main(String[] args) {
        long targetNumber = 1500000; // 150만

        // 1. 한국어 로케일 (10,000 단위 분기)
        NumberFormat krShort = NumberFormat.getCompactNumberInstance(Locale.KOREA, NumberFormat.Style.SHORT);
        krShort.setMaximumFractionDigits(1); // 소수점 첫째 자리까지 표시
        System.out.println("한국어 SHORT: " + krShort.format(targetNumber)); // 출력: 150만

        // 2. 영어 로케일 (1,000 단위 분기)
        NumberFormat usShort = NumberFormat.getCompactNumberInstance(Locale.US, NumberFormat.Style.SHORT);
        usShort.setMaximumFractionDigits(1);
        System.out.println("미국 SHORT: " + usShort.format(targetNumber));   // 출력: 1.5M

        // 3. 영어 로케일 LONG 스타일 비교
        NumberFormat usLong = NumberFormat.getCompactNumberInstance(Locale.US, NumberFormat.Style.LONG);
        usLong.setMaximumFractionDigits(1);
        System.out.println("미국 LONG: " + usLong.format(targetNumber));     // 출력: 1.5 million
    }
}
```
```plaintext
[ 로케일별 숫자 축약 분기 로직 ]

입력값: 15,000,000 (천오백만)

       ┌── (Locale.US) ──▶ [1,000 단위 나눗셈] ──▶ 15 M (또는 million)
       │
1.5E7 ─┤
       │
       └── (Locale.KOREA) ──▶ [10,000 단위 나눗셈] ──▶ 1,500 만
```

### 💡 Note Context
- **수정/개선점**: 원본 노트의 첫 번째 예시 코드에서 사용된 '사용자 정의 접미사 배열(`new String[] { "", "천", "백만", "십억", "조" }`)'은 한국어의 자연스러운 숫자 읽기 방식(1만 단위)을 무시하고 서구권의 1,000 단위 분기에 억지로 한글을 끼워 맞춘 안티패턴(Anti-Pattern)이므로 삭제함. 대신 Java 12의 내장 기능을 활용하여 `getCompactNumberInstance()`로 한국어(만 단위)와 영어(천 단위)가 자연스럽게 포맷팅되는 표준 방식을 적용하여 코드를 전면 재작성함.
- **AI 인사이트**: `CompactNumberFormat`은 프론트엔드/클라이언트 위주의 포맷팅이 서버 사이드로 넘어올 때 유용함. 단, 글로벌 서비스 설계 시 서버는 원본 숫자(Raw Data)를 API로 내려주고, 실제 문자열 포맷팅은 클라이언트(웹/앱의 `Intl.NumberFormat` 등)에서 처리하는 것이 확장성과 로케일 대응 측면에서 더 유리한 아키텍처일 수 있음.

## 📘 java.text 핵심 포맷팅 및 비교 클래스 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - `java.text` 패키지의 핵심 클래스들로, **숫자/통화 포맷팅**과 **다국어 메시지 처리**, **로케일 기반 문자열 비교** 기능을 제공함.
>     
> - **국제화(i18n)** 지원에 필수적이며, 특히 `NumberFormat` 계열은 데이터 출력 시 **지역별 형식**을 적용하는 표준 방법임.
>     

---

## 목적 🎯

- **데이터 국제화:** 로케일(Locale) 기반으로 숫자, 통화, 메시지, 문자열 비교를 처리하여 다국어 환경을 지원.
    
- **출력 일관성 및 가독성 확보:** 숫자, 통화, 메시지 등의 출력을 사용자에게 익숙한 지역 형식으로 통일.
    

---

## 핵심 개념 💡

### ✅ 포맷팅, 메시징, 비교 기능

| **클래스명**            | **용도**                       | **패턴 및 예시**                                              |
| ------------------- | ---------------------------- | -------------------------------------------------------- |
| **`NumberFormat`**  | 숫자, 통화, 퍼센트 형식 지정            | `NumberFormat.getCurrencyInstance()` <br>(로케일별 통화 포맷)    |
| **`DecimalFormat`** | 세밀한 숫자 포맷 지정                 | `"#,###.00"` (패턴 기반 정밀 포맷팅)                              |
| **`MessageFormat`** | 자리 표시자 기반 문자열 조합, 다국어 메시지 처리 | `"사용자 {0}님, {1}건의 메시지가 있습니다."`                           |
| **`Collator`**      | 문자열 비교 시 로케일별 규칙 적용          | `Collator.getInstance(Locale.KOREAN)` <br>(언어별 정렬 규칙 적용) |

---

## 특징 ✨

### ✅ 포맷팅 및 파싱 지원

- **`NumberFormat` 및 `DecimalFormat`:** 객체(`Number`)를 문자열로 변환하는 **포맷팅(Formatting)**과 문자열을 다시 객체로 변환하는 **파싱(Parsing)** 기능을 모두 제공함.
    
- **로케일 기반:** `Locale`에 따라 통화 기호, 소수점/천 단위 구분자가 자동으로 변경되어 **국제화 지원이 용이**함.
    

### ✅ 유연한 텍스트 조합 및 비교

- **`MessageFormat`:** 다양한 타입의 객체(숫자, 날짜 등)를 **자리 표시자**에 넣어 메시지를 동적으로 완성하며, 삽입되는 객체에도 로케일 기반의 포맷이 적용됨.
    
- **`Collator`:** 단순 유니코드 값 비교가 아닌, **언어학적 규칙**을 기반으로 문자열을 비교하여 **다국어 정렬의 정확성**을 높임.
    

---

## 팁과 주의점 ⚠️

- **멀티스레드 환경 주의:** `DecimalFormat`은 **스레드 안전성 문제**가 있을 수 있으므로, 멀티스레드 환경에서는 **`ThreadLocal`** 사용 또는 **메서드 호출 시마다 새 인스턴스 생성**을 권장함.
    
- **`MessageFormat` 이스케이프:** 메시지 패턴 내에서 중괄호 `{`를 일반 문자로 사용하려면 **작은따옴표 두 개**(`''`)로 이스케이프 처리해야 함.
    
- **`Collator` 필수성:** 한국어, 독일어 등 언어별 정렬 규칙이 복잡한 문자열을 정확하게 정렬해야 할 때는 반드시 `Collator`를 사용해야 함.

## 예시 📌

### ✅ 통화 포맷팅 및 메시지 조합 사례

로케일(`Locale.US`) 기반으로 숫자를 통화 형식으로 포맷하고, 그 결과를 `MessageFormat`을 사용하여 메시지에 삽입하는 코드 실행 사례.

```java
import java.text.NumberFormat;
import java.text.MessageFormat;
import java.util.Locale;

public class FormatExample {
    public static void main(String[] args) {
        double balance = 987654.32;
        
        // 1. 미국 로케일 기반의 통화 포맷터 획득 (예: $987,654.32)
        NumberFormat usCurrencyFormat = NumberFormat.getCurrencyInstance(Locale.US);
        
        // 2. MessageFormat 패턴 설정 (자리 표시자 {0}에 포맷된 통화 삽입)
        String pattern = "고객님의 현재 잔액은 {0}입니다. 감사합니다.";
        MessageFormat formatter = new MessageFormat(pattern, Locale.KOREA);
        
        // 3. 통화 포맷터를 MessageFormat에 적용하고 포맷팅
        // (Locale.US로 포맷된 통화 문자열이 {0}에 삽입됨)
        String formattedCurrency = usCurrencyFormat.format(balance);
        
        // 4. 최종 메시지 출력
        Object[] arguments = {formattedCurrency};
        String finalMessage = formatter.format(arguments);

        System.out.println("--- 실행 흐름 ---");
        System.out.println("포맷된 통화: " + formattedCurrency);
        System.out.println("최종 메시지: " + finalMessage);
        
        /*
        --- 실행 흐름 ---
        포맷된 통화: $987,654.32
        최종 메시지: 고객님의 현재 잔액은 $987,654.32입니다. 감사합니다.
        */
    }
}
```

---

**개선점:**

- **내용 압축:** `java.text` 패키지 클래스 중 요청된 4가지 클래스에만 집중하여 내용을 구성함.
    
- **정확한 설명:** `NumberFormat`과 `DecimalFormat`은 비추천 클래스가 아님을 명확히 하고, **스레드 안전성 주의점**을 팁에 기술함.
    
- **형식 준수:** 요청된 모든 마크다운 형식(콜아웃, 구분선, 명사형 종결, 볼드, 표)을 준수함.

# java.text 핵심 클래스 아키텍처
> [!summary] 핵심 요약
> 
> - **개념**: 데이터(숫자/문자열)를 지역별(Locale) 형식으로 변환하거나 언어권 기준에 맞춰 비교하는 Java 표준 API 세트
>     
> - **왜?**: 국가마다 다른 숫자 표기(천 단위 구분자, 통화 기호 등)와 언어별 정렬 규칙을 일관되게 처리하기 위함
>     
> - **언제?**: 글로벌 서비스(다국어 지원) 개발 시, 동적 메시지 조립 및 사용자 화면(View)에 데이터 출력 계층
>     
> - **핵심**: 멀티스레드 환경에서 `DecimalFormat` 사용 시 발생하는 ==스레드 안전성(Thread-Safety) 문제 인지 및 대응==
>     
## 핵심 클래스 및 기능
- **포맷팅 및 메시징 도구**
    - 숫자, 통화, 다국어 메시지 처리를 위한 `java.text` 패키지의 주요 구성 요소

|**클래스명**|**용도 및 역할**|**주요 특징**|
|---|---|---|
|`NumberFormat`|국가별 표준 숫자/통화/퍼센트 포맷 지정|팩토리 메서드(`getInstance`)를 통해 로케일 기반 객체 생성|
|`DecimalFormat`|사용자 정의 기호(`#`, `0`)를 활용한 정밀 숫자 포맷 지정|`NumberFormat`의 하위 클래스로 상세한 패턴 설정 가능|
|`MessageFormat`|다국어 템플릿에 동적 데이터를 주입하여 메시지 완성|매개변수 인덱스(`{0}`, `{1}`)를 활용한 데이터 바인딩|
|`Collator`|언어권별 사전적 정렬 규칙 적용|단순 유니코드가 아닌 언어학적 정렬 규칙 지원|

## 숫자 데이터 포맷팅 (Formatting)
- **NumberFormat 및 DecimalFormat**
    - 데이터(객체)를 지정된 형식의 문자열로 변환(Format)하거나, 문자열을 다시 데이터로 변환(Parse)하는 양방향 처리 지원
    - `Locale` 객체와 결합하여 통화 기호나 소수점 구분자를 자동으로 치환
    - 멀티스레드 환경 적용 시: `DecimalFormat`은 내부 상태를 변경하므로 ==공유 자원으로 사용 시 동시성 이슈 발생 (ThreadLocal 객체로 래핑하거나 지역 변수로 생성하여 격리 필수)==
    - **어떻게?**: 안전한 포맷터 래핑 및 로케일 기반 생성
```java
import java.text.DecimalFormat;
import java.text.NumberFormat;
import java.util.Locale;

public class FormatterExample {
    // Thread-Safe를 보장하기 위한 ThreadLocal 래핑
    private static final ThreadLocal<DecimalFormat> safeFormatter = 
        ThreadLocal.withInitial(() -> new DecimalFormat("#,###.00"));

    public static void main(String[] args) {
        double balance = 987654.32;

        // 1. Thread-Safe한 사용자 정의 정밀 포맷 출력
        // 0은 빈자리를 채우고, #은 빈자리를 채우지 않음
        System.out.println("커스텀 포맷: " + safeFormatter.get().format(balance)); // 987,654.32
        System.out.println("사번 포맷: " + new DecimalFormat("000000").format(123)); // 000123

        // 2. Locale 기반 표준 통화 포맷 출력 (US/KOREA)
        NumberFormat usCurrency = NumberFormat.getCurrencyInstance(Locale.US);
        NumberFormat krCurrency = NumberFormat.getCurrencyInstance(Locale.KOREA);
        
        System.out.println("미국 통화: " + usCurrency.format(balance)); // $987,654.32
        System.out.println("한국 통화: " + krCurrency.format(balance)); // ₩987,654
    }
}
```
## 동적 메시지 및 정렬 (Messaging & Collation)
- **MessageFormat**
    - 문자열 내부에 자리 표시자(`{0}`, `{1}`)를 배치하고 배열 형태의 데이터를 동적으로 바인딩
    - 주입되는 데이터(숫자 등) 역시 지정된 로케일 포맷을 따름
    - 특수문자 제약: 템플릿 내에서 ==중괄호 `{` 기호를 단순 문자로 노출하려면 반드시 작은따옴표 두 개(`''`)로 이스케이프 처리==
    - **어떻게?**: 다국어 템플릿에 로케일이 적용된 데이터 주입
```plaintext
[ MessageFormat 데이터 주입 흐름 (Data Injection Flow) ]

    데이터 배열 (Arguments)                  메시지 템플릿 (Template)
    Object[] args                     "사용자 {0}님, 잔액은 {1}입니다."
  ┌─────────────────┐                  │                   │
  │ [0]: "홍길동"   │────────(매핑)───▶│                   │
  │ [1]: $987,654.32│────────(매핑)────┴──────────────────▶│
  └─────────────────┘                                      ▼
                                                  [ 최종 완성 문자열 ]
                                   "사용자 홍길동님, 잔액은 $987,654.32입니다."
```
- **Collator**
    - 독일어 움라우트, 프랑스어 악센트, 한국어 자모음 순서 등 언어학적 특성을 반영한 정밀 문자열 비교
    - 단순 `String.compareTo()`의 유니코드 값 비교가 가지는 한계 극복
    - **어떻게?**: `getInstance(Locale)`을 통해 특정 국가의 사전적 정렬 규칙을 컬렉션 정렬에 주입
```java
import java.text.MessageFormat;
import java.text.Collator;
import java.util.Arrays;
import java.util.List;
import java.util.Locale;

public class MessagingAndCollationExample {
    public static void main(String[] args) {
        // 1. MessageFormat (동적 데이터 주입)
        String template = "알림: {0}님의 등급은 [{1}]이며, 포인트는 {2}점입니다.";
        Object[] argsData = {"홍길동", "VIP", 5000};
        System.out.println(MessageFormat.format(template, argsData));
        // 출력: 알림: 홍길동님의 등급은 [VIP]이며, 포인트는 5,000점입니다.

        // 2. Collator (언어권별 사전적 정렬)
        List<String> words = Arrays.asList("apple", "Banana", "cherry");
        
        // 단순 유니코드(대문자 우선)가 아닌, 실제 언어 사전 규칙(소문자/대문자 무관한 알파벳 순)에 따라 정렬
        words.sort(Collator.getInstance(Locale.US)); 
        System.out.println("정렬 결과: " + words); 
        // 출력: [apple, Banana, cherry]
    }
}
```

### 💡 Note Context
- **수정/개선점**: 단일 스니펫으로 묶여 있던 코드 블록을 개념별(`NumberFormat`, `MessageFormat/Collator`)로 분리 및 배치하여 가독성 개선, `MessageFormat` 바인딩 원리를 도식화(ASCII Art)하여 추가함.
- **AI 인사이트 (내용 보완 및 연관 키워드 제안)**: 글로벌 서비스에서는 시간 처리도 필수적이므로, `java.time.format.DateTimeFormatter` (Thread-Safe)와의 연계 학습을 권장함. 기존 구형 클래스인 `SimpleDateFormat`의 멀티스레드 취약점과 비교하는 내용을 추가로 구성하면 좋음.

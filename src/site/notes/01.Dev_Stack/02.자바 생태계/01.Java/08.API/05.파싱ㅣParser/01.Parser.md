# Java 파서 (Parser) 핵심 및 구현
> [!summary] 핵심 요약
> 
> - **개념**: 외부 원시 데이터(JSON, XML, CSV, 일반 문자열 등)를 읽어 분석한 뒤, 자바 애플리케이션에서 사용 가능한 구조화된 객체(Object)나 데이터 트리로 변환하는 도구
>     
> - **왜?**: 네트워크 통신, 파일 읽기 등으로 수신한 텍스트 덩어리는 프로그램 로직에서 즉시 계산이나 조작이 불가능하므로, 의미를 가진 타입으로 매핑해야 함
>     
> - **언제?**: 외부 API 응답 처리, 애플리케이션 설정 파일(XML/JSON) 로드, 대용량 로그 및 CSV 파일 데이터 추출 시 주로 사용
>     
> - **핵심**: 데이터의 크기, 검색 필요성, 구조의 복잡성에 따라 **메모리 기반(DOM)과 스트림 기반(SAX) 파싱을 적절히 선택**하여 자원(메모리)을 최적화하는 것
>     
## 파싱의 동작 원리 및 파서 분류
- **파싱의 기본 동작 파이프라인**
    - 입력된 텍스트를 파서가 가장 작은 의미 단위인 ==토큰(Token)==으로 분리 (Lexical Analysis)
    - 토큰 간의 구조와 문법적 유효성 및 데이터 타입 검증 (Syntax Analysis)
    - 검증된 데이터를 바탕으로 메모리 상에 즉시 사용 가능한 인스턴스(객체) 생성
    - 입력 데이터의 포맷이 정의된 규격이나 내부 클래스 구조와 1자리라도 일치하지 않으면 즉시 파싱 예외 발생
```plaintext
[ Data Parsing Flow ]

  Raw Data           1. Tokenizer         2. Syntax Logic        Object Result
 ┌──────────┐       ┌──────────┐         ┌──────────────┐       ┌──────────┐
 │ {"id":1} │ ────▶ │ "{", "id"│ ──────▶ │ Valid Check  │ ────▶ │ User Obj │
 │          │       │  ":", "1"│         │              │       │  id = 1  │
 └────┬─────┘       └────┬─────┘         └──────┬───────┘       └────┬─────┘
      │                  │                      │                    │
  원본 문자열       어휘 단위 분리       타입/문법 검증        자바 객체 생성
```

| **파서 종류**     | **대표 클래스 및 라이브러리**        | **데이터 처리 방식**      | **용도 및 핵심 특징**                 |
| ------------- | ------------------------- | ------------------ | ------------------------------ |
| **단순 문자열**    | `String.split`, `Scanner` | 메모리 로드 및 구분        | 정규식이나 특정 구분자로 텍스트 단순 분할        |
| **JSON**      | `Jackson`, `Gson`         | 양방향 객체 매핑          | ==웹 API 통신 표준==, 구조 변환의 유연성 제공 |
| **XML (트리)**  | `DOMParser`               | 메모리 전체 적재 (Tree)   | 문서 전체 구조 파악 및 특정 노드 검색/수정에 유리  |
| **XML (스트림)** | `SAXParser`               | 순차적 스트림 읽기 (Event) | 메모리 소모가 극히 적어 대용량 문서 처리에 필수적   |

## 주요 파서 종류별 특징 및 구현 예시
### 1. 단순 문자열 파서 (String / Scanner)
- **개념**: 별도의 복잡한 라이브러리 없이 특정 구분자(Delimiter)를 기준으로 문자열을 배열이나 토큰으로 분할하는 가장 기본적인 파싱 방식
- **어떻게?**: `String.split()`으로 배열화하거나, `Scanner`의 `useDelimiter()`를 활용하여 스트림 텍스트 처리
```java
String csvData = "홍길동,30,서울시";

// 1. split()을 이용한 단순 파싱
String[] tokens = csvData.split(",");
System.out.println("이름: " + tokens[0]); // 홍길동

// 2. Scanner를 이용한 토큰 파싱(구식)
try (Scanner sc = new Scanner(csvData)) {
	sc.useDelimiter(",");
	while (sc.hasNext()) {
		System.out.println("데이터: " + sc.next());
	}
}

// 3. Stream을 이용한 방식(현대적)
List<String> result = Arrays.stream(tokens) // 1. 스트림 생성
		.map(String::trim)                  // 2. 공백 제거 (가공)
		.filter(s -> !s.isEmpty())          // 3. 빈 문자열 제외 (필터링)
		.collect(Collectors.toList());      // 4. 결과 저장
```
### 2. JSON 파서 (Jackson 라이브러리)
- **개념**: JSON 형식의 문자열을 자바 객체(DTO)로 변환(역직렬화)하는 웹 통신의 표준 방식
- `ObjectMapper` 인스턴스는 생성 비용이 매우 높으므로 반드시 ==상수로 선언하거나 스프링 빈(Bean)으로 등록==하여 애플리케이션 전역에서 재사용할 것
- **어떻게?**: `readValue()`로 특정 클래스 타입에 직접 매핑 (현대 자바에서는 `record`와 `Text Block`을 조합하여 가독성과 불변성을 확보하는 것을 권장)
```java
// 1. Record를 활용한 간결한 DTO
public record User(String name, int age) {}

public class JsonParserExample {
    // 2. 성능 최적화를 위한 싱글톤/상수 파서 객체 생성
    private static final ObjectMapper MAPPER = new ObjectMapper();

    public static void main(String[] args) {
        // 3. Text Block (Java 15+) 사용으로 JSON 문자열 가독성 향상
        String jsonText = """
                {
                  "name": "Alice",
                  "age": 25
                }
                """;

        try {
            // 4. 파싱 (JSON 문자열 -> 자바 Record 객체)
            User user = MAPPER.readValue(jsonText, User.class);
            
            // record는 getter(getName) 대신 필드명과 동일한 name() 메서드 제공
            System.out.println("파싱 완료: " + user.name()); 
        } catch (Exception e) {
            System.err.println("JSON 문법 오류 또는 타입 불일치");
        }
    }
}
```
### 3. XML 파서 - DOM (Document Object Model) 기반
- **개념**: XML 문서 전체를 읽어 메모리 상에 트리(Tree) 구조로 올려놓고 분석하는 방식
- **특징**: 특정 노드의 검색, 추가, 삭제 등 문서 조작이 매우 자유로우나, 파일 크기가 클 경우 `OutOfMemoryError` 발생 위험이 큼
- **어떻게?**: `DocumentBuilder`를 통해 문서를 파싱하고 `getElementsByTagName` 등으로 탐색
```java
import javax.xml.parsers.DocumentBuilder;
import javax.xml.parsers.DocumentBuilderFactory;
import org.w3c.dom.Document;
import org.w3c.dom.NodeList;
import java.io.ByteArrayInputStream;

public class DomParserExample {
    public static void main(String[] args) {
        String xml = "<users><user><name>Bob</name></user></users>";
        
        try {
            // 1. DOM 파서 팩토리 및 빌더 생성
            DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
            DocumentBuilder builder = factory.newDocumentBuilder();
            
            // 2. 전체 문서를 메모리에 로드하여 Document 트리 객체로 파싱
            Document doc = builder.parse(new ByteArrayInputStream(xml.getBytes()));
            
            // 3. 트리 탐색 (특정 태그 검색)
            NodeList nameNodes = doc.getElementsByTagName("name");
            System.out.println("추출된 이름: " + nameNodes.item(0).getTextContent()); // Bob
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
### 4. XML 파서 - SAX (Simple API for XML) 기반
- **개념**: 문서를 메모리에 적재하지 않고, 텍스트 스트림을 처음부터 끝까지 순차적으로 읽으면서 이벤트를 발생시키는 방식
- ==대규모 공공 데이터(수십 MB 이상의 XML 파일) 처리 시 메모리 고갈을 방지하기 위한 유일한 대안이자 필수 기술==
- **어떻게?**: `DefaultHandler`를 상속받아 태그의 시작(`startElement`), 텍스트 내용(`characters`), 종료 시점의 콜백 메서드를 구현
```java
import javax.xml.parsers.SAXParser;
import javax.xml.parsers.SAXParserFactory;
import org.xml.sax.Attributes;
import org.xml.sax.helpers.DefaultHandler;
import java.io.ByteArrayInputStream;

public class SaxParserExample {
    public static void main(String[] args) {
        String xml = "<users><user><name>Charlie</name></user></users>";
        
        try {
            SAXParserFactory factory = SAXParserFactory.newInstance();
            // 보안을 위한 외부 엔티티(XXE) 참조 비활성화 (실무 표준)
            // XML 파서가 내 허락 없이 외부(파일, 인터넷 주소)의 데이터를 함부로 가져다 쓰지 못하게 막기
            factory.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);
            SAXParser saxParser = factory.newSAXParser();

            // 1. 이벤트 핸들러 정의 - 아래 관련 내용 참조
            DefaultHandler handler = new DefaultHandler() {
                boolean isName = false;
				
                @Override
                public void startElement(String uri, String localName, String qName, Attributes attributes) {
	                // 
                    if (qName.equalsIgnoreCase("name")) isName = true;
                }

                @Override
                public void characters(char[] ch, int start, int length) {
                    if (isName) {
                        System.out.println("스트림 추출 이름: " + new String(ch, start, length));
                        isName = false;
                    }
                }
            };

            // 2. 스트림을 읽으며 이벤트 발생 (메모리 절약)
            saxParser.parse(new ByteArrayInputStream(xml.getBytes()), handler);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

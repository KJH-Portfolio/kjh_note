# SAX 파서 콜백 메서드 및 파라미터 구조
> [!summary] 핵심 요약
> 
> - **개념**: SAX 파서가 XML 문서 스트림을 읽으며 태그 시작(`startElement`)과 내부 텍스트(`characters`)를 만날 때 호출하는 이벤트 메서드
>     
> - **왜?**: 대용량 XML 전체를 메모리에 올리지 않고, 현재 읽고 있는 조각 데이터의 위치와 상태값만 전달하여 메모리를 극도로 절약하기 위함
>     
> - **언제?**: `DefaultHandler`를 상속받아 특정 태그나 텍스트 데이터만 선별적으로 추출하는 커스텀 로직 구현 시
>     
> - **핵심**: 텍스트를 매번 새 객체로 쪼개지 않고, ==단일 버퍼 배열(`char[]`)의 참조값과 위치(start, length)만 재사용==하는 메모리 최적화 메커니즘
>     
## startElement 파라미터와 태그 식별
- **여는 태그 탐지 콜백**
    - XML 파싱 중 시작 태그(`<tag>`) 도달 시 자동 호출
    - 네임스페이스, 태그 이름, 내부 속성(Attribute) 정보 제공
    - **어떻게?**: 파라미터로 전달된 `qName`과 타겟 태그명을 비교하여 추출 플래그(boolean) 활성화
```xml
<?xml version="1.0" encoding="UTF-8"?>
<root xmlns:api="http://www.easyearth.com/schema/api"> 
    
    <user_info>
        <api:shop id="S001" category="zero-waste">
            <api:name>초록상점</api:name>
            <api:address>경기도 용인시</api:address>
        </api:shop>
    </user_info>
    
</root>
```
```java
@Override
public void startElement(String uri, String localName, String qName, Attributes attributes) {
    // 1. 태그 이름 검증 (qName: 접두사가 포함된 전체 태그명)
    if (qName.equalsIgnoreCase("api:name")) {
        isName = true; // 텍스트 추출을 위한 상태 플래그 켜기
    }
    
    // 2. 필요 시 태그 내부의 속성값(id 등) 추출 가능
    // String idValue = attributes.getValue("id"); 
}
```

|**항목**|**의미**|**역할 및 반환 타입 등 기타 속성**|
|---|---|---|
|**`uri`**|네임스페이스 URI|해당 태그가 속한 네임스페이스의 고유 주소 (설정 없으면 빈 문자열)|
|**`localName`**|지역 이름|네임스페이스 접두사(Prefix)를 제외한 순수 태그 이름|
|**`qName`**|수식 이름|접두사를 포함한 전체 태그 이름 (태그 식별 시 최우선으로 사용)|
|**`attributes`**|속성 정보|시작 태그 내부에 포함된 속성(`id`, `class` 등)들의 목록 데이터|
```text
// [uri] -> "http://www.easyearth.com/schema/api"
// XML 상단의 xmlns:api에 적힌 고유 주소값입니다.

// [localName] -> "shop"
// 접두사(api:)를 떼어낸 순수 태그 이름입니다.

// [qName] -> "api:shop"
// 눈에 보이는 그대로의 전체 이름입니다. (가장 많이 사용됨)

// [attributes] -> "id=S001", "category=zero-waste"
// 태그 안의 속성들이 담긴 객체입니다. 
// attributes.getValue("id")를 하면 "S001"이 반환됩니다.z
```
## characters 파라미터와 텍스트 조립
- **메모리 최적화 텍스트 버퍼**
    - 태그 내부의 순수 텍스트(Text Node) 판독 시 자동 호출
    - 파서가 내부적으로 쥐고 있는 거대한 단일 배열의 참조값만 전달하여 메모리 누수 원천 차단
    - **어떻게?**: 전달된 원본 배열(`ch`)에서 시작점(`start`)부터 길이(`length`)만큼 잘라내어 `new String()`으로 조립
```plaintext
[ SAX Memory Optimization ]

  Buffer (char[]) 
 ┌──────────────────────────────────────────────────────┐
 │ < a p i : n a m e > C h a r l i e < / a p i : n a ...│ <-- 거대한 내부 버퍼 배열 재사용
 └─────────────────────┬─────────────┬──────────────────┘
                       │ length = 7  │                    <-- 읽을 글자 수
                   start = 10        │                    <-- 시작 인덱스
                       │             │
                       ▼             ▼
                new String(ch, start, length)             <-- 필요한 순간에만 String 생성
                       │
 ┌─────────────────────┴─────────────┴──────────────────┐
 │                  "Charlie"                           │ <-- 최종 추출된 문자열 객체
 └──────────────────────────────────────────────────────┘
```
```java
@Override
public void characters(char[] ch, int start, int length) {
    // 1. startElement에서 켜둔 플래그 확인
    if (isName) {
        // 2. 거대한 ch 배열에서 필요한 부분만 잘라내어 String 객체 생성
        String extractedText = new String(ch, start, length);
        System.out.println("스트림 추출 이름: " + extractedText);
        
        // 3. 추출 완료 후 플래그 끄기 (초기화)
        isName = false;
    }
}
```

|**항목**|**의미**|**역할 및 반환 타입 등 기타 속성**|
|---|---|---|
|**`ch`**|문자 배열|파서가 현재 재사용하며 읽고 있는 텍스트 버퍼 (거대한 `char` 배열)|
|**`start`**|시작 인덱스|추출해야 할 텍스트가 배열 내에 위치한 시작점 지표|
|**`length`**|읽을 길이|`start` 위치부터 잘라내어 읽어들일 글자 수|

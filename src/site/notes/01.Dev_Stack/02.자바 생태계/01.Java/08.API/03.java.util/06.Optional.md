# java.util.Optional (Java 8 Null 안정성 제어)
> [!summary] 핵심 요약
> 
> - **개념**: `null`이 될 가능성이 있는 객체를 안전하게 감싸는 래퍼(Wrapper) 컨테이너 클래스
>     
> - **왜?**: 런타임에 발생하는 치명적인 `NullPointerException`(NPE)을 방지하고, 명시적으로 '값이 부재할 수 있음'을 API 반환 타입에 표현하기 위함
>     
> - **언제?**: 메서드의 반환 타입(Return Type)으로 사용하여, 호출자가 빈 결과값에 대한 방어 로직을 누락하지 않도록 강제할 때
>     
> - **핵심**: 조건문(`if(obj != null)`)을 대체하여, 내장 함수형 API를 통한 ==우아하고 안전한 Null 처리 파이프라인== 구축
>     
## Optional 객체의 생성과 사용 원칙
- **생성 방식 (팩토리 메서드)**
    - `Optional.empty()`: 값이 없는 빈(Empty) 객체 생성
    - `Optional.of(value)`: 전달된 값이 절대 `null`이 아님을 보장할 때 사용 (만약 `null`이 입력되면 즉시 NPE 발생)
    - `Optional.ofNullable(value)`: 전달된 값이 `null`일 수도, 아닐 수도 있을 때 사용 (가장 빈번하게 활용됨)
- **실무 적용 시 제약 및 경고 사항**
    - Optional은 반환값(Return) 목적으로만 설계된 객체이므로 생성 비용과 직렬화(Serialization) 미지원 문제가 존재함
    - 따라서 ==필드, 생성자, 메서드 파라미터, 컬렉션의 원소로 사용하는 것을 엄격히 금지==하며, 오직 메서드의 반환 타입으로만 사용해야 함
```java
public class OptionalCreation {
    public Optional<Member> findMember(String id) {
        // 1. null이 절대 아닐 때 
        Optional<String> name = Optional.of("홍길동");

        // 2. null일 가능성이 있을 때 (DB 조회 결과 등)
        Member member = repository.findById(id);
        
        // member가 null이면 내부적으로 Optional.empty()를 반환함
        return Optional.ofNullable(member); 
    }
}
```
## 안전한 값 추출과 내장 기능 (Extraction)
- **Optional 내부 값 접근 로직**
    - Optional 객체에서 실제 값을 꺼낼 때는 `get()` 사용을 지양하고, 값이 없을 상황을 대비한 안전한 대체 메서드를 활용함
- **orElse와 orElseGet의 성능 차이 인지**
    - `orElse`는 내부 값이 존재하더라도 괄호 안의 매개변수가 무조건 평가(실행)되므로, 리소스를 소모하는 객체 생성 로직에는 적합하지 않음
    - 비용이 큰 객체 생성이나 DB 연산이 필요할 경우 ==값이 비어있을 때만 지연 실행(Lazy Evaluation)되는 orElseGet을 사용==해야 함

| **추출 메서드**            | **동작 원리 및 역할**          | **예외 및 특이사항**                                              |
| --------------------- | ----------------------- | ---------------------------------------------------------- |
| `get()`               | 컨테이너 내부의 실제 값을 꺼냄       | 값이 없으면 `NoSuchElementException` 예외 발생 (사용 지양)              |
| `orElse(T)`           | T반환 or 전달된 '기본값' 반환     | 값이 있든 없든 내부 로직이 무조건 실행되어 평가됨                               |
| `orElseGet(Supplier)` | T반환 or 전달된 '함수'를 실행해 반환 | 값이 비어있을 때만 함수가 실행됨 ==(성능 최적화 유리)==                         |
| `orElseThrow()`       | T반환 or 특정 예외를 발생시킴      | Spring 등에서 커스텀 예외(e.g., `UserNotFoundException`) 처리에 주로 활용 |
| `ifPresent(Consumer)` | 값이 존재할 때만 특정 로직(콜백) 실행  | 반환값이 없으며(`void`), 출력이나 객체의 상태 변경에 주로 사용                    |

```java
public class OptionalExtraction {
    public void processMember(Optional<Member> optionalMember) {
        // [안티 패턴] 구시대적 Null 체크와 다를 바 없는 접근 방식
        if (optionalMember.isPresent()) {
            System.out.println(optionalMember.get().getName());
        }

        // [권장 패턴 1] 값이 있을 때만 안전하게 실행 (람다/메서드 참조 활용)
        optionalMember.ifPresent(m -> System.out.println(m.getName()));

        // [권장 패턴 2] 값이 없을 경우 예외 던지기 (Fail-fast)
        Member validMember = optionalMember.orElseThrow(() -> 
                                    new IllegalArgumentException("회원 정보를 찾을 수 없습니다."));

        // [자주 헷갈리는 차이점] orElse vs orElseGet
        // 주의: optionalMember에 값이 있어도 createNewMember() 메서드가 호출됨
        Member member1 = optionalMember.orElse(createNewMember());
        
        // 정상: optionalMember가 비어있을 때만 createNewMember() 메서드가 호출됨
        Member member2 = optionalMember.orElseGet(() -> createNewMember());
    }
}
```
## Optional 파이프라인과 데이터 가공 (Chaining)
- **컬렉션 Stream과 유사한 변환 연산**
    - `filter()`, `map()`, `flatMap()` 등의 메서드를 연쇄적으로 호출하여, `null` 여부를 검사하는 `if`문 없이도 데이터 가공 및 필터링 작업을 간결하게 처리함
    - 파이프라인 중간 단계에서 값이 없어지면(Null 도출), 이후의 모든 연산은 무시되고 안전하게 하위로 통과됨
```plaintext
[ Optional 파이프라인 연산 흐름 (Pipeline Flow) ]

 [초기 상태] Optional<String> (" hello ")
                 │
                 ▼
 1. map(String::trim) ─────────────▶ 공백 제거 로직 적용 ("hello")
                 │                   (※ 만약 비어있었다면 연산 무시하고 그대로 통과)
                 ▼
 2. filter(s -> s.length() > 5) ───▶ 조건 불만족 (길이가 5 이하임)
                 │                   (※ 조건을 만족하지 못해 빈 상태(empty)로 전환)
                 ▼
 3. orElse("기본값") ──────────────▶ 최종 상태가 비어있으므로 대체 텍스트 반환
                 │
                 ▼
           [최종 반환: "기본값"]
```

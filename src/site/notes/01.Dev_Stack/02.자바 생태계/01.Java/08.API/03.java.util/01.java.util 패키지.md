# 📘 java.util 주요 유틸리티 클래스 정리
> [!summary] 핵심 요약
> 
> - **개념**: 자바 프로그래밍에 필요한 자료구조, 입력 처리, 정렬 등 필수 도구를 모아둔 표준 패키지
>     
> - **왜?**: 직접 구현하기 복잡한 알고리즘(정렬, 검색)과 자료구조(리스트, 맵)를 표준 API로 제공하여 개발 생산성 극대화
>     
> - **언제?**: 데이터 그룹화(컬렉션), 배열 조작, 사용자 콘솔 입력 처리, 난수 생성 등 애플리케이션 개발 전반
>     
> - **핵심**: 목적에 맞는 `Collection` 자료구조 선택, `Stream API`를 활용한 데이터 제어, 그리고 스레드 안전성을 고려한 클래스 선별 사용
>     
## 핵심 자료구조 (Collections Framework)
- **컬렉션 프레임워크 (Collections Framework)**
    - 데이터를 효율적으로 저장, 삭제, 검색하기 위한 표준 인터페이스와 구현체 모음
    - **List 계열 (`ArrayList` 등)**: 입력 순서 유지, 데이터 중복 허용
    - **Set 계열 (`HashSet` 등)**: 순서 무관, 데이터 중복 불가
    - **Map 계열 (`HashMap` 등)**: Key-Value 쌍으로 저장, Key 중복 불가
    - **어떻게?**: 인터페이스 타입으로 선언하고 구체 클래스로 인스턴스화하여 다형성 확보
- **데이터 파이프라인 (`Stream API` - Java 8+)**
    - 컬렉션 데이터를 `for`문 없이 선언적으로 제어하기 위한 모던 API
    - 필터링, 매핑, 정렬 등의 복잡한 로직을 메서드 체이닝으로 간결하게 처리
    - ==스트림 처리는 원본 컬렉션 데이터를 절대 변경하지 않음 (불변성 유지)==
    - **어떻게?**: `컬렉션.stream()` 호출 후 중간 연산과 최종 연산을 연결하여 구현
```java
// 1. List 생성 (자료구조)
List<String> userList = new ArrayList<>();
userList.add("Alice");
userList.add("Bob");
userList.add("Charlie");
userList.add("Alex");

// 2. Stream API를 활용한 데이터 선언적 처리
// 'A'로 시작하는 이름만 필터링하여 대문자로 변환 후 새로운 리스트로 반환
List<String> filteredUsers = userList.stream()
		.filter(name -> name.startsWith("A")) // 중간 연산 (조건)
		.map(String::toUpperCase)             // 중간 연산 (가공)
		.toList();                            // 최종 연산 (수집) : 수정 불가ver
		//.collect(Collectors.toList())       // 최종 연산 (수집) : 수정 가능ver
		
System.out.println("필터링된 결과: " + filteredUsers); // [ALICE, ALEX]
System.out.println("원본 리스트 보존 확인: " + userList); // [Alice, Bob, Charlie, Alex]
```
```plaintext
[ Stream API 처리 흐름도 ]

데이터 소스 (List, Set)
       │
       ▼
스트림 생성 (.stream())
       │
       ▼
중간 연산 (.filter(), .map() 등) ──▶ 조건 필터링 및 데이터 가공 (지연 평가)
       │
       ▼
최종 연산 (.collect(), .toList()) ──▶ 결과물 반환 (새로운 컬렉션 생성)
```
## 배열 및 컬렉션 제어 (Arrays & Collections)
- **배열 유틸리티 (`Arrays`)**
    - 정적 배열(`int[]`, `String[]` 등)의 정렬(`sort`), 검색(`binarySearch`), 복사, 문자열 변환 기능 제공
    - ==`binarySearch()`는 반드시 정렬(`sort`)이 선행된 배열에서만 정상 동작함==
    - **어떻게?**: `Arrays.메서드명(배열)` 형태의 정적(Static) 호출
- **컬렉션 유틸리티 (`Collections`)**
    - 동적 자료구조(`List`, `Set`, `Map` 등)를 조작하기 위한 정적 메서드 제공
    - 요소 뒤집기(`reverse`), 섞기(`shuffle`), 동기화 처리, 불변(Immutable) 객체 생성 등 지원
```plaintext
[ 유틸리티 클래스 조작 대상 비교 ]

      ┌─ 정적 배열 (고정 크기, 기본형/참조형)
      │  └─ 대상: int[], String[] 등
      ▼
Arrays 클래스 ──(sort, binarySearch 등)──▶ 결과 반환

      ┌─ 동적 자료구조 (가변 크기, 참조형 전용)
      │  └─ 대상: ArrayList, HashSet 등
      ▼
Collections 클래스 ──(reverse, shuffle 등)──▶ 원본 상태 변경
```
## 날짜와 시간 처리 (Legacy vs Modern)
- **구형 날짜 클래스 (`java.util.Date`, `Calendar`)**
    - 자바 초기 버전부터 제공된 기본 날짜 및 시간 제어 클래스
    - 내부 상태가 변경 가능한(Mutable) 구조로 설계됨
    - ==멀티스레드 환경에서 동시 접근 시 데이터 오염(Thread-unsafe) 위험이 치명적임==
- **모던 날짜 API (`java.time` 패키지 - Java 8+)**
    - 구형 클래스의 결함을 해결하기 위해 도입된 새로운 표준
    - `LocalDateTime`, `LocalDate` 등은 생성 후 상태가 변하지 않는 불변(Immutable) 객체로 설계됨
    - **어떻게?**: 신규 자바 프로젝트에서는 `java.util.Date` 대신 반드시 `java.time` 패키지를 기본으로 사용
```java
// [지양] 구형 Date 방식 (가변 객체, 스레드 불안정)
Date legacyDate = new Date();
legacyDate.setYear(126); // 현재 연도 계산 등의 API가 직관적이지 않음 (Deprecated)

// [권장] 모던 LocalDateTime 방식 (불변 객체, 스레드 안전)
LocalDateTime now = LocalDateTime.now();
LocalDateTime nextWeek = now.plusDays(7); // 원본(now)은 유지되고 새로운 객체 반환

DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
System.out.println("일주일 뒤: " + nextWeek.format(formatter));
```
## 기타 필수 유틸리티 (입력 및 난수)
- **입력 처리 (`Scanner`)**
    - 콘솔, 파일, 문자열 등 다양한 입력 소스에서 데이터를 읽고 타입별로 파싱
    - ==메모리 누수 방지를 위해 사용 후 반드시 `close()` 처리 (try-with-resources 권장)==
- **난수 생성 (`ThreadLocalRandom`)**
    - 멀티스레드 환경에 최적화된 고성능 난수 생성기
    - 기존 `java.util.Random` 클래스가 갖는 스레드 경합(Contention) 병목 현상 해결
    - **어떻게?**: 인스턴스 생성 없이 `ThreadLocalRandom.current().nextInt(min, max)` 호출
```java
// 1. Scanner: try-with-resources 구문을 통한 자동 자원 반납
try (Scanner scanner = new Scanner(System.in)) {
	System.out.print("나이를 입력하세요: ");
	int age = scanner.nextInt(); 
	// (실행 시 사용자 입력 대기)
} 

// 2. ThreadLocalRandom: 경계값을 지정한 난수 생성
// 시작값(inclusive)부터 끝값(exclusive) 직전까지의 난수 발생
int randomNum = ThreadLocalRandom.current().nextInt(1, 101); // 1 ~ 100 반환
System.out.println("생성된 난수: " + randomNum); 
```
## 주요 클래스 요약

| **클래스/항목**                  | **용도**         | **특징 및 역할**                              |
| --------------------------- | -------------- | ---------------------------------------- |
| **`HashMap` / `ArrayList`** | 자료구조           | 가장 빈번하게 사용되는 Map, List 인터페이스 구현체         |
| **`Stream`**                | 데이터 파이프라인      | 컬렉션 데이터를 선언적으로 필터링/가공하는 불변성 보장 API       |
| **`Date` / `Calendar`**     | 날짜 제어 (Legacy) | 구형 날짜 클래스 (멀티스레드 환경 불완전, 사용 지양)          |
| **`Arrays`**                | 배열 유틸리티        | 정적 배열의 정렬, 탐색, 비교, 복사를 돕는 정적 메서드 제공      |
| **`Collections`**           | 컬렉션 유틸리티       | List, Set 등 동적 자료구조의 정렬, 뒤집기, 스레드 안전화 지원 |
| **`Scanner`**               | 입력 처리          | 콘솔, 파일 등에서 데이터를 타입별로 읽어오는 파서 역할          |
| **`Formatter`**             | 출력 포맷팅         | printf 스타일의 형식화된 문자열 생성 지원               |
| **`ThreadLocalRandom`**     | 난수 생성          | 멀티스레드 환경에서 안전하고 빠르게 난수를 생성               |
| **`Concurrent` (패키지)**      | 병렬 프로그래밍       | 멀티스레드 환경을 위한 고성능 자료구조 및 동기화 도구 포함        |
| **`Optional`**              | Null 방지        | Null 포인터 예외를 방지하기 위한 래퍼 클래스              |

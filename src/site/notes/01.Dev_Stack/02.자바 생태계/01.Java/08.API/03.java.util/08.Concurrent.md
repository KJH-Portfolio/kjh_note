# Java Concurrent API (`java.util.concurrent`)
> [!summary] 핵심 요약
> 
> - **개념**: 멀티스레드 환경에서 안전하고 효율적인 병렬 처리를 지원하는 자바의 표준 동시성 유틸리티 패키지
>     
> - **왜?**: 기존 `Thread` 직접 생성 방식의 리소스 낭비를 막고, `synchronized` 키워드로 인한 병목 현상(데드락, 성능 저하)을 해결하기 위함
>     
> - **언제?**: 대용량 트래픽 처리, 다중 I/O 비동기 작업, 스레드 안전(Thread-Safe)한 데이터 공유가 필요한 백엔드 실무 환경
>     
> - **핵심**: 스레드 생명주기를 관리하는 **Executor 프레임워크**와 락(Lock) 경합을 최소화한 **동시성 컬렉션**의 적절한 선택 및 활용
>     
## Executor 프레임워크 (스레드 풀 관리)
- **스레드 생성 및 관리의 추상화**
    - 스레드 생성 비용(Context Switching 및 메모리 할당)을 줄이기 위해 **스레드 풀(Thread Pool)**을 미리 생성하여 재사용
    - 작업(Task)의 제출(Submit)과 실행(Execution)을 완벽하게 분리
    - ==스레드 개수를 제한하여 시스템 자원 고갈을 방지하는 핵심 안전장치 역할 수행==
    - **어떻게?**: `Executors` 팩토리 클래스를 통해 목적에 맞는 `ExecutorService`를 생성하고 `submit()`으로 작업을 위임
```plaintext
[ ExecutorService Flow ]

  Task (Runnable/Callable)
         │
         ▼
 ┌──────────────┐      ┌──────────────┐
 │  Task Queue  │ ───▶ │  ThreadPool  │ ──▶ Execution (Result)
 └──────────────┘      └──────────────┘
  (작업 대기열)          (미리 생성된 스레드 묶음)
```
```java
public class ExecutorExample {
    public static void main(String[] args) {
        // 1. 고정된 크기(스레드 5개)의 스레드 풀 생성
        ExecutorService executor = Executors.newFixedThreadPool(5);

        for (int i = 0; i < 10; i++) {
            final int taskId = i;
            // 2. 작업 제출 (비동기 실행)
            executor.submit(() -> {
                String threadName = Thread.currentThread().getName();
                System.out.println("작업 실행 중: " + taskId + " by " + threadName);
            });
        }

        // 3. 스레드 풀 종료 (새로운 작업 수용 거부 및 기존 작업 완료 후 종료)
        executor.shutdown();
        
        try {
            // 최대 1시간 동안 모든 작업이 완료되기를 대기
            if (!executor.awaitTermination(1, TimeUnit.HOURS)) {
                executor.shutdownNow(); // 시간 초과 시 강제 종료
            }
        } catch (InterruptedException e) {
            executor.shutdownNow();
        }
    }
}
```
## Concurrent Collections (동시성 컬렉션)
- **스레드 안전성(Thread-Safety)과 성능의 양립**
    - `Collections.synchronizedXXX()` 방식의 전역 락(Global Lock) 한계를 극복
    - **CAS(Compare-And-Swap)** 알고리즘 및 부분 락(Lock Stripping) 기법을 사용하여 ==여러 스레드가 동시에 접근해도 병목을 최소화함==
    - **어떻게?**: 멀티스레드 환경에서 공유되는 데이터 구조는 반드시 `java.util.concurrent` 하위의 전용 컬렉션으로 선언하여 사용

| **클래스명**               | **동작 원리 및 역할**                                                              | **반환 타입 등 기타 속성**     |
| ---------------------- | --------------------------------------------------------------------------- | --------------------- |
| `ConcurrentHashMap`    | Map을 구역별로 나눠서(Lock Striping) 효율을 높이고,<br>읽기 작업은 방해 없이(Lock-free) 수행         | `ConcurrentMap<K, V>` |
| `CopyOnWriteArrayList` | 수정 작업 발생 시 내부 배열을 복사(Copy)하여 새로 구성하는 List. <br>(==읽기 작업==이 압도적으로 많은 환경에 적합) | `List<E>`             |
| `BlockingQueue`        | 큐가 비어있으면 데이터가 들어올 때까지, 꽉 차면 빈자리가 날 때까지 스레드를 대기(Block)시킴                     | `Queue<E>`            |

## 동기화 보조 도구 (Synchronizers & Locks)
- **스레드 간의 실행 순서 제어 및 세밀한 락(Lock) 관리**
    - `Object.wait()`, `notify()`의 복잡하고 위험한 수동 동기화를 대체하는 고수준 API 제공
    - `java.util.concurrent.locks` 및 `atomic` 패키지와 연계하여 정밀한 제어 가능
    - **어떻게?**: 특정 이벤트 발생을 기다리거나, 자원 접근 인원을 제한할 때 목적에 맞는 클래스 객체를 생성하여 스레드 간 공유

| **클래스명**         | **동작 원리 및 역할**                                                    | **반환 타입 등 기타 속성**                 |
| ---------------- | ----------------------------------------------------------------- | --------------------------------- |
| `CountDownLatch` | 정해진 횟수만큼 카운트가 차감될 때까지 다른 스레드의 실행을 대기시킴 (일회용)                      | `void` (`await()`, `countDown()`) |
| `CyclicBarrier`  | 지정된 수의 스레드가 장벽(Barrier)에 도달할 때까지 모두 대기시켰다가 동시에 재개 (재사용 가능)        | `int` (`await()`)                 |
| `Semaphore`      | 동시에 접근 가능한 스레드의 개수(Permit)를 제한하여 리소스 과부하 방지                       | `void` (`acquire()`, `release()`) |
| `ReentrantLock`  | 기본 `synchronized` 블록을 대체하며, 공정성(Fairness) 설정 및 타임아웃 처리가 가능한 명시적 락 | `Lock` 구현체                        |
#### 직관적인 기능 비유

| **클래스명**           |                                                    |
| ------------------ | -------------------------------------------------- |
| **CountDownLatch** | **"남들 다 끝났니? 다 됐으면 나(Main) 출발한다."** (종료 대기/일회용)    |
| **CyclicBarrier**  | **"우리 다 모였니? 다 왔으면 우리 같이 다음 단계 가자."** (동기 맞추기/재사용) |
| **Semaphore**      | **"남은 자리 있니? 정원 초과니까 나갈 때까지 기다려."** (자원 개수 제한)     |
| **ReentrantLock**  | **"내가 열쇠 가졌어! 다 쓸 때까지 순서대로 대기해."** (똑똑한 수동 자물쇠)    |
- 아래 예시에서 IntStram.range().parallel()을 사용해도 좋음
#### ① CountDownLatch (설정 로딩 기다리기)
```java
// 3개의 설정 파일이 모두 읽혀야 메인 서버가 시작됨
CountDownLatch latch = new CountDownLatch(3);

IntStream.range(0, 3).forEach(i -> {
    new Thread(() -> {
        System.out.println("설정 파일 " + i + " 로딩 중...");
        latch.countDown(); // 카운트 1 감소
    }).start();
});

latch.await(); // 0이 될 때까지 메인 스레드는 여기서 대기
System.out.println("모든 설정 완료! 서버를 시작합니다.");
```
#### ② CyclicBarrier (단계별 공동 작업)
```java
// 3명이 모두 모여야 '점심 식사' 단계로 넘어감
CyclicBarrier barrier = new CyclicBarrier(3, () -> System.out.println("== 모두 모였다! 식사 시작 =="));

IntStream.range(0, 3).forEach(i -> {
    new Thread(() -> {
        try {
            System.out.println("스레드 " + i + ": 등산 중...");
            barrier.await(); // 3명이 올 때까지 여기서 대기 (다 모이면 위 람다식 실행)
            System.out.println("스레드 " + i + ": 밥 먹는 중!");
        } catch (Exception e) { e.printStackTrace(); }
    }).start();
});
```
#### ③ Semaphore (DB 연결 개수 제한)
```java
// 동시에 2명만 접속 가능한 DB (허가권 2개)
Semaphore semaphore = new Semaphore(2);

IntStream.range(0, 5).forEach(i -> {
    new Thread(() -> {
        try {
            semaphore.acquire(); // 허가권 획득 (없으면 대기)
            System.out.println("스레드 " + i + ": DB 사용 중...");
            Thread.sleep(1000); 
            System.out.println("스레드 " + i + ": DB 반납.");
            semaphore.release(); // 허가권 반납
        } catch (Exception e) { e.printStackTrace(); }
    }).start();
});

//해당 작업은 순서를 보장할 수 없음
```
#### ④ ReentrantLock (공정한 입금 처리)
```java
ReentrantLock lock = new ReentrantLock(true); // true 설정 시 먼저 온 사람부터 (공정성)

IntStream.range(0, 3).forEach(i -> {
    new Thread(() -> {
        lock.lock(); // 자물쇠 잠금
        try {
            System.out.println("스레드 " + i + ": 금고 열고 입금 중...");
        } finally {
            lock.unlock(); // 반드시 해제 (안 하면 무한 대기 발생!)
            System.out.println("스레드 " + i + ": 작업 완료.");
        }
    }).start();
});

// 해당 작업도 n번 스레드가 먼저 작업할 것이라는 보장은 없음
// 다만, 먼저 작업 요청을 넣은 스레드 순서대로 작업하는것은 보장됨 
```

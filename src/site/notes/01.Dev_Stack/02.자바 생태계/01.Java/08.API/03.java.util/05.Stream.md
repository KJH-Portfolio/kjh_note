# Java Stream (선언적 데이터 처리 API)
> [!summary] 핵심 요약
> 
> - **개념**: 배열, 컬렉션 등의 데이터 소스를 함수형 프로그래밍 방식으로 처리하는 파이프라인
>     
> - **왜?**: `for` 루프와 `if` 조건문을 없애고 메서드 체이닝을 통해 코드를 간결하게 작성하며, 원본 데이터를 훼손하지 않는 안전한 처리를 위함
>     
> - **언제?**: 대량의 컬렉션 데이터를 필터링, 변환, 집계하거나 멀티코어를 활용한 병렬 처리가 필요할 때 주로 사용
>     
> - **핵심**: 지연 연산(Lazy Evaluation)의 원리를 이해하고 중간 연산과 최종 연산의 역할을 명확히 구분하여 파이프라인을 설계
>     
## 스트림 파이프라인 동작 원리
- **데이터 처리 흐름 및 특성**
    - `생성(Source)` -> `가공(Intermediate)` -> `결과 도출(Terminal)`의 3단계 파이프라인으로 구성
    - **지연 연산(Lazy Evaluation)**: 최종 연산이 호출되기 전까지는 중간 연산이 실제로 실행되지 않아 연산 횟수를 최적화함
    - **내부 반복(Internal Iteration)**: 개발자가 직접 반복을 제어하는 외부 반복(`for`문)과 달리, 스트림이 내부적으로 요소를 순회하며 처리함
    - ==최종 연산이 완료된 스트림은 닫히므로 절대 재사용 불가 (일회성)==
    - **어떻게?**: 데이터 소스에서 `.stream()`을 호출하여 파이프라인을 열고 체이닝 시작
```plaintext
[ Stream Pipeline Flow ]

  Data Source (List, Array)
       │
       ▼ .stream() 
 ┌────────────┐
 │ Inter-Op   │  <-- 중간 연산: filter(), map() 등 (지연 실행)
 └────┬───────┘
       │ New Stream
       ▼
 ┌────────────┐
 │ Final-Op   │  <-- 최종 연산: collect(), count() 등 (즉시 실행 후 닫힘)
 └────────────┘
       │
       ▼
     Result
```
## 주요 스트림 연산 메서드 (Operations)
- **중간 연산과 최종 연산의 조합**
    - 중간 연산은 처리가 끝난 후 또 다른 `Stream`을 반환하여 연속적인 체이닝을 허용함
    - 최종 연산은 파이프라인을 닫고 단일 값(숫자, 컬렉션, `void` 등)을 반환함
    - **어떻게?**: 데이터 가공 목적에 맞는 중간 연산을 여러 개 나열한 뒤, 원하는 결과물 형태에 맞춰 최종 연산을 한 번 호출

|**메서드**|**구분**|**동작 원리 및 역할**|**반환 타입 등 기타 속성**|
|---|---|---|---|
|`filter()`|중간|조건(Predicate)에 참인 요소만 추출|새로운 `Stream`|
|`map()`|중간|요소들을 지정된 형태(Function)로 변환|새로운 `Stream`|
|`distinct()`|중간|스트림 내 중복 요소 제거|새로운 `Stream`|
|`sorted()`|중간|지정된 기준이나 기본 오름차순으로 요소 정렬|새로운 `Stream`|
|`collect()`|최종|요소들을 리스트, 맵 등의 컬렉션으로 수집|`List`, `Set`, `Map` 등|
|`forEach()`|최종|각 요소를 순회하며 지정된 작업 수행 (출력 등)|`void`|
|`count()`|최종|파이프라인을 통과한 최종 요소의 총 개수 반환|`long`|
|`toArray()`|최종|스트림의 요소를 배열 형태로 변환|`Object[]` 등|

```java
List<String> names = Arrays.asList("alice", "bob", "Charlie", "alice", "david", "BOB");

// 파이프라인 구성 및 실행 (중간 연산 -> 최종 연산)
List<String> processedNames = names.stream()
		.map(String::toUpperCase)          // 1. 대문자 변환: ALICE, BOB, CHARLIE, ALICE, DAVID, BOB
		.filter(n -> n.length() > 3)       // 2. 3글자 초과 필터: ALICE, CHARLIE, ALICE, DAVID
		.distinct()                        // 3. 중복 제거: ALICE, CHARLIE, DAVID
		.sorted()                          // 4. 정렬 (알파벳 순)
		.collect(Collectors.toList());     // 5. 최종 리스트로 묶기 (스트림 닫힘)

System.out.println(processedNames); // [ALICE, CHARLIE, DAVID]

// 주의: 여기서 names.stream()을 다시 열지 않고 기존 스트림 변수를 재사용하면 IllegalStateException 발생
```
## 스트림 사용 시 주의사항 및 최적화
- **성능 및 안정성 관리**
    - **오토박싱 방지**: `int`, `long` 등 기본형 데이터를 다룰 때는 범용 `Stream<T>` 대신 `IntStream`, `LongStream`을 사용하여 메모리 및 성능 낭비 방지
    - **부작용(Side Effect) 금지**: ==스트림 파이프라인 연산 내부에서 외부 변수나 원본 데이터를 직접 수정하지 말 것== (스레드 세이프티 위반 및 불변성 훼손)
    - **병렬 처리 오용 주의**: 데이터 크기가 작거나 로직이 단순할 때 `.parallel()`을 남용하면 스레드 생성 오버헤드로 인해 순차 처리보다 성능이 저하될 수 있음

## 📘Stream

## Summary 📝

> [!summary] Summary 📝
> 
> - 스트림(`Stream`)은 Java 8에서 도입된 **함수형 프로그래밍**을 지원하는 API로, 배열이나 컬렉션 등의 데이터 소스를 처리하는 **선언적(Declarative)** 방법을 제공함.
>     
> - 데이터 소스의 요소를 **파이프라인**을 통해 처리하며, **불변성**을 지향하고 **내부 반복**을 사용하여 코드의 **간결성**과 **병렬 처리**의 용이성을 높임.
>     
> - **중간 연산(Intermediate Operation)**과 **최종 연산(Terminal Operation)**으로 구성됨.
>     

---

## 목적 🎯

- **코드 간결성 확보:** 반복문(`for` 루프)과 조건문(`if`)을 제거하고 **메서드 체인**을 통해 코드를 선언적으로 작성하여 가독성을 향상.
    
- **병렬 처리 용이성:** **`parallel()`** 메서드 호출만으로 멀티스레드 기반의 병렬 처리를 쉽게 구현하여 대용량 데이터 처리 성능을 향상.
    
- **데이터 불변성 유지:** 원본 데이터 소스를 직접 변경하지 않고 **새로운 결과**를 생성하여 함수의 부작용(Side Effect)을 최소화.

## 핵심 개념 💡

### ✅ 스트림 파이프라인 구성

스트림은 데이터 소스, 0개 이상의 중간 연산, 그리고 단 하나의 최종 연산으로 구성된 **파이프라인**을 통해 데이터를 처리합니다.

|**구분**|**역할**|**예시 메서드**|**특징**|
|---|---|---|---|
|**데이터 소스**|스트림이 처리할 원본 데이터|배열, 컬렉션, 파일 I/O 등|`Arrays.stream(arr)`, `list.stream()`|
|**중간 연산**|스트림을 변환하거나 필터링하여 새로운 스트림을 반환|`filter()`, `map()`, `sorted()`, `distinct()`, `limit()`|**지연 연산(Lazy)**: 최종 연산이 호출될 때까지 실행되지 않음.|
|**최종 연산**|스트림 파이프라인을 종료하고 결과를 반환|`collect()`, `sum()`, `forEach()`, `count()`, `toArray()`, `reduce()`|**즉시 실행:** 최종 연산이 호출되는 순간 모든 중간 연산이 실행됨.|

---

### ✅ 자주 사용하는 메서드 요약

| **메서드**                  | **구분** | **기능**                 | **예시**                          |
| ------------------------ | ------ | ---------------------- | ------------------------------- |
| **`filter(Predicate)`**  | 중간     | 조건에 맞는 요소만 남김.         | `.filter(n -> n > 5)`           |
| **`map(Function)`**      | 중간     | 요소를 다른 형태로 변환.         | `.map(s -> s.toUpperCase())`    |
| **`distinct()`**         | 중간     | 스트림 내 중복 요소를 제거.       | `.distinct()`                   |
| **`sorted()`**           | 중간     | 요소를 정렬.                | `.sorted()`                     |
| **`collect(Collector)`** | 최종     | 스트림 요소를 리스트, 맵 등으로 수집. | `.collect(Collectors.toList())` |
| **`forEach(Consumer)`**  | 최종     | 각 요소에 대해 지정된 작업을 수행.   | `.forEach(System.out::println)` |
| **`count()`**            | 최종     | 스트림 내 요소의 개수를 반환.      | `.count()`                      |
| **`toArray()`**          | 최종     | 스트림의 요소를 배열로 반환.       | `.toArray(String[]::new)`       |

## 특징 ✨

### ✅ 지연 연산 (Lazy Evaluation)

- **효율성:** 중간 연산은 파이프라인이 모두 구축된 후 최종 연산이 필요로 하는 시점에 한 번만 실행되어, 불필요한 연산을 줄여 효율적임.
    
- **쇼트 서킷(Short-circuiting):** `limit()` 같은 연산으로 일부 요소만 처리하고 전체 탐색을 중단할 수 있음.
    

### ✅ 일회용 (Consumable)

- 스트림은 **한 번 사용**하면 재사용할 수 없음. 최종 연산이 호출된 스트림은 닫히므로, 다시 사용하려면 데이터 소스로부터 **새로운 스트림**을 생성해야 함.
    

### ✅ 내부 반복 (Internal Iteration)

- 개발자가 직접 `for` 루프를 작성하는 **외부 반복**과 달리, 스트림은 내부적으로 최적화된 방법으로 요소를 반복 처리하는 **내부 반복**을 사용함.
    

---

## 팁과 주의점 ⚠️

- **기본형 스트림 사용:** `int`, `long`, `double` 타입의 배열을 처리할 때는 **`IntStream`**, **`LongStream`**, **`DoubleStream`**과 같은 기본형 특화 스트림을 사용하면 **오토박싱(Autoboxing)**으로 인한 성능 저하를 방지할 수 있음.
    
- **Side Effect 피하기:** 스트림 연산 내에서 원본 데이터 소스의 요소를 직접 변경하는 행위(Side Effect)는 예상치 못한 결과를 초래할 수 있으므로 피해야 함.
    
- **`parallel()` 오용 금지:** 병렬 처리가 항상 빠른 것은 아님. 데이터 크기가 작거나, 중간 연산이 복잡하지 않을 때는 오히려 스레드 생성 오버헤드 때문에 성능이 저하될 수 있음.

## 예시 📌

### ✅ 자주 사용하는 메서드를 이용한 데이터 처리 사례

```java
import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;

public class StreamMethodsInOne {

    public static void main(String[] args) {
        
        // 데이터 소스: 중복된 요소와 정렬되지 않은 문자열 리스트
        List<String> rawNames = Arrays.asList(
            "alice", "bob", "Charlie", "alice", "david", "BOB"
        );
        
        System.out.println("--- 1. 데이터 소스 ---");
        System.out.println("원본: " + rawNames);
        
        // ------------------------------------------------------------------
        // 파이프라인 구성: 중간 연산 (Filter, Map, Distinct, Sorted)
        // ------------------------------------------------------------------
        
        List<String> processedList = rawNames.stream()
            
            // 1. map(Function): 모든 요소를 대문자로 변환
            .map(String::toUpperCase) // alice -> ALICE, bob -> BOB, Charlie -> CHARLIE, ...
            
            // 2. filter(Predicate): 길이가 3보다 긴 요소만 필터링
            .filter(name -> name.length() > 3) // BOB 제외 (ALICE, CHARLIE, DAVID, ALICE)
            
            // 3. distinct(): 중복 요소 제거
            .distinct() // (ALICE, CHARLIE, DAVID)
            
            // 4. sorted(): 알파벳 순으로 정렬
            .sorted() // (ALICE, CHARLIE, DAVID)
            
            // 5. collect(Collector): 최종 결과를 List<String>으로 수집 (최종 연산)
            .collect(Collectors.toList());
            
        System.out.println("\n--- 2. 최종 리스트 (Collect) ---");
        System.out.println("결과 리스트: " + processedList); 
        // 결과: [ALICE, CHARLIE, DAVID]
        
        // ------------------------------------------------------------------
        // 추가 최종 연산 (Count, ToArray, ForEach)
        // ------------------------------------------------------------------
        
        // 새로운 스트림 생성 및 최종 연산 적용
        System.out.println("\n--- 3. 추가 최종 연산 ---");
        
        long count = rawNames.stream()
            .map(String::toUpperCase)
            .distinct()
            .count(); // 6. count(): 중복 제거된 요소의 개수 세기
        
        System.out.println("원본에서 중복 제거된 요소 개수 (count): " + count); // 4 (ALICE, BOB, CHARLIE, DAVID)
        
        // 7. toArray(): 최종 결과를 배열로 반환
        Object[] nameArray = rawNames.stream()
            .map(String::toUpperCase)
            .toArray();
            
        System.out.println("전체 요소 배열 (toArray): " + Arrays.toString(nameArray));
        // 결과: [ALICE, BOB, CHARLIE, ALICE, DAVID, BOB]
        
        // 8. forEach(Consumer): 각 요소에 대해 작업 수행
        System.out.println("\n각 요소 출력 (forEach):");
        rawNames.stream()
            .map(String::toUpperCase)
            .forEach(name -> System.out.print(name + " | ")); // 각 요소를 대문자로 변환 후 출력
        // 결과: ALICE | BOB | CHARLIE | ALICE | DAVID | BOB | 
    }
}
```

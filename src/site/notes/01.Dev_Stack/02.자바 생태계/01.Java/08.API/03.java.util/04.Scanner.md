# Java Scanner (입력 스트림 파싱 유틸리티)
> [!summary] 핵심 요약
> 
> - **개념**: 키보드 입력, 파일, 문자열 등으로부터 데이터를 읽어 원하는 타입(정수, 실수, 문자열 등)으로 변환해주는 유틸리티 클래스
>     
> - **왜?**: 순수 바이트 단위의 입력 스트림(`System.in`)을 개발자가 다루기 쉬운 기본 데이터 타입으로 자동 파싱(Parsing)하기 위함
>     
> - **언제?**: 콘솔 기반 프로그램에서 사용자와 상호작용(입력)하거나, 정형화된 텍스트 파일의 데이터를 순차적으로 읽어들일 때 주로 사용
>     
> - **핵심**: `nextInt()` 등의 토큰 기반 입력과 `nextLine()`의 줄 단위 입력 혼용 시 발생하는 **버퍼 잔류 문제**의 원인 파악 및 해결 전략 숙달
>     
## Scanner 기본 동작 원리 및 객체 생성
- **입력 스트림 연결 및 객체 초기화**
    - `java.util.Scanner` 패키지 임포트 필수
    - `System.in`(표준 입력), `File`, `String` 등 다양한 입력 소스와 연결 가능
    - 공백(Space, Tab, Enter)을 구분자(Delimiter)로 인식하여 데이터를 **토큰(Token)** 단위로 분리
    - **어떻게?**: 생성자의 매개변수로 입력 소스를 전달하여 객체를 생성한 후 활용
```plaintext
[ Token Parsing Flow ]

  Input: "10 20\n"
         │
         ▼
  ┌────────────┐
  │ Input Buf  │ [ 10 ] [공백] [ 20 ] [\n]    <-- 입력 버퍼 대기열
  └────┬───────┘
       │ sc.nextInt() 호출
       ▼
  ┌────────────┐
  │  Scanner   │ ──▶ 공백 이전의 "10"을 읽어 int 타입으로 파싱 후 반환
  └────────────┘
```
```java
// 1. 표준 입력(콘솔)으로부터 데이터를 읽는 Scanner 생성
Scanner consoleSc = new Scanner(System.in);

// 2. 파일로부터 데이터를 읽는 Scanner 생성 (예외 처리 필요)
Scanner fileSc = new Scanner(new File("src/data.txt"));

// 3. 단순 문자열을 파싱하는 Scanner 생성
Scanner stringSc = new Scanner("100 200 300");
```
## 핵심 입력 메서드 (Input Methods)
- **데이터 타입별 전용 파싱 메서드**
    - 입력된 텍스트를 정해진 타입으로 변환하여 반환
    - 예상치 못한 문자(예: 숫자가 와야 할 곳에 문자가 옴) 입력 시 `InputMismatchException` 런타임 에러 발생
    - **어떻게?**: 변수에 담을 데이터 타입과 일치하는 `nextXXX()` 메서드 호출

|**메서드**|**동작 원리 및 역할**|**반환 타입 등 기타 속성**|
|---|---|---|
|`next()`|공백(Space, Tab, Enter) 전까지의 **단어(토큰)** 하나를 읽음|`String`|
|`nextLine()`|개행 문자(Enter) 전까지의 **한 줄 전체**를 읽고 개행 문자는 버림|`String`|
|`nextInt()`|입력된 값을 파싱하여 정수형으로 반환|`int`|
|`nextDouble()`|입력된 값을 파싱하여 실수형으로 반환|`double`|
|`nextBoolean()`|입력된 값을 파싱하여 논리형으로 반환|`boolean` (`true`\|`false`)|

## 개행 문자(Enter) 버퍼 잔류 문제와 해결
- **토큰 기반 메서드와 줄 단위 메서드 혼용 시 충돌 현상**
    - `nextInt()`, `nextDouble()` 등은 숫자만 읽고 사용자가 누른 **엔터(`\n`)는 버퍼에 그대로 남겨둠**
    - 이 상태에서 직후에 `nextLine()`을 호출하면, 버퍼에 남아있던 `\n`을 "입력이 끝났다"는 신호로 오해하여 빈 문자열을 반환하고 즉시 종료됨
    - **어떻게?**: ==숫자 입력 직후 `nextLine()`을 빈 채로 한 번 호출하여 버퍼에 남은 엔터를 강제 소거(Flush)해야 함==
```plaintext
[ Buffer Flush Flow ]

 1. 사용자 입력 : 25 + [Enter]
 2. 버퍼 상태   : [ "25" ] [ "\n" ]
 3. nextInt()   : [ "25" ]만 가져감 ──▶ 버퍼에 [ "\n" ] 잔류

 [ 문제 상황 ] 
 4. nextLine()  : 버퍼에 남은 [ "\n" ]을 즉시 발견 ──▶ 빈 문자열("") 반환 후 강제 종료

 [ 해결 방안 ]
 4. 빈 nextLine() 호출 ──▶ 남은 [ "\n" ]을 꺼내서 버림 (Flush)
 5. 다시 nextLine() 호출 ──▶ 정상적으로 다음 문자열 입력 대기
```
```java
Scanner sc = new Scanner(System.in);

System.out.print("나이 입력: ");
int age = sc.nextInt(); // 25 입력 후 Enter

// [해결책] 버퍼에 남아있는 개행 문자(\n)를 읽어서 버림 (Buffer Flush)
sc.nextLine(); 

System.out.print("이름 한 줄 입력: ");
// 위에서 Flush를 하지 않았다면, 여기서 입력을 대기하지 않고 곧바로 넘어가버림
String name = sc.nextLine(); 

System.out.println("나이: " + age + " / 이름: " + name);
sc.close();
```
## 자원 해제 및 주의사항 (Resource Management)
- **Scanner 스트림 닫기**
    - 파일이나 네트워크 등 외부 자원과 연결된 경우 `close()`를 호출하여 메모리 누수 방지
    - 단, ==`System.in` 기반의 Scanner를 `close()`하면 자바 애플리케이션 전체의 표준 입력 스트림이 영구적으로 닫힘==
    - **어떻게?**: `System.in`용 Scanner는 프로그램 당 하나만 생성하여 공유하고, 애플리케이션이 완전히 종료되기 직전에만 해제할 것

# Java ThreadLocalRandom (멀티스레드 최적화 난수 생성기)
> [!summary] 핵심 요약
> 
> - **개념**: 멀티스레드 환경에서 스레드 간 경합 없이 안전하고 빠르게 난수를 생성하는 동시성(Concurrency) 유틸리티 클래스
>     
> - **왜?**: 기존 `java.util.Random`이 가진 단일 인스턴스 공유 방식의 동기화 락(Lock) 경합으로 인한 심각한 성능 저하를 극복하기 위함
>     
> - **언제?**: 웹 애플리케이션, 병렬 스트림(Parallel Stream), 스레드 풀 등 동시다발적인 난수 생성이 필요한 고성능 백엔드 환경
>     
> - **핵심**: 인스턴스 강제 생성(`new`)을 피하고 `ThreadLocalRandom.current()`를 호출하여 현재 스레드 전용 난수 생성기를 할당받는 패턴 숙달
>     
## 기본 동작 원리 및 인스턴스 획득
- **스레드 로컬(Thread-Local) 기반의 격리**
    - 내부적으로 `ThreadLocal` 메커니즘을 적용하여 호출하는 각 스레드마다 독립적인 `Random` 인스턴스를 메모리에 할당
    - 스레드 간 상태(Seed) 공유가 발생하지 않아 ==동기화(Lock) 대기 과정이 원천적으로 생략되며 처리량(Throughput)이 극대화됨==
    - 초기 시드(Seed)는 시스템에 의해 자동 설정 및 관리됨
    - **어떻게?**: ==생성자(`new`) 사용이 문법적으로 금지==되어 있으므로 정적 팩토리 메서드인 `current()`를 호출하여 획득
```plaintext
[ Random vs ThreadLocalRandom 구조 차이 ]

 1. java.util.Random (Contention)        2. ThreadLocalRandom (Thread-Local)

 Thread A ─┐                             Thread A ──▶ ┌────────┐
           │                                          │ Random │ <-- A 스레드 전용
           ▼                                          └────────┘
 Thread B ─┼─▶ ┌────────┐                Thread B ──▶ ┌────────┐
           │   │ Random │ <-- 공유 객체               │ Random │ <-- B 스레드 전용
           ▼   └────────┘                             └────────┘
 Thread C ─┘                             Thread C ──▶ ┌────────┐
  (Lock 병목 발생 대기)                               │ Random │ <-- C 스레드 전용
                                                      └────────┘
```
## 핵심 메서드 (범위 지정 난수 생성)
- **직관적인 Range API 제공**
    - 최소값(포함)과 최대값(미포함)을 명시적으로 인수로 받는 오버로드 메서드를 기본 제공함
    - 과거 `Random` 클래스에서 요구하던 복잡한 모듈러 연산(`%`) 및 덧셈 보정을 생략할 수 있음
    - **어떻게?**: `current()` 호출 직후 원하는 반환 타입에 맞추어 `nextXXX(origin, bound)` 메서드 호출

|**메서드**|**동작 원리 및 역할**|**반환 타입 등 기타 속성**|
|---|---|---|
|`current()`|현재 실행 중인 스레드에 바인딩된 고유 인스턴스를 반환|`ThreadLocalRandom`|
|`nextInt(int bound)`|`0`부터 `bound - 1`까지의 정수 난수 반환|`int`|
|`nextInt(int origin, int bound)`|`origin`부터 `bound - 1`까지의 지정된 범위 정수 난수 반환|`int`|
|`nextDouble(double origin, double bound)`|`origin` 이상 `bound` 미만의 지정된 범위 실수 난수 반환|`double`|

```java
// 5개의 작업을 병렬로 실행하여 난수 동시 생성
IntStream.range(0, 5).parallel().forEach(i -> {
	// 1. 현재 스레드 전용 난수 생성기 인스턴스 획득 
	// (주의: 객체를 필드에 저장해두고 공유하면 안 됨. 매번 current() 호출 권장)
	ThreadLocalRandom random = ThreadLocalRandom.current(); 

	// 2. 범위 지정 난수 생성 (1 이상 11 미만 -> 1~10)
	int randomNumber = random.nextInt(1, 11); 
	
	String threadName = Thread.currentThread().getName();
	System.out.println("Thread [" + threadName + "] - Result: " + randomNumber);
});
```
## 주의사항 및 한계점 (Limitations)
- **인스턴스 공유 금지 및 보안적 한계**
    - `current()`로 얻은 인스턴스를 전역 변수나 다른 스레드에 넘겨서 공유하면 스레드 격리의 의미가 상실되고 예기치 않은 결과가 발생함
    - 선형 합동 생성기(LCG) 기반의 예측 가능한 알고리즘을 사용하므로 보안 목적으로는 절대 부적합함
    - **어떻게?**: 암호학적 안정성이 요구되는 토큰, 세션 ID, 비밀번호 생성 등에는 ==반드시 `java.security.SecureRandom`을 대체 사용==해야 함

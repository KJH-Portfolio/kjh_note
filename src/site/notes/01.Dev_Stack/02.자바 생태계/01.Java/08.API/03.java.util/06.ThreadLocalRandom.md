## 📘 ThreadLocalRandom 클래스 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - `ThreadLocalRandom` 클래스는 **멀티스레드 환경**에서 안전하고 고성능의 난수(Random Number) 생성을 위한 클래스.
>     
> - `java.util.Random`의 **스레드 안전성 문제와 경합(Contention) 문제**를 해결하기 위해 Java 7부터 도입.
>     
> - 각 스레드마다 **독립적인 난수 생성기** 인스턴스를 제공하여 락(Lock) 없이 작동.
>     

---

## 목적 🎯

- **멀티스레드 환경**에서 **`java.util.Random` 사용 시 발생하는 성능 저하**를 방지.
    
- 각 스레드가 독립적인 난수 생성기를 사용하여 **경합 없이** 높은 처리량(Throughput) 확보.
    
- 간단하고 직관적인 **범위 지정 난수 생성 메서드** 제공.
    

---

## 핵심 개념 💡

### ✅ 기본 동작 원리

- **스레드 로컬(Thread-Local):** 내부적으로 `ThreadLocal` 메커니즘을 사용하여 호출하는 **각 스레드에 전용 `Random` 인스턴스**를 바인딩.
    
- **`current()` 팩토리:** 인스턴스를 직접 생성하지 않고, **`ThreadLocalRandom.current()`** 정적 메서드를 통해 현재 스레드의 인스턴스를 얻어 사용.
    
- **시드(Seed) 자동 관리:** 최초 호출 시 자동으로 시드(Seed)를 설정하며, 사용자가 명시적으로 시드를 설정할 필요가 없음.
    

|**클래스**|**스레드 안전성**|**성능 (멀티스레드)**|**인스턴스 획득**|
|---|---|---|---|
|**`Random`**|O (Thread-Safe)|**저하** (내부적으로 락 사용)|`new Random()`|
|**`ThreadLocalRandom`**|O (Thread-Safe)|**우수** (락 미사용, 경합 없음)|`ThreadLocalRandom.current()`|

## 특징 ✨

### ✅ 직관적인 범위 지정 메서드

- 난수 생성 시 **최소값(포함)과 최대값(미포함)**을 인수로 받는 오버로드된 메서드를 제공하여 난수 범위 계산이 매우 간결함. (예: `nextInt(origin, bound)`)
    

### ✅ 불변(Immutable) 객체 아님

- 클래스 자체는 스레드에 로컬하게 바인스되어 있어 안전하지만, 인스턴스 자체는 `Random`을 상속받았으므로 **불변 객체는 아님**. 그러나 각 스레드가 고유 인스턴스를 가지므로 문제가 발생하지 않음.
    

### ✅ `java.util.concurrent` 소속

- **동시성(Concurrency)** 문제를 해결하기 위해 설계된 클래스임을 패키지 소속을 통해 명확히 함.

## 팁과 주의점 ⚠️

- **싱글스레드 환경:** 싱글스레드 환경에서는 성능 이점이 크지 않으므로 `java.util.Random` 또는 `ThreadLocalRandom` 중 무엇을 사용해도 무방함.
    
- **인스턴스화 금지:** 생성자가 **`private`**으로 선언되어 있으므로, **`new ThreadLocalRandom()`**으로 인스턴스화할 수 없음. 반드시 **`current()`** 메서드를 통해 획득해야 함.
    
- **보안 목적 제외:** 암호학적으로 안전한 난수(Cryptographically Secure Random Number)가 필요한 경우, **`java.security.SecureRandom`** 클래스를 사용해야 함. `ThreadLocalRandom`은 일반적인 시뮬레이션이나 게임 등에 적합.

## 예시 📌

### ✅ 멀티스레드 환경 난수 생성 사례

여러 스레드가 동시에 1부터 10까지의 난수를 생성할 때, `ThreadLocalRandom`이 어떻게 각 스레드에 독립적으로 동작하는지 보여주는 코드 실행 사례.

```java
import java.util.concurrent.ThreadLocalRandom;
import java.util.stream.IntStream;

public class ThreadLocalRandomFlow {

    public static void main(String[] args) {
        // 5개의 스레드를 병렬로 실행
        IntStream.range(0, 5).parallel().forEach(i -> {
            // ThreadLocalRandom.current()를 통해 현재 스레드 전용 인스턴스 획득
            ThreadLocalRandom random = ThreadLocalRandom.current(); 

            // 1부터 10 (11 미만) 사이의 난수 생성
            int randomNumber = random.nextInt(1, 11); 
            
            // 각 스레드의 이름과 생성된 난수 출력
            String threadName = Thread.currentThread().getName();
            System.out.println("스레드 [" + threadName + "] - 생성된 난수: " + randomNumber);
        });
        
        // --- 실행 결과 (순서는 실행 시마다 다를 수 있음) ---
        /*
        스레 [ForkJoinPool.commonPool-3] - 생성된 난수: 5
        스레 [main] - 생성된 난수: 9
        스레 [ForkJoinPool.commonPool-5] - 생성된 난수: 10
        스레 [ForkJoinPool.commonPool-2] - 생성된 난수: 2
        스레 [ForkJoinPool.commonPool-4] - 생성된 난수: 7
        */
    }
}
```

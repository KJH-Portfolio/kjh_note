- # Java 전역 시점과 시간대 처리 (`Instant` & `ZonedDateTime`)
> [!summary] 핵심 요약
> 
> - **개념**: 전 세계 어디서나 동일한 '찰나'를 의미하는 `Instant`와 특정 지역의 시간 관습을 반영한 `ZonedDateTime`의 결합
>     
> - **왜?**: 서버 간 데이터 교환 시에는 시간대 독립적인 시점이 필요하고, 사용자 화면에는 지역별 현지 시간이 필요하기 때문
>     
> - **언제?**: DB 저장/로그 기록(Instant), 스케줄링/사용자 화면 표시(ZonedDateTime)
>     
> - **핵심**: ==모든 시점의 기준은 UTC(Instant)이며==, 사용자의 위치에 따라 동적으로 지역 시간대를 적용하는 흐름을 숙달해야 함
>     
---
## 1. Instant (기계 친화적 절대 시점)
- **UTC 기준의 타임스탬프**
    - 1970년 1월 1일 0시 0분 0초(Epoch Time)를 기준으로 경과된 시간을 나노초 단위로 표현
    - 타임존 정보가 없으므로 ==로그 기록이나 데이터베이스 저장용 표준==으로 가장 적합
    - **어떻게?**: `Instant.now()`를 호출하면 항상 전 세계 공통 시점(Z가 붙은 형태)이 생성됨
```plaintext
[ Instant Visualization ]

  ┌───────────────┐
  │    Instant    │  <-- 1713182300.123 (Long Value)
  └───────┬───────┘
          │
    (UTC Standard)   <-- 전 세계 어디서나 동일한 "찰나"
```
```java
// 1. 현재 시점 (UTC)
Instant now = Instant.now(); 

// 2. 밀리초 단위 변환 (DB 저장/통신용)
long epochMilli = now.toEpochMilli();

// 3. 비교 연산
Instant earlier = now.minusSeconds(3600);
boolean isBefore = earlier.isBefore(now); // true
```

## 2. ZonedDateTime (인간 친화적 지역 시간)
- **지역 시간대와 섬머타임의 결합**
    - `LocalDateTime` + `ZoneId` + `ZoneOffset`이 합쳐진 형태
    - ==섬머타임(DST) 및 지역별 시간 변경 규칙을 자동으로 계산==해줌
    - **어떻게?**: `atZone()` 메서드를 사용하여 `Instant`에 특정 지역의 ID(예: `Asia/Seoul`)를 주입
```plaintext
[ Zoned Time Box Model ]

  ┌─────────────────────────┐
  │      ZonedDateTime      │
  ├────────┬────────┬───────┤
  │  Date  │  Time  │ Zone  │  <-- 지역 시각 + 타임존 아이디
  └────────┴────────┴───────┘
      │        │        └────  Asia/Seoul (UTC+09:00)
      └────────┴─────────────  2026-04-15 18:18:00
```
```java
// 1. UTC 시점을 한국 시간대로 변환
ZonedDateTime krTime = Instant.now().atZone(ZoneId.of("Asia/Seoul"));

// 2. 동일 시점의 뉴욕 시간 확인 (시간 표기만 변경됨)
ZonedDateTime nyTime = krTime.withZoneSameInstant(ZoneId.of("America/New_York"));

// 3. 특정 일시 + 특정 시간대 지정 생성
ZonedDateTime appointment = ZonedDateTime.of(2026, 5, 1, 14, 0, 0, 0, ZoneId.of("Europe/Paris"));
```

## 3. 클래스별 기능 비교

|**항목**|**Instant**|**LocalDateTime**|**ZonedDateTime**|
|---|---|---|---|
|**기준**|UTC (전 세계 공통)|로컬 (특정 지역)|로컬 + 타임존 정보|
|**주 용도**|서버 로그, DB 저장|내부 연산, 일정 예약|사용자 화면 출력|
|**섬머타임**|고려 안 함|고려 안 함|==자동 계산 및 적용==|
|**표기 형식**|`2026-04-15T09:18:00Z`|`2026-04-15T18:18:00`|`2026-04-15T18:18:00+09:00[Asia/Seoul]`|

# Java 지역 날짜 및 시간 처리 (`LocalDateTime`)
> [!summary] 핵심 요약
> 
> - **개념**: Java 8에서 도입된 스레드 안전(Thread-Safe)하고 불변성(Immutable)을 보장하는 날짜 및 시간 통합 처리 클래스
>     
> - **왜?**: 가변성(Mutable)으로 인해 멀티스레드 환경에서 치명적인 버그를 유발하던 기존 `Date`와 `Calendar` 클래스를 대체하기 위함
>     
> - **언제?**: 타임존(Timezone) 정보가 필요 없는 로컬 시스템 기준의 시간 기록, 일정 계산, 도메인 비즈니스 로직의 기한 설정 등에 사용
>     
> - **핵심**: 모든 연산 메서드는 원본 데이터를 변경하지 않고 연산이 적용된 **새로운 복사본 객체를 반환(Immutable)**함
>     
## 객체 생성 및 데이터 추출
- **현재 시점 획득 및 특정 시점 지정**
    - 시스템의 기본 시간대를 기준으로 현재 시각을 가져오거나, 연/월/일/시/분/초를 직접 지정하여 인스턴스 생성
    - `LocalDateTime`은 내부적으로 `LocalDate`(날짜)와 `LocalTime`(시간)을 조합한 형태를 띰
    - ==타임존(UTC 오프셋 등) 정보를 전혀 포함하지 않으므로, 글로벌 서비스의 다국어 시간 처리 시에는 `ZonedDateTime` 사용을 권장함==
    - **어떻게?**: 정적 팩토리 메서드 `now()` 또는 `of()`를 사용하여 객체를 생성한 후, `getXXX()` 메서드로 세부 필드값 추출
```plaintext
[ LocalDateTime Structure ]

  ┌───────────────┐
  │ LocalDateTime │  <-- 날짜(Date)와 시간(Time)의 결합 객체
  └──────┬─┬──────┘
         │ │
   ┌─────┘ └─────┐
   ▼             ▼
┌───────────┐ ┌───────────┐
│ LocalDate │ │ LocalTime │  <-- 필요 시 분리하여 단독 사용 가능
└───────────┘ └───────────┘
```
```java
// 1. 현재 날짜와 시간 생성
LocalDateTime now = LocalDateTime.now();

// 2. 특정 시점 지정 (2025년 12월 31일 23:59:59)
LocalDateTime endOfYear = LocalDateTime.of(2025, 12, 31, 23, 59, 59);

// 3. 내부 데이터 추출
int year = now.getYear();             // 연도 추출
int month = now.getMonthValue();      // 월 추출 (1~12로 직관적 반환)
int day = now.getDayOfMonth();        // 일 추출
int hour = now.getHour();             // 시간 추출 (24시간제)
```

|**기능**|**메서드명**|**매개변수**|**반환 타입**|**설명**|
|---|---|---|---|---|
|**생성**|`now()`|없음|`LocalDateTime`|시스템 기본 시간대의 현재 시각 반환|
|**생성**|`of()`|`year, month, day, hour, min...`|`LocalDateTime`|입력한 값으로 특정 시점의 객체 생성|
|**추출**|`getYear()`, `getMonthValue()`|없음|`int`|연, 월 등 특정 단위의 정수값 추출|
|**변환**|`toLocalDate()`, `toLocalTime()`|없음|`LocalDate` 등|날짜 또는 시간 부분만 절삭하여 반환|

## 날짜 연산 및 불변성 제어
- **불변 객체의 안전한 시간 연산**
    - `plusXXX()`, `minusXXX()` 메서드를 통해 직관적인 시간 가감 연산 지원
    - ==원본 객체의 상태를 절대 변경하지 않으므로, 연산 결과를 반드시 새로운 변수에 재할당해야 함==
    - `withXXX()` 메서드를 사용하면 특정 필드(예: 시간만 14시로 변경)만 교체된 새로운 객체 생성 가능
    - **어떻게?**: 연산 메서드들을 체이닝(Chaining) 방식으로 연결하여 복합적인 시간 계산 로직 구현
```java
LocalDateTime now = LocalDateTime.now();

// [잘못된 사용법] 원본(now)은 변경되지 않으므로 연산 결과가 유실됨
now.plusDays(5); 

// [올바른 사용법] 반환된 새 객체를 변수에 할당 & 체이닝 기법 적용
// 현재 시간에서 5일 뒤, 시간을 오후 2시(14시)로 고정한 배송 예정일 계산
LocalDateTime deliveryTime = now.plusDays(5)
								.withHour(14)
								.withMinute(0)
								.withSecond(0);
								
return deliveryTime;
```

|**기능**|**메서드명**|**매개변수 타입**|**반환 타입**|**설명**|
|---|---|---|---|---|
|**더하기**|`plusDays()`, `plusHours()`|`long amountToAdd`|`LocalDateTime`|지정된 수치만큼 더한 새 객체 반환|
|**빼기**|`minusMonths()`, `minusWeeks()`|`long amountToSubtract`|`LocalDateTime`|지정된 수치만큼 뺀 새 객체 반환|
|**변경**|`withYear()`, `withHour()`|`int newValue`|`LocalDateTime`|특정 필드의 값만 교체한 새 객체 반환|

## 문자열 포맷팅 및 파싱
- **객체와 문자열 간의 양방향 변환**
    - `DateTimeFormatter`를 사용하여 날짜 객체를 특정 형태의 문자열로 변환하거나, 문자열을 객체로 파싱
    - ==`DateTimeFormatter`는 스레드에 안전한 불변 객체이므로 실무에서는 메서드 내부가 아닌 `static final` 상수로 선언하여 재사용해야 함==
    - **어떻게?**: `ofPattern()`으로 포맷을 지정하고, `format()`(출력) 또는 `parse()`(입력) 메서드에 주입하여 사용
```java
public class FormattingExample {
    // 실무 권장: 포맷터는 불변 객체이므로 상수로 캐싱하여 애플리케이션 전역에서 재사용
    private static final DateTimeFormatter DEFAULT_FORMATTER = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");

    public void formatAndParse() {
        // 1. Format: 객체 -> 문자열 변환
        LocalDateTime now = LocalDateTime.now();
        String formattedDate = now.format(DEFAULT_FORMATTER);
        
        // 2. Parse: 문자열 -> 객체 역변환
        String inputStr = "2026-12-25 09:30:00";
        LocalDateTime parsedDate = LocalDateTime.parse(inputStr, DEFAULT_FORMATTER);
    }
}
```

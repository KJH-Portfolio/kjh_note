# Java 멀티스레딩 핵심: Thread와 Synchronized
> [!summary] 핵심 요약
> 
> **Thread**는 프로세스 내 독립적인 실행 흐름을 담당하는 최소 작업 단위이며, **Synchronized**는 공유 자원 접근 시 발생하는 데이터 불일치를 방지하는 동기화 메커니즘임. 두 요소 모두 **Java SE** 표준 기술로서 효율적인 병렬 처리와 데이터 무결성 보장을 위해 필수적으로 사용됨.
## 1. Thread: 병렬 실행의 기초
- **스레드(Thread)의 정의**
    - 프로세스 내에서 자원을 공유하며 실제 작업을 수행하는 최소 단위.
    - 개별적인 **Stack 영역** 할당 및 **Heap 메모리** 공유를 통한 효율적 통신.
- **스레드 구현 방식 비교**
    - **Thread 클래스 상속**: `run()` 메서드 오버라이드 방식, 구현의 간결함.
    - **Runnable 인터페이스 구현**: 다중 상속 제약 극복 및 코드 재사용성 증대로 인해 권장되는 방식.
```java
// 1. Thread 클래스 상속 방식
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread 상속 실행");
    }
}

// 2. Runnable 인터페이스 구현 방식 (권장)
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable 구현 실행");
    }
}

public class Main {
    public static void main(String[] args) {
        new MyThread().start();
        new Thread(new MyRunnable()).start();
    }
}
```
- **스레드 생명주기 (Lifecycle)**
    - `NEW`: 객체 생성 후 실행 대기 전.
    - `RUNNABLE`: 실행 중 혹은 실행 가능 상태.
    - `WAITING / BLOCKED`: 자원 대기 및 일시 정지.
    - `TERMINATED`: 실행 완료 및 종료.
## 2. Synchronized: 데이터 무결성 보장
- **동기화의 필요성**
    - **경쟁 조건(Race Condition)**: 여러 스레드가 동시에 공유 변수를 수정할 때 결과값이 왜곡되는 현상.
    - **원자성(Atomicity)**: 작업이 분리되지 않고 한 번에 수행됨을 보장하여 일관성 유지.
- **동기화 적용 방식**
    - **메서드 레벨**: 메서드 전체에 락(Lock)을 걸어 객체 단위의 접근 제어.
    - **블록 레벨**: 특정 코드 영역만 임계 영역으로 설정하여 성능 저하 최소화.
```java
public class Counter {
    private int count = 0;

    // 메서드 동기화: 해당 객체 전체에 락을 걺
    public synchronized void incrementMethod() {
        count++;
    }

    // 블록 동기화: 특정 객체에 대해서만 락을 걺 (효율적)
    public void incrementBlock() {
        synchronized(this) {
            count++;
        }
    }
}
```
## 3. Java SE 및 API 소속 정보
- **표준 라이브러리 지원**
    - `java.lang.Thread`: 별도의 import 없이 사용 가능한 자바 기본 클래스.
    - `synchronized` 키워드: 자바 언어 문법 자체에 내장된 예약어.
- **패키지 위치**
    - 모든 스레드 관련 핵심 API는 **Java SE (Standard Edition)** 환경에 포함됨.

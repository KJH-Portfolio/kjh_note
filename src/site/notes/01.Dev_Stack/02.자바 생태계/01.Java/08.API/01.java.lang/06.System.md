# 📘 java.lang.System 클래스 심층 분석
> [!summary] 핵심 요약
> 
> - 운영체제(OS)와 자바 프로그램 사이의 연결 고리 역할을 수행하는 핵심 유틸리티 클래스
>     
> - 목적: 표준 입출력, 환경 변수, 배열 복사, 시간 측정 등 OS 레벨의 시스템 자원에 쉽게 접근하고 제어하기 위함
>     
> - 핵심: 생성자가 `private`으로 감춰져 있어 객체 인스턴스화가 불가능하며, ==모든 필드와 메서드를 `static`으로 직접 호출==해야 함
>     
## 표준 입출력 스트림 (Standard I/O Streams)
- **스트림 자동 연결**
    - JVM이 시작될 때 OS와 자동으로 연결되는 3가지 기본 스트림 제공
    - `System.out.println(객체)` 호출 시 내부적으로 해당 객체의 `toString()`을 자동 실행하여 문자열로 출력
```plaintext
System Standard Streams Architecture
 ├─ System.in  (InputStream)  : 키보드 등 표준 입력 장치와 연결 (바이트 단위 읽기)
 ├─ System.out (PrintStream)  : 콘솔 등 표준 출력 장치와 연결 (일반 정보 출력)
 └─ System.err (PrintStream)  : 콘솔 등 표준 에러 장치와 연결 (버퍼를 우회하여 즉각적인 에러 출력)
```

|**필드 / 메서드**|**설명**|**주요 특징**|
|---|---|---|
|`System.in`|표준 입력 스트림|단일 바이트 처리만 가능하여 실무에서는 `Scanner`나 `BufferedReader`로 감싸서 사용 필수|
|`System.out`|표준 출력 스트림|`print()`, `println()`, `printf()` 등의 출력 포맷팅 메서드 제공|
|`System.err`|표준 에러 스트림|IDE 콘솔에서 주로 빨간색 텍스트로 시각적 구분되어 출력됨|

```java
import java.util.Scanner;

public class SystemIOExample {
    public static void main(String[] args) {
        // 1. 표준 출력 (형식 지정)
        int status = 200;
        System.out.printf("서버 상태 코드: %d\n", status);

        // 2. 표준 에러 출력
        System.err.println("경고: 비정상적인 접근이 감지되었습니다.");

        // 3. 표준 입력 (Scanner 보조 스트림 활용)
        Scanner scanner = new Scanner(System.in);
        System.out.print("명령어 입력: ");
        String command = scanner.nextLine();
        
        System.out.println("입력된 명령어: " + command);
    }
}
```
## 시간 측정 (Time Measurement)
- **시간의 절대성과 상대성 구분**
    - 용도에 맞지 않는 시간 측정 메서드 사용 시 윤초(Leap Second)나 시스템 시간 동기화에 의해 시간 역전 현상 발생 가능

|**메서드 시그니처**|**반환 단위**|**측정 기준**|**주요 용도**|
|---|---|---|---|
|`currentTimeMillis()`|Millisecond (`1/1000`초)|1970년 1월 1일 UTC 기준 절대 시간|현재 날짜 및 시간 계산, DB 타임스탬프 기록|
|`nanoTime()`|Nanosecond (`1/10^9`초)|JVM이 시작된 시점 등의 임의의 상대적 시간|==로직 실행 소요 시간(성능) 정밀 측정==|

```java
// 애플리케이션 성능(소요 시간) 측정 기본 패턴
long startTime = System.nanoTime(); 

// ... (시간을 측정할 비즈니스 로직 실행) ...
for (int i = 0; i < 1_000_000; i++) { }

long endTime = System.nanoTime();
System.out.println("실행 소요 시간: " + (endTime - startTime) + " ns");
```
## 배열 고속 복사 (`arraycopy`)
- **OS 레벨의 메모리 복사**
    - 반복문(`for`)을 사용한 원소 복사보다 JNI(Java Native Interface)를 통한 메모리 직접 복사를 수행하므로 대량의 데이터 처리 시 성능 우수
    - **치명적 주의사항**: 기본 데이터 타입(Primitive)은 실제 값을 복사하지만, 객체를 담은 참조형 배열은 ==메모리 주소만 복사하는 얕은 복사(Shallow Copy)==를 수행함

|**메서드 시그니처**|**주요 기능**|
|---|---|
|`arraycopy(src, srcPos, dest, destPos, length)`|원본(`src`)의 `srcPos` 인덱스부터 `length`만큼을 대상(`dest`)의 `destPos` 인덱스로 고속 복사|

```java
import java.util.Arrays;

public class ArrayCopyExample {
    public static void main(String[] args) {
        String[] origin = {"A", "B", "C", "D"};
        String[] target = new String[4];

        // origin[1]부터 2개의 요소를 target[2] 위치에 복사
        System.arraycopy(origin, 1, target, 2, 2);
        
        // 결과: [null, null, B, C]
        System.out.println(Arrays.toString(target));
    }
}
```
## 환경 정보 및 시스템 제어
- **환경 변수 vs 시스템 속성**
    - `getenv()`: OS 레벨에 설정된 환경 변수(예: `PATH`, `JAVA_HOME`) 읽기
    - `getProperty()`: JVM 구동 시 전달된 속성이나 자바 실행 환경 정보(예: `os.name`, `user.dir`) 읽기

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`getenv(String name)`|`String`|운영체제의 시스템 환경 변수 값 반환|
|`getProperty(String key)`|`String`|JVM 시스템 프로퍼티 값 반환|
|`gc()`|`void`|가비지 컬렉터(GC) 실행 명시적 요청 (JVM이 무시할 수 있음)|
|`exit(int status)`|`void`|JVM 실행 즉각 강제 종료 (상태 코드 `0`: 정상, 비 `0`: 에러)|

```java
// 1. 시스템 속성 및 환경 변수 조회
System.out.println("운영체제: " + System.getProperty("os.name"));
System.out.println("JAVA_HOME: " + System.getenv("JAVA_HOME"));

// 2. 시스템 강제 종료 (웹 서버 환경에서 사용 엄금)
int errorCode = 1;
// Tomcat, Spring Boot 등의 컨테이너 전체가 다운되므로 ==실무 비즈니스 로직 내 호출 절대 금지==
System.exit(errorCode); 
```

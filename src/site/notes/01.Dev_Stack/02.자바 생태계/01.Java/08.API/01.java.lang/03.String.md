# 📘 String 클래스 심층 분석
> [!summary] 핵심 요약
> 
> - 자바에서 텍스트 데이터를 다루기 위해 가장 빈번하게 사용되는 ==불변(Immutable)== 클래스
>     
> - 목적: 문자열 리터럴의 재사용을 통한 메모리 최적화 및 멀티스레드 환경에서의 안전성 확보
>     
> - 핵심: 객체의 참조값이 아닌 실제 문자열 내용을 비교할 때는 반드시 ==`equals()`== 메서드 사용 필수
>     
## String Pool과 객체 생성 방식
- **문자열 할당 메커니즘**
    - 문자열 리터럴(`""`) 사용 시 JVM의 Heap 메모리 내 `String Constant Pool`에 저장되어 동일 문자열 재사용
    - `new` 키워드 사용 시 Pool의 내용과 무관하게 일반 Heap 영역에 독립적인 객체 생성
```plaintext
String Memory Allocation
 ├─ Literal Type: String s = "hello";
 │   └─ [String Pool]에 "hello" 생성 -> s가 참조 (동일 리터럴은 참조 공유)
 │
 └─ Object Type: String s = new String("hello");
     └─ [일반 Heap]에 새로운 객체 생성 -> s가 참조 (메모리 낭비 발생)
```

| **생성 방식**      | **저장 위치**   | **== 비교 결과 (동일 리터럴 기준)** | **메모리 특징**                               |
| -------------- | ----------- | ------------------------ | ---------------------------------------- |
| `"문자열"`        | String Pool | `true` (참조 주소 동일)        | 동일한 문자열은 메모리에 단 하나만 존재 (메모리 절약)          |
| `new String()` | Heap        | `false` (참조 주소 다름)       | 호출 시마다 새로운 객체 생성 (사용 지양)                 |
| `intern()`     | String Pool | `true`                   | 일반 Heap에 있는 문자열을 강제로 Pool에 등록 후 해당 참조 반환 |

```java
// 1. 객체 생성 및 비교 예시
String a = "hello";
String b = "hello";
String c = new String("hello");

System.out.println(a == b);      // true: 동일한 String Pool 참조
System.out.println(a == c);      // false: 서로 다른 Heap 메모리 참조
System.out.println(a.equals(c)); // true: 내부 문자열 값은 동일

// 2. intern() 활용 예시
String d = c.intern();           // 객체 c의 문자열을 Pool에서 검색/등록 후 참조 반환
System.out.println(a == d);      // true: Pool 내부의 동일한 참조를 가짐
```
## 문자열의 불변성(Immutability)과 성능
- **불변 객체의 특징**
    - 한 번 생성된 `String` 객체는 내부의 문자열 값을 변경 불가
    - 문자열을 수정(결합, 자르기 등)하면 기존 객체가 변경되는 것이 아니라 새로운 객체가 생성됨
    - 스레드 환경에서 데이터 변경으로 인한 동기화 문제가 발생하지 않음 (Thread-safe)
- **문자열 결합(`+`) 컴파일러 최적화 원리**
    - 기존(Java 8 이하): `+` 연산 시 컴파일러가 매번 `new StringBuilder().append()` 구문으로 자동 변환하여 결합 수행
    - 변경(Java 9 이상): ==`invokedynamic`== 바이트코드 명령어를 도입하여, 런타임 시점에 최적의 결합 방식(`StringConcatFactory`)을 동적으로 결정
    - 효과: 컴파일 타임에 고정된 방식이 아니므로 불필요한 객체 생성을 줄이고 향후 JVM 버전에 따른 지속적 성능 향상 확보
```plaintext
String Concatenation Process (str1 + str2)
 ├─ [Java 8 이전] 컴파일 타임 고정
 │   └─ new StringBuilder(str1).append(str2).toString() -> 반복문 사용 시 매번 객체 생성 오버헤드
 │
 └─ [Java 9 이후] 런타임 동적 최적화
     └─ ==invokedynamic== 명령어 호출 -> JVM이 런타임에 최적의 결합 수행 (StringConcatFactory)
```
- **성능 최적화 주의점**
    - `invokedynamic`을 통한 내부적 최적화가 이루어졌더라도, 반복문(Loop) 내부에서의 잦은 문자열 결합은 여전히 막대한 가비지(Garbage)를 양산하므로 ==명시적인 `StringBuilder` 사용 필수==
```java
// 단일 결합은 컴파일러 최적화(invokedynamic)로 문제없음
String str = "Hello" + " World"; 

// 반복문 내부 결합은 최적화 한계로 인해 StringBuilder 명시적 사용 권장
StringBuilder sb = new StringBuilder("Hello");
for(int i=0; i<100; i++) {
    sb.append(" World"); 
}
System.out.println(sb.toString());
```
## 주요 문자열 조작 메서드

|**메서드 시그니처**|**반환형**|**기능 설명 및 예시**|
|---|---|---|
|`length()`|`int`|문자열의 길이 반환 (예: `"Hi".length()` → `2`)|
|`equals(Object)`|`boolean`|문자열의 내용이 정확히 일치하는지 비교|
|`substring(int, int)`|`String`|지정한 시작 인덱스부터 끝 인덱스 직전까지 추출 (예: `"Hello".substring(0,2)` → `"He"`)|
|`replace(CharSequence, CharSequence)`|`String`|특정 문자나 문자열을 다른 문자열로 모두 치환|
|`split(String)`|`String[]`|지정한 정규식 또는 구분자를 기준으로 문자열 분리|
|`trim()`|`String`|문자열의 양 끝단에 있는 공백(Whitespace) 제거|
|`contains(CharSequence)`|`boolean`|특정 문자열의 포함 여부 반환|
|`indexOf(String)`|`int`|특정 문자열이 처음 나타나는 인덱스 반환 (없으면 `-1`)|
|`toUpperCase()` / `toLowerCase()`|`String`|모든 문자를 대문자 혹은 소문자로 일괄 변환|
|`valueOf(Object)`|`String`|기본형(Primitive) 데이터 등 다른 타입의 값을 문자열로 변환|
|`toCharArray()`|`char[]`|문자열을 개별 문자 배열로 분해하여 반환|

```java
String str = "  Hello World  ";

// 1. length(): 문자열의 길이 (공백 포함) 반환
int len = str.length(); // 15

// 2. equals(Object): 내용이 완전히 같은지 비교
boolean isEqual = "Hello".equals("hello"); // false (대소문자 구분)

// 3. substring(int, int): 0번부터 2번 '직전'까지 추출
String sub = "Hello".substring(0, 2); // "He"

// 4. replace(old, new): 모든 "l"을 "L"로 치환
String rep = "Hello".replace("l", "L"); // "HeLLo"

// 5. split(String): 공백을 기준으로 나누어 배열 반환
String[] words = "A B C".split(" "); // ["A", "B", "C"]

// 6. trim(): 앞뒤의 불필요한 공백만 제거
String trimmed = str.trim(); // "Hello World"

// 7. contains(CharSequence): "World"가 포함되어 있는지 확인
boolean hasWorld = str.contains("World"); // true

// 8. indexOf(String): "W"가 처음 나타나는 위치(인덱스) 반환
int idx = str.indexOf("W"); // 8 (공백 포함 시)

// 9. toUpperCase() / toLowerCase(): 대/소문자 일괄 변환
String upper = "java".toUpperCase(); // "JAVA"

// 10. valueOf(Object): 숫자 100을 문자열 "100"으로 변환
String val = String.valueOf(100); // "100"

// 11. toCharArray(): 문자열을 하나씩 쪼개어 char 배열 생성
char[] chars = "Hi".toCharArray(); // ['H', 'i']'

// 자주 쓰이는 문자열 조작 실무 예시 (공백 제거 및 분리)
String rawData = "  apple, banana, orange  ";
String cleaned = rawData.trim().replace(" ", ""); // "apple,banana,orange"
String[] fruits = cleaned.split(","); // {"apple", "banana", "orange"}
```

# 📘 자바 String 클래스 아키텍처 및 최적화 가이드
> [!summary] 핵심 요약
> 
> - **개념**: 자바에서 텍스트 데이터를 처리하기 위해 제공되는 ==불변(Immutable)== 객체
>     
> - **왜?**: 동일한 문자열 리터럴의 재사용을 통한 메모리 절약 및 멀티스레드 환경의 안전성(Thread-safe) 확보
>     
> - **언제?**: 애플리케이션 내의 모든 텍스트 데이터 보관, 조작, 전달 시 기본 타입처럼 사용
>     
> - **핵심**: 객체의 물리적 주소가 아닌 논리적 문자열 내용을 비교할 때는 반드시 ==equals()== 메서드 사용 필수
>     
## String Pool과 객체 생성 메커니즘
- **문자열 할당 방식 및 저장 위치**
    - 리터럴(`""`) 선언: JVM Heap 영역 내 `String Constant Pool`에 저장되며, 동일한 값은 참조를 공유
    - 객체(`new`) 선언: 메모리 낭비를 유발하므로 ==new String()== 방식의 생성 지양
    - 캐싱 처리: 일반 Heap의 문자열도 ==intern()== 메서드 호출 시 Pool에 강제 등록 후 참조 반환

|**생성 방식**|**저장 위치**|**== 비교 (동일 리터럴)**|**메모리 특징**|
|---|---|---|---|
|`"문자열"`|String Pool|`true` (참조 공유)|동일 문자열은 메모리에 단 하나만 존재 (메모리 최적화)|
|`new String()`|일반 Heap|`false` (독립 객체)|호출 시마다 새로운 객체 생성 (사용 지양)|
|`intern()`|String Pool|`true` (참조 공유)|일반 Heap의 문자열을 Pool에 등록 및 참조 반환|

```plaintext
[ String Memory Allocation ]

 [Stack]             [Heap Area]
                   ┌────────────────────────────────────────┐
  String a ──┐     │  [String Constant Pool]                │
             ├───▶ │   "hello" (Address: 0x10)              │
  String b ──┘     │                                        │
                   ├────────────────────────────────────────┤
                   │  [General Heap]                        │
  String c ──────▶ │   new String("hello") (Address: 0x50)  │
                   └────────────────────────────────────────┘
```
```java
// 1. 객체 생성 및 참조 비교
String a = "hello";
String b = "hello";
String c = new String("hello");

System.out.println(a == b);      // true (동일한 String Pool 참조)
System.out.println(a == c);      // false (서로 다른 Heap 메모리 참조)
System.out.println(a.equals(c)); // true (내부 문자열 논리적 값 동일)

// 2. intern()을 통한 수동 캐싱
String d = c.intern();           // c의 문자열을 Pool에서 검색/등록 후 해당 참조 반환
System.out.println(a == d);      // true (Pool 내부의 동일 참조)
```
## 문자열의 불변성(Immutability)과 성능 고려사항
- **불변 객체의 구조적 특징**
    - 한 번 생성된 `String` 객체는 내부 문자열 데이터 수정 불가
    - 문자열 수정(결합, 자르기 등) 발생 시, 기존 객체 대신 새로운 객체 생성 및 반환
    - 상태가 변하지 않으므로 스레드 간 공유 시 동기화 처리 불필요 (Thread-safe)
- **문자열 결합(`+`) 동작 원리 및 한계**
    - Java 8 이전: 매번 `new StringBuilder().append()`로 변환되어 객체 생성 오버헤드 발생
    - Java 9 이후: ==invokedynamic== 바이트코드를 통해 런타임에 최적의 결합 방식 동적 결정
    - 주의사항: 단일 결합은 최적화되나, 반복문 내부의 결합은 최적화 한계로 가비지를 양산하므로 ==명시적인 StringBuilder 사용 필수==
```plaintext
[ String Concatenation Flow (str1 + str2) ]

 ├─ Java 8 이하 (정적 변환)
 │   └─ new StringBuilder(str1).append(str2).toString() ──▶ 반복문 사용 시 메모리 낭비 심각
 │
 └─ Java 9 이상 (동적 최적화)
     └─ invokedynamic 명령어 호출 ──▶ JVM이 런타임에 최적의 결합 알고리즘(StringConcatFactory) 적용
```
```java
// 단일 결합: 컴파일러 최적화(invokedynamic)로 성능 저하 없음
String str = "Hello" + " World"; 

// 반복 결합: 반복문 내부에서는 반드시 StringBuilder 명시적 사용
StringBuilder sb = new StringBuilder("Hello");
for(int i = 0; i < 100; i++) {
    sb.append(" World"); // 새로운 객체 생성 없이 내부 버퍼만 조작
}
System.out.println(sb.toString());
```
## 핵심 문자열 조작 API
- **주요 내장 메서드 기능 요약**
    - 원본 문자열을 훼손하지 않고 조작된 새로운 문자열 데이터를 반환함

|**메서드 시그니처**|**반환형**|**기능 설명 및 예시**|
|---|---|---|
|`length()`|`int`|문자열의 총 길이 반환 (공백 포함)|
|`equals(Object)`|`boolean`|대소문자를 엄격히 구분하여 논리적 내용 일치 여부 검증|
|`substring(int, int)`|`String`|시작 인덱스부터 끝 인덱스 직전까지 문자열 추출|
|`replace(old, new)`|`String`|특정 문자/문자열을 찾아 다른 문자열로 일괄 치환|
|`split(String)`|`String[]`|지정한 정규식이나 구분자를 기준으로 문자열 분해 및 배열 반환|
|`trim()`|`String`|문자열 양 끝단의 불필요한 공백(Whitespace) 제거|
|`contains(CharSeq)`|`boolean`|특정 문자열의 포함 여부 반환|
|`indexOf(String)`|`int`|특정 문자열의 최초 발생 위치 인덱스 반환 (없을 시 `-1`)|

```java
String rawData = "  apple, banana, orange  ";

// 1. 공백 제거 및 치환
String trimmed = rawData.trim();                  // "apple, banana, orange"
String noSpace = trimmed.replace(" ", "");        // "apple,banana,orange"

// 2. 구분자 기준 분리
String[] fruits = noSpace.split(",");             // ["apple", "banana", "orange"]

// 3. 인덱스 추출 및 자르기
int bananaIdx = noSpace.indexOf("banana");        // 6
String target = noSpace.substring(bananaIdx, bananaIdx+6); // "banana"
```

# 📘 Wrapper 클래스 심층 분석
> [!summary] 핵심 요약
> 
> - **개념**: 기본형(Primitive) 데이터를 내부 상태로 포장하여 객체(Object) 형태로 다룰 수 있게 해주는 클래스 묶음
>     
> - **왜?**: 자바의 제네릭(Generics)과 컬렉션 프레임워크(Collection Framework)는 기본형을 직접 다룰 수 없으므로, 이를 객체화하여 저장하고 다양한 유틸리티 메서드를 활용하기 위함
>     
> - **언제?**: `List`, `Map` 등의 자료구조에 데이터를 담거나, 문자열을 숫자로 파싱할 때, 혹은 데이터베이스의 `null` 값을 표현해야 하는 상황
>     
> - **핵심**: 기본형과 Wrapper 객체 간의 변환을 컴파일러가 자동으로 처리하는 ==Auto Boxing/Unboxing== 메커니즘의 이해와 남용 방지
>     
## 기본형과 Wrapper 클래스 매핑
- **타입 매핑 구조**
    - 자바의 8가지 기본 데이터 타입에 각각 1:1로 대응하는 참조형 클래스 제공
    - `char`와 `int`를 제외한 나머지 타입은 첫 글자만 대문자로 변환된 이름을 가짐

|**기본형 (Primitive)**|**Wrapper 클래스**|
|---|---|
|`byte`|`Byte`|
|`short`|`Short`|
|`int`|**`Integer`**|
|`long`|`Long`|
|`float`|`Float`|
|`double`|`Double`|
|`char`|**`Character`**|
|`boolean`|`Boolean`|

## Boxing과 Unboxing 메커니즘
- **변환 원리**
    - **Boxing (박싱)**: 기본형 데이터를 Wrapper 객체 내부로 포장하는 과정
    - **Unboxing (언박싱)**: Wrapper 객체에서 기본형 데이터 값을 추출하는 과정
    - **Auto (오토)**: Java 5부터 컴파일러가 코드 문맥에 맞게 박싱과 언박싱을 자동으로 삽입 처리
- **성능 고려사항**
    - 반복문(Loop) 내부에서 불필요한 오토 박싱이 누적될 경우 막대한 가비지(Garbage) 객체와 오버헤드가 발생하므로 ==대량의 산술 연산 시에는 반드시 기본형(Primitive) 사용 필수==
```plaintext
[ Auto Boxing & Unboxing Flow ]

   Stack Memory                       Heap Memory (Object Area)
 ┌──────────────┐                   ┌───────────────────────────┐
 │              │ ─── (Boxing) ───▶ │ [Integer Object]          │
 │ int val = 10 │                   │  value: 10                │
 │              │ ◀── (Unboxing) ── │                           │
 └──────────────┘                   └───────────────────────────┘
   기본형 변수                          Wrapper 참조 객체
```
## 핵심 유틸리티 메서드
- **주요 내장 메서드**
    - 각 Wrapper 클래스는 문자열 변환, 대소 비교, 데이터 검증 등을 위한 정적(static) 메서드 제공
    - `parseXxx`는 순수 기본형을 반환하고, `valueOf`는 객체를 반환하는 차이점 존재

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`parseXxx(String)`|기본형|문자열을 해석하여 순수 기본형 데이터로 파싱 (예: `Integer.parseInt("123")`)|
|`valueOf(String)`|Wrapper|문자열을 해석하여 Wrapper 객체 형태로 반환|
|`xxxValue()`|기본형|인스턴스에 포장된 기본형 값을 추출 (예: `boxed.intValue()`)|
|`toString(기본형)`|`String`|기본형 값을 문자열로 변환 (예: `Integer.toString(123)` → `"123"`)|
|`compareTo(객체)`|`int`|두 Wrapper 객체의 값을 비교하여 대소 관계 반환 (`-1`, `0`, `1`)|
|`isDigit(char)`|`boolean`|`Character` 클래스 전용 메서드로, 해당 문자가 숫자인지 판별|

## 실무 활용 시 필수 주의사항
- **객체 간 동등성 비교 (Equality)**
    - Wrapper는 참조형 객체이므로 `==` 연산자 사용 시 값이 아닌 메모리 주소를 비교하게 됨
    - 값의 논리적 일치 여부를 대조할 때는 ==반드시 `equals()` 메서드 사용 필수==
- **오토 언박싱과 예외 (NullPointerException)**
    - 객체는 `null`을 가질 수 있지만 기본형은 `null`을 가질 수 없음
    - `null` 상태인 Wrapper 객체를 기본형 변수에 대입(오토 언박싱)하려 시도할 경우 치명적인 ==`NullPointerException` 예외 발생==
```java
public class WrapperExample {
    public static void main(String[] args) {
        // 1. Auto Boxing & Unboxing
        int num = 10;
        Integer autoBoxed = num;      // 기본형 -> 객체 자동 포장
        int autoUnboxed = autoBoxed;  // 객체 -> 기본형 자동 추출

        // 2. 값 비교의 함정
        Integer a = new Integer(100); 
        Integer b = new Integer(100);
        System.out.println(a == b);      // false (메모리 참조 주소가 다름)
        System.out.println(a.equals(b)); // true (내부 값이 동일함)

        // 3. 문자열 데이터 변환
        int parsed = Integer.parseInt("456");       // 456 (int 반환)
        Integer wrapped = Integer.valueOf("789");   // 789 (Integer 객체 반환)
        String str = Double.toString(3.14);         // "3.14" (String 변환)

        // 4. NPE 발생 주의 구간
        Integer nullableObject = null;
        try {
            int primitiveVal = nullableObject; // null을 언박싱 시도 -> NPE 터짐
        } catch (NullPointerException e) {
            System.out.println("null 상태의 Wrapper는 기본형으로 변환할 수 없습니다.");
        }
    }
}
```

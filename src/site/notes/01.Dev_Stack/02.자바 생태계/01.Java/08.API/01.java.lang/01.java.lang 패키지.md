# java.lang 패키지
> [!summary] 핵심 요약
> 
> - **개념**: Java 프로그래밍의 가장 기본적이고 필수적인 클래스들을 모아둔 표준 라이브러리
>     
> - **왜?**: 문자열 처리, 수학 연산, 시스템 입출력 등 모든 프로그램에서 공통으로 필요한 토대 기능 제공
>     
> - **언제?**: Java 애플리케이션 개발 시 모든 계층 및 상황에서 항시 사용
>     
> - **핵심**: 별도의 `import` 선언 없이 자동으로 로드되는 유일한 패키지
>     
## 주요 클래스 구성 및 계층도
- **클래스 계층 구조**
    - 모든 자바 객체의 근간을 이루는 뼈대 역할
    - 최상위 부모인 `Object`를 상속받아 세부 기능 구현
```plaintext
[ java.lang 핵심 클래스 계층도 ]

Object (모든 객체의 최상위 조상)
 ├── String, StringBuilder, StringBuffer (문자열 처리)
 ├── System (운영체제/시스템 제어)
 ├── Math (수학적 연산/난수)
 ├── Thread (동시성/멀티스레딩)
 ├── Record (불변 데이터 객체의 뼈대) // Java 14+ 
 ├── Throwable (오류 및 예외 처리의 근원)
 │    ├── Error (시스템 치명적 오류)
 │    └── Exception (프로그램 예외)
 └── Number (숫자형 Wrapper 최상위)
      ├── Integer, Long, Short, Byte
      └── Double, Float
```

|**분류**|**핵심 클래스**|**주 사용 목적 및 특징**|
|---|---|---|
|**최상위 부모**|`Object`|자바의 모든 객체가 상속받는 기본 틀 (객체 비교, 해시, 문자열 반환)|
|**문자열**|`String`|텍스트 데이터 보관 및 처리 (불변성)|
|**시스템**|`System`|표준 입출력(out, in) 제어 및 현재 시간, 환경 변수 조회|
|**수학 연산**|`Math`|정적(Static) 메서드 기반의 수치 계산 및 난수(`random()`) 생성|
|**포장 객체**|`Wrapper`|기본형(Primitive Type) 데이터를 객체로 다루기 위한 변환 (`Integer` 등)|
|**스레드**|`Thread`|멀티스레드 환경의 프로세스 실행 및 제어|
|**레코드**|`Record`|DTO 등 불변 데이터 객체 생성을 위한 뼈대 (보일러플레이트 코드 제거)|
|**예외 처리**|`Throwable`|런타임 내 발생하는 모든 예외(`Exception`)와 오류(`Error`)의 최상위 기준|

## 핵심 클래스별 동작 원리 및 실무 가이드
### 1. Object 클래스
- 객체의 메모리 주소(참조값)를 기준으로 동작하는 기본 메서드 내장
- 논리적 동등성 비교 및 상태 출력을 위해 `equals()`, `hashCode()`, `toString()`의 오버라이딩 필수
- **어떻게?**: 도메인 객체(Entity, DTO 등) 생성 시 내부 데이터 기준으로 재정의
```java
class Person {
    String name;
    Person(String name) { this.name = name; }

    @Override
    public String toString() {
        // 참조값(메모리 주소)이 아닌 실제 의미 있는 데이터 출력으로 재정의
        return "Person Name: " + this.name;
    }
}
```
### 2. String 클래스
- 한 번 생성되면 내부 상태가 변하지 않는 불변(Immutable) 객체
- 반복적인 문자열 결합(`+` 연산) 시 새로운 객체가 계속 생성되어 메모리 누수 위험 증가
- **어떻게?**: 동적인 문자열 조작이 잦은 루프문 내부 등에서는 `StringBuilder` 활용
```plaintext
[ String의 불변성 (Immutability) 메모리 구조 ]

  Heap 영역 (String Pool)
 ┌─────────────────────────┐
 │                         │
 │  "Hello" ◀──── (1. 최초 문자열 생성 및 참조)
 │                         │
 │  "Hello World" ◀── (2. 결합 연산 시 기존 값 수정이 아닌 새 객체 할당)
 │                         │
 └─────────────────────────┘
 ※ "Hello" 객체는 수정되지 않고 메모리에 잔류함 (가비지 컬렉션 대상)
```
### 3. System 클래스
- JVM과 운영체제(OS) 간의 상호작용 채널 제공
- `System.exit()` 호출 시 JVM 자체가 즉시 종료되므로 웹 서버 환경 등에서는 실무 사용 엄금
- **어떻게?**: 콘솔 출력(`out`), 에러 출력(`err`), 밀리초 단위 현재 시간 측정(`currentTimeMillis()`)에 활용
### 4. Math 클래스
- 객체 생성 없이 직접 호출 가능한 정적(Static) 메서드와 상수(`PI` 등)만으로 구성
- `Math.random()`은 `0.0 <= x < 1.0` 범위의 `double` 난수 반환
- **어떻게?**: 정수 범위의 난수 추출 시 원하는 최대값을 곱한 후 명시적 정수형 변환(Casting) 적용
```java
// 1부터 10까지의 무작위 정수 추출 공식
int randomNum = (int)(Math.random() * 10) + 1;

// 2의 3제곱 연산 (반환형이 double임에 주의)
double powerResult = Math.pow(2, 3); // 8.0
```
### 5. Wrapper 클래스
- 기본 데이터 타입(`int`, `double` 등)을 객체 형태로 감싸는 포장지 역할
- `List`, `Map` 등 컬렉션 프레임워크는 객체 타입만 허용하므로 사용 필수
- **어떻게?**: 컴파일러가 지원하는 자동 변환(Auto-boxing/Unboxing) 기능을 통해 자연스러운 맵핑 수행
```java
int primitiveNum = 10;

// 명시적 래핑 (과거 방식)
Integer explicitBoxed = Integer.valueOf(primitiveNum);

// 오토박싱 (Auto-boxing): 컴파일러가 자동으로 객체 변환 수행 (Java 5+)
Integer autoBoxed = 20;

// 언박싱 (Unboxing): 래퍼 객체 연산 시 자동으로 기본형으로 변환
int result = autoBoxed + 5; // 25
```
### 6. Record 클래스 (Java 14+)
- 한 번 값이 설정되면 변하지 않는 데이터를 안전하게 담고 전달하기 위한 특수 클래스
- DTO(Data Transfer Object) 등을 만들 때 매번 작성해야 했던 생성자, `Getter`, `equals()`, `toString()` 등을 생략할 수 있어 코드가 극적으로 짧아짐
- **어떻게?**: `class` 대신 `record` 키워드를 사용해 선언하면, 컴파일러가 내부적으로 `java.lang.Record`를 자동으로 상속시키고 필수 메서드들을 생성해 줌
```java
// 1. Record를 활용한 간결한 DTO
public record User(String name, int age) {}
```

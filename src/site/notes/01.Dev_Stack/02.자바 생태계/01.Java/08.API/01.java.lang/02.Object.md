# 📘 java.lang.Object 클래스 심층 분석
> [!summary] 핵심 요약
> 
> - 자바의 모든 클래스가 묵시적으로 상속받는 계층 구조 상의 최상위(Root) 클래스
>     
> - 목적: 자바의 모든 객체가 공통으로 가져야 할 기본 동작(객체 비교, 정보 출력, 스레드 제어 등)의 표준 명세 제공
>     
> - 핵심: 참조값이 아닌 내부 데이터의 논리적 값을 비교하려면 `equals()`와 `hashCode()`를 ==반드시 함께 오버라이딩==해야 함
>     
## 계층 구조 및 기본 특징
- **자동 상속 메커니즘**
    - `extends` 키워드로 다른 클래스를 명시적으로 상속하지 않으면 컴파일러가 자동으로 `extends Object` 추가
    - 모든 자바 객체(배열 포함)는 최상위 부모인 `Object` 타입으로 업캐스팅(Upcasting)이 가능하여 강력한 다형성 제공 가능
```plaintext
Object Inheritance Architecture
 └─ java.lang.Object (최상위 Root)
     ├─ 기본 라이브러리 클래스 (String, System, Math 등)
     └─ 사용자 정의 클래스 (Member, Order 등)
         └─ 컴파일 시 자동 상속 처리됨
```
## 주요 메서드 명세표

| **메서드 시그니처**         | **반환형**    | **주요 기능 및 특징**                                       |
| -------------------- | ---------- | ---------------------------------------------------- |
| `equals(Object obj)` | `boolean`  | 두 객체의 참조값(메모리 주소) 동일성 비교 (오버라이딩으로 논리적 동등성 비교 지원)     |
| `hashCode()`         | `int`      | 객체를 식별하는 정수값(해시코드) 반환 (해시 기반 컬렉션에서 객체 검색 시 사용)       |
| `toString()`         | `String`   | 객체의 정보를 문자열로 반환 (기본값: `클래스명@16진수해시코드`)               |
| `getClass()`         | `Class<?>` | 런타임 시점의 객체 클래스(타입) 정보 반환 (리플렉션에 활용)                  |
| `clone()`            | `Object`   | 객체의 얕은 복사본(Shallow Copy) 생성 및 반환 (`Cloneable` 구현 필수) |
| `wait()`, `notify()` | `void`     | 멀티스레드 환경에서 스레드 간 동기화 및 락(Lock) 제어                    |
| `finalize()`         | `void`     | 가비지 컬렉터(GC)에 의해 객체 소멸 시 호출 (Java 9부터 ==실무 사용 금지==)   |

## 핵심 메서드 활용 및 주의점
### 1. 객체 비교 (`equals` & `hashCode`)
- **동일성 vs 동등성**
    - 동일성(Identity): `==` 연산자 사용, 물리적으로 완전히 같은 메모리 주소를 가리키는지 확인
    - ==동등성(Equality)==: `equals()` 오버라이딩 사용, 메모리 주소가 달라도 내부 데이터(값)가 같은지 논리적 확인
- **`hashCode()` 동반 오버라이딩 법칙**
    - `equals()` 비교 결과가 `true`인 두 객체는 반드시 같은 `hashCode()` 값을 반환해야 함
    - 규칙 위반 시 `HashSet`, `HashMap` 등 해시 기반 자료구조에서 동일 객체로 인식하지 못해 데이터 누락 등 치명적 오류 발생
```java
import java.util.Objects;

public class User {
    private String id;

    public User(String id) { this.id = id; }

    // 논리적 동등성 비교를 위한 equals 오버라이딩
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true; // 1. 물리적 주소가 같으면 통과
        if (obj == null || getClass() != obj.getClass()) return false; // 2. 타입이 다르면 실패
        User user = (User) obj;
        return Objects.equals(id, user.id); // 3. 내부 핵심 값(id) 비교
    }

    // equals를 재정의했으므로 hashCode도 반드시 재정의
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}
```
### 2. 객체 정보 출력 (`toString`)
- **가독성 향상을 위한 재정의**
    - 객체를 `System.out.println()`으로 출력하거나 문자열과 결합(`+`)할 때 내부적으로 자동 호출됨
    - 기본 반환값은 식별이 어려우므로, 디버깅 및 로깅을 위해 객체의 현재 상태(주요 필드값)를 직관적으로 보여주도록 즉시 오버라이딩 권장
```java
public class Product {
    private String name;
    private int price;

    public Product(String name, int price) {
        this.name = name;
        this.price = price;
    }

    // 기본값(Product@15db9742) 대신 유의미한 상태 데이터 출력
    @Override
    public String toString() {
        return "Product{name='" + name + "', price=" + price + "}";
    }
}
```
### 3. 객체 복사 (`clone`)
- **안전한 복사를 위한 전제 조건**
    - 해당 객체의 클래스가 반드시 ==`Cloneable` 인터페이스를 구현(implements)==해야만 정상 작동
    - 미구현 상태에서 `clone()` 호출 시 `CloneNotSupportedException` 예외 발생
    - `Object.clone()`은 필드 값만 복사하는 얕은 복사(Shallow Copy)이므로, 내부에 참조 타입(배열, 다른 객체 등)이 있다면 원본과 복사본이 주소를 공유하게 되어 깊은 복사(Deep Copy)를 위한 별도 로직 구현 필요

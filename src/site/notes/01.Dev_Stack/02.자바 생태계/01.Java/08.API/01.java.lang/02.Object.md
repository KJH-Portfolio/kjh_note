# 📘 java.lang.Object 클래스 심층 분석
> [!summary] 핵심 요약
> 
> - **개념**: 자바의 모든 클래스가 묵시적으로 상속받는 계층 구조 상의 최상위(Root) 클래스
>     
> - **왜?**: 객체 비교, 정보 출력, 스레드 제어 등 모든 자바 객체가 공통으로 가져야 할 기본 동작의 표준 명세 제공
>     
> - **언제?**: 사용자 정의 객체의 논리적 비교나 상태 출력 방식(동작)을 재정의할 때
>     
> - **핵심**: 참조값이 아닌 내부 데이터의 논리적 값을 비교하려면 `equals()`와 `hashCode()`를 ==반드시 함께 오버라이딩==해야 함
>     
## 계층 구조 및 기본 특징
- **자동 상속 메커니즘**
    - `extends` 키워드 미사용 시 컴파일러가 `extends Object`를 자동 추가
    - 모든 자바 객체(배열 포함)는 `Object` 타입으로 업캐스팅(Upcasting) 가능 (강력한 다형성 기반)
```plaintext
[ Object Inheritance Architecture ]
 └─ java.lang.Object (최상위 Root)
     ├─ 기본 라이브러리 클래스 (String, System, Math 등)
     └─ 사용자 정의 클래스 (Member, Order 등)
         └─ 컴파일 시 자동 상속 처리됨
```
## 주요 메서드 명세

| **메서드 시그니처**         | **반환형**    | **주요 기능 및 특징**                                         |
| -------------------- | ---------- | ------------------------------------------------------ |
| `equals(Object obj)` | `boolean`  | 객체의 참조값(주소) 동일성 비교 (오버라이딩을 통해 논리적 동등성 비교 지원)           |
| `hashCode()`         | `int`      | 객체 식별용 고유 정수값 반환 (해시 기반 자료구조의 검색 키로 사용)                |
| `toString()`         | `String`   | 객체 상태를 문자열로 반환 (기본값: `클래스명@16진수해시코드`)                  |
| `getClass()`         | `Class<?>` | 런타임 시점의 객체 클래스(타입) 메타 정보 반환 (리플렉션 활용)                  |
| `clone()`            | `Object`   | 객체의 얕은 복사본(Shallow Copy) 생성 (`Cloneable` 구현 필수)        |
| `wait()`,`notify()`  | `void`     | 멀티스레드 환경의 스레드 동기화 및 락(Lock) 제어                         |
| `finalize()`         | `void`     | 객체 소멸 시 GC에 의해 호출 (예측 불가능성으로 인해 Java 9부터 ==실무 사용 금지==) |

## 핵심 메서드 활용 및 주의점
### 1. 객체 비교 (`equals` & `hashCode`)
- **동일성 vs 동등성 구분**
    - 동일성(Identity): `==` 연산자, 물리적으로 완전히 같은 메모리 주소인지 확인
    - ==동등성(Equality)==: `equals()` 오버라이딩, 메모리 주소가 달라도 내부 데이터(값)가 같은지 논리적 확인
- **`hashCode()` 동반 오버라이딩 법칙**
    - `equals()`가 `true`인 두 객체는 반드시 동일한 `hashCode()` 값 반환 필수
    - 규칙 위반 시 `HashSet`, `HashMap` 등에서 같은 객체로 인식 불가하여 ==데이터 누락 오류== 발생
```plaintext
[ 동등성 비교 흐름도 (equals 재정의 시) ]
 1. 주소 비교 (==) ──────(같음)──────▶ [ true 반환 ]
      │ (다름)
      ▼
 2. 타입 검사 (instanceof) ──(다름)──▶ [ false 반환 ]
      │ (같음)
      ▼
 3. 내부 핵심 값 비교 ──────(일치)───▶ [ true 반환 ]
```
```java
import java.util.Objects;

public class User {
    private String id;

    public User(String id) { this.id = id; }

    // 논리적 동등성 비교를 위한 equals 오버라이딩
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true; // 1. 물리적 주소가 같으면 통과
        if (obj == null || getClass() != obj.getClass()) return false; // 2. 타입 검증
        User user = (User) obj;
        return Objects.equals(id, user.id); // 3. 내부 핵심 값(id) 비교
    }

    // equals 재정의 시 hashCode도 반드시 함께 재정의
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}
```
### 2. 객체 정보 출력 (`toString`)
- **가독성 향상을 위한 재정의**
    - `System.out.println()` 출력 또는 문자열 결합(`+`) 시 내부적으로 자동 호출됨
    - 기본 반환값은 식별이 어려우므로, 디버깅 및 로깅을 위해 주요 필드값을 출력하도록 즉시 재정의 권장
```java
public class Product {
    private String name;
    private int price;

    public Product(String name, int price) {
        this.name = name;
        this.price = price;
    }

    // 기본값(Product@15db9742) 대신 상태 데이터 출력으로 재정의
    @Override
    public String toString() {
        return "Product{name='" + name + "', price=" + price + "}";
    }
}
```
### 3. 객체 복사 (`clone`)
- **안전한 복사를 위한 전제 조건**
    - 해당 클래스가 반드시 ==`Cloneable` 인터페이스 구현== 필수 (미구현 시 `CloneNotSupportedException` 발생)
    - `Object.clone()`은 필드 값만 복사하는 얕은 복사(Shallow Copy)로 동작
    - 내부에 참조 타입(배열, 다른 객체) 필드가 있을 경우 주소를 공유하므로, 완벽한 분리를 원한다면 깊은 복사(Deep Copy) 로직 별도 구현 필요
    - 최신 문법은 ==복사 생성자==를 생성하여 사용하는 추세
```java
public class User {
    private String id;
    private List<String> roles; // 참조 타입 (배열이나 리스트)

    // 일반 생성자
    public User(String id, List<String> roles) {
        this.id = id;
        this.roles = roles;
    }

    // [추가] 복사 생성자: 깊은 복사 구현
    public User(User other) {
        this.id = other.id; // String은 불변(Immutable)이므로 그냥 복사해도 안전
        
        // 핵심: 새로운 리스트 객체를 생성하여 내부 값만 복사 (깊은 복사)
        this.roles = new ArrayList<>(other.roles); 
    }
}
```

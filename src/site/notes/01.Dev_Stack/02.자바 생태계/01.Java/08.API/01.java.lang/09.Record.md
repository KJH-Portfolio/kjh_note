# Java Record 클래스 핵심 및 활용
> [!summary] 핵심 요약
> 
> - **개념**: 불변(Immutable) 데이터 객체를 생성하기 위해 자바 14에 프리뷰로 도입되어 16부터 정식 표준이 된 특수 목적 클래스
>     
> - **왜?**: 데이터를 담고 전달하는 객체(DTO 등)에서 반복적으로 발생하는 막대한 보일러플레이트 코드(Getter, 생성자, `equals` 등)를 제거하기 위함
>     
> - **언제?**: API 응답/요청 DTO, 데이터베이스 조회 결과 매핑 등 상태 변경 없이 데이터 묶음을 전달해야 할 때 주로 사용
>     
> - **핵심**: 모든 필드는 `private final`로 선언되며, ==상태 변경(Setter)이 원천적으로 불가능한 불변 객체==임
>     
## Record의 탄생 배경과 기본 문법
- **기존 POJO 클래스의 한계**
    - 데이터를 담기 위해 클래스를 선언하면 필드 선언 외에도 생성자, Getter, `equals()`, `hashCode()`, `toString()`을 모두 오버라이딩해야 함
    - 핵심 비즈니스 로직 없이 코드 길이만 길어져 가독성 저하 및 유지보수 비용 증가
- **Record의 해결책**
    - `class` 대신 `record` 키워드를 사용하고 클래스 이름 옆에 상태(필드)를 괄호로 나열
    - 기존 `get필드명()` 관례 대신 ==필드명과 동일한 이름의 메서드(예: name())==를 통해 데이터 조회
```java
// 1. 기존 방식 (Lombok 또는 IDE 자동 생성 의존)
public class LegacyUserDto {
    private final String name;
    private final int age;

    public LegacyUserDto(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public String getName() { return name; }
    public int getAge() { return age; }
    // ... equals, hashCode, toString 생략 (수십 줄 차지)
}

// 2. Record 방식 (위의 LegacyUserDto와 완벽히 동일한 역할 수행)
public record UserRecord(String name, int age) {}

public class Main {
    public static void main(String[] args) {
        UserRecord user = new UserRecord("Alice", 25);
        
        // getter 대신 필드명() 형태로 호출
        System.out.println(user.name()); 
        System.out.println(user); // toString() 자동 지원: UserRecord[name=Alice, age=25]
    }
}
```
## 내부 동작 원리 및 제약 사항
- **컴파일 시점 자동 완성**
    - 자바 컴파일러(`javac`)가 레코드 헤더에 정의된 상태를 기반으로 바이트코드에 필수 메서드를 자동 삽입
- **상속 및 구조적 한계**
    - 컴파일 시 내부적으로 `java.lang.Record` 클래스를 자동으로 상속함
    - 자바는 다중 상속을 지원하지 않으므로 ==다른 클래스를 상속(extends)할 수 없음==
    - 단, 인터페이스 구현(implements)은 자유롭게 가능
    - 헤더에 정의되지 않은 인스턴스 필드(non-static 변수) 추가 불가
```plaintext
[ Record Compilation Structure ]

 Source Code                 Compiler                 Bytecode (.class)
 ┌────────┐                 ┌────────┐               ┌─────────────────┐
 │ Record │ ──────────────▶ │ javac  │ ────────────▶ │ extends Record  │
 └────────┘   (Auto Gen)    └────────┘               ├─────────────────┤
  (Header)                                           │ - private final │
                                                     │ - constructor   │
                                                     │ - name(), age() │ -> 필드값 반환
                                                     │ - equals()      │
                                                     │ - hashCode()    │
                                                     │ - toString()    │
                                                     └─────────────────┘
```

|**항목**|**동작 원리 및 역할**|**반환 타입 등 기타 속성**|
|---|---|---|
|**`equals()`**|객체의 메모리 주소가 아닌, 모든 필드의 값이 일치하는지 논리적 동등성 비교|`boolean`|
|**`hashCode()`**|모든 필드 값을 기반으로 해시 코드를 생성하여 해시 기반 컬렉션(Map, Set)에서 안전한 사용 보장|`int`|
|**`toString()`**|객체의 타입 이름과 각 필드의 이름 및 값을 맵핑하여 가독성 좋은 문자열로 출력|`String`|

## 컴팩트 생성자 (Compact Constructor) 활용
- **개념**: Record가 초기화될 때 데이터의 유효성을 검증(Validation)하거나 방어적 복사(Defensive Copy)를 수행하기 위한 특수 생성자
- **어떻게?**: 일반 클래스 생성자와 달리 ==매개변수 선언부와 필드 할당문(this.x = x)을 생략==하여 코드의 중복을 제거
```java
// 파일명: Product.java
public record Product(String id, int price) {
    
    // 컴팩트 생성자 (record 내부)
    public Product {
        if (price < 0) {
            throw new IllegalArgumentException("가격은 0보다 작을 수 없습니다.");
        }
        
        if (id == null || id.isBlank()) {
            id = "UNKNOWN";
        }
    }

    // 일반 메서드도 추가 가능합니다.
    public void printInfo() {
        System.out.println("상품ID: " + id + ", 가격: " + price);
    }
}
```

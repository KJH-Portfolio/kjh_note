# 📘 java.lang.Math 클래스 정리
> [!summary] 핵심 요약
> 
> - `Math` 클래스는 자바에서 각종 수학 연산을 수행하기 위해 제공되는 정적 유틸리티(Static Utility) 클래스
>     
> - 목적: 빈번하게 사용되는 수학 상수와 함수를 표준화하여 코드 간결화 및 계산 효율성 극대화
>     
> - 핵심: 모든 필드와 메서드가 `static`으로 선언되어 있어 ==객체 생성(new) 없이 클래스명으로 직접 호출==해야 함
>     
## 내부 구조 및 설계 특징
- **인스턴스화 차단 (Non-instantiability)**
    - 기본 생성자가 `private Math() {}` 형태로 감춰져 있어 외부에서 `new Math()`로 객체 생성 불가
    - 내부 상태(State) 필드를 가지지 않는 순수 함수형 메서드들의 집합체로 구성
```java
// 객체 생성 시도 시 컴파일 에러 발생
// Math math = new Math(); (X)

// 올바른 호출 방식: 클래스명.메서드명()
int maxValue = Math.max(10, 20); // (O)
```
## 주요 수학 상수 및 기본 연산
- **내장 상수 (Constants)**
    - 메서드가 아닌 `static final` 필드이므로 소괄호 `()` 없이 직접 접근

|**상수 필드**|**데이터 타입**|**설명**|**호출 예시**|
|---|---|---|---|
|`Math.PI`|`double`|원주율 (파이)|`Math.PI` (3.141592...)|
|`Math.E`|`double`|자연로그의 밑 (오일러의 수)|`Math.E` (2.718281...)|

- **기본 연산 및 절댓값**

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`max(a, b)`|입력 타입 동일|두 인자 중 더 큰 값 반환|
|`min(a, b)`|입력 타입 동일|두 인자 중 더 작은 값 반환|
|`abs(a)`|입력 타입 동일|음수를 양수로 변환한 절댓값 반환|

## 안전한 산술 연산 (Overflow 감지)
- **오버플로우(Overflow) 예방 메커니즘**
    - 일반적인 정수형 연산 시 변수 허용 범위를 초과하면 에러 없이 비정상적인 값(순환된 음수/양수)을 반환하여 치명적 버그 유발
    - Java 8부터 도입된 ==Exact 계열 메서드==는 연산 중 범위 초과 감지 시 즉각 `ArithmeticException` 예외를 발생시켜 시스템의 데이터 무결성 보호

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`addExact(x, y)`|`int` / `long`|두 값의 덧셈 수행 및 오버플로우 예외 감지|
|`subtractExact(x, y)`|`int` / `long`|두 값의 뺄셈 수행 및 언더플로우 예외 감지|
|`multiplyExact(x, y)`|`int` / `long`|두 값의 곱셈 수행 및 오버플로우 예외 감지|
|`toIntExact(long value)`|`int`|`long` 값을 `int`로 다운캐스팅 시 데이터 유실 예외 감지|

```java
// 1. 일반 연산의 Silent Overflow (치명적 버그 유발)
int max = Integer.MAX_VALUE; // 2147483647
int result = max + 1;        // -2147483648 (에러 없이 순환됨)

// 2. Exact 연산을 통한 안전한 예외 처리 (재무/결제 등 중요 로직 필수)
try {
    int safeResult = Math.addExact(max, 1);
} catch (ArithmeticException e) {
    System.out.println("오버플로우가 감지되어 연산을 안전하게 중단합니다.");
}
```
## 실수 처리 및 반올림 연산
- **소수점 제어 메서드 비교**
    - 금융, 과학 계산 등 ==정밀하고 오차 없는 소수점 연산이 필요할 때는 `Math` 대신 `BigDecimal` 클래스 사용 필수==

|**메서드 시그니처**|**반환형**|**기능 설명 (입력값 2.5 기준)**|**결과**|
|---|---|---|---|
|`ceil(double a)`|`double`|소수점 첫째 자리에서 무조건 올림 (Ceiling)|`3.0`|
|`floor(double a)`|`double`|소수점 첫째 자리에서 무조건 버림 (Floor)|`2.0`|
|`round(double a)`|`long`|소수점 첫째 자리에서 반올림 (입력이 `float`이면 `int` 반환)|`3`|

## 지수, 로그 및 난수 연산
- **거듭제곱 및 제곱근**
    - 반환형이 항상 `double`이므로 정수형 결과가 필요할 경우 명시적 캐스팅(Casting) 수반
- **난수 생성 (`Math.random` vs `ThreadLocalRandom`)**
    - `Math.random()`: `0.0` 이상 `1.0` 미만의 `double` 반환. 단, 단일 Seed를 공유하므로 멀티스레드 환경에서 동기화(Lock) 경합으로 인한 성능 저하(병목) 발생
    - `ThreadLocalRandom`: 멀티스레드 환경에서 난수 병목을 해결한 클래스. 스레드별로 Seed를 격리하며 지정된 범위의 정수를 즉시 반환하여 성능 최적화 달성

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`pow(double a, double b)`|`double`|`a`의 `b` 거듭제곱 계산|
|`sqrt(double a)`|`double`|`a`의 양의 제곱근 계산|
|`random()`|`double`|`0.0 <= x < 1.0` 범위의 의사 난수 반환|

```java
// 1. Math.random() 방식 (1부터 10까지 정수 난수, 캐스팅 수식 필요)
int randInt1 = (int)(Math.random() * 10) + 1; 

// 2. ThreadLocalRandom 방식 (1부터 10까지 정수 난수, 실무 멀티스레드 권장)
import java.util.concurrent.ThreadLocalRandom;
int randInt2 = ThreadLocalRandom.current().nextInt(1, 11); // 끝 인덱스는 미포함(Exclusive)
```
## 삼각함수 연산
- **라디안(Radian) 변환 주의점**
    - 모든 삼각함수 메서드는 일반 각도(Degree)가 아닌 라디안 값을 인자로 요구
    - `Math.toRadians(각도)`를 사용하여 명시적으로 변환한 뒤 계산해야 정확한 결과 도출

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`sin(double a)`|`double`|라디안 `a`의 사인(Sine) 값 반환|
|`cos(double a)`|`double`|라디안 `a`의 코사인(Cosine) 값 반환|
|`tan(double a)`|`double`|라디안 `a`의 탄젠트(Tangent) 값 반환|

```java
// 90도 일반 각도를 라디안으로 변환하여 사인 값 계산
double sinVal = Math.sin(Math.toRadians(90)); // 1.0
double cosVal = Math.cos(Math.toRadians(0));  // 1.0
```

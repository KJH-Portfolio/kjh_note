# 📘 java.lang.Math 클래스 아키텍처 및 최적화 가이드
> [!summary] 핵심 요약
> 
> - **개념**: 자바에서 각종 수학적 연산을 수행하기 위해 제공되는 정적 유틸리티(Static Utility) 클래스
>     
> - **왜?**: 빈번하게 사용되는 수학 상수와 함수를 표준화하여 코드 간결성 확보 및 연산 로직 재사용
>     
> - **언제?**: 산술 비교, 반올림, 난수 생성, 지수/삼각함수 등 기본 수리 연산이 필요할 때
>     
> - **핵심**: 모든 필드와 메서드가 `static`으로 선언되어 있어 ==객체 생성(new) 없이 클래스명으로 직접 호출== 필수
>     
## 내부 구조 및 설계 특징
- **인스턴스화 원천 차단 (Non-instantiability)**
    - 기본 생성자가 `private Math() {}` 형태로 은닉되어 외부의 무의미한 객체 생성 방지
    - 내부 상태값(State)을 가지지 않고 입력값에만 의존하는 순수 함수형 메서드들의 집합체
- **정적 접근 메커니즘**
    - 모든 기능을 `Math.메서드명()` 형태로 호출하여 메모리 낭비 제어
```java
// 객체 생성 시도 시 컴파일 에러 발생 (생성자 private 접근 제어)
// Math math = new Math(); 

// 올바른 호출 방식: 클래스명을 통한 정적(Static) 직접 접근
int maxValue = Math.max(10, 20); 
```
## 주요 수학 상수 및 기본 산술 연산
- **내장 상수 (Constants)**
    - 변하지 않는 수학적 기준값을 제공하며, 메서드가 아닌 필드이므로 소괄호 없이 직접 참조

|**상수 필드**|**데이터 타입**|**설명**|**호출 예시**|
|---|---|---|---|
|`Math.PI`|`double`|원주율 (파이, 약 3.14159)|`double area = Math.PI * r * r;`|
|`Math.E`|`double`|자연로그의 밑 (오일러의 수, 약 2.71828)|`double logVal = Math.E;`|

- **기본 비교 및 절댓값**

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`max(a, b)`|입력 타입 동일|두 인자 중 더 큰 값 반환|
|`min(a, b)`|입력 타입 동일|두 인자 중 더 작은 값 반환|
|`abs(a)`|입력 타입 동일|음수를 양수로 변환한 절댓값 반환|

## 안전한 산술 연산 (Overflow 방어)
- **오버플로우(Overflow) 예방 메커니즘**
    - 기본 정수 연산은 한계치 초과 시 에러 없이 음/양수가 뒤바뀌는 현상(Silent Overflow)이 발생하여 치명적 버그 유발
    - Java 8에 도입된 ==Exact 계열 메서드==는 연산 중 허용 범위 초과를 감지하면 즉각 예외를 발생시켜 시스템의 데이터 무결성 보호
```plaintext
[ Integer 연산 한계 초과 방어 메커니즘 ]

       일반 연산 (+ 연산자)                      Exact 연산 (Math.addExact)
 ┌──────────────────────────────────┐        ┌──────────────────────────────────┐
 │ MAX_VALUE (2147483647)           │        │ MAX_VALUE (2147483647)           │
 │            + 1                   │        │            + 1                   │
 ├──────────────────────────────────┤        ├──────────────────────────────────┤
 │ [X] -2147483648 (에러 없음)      │        │ [!] ArithmeticException 발생     │
 │ (비정상 값으로 로직 강행)        │        │ (시스템 즉각 중단 및 데이터 보호)│
 └──────────────────────────────────┘        └──────────────────────────────────┘
```

| **메서드 시그니처**          | **반환형**        | **주요 기능 및 특징**                        |
| --------------------- | -------------- | ------------------------------------- |
| `addExact(x, y)`      | `int` / `long` | 두 값의 덧셈 수행 및 오버플로우 예외 감지              |
| `subtractExact(x, y)` | `int` / `long` | 두 값의 뺄셈 수행 및 언더플로우 예외 감지              |
| `multiplyExact(x, y)` | `int` / `long` | 두 값의 곱셈 수행 및 오버플로우 예외 감지              |
| `toIntExact(long)`    | `int`          | `long` 값을 `int`로 다운캐스팅 시 데이터 유실 예외 감지 |

```java
// 1. 일반 연산의 치명적 결함 (Silent Overflow)
int max = Integer.MAX_VALUE; 
int result = max + 1;        // 결과: -2147483648 (에러 없이 순환됨)

// 2. Exact 연산을 통한 안전한 예외 처리 (재무/결제 등 핵심 비즈니스 로직 필수)
try {
    int safeResult = Math.addExact(max, 1);
} catch (ArithmeticException e) {
    System.out.println("오버플로우가 감지되어 연산을 안전하게 중단합니다.");
}
```
## 실수 처리 및 반올림 연산
- **소수점 제어 메서드 비교**
    - 주의: `Math` 클래스의 소수점 연산은 부동소수점 오차를 내포하므로, 금액 계산 등 ==정밀한 무오차 연산이 필요할 때는 BigDecimal 클래스 사용 필수==

|**메서드 시그니처**|**반환형**|**기능 설명 (입력값 2.5 기준)**|**결과**|
|---|---|---|---|
|`ceil(double a)`|`double`|소수점 첫째 자리에서 무조건 올림 (Ceiling)|`3.0`|
|`floor(double a)`|`double`|소수점 첫째 자리에서 무조건 버림 (Floor)|`2.0`|
|`round(double a)`|`long`|소수점 첫째 자리에서 반올림 (입력이 `float`이면 `int` 반환)|`3`|

## 지수, 로그 및 난수 연산
- **거듭제곱 및 난수 생성 최적화**
    - 지수 연산의 반환형은 항상 `double`이므로 정수 결과 도출 시 캐스팅 필요
    - 난수 생성 시 `Math.random()`은 내부적으로 단일 Seed를 공유하므로 ==멀티스레드 환경에서는 ThreadLocalRandom 사용으로 병목(Lock) 해소 필수==

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`pow(a, b)`|`double`|`a`의 `b` 거듭제곱 계산 (예: $2^3$)|
|`sqrt(a)`|`double`|`a`의 양의 제곱근 계산 (예: $\sqrt{16}$)|
|`random()`|`double`|`0.0` 이상 `1.0` 미만 범위의 의사 난수(Pseudo-random) 반환|

```java
// 1. Math.random() 방식 (1~10 난수) - 단일 스레드용, 수식 복잡함
int randInt1 = (int)(Math.random() * 10) + 1; 

// 2. ThreadLocalRandom 방식 (1~10 난수) - 멀티스레드 성능 최적화 (실무 권장)
import java.util.concurrent.ThreadLocalRandom;
// nextInt(시작값, 끝값) : 끝값은 미포함(Exclusive)되므로 11로 설정
int randInt2 = ThreadLocalRandom.current().nextInt(1, 11); 
```
## 삼각함수 연산
- **라디안(Radian) 변환 주의점**
    - `Math` 클래스의 모든 삼각함수 메서드는 우리가 흔히 아는 360도 일반 각도(Degree)가 아닌 라디안 값을 요구함
    - `Math.toRadians(각도)`를 선행 호출하여 단위를 맞추어야 정확한 값 도출 가능

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`sin(double a)`|`double`|라디안 `a`의 사인(Sine) 값 반환|
|`cos(double a)`|`double`|라디안 `a`의 코사인(Cosine) 값 반환|
|`tan(double a)`|`double`|라디안 `a`의 탄젠트(Tangent) 값 반환|

```java
// 90도(Degree)를 라디안으로 변환하여 올바른 삼각함수 값 계산
double sinVal = Math.sin(Math.toRadians(90)); // 결과: 1.0
double cosVal = Math.cos(Math.toRadians(0));  // 결과: 1.0
```

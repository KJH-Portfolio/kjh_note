# 📘 java.lang.System 클래스 심층 분석
> [!summary] 핵심 요약
> 
> - **개념**: 운영체제(OS)와 자바 프로그램 사이의 연결 고리 역할을 수행하는 핵심 유틸리티 클래스
>     
> - **왜?**: 표준 입출력, 환경 변수, 배열 고속 복사 등 OS 레벨의 시스템 자원에 직접 접근하고 제어하기 위함
>     
> - **언제?**: 애플리케이션 콘솔 입출력 처리, 로직 성능(시간) 측정, 시스템 속성 조회 및 강제 제어가 필요할 때
>     
> - **핵심**: 생성자가 숨겨져 있어 객체 인스턴스화가 불가능하며, 모든 기능을 ==static== 메서드와 필드로만 호출 필수
>     
## 표준 입출력 스트림 (Standard I/O Streams)
- **스트림 자동 연결 동작**
    - JVM 시작 시 OS와 자동으로 연결되는 3가지 기본 스트림 (입력, 일반 출력, 에러 출력)
    - `System.out.println(객체)` 호출 시 내부적으로 해당 객체의 `toString()` 자동 실행
```plaintext
[ System Standard Streams Architecture ]

 JVM (Java Application)                     OS (Operating System)
  ┌──────────────────┐                       ┌─────────────────────┐
  │                  │ ◀───(System.in)────── │ Keyboard (Input)    │
  │   System Class   │ ────(System.out)────▶ │ Console (Output)    │
  │                  │ ────(System.err)────▶ │ Console (Error)     │
  └──────────────────┘                       └─────────────────────┘
```

|**필드 / 메서드**|**설명**|**주요 특징**|
|---|---|---|
|`System.in`|표준 입력 스트림|단일 바이트 단위 처리 (실무에서는 `Scanner`나 `BufferedReader`로 감싸서 사용)|
|`System.out`|표준 출력 스트림|`print()`, `println()`, `printf()` 등 다양한 출력 포맷팅 지원|
|`System.err`|표준 에러 스트림|출력 버퍼를 우회하여 즉각 출력됨 (IDE 콘솔에서 붉은색 텍스트로 시각적 구분)|

```java
import java.util.Scanner;

public class SystemIOExample {
    public static void main(String[] args) {
        // 1. 표준 출력 (포맷 지정)
        int status = 200;
        System.out.printf("서버 상태 코드: %d\n", status);

        // 2. 표준 에러 출력 (버퍼 우회 즉각 출력)
        System.err.println("경고: 비정상적인 접근이 감지되었습니다.");

        // 3. 표준 입력 (Scanner 보조 스트림 결합)
        Scanner scanner = new Scanner(System.in);
        System.out.print("명령어 입력: ");
        String command = scanner.nextLine();
        
        System.out.println("입력된 명령어: " + command);
    }
}
```
## 시간 측정 (Time Measurement)
- **시간의 절대성과 상대성 구분**
    - 용도에 맞지 않는 메서드 사용 시 윤초(Leap Second) 보정이나 OS 시간 동기화에 의해 오차 발생 가능성 존재
    - 절대적 시간은 `currentTimeMillis()`, 상대적 경과 시간은 `nanoTime()`으로 명확히 구분하여 사용

|**메서드 시그니처**|**반환 단위**|**측정 기준**|**주요 용도**|
|---|---|---|---|
|`currentTimeMillis()`|Millisecond (`1/1000`초)|1970년 1월 1일 UTC 기준 절대 시간|날짜/시간 계산, DB 타임스탬프 기록|
|`nanoTime()`|Nanosecond (`1/10^9`초)|JVM 구동 시점 등 임의의 상대적 시간|==로직 소요 시간 정밀 측정 (성능 테스트)==|

```java
// 애플리케이션 성능(소요 시간) 측정 기본 패턴
long startTime = System.nanoTime(); 

// ... (시간을 측정할 비즈니스 로직 실행) ...
for (int i = 0; i < 1_000_000; i++) { }

long endTime = System.nanoTime();
System.out.println("실행 소요 시간: " + (endTime - startTime) + " ns");
```
## 배열 고속 복사 (`arraycopy`)
- **OS 레벨의 메모리 복사 메커니즘**
    - 자바 반복문(`for`)이 아닌 JNI(Java Native Interface)를 통한 OS 레벨 메모리 직접 복사로 대량 데이터 처리 최적화
    - 객체를 담은 참조형 배열 복사 시 실제 객체가 아닌 메모리 주소만 복사되는 ==얕은 복사(Shallow Copy)== 동작 주의

|**메서드 시그니처**|**주요 기능**|
|---|---|
|`arraycopy(src, srcPos, dest, destPos, length)`|원본(`src`)의 `srcPos` 위치부터 `length`만큼을 대상(`dest`)의 `destPos` 위치로 고속 복사|

```java
import java.util.Arrays;

public class ArrayCopyExample {
    public static void main(String[] args) {
        String[] origin = {"A", "B", "C", "D"};
        String[] target = new String[4];

        // origin[1]부터 2개의 요소("B", "C")를 target[2] 위치에 복사
        System.arraycopy(origin, 1, target, 2, 2);
        
        // 결과: [null, null, B, C]
        System.out.println(Arrays.toString(target));
    }
}
```
## 환경 정보 및 시스템 제어
- **환경 변수와 시스템 속성의 차이**
    - 환경 변수(`getenv`): OS 자체에 설정된 전역 변수값 읽기
    - 시스템 속성(`getProperty`): JVM 구동 환경 및 자바 실행 시 전달된 파라미터 값 읽기
> [!warning] 강제 종료 메서드 사용 주의
> 
> `System.exit()` 호출 시 현재 실행 중인 JVM 프로세스 자체가 즉각 종료됩니다. Spring Boot나 Tomcat 같은 웹 애플리케이션 서버(WAS) 환경에서는 전체 서비스 다운으로 직결되므로 ==실무 비즈니스 로직 내 호출 절대 금지== 원칙을 준수해야 합니다.

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`getenv(String name)`|`String`|운영체제의 시스템 환경 변수 값 반환 (예: `PATH`, `JAVA_HOME`)|
|`getProperty(String key)`|`String`|JVM 시스템 프로퍼티 값 반환 (예: `os.name`, `user.dir`)|
|`gc()`|`void`|가비지 컬렉터(GC) 실행 명시적 요청 (단, JVM이 무시할 수 있음)|
|`exit(int status)`|`void`|JVM 즉각 강제 종료 (상태 코드 `0`: 정상, 비 `0`: 에러)|

```java
// 1. 환경 변수 및 시스템 속성 조회
System.out.println("운영체제 이름: " + System.getProperty("os.name"));
System.out.println("JAVA_HOME 경로: " + System.getenv("JAVA_HOME"));

// 2. 시스템 강제 종료 (일반적인 CLI 배치 프로그램 등에서만 제한적 사용)
int errorCode = 1;
System.exit(errorCode); 
```

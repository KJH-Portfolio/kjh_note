# 📘 StringBuilder & StringBuffer 심층 분석
> [!summary] 핵심 요약
> 
> - **개념**: 내부 버퍼(Buffer)를 활용해 새로운 객체 생성 없이 텍스트를 직접 수정하는 ==가변(Mutable)== 문자열 클래스
>     
> - **왜?**: 잦은 문자열 연산 시 기존 `String` 객체가 유발하는 막대한 메모리 낭비와 가비지 컬렉션(GC) 오버헤드 해결
>     
> - **언제?**: 반복문(Loop) 내부에서 대량의 텍스트 결합, 삽입, 삭제 등의 연속적인 문자열 조작이 필요할 때
>     
> - **핵심**: 단일 스레드 환경에서는 성능이 압도적인 ==`StringBuilder`==를, 멀티 스레드 환경에서는 동기화가 보장되는 ==`StringBuffer`==를 분리하여 적용
>     
## 가변(Mutable) 문자열의 동작 원리
- **내부 버퍼(Buffer) 메커니즘**
    - 객체 생성 시 입력된 문자열 길이보다 넉넉한 여유 공간(기본 16문자)을 가진 내부 배열 할당
    - 문자열 추가/수정 시 기존 배열의 빈 공간을 그대로 활용하여 메모리 재할당 비용 제거
    - 버퍼 용량(`capacity`) 초과 시에만 기존 크기의 약 2배 크기로 새로운 배열을 생성 후 기존 데이터 복사 진행
```plaintext
[ Mutable String Processing Flow ]

 1. 초기화: new StringBuilder("Hi"); 
    └─ [Heap] 단일 객체 생성 및 여유 버퍼 확보
       ▶ [ 'H', 'i', \0, \0, \0, \0, ... ]  (빈 공간 대기)

 2. 데이터 조작: .append("!");
    └─ [Heap] 새 객체 생성 없이 기존 버퍼의 빈 공간에 데이터 직접 삽입
       ▶ [ 'H', 'i', '!', \0, \0, \0, ... ] (수정 완료)

 3. 최종 반환: .toString();
    └─ [Heap] 버퍼에 저장된 상태를 읽어 새로운 불변 String 객체로 변환 반환
```
## 클래스별 성능 및 환경 최적화 비교
- **스레드 안전성(Thread-Safe)과 성능의 상관관계**
    - `StringBuffer`: 내부 메서드에 `synchronized` 키워드가 적용되어 스레드 간 동시 접근을 제어(Lock)하므로, 데이터 무결성은 보장되나 성능 저하 발생
    - `StringBuilder`: 동기화 처리가 생략되어 있어 멀티 스레드 환경에서는 데이터가 깨질 수 있으나, 단일 스레드 환경에서는 연산 속도 극대화
- **메모리 할당 최적화 전략**
    - 대량의 텍스트 병합이 예상될 경우, 객체 생성 시점에 ==초기 `capacity`(용량)를 명시적으로 설정==하여 런타임 배열 복사 비용 최소화 필수

|**구분**|**String**|**StringBuilder**|**StringBuffer**|
|---|---|---|---|
|**객체 속성**|불변 (Immutable)|가변 (Mutable)|가변 (Mutable)|
|**동기화 지원**|지원 (불변이므로 안전)|❌ 비지원 (스레드 불안전)|✅ 지원 (스레드 안전)|
|**연산 성능**|매우 낮음 (잦은 병목)|==매우 높음==|보통 (Lock 오버헤드)|
|**권장 환경**|고정된 문자열 참조/전달|단일 스레드 및 로컬 변수 영역|멀티 스레드 공유 자원 영역|

## 핵심 문자열 조작 메서드 명세표

|**메서드 시그니처**|**반환형**|**주요 기능 및 특징**|
|---|---|---|
|`append(Object)`|`StringBuilder`|문자열 맨 끝에 데이터 추가 (메서드 체이닝 핵심)|
|`insert(int, Object)`|`StringBuilder`|지정한 인덱스 위치를 밀어내고 새로운 데이터 삽입|
|`delete(int, int)`|`StringBuilder`|시작 인덱스부터 끝 인덱스 직전까지의 구간 일괄 삭제|
|`deleteCharAt(int)`|`StringBuilder`|특정 인덱스 위치의 단일 문자만 삭제|
|`replace(int, int, String)`|`StringBuilder`|지정한 구간의 텍스트를 새로운 문자열로 완전 치환|
|`reverse()`|`StringBuilder`|버퍼 내 전체 문자열의 배열 순서를 역순으로 뒤집기|
|`ensureCapacity(int)`|`void`|내부 배열의 최소 버퍼 크기를 런타임에 강제로 확보|
|`toString()`|`String`|모든 가변 조작을 마친 데이터를 최종 불변 객체로 변환|

```java
// 실무 활용 예시: 메서드 체이닝(Method Chaining)을 통한 연속 조작
public class BufferExample {
    public static void main(String[] args) {
        
        // 1. 객체 생성 및 여유 용량(50) 명시적 확보 (배열 복사 방지)
        StringBuilder sb = new StringBuilder(50);
        
        // 2. 메서드 체이닝을 활용한 직관적인 데이터 조작
        sb.append("Hello")
          .append(" World")      // 결과: "Hello World"
          .insert(5, ",")        // 결과: "Hello, World"
          .replace(0, 5, "Hi")   // 결과: "Hi, World"
          .delete(2, 3);         // 결과: "Hi World" (쉼표 삭제)
          
        // 3. 역순 정렬 테스트
        sb.reverse();            // 결과: "dlroW iH"
        sb.reverse();            // 결과: "Hi World" (원상 복구)
        
        // 4. 조작 완료 후 반드시 String으로 변환하여 사용/반환
        String finalResult = sb.toString();
        System.out.println(finalResult);
    }
}
```

# 📘 StringBuilder & StringBuffer 클래스 심층 분석
> [!summary] 핵심 요약
> 
> - 내부 버퍼(Buffer)를 활용해 새로운 객체 생성 없이 문자열을 직접 변경할 수 있는 ==가변(Mutable)== 클래스
>     
> - 목적: 잦은 문자열 연산에서 발생하는 `String`의 메모리 낭비 문제 해결 및 애플리케이션 성능 최적화
>     
> - 핵심: 단일 스레드 환경에서는 성능이 우수한 ==`StringBuilder`==를, 스레드 경합이 예상되는 환경에서는 동기화가 보장되는 ==`StringBuffer`==를 선택 적용
>     
## 가변(Mutable) 문자열의 동작 원리
- **내부 버퍼(Buffer) 메커니즘**
    - 초기 생성 시 문자열 길이보다 넉넉한 여유 공간(기본 16글자 추가 버퍼)을 가진 내부 배열 할당
    - 문자열을 추가하거나 수정할 때 기존 배열의 빈 공간을 활용하므로 메모리 할당/해제 오버헤드 대폭 감소
    - 버퍼 용량(`capacity`) 초과 시에만 더 큰 새로운 배열을 생성하고 기존 데이터를 복사하는 방식 사용
```plaintext
Mutable String Processing (StringBuilder / StringBuffer)
 ├─ 1. 초기화: new StringBuilder("Hi"); 
 │   └─ [Heap] 단일 객체 생성 및 여유 버퍼 확보 -> [ 'H', 'i', \0, \0, \0, ... ]
 │
 ├─ 2. 데이터 조작: .append("!");
 │   └─ [Heap] 새 객체 생성 없이 기존 버퍼의 빈 공간에 데이터 직접 삽입
 │   └─ [ 'H', 'i', '!', \0, \0, ... ]
 │
 └─ 3. 최종 반환: .toString();
     └─ [Heap] 버퍼에 저장된 최종 상태를 불변 객체인 String으로 변환하여 반환
```
## 클래스별 성능 및 환경 최적화 비교
- **스레드 안전성(Thread-Safe)과 성능의 트레이드오프**
    - `StringBuffer`는 각 메서드에 `synchronized` 키워드가 적용되어 있어 스레드 간 동시 접근을 제어(Lock)하므로 상대적으로 연산 속도 저하 발생
    - 로컬 변수 기반의 문자열 조작이나 단일 스레드 환경에서는 동기화 오버헤드가 없는 `StringBuilder` 사용이 압도적으로 유리
- **성능 최적화 팁**
    - 대량의 데이터를 반복문으로 조작할 것이 예상된다면 객체 생성 시점에 ==`capacity(용량)`를 미리 설정==하여 런타임 배열 복사 비용 최소화 권장

|**구분**|**String**|**StringBuilder**|**StringBuffer**|
|---|---|---|---|
|**속성**|불변 (Immutable)|가변 (Mutable)|가변 (Mutable)|
|**동기화 지원**|불필요|❌ 비지원 (비동기)|✅ 지원 (스레드 안전)|
|**연산 성능**|낮음 (잦은 병목)|**매우 높음**|보통 (Lock 오버헤드)|
|**사용 환경**|고정된 문자열 참조|단일 스레드 또는 로컬 변수 환경|멀티 스레드 공유 자원 환경|

## 핵심 문자열 조작 메서드 명세표

| **메서드 시그니처**                | **반환형**         | **주요 기능 및 특징**                           |
| --------------------------- | --------------- | ---------------------------------------- |
| `append(Object)`            | `StringBuilder` | 기존 문자열의 맨 끝에 새로운 데이터 추가 (메서드 체이닝 지원)     |
| `insert(int, Object)`       | `StringBuilder` | 지정한 인덱스 위치에 새로운 데이터 삽입(==추가==)           |
| `delete(int, int)`          | `StringBuilder` | 시작 인덱스부터 끝 인덱스 직전까지의 구간 삭제               |
| `deleteCharAt(int)`         | `StringBuilder` | 특정 인덱스 위치의 단일 문자 삭제                      |
| `replace(int, int, String)` | `StringBuilder` | 시작 인덱스부터 끝 인덱스 직전까지를 새로운 문자열로 치환         |
| `reverse()`                 | `StringBuilder` | 버퍼 내의 전체 문자열 순서를 역순으로 뒤집기                |
| `ensureCapacity(int)`       | `void`          | 내부 배열의 최소 버퍼 크기를 명시적으로 확보 (성능 튜닝용)       |
| `toString()`                | `String`        | 가변 조작이 완료된 데이터를 최종 불변 `String` 객체로 변환 반환 |

```java
// 실무 활용 예시: 메서드 체이닝(Method Chaining)을 통한 연속 조작
public class BufferExample {
    public static void main(String[] args) {
        // 1. 객체 생성 및 여유 용량(50) 명시적 확보
        StringBuilder sb = new StringBuilder(50);
        
        // 2. 메서드 체이닝을 활용한 연속적인 데이터 조작
        sb.append("Hello")
          .append(" World")      // "Hello World"
          .insert(5, ",")        // "Hello, World"
          .replace(0, 5, "Hi")   // "Hi, World"
          .delete(2, 3);         // "Hi World" (쉼표 삭제)
          
        // 3. 역순 정렬 테스트
        sb.reverse();            // "dlroW iH"
        sb.reverse();            // "Hi World" (원상 복구)
        
        // 4. 조작 완료 후 불변 객체로 변환하여 최종 사용
        String result = sb.toString();
        System.out.println(result);
    }
}
```

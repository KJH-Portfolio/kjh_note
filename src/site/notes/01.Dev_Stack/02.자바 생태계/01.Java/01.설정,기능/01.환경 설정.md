# 🛠️ Java 및 Eclipse 개발 환경 아키텍처 (LTS 17)
> [!summary] 핵심 요약
> 
> Java 17 LTS 및 Tomcat 10.1 기반의 현대적 웹 개발 환경 구축 가이드.
> 
> Jakarta EE 전환에 따른 패키지 명명 규칙 변경 사항 및 UTF-8 인코딩 통일 전략 수립.
> 
> DevStyle 테마 및 Spring Boot 3.x 호환성을 고려한 IDE 최적화 설정 명시.
## 📌 핵심 인프라 구성 및 버전 전략
- **런타임 호환성 확보**
    - Java 17 (LTS) 기반의 안정적 빌드 환경 구축.
    - Spring Boot 3.x 최소 요구 사양 준수.
- **표준 규격 전환**
    - Jakarta EE 10 지원을 위한 Tomcat 10.1 서버 채택.
    - `javax.*`에서 `jakarta.*`로의 패키지 네임스페이스 전면 교체 대응.

|**항목**|**권장 사양 / 버전**|**비고**|
|---|---|---|
|**JDK**|Amazon Corretto 17|LTS 버전, 높은 안정성|
|**IDE**|Eclipse 2024-x|최신 유지보수 버전|
|**Web Server**|Apache Tomcat 10.1|Servlet 6.0 규격 지원|
|**ORM**|MyBatis 3.5.16|Boot Starter 3.0.5 연동|

---
## ⚙️ Eclipse Workspace 최적화 설정
### 1. 인코딩 및 기본 환경
- **UTF-8 통일**
    - Workspace, Spelling, 리소스 파일 전체 인코딩 설정.
    - 기종 간 소스 공유 시 한글 깨짐 방지.
- **디렉터리 구조 관리**
    - `src` (Source)와 `bin` (Binary) 경로의 명확한 분리.
### 2. JRE 및 컴파일러 구성
- **Installed JREs**
    - Amazon Corretto 17 등록 및 기본값(Default) 지정.
- **Compiler Compliance**
    - JDK Compliance 레벨을 17로 고정하여 문법 오류 방지.
- **Module System**
    - 일반 웹 프로젝트 시 `module-info.java` 생성 제외 (설계 복잡도 감소).

- **Project Facets 설정**
    - 프로젝트 성격에 맞는 기술 스택 버전 매핑.

|---|---|---|

> [!WARNING] Jakarta EE 전환 주의사항
> 
> Tomcat 10 버전 이상부터는 기존의 **`javax.servlet`** 패키지를 사용할 수 없으며, 반드시 **`jakarta.servlet`** 패키지를 임포트해야 정상 작동함.

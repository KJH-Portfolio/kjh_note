# Eclipse IDE Source 메뉴 및 소스 코드 자동화 가이드
> [!summary] 핵심 요약
> 
> - 보일러플레이트 코드의 자동 생성 및 정리를 통한 개발 생산성 및 코드 품질 극대화.
>     
> - 팀 표준 포맷팅(Format) 및 클린업(Clean Up) 규칙 적용을 통한 코드 일관성 확보.
>     
> - 객체 기본 메서드(Equals, HashCode 등)의 표준 구현 지원 및 유지보수성 제고.
>     
## 주요 기능 및 역할

| **기능 항목**                    | **핵심 역할**                | **기대 효과**          |
| ---------------------------- | ------------------------ | ------------------ |
| **Format**                   | 코드 정렬 및 들여쓰기 자동 교정       | 코드 가독성 및 일관성 향상    |
| **Organize Imports**         | 불필요한 import 제거 및 자동 추가   | 패키지 의존성 최적화        |
| **Clean Up**                 | 코드 스타일 일괄 정리 및 잠재적 버그 제거 | 전역적 코드 품질 관리       |
| **Generate Getters/Setters** | 캡슐화를 위한 필드 접근 메서드 생성     | 캡슐화 원칙 준수 및 시간 절감  |
| **Generate Constructor**     | 필드 기반의 생성자 자동 구현         | 객체 초기화 로직 표준화      |
| **Override Methods**         | 부모 클래스/인터페이스 메서드 재정의     | 상속 구조의 명확한 구현      |
| **HashCode / Equals**        | 객체 비교 및 해시 기반 컬렉션 활용 지원  | 논리적 동등성 보장 및 오류 방지 |
| **toString()**               | 객체 상태의 문자열 표현 생성         | 디버깅 및 로깅 효율화       |
| **Externalize Strings**      | 하드코딩 문자열의 리소스 번들 분리      | 다국어 지원(i18n) 기반 마련 |

## 소스 자동화 핵심 원칙
- **표준 규격 준수**
    - `Google Java Style Guide` 등 팀 프로젝트 표준 포맷터 설정 파일 적용 연계.
    - 개별 개발자의 코딩 습관 배제 및 프로젝트 단위의 단일 스타일 유지.
- **무결성 검증**
    - 자동 생성된 코드의 `@Override` 어노테이션 부여를 통한 컴파일 시점의 시그니처 검증.
    - `equals` 및 `hashCode` 생성 시 필드 선택을 통한 논리적 비교 범위 설정.
- **리팩토링 유연성**
    - `Surround With` 기능을 활용한 예외 처리(`try-catch`) 및 조건문 블록의 안전한 구조화.
    - `Generate Delegate`를 통한 구성(Composition) 기반의 메서드 위임 구조 자동화.

## 코드 구현 및 적용 예시
### 필드 기반 생성자 자동 생성
```java
public class User {
    private String id;
    private String email;

    // Source -> Generate Constructor using Fields 적용 결과
    public User(String id, String email) {
        this.id = id;
        this.email = email;
    }
}
```
### 객체 논리적 동등성 구현 (Equals & HashCode)
```java
@Override
public boolean equals(Object obj) {
    if (this == obj) return true; // 동일 객체 참조 확인
    if (obj == null || getClass() != obj.getClass()) return false; // 타입 일치 검증
    
    Point other = (Point) obj;
    return x == other.x && y == other.y; // 핵심 상태값 비교
}

@Override
public int hashCode() {
    return Objects.hash(x, y); // 필드 기반 해시값 생성
}
```

## 운영 가이드 및 주의사항
- **포맷팅 범위 관리**
    - 전체 프로젝트 적용 전 파일 단위 테스트 수행을 통한 의도치 않은 변경 방지.
    - Git 등의 형상 관리 도구에서 포맷팅 차이로 인한 `diff` 혼란 주의.
- **생성 코드 검토**
    - `toString()` 생성 시 순환 참조(Circular Reference) 발생 가능 필드 제외 처리.
    - 자동 생성된 스텁(Stub) 코드 내의 주석 및 임시 반환값 수정 필수.

# Eclipse IDE Source 메뉴 및 소스 코드 자동화 가이드
> [!summary] 핵심 요약
> 
> - 주제: Eclipse IDE의 Source 메뉴를 활용한 보일러플레이트(표준화) 코드 생성 및 품질 관리 자동화
>     
> - 목적: 반복 작업 최소화를 통한 개발 생산성 향상 및 팀 내 코드 스타일 일관성 유지
>     
> - 핵심: ==표준 포맷터== 적용 및 ==객체 기본 메서드== 자동 생성 기능 숙달
>     
## 주요 자동화 기능 명세

|**기능 항목**|**핵심 역할**|**기대 효과**|
|---|---|---|
|**Format**|코드 정렬 및 들여쓰기 자동 교정|==코드 가독성 확보==|
|**Organize Imports**|불필요한 import 제거 및 자동 추가|패키지 의존성 최적화|
|**Clean Up**|코드 스타일 일괄 정리 및 버그 제거|전역적 코드 품질 관리|
|**Generate Getters/Setters**|캡슐화를 위한 필드 접근 메서드 생성|==캡슐화 원칙 준수==|
|**Generate Constructor**|필드 기반의 생성자 자동 구현|객체 초기화 로직 표준화|
|**Override Methods**|부모 클래스/인터페이스 메서드 재정의|상속 구조의 명확한 구현|
|**HashCode / Equals**|객체 논리적 동등성 비교 로직 생성|==논리적 오류 방지==|
|**toString()**|객체 상태의 문자열 표현 생성|디버깅 및 로깅 효율화|

## 소스 자동화 핵심 원칙
- **표준 규격 준수**
    - `Google Java Style Guide` 등 팀 프로젝트 ==표준 포맷터 설정 적용==
    - 개별 코딩 습관을 배제하고 프로젝트 단위의 ==단일 스타일 유지==
- **무결성 검증**
    - 자동 생성 코드에 `@Override` 어노테이션을 부여하여 ==컴파일 시점 검증== 강화
    - `equals` 및 `hashCode` 생성 시 비교 대상 필드를 엄격히 선택하여 논리적 범위 설정
- **리팩토링 유연성**
    - `Surround With` 기능을 통한 예외 처리 및 조건문 블록의 구조화
    - ==메서드 위임 구조== 자동화를 위한 `Generate Delegate` 활용

## 코드 구현 및 적용 예시
- **객체 논리적 동등성 비교 절차**
```plaintext
객체 비교 내부 로직 (equals)
 ├─ 1. 주소값 확인 (==)
 │   └─ 일치: 동일 객체임 -> [True 반환]
 │
 └─ 2. 타입 및 상태 확인
     ├─ [검증 A] Null 체크 및 클래스 타입 비교
     │   └─ 불일치: 비교 불가 객체 -> [False 반환]
     │
     └─ [검증 B] 필드 데이터 대조
         ├─ 모든 지정 필드 일치: 동등한 객체 확정
         └─ 하나라도 다름: 다른 객체로 간주
```
- **필드 기반 생성자 및 동등성 구현 코드**
```java
public class User {
    private String id;
    private String email;

    // 1. Source -> Generate Constructor using Fields 적용
    public User(String id, String email) {
        this.id = id;
        this.email = email;
    }

    // 2. Source -> Generate hashCode() and equals() 적용
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true; // 동일 참조 시 즉시 성공
        if (obj == null || getClass() != obj.getClass()) return false; // 타입 검증
        
        User other = (User) obj;
        return Objects.equals(id, other.id) && Objects.equals(email, other.email); // 상태 비교
    }

    @Override
    public int hashCode() {
        return Objects.hash(id, email); // 필드 기반 해시값 생성
    }
}
```

## 운영 가이드 및 주의사항
- **포맷팅 범위 관리**
    - 전체 프로젝트 적용 전 반드시 ==파일 단위 테스트== 수행
    - 형상 관리 도구(Git)에서의 `diff` 혼란 방지를 위해 프로젝트 초기 단계에서 일괄 적용 권장
- **생성 코드 사후 검토**
    - `toString()` 생성 시 ==순환 참조 발생 필드 제외== 처리 확인
    - 자동 생성된 스텁(Stub) 코드 내의 임시 반환값 및 주석 수정 여부 점검

# Java 객체 지향 프로그래밍 (OOP) 설계 및 구현 가이드
> [!summary] 핵심 요약
> 
> - 현실 세계의 개념을 객체 단위로 추상화하여 소프트웨어에 투영하는 프로그래밍 패러다임
>     
> - 목적: 소프트웨어의 ==유지보수성== 향상 및 코드의 재사용성 극대화
>     
> - 핵심: 데이터(필드)와 행위(메서드)를 하나로 묶고 외부 노출을 최소화하는 객체 모델링
>     
## 1. 객체 지향의 목적 및 비즈니스 가치

|**항목**|**상세 내용**|
|---|---|
|**현실 모델링**|실세계 비즈니스 로직을 도메인 객체와 행위로 코드화|
|**모듈화**|객체 단위로 책임을 분리하여 수정 시 ==파급 효과 최소화==|
|**재사용성**|상속 및 합성을 활용하여 기존에 검증된 코드의 활용도 제고|
|**확장성**|다형성을 통해 기존 코드 수정 없이 기능 추가 가능성 확보|

---
## 2. 6대 핵심 메커니즘

|**개념**|**설명**|
|---|---|
|**클래스 (Class)**|객체 생성을 위한 상태와 행위가 정의된 틀(설계도)|
|**인스턴스 (Instance)**|클래스를 바탕으로 ==Heap 메모리==에 실제 할당된 객체|
|**캡슐화 (Encapsulation)**|`private` 접근 제어자로 정보를 은닉하고 public 메서드로 소통|
|**상속 (Inheritance)**|부모의 특성을 물려받아 계층 구조를 형성하고 코드를 재사용|
|**다형성 (Polymorphism)**|하나의 타입으로 여러 구현 객체를 참조하여 ==결합도 완화==|
|**추상화 (Abstraction)**|복잡한 내부 구현을 배제하고 핵심적인 인터페이스만 정의|

```plaintext
객체 비교 절차
 ├─ 1. 주소값 확인 (==)
 │    └─ 일치: 동일한 존재 확인 -> [종료: 성공]
 │
 └─ 2. 불일치 시: 내용물 비교 시작
      ├─ [단계 A] 해시값 비교 (속도 최적화)
      │    ├─ 다름: 확실히 다른 객체임 -> [종료]
      │    └─ 같음: 내용이 같을 가능성 있음 -> [다음 단계 진입]
      │
      └─ [단계 B] 논리적 비교 (.equals)
           ├─ 유효성 검사 (안정성)
           │    └─ 비어있거나(Null) 타입이 다름 -> [종료]
           └─ 세부 데이터 대조 (정밀 확인)
                ├─ 모든 데이터 일치: 동등한 객체로 최종 확정
                └─ 하나라도 다름: 다른 객체임 -> [종료]
```

## 3. 유연한 설계를 위한 SOLID 원칙

|**원칙**|**상세 정의**|**핵심 가치**|
|---|---|---|
|**SRP (단일 책임)**|하나의 클래스는 ==단 하나의 변경 이유==만 보유해야 함|높은 응집도|
|**OCP (개방-폐쇄)**|확장은 열려 있어야 하고, 수정은 닫혀 있어야 함|유연한 확장|
|**LSP (리스코프 치환)**|자식 클래스는 부모 클래스의 기능을 완벽히 대체 가능해야 함|계층의 일관성|
|**ISP (인터페이스 분리)**|범용 인터페이스보다 클라이언트를 위한 구체적 분리 권장|의존성 최적화|
|**DIP (의존 역전)**|고수준 모듈은 ==인터페이스(추상)에 의존==해야 함|결합도 감소|

## 4. 실무 설계 가이드라인 및 주의 사항
> [!tip] 고품질 OOP 설계를 위한 액션 플랜
> 
> - **정보 은닉 철저화**: 모든 필드는 ==`private`으로 선언==하고 최소한의 Getter만 제공
>     
> - **합성(Composition) 우선**: 상속의 경직성을 피하기 위해 ==기능 확장은 합성을 권장==
>     
> - **의도 명확화**: ==무분별한 Setter 사용을 금지==하고 행위의 의도가 드러나는 메서드명 사용
>     
> - **인터페이스 기반 설계**: 구체 클래스보다 인터페이스 타입을 참조하여 유연성 확보
>

## 5. 구현 예시 아키텍처
### 캡슐화 및 도메인 로직 처리
```java
public class User {
    private String name;
    private int age;

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    /**
     * 객체 내부 데이터에 기반한 판단 로직 (캡슐화)
     * 외부에서 age를 가져가서 판단하는 것이 아니라 객체에게 물어봄
     */
    public boolean isAdult() {
        return age >= 18;
    }
}
```
### 다형성을 활용한 인터페이스 기반 구현
```java
//결제 인터페이스
public interface Payment {
    void process();
}

//카카오페이 도메인(결제 인터페이스를 구현)
public class KakaoPay implements Payment {
    @Override
    public void process() {
        System.out.println("카카오페이 결제 진행");
    }
}

//결제 서비스(해당 도메인에 해당하는 결제 )
public class PaymentService {
    // 특정 구현체가 아닌 인터페이스에 의존 (DIP 준수)
    public void execute(Payment payment) {
	    payment.process(); 
    }
}

//호출하는 메인 클래스
public static void main(string[] args){
	
	//서비스 객체 호출
	PaymentService paymentService = new PaymentService();
	
	//카카오 페이 결제 진행
	Payment kakao = new KakaoPay();
	patmentService.execute(kakao);
}
```

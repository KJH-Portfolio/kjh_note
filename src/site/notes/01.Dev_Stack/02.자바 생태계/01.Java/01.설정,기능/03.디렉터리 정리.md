# Java 프로젝트 표준 아키텍처 및 규약 가이드
> [!summary] 핵심 요약
> 
> - 주제: Java 프로젝트의 표준 디렉터리 구조 및 명명 규칙 수립
>     
> - 목적: 협업 생산성 향상 및 빌드 자동화 환경 최적화
>     
> - 핵심: Maven/Gradle ==표준 레이아웃 준수== 및 도메인 역순 패키징
>     
## 프로젝트 표준 명명 규약

|**항목**|**권장 규칙**|**세부 내용**|
|---|---|---|
|**패키지 명**|`com.회사명.프로젝트명.모듈명`|전체 소문자 사용 및 ==도메인 역순 배치==|
|**클래스 명**|==파스칼 케이스 (PascalCase)==|명사형 단어 조합 (예: `UserRepository`)|
|**메서드 명**|카멜 케이스 (camelCase)|==동사형 시작== (예: `findUserById`)|
|**상수 명**|스네이크 케이스 (UPPER_SNAKE)|==전체 대문자 및 언더바== (예: `MAX_COUNT`)|

## 프로젝트 물리 구조 및 디렉터리 위계
- **소스 코드 격리**
    - `src/main/java`: 애플리케이션의 실제 로직을 담당하는 ==Java 소스 파일 배치==
    - `src/test/java`: 단위 테스트 및 통합 테스트 코드를 위한 ==별도 격리 공간==
- **리소스 및 설정 관리**
    - `src/main/resources`: XML, Properties, YAML 등 ==실행 시 필요한 설정 파일== 관리
- **빌드 스크립트 위치**
    - `pom.xml` 또는 `build.gradle`: 프로젝트의 의존성 관리를 위해 반드시 ==루트 디렉터리 위치 고수==
```plaintext
Project Root (프로젝트 최상위)
 ├─ src (소스 루트)
 │   ├─ main (배포용 코드)
 │   │   ├─ java (Java 소스)
 │   │   │   └─ com/company/project
 │   │   │       ├─ config (설정)
 │   │   │       ├─ controller (진입점)
 │   │   │       ├─ service (비즈니스 로직)
 │   │   │       └─ repository (데이터 접근)
 │   │   └─ resources (설정 및 정적 파일)
 │   │       ├─ static (이미지/JS/CSS)
 │   │       ├─ templates (HTML 템플릿)
 │   │       └─ application.yml (환경 설정)
 │   └─ test (검증용 코드)
 │       └─ java (테스트 소스)
 ├─ .gitignore (Git 제외 목록)
 └─ build.gradle (Gradle 빌드 스크립트)
```

## 소스 코드 작성 및 진입점 규격
- **애플리케이션 진입점**
    - 메인 클래스는 최상위 패키지에 위치하여 하위 패키지 스캔을 용이하게 함
    - 관례에 따라 `ProjectName + Application` 형식의 명칭 사용
```java
package com.company.project;

/**
 * 애플리케이션 실행을 위한 표준 메인 클래스
 * 1. 최상위 패키지 위치 준수
 * 2. 명확한 실행 로그 기록
 */
public class ProjectMyWorldApplication {
    public static void main(String[] args) {
        // 애플리케이션 시작 로직 수행
        System.out.println("Service Started Successfully.");
    }
}
```

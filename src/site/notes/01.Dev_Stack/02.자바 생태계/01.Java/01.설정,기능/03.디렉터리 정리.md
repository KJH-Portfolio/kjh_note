# Java 프로젝트 표준 아키텍처 및 규약 가이드
> [!summary] 핵심 요약
> 
> - Maven 및 Gradle 표준 레이아웃에 기반한 소스 코드와 리소스의 물리적 분리 원칙.
>     
> - 도메인 역순 패키징 및 파스칼 케이스 등 Java 생태계의 표준 명명 규칙 준수.
>     
> - 빌드 도구와의 호환성 및 유지보수성 극대화를 위한 디렉터리 위계 질서 확립.
>     
## 핵심 구축 목적
- **협업 표준화**
    - 일관된 디렉터리 구조 설계를 통한 팀원 간 코드 탐색 효율성 증대.
    - 프로젝트 온보딩 비용 최소화 및 코드 리뷰 생산성 향상.
- **빌드 자동화 및 안정성**
    - 표준 레이아웃(Standard Directory Layout) 준수를 통한 빌드 도구와의 설정 간소화.
    - 컴파일, 테스트, 패키징 공정의 오류 가능성 차단.
- **코드 무결성 유지**
    - 논리적 패키징을 통한 클래스 충돌 방지 및 접근 제어자 활용 최적화.
---
## 시스템 설계 원칙
- **관례 우선 원칙 (Convention over Configuration)**
    - 빌드 도구의 기본 설정을 그대로 수용하여 불필요한 설정 파일 비대화 방지.
- **관심사 분리 (Separation of Concerns)**
    - 실행 코드(`main`)와 검증 코드(`test`)의 엄격한 물리적 격리.
    - 비즈니스 로직과 설정 데이터(`resources`)의 관리 주체 분리.
- **명명 일관성**
    - 파일 시스템의 파일명과 언어적 클래스명의 1:1 매칭 보장.
---
## 표준 명명 및 관리 규약

| **항목**      | **권장 규칙**                   | **세부 내용**                       |
| ----------- | --------------------------- | ------------------------------- |
| **패키지 명**   | `com.회사명.프로젝트명.모듈명`         | 전체 소문자 사용, 도메인 역순 배치            |
| **클래스 명**   | 파스칼 케이스 (PascalCase)        | 명사형 단어 조합 (예: `UserRepository`) |
| **메서드 명**   | 카멜 케이스 (camelCase)          | 동사형 시작 (예: `findUserById`)      |
| **상수 명**    | 스네이크 케이스 (UPPER_SNAKE)      | 전체 대문자 및 언더바 (예: `MAX_COUNT`)   |
| **테스트 경로**  | 대상 클래스와 동일한 패키지 구조          | `src/test/java` 하위 배치           |
| **빌드 스크립트** | `pom.xml` 또는 `build.gradle` | 루트 디렉터리 위치 고수                   |

## 디렉터리 및 파일 물리 구조
### 표준 프로젝트 트리 구조
```plaintext
Project Root
 ├─ src
 │   ├─ main
 │   │   ├─ java (Java 소스 파일)
 │   │   │   └─ com/company/project
 │   │   │       ├─ config (설정 클래스)
 │   │   │       ├─ controller (진입점)
 │   │   │       ├─ service (비즈니스 로직)
 │   │   │       └─ repository (데이터 접근)
 │   │   └─ resources (설정 및 정적 파일)
 │   │       ├─ static (이미지/JS/CSS)
 │   │       ├─ templates (HTML 템플릿)
 │   │       └─ application.yml (환경 설정)
 │   └─ test
 │       └─ java (단위 및 통합 테스트)
 ├─ .gitignore (Git 제외 목록)
 └─ build.gradle (Gradle 빌드 스크립트)
```
### 진입점 클래스 규격 예시
```java
package com.company.project;

/**
 * 애플리케이션 실행을 위한 표준 메인 클래스
 */
public class ProjectApplication {
    public static void main(String[] args) {
        // 애플리케이션 시작 로직
        System.out.println("Service Started Successfully.");
    }
}
```

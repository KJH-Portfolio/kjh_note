# Java 형변환 (Type Casting) 아키텍처
> [!summary] 핵심 요약
> 
> - 자바(Java) 원시 타입(Primitive Type) 간 데이터 타입 변환 메커니즘
>     
> - 목적: 서로 다른 자료형 간 연산 및 대입 시 타입 안정성 확보
>     
> - 핵심: ==자동 형변환(Widening)==과 ==강제 형변환(Narrowing)==의 명확한 구분 및 데이터 손실 방지
>     
## 형변환 핵심 메커니즘
- **형변환(Type Casting)의 정의**: 특정 값의 자료형을 다른 타입으로 변환하는 과정
- **목적**: 서로 다른 자료형 간 연산 가능성 확보 및 메모리 효율성 보장
- **발생 시점**: 양변의 자료형이 다른 상태에서의 연산 및 대입 시 발생

| **구분**               | **방향 및 조건**                    | **특징 및 문법**                   |
| -------------------- | ------------------------------ | ----------------------------- |
| 자동 형변환<br>(Implicit) | 작은 범위 타입 $\rightarrow$ 큰 범위 타입 | 데이터 손실 없음, JVM에 의한 자동 처리      |
| 강제 형변환<br>(Explicit) | 큰 범위 타입 $\rightarrow$ 작은 범위 타입 | ==데이터 손실 위험==, `(자료형)값` 명시 필수 |

## 연산 및 대입 규칙
- **기본 연산 원칙**: 동일한 자료형 간 연산 수행
- **정수 타입 자동 승격**: 연산 시 `byte`, `short`, `char` 타입은 자동으로 `int`형으로 ==승격(Promotion)==
- **혼합 타입 연산**: 서로 다른 타입(예: 정수형과 실수형) 연산 시 더 큰 범위의 자료형(실수형)으로 자동 변환
- **문자열 결합**: 문자(`char`)와 문자열(`String`) 결합 시 문자열 타입으로 일괄 변환
## 주요 주의사항
- **데이터 손실**: 실수 타입(`double`)에서 정수 타입(`int`)으로 강제 변환 시 ==소수점 이하 값 절삭== 발생
- **문자형 수치 변환**: `char` 타입과 `int` 타입 간의 변환은 유니코드(Unicode) 십진수 값 기반 매핑
- **오버플로우(Overflow) 주의**: 큰 정수(`long`)를 작은 정수(`int`)로 강제 변환 시 표현 범위를 넘어서면 예기치 않은 값으로 변환
## 변환 예시 및 문법
### 1. 자동 형변환 (Implicit Casting)
```java
int x = 10;
double y = x;          // int → double (자동 형변환)
System.out.println(y); // 출력: 10.0
```
### 2. 강제 형변환 (Explicit Casting)
```java
double a = 3.14;
int b = (int)a;        // double → int (소수점 손실 발생)
System.out.println(b); // 출력: 3
```
### 3. 타입 혼합 연산 및 문자열 변환
```java
// 정수형과 실수형의 혼합 연산
int num = 5;
double val = 2.5;
double result = num + val; // 자동 형변환 (int → double)

// 문자형과 정수형 간 변환 (유니코드 매핑)
char c = 'A';
int code = c;          // char → int (유니코드 값 65)
char d = (char)66;     // int → char ('B')

// 문자형과 문자열의 결합
char ch1 = 'A';
char ch2 = 'B';
String strResult = "" + ch1 + ch2;  // char + String → String 자동 형변환
System.out.println(strResult);      // 출력: AB
```

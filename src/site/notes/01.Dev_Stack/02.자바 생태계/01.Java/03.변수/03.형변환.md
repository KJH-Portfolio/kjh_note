# Java 형변환 (Type Casting) 핵심 아키텍처
> [!summary] 핵심 요약
> 
> - Java 원시 타입(Primitive Type) 간 데이터 타입 변환 메커니즘
>     
> - 목적: 서로 다른 자료형 간 연산 및 대입 시 발생 가능한 타입 불일치 해결
>     
> - 핵심: ==자동 형변환(Widening)==과 데이터 손실 위험이 있는 ==강제 형변환(Narrowing)==의 명확한 구분
>     
## 1. 형변환의 기본 개념과 방향성
- **형변환(Type Casting)**
    - 특정 데이터의 자료형을 다른 타입으로 변환하는 과정
    - 서로 다른 자료형이 대입되거나 혼합 연산될 때 발생
```plaintext
[ 데이터 크기 및 형변환 방향성 ]

   [작은 크기] -----------------------------------------------------> [큰 크기]
   byte(1) -> short(2) -> int(4) -> long(8) -> float(4) -> double(8)
              char(2) ↗

   - 우측 이동(->) : 공간 넉넉함 -> 자동 형변환 (데이터 보존)
   - 좌측 이동(<-) : 공간 부족함 -> 강제 형변환 (데이터 손실 위험)
```

|**구분**|**방향 및 조건**|**특징 및 문법**|
|---|---|---|
|**자동 형변환 (Implicit)**|작은 범위 타입 $\rightarrow$ 큰 범위 타입|데이터 손실 없음, JVM에 의한 자동 처리|
|**강제 형변환 (Explicit)**|큰 범위 타입 $\rightarrow$ 작은 범위 타입|==데이터 손실 위험==, `(자료형)값` 명시 필수|

## 2. 자동 형변환 (Implicit Casting / Widening)
- **개념 및 특징**
    - 작은 데이터 타입이 큰 데이터 타입의 그릇에 담길 때 발생
    - 값의 누락이 없으므로 개발자의 추가 조치 없이 컴파일러가 자동 수행
- **정수 타입 자동 승격(Promotion)**
    - 산술 연산 시 `byte`, `short`, `char` 타입은 `int`형으로 ==자동 승격==
```java
// 1. 단순 대입 시 자동 형변환
int intValue = 10;
double doubleValue = intValue;      // int(4byte) -> double(8byte) 자동 승격
System.out.println(doubleValue);    // 출력: 10.0

// 2. 연산 시 int 승격 현상
byte b1 = 10;
byte b2 = 20;
// byte result = b1 + b2;           // ❌ 에러: 더하는 순간 int로 승격되어 byte 변수에 담을 수 없음
int intResult = b1 + b2;            // 정상 작동
```
## 3. 강제 형변환 (Explicit Casting / Narrowing)
- **개념 및 특징**
    - 큰 데이터 타입을 작은 데이터 타입으로 억지로 구겨 넣는 과정
    - 캐스팅 연산자 `(바꿀자료형)`을 값 앞에 명시해야만 컴파일 에러 방지 가능
- **주요 위험성**
    - 실수에서 정수로 변환 시 ==소수점 이하 값 절삭== 발생
    - 대상 타입의 허용 범위를 초과하는 값 대입 시 의도치 않은 쓰레기값이 저장되는 오버플로우(Overflow) 발생
```java
// 1. 소수점 절삭
double pi = 3.14159;
int intPi = (int)pi;               // double(8byte) -> int(4byte) 강제 형변환
System.out.println(intPi);         // 출력: 3 (0.14159 손실)

// 2. 오버플로우 (표현 범위 초과)
int maxInt = 2147483647;           // int 최대값
byte overflowByte = (byte)maxInt;  // 강제 형변환
System.out.println(overflowByte);  // 출력: -1 (의도하지 않은 쓰레기값 발생)
```
## 4. 혼합 연산 및 문자열 특수 변환
- **혼합 타입 연산**
    - 서로 다른 자료형 간 연산 시, 더 큰 범위의 자료형을 기준으로 일괄 자동 변환
    - 예: `int`와 `double` 연산 시 `double`로 계산 수행
- **문자와 정수 간 변환**
    - `char` 타입과 `int` 타입 간의 상호 변환은 유니코드(Unicode) 십진수 값을 기반으로 매핑
- **문자열 결합 (String Concatenation)**
    - 어떤 타입이든 문자열(`String`)과 `+` 연산으로 결합 시 ==문자열 타입으로 완전 자동 변환==
```java
// 1. 혼합 연산 시 큰 타입으로 자동 변환
int num = 5;
double val = 2.5;
double result = num + val;          // num이 5.0으로 변환 후 계산되어 7.5 저장

// 2. 문자형(char)과 정수형(int) 유니코드 매핑
char character = 'A';
int code = character;               // 'A'의 유니코드 값 (65) 자동 형변환
char nextChar = (char)(code + 1);   // (char)66 -> 'B' 강제 형변환

// 3. 문자열 결합을 통한 변환 안전성
char ch1 = 'A';
char ch2 = 'B';
// System.out.println(ch1 + ch2);   // 131 출력 (A와 B의 유니코드 합산, int 승격)
String strResult = "" + ch1 + ch2;  // 빈 문자열("")이 더해져 "AB" 문자열로 안전하게 변환
```

# 📘 Java 매개변수(Parameter)와 인자(Argument) 아키텍처
> [!summary] 핵심 요약
> 
> - Java 메서드의 데이터 입력 매개체인 ==파라미터와 아규먼트의 명확한 구분== 및 동작 원리
>     
> - 목적: 메서드의 유연한 설계 및 데이터 전달 시 발생하는 메모리 참조 방식의 이해
>     
> - 핵심: 선언부와 호출부의 용어 구분 및 ==항상 값에 의한 호출(Pass-by-Value)==이 발생한다는 원칙 숙지
>     
## 개념 및 용어 구분
- **매개변수 (Parameter)**
    - 메서드 선언부(Signature) 괄호 안에 정의되는 **입력값의 변수명**
    - 메서드 내부에서만 유효한 지역 변수(Local Variable) 형태
    - 메서드 호출 시 메모리 할당 및 종료 시 즉각 소멸 (생명주기)
- **인자 (Argument)**
    - 메서드 호출부에서 파라미터로 실제 **전달되는 구체적인 값** (데이터)
## 데이터 전달 메커니즘 (Pass-by-Value)
- Java의 메서드 호출은 ==항상 값 복사(Pass-by-Value)== 방식만 존재
- **기본형(Primitive Type) 전달**
    - 실제 데이터 값 자체의 복사
    - 메서드 내부에서 값 변경 시 원본 데이터 불변 (안전성 보장)
- **참조형(Reference Type) 전달**
    - 실제 객체가 아닌 ==참조값(메모리 주소)의 복사==
    - 매개변수를 통해 원본 객체의 상태(필드값 등) 변경 가능성 존재 (사이드 이펙트 주의)
```plaintext
Java Argument Passing Logic
 ├─ Primitive Type (기본형)
 │   └─ 실제 '값'의 복사 -> [원본 상태 보존: Safe]
 └─ Reference Type (참조형: 배열, 객체)
     ├─ '메모리 주소값'의 복사
     └─ 동일한 객체 인스턴스 참조 -> [객체 내부 상태 변경 시 원본 영향: Warning]
```
## 매개변수 주요 유형 및 규칙

|**유형**|**데이터 전달 형태**|**특징 및 제약 사항**|
|---|---|---|
|**기본형 (Primitive)**|값 복사|원본 불변 보장, 단순 데이터 전달용|
|**참조형 (Reference)**|주소값 복사|배열, 객체 등 전달, 원본 데이터 제어 가능|
|**가변인자 (Varargs)**|배열 형태의 참조형 반환|동적 개수의 인자 처리, ==반드시 매개변수 목록의 마지막에 위치==|

## 구현 예시
### Parameter vs Argument의 구분
```java
public class User {
    // name: 매개변수(Parameter) - 선언부
    public void printName(String name) { 
        System.out.println("이름: " + name);
    }

    public static void main(String[] args) {
        User user = new User();
        // "초코": 인자(Argument) - 호출부
        user.printName("초코"); 
    }
}
```
### 매개변수 선언 방식(가변인자 포함)
```java
// 1. 매개변수 없음
void emptyMethod() {}

// 2. 기본형 매개변수
void primitiveMethod(int count) {}

// 3. 참조형 매개변수 (배열)
void referenceMethod(int[] numbers) {}

// 4. 가변인자 (String 타입 0개 이상의 인자 허용)
void varargsMethod(int id, String... names) {} // 가변인자는 항상 마지막에 선언
```

# Java 열거형(Enum) 완벽 가이드
> [!summary] 핵심 요약
> 
> - 서로 연관된 상수들의 집합을 깔끔하게 정의하고 관리하는 자바의 특수 클래스
>     
> - 목적: 데이터의 허용 범위를 제한하여 ==컴파일 시점의 타입 안정성(Type Safety)== 확보
>     
> - 핵심: 상태, 권한, 요일 등 고정된 값의 묶음을 다룰 때 오타나 잘못된 값의 유입을 원천 차단
>     
## 1. Enum의 개념과 도입 배경 (Why)
- **기존 상수 관리 방식(`String` 또는 `public static final int`)의 치명적 한계**
    - 타입 검증 불가: 의미가 전혀 다른 상수끼리 비교되거나 연산되는 논리적 오류 발생 위험
    - 런타임 에러 노출: 문자열(`String`)로 상태를 관리할 경우, 오타("vVIP")가 발생해도 실행(Runtime) 중에야 에러가 발견됨
- **Enum이 제공하는 핵심 해결책**
    - ==오류의 조기 발견==: 지정된 Enum 타입 외의 값은 코드 작성 단계(컴파일 시점)에서 즉각 차단
    - 높은 가독성: 코드의 의도가 명확해지고 IDE의 자동 완성 기능을 완벽하게 지원받음
```plaintext
입력값 검증 비교 (회원 등급: BASIC, VIP)
 ├─ String 타입 사용 시
 │   └─ "vVIP" 입력 -> 통과됨(문자열 형태가 맞으므로) -> [시스템 장애 / 런타임 에러 발생]
 │
 └─ Enum 타입 사용 시
     └─ Grade.vVIP 입력 -> 없는 상태값임 -> [IDE에 빨간줄(컴파일 에러)로 즉시 경고]
```
## 2. Enum의 적절한 활용 시기 (When)
- **적용 기준**
    - 도메인 내에서 ==고정된 개수의 연관된 상수==를 정의해야 할 때 필수적으로 적용
    - 데이터의 '분류', '상태', '권한' 등을 표현하는 모든 비즈니스 로직에 활용

|**카테고리**|**활용 예시**|**설명**|
|---|---|---|
|**상태 관리**|`ORDERED`, `SHIPPING`, `DELIVERED`|쇼핑몰의 주문, 배송, 결제 진행 상태|
|**권한/역할**|`GUEST`, `USER`, `ADMIN`|시스템 및 게시판 접근 권한 등급|
|**고정 분류**|`SPRING`, `SUMMER`, `FALL`, `WINTER`|계절, 요일(`MONDAY`), 방위(`NORTH`) 등|
|**단순 플래그**|`YES`, `NO`|데이터베이스에 매핑되는 명확한 긍정/부정 조건|

## 3. Enum의 기본 문법과 활용 (How)
- **선언 및 명명 규칙**
    - `class` 대신 `enum` 키워드를 사용하여 선언
    - 내부의 ==상수명은 대문자 스네이크 표기법(`UPPER_SNAKE`)== 원칙을 엄격하게 준수
```java
// 1. 단순 Enum 선언
public enum Role {
    GUEST, USER, ADMIN
}

public class Member {
    private String name;
    private Role role; // String 타입 대신 Enum 타입 적용

    public void assignRole(Role role) {
        this.role = role; // GUEST, USER, ADMIN 셋 중 하나만 들어올 수 있음
    }
}
```
- **Enum의 핵심 내장 메서드**

|**메서드**|**반환 타입**|**설명**|
|---|---|---|
|`name()`|`String`|Enum 상수의 이름을 문자열 그대로 반환|
|`values()`|`Enum[]`|선언된 모든 Enum 상수를 배열로 담아 반환 (반복문에 유용)|
|`valueOf()`|`Enum`|매개변수로 들어온 문자열과 일치하는 Enum 객체를 찾아 반환|
|`ordinal()`|`int`|선언된 순서(0부터 시작) 반환 (※ ==중간에 값이 추가되면 순서가 꼬이므로 사용 지양==)|

```java
// 2. Enum 내장 메서드 활용 예시
public class EnumExample {
    public static void main(String[] args) {
        
        // 데이터 할당
        Role myRole = Role.ADMIN;

        // 1. name(): 이름 출력
        System.out.println("내 권한: " + myRole.name()); // "ADMIN" 출력

        // 2. values(): 모든 상태 순회
        for (Role r : Role.values()) {
            System.out.println("시스템 허용 권한: " + r.name());
        }

        // 3. valueOf(): 외부 문자열 데이터를 Enum으로 변환 (API 통신 시 자주 사용)
        Role guestRole = Role.valueOf("GUEST");
    }
}
```

### 💡 Note Context
- 수정/개선점: 초심자의 눈높이에 맞춰 기존 문자열 기반 상태 관리의 한계를 텍스트 트리 구조로 도식화하여 '왜 Enum을 써야 하는가'에 대한 직관적 이해도 향상.
- AI 인사이트 (내용 보완 및 연관 키워드 제안): 단순 상수 나열을 넘어, 상수마다 특정 데이터나 문자열("관리자", "일반유저")을 품게 만드는 **'Enum 필드 추가 및 생성자 활용법'**으로의 후속 학습을 권장함. 또한, Enum 내부에 다형성을 적용하는 **'상태 패턴(State Pattern)으로서의 Enum 활용'**은 실무에서 매우 강력한 시너지를 냄.

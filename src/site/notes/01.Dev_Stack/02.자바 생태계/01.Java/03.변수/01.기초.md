# Java 변수와 상수의 핵심 아키텍처
> [!summary] 핵심 요약
> 
> - 변수 및 상수의 정의, 선언 방식, 메모리 저장 원칙에 관한 가이드
>     
> - 목적: 데이터의 유연한 활용 및 ==코드의 안정성(상수 활용)== 확보
>     
> - 핵심: 자료형(Type) 지정, 식별자 명명 규칙 준수, `final` 키워드 활용
>     
## 1. 변수(Variable)와 리터럴(Literal)
- **변수(Variable)**
    - 데이터 저장 및 참조를 위한 메모리 공간의 이름
    - 선언과 동시에 자료형에 따른 메모리 크기 할당
    - ==동일 스코프 내 중복 선언 절대 금지==
- **리터럴(Literal)**
    - 소스 코드 내에 직접 기록된 변하지 않는 값 자체
    - 변수에 대입되는 실제 데이터(정수, 실수, 문자, 불리언 등)
```plaintext
Data Storage Structure
 ├─ Variable: 데이터 보관을 위한 유동적 컨테이너
 │   └─ Identity: 변수명(식별자)
 │   └─ Value: 할당된 리터럴
 └─ Literal: 고정된 값의 형태 (20, "Tom", 3.14)
```
## 2. 상수(Constant)의 정의와 특징
- **개념**
    - 프로그램 실행 중 ==값이 변하지 않는== 고정된 데이터를 담는 식별자
    - `final` 예약어를 사용하여 수정 불가능한 상태로 선언
- **제약 사항**
    - 선언 시 또는 생성자를 통한 단 1회의 초기화만 허용
    - 초기화 이후 값 재할당 시 컴파일 에러 발생

|**구분**|**변수 (Variable)**|**상수 (Constant)**|
|---|---|---|
|**선언 키워드**|자료형 (int, String 등)|`final` + 자료형|
|**값 변경 여부**|자유로운 재할당 가능|==최초 초기화 후 변경 불가==|
|**명명 관례**|카멜 표기법(`camelCase`)|스네이크 대문자 표기(`UPPER_SNAKE`)|
|**주요 용어**|리터럴의 저장소|매직 넘버 방지 및 환경 설정값 저장|

## 3. 데이터 식별 및 명명 규칙
- **식별자 작성 원칙**
    - 숫자로 시작 불가 및 특수문자는 `_`, `$`만 허용
    - 예약어(class, public, int 등) 사용 엄격 금지
    - 의미 있는 단어 조합을 통한 ==가독성 확보 필수==

|**항목**|**상세 규칙**|**비고**|
|---|---|---|
|**변수명**|소문자로 시작하는 `camelCase` 적용|`userAge`, `totalCount`|
|**상수명**|모든 문자를 대문자로 작성, 언더바(`_`) 연결|`MAX_VALUE`, `PI`|
|**클래스명**|대문자로 시작하는 `PascalCase` 적용|`VariableManager`|
|**사용 범위**|선언된 블록(`{ }`) 내에서만 유효|==지역 변수(Local Variable)== 개념|

## 4. 실습 코드 및 예외 처리
- **변수 및 리터럴 할당**
```java
int age = 20;         // age: 변수, 20: 정수 리터럴
String name = "Tom";  // name: 변수, "Tom": 문자열 리터럴
```
- **상수 활용 및 오류 케이스**
```java
final int MAX_SPEED = 120;
// MAX_SPEED = 150;   // ❌ 컴파일 오류: final 변수에 값 재할당 불가

System.out.println(Math.PI); // 자바 표준 라이브러리 내 대표적 상수
```

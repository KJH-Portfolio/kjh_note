# 인터페이스 심화
> [!summary] 핵심 요약
> 
> - Java 8/9 이후 도입된 인터페이스의 확장 기능과 람다식을 활용한 함수형 프로그래밍 기법
>     
> - 목적: 인터페이스의 하위 호환성 유지 및 보일러플레이트 코드를 줄여 가독성 높은 코드 작성
>     
> - 핵심: `default` 및 `private` 메서드의 활용 목적 이해, 함수형 인터페이스(`@FunctionalInterface`)와 람다식 문법 숙달
>     
## 인터페이스의 확장 기능 (Java 8+)
- **도입 배경**: 기존 구현체들의 코드 수정 없이 새로운 공통 기능을 추가하기 위한 하위 호환성 보장 및 캡슐화

| **메서드 타입** | **도입 버전** | **주요 특징 및 역할**                                       | **제약 및 주의사항**                                     |
| ---------- | --------- | ---------------------------------------------------- | ------------------------------------------------- |
| `default`  | Java 8    | 예외적으로 구현부(`{}`)를 가지는 메서드                             | 다중 인터페이스 구현 시 충돌 발생하면 하위 클래스에서 ==반드시 오버라이딩(재정의)== |
| `static`   | Java 8    | 객체 생성 없이 호출 가능한 유틸리티 메서드                             | `인터페이스명.메서드명()`으로 직접 호출                           |
| `private`  | Java 9    | `default`/`static` 메서드 간의 ==중복 코드 제거 및 캡슐화== (내부 헬퍼) | 구현부 필수, 인터페이스 내부에서만 호출 가능 (자식 클래스로 상속 불가)         |

```java
interface Printer {
    void print(String msg);

    // default 메서드 1
    default void printInfo(String msg) {
        logHeader("INFO");
        print(msg);
    }

    // default 메서드 2
    default void printError(String msg) {
        logHeader("ERROR");
        print(msg);
    }

    // private 메서드 (Java 9+): default 메서드들의 중복 로직 캡슐화
    private void logHeader(String level) {
        System.out.println("=== [" + level + "] 출력 시작 ===");
    }
}

class ConsolePrinter implements Printer {
    // 순수 추상 메서드만 필수 오버라이딩
    @Override
    public void print(String msg) {
        System.out.println(msg);
    }
}

public class Main {
    public static void main(String[] args) {
        Printer p = new ConsolePrinter();
        p.printInfo("시스템 정상 작동");  // default 메서드 호출 (내부적으로 private 메서드 실행)
    }
}
```
## 함수형 인터페이스와 람다식 (Functional Interface & Lambda)
- **함수형 인터페이스 (Functional Interface)**
    - 조건: 내부에 ==단 하나의 추상 메서드==만 존재하는 특수 인터페이스 (default/static 메서드 개수 무관)
    - 검증: `@FunctionalInterface` 어노테이션 명시 시 컴파일러가 추상 메서드 단일성 제약 엄격 검증
- **람다식 (Lambda Expression)**
    - 역할: 익명 내부 클래스(Anonymous Inner Class)의 불필요한 보일러플레이트 코드 획기적 제거
    - 구조: `(매개변수) -> { 실행문 }`
    - 특징: 동작(메서드) 자체를 값처럼 취급하여 전달 가능
> [!example] 익명 내부 클래스와 람다식의 구현 비교
> 
> 람다식을 활용하면 인터페이스의 구현 객체 생성을 매우 직관적이고 간결하게 표현할 수 있습니다.
```java
// 1. 함수형 인터페이스 선언 (추상 메서드가 1개뿐이어야 함)
@FunctionalInterface
interface Calculator {
    int calculate(int x, int y);
    // int add(int x, int y); // 주석 해제 시 다중 추상 메서드로 인해 컴파일 에러 발생
}

public class Main {
    public static void main(String[] args) {
        // [방식 A] 기존 익명 내부 클래스 활용: 코드가 길고 가독성 저하
        Calculator calc1 = new Calculator() {
            @Override
            public int calculate(int x, int y) {
                return x + y;
            }
        };

        // [방식 B] 람다식 적용: 인터페이스 구현의 극단적 간소화
        Calculator calc2 = (x, y) -> x + y; // 매개변수 타입과 return 키워드 생략 가능

        System.out.println(calc1.calculate(10, 20)); // 30
        System.out.println(calc2.calculate(10, 20)); // 30
    }
}
```

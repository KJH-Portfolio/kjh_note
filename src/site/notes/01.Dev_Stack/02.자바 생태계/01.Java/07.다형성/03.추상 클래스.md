# 📘 추상 클래스 (Abstract Class)
> [!summary] 핵심 요약
> 
> - 공통 기능 제공과 세부 구현의 강제를 동시에 수행하는 미완성 설계도
>     
> - 목적: 부모 클래스의 공통 로직 공유 및 자식 클래스의 필수 메서드 구현 보장
>     
> - 핵심: ==`abstract`== 키워드 사용 및 다형성(Polymorphism)을 통한 유연한 참조 타입 활용
>     
## 추상 클래스의 구조 및 핵심 개념
- **추상화(Abstraction) 설계 원리**
    - 여러 클래스의 공통된 특성(필드, 메서드) 추출 및 상위 클래스로의 통합
    - 팀 개발 환경에서의 설계 일관성 유지 및 필수 기능 누락 방지

|**구분**|**구조 및 세부 특징**|
|---|---|
|**정의**|하나 이상의 추상 메서드를 포함하는 클래스 (단독 객체 생성 불가)|
|**역할**|부모의 기본 로직 제공 및 자식의 특화된 세부 동작 구현 유도|
|**선언 형식**|`abstract class 클래스명 { }` 형태|
|**추상 메서드**|`abstract` 키워드 명시 및 ==구현부(Body) 부재== (`{ }` 없음)|
|**일반 멤버**|일반 인스턴스 변수 및 일반 메서드의 자유로운 포함|

> [!example] 예시 1: 추상 클래스의 기본 선언과 구조
> 
> - 일반 메서드(공통 기능)와 추상 메서드(구현 강제)의 혼용 형태
>     
```java
public abstract class Shape {
    // 1. 일반 메서드: 모든 자식이 공유하는 기본 기능
    public void info() {
        System.out.println("도형입니다.");
    }
    
    // 2. 추상 메서드: 자식마다 그려지는 방식이 다르므로 구현을 강제
    public abstract void draw(); 
}
```
> [!example] 예시 2: 상속과 오버라이딩을 통한 세부 구현
> 
> - 부모 생성자 호출(`super()`) 및 추상 메서드의 구체화
>     
```java
abstract class Animal {
    String name;
    
    Animal(String name) { 
        this.name = name; 
    }

    public void eat() {
        System.out.println(name + "이(가) 밥을 먹습니다.");
    }
    
    // 자식 클래스에서 반드시 구현해야 할 추상 메서드
    public abstract void sound();
}

class Dog extends Animal {
    Dog(String name) { 
        super(name); 
    }
    
    @Override
    public void sound() { 
        System.out.println("멍멍!"); 
    }
}

class Cat extends Animal {
    Cat(String name) { 
        super(name); 
    }
    
    @Override
    public void sound() { 
        System.out.println("야옹!"); 
    }
}
```
## ⚖️ 추상 클래스 vs 인터페이스 비교
- **설계 목적에 따른 적절한 도구 선택**
    - 공통 기능의 '상속'이 중심인지, 동작의 '계약'이 중심인지에 따른 구분

|**구분**|**추상 클래스 (Abstract Class)**|**인터페이스 (Interface)**|
|---|---|---|
|**상속/구현**|단일 상속만 허용 (`extends`)|다중 구현 허용 (`implements`)|
|**메서드 구성**|추상 메서드 + 일반 메서드 혼용|==순수 추상 메서드 위주== (`default`/`static` 예외)|
|**상태(필드)**|일반 변수 및 상수 모두 선언 가능|상수(`public static final`)만 선언 가능|
|**설계 목적**|공통 코드의 공유 및 부분적 추상화|순수 기능 계약의 정의 및 다형성 극대화|

## 설계 시 주의점 및 제약 사항
> [!warning] 컴파일 에러 유발 주의 사항
> 
> - **인스턴스화 불가**: `new` 키워드를 통한 ==직접적인 객체 생성 절대 불가== (반드시 상속 후 자식 객체로 생성)
>     
> - **구현의 강제성**: 자식 클래스는 부모의 모든 추상 메서드를 ==반드시 오버라이딩(재정의)== 해야 함 (미구현 시 자식도 추상 클래스가 됨)
>     
> - **단일 상속 제약**: 일반 클래스와 동일하게 다중 상속 불가
>     

## 템플릿 메서드 패턴 (Template Method Pattern) 활용
- **패턴 설계 원리**
    - 알고리즘의 전체 실행 흐름(뼈대)은 부모 클래스에서 정의하고, 세부 구현은 자식에게 위임하는 설계 기법
    - `final` 키워드와 추상 메서드의 전략적 조합 활용
- **구현 방식 및 효과**
    - **템플릿 메서드 (`final`)**: 오버라이딩을 금지하여 ==실행 순서와 핵심 흐름의 임의 변경 방지==
    - **추상 메서드 (`abstract`)**: 자식 클래스에서 반드시 구현해야 할 ==가변적인 세부 동작 강제==
    - **일반 메서드 (Hook)**: 필요에 따라 자식 클래스에서 선택적으로 재정의할 수 있는 기본 동작 제공
> [!example] 예시 4: 템플릿 메서드 패턴의 실제 적용
> 
> - `final` 메서드로 흐름을 고정하고, `abstract` 메서드로 세부 구현을 강제하는 구조
>     
```java
abstract class Car {
    // 자식 클래스에서 반드시 구현해야 하는 세부 동작
    public abstract void drive();
    public abstract void stop();

    // 선택적으로 오버라이딩 가능한 Hook 메서드
    public void turnOnRadio() {
        System.out.println("라디오를 켭니다.");
    }

    // 템플릿 메서드: 전체 실행 흐름 고정 (오버라이딩 절대 불가)
    public final void run() {
        System.out.println("--- 시동을 켭니다 ---");
        drive();          // 자식의 구현부 호출
        turnOnRadio();    // 기본 또는 재정의된 동작 호출
        stop();           // 자식의 구현부 호출
        System.out.println("--- 시동을 끕니다 ---");
    }
}

class AICar extends Car {
    @Override
    public void drive() { 
        System.out.println("자율 주행 시스템이 조향합니다."); 
    }
    
    @Override
    public void stop() { 
        System.out.println("장애물 감지 후 스스로 정지합니다."); 
    }
    
    @Override
    public void turnOnRadio() { 
        System.out.println("인공지능이 추천 음악을 틉니다."); 
    }
}

public class TemplateMethodTest {
    public static void main(String[] args) {
        Car myCar = new AICar();
        // 부모의 템플릿 메서드 실행 -> 정해진 시나리오대로 자식 메서드 자동 호출
        myCar.run(); 
    }
}
```

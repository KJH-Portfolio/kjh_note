# 📘 인터페이스 (Interface)
> [!summary] 핵심 요약
> 
> - 구현 로직이 아닌 객체의 역할과 동작에 대한 ==계약(Contract)==을 정의하는 추상적 타입
>     
> - 목적: 기능의 표준화, 다중 구현을 통한 다형성 극대화 및 클래스 간 결합도 감소
>     
> - 핵심: `implements` 키워드 활용, 순수 추상 메서드 구현 강제 및 람다식을 통한 함수형 프로그래밍 지원
>     
## 인터페이스의 기본 구조와 특징
- **기본 설계 원칙**
    - 순수하게 기능의 명세서 역할만 수행하며 상위 참조 타입으로 활용
    - 일반 클래스의 단일 상속(`extends`) 한계를 ==다중 구현(`implements`)==으로 극복
    - 협업 시 개발자 간의 명확한 역할 분담 및 구현체의 유연한 교체 지원

|**구분**|**세부 스펙 및 특징**|
|---|---|
|**상태 (변수)**|묵시적으로 ==`public static final`== 적용 (상수만 선언 가능)|
|**행위 (메서드)**|묵시적으로 ==`public abstract`== 적용 (구현부 없음)|
|**인스턴스화**|`new` 키워드를 통한 ==객체 직접 생성 절대 불가==|
|**관계 확장**|인터페이스 간 확장은 `extends` 사용 (인터페이스 간 다중 상속 허용)|

```java
public interface Drawable {
    // public static final 생략 가능
    int WIDTH = 100;   
    
    // public abstract 생략 가능
    void draw();       
}

class Circle implements Drawable {
    @Override
    public void draw() {
        System.out.println("원을 그립니다.");
    }
}
```

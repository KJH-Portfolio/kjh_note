# 객체지향 인터페이스 (Interface)
> [!summary] 핵심 요약
> 
> - **개념**: 클래스가 구현해야 할 동작(메서드)의 ==규약(Contract)==을 정의하는 순수한 추상화 타입
>     
> - **왜?**: 객체 간 결합도를 낮추고, 여러 구현체가 동일한 동작 방식을 보장하도록 기능의 표준화를 강제하기 위함
>     
> - **언제?**: 서로 연관성이 없는 클래스들에 공통적인 행위를 부여하거나, 다중 구현을 통해 다형성을 극대화해야 할 때
>     
> - **핵심**: `implements` 키워드를 통한 다중 구현 지원 및 모던 자바(Java 8+)의 `default` 메서드를 활용한 유연한 설계 확장
>     
## 인터페이스 기본 명세 및 구조
- **기본 설계 원칙**
    - 객체의 고유한 상태(인스턴스 변수)를 가지지 않고, 외부로 노출할 행위(메서드)만을 명세
    - 일반 클래스의 단일 상속(`extends`) 한계를 ==다중 구현(`implements`)== 메커니즘으로 우회 및 극복
    - 협업 시 개발자 간의 명확한 역할 분담 및 구현체의 유연한 교체(Loose Coupling) 지원

|**구성 요소**|**세부 스펙 및 제약 사항**|**비고**|
|---|---|---|
|**상태 (필드)**|묵시적으로 `public static final` 적용|상수만 선언 가능 (객체 상태 보관 불가)|
|**행위 (메서드)**|묵시적으로 `public abstract` 적용|자식 클래스에서 반드시 오버라이딩 필수|
|**인스턴스화**|`new` 키워드를 통한 ==객체 직접 생성 절대 불가==|반드시 구현체(클래스) 생성 후 업캐스팅 활용|
|**디폴트/정적 메서드**|`default` 및 `static` 키워드 지원 (Java 8+)|인터페이스 내부에 예외적으로 구현부 작성 가능|

## 다중 구현 및 확장 원리
- **인터페이스 간의 상속과 확장**
    - 인터페이스가 다른 인터페이스의 규약을 이어받을 때는 `extends` 키워드 사용
    - 일반 클래스와 달리 인터페이스 간에는 ==다중 상속 허용== (메서드 시그니처만 상속받으므로 충돌 위험이 없음)
- **어떻게?**
    - 여러 인터페이스를 쉼표(`,`)로 구분하여 하나의 클래스에서 동시 구현 가능
```plaintext
[ 인터페이스 다중 구현 및 상속 계층도 ]

  인터페이스 A           인터페이스 B
  (메서드 a)             (메서드 b)
     │                     │
     └─> 다중 상속(extends) ─┘
             ▼
        인터페이스 C
      (메서드 a, b, c)
             │
             └─> 구현(implements)
                     ▼
                구현체 클래스
         (메서드 a, b, c 모두 구현 필수)
```
```java
public interface Drawable {
    // 1. public static final 묵시적 적용 (상수)
    int MAX_WIDTH = 100;   
    
    // 2. public abstract 묵시적 적용 (추상 메서드)
    void draw();       
}

public interface Resizable {
    void resize(int ratio);
}

// 3. 다중 구현 (여러 인터페이스의 규약을 동시에 넘겨받음)
class Shape implements Drawable, Resizable {
    @Override
    public void draw() {
        System.out.println("도형을 그립니다. 최대 너비: " + MAX_WIDTH);
    }

    @Override
    public void resize(int ratio) {
        System.out.println(ratio + "% 비율로 크기를 조정합니다.");
    }
}
```

# 📘 다형성 (Polymorphism)
> [!summary] 핵심 요약
> 
> - 부모 타입의 참조 변수로 여러 자식 객체를 다루는 객체지향의 핵심 성질
>     
> - 목적: 객체 간 결합도를 낮추고 코드의 재사용성 및 유연성 확보
>     
> - 핵심: **업캐스팅(Upcasting)**과 **동적 바인딩(Dynamic Binding)** 메커니즘 숙달
>     
## 다형성의 핵심 메커니즘
- **바인딩 (Binding) 규칙**
    - 메서드: ==동적 바인딩(Dynamic Binding)== 적용 (런타임에 실제 생성된 객체 타입 기준 실행)
    - 필드: ==정적 바인딩(Static Binding)== 적용 (컴파일 타임에 참조 변수 타입 기준 접근)
- **형변환 (Casting) 비교**

|**구분**|**업캐스팅 (Upcasting)**|**다운캐스팅 (Downcasting)**|
|---|---|---|
|**방향**|자식 $\rightarrow$ 부모|부모 $\rightarrow$ 자식|
|**캐스팅 방식**|자동 (암시적)|==명시적 (강제)==|
|**안전성**|항상 안전|런타임 에러(`ClassCastException`) 발생 가능|
|**목적**|다형성 실현, 객체 통합 관리|자식 객체의 고유 멤버(메서드/필드) 접근|

- **객체 생성 및 멤버 접근 결과**

|**코드**|**참조 변수 타입**|**실제 생성 객체**|**메서드 실행 (동적 바인딩)**|**필드 접근 (정적 바인딩)**|
|---|---|---|---|---|
|`Parent p = new Child();`|부모|자식|자식의 오버라이드된 메서드|부모의 필드|
|`Child c = new Parent();`|자식|부모|❌ 컴파일 에러|❌ 컴파일 에러|

```java
// 업캐스팅과 바인딩 규칙 예시
class Animal {
    String name = "Animal";
    void sound() { System.out.println("동물 소리"); }
}

class Dog extends Animal {
    String breed = "Retriever";
    
    @Override
    void sound() { System.out.println("멍멍!"); }
    void bark() { System.out.println("짖는다!"); }
}

public class Main {
    public static void main(String[] args) {
        Animal animal = new Dog();  // 업캐스팅
        
        animal.sound();             // "멍멍!" (동적 바인딩: 실제 객체인 Dog의 메서드)
        System.out.println(animal.name); // "Animal" (정적 바인딩: 참조 타입인 Animal의 필드)
        
        // animal.bark();           // ❌ 컴파일 에러 (부모 타입 참조 변수로는 자식 고유 메서드 호출 불가)
    }
}
```
## 다형성 활용 패턴
- **다형적 참조와 배열 활용**
    - 부모 타입 배열 하나로 다양한 자식 객체 동시 관리 가능
    - 새로운 자식 클래스가 추가되어도 기존 반복문 코드 수정 불필요 (확장성 보장)
```java
// 배열을 활용한 객체 통합 관리 예시
class Employee {
    String name;
    Employee(String name) { this.name = name; }
    void introduce() { System.out.println("직원: " + name); }
}

class Manager extends Employee {
    Manager(String name) { super(name); }
    @Override
    void introduce() { System.out.println("매니저: " + name); }
}

class Intern extends Employee {
    Intern(String name) { super(name); }
    @Override
    void introduce() { System.out.println("인턴: " + name); }
}

public class Company {
    public static void main(String[] args) {
        // 부모 타입 배열로 자식 객체들을 한 번에 관리
        Employee[] list = {
            new Manager("Alice"),
            new Intern("Bob"),
            new Employee("Charlie")
        };
        
        for (Employee e : list) {
            e.introduce(); // 각 객체의 오버라이드된 메서드가 알아서 실행됨 (동적 바인딩)
        }
    }
}
```
- **안전한 다운캐스팅과 `instanceof`**
    - 부모 타입으로 업캐스팅된 객체에서 자식의 고유 메서드를 호출해야 할 때 사용
    - 다운캐스팅 전 반드시 `instanceof` 연산자로 실제 객체 타입 검증 필수 (예외 방지)
    - **Pattern Matching for `instanceof` (Java 16+)**
        - 타입 확인과 동시에 형변환된 변수 선언 가능
        - 불필요한 ==명시적 캐스팅(Boilerplate) 제거== 및 코드 가독성 향상
        - 선언된 변수는 조건문이 `true`인 블록 내부에서만 유효 (안전한 스코프 보장)
```java
// 다운캐스팅과 instanceof 활용 예시
Animal animal = new Dog();  // 업캐스팅

// [전통적인 방식] 타입 확인 후 명시적 다운캐스팅 필요
if (animal instanceof Dog) {
    Dog dog = (Dog) animal;  // 강제 캐스팅
    dog.bark();              // Dog 고유 메서드 호출
}

// [Java 16 이후] Pattern Matching 활용 (권장)
if (animal instanceof Dog dog) {
    dog.bark();              // 명시적 캐스팅 없이 즉시 사용 가능
}
```
- **상속과 구현의 결합**
    - `extends`로 핵심 속성/기본 동작을 상속받고, `implements`로 특정 부가 능력(기능 계약) 부여
    - 다중 다형성 적용 가능
```java
// 클래스 단일 상속과 인터페이스 다중 구현의 결합 예시
class Animal {
    void eat() { System.out.println("먹는다"); }
}

interface Pet { 
    void play(); 
}

class Dog extends Animal implements Pet {
    @Override
    public void play() { 
        System.out.println("공놀이"); 
    }
}
```

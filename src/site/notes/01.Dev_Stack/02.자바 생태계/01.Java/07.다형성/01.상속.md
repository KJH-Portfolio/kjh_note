# 객체지향 상속 (Inheritance)
> [!summary] 핵심 요약
> 
> - **개념**: 상위(부모) 클래스의 필드와 메서드를 하위(자식) 클래스가 물려받아 재사용 및 확장하는 기법
>     
> - **왜?**: 코드 중복을 제거하고 공통 로직을 단일화하여 유지보수성을 높이고 다형성(Polymorphism)을 실현하기 위함
>     
> - **언제?**: 객체 간에 명확한 계층적 관계(is-a)가 존재하고, 기존 코드를 기반으로 새로운 기능을 확장할 때
>     
> - **핵심**: `extends` 키워드를 통한 기능 상속 및 부모 메서드의 ==오버라이딩(Overriding)==
>     
## 상속의 구조 및 기본 원리
- **기본 동작 방식**
    - 코드 중복 제거 및 재사용성 확장을 위한 계층적 구조화
    - 부모 클래스의 변경 사항에 대한 자식 클래스의 자동 반영

|**구분**|**세부 특징**|
|---|---|
|**문법 형식**|`public class 자식클래스 extends 부모클래스 { }` 형태|
|**접근 제어**|부모의 `public`, `protected` 멤버만 접근 가능 (==`private` 멤버 상속 불가==)|
|**생성자 처리**|생성자 및 초기화 블록 상속 불가 (자식 객체에서 `super()`로 명시적/묵시적 호출)|
|**확장성**|자식 클래스 내 새로운 필드 및 메서드의 자유로운 추가|

## 설계 제약 사항 및 필수 규칙
- **구조적 제약**
    - 자바 클래스의 ==다중 상속 전면 금지== (단일 상속만 허용, 다중 상속 필요 시 인터페이스 활용)
    - `final` 클래스 상속 불가 및 `final` 메서드 오버라이딩 불가
- **부모 생성자 호출 (`super()`)**
    - 자식 객체 생성 전 부모 객체의 선행 초기화 필수
    - 명시하지 않을 경우 컴파일러가 기본 생성자 `super()` 자동 삽입
    - 명시적 호출 시 반드시 ==자식 생성자의 첫 번째 줄==에 위치
- **메서드 오버라이딩 (Overriding)**
    - 상속받은 부모 메서드의 동작을 자식 클래스 목적에 맞게 재정의
    - 컴파일러 문법 검증을 위한 `@Override` 어노테이션 명시 권장
## 다형성 구현 및 동작 원리
- **업캐스팅(Upcasting)과 동적 바인딩(Dynamic Binding)**
    - 부모 타입의 참조 변수로 자식 객체를 제어하는 유연한 객체 관리
    - 런타임에 실제 할당된 객체의 재정의된 메서드 실행
```plaintext
[ 상속 객체 생성 흐름 및 메모리 구조 ]

Stack 영역              Heap 영역
[ 참조 변수 a ]  ---->  [ Dog 인스턴스 ]
                        ├── [ Animal (부모) 부분 ] : name 데이터, super() 호출로 생성
                        └── [ Dog (자식) 부분 ]    : num 데이터, sound() 오버라이딩
```
```java
class Animal {
    String name;
    
    Animal(String name) {
        this.name = name;
    }
    
    void sound() { 
        System.out.println("동물 소리"); 
    }
}

class Dog extends Animal {
    int num;
    
    Dog(String name, int num) {
        super(name);   // 부모 생성자 선행 호출 (반드시 첫 줄에 위치)
        this.num = num;
    }
    
    @Override
    void sound() { 
        System.out.println("멍멍!"); 
    }
}

public class Main {
    public static void main(String[] args) {
        // 1. 다형성을 활용한 업캐스팅(Upcasting)
        Animal a = new Dog("바둑이", 1); 

        // 2. 동적 바인딩을 통한 재정의 메서드 호출
        a.sound(); // 출력: "멍멍!" (런타임에 실제 할당된 Dog 객체의 메서드 실행)
    }
}
```
> [!warning] 오버로딩(Overloading)과 오버라이딩(Overriding)의 명확한 구분
> 
> 자주 혼동되는 두 다형성 기법은 목적과 동작 방식이 완전히 다름.
> 
> - **오버로딩**: 매개변수 시그니처 차이를 이용해 동일 이름의 메서드를 ==새로 추가(New)==하는 기법
>     
> - **오버라이딩**: 상속 구조에서 기존 부모 메서드의 내부 로직을 자식에 맞게 ==재정의(Modify)==하는 기법
>     
```java
class Calculator {
    // [오버로딩] 매개변수의 타입이나 개수가 다름
    int add(int a, int b) { return a + b; }          
    double add(double a, double b) { return a + b; } 
}

class Child extends Parent {
    // [오버라이딩] 상속받은 greet() 메서드의 내용만 변경
    @Override
    void greet() { System.out.println("Hi"); } 
}
```

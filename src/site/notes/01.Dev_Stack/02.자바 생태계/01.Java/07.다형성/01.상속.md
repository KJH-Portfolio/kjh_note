# 📘 객체지향 상속 (Inheritance)
> [!summary] 핵심 요약
> 
> - 기존 클래스의 속성과 기능을 하위 클래스가 물려받아 재사용 및 확장하는 객체지향 핵심 개념
>     
> - 목적: 공통 로직 단일화를 통한 유지보수성 향상 및 다형성(Polymorphism) 실현
>     
> - 핵심: `extends` 키워드를 통한 기능 확장 및 부모 메서드의 ==오버라이딩(Overriding)==
>     
## 상속의 구조 및 핵심 개념
- **상속 기본 원리**
    - 코드 중복 제거 및 재사용성 확장을 위한 계층적 구조화
    - 부모 클래스의 변경 사항에 대한 자식 클래스의 자동 반영

| **구분**     | **구성 요소 및 세부 특징**                                            |
| ---------- | ------------------------------------------------------------ |
| **정의**     | 상위(부모) 클래스의 필드와 메서드를 하위(자식) 클래스가 물려받아 사용하는 기법                |
| **문법 형식**  | `public class 자식클래스 extends 부모클래스 { }` 형태                    |
| **접근 제어**  | 부모의 `public`, `protected` 멤버만 접근 가능 (==`private` 멤버 상속 불가==) |
| **생성자 처리** | 생성자 및 초기화 블록의 상속 불가 (자식 객체에서 `super()`를 통한 명시적/묵시적 호출 필수)    |
| **확장성**    | 자식 클래스 내 새로운 필드 및 메서드의 자유로운 추가                               |

## 상속 설계 시 제약 사항 및 필수 규칙
- **구조적 제약**
    - 자바 클래스의 ==다중 상속 전면 금지== (단일 상속만 허용, 필요 시 인터페이스로 보완)
    - `final` 클래스의 상속 및 `final` 메서드의 오버라이딩 불가
- **부모 생성자 호출 (`super()`)**
    - 호출하지 않을 시 묵시적으로 들어감
    - 자식 객체 생성 전 부모 객체의 선행 초기화
    - `super()` 메서드 호출 시 반드시 ==자식 생성자의 첫 번째 줄== 위치
- **메서드 오버라이딩 (Overriding)**
    - 상속받은 부모 메서드의 동작을 자식 클래스 목적에 맞게 재정의
    - 컴파일러 문법 검증을 위한 `@Override` 어노테이션 명시 권장
## 다형성 구현 및 동작 예시
> [!example] 상속 관계의 객체 생성 및 생성자 호출
> 
> - `super()`를 활용한 부모 객체 초기화 및 자식 클래스 확장 구조
>     
```java
class Animal {
    String name;
    
    Animal(String name) {
        this.name = name;
    }
    
    void sound() { 
        System.out.println("동물 소리"); 
    }
}

class Dog extends Animal {
    int num;
    
    Dog(String name, int num) {
        super(name);   // ==부모 생성자 선행 호출 (반드시 첫 줄)==
        this.num = num;
    }
    
    @Override
    void sound() { 
        System.out.println("멍멍!"); 
    }
}
```
> [!tip] 다형성(Polymorphism)과 동적 바인딩(Dynamic Binding)
> 
> - 부모 타입의 참조 변수로 자식 객체를 제어하는 유연한 객체 관리
>     
```java
// 1. 다형성을 활용한 업캐스팅(Upcasting)
Animal a = new Dog("바둑이", 1); 

// 2. 동적 바인딩을 통한 재정의 메서드 호출
a.sound(); // 출력: "멍멍!" (런타임에 실제 할당된 Dog 객체의 메서드 실행)
```
> [!warning] 오버로딩(Overloading)과의 명확한 구분
> 
> - 자주 혼동되는 두 다형성 기법의 목적 차이 비교
>     
```java
class Calculator {
    // [오버로딩] 매개변수 시그니처 차이를 이용한 '동일 이름의 새로운 메서드' 추가
    int add(int a, int b) { return a + b; }          
    double add(double a, double b) { return a + b; } 
}

class Child extends Parent {
    // [오버라이딩] 상속 구조에서 기존 부모 메서드의 내부 로직을 '재정의'
    @Override
    void greet() { System.out.println("Hi"); } 
}
```

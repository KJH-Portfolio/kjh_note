# 📘 Singleton 패턴 정리 (개선 버전)

## Summary 📝

> [!summary] Summary 📝
> 
> - Singleton 패턴은 **인스턴스를 하나만 유지**하고 전역적으로 공유해야 할 때 사용
> - 구현 방식: **Eager**, **Lazy**, **Thread-Safe(Double-Checked Locking)**, **Bill Pugh(권장)**
> - ✅ 장점: 메모리 절약, 일관성 유지
> - ⚠️ 단점: 테스트 어려움, 의존성 증가, 멀티스레드 환경 주의 필요

---

## 목적 🎯

- **전역적으로 하나의 객체**만 필요할 때 사용
- **자원 낭비 방지**: 불필요한 객체 생성 차단
- **상태 공유**: 동일한 인스턴스를 여러 곳에서 활용
- **일관성 유지**: 시스템 전역에서 동일한 동작 보장

---

## 핵심 개념 💡

| 구분     | 설명                                      |
| ------ | --------------------------------------- |
| 정의     | 클래스의 인스턴스를 **하나만 생성**하고 어디서든 접근 가능하게 보장 |
| 생성자    | `private`으로 선언해 외부에서 `new` 불가           |
| 접근 방식  | `getInstance()` 정적 메서드로 전역 접근 지점 제공     |
| 인스턴스 수 | 단 하나만 유지                                |

## 구현 방식 📌

### 1. 기본 Singleton (Eager Initialization)

```java
public class Singleton {
    // 클래스 로딩 시점에 인스턴스 생성
    private static final Singleton instance = new Singleton();

    private Singleton() {} // 외부 생성 차단

    public static Singleton getInstance() {
        return instance;
    }
}
```

- **장점**: 구현 간단, thread-safe
- **단점**: 사용하지 않아도 인스턴스가 생성됨 (자원 낭비 가능)

---

### 2. Lazy Initialization (필요할 때 생성)

```java
public class Singleton {
    private static Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) { // 최초 호출 시 생성
            instance = new Singleton();
        }
        return instance;
    }
}
```

- **장점**: 필요할 때만 생성 → 자원 절약
- **단점**: 멀티스레드 환경에서 동기화 문제 발생 가능

---

### 3. Thread-Safe Singleton (Double-Checked Locking)

```java
public class Singleton {
    private static volatile Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) { // 첫 번째 체크
            synchronized (Singleton.class) {
                if (instance == null) { // 두 번째 체크
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

- **장점**: 멀티스레드 환경에서도 안전
- **단점**: 코드 복잡도 증가

---

### 4. Bill Pugh Singleton (권장 방식)

```java
public class Singleton {
    private Singleton() {}

    // 내부 정적 클래스 사용
    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}
```

- **장점**: Lazy Initialization + Thread-Safe + 코드 간결
- **실무에서 가장 많이 사용되는 방식**

# Java 배열 복사 (Array Copy)
> [!summary] 핵심 요약
> 
> - **개념**: 자바에서 배열 데이터를 다른 배열로 복제할 때 적용되는 얕은 복사(Shallow Copy)와 깊은 복사(Deep Copy) 메커니즘
>     
> - **왜?**: 원본 배열과 복사된 배열 간의 참조 공유로 인한 사이드 이펙트(Side Effect)를 방지하고, 데이터의 독립성을 확보하기 위함
>     
> - **언제?**: 원본 데이터를 보존한 상태로 정렬, 필터링, 데이터 가공 등의 조작이 필요한 자바 애플리케이션 계층 전반
>     
> - **핵심**: 원시 타입(Primitive) 1차원 배열은 기본 API로 깊은 복사가 가능하지만, 다차원 및 객체 배열은 기본 API 사용 시 ==최상위 껍데기만 복사==되므로 직접 내부 요소를 순회하며 복사해야 함
>     
## 얕은 복사와 깊은 복사 (Shallow vs Deep)
- **메모리 할당 및 참조 방식 비교**

|**구분**|**메모리 구조**|**데이터 독립성**|**핵심 특징**|
|---|---|---|---|
|**얕은 복사 (Shallow)**|동일한 Heap 메모리 주소 참조|종속적 (원본 변경 시 복사본 동기화)|단순 대입 연산자(`=`)를 통해 객체의 ==메모리 주소값만 복사==|
|**깊은 복사 (Deep)**|Heap 메모리에 새로운 공간 할당|완전 독립 (원본과 복사본 무관)|실제 데이터 값을 새로운 메모리 공간에 복제하여 ==완전한 개별 인스턴스 생성==|

## 깊은 복사 지원 API 및 구현 패턴
- **주요 복사 메서드 스펙 및 권장 사항**

|**API (메서드)**|**파라미터 구조**|**특징 및 권장 상황**|
|---|---|---|
|`clone()`|매개변수 없음|1차원 원시 타입 배열을 가장 간편하게 전체 복사할 때 사용|
|`System.arraycopy()`|`(원본, 시작점, 대상, 시작점, 길이)`|OS 레벨 네이티브 구현으로 ==대용량 배열 복사 시 최고 성능== 보장|
|`Arrays.copyOf()`|`(원본 배열, 복사할 길이)`|처음부터 지정된 길이만큼 복사 (배열 크기 확장/축소 시 유용)|
|`Arrays.copyOfRange()`|`(원본 배열, 시작 인덱스, 끝 인덱스)`|배열의 특정 구간 데이터만 추출하여 복사할 때 사용|

- **1차원 원시 타입(Primitive Type) 배열 복사 예시**
    - **어떻게?**: 제공되는 API를 활용하여 배열의 목적과 성능 요구사항에 맞게 복사 로직 적용
```java
int[] original = {1, 2, 3};

// [얕은 복사] 주소값만 공유 (original 변경 시 shallowCopy도 변경됨)
int[] shallowCopy = original;

// [깊은 복사 1] 간편한 전체 복사
int[] deepCopyClone = original.clone();

// [깊은 복사 2] 배열 크기를 확장하며 복사 (크기 5인 새 배열에 복사, 남은 공간은 0)
int[] deepCopyArrays = Arrays.copyOf(original, 5);

// [깊은 복사 3] 성능 최적화 및 세밀한 위치 제어 (가장 빠름)
int[] deepCopySys = new int[3];
// 구조: (원본배열, 원본 시작위치, 대상배열, 대상 시작위치, 복사할 개수)
System.arraycopy(original, 0, deepCopySys, 0, original.length);
```
## 다차원 및 객체 배열 복사의 한계와 해결책
- **참조 타입(Reference Type) 복사 시 발생하는 문제점**
    - 다차원 배열이나 객체 배열에 `clone()`, `copyOf()` 등 기본 API 적용 시, 요소가 품고 있는 '하위 주소값' 자체만 복사됨
    - 즉, 최상위 껍데기는 새로 생성되지만, 내부의 실제 객체나 서브 배열은 ==여전히 동일한 메모리를 참조하는 얕은 복사 상태==로 남음
- **해결책 (완전한 깊은 복사)**
    - 반복문(for-loop)이나 Stream API를 사용하여 배열 내부를 순회하며 ==각 요소를 명시적으로 새로 생성(`new`)==하여 할당해야 함
```plaintext
[ 다차원 / 객체 배열 복사 참조 구조 ]

 1. 기본 API (clone 등) 사용 시 (불완전한 복사)
    [원본 배열] ----> [서브 배열 A (0x100)] <---- [복사본 배열]
                     [서브 배열 B (0x200)] <---- (껍데기만 다르고 속은 같은 주소 공유)

 2. 요소 순회 개별 복사 시 (완전한 깊은 복사)
    [원본 배열] ----> [원본 서브 배열 A (0x100)]
    [복사본 배열] ---> [신규 서브 배열 A (0x300)] (메모리가 완벽히 분리됨)
```
- **객체 배열의 완전한 깊은 복사 예시**
    - **어떻게?**: Stream API의 `map()`을 활용해 각 요소의 복사 생성자를 호출하여 완전한 새 객체 반환
```java
class Student {
    String name;
    Student(String name){ this.name = name; }
}

Student[] original = { new Student("홍길동"), new Student("김철수") };

// [주의] original.clone() 사용 시 Student 객체 자체는 얕은 복사됨

// [해결] 완전한 깊은 복사 (Stream 활용)
Student[] deepCopy = Arrays.stream(original)
                           // 핵심: 각 요소마다 명시적으로 새로운 인스턴스 할당
                           .map(Student::new) 
                           .toArray(Student[]::new);
```
- **2차원(다차원) 배열의 완전한 깊은 복사 예시**
    - **어떻게?**: 반복문으로 각 행(1차원 배열)에 접근하여 개별적으로 `clone()` 수행
```java
int[][] original2D = {{1, 2}, {3, 4}};

// 1. for문을 활용한 2차원 배열의 완전한 깊은 복사
int[][] deepCopyFor = new int[original2D.length][];
for (int i = 0; i < original2D.length; i++) {
    // 2차원 배열의 요소인 1차원 배열(원시 타입) 각각을 깊은 복사
    deepCopyFor[i] = original2D[i].clone(); 
}

// 2. Stream API를 활용한 방식 (Java 8+)
int[][] deepCopyStream = Arrays.stream(original2D)
                               .map(int[]::clone)
                               .toArray(int[][]::new);
```

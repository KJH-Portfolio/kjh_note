# Java 객체 배열 (Object Array)
> [!summary] 핵심 요약
> 
> - **개념**: 자바에서 참조형 데이터(객체)를 그룹화하여 관리하기 위한 고정 크기의 자료구조
>     
> - **왜?**: 동일한 타입의 다수 객체를 인덱스 기반으로 일괄 제어하고 메모리를 효율적으로 관리하기 위함
>     
> - **언제?**: 동적 크기 변환이 필요 없는 다수의 인스턴스를 다루는 자바 애플리케이션 계층 전반 (컬렉션 프레임워크 도입 전 단계)
>     
> - **핵심**: 배열 공간 할당 시 ==각 요소는 `null`로 초기화==되므로, 사용 전 반드시 개별 객체의 인스턴스화(`new`) 필요
>     
## 객체 배열의 핵심 특징
- **참조형 데이터 보관**
    - 실제 데이터 값이 아닌 객체가 위치한 ==메모리 주소(참조값)== 저장
- **배열 크기의 고정성**
    - 최초 선언 시 지정된 배열의 물리적 크기 변경 불가
    - **어떻게?**: 데이터 추가/삭제에 따른 동적 크기 조절이 필요한 경우 `ArrayList` 등 컬렉션 프레임워크로 대체
- **예외 발생 위험성**
    - 인덱스 공간만 확보된 상태(null)에서 객체 메서드 접근 시 ==`NullPointerException` 발생==
- **메모리 회수(GC) 연계**
    - 불필요해진 배열 인덱스에 명시적으로 `null`을 할당하여 가비지 컬렉터의 메모리 회수 유도
## 기본형 배열과 객체 배열의 구조 비교

| **구분**      | **기본형 배열 (Primitive Array)** | **객체 배열 (Object Array)**                       |
| ----------- | ---------------------------- | ---------------------------------------------- |
| **저장 데이터**  | 실제 값 (`int`, `double` 등)     | 객체의 ==참조값(메모리 주소)==                            |
| **초기화 기본값** | `0`, `0.0`, `false` 등        | `null`                                         |
| **생성 및 선언** | `int[] arr = new int[5];`    | `Student[] arr = new Student[5];`              |
| **메모리 할당**  | 힙(Heap) 영역에 실제 값을 직접 할당      | 힙(Heap)에 주소 배열 생성 후,<br>실제 객체는 별도의 힙(Heap)에 생성 |

```plaintext
[ Object Array Memory Structure ]

  스택(Stack) 영역                   힙(Heap) 영역
____________________            _____________________________________
|                    |          | [배열 객체] Student[] (크기: 3)       |
| 변수 arr (0x100)    |────┐     |  ├─ index 0: 0x200 (Student 1)    |
|____________________|    │     |  ├─ index 1: 0x300 (Student 2)    |
                          └────▶|  └─ index 2: null (미초기화)        |
                                |                                   |
                                | [실제 인스턴스]                      |
                                |  ├─ [0x200] Student("홍길동", 20)  |
                                |  └─ [0x300] Student("김철수", 22)  |
                                |___________________________________|
```
## 객체 배열 초기화 및 활용 패턴
- **선언과 개별 인덱스 할당**
    - 배열 공간을 먼저 확보한 뒤, 필요한 시점에 각 인덱스에 명시적으로 객체 주입
```java
// 1. 3개의 요소를 담을 수 있는 빈 배열 생성 (모두 null 상태)
Student[] students = new Student[3]; 

// 2. 개별 인덱스에 실제 객체 생성 및 주소 할당
students[0] = new Student("홍길동", 20);
students[1] = new Student("김철수", 22);
// 주의: students[2]는 여전히 null 상태임
```
- **배열 리터럴을 통한 동시 초기화**
    - 선언과 동시에 중괄호 `{}` 내부에서 다수의 객체를 즉시 생성하여 묶음 할당
```java
Student[] students = new Student[] {
    new Student("홍길동", 20),
    new Student("김철수", 22)
};
```
- **반복문 기반의 객체 제어 및 안전성 검증**
    - 배열 데이터 순회 시, 초기화되지 않은 인덱스 접근에 의한 애플리케이션 크래시를 막기 위해 ==Null 체크 필수== 적용
    - **어떻게?**: 조건문을 통해 해당 인덱스가 `null`이 아닐 때만 객체 내부 데이터 접근
```java
for (Student s : students) {
    // 안정성 확보: NullPointerException 사전 차단
    if (s != null) { 
        System.out.println(s.getName() + " - " + s.getAge());
    }
}
```

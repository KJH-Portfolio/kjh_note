# Java 객체 배열 (Object Array)
> [!summary] 핵심 요약
> 
> - 자바에서 참조형 데이터(객체)를 그룹화하여 관리하기 위한 고정 크기의 자료구조
>     
> - 목적: 동일한 타입의 다수 객체를 인덱스 기반으로 일괄 제어 및 메모리 관리
>     
> - 핵심: 배열 생성 시 ==각 요소는 `null`로 초기화==되며, 반드시 개별 객체 생성(`new`) 필요
>     
## 객체 배열의 핵심 특징
- **참조형 데이터 보관**
    - 실제 값이 아닌 객체의 ==메모리 주소== 저장
- **배열 크기의 고정성**
    - 선언 시 지정된 크기 변경 불가 (동적 크기 필요 시 `ArrayList` 등 컬렉션 프레임워크 활용)
- **예외 발생 위험성**
    - `new`를 통한 객체 인스턴스화 없이 메서드 접근 시 `NullPointerException` 발생
- **가비지 컬렉션(GC) 연계**
    - 불필요한 객체 배열 인덱스에 `null` 할당을 통한 메모리 회수 유도
- **String 클래스의 예외적 허용**
    - 일반 클래스와 달리 `String` 배열은 리터럴을 통한 직접 초기화 가능
## 기본 배열과 객체 배열의 구조 비교

|**구분**|**기본형 배열 (Primitive Array)**|**객체 배열 (Object Array)**|
|---|---|---|
|**저장 데이터**|실제 값 (int, double 등)|객체의 ==참조값(메모리 주소)==|
|**초기화 기본값**|`0`, `0.0`, `false` 등|`null`|
|**생성 및 선언 방식**|`int[] arr = new int[5];`|`Student[] arr = new Student[5];`|
|**메모리 할당 구조**|힙(Heap) 영역에 값 직접 할당|힙(Heap) 영역에 참조 배열 생성 후, 개별 객체 별도 할당|

```plaintext
Object Array Memory Structure
 ├─ 1. Stack Area
 │   └─ 변수 (Heap 영역의 배열 시작 주소 보관)
 └─ 2. Heap Area
     ├─ [배열 객체] Student[] (크기 지정)
     │   ├─ index 0: Student 객체 1의 주소 (ex: 0x100)
     │   ├─ index 1: Student 객체 2의 주소 (ex: 0x200)
     │   └─ index 2: null (미초기화 상태)
     │
     └─ [실제 객체] (개별 new 선언 시 생성)
         ├─ Student 객체 1 (0x100 위치)
         └─ Student 객체 2 (0x200 위치)
```
## 객체 배열 초기화 및 활용 패턴
- **선언과 개별 인덱스 할당**
    - 배열 공간 확보 후 각 인덱스에 명시적 객체 주입
```java
    Student[] students = new Student[3]; // [null, null, null]
    students[0] = new Student("홍길동", 20);
    students[1] = new Student("김철수", 22);
    // students[2]는 여전히 null
```
- **배열 리터럴을 통한 동시 초기화**
    - 선언과 동시에 다수의 객체를 생성하여 배열 묶음으로 할당
```java
    Student[] students = new Student[] {
        new Student("홍길동", 20),
        new Student("김철수", 22)
    };
```
- **반복문 기반의 객체 제어 및 안전성 검증**
    - 데이터 순회 시 ==Null 체크 필수== (미초기화 인덱스 접근에 따른 크래시 방지)
```java
    for (Student s : students) {
        if (s != null) { // NullPointerException 사전 차단
            System.out.println(s.getName() + " - " + s.getAge());
        }
    }
```

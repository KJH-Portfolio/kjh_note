# 📘 Java 배열(Array) 아키텍처
> [!summary] 핵심 요약
> 
> - 동일한 타입의 데이터를 연속된 메모리 공간에 저장하는 고정 크기의 참조형 자료구조
>     
> - 목적: 인덱스를 통한 데이터의 빠른 접근 및 반복문을 활용한 대량 데이터 관리 효율성 극대화
>     
> - 핵심: 배열 생성 시 크기 ==변경 절대 불가(고정 길이)== 원칙 및 `Arrays.toString()`을 활용한 올바른 데이터 출력 방식 숙지
>     
## 배열 핵심 메커니즘
- **자료구조 특성**: 단일 변수에 동일 타입의 데이터를 다수 저장 및 관리
- **메모리 구조**: 실제 데이터는 Heap 영역에 저장되며, 배열 변수는 Stack 영역에서 해당 객체의 ==참조값(메모리 주소)== 보유
- **인덱스 체계**: `0`부터 시작하는 제로 베이스(Zero-based) 인덱싱 채택
- **크기 제약**: 배열 생성 시 크기 지정 필수 및 이후 ==크기 변경 불가== (동적 크기 할당 필요 시 `ArrayList` 사용 권장)
## 메모리 할당 및 기본값 초기화
- **자동 초기화 메커니즘**: `new` 키워드를 통한 배열 객체 생성 시, 할당된 메모리 공간은 자료형에 따라 자동 초기화 진행

|**자료형 분류**|**구체적 자료형**|**초기화 기본값**|
|---|---|---|
|**정수형**|`byte`, `short`, `int`, `long`|`0` (long은 `0L`)|
|**실수형**|`float`, `double`|`0.0` (float는 `0.0f`)|
|**논리형**|`boolean`|`false`|
|**참조형**|`String`, 객체, 배열 등|`null`|

## 배열 선언 및 제어 문법
- **권장 선언 방식**: 타입명 뒤에 대괄호 표기 (`int[] arr` 형태 권장, `int arr[]` 방식은 가독성 저하로 지양)
- **메모리 참조 출력**: 일반 출력문(`System.out.println(arr)`) 사용 시 실제 값이 아닌 객체의 Heap 영역 메모리 주소값 출력 (단, `char[]` 타입은 문자열 형태로 연속 출력되는 예외 존재)
- **데이터 확인 유틸리티**: `java.util.Arrays` 클래스 메서드 활용 필수
    - 1차원 배열 값 확인: `Arrays.toString(배열명)`
    - 다차원 배열 값 확인: `Arrays.deepToString(배열명)`
    - & java/lang/[[3. Resource/01.Dev_Stack/02.자바 생태계/01.Java/08.API/01.java.lang/02.Array\|02.Array]] 함수 사용
```plaintext
Array Initialization & Printing Process
 ├─ 1. Declaration & Instantiation
 │   ├─ int[] arr = new int[3];  -> [결과: 기본값 0으로 자동 초기화]
 │   └─ int[] arr = {1, 2, 3};   -> [결과: 선언과 동시에 명시적 값 할당]

```
## 주요 예외 및 주의 사항
- **인덱스 범위 초과**: 유효하지 않은 인덱스(음수이거나 배열 길이 이상의 값) 접근 시 ==`ArrayIndexOutOfBoundsException`== 런타임 에러 발생 주의
- **다차원 배열 관리**: 2차원 이상의 배열(`int[][]`)은 '배열의 배열' 구조를 가지며, 각 행마다 다른 길이의 배열을 참조하는 가변 배열(Ragged Array) 형태로도 생성 가능

# Java 배열(Array) 아키텍처
> [!summary] 핵심 요약
> 
> - **개념**: 동일한 타입의 데이터를 연속된 메모리 공간에 저장하는 고정 크기의 참조형 자료구조
>     
> - **왜?**: 인덱스를 통한 빠른 데이터 접근 및 반복 처리를 활용해 대량의 데이터 관리 효율성 확보
>     
> - **언제?**: 데이터의 개수가 변하지 않는 정적 데이터 집합 관리, 행렬 연산, 캐싱 처리 등
>     
> - **핵심**: 배열 생성 시 ==크기 변경 절대 불가(고정 길이)== 원칙 준수 및 실제 데이터 출력 시 `Arrays.toString()` 유틸리티 활용
>     
## 배열의 메모리 구조 및 특성
- **자료구조 특성**: 단일 변수에 동일 타입의 데이터를 다수 저장 및 순차적으로 관리
- **메모리 매핑**: 실제 데이터는 Heap 영역에 저장되며, 배열 변수는 Stack 영역에서 해당 객체의 ==참조값(메모리 주소)== 보유
- **인덱스 체계**: `0`부터 시작하는 제로 베이스(Zero-based) 인덱싱 채택
- **크기 불변성**: 최초 생성 시 할당된 메모리 공간 크기는 변경 불가 (동적 크기 할당 필요 시 `ArrayList` 객체 활용 권장)
- **어떻게?**: 타입명 뒤에 대괄호를 표기하여 선언 (`int[] arr` 형태 권장, `int arr[]` 방식은 가독성 저하로 지양)
## 배열 생성 및 자동 초기화
- **초기화 메커니즘**: `new` 키워드를 통한 배열 객체 생성 시, 할당된 메모리 공간은 선언된 자료형에 따라 자동 초기화 진행
- **명시적 할당**: 선언과 동시에 중괄호 `{}`를 사용하여 원하는 데이터로 즉시 초기화 가능

|**자료형 분류**|**구체적 자료형**|**초기화 기본값**|
|---|---|---|
|**정수형**|`byte`, `short`, `int`, `long`|`0` (long은 `0L`)|
|**실수형**|`float`, `double`|`0.0` (float는 `0.0f`)|
|**논리형**|`boolean`|`false`|
|**참조형**|`String`, 객체, 배열 등|`null`|

## 데이터 제어 및 주의사항
- **메모리 참조 출력**: 일반 출력문(`System.out.println(arr)`) 사용 시 실제 값이 아닌 객체의 Heap 영역 메모리 주소값 반환 (단, `char[]` 타입은 내부 문자가 연속 출력되는 예외 존재)
- **데이터 확인 유틸리티**: 배열 내부 데이터 확인 시 `java.util.Arrays` 클래스 메서드 ==반드시 활용==
    - 1차원 배열 값 확인: `Arrays.toString(배열명)`
    - 다차원 배열 값 확인: `Arrays.deepToString(배열명)`
- **인덱스 범위 에러**: 유효하지 않은 인덱스(음수 또는 배열 길이 이상의 값) 접근 시 ==`ArrayIndexOutOfBoundsException`== 런타임 에러 발생 주의
- **다차원 배열 구조**: 2차원 이상의 배열(`int[][]`)은 '배열의 배열' 구조로 동작하며, 각 행마다 다른 길이의 하위 배열을 참조하는 가변 배열(Ragged Array) 형태로 구성 가능
```plaintext
[ Java 배열 초기화 및 출력 메모리 흐름 ]

1. 배열 선언 및 인스턴스화
 ├─ int[] arr = new int[3];  -> [결과: Heap 영역에 공간 할당 후 0으로 기본값 초기화]
 └─ int[] arr = {1, 2, 3};   -> [결과: 공간 할당과 동시에 1, 2, 3 명시적 값 셋팅]

2. 배열 데이터 출력 (Output Handling)
 ├─ System.out.println(arr);
 │   └─ [출력: "[I@15db9742" -> 배열 객체의 메모리(Heap) 참조 주소값 반환]
 │
 └─ System.out.println(Arrays.toString(arr));
     └─ [출력: "[1, 2, 3]" -> 내부 요소를 문자열로 변환하여 실제 데이터 반환]
```

# ThreadLocal (Java 동시성 제어: 스레드 전용 로컬 변수)
> [!summary] 핵심 요약
> 
> - **개념**: 멀티스레드 환경에서 각 스레드마다 독립적으로 할당되는 전용 변수 저장소 (스레드 전용 비밀 금고)
>     
> - **왜?**: 동기화(Lock) 과정 없이 스레드 안전성(Thread-Safety)을 확보하고, 복잡한 파라미터 전달 없이 데이터를 전역적으로 공유하기 위함
>     
> - **언제?**: 사용자 인증 정보 보관(Spring Security), 트랜잭션 DB 커넥션 유지, 사용자 요청 추적(MDC) 등 스레드 단위의 데이터 상태 유지가 필요할 때
>     
> - **핵심**: 스레드 풀(Thread Pool) 환경에서는 작업 완료 후 ==반드시 `remove()`를 호출하여 자원을 반납==해야 함
>     
## ThreadLocal 핵심 문법과 메모리 관리
- **객체 생성 및 안전한 초기화**
    - 단순 생성 시 초기값 부재로 인한 `NullPointerException` 발생 위험 존재
    - Java 8부터 지원하는 `ThreadLocal.withInitial()` 팩토리 메서드를 사용하여 기본값(Default Value) 세팅 권장
- **데이터의 은닉과 활용**
    - `set()` 메서드를 통해 현재 실행 중인 스레드의 독립된 공간에만 안전하게 데이터 저장
    - 메서드 매개변수(Parameter)로 넘기지 않아도, 동일한 스레드 내의 어떤 메서드 계층에서든 `get()`으로 즉시 조회 가능
- **스레드 풀(Thread Pool) 환경의 치명적 부작용 및 주의사항**
    - 웹 애플리케이션(Tomcat 등)은 성능 극대화를 위해 스레드를 매번 생성하지 않고 풀에서 꺼내 재사용함
    - 작업 종료 시 `remove()`로 초기화하지 않으면, ==이전 사용자의 데이터가 동일한 스레드를 배정받은 타인에게 노출==되는 보안 사고 발생
    - 가비지 컬렉터(GC)가 만료된 데이터를 회수하지 못해 점진적인 메모리 누수(Memory Leak) 유발
```java
public class ThreadLocalExample {
    // [권장 패턴] withInitial()을 사용하여 초기값("GUEST")을 지정해 Null 방지
    private static final ThreadLocal<String> userContext = 
        ThreadLocal.withInitial(() -> "GUEST");

    public void processRequest(String userName) {
        try {
            // 1. 현재 스레드만의 독립된 공간에 사용자 이름 저장
            userContext.set(userName);
            
            // 2. 파라미터로 넘기지 않아도 하위 비즈니스 로직에서 즉시 조회 가능
            doBusinessLogic();
            
        } finally {
            // [치명적 주의] 3. 작업이 끝나면 무조건 finally 블록에서 삭제하여 메모리 누수 및 데이터 오염 방지
            userContext.remove(); 
        }
    }

    private void doBusinessLogic() {
        // processRequest()에서 세팅한 값을 동일한 스레드이므로 그대로 꺼내 씀
        String currentUser = userContext.get();
        System.out.println("비즈니스 로직 실행 중 - 현재 작업자: " + currentUser);
    }
}
```

|**메서드**|**동작 원리 및 역할**|**반환 타입 등 기타 속성**|
|---|---|---|
|`set(T value)`|현재 스레드의 전용 공간에 값 저장|`void`|
|`get()`|현재 스레드의 전용 공간에서 값 조회|값이 없으면 `null` 반환 (단, 초기값이 세팅된 경우 해당 값 반환)|
|`remove()`|현재 스레드 공간에 저장된 값 완전 삭제|`void` / **사용 종료 시 필수 호출**|
|`withInitial()`|변수 생성과 동시에 람다식으로 초기값 세팅|`ThreadLocal<T>` / `null` 방어용 팩토리 메서드|

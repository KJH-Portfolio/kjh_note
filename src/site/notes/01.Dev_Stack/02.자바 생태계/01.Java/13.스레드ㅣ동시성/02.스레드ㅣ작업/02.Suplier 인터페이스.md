# Java Supplier 인터페이스 핵심 가이드
> [!summary] 핵심 요약
> 
> - **개념**: 매개변수를 받지 않고, 특정 타입의 결과값을 반환하는 자바의 함수형 인터페이스입니다.
>     
> - **왜?**: 값이 실제로 필요한 시점까지 객체 생성이나 연산을 미루는 **지연 연산(Lazy Evaluation)**을 구현하기 위함입니다.
>     
> - **언제?**: 무거운 객체의 초기화 지연, 비동기 작업(`CompletableFuture`), `Optional.orElseGet()` 등에서 주로 활용됩니다.
>     
> - **핵심**: 입력값 없이 오직 출력값만 제공하며, 반드시 **`get()` 메서드**를 통해 최종적으로 값을 꺼내야 합니다.
>     
---
## 1. 기본 개념 및 동작 원리
- **핵심 특징 및 구조**
    - `java.util.function` 패키지에 포함된 표준 함수형 인터페이스입니다.
    - 매개변수가 없고 반환값만 존재하는 `T get()` 추상 메서드 단 하나만 가집니다.
    - 주로 데이터를 생성(Produce)하거나 공급하는 역할을 수행합니다.
```plaintext
[ Supplier 동작 흐름 ]

   (호출자)               (Supplier 객체)               (결과물)
  [Client] ─── get() ──▶ [ () -> T ] ──── return ─▶ [Type T]

  ※ 핵심 요약:
  1. 입력 : 텅 비어있음 (No Arguments)
  2. 출력 : 지정된 제네릭 타입 T 반환
```
- **어떻게? (기본 사용법)**
    - 람다식(Lambda)을 활용하여 구현부를 정의합니다.
    - 값의 반환이 필요한 로직을 블록(`{}`) 내부에 작성하거나 단일 식 형태로 작성합니다.
```java
import java.util.function.Supplier;

public class SupplierExample {
    public static void main(String[] args) {
        // 1. 단순 문자열을 반환하는 Supplier
        Supplier<String> stringSupplier = () -> "Hello, Supplier!";
        System.out.println(stringSupplier.get()); 

        // 2. 팩토리(Factory) 패턴: 호출될 때마다 새로운 난수 생성
        Supplier<Double> randomSupplier = () -> Math.random();
        System.out.println("난수: " + randomSupplier.get());

        // 3. 복잡한 로직이 포함된 Supplier (명시적 return 필요)
        Supplier<String> timeSupplier = () -> {
            long currentTime = System.currentTimeMillis();
            return "현재 시간(ms): " + currentTime;
        };
        System.out.println(timeSupplier.get());
    }
}
```

## 2. 실무 활용: 지연 연산 (Lazy Evaluation)
- **지연 연산의 필요성**
    - 값이 즉시 필요하지 않은 상황에서 무거운 연산을 미리 수행하는 것을 방지합니다.
    - 호출되는 시점(`get()`)에만 연산이 실행되므로 불필요한 리소스 낭비를 막을 수 있습니다.
- **예시: `Optional`에서의 활용**
    - `orElse()`는 데이터 유무와 상관없이 매개변수 연산을 수행하지만, `orElseGet(Supplier)`은 데이터가 **null일 때만** 연산을 수행합니다.
```java
import java.util.Optional;

public class LazyEvaluationExample {
    public static void main(String[] args) {
        String data = "Existing Data";

        // [Bad] orElse: data가 null이 아니어도 heavyOperation()이 무조건 실행됨
        String result1 = Optional.ofNullable(data)
                                 .orElse(heavyOperation());

        // [Good] orElseGet: data가 null일 때만 heavyOperation()이 실행됨
        String result2 = Optional.ofNullable(data)
                                 .orElseGet(() -> heavyOperation());
    }

    private static String heavyOperation() {
        System.out.println("무거운 작업 실행 중...");
        return "New Data";
    }
}
```

## 3. 스레드 및 비동기 프로그래밍에서의 활용
- **비동기 작업 공급자 역할**
    - 별도의 스레드에서 무거운 연산을 처리하고 그 결과를 반환받아야 할 때 사용됩니다.
    - `CompletableFuture.supplyAsync()`는 인자로 Supplier를 받아 백그라운드 스레드에서 작업을 수행합니다.
```plaintext
[ CompletableFuture와 Supplier 실행 흐름 ]

  메인 스레드 (Main Thread)              백그라운드 스레드 (Worker)
 ┌───────────────────────────┐          ┌──────────────────────────────────┐
 │                           │          │                                  │
 │ supplyAsync(Supplier) 호출│────위임──▶│ Supplier.get() 로직 실행        │
 │ (즉시 다음 코드로 넘어감) │          │ (DB 조회 등 무거운 작업)         │
 │                           │          │                                  │
 │ 다른 작업 수행            │          │                 │                │
 │                           │          │                 ▼                │
 │ future.join() 결과 수신   │◀──반환───│ 작업 완료 후 결과값 리턴         │
 └───────────────────────────┘          └──────────────────────────────────┘
```

## 4. 자바 주요 함수형 인터페이스 비교

|**인터페이스**|**파라미터**|**반환 타입**|**핵심 메서드**|**주 사용 목적**|
|---|---|---|---|---|
|**`Supplier<T>`**|**없음**|**`T`**|**`get()`**|**데이터 생성 및 공급, 지연 연산**|
|`Consumer<T>`|`T`|`void`|`accept()`|데이터 소비 (출력, 상태 변경 등)|
|`Function<T, R>`|`T`|`R`|`apply()`|데이터 변환 (T -> R)|
|`Predicate<T>`|`T`|`boolean`|`test()`|조건 검사 및 필터링|

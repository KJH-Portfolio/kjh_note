# Hashtable의 통잠금 구조와 성능 한계 (Java 동시성 제어 1단계)
> [!summary] 핵심 요약
> 
> - **개념**: 자바 초기 버전(1.0)부터 존재한 1세대 동기화 컬렉션으로, 모든 메서드에 `synchronized` 키워드가 적용된 Map 구현체
>     
> - **왜?**: 멀티 스레드 환경에서 여러 스레드가 동시에 접근할 때 발생하는 데이터 오염을 막기 위함
>     
> - **언제?**: 레거시 시스템을 유지보수할 때만 예외적으로 사용 (현대 자바 애플리케이션에서는 ==절대 신규 도입하지 않음==)
>     
> - **핵심**: 안전성은 보장되지만, 하나의 스레드가 객체를 점유하면 ==다른 모든 스레드가 대기해야 하는 치명적 병목(통잠금) 발생==
>     
## Hashtable의 동기화 방식과 동작 원리
- **메서드 레벨 동기화 (Method-Level Synchronization)**
    - 내부의 모든 주요 연산(`put()`, `get()`, `remove()` 등)에 `synchronized` 키워드를 부여하여 구현함
    - 메서드 호출 시 대상 객체(Instance) 전체에 락(Lock)을 걸어 한 번에 단 하나의 스레드만 진입을 허용함
- **어떻게? (구현 및 한계점)**
    - 스레드 A가 단순 조회를 위해 `get()`을 호출하는 동안에도 Hashtable 객체 전체가 잠금 처리됨
    - 스레드 B가 완전히 다른 데이터(다른 Key)를 삽입하거나 조회하려 해도, 스레드 A의 락이 풀릴 때까지 ==무한 대기(Block) 상태==로 전환됨
    - 결과적으로 멀티 코어 CPU의 이점을 전혀 살리지 못하고 직렬화(Serialization)되어 처리되므로 극심한 성능 저하 유발
```java
// Hashtable 내부 구조 (Java 소스 코드 일부 발췌 및 단순화)
public class Hashtable<K,V> extends Dictionary<K,V> implements Map<K,V> {
    
    // 1. 객체 단위 락(Lock): 메서드에 진입하는 순간 객체 전체가 잠김
    public synchronized V put(K key, V value) {
        // null 값을 엄격히 제한함 (NullPointerException 발생)
        if (value == null) {
            throw new NullPointerException();
        }
        // ... 데이터 삽입 로직 ...
        return null;
    }

    // 2. 읽기 작업(get) 시에도 락이 강제됨 (동시 읽기 불가)
    public synchronized V get(Object key) {
        // ... 데이터 조회 로직 ...
        return null;
    }
}
```
```plaintext
[ Hashtable 병목 현상 (무식한 통잠금) ]

  스레드 A (단순 읽기 요청)
    │
    ▼ (Lock 획득)
  ┌────────────────────────────────────────┐
  │  Hashtable 인스턴스 [전체 LOCKED]      │
  │  ├─ [Key: A, Val: 10] (스레드 A 접근)  │
  │  ├─ [Key: B, Val: 20]                  │
  │  └─ [Key: C, Val: 30]                  │
  └────────────────────────────────────────┘
    ▲ (Lock 반환 시까지 철저히 진입 차단)
    │
    ├─ 스레드 B (Key: B 읽기 대기) ──▶ [BLOCKED]
    ├─ 스레드 C (Key: C 쓰기 대기) ──▶ [BLOCKED]
    └─ 스레드 D (Key: D 삭제 대기) ──▶ [BLOCKED]
    
  ※ 문제점: 서로 다른 데이터를 건드리려 하거나, 데이터 변경이 없는 
     단순 조회만 하려는 스레드들까지 전부 멈춰서 기다려야 함.
```
## Hashtable 주요 특성 및 복잡도

| **항목**          | **동작 원리 및 역할**             | **반환 타입 등 기타 속성**                   |
| --------------- | -------------------------- | ----------------------------------- |
| **동기화 범위**      | 객체 전체                      | 한 스레드가 메서드 점유 시 타 스레드 접근 불가         |
| **Thread-Safe** | 지원함 (`true`)               | 멀티 스레드 환경에서 데이터 무결성은 보장됨            |
| **Null 허용 여부**  | Key, Value 모두 `null` 불허    | 삽입 시도시 즉시 `NullPointerException` 발생 |
| **발전형**         | `ConcurrentHashMap`을 사용해야함 | Java 5 이후 실무 표준 동기화 컬렉션             |

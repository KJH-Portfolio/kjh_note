# 📘 Java 네트워크 프로그래밍 이론 정리 
## Summary 📝

> [!summary] Summary 📝
> 
> - 네트워크는 여러 컴퓨터와 통신 장치가 연결되어 데이터를 주고받는 구조
> - Java는 `java.net` 패키지를 통해 네트워크 프로그래밍을 지원
> - 서버(Server)는 서비스를 제공, 클라이언트(Client)는 서비스를 요청 및 이용
> - 통신은 **IP 주소 + Port 번호**로 식별, **소켓(Socket)**을 통해 데이터 송수신
> - TCP는 연결 지향적이며 신뢰성이 높고, UDP는 비연결 지향적이며 속도가 빠름
> - 서버-클라이언트 구조에서 스트림을 통해 메시지를 주고받으며, `flush()`로 즉시 전송 보장

---

## 목적 🎯

- 네트워크의 기본 개념(IP, Port, MAC, 프로토콜) 이해
- Java에서 서버/클라이언트 소켓 프로그래밍 흐름 학습
- 스트림을 통한 데이터 송수신 과정 이해
- TCP 기반 채팅 프로그램 구조 실습

---

## 핵심 개념 💡

### ✅ 네트워크 기본

- 네트워크: 여러 컴퓨터/통신 장치가 연결된 구조
- 목적: 데이터를 손쉽게 주고받기 위함
- 서버(Server): 서비스 제공, 요청 수락 및 응답
- 클라이언트(Client): 서비스 요청 및 이용

### ✅ 주소 체계

|구분|설명|
|---|---|
|IP 주소|네트워크 상에서 컴퓨터를 식별하는 고유 번호 (중복 불가)|
|Port 번호|같은 컴퓨터 내에서 프로그램을 구분하는 번호 (중복 불가)|
|MAC 주소|네트워크 장치의 물리적 고유 번호 (변하지 않음)|
|IP + Port|클라이언트가 서버에 접속하기 위해 필요한 정보|

### ✅ 소켓과 프로토콜

- 소켓(Socket): 프로세스 간 통신의 양 끝단
- TCP: 연결 지향, 신뢰성 보장, 속도 느림
- UDP: 비연결 지향, 신뢰성 낮음, 속도 빠름
- 프로토콜: 통신 방법에 대한 약속

## 특징 ✨

### ✅ 주요 클래스와 메서드

- `InetAddress`: IP, 도메인 정보 조회
- `ServerSocket`: 서버 소켓 생성, 클라이언트 접속 대기
- `Socket`: 클라이언트와 서버 간 연결 통로
- `BufferedReader`, `PrintWriter`: 문자 기반 보조 스트림, 한 줄 단위 송수신 지원
- `flush()`: 버퍼에 남은 데이터 즉시 전송

### ✅ 서버 연결 순서 (ServerRun)

1. 서버 소켓 생성 (`ServerSocket`)
2. 클라이언트 접속 요청 대기 (`accept()`)
3. 요청 수락 후 소켓 생성 (`Socket`)
4. 클라이언트 정보 저장
5. 출력 스트림 생성 (`PrintWriter`)
6. 입력 스트림 생성 (`BufferedReader`)
7. 스트림을 통한 데이터 송수신
8. 연결 종료 (`close()`)

### ✅ 클라이언트 연결 순서 (ClientRun)

1. 서버 IP와 Port로 소켓 생성 (`Socket`)
2. I/O 스트림 준비 (`InputStreamReader`, `PrintWriter`)
3. 보조 스트림으로 변환 (`BufferedReader`)
4. 데이터 송수신 (`readLine()`, `println()`)
5. 통신 종료 (`close()`)

---

## 팁과 주의점 ⚠️

- 포트 충돌: 이미 사용 중인 포트는 `BindException` 발생 → 다른 포트 사용 또는 프로세스 종료 필요
- 자원 해제: `close()` 반드시 호출해야 포트 점유 해제
- flush(): 실시간 통신에서는 반드시 호출해야 데이터가 즉시 전송됨
- 무한 루프: `while(true)` 사용 시 종료 조건 필요 (예: "exit" 입력 시 break)

## 예시 📌

### ✅ flush()의 필요성

- 출력 스트림(`PrintWriter`)은 내부적으로 버퍼 사용
- `println()` 호출 시 버퍼에만 저장될 수 있음
- `flush()` 호출 시 버퍼에 남은 데이터를 즉시 전송

```java
pw.print("Hello");   // 버퍼에만 저장됨
pw.flush();          // 즉시 전송
```

**비유**

- `print()`만 하면 편지를 책상 위에 올려둔 상태
- `flush()`를 해야 우체통에 넣어 상대방에게 전달됨

---

### ✅ 실행 흐름도

**서버(ServerRun)**

```
[ServerSocket 생성] → [accept() 대기] → [클라이언트 연결 수락] 
→ [입출력 스트림 연결] → [메시지 송수신 반복] → [flush()로 즉시 전송] → [close()]
```

**클라이언트(ClientRun)**

```
[Socket 생성(ip, port)] → [스트림 연결] → [메시지 송신 pw.println()] 
→ [flush()로 전송] → [서버 응답 수신 br.readLine()] → [반복] → [close()]
```

---
### ✅ 서버 실행 흐름
### 1. 준비 단계

- **입력 장치 준비**
    
    ```java
    Scanner sc = new Scanner(System.in);
    ```
    
    → 서버에서 직접 메시지를 입력해 클라이언트에게 보낼 수 있도록 `Scanner` 준비.
    
- **보조 스트림 선언**
    
    ```java
    PrintWriter pw = null;
    BufferedReader br = null;
    ```
    
    → 클라이언트와의 데이터 송수신을 위해 입출력 스트림 객체를 미리 선언.
    
- **포트 번호 지정**
    
    ```java
    int port = 3000;
    ```
    
    → 서버가 열릴 포트 번호를 지정. (다른 프로그램과 충돌하지 않도록 3000번대 사용)
    
- **소켓 객체 선언**
    
    ```java
    ServerSocket server = null; 
    Socket socket = null;
    ```
    
    → 서버용 소켓(`ServerSocket`)과 클라이언트와 연결될 소켓(`Socket`) 준비.
    

---

### 2. 서버 소켓 생성 및 클라이언트 연결 대기

```java
server = new ServerSocket(port);
System.out.println("클라이언트 접속 대기중...");
socket = server.accept();
System.out.println(socket.getInetAddress()+" 접속 수락");
```

- `ServerSocket`을 생성해 지정된 포트에서 클라이언트 요청을 기다림.
- `accept()` 호출 시 클라이언트가 접속할 때까지 블로킹 상태.
- 접속이 수락되면 클라이언트의 IP 주소 출력.

---

### 3. 스트림 연결

- **입력 스트림 (클라이언트 → 서버)**
    
    ```java
    br = new BufferedReader(new InputStreamReader(socket.getInputStream()));
    ```
    
    → 클라이언트가 보낸 메시지를 읽기 위해 `BufferedReader` 사용.
    
- **출력 스트림 (서버 → 클라이언트)**
    
    ```java
    pw = new PrintWriter(socket.getOutputStream());
    ```
    
    → 서버에서 클라이언트로 메시지를 보낼 때 사용.
    

---

### 4. 메시지 송수신 처리 (무한 루프)

```java
while(true) {
    // 클라이언트 메시지 수신
    String message = br.readLine();
    System.out.println("클라이언트가 보낸 메시지 : " + message);

    // 서버에서 메시지 입력 후 전송
    System.out.println("클라이언트에게 보낼 메시지 : ");
    String sendMsg = sc.nextLine();
    pw.println(sendMsg);
    pw.flush(); // 버퍼 비우고 즉시 전송
}
```

- 클라이언트가 보낸 메시지를 읽어 출력.
- 서버 사용자가 입력한 메시지를 클라이언트로 전송.
- `flush()`로 버퍼에 남은 데이터를 강제로 내보냄.

---

### 5. 종료 처리 (finally 블록)

```java
pw.close();
br.close();
socket.close();
server.close();
sc.close();
```

- 모든 스트림과 소켓, 스캐너를 닫아 자원 누수 방지.


### ✅ 클라이언트 실행 흐름 
### 1. 준비 단계

- **사용자 입력 장치 준비**
    
    ```java
    Scanner sc = new Scanner(System.in);
    ```
    
    → 클라이언트에서 서버로 보낼 메시지를 입력받기 위해 `Scanner` 사용.
    
- **보조 스트림 선언**
    
    ```java
    PrintWriter pw = null;
    BufferedReader br = null;
    ```
    
    → 서버와의 데이터 송수신을 위해 입출력 스트림 객체 선언.
    
- **서버 접속 정보 준비**
    
    ```java
    String ip = "192.168.30.194";
    int port = 3000;
    ```
    
    → 접속할 서버의 IP 주소와 포트 번호 지정.
    
- **소켓 선언**
    
    ```java
    Socket socket = null;
    ```
    
    → 서버와 연결할 클라이언트 소켓 준비.
    

---

### 2. 서버 연결

```java
socket = new Socket(ip, port);
System.out.println("서버와 연결 성공");
```

- 지정된 IP와 포트 번호로 서버에 접속 시도.
- 연결 성공 시 메시지 출력.

---

### 3. 스트림 연결

- **입력 스트림 (서버 → 클라이언트)**
    
    ```java
    br = new BufferedReader(new InputStreamReader(socket.getInputStream()));
    ```
    
    → 서버가 보낸 메시지를 읽기 위해 `BufferedReader` 사용.
    
- **출력 스트림 (클라이언트 → 서버)**
    
    ```java
    pw = new PrintWriter(socket.getOutputStream());
    ```
    
    → 클라이언트에서 서버로 메시지를 보낼 때 사용.
    

---

### 4. 메시지 송수신 처리 (무한 루프)

```java
while(true) {
    // 서버로 메시지 전송
    System.out.println("서버에 보낼 메시지 : ");
    String sendMsg = sc.nextLine();
    pw.println(sendMsg);
    pw.flush();

    // 서버로부터 메시지 수신
    String message = br.readLine();
    System.out.println("서버로부터 받은 메시지 : " + message);
}
```

- 클라이언트가 입력한 메시지를 서버로 전송.
- 서버가 보낸 응답 메시지를 읽어 출력.
- `flush()`로 버퍼에 남은 데이터를 즉시 전송.

---

### 5. 종료 처리 (finally 블록)

```java
br.close();
pw.close();
socket.close();
sc.close();
```

- 모든 스트림과 소켓, 스캐너를 닫아 자원 누수 방지.

## 📊 전체 흐름 요약

1. **서버 접속 정보 준비** → IP, Port 지정
2. **소켓 생성** → `new Socket(ip, port)`
3. **스트림 연결** → `BufferedReader`, `PrintWriter`
4. **메시지 송수신 반복** → `while(true)`
5. **종료 시 자원 해제** → `close()`

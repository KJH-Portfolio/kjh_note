# Java 연산자 (Operators) 핵심 정리
> [!summary] 핵심 요약
> 
> - **개념**: 데이터를 가공, 비교, 제어하기 위해 제공되는 Java의 기본 연산 기호 체계
>     
> - **왜?**: 프로그램의 조건 분기, 수학적 계산, 메모리 참조 제어 등 논리적 흐름 구현
>     
> - **언제?**: 조건문(if, switch), 반복문(for, while), 변수 할당 등 애플리케이션 전반의 로직 구현부
>     
> - **핵심**: 문자열 비교 시 `==` 연산자 사용을 엄격히 금지하며, 논리 연산의 ==단락 평가(Short-Circuit)== 특성을 이해하여 의도치 않은 로직 누락 방지
>     
## 연산자 계층 및 분류

| **분류**     | **연산자 종류**         | **기호 및 문법**                                                              |
| ---------- | ------------------ | ------------------------------------------------------------------------ |
| **접근/그룹화** | 괄호, 배열/멤버 접근       | `()`, `[]`, `.`, `{}`                                                    |
| **단항 연산자** | 부호, 논리 반전, 증감, 형변환 | `+`, `-`, `!`, `++`, `--`, `~`, `(자료형)`                                  |
| **이항 연산자** | 산술 연산              | `*`, `/`, `%`, `+`, `-`                                                  |
|            | 비트 이동 (Shift)      | `<<`, `>>`, `>>>`                                                        |
|            | 크기 및 동등 비교         | `>`, `<`, `>=`, `<=`, `==`, `!=`                                         |
|            | 비트 논리 연산           | `&`, `^`, \|                                                             |
|            | 일반 논리 연산           | `&&`, \|\|                                                               |
| **삼항 연산자** | 조건 분기              | `(조건식) ? 값1 : 값2`                                                        |
| **대입 연산자** | 기본 및 복합 대입         | `=`, `+=`, `-=`, `*=`, `/=`, `%=`, `<<=`, `>>=`, `>>>=`, `&=`, `^=`, \|= |
| **나열 연산자** | 다중 선언              | `,`                                                                      |

## 핵심 연산자 동작 원리 및 실무 적용
### 동등 비교 (`==`)와 문자열 비교
- **기본형(Primitive) 비교**: 메모리에 저장된 '실제 값(Value)' 대조
- **참조형(Reference) 비교**: 메모리상의 ==물리적 주소(Address)== 대조 (자주 헷갈리는 것)
- **문자열(String) 비교 규칙**: 주소 비교로 인한 논리적 오류 방지를 위해 반드시 ==`equals()` 메서드 사용== (금지·경고)
- **어떻게?**: 객체 생성 방식(리터럴 vs `new` 키워드)에 따라 할당되는 메모리 영역이 달라지므로, 값 자체를 검증할 때는 `equals()`를 호출
```plaintext
[ 문자열 비교 동작 방식 ]

  Stack 영역           Heap 영역 (String Constant Pool 포함)
_____________          _________________________________
| a (0x10)  |---(참조)-->| "hello" (0x10)                |
| b (0x20)  |---(참조)-->| new String("hello") (0x20)    |

- a == b 결과: 0x10과 0x20의 주소값 대조 -> [false]
- a.equals(b): "hello"와 "hello"의 내부 문자 배열 대조 -> [true]
```
```java
public class OperatorExample {
    public static void main(String[] args) {
        String a = "hello";
        String b = new String("hello");

        System.out.println(a == b);      // false (메모리 주소 비교)
        System.out.println(a.equals(b)); // true (논리적 값 비교)
    }
}
```
### 단락 평가 (Short-Circuit Evaluation)
- **개념**: 논리 연산(`&&`, `||`) 시 좌측 항의 결과만으로 전체 결과가 확정되면 ==우측 항 연산을 즉시 생략== (핵심 개념)
- **주의사항**: 우측 항에 증감 연산자나 상태를 변경하는 메서드 배치 시 ==로직 실행 누락== 발생 위험 (경고)
- **어떻게?**: 상태를 변경하는 연산(증감 연산자, 메서드 호출 등)은 조건문 내부가 아닌 외부에 분리하여 작성
```plaintext
[ 단락 평가(Short-Circuit) 흐름도 ]

입력: 조건 A && 조건 B

  조건 A 평가
   ├─ [false] ──▶ 전체 결과 false 확정
   │                └─ 조건 B는 아예 실행되지 않음 (Skip!)
   │
   └─ [true] ───▶ 전체 결과 미정
                    └─ 조건 B 평가 후 최종 결과 결정
```
```java
int i = 0;
// 좌측 항(1 == 0)이 false이므로 && 우측 항(2 > i++)은 평가조차 되지 않음
if (1 == 0 && 2 > i++) {
    // 실행 블록 진입 불가
}
System.out.println(i); // 출력 결과: 0 (i++ 연산이 무시됨)
```
### 증감 연산자 (Increment/Decrement)
- **전위 연산 (`++x`)**: 변수 참조 전 즉시 증감을 수행한 뒤 결과값 반환
- **후위 연산 (`x++`)**: 현재 값을 먼저 반환(평가)한 후 증감 수행 (실행 순서 혼동 주의)
- **어떻게?**: 복잡한 수식 내에 증감 연산자를 섞어 쓰지 말고, 가독성을 위해 ==단독 라인으로 분리==하여 작성
```java
int x = 3;
int y = x++ + ++x; 
// 평가 순서:
// 1. x++ -> 3 평가 후 x=4로 증가
// 2. ++x -> x=4에서 5로 증가 후 5 평가
// 3. y = 3 + 5 = 8 도출

System.out.println("x=" + x + ", y=" + y); // 출력 결과: x=5, y=8
```
### 삼항 연산자 및 대입 연산자
- **삼항 연산자**: `if-else` 분기문을 1줄의 표현식으로 압축하여 코드 가독성 향상
- **객체 대입 (`=`)**: 참조형 변수에 대입 연산자 적용 시, 객체 내부 값이 아닌 주소값만 복사되는 ==얕은 복사(Shallow Copy)== 발생
- **어떻게?**: 단순 분기 처리나 기본값 할당 로직에 삼항 연산자를 적극 활용
```java
int score = 75;
// 조건식이 true면 "합격", false면 "불합격" 대입
String result = (score >= 60) ? "합격" : "불합격";

System.out.println(result); // 출력 결과: 합격
```

# Java 제어문 - 조건문 (Conditional Statements)
> [!summary] 핵심 요약
> 
> - 프로그램 로직의 동적 제어를 위한 `if`, `switch` 문 및 삼항 연산자의 구조와 활용
>     
> - 목적: 특정 조건에 따른 실행 흐름의 의사결정 분기 처리 및 가독성 높은 코드 구조화
>     
> - 핵심: 전통적 `switch-case`의 Fall-through 방지 및 ==Java 14+ 향상된 `switch`(`->`, `yield`)를 활용한 간결하고 안전한 값 반환==
>     
## 조건문 종류 및 핵심 스펙

|**구분**|**분기 방식**|**핵심 동작 및 특징**|
|---|---|---|
|`if` 문|단일 조건 평가|조건식이 참(`true`)일 경우에만 내부 코드 블록 실행|
|`if-else` 문|양자택일 분기|조건식의 참/거짓 여부에 따른 두 갈래의 명확한 실행 분기|
|다중 `if-else`|순차적 조건 검사|상위 조건부터 아래로 순차 평가하며, 최초로 일치하는 조건 블록 하나만 실행|
|전통적 `switch-case`|단일 값 매칭|평가 대상 변수의 값과 정확히 일치하는 `case` 블록으로 이동 및 `break` 전까지 실행|
|향상된 `switch` (Java 14+)|다중 매칭 및 반환|화살표(`->`) 구문 지원, 블록 내 최종 결괏값을 반환하는 `yield` 키워드 도입|
|삼항 연산자|인라인 조건 분기|`(조건식) ? 참값 : 거짓값` 형태의 식을 통한 간결한 데이터 반환|

## 주요 특징 및 실무 주의점
- **전통적 `switch-case` 동작 원리 및 한계**
    - 지원 데이터 타입: `byte`, `short`, `int`, `char`, `String`, `Enum` (==실수형 `float`, `double` 사용 불가==)
    - 부등호(`<`, `>`)를 활용한 ==범위(Range) 비교 불가==
    - `break` 누락 시 조건과 무관하게 하위의 모든 `case` 블록이 순차 실행되는 ==Fall-through 현상 발생 경고==
- **향상된 `switch` 문 (Java 14+)의 구조적 이점**
    - **화살표 구문 (`->`)**: 조건 매칭 시 해당 구문만 실행 후 자동 탈출 (Fall-through 원천 차단 및 `break` 생략)
    - **다중 조건 매칭**: 쉼표(`,`)를 활용한 여러 값의 단일 라인 처리
    - **표현식 반환 (Switch Expression)**: `switch` 블록의 평가 결과를 변수에 직접 대입 가능
    - **`yield` 키워드 사용**: 다중 라인 코드 블록(`{}`) 내부에서 연산을 수행한 뒤 ==최종 산출값을 반환할 때 필수 사용==
- **가독성 및 유지보수 최적화 기준**
    - 복잡한 범위 연산 및 다중 논리 연산(`&&`, `||`): 다중 `if-else` 문 활용
    - 다수의 명확하고 고정된 값 매칭: `switch` 문을 통한 코드 가독성 확보
    - 단순 대입용 조건 분기: 삼항 연산자를 활용한 코드 라인 수 단축 (중첩 사용 지양)
## 조건문 활용 예시
> [!example] 다중 `if-else` 문 (범위 분기)
```java
int score = 85;
if (score >= 90) {
    System.out.println("A"); // 조건 불일치 -> 스킵
} else if (score >= 80) {
    System.out.println("B"); // 조건 일치 -> 블록 실행 후 전체 if문 탈출
} else {
    System.out.println("C");
}
// 최종 출력: B
```
> [!example] 전통적 `switch-case` 문 (값 매칭 및 `break` 제어)
```java
int day = 3;
switch (day) {
    case 1:
        System.out.println("월요일");
        break; // 실행 블록 탈출 제어
    case 2:
        System.out.println("화요일");
        break;
    case 3:
        System.out.println("수요일");
        break; // day가 3이므로 여기까지만 실행 후 switch문 종료
    default:
        System.out.println("기타 요일"); // 일치하는 case가 없을 때의 기본 실행 블록
}
// 최종 출력: 수요일
```
> [!example] 향상된 `switch` 표현식 (Java 14+, 화살표 구문 및 `yield`)
```java
String customerGrade = "GOLD";
int purchaseAmount = 100000; // 10만원 구매

// 목적: 고객 등급에 따른 최종 할인 금액(discount)을 결정한다.
int discount = switch (customerGrade) {
    case "BRONZE" -> 1000; // 브론즈는 1,000원 고정 할인
    case "SILVER", "GOLD" -> 5000; // 실버, 골드는 5,000원 고정 할인
    
    case "VIP" -> {
        // VIP는 구매 금액의 10%를 계산하는 복잡한 로직이 필요함
        System.out.println("VIP 특별 혜택을 계산 중입니다...");
        int calculated = purchaseAmount / 10; 
        yield calculated; // 최종 계산된 값을 반환
    }
    
    default -> {
        System.out.println("비회원입니다.");
        yield 0;
    }
};

System.out.println(customerGrade + " 등급의 할인 금액: " + discount);
```
> [!example] 삼항 연산자 (인라인 조건 분기)
```java
int a = 10, b = 20;
// (10 > 20)은 거짓이므로 콜론(:) 우측의 b 반환
int max = (a > b) ? a : b; 
System.out.println("최대값: " + max); 
// 최종 출력: 최대값: 20
```

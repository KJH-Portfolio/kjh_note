# Java 제어문: 조건문 (Conditional Statements)
> [!summary] 핵심 요약
> 
> - **개념**: 프로그램의 논리적 흐름을 제어하고 분기를 처리하기 위한 기본 문법 체계
>     
> - **왜?**: 동적인 데이터 변화나 사용자 입력에 따라 각기 다른 코드 블록을 실행하여 로직의 의사결정을 자동화하기 위함
>     
> - **언제?**: 비즈니스 로직 내 예외 처리, 권한 분기, 데이터 상태에 따른 제어 흐름 등 애플리케이션 전반
>     
> - **핵심**: 전통적 `switch-case`의 단점(Fall-through)을 보완한 ==Java 14+ 향상된 `switch` 구문의 활용 능력==
>     
## 조건문 계층 및 분류

|**분기 방식**|**제어문 종류**|**핵심 동작 및 특징**|
|---|---|---|
|**단일/다중 조건**|`if` / `if-else`|조건식의 논리적 참/거짓(`true`/`false`) 여부에 따른 실행 흐름 분기|
|**순차적 검사**|다중 `if-else`|최상단부터 순차 평가하며, 최초로 일치하는 조건 블록 단 하나만 실행|
|**값 기반 매칭**|전통적 `switch-case`|변수의 실제 값과 정확히 일치하는 `case` 로 이동하여 실행|
|**다중 매칭/반환**|향상된 `switch` (Java 14+)|화살표(`->`) 구문 지원, 블록 내 연산 결과를 `yield`로 직접 반환|
|**인라인 분기**|삼항 연산자|`(조건) ? 참 : 거짓` 형태로 간단한 대입문을 1줄로 압축|

## `if` 문 구조 및 동작 원리
- **다중 `if-else` 분기**
    - **특징**: 복잡한 조건식(범위 연산, 다중 논리 연산 `&&`, `||`)을 유연하게 처리
    - **주의점**: 상위 조건이 충족되면 하위 `else if` 구문은 ==평가조차 하지 않고 스킵됨==
    - **어떻게?**: 조건의 교집합이 발생하지 않도록 범위의 순서(가장 좁은 범위부터 넓은 범위로)를 논리적으로 배치
```java
int score = 85;

if (score >= 90) {
    System.out.println("A"); // 조건 불일치 -> 스킵
} else if (score >= 80) {
    System.out.println("B"); // 조건 일치 -> 블록 실행 후 전체 if문 즉시 탈출
} else {
    System.out.println("C");
}
// 최종 출력: B
```
## `switch` 문: 전통적 방식 vs 향상된 방식
- **전통적 `switch-case`**
    - **타입 제약**: `byte`, `short`, `int`, `char`, `String`, `Enum`만 지원 (==실수형 `float`, `double` 사용 불가==)
    - **한계점**: 부등호(`<`, `>`)를 활용한 범위(Range) 단위 비교 불가능
    - **위험성**: `break` 누락 시 조건 충족 여부와 무관하게 하위 모든 `case` 블록이 실행되는 ==Fall-through 현상 발생 주의==
```java
int day = 3;
switch (day) {
    case 1:
        System.out.println("월요일");
        break; // 실행 블록 탈출
    case 2:
        System.out.println("화요일");
        break;
    case 3:
        System.out.println("수요일");
        break; // day가 3이므로 여기서 탈출
    default:
        System.out.println("기타 요일"); // 일치하는 값이 없을 때 기본 실행
}
```
- **향상된 `switch` 문 (Java 14+)**
    - **화살표 구문 (`->`)**: 조건 매칭 시 해당 블록만 실행 후 자동 탈출하므로 `break` 작성이 불필요
    - **표현식 반환 (Switch Expression)**: 제어문의 실행 결과를 변수에 직접 할당 가능
    - **다중 조건 매칭**: 쉼표(`,`)를 활용하여 여러 값을 하나의 `case`로 통합 처리
    - **`yield` 키워드**: 다중 라인 코드 블록(`{}`)에서 연산을 수행한 뒤 ==최종 산출값을 반환할 때 반드시 사용== 
        - switch문 내에서 `return`과 비슷한 역할을 함
```java
String customerGrade = "GOLD";
int purchaseAmount = 100000;

// 목적: 고객 등급에 따른 최종 할인 금액(discount) 계산 후 즉시 대입
int discount = switch (customerGrade) {
    case "BRONZE" -> 1000; // 단일 실행문은 직접 값 반환
    case "SILVER", "GOLD" -> 5000; // 다중 매칭 가능
    
    case "VIP" -> {
        // 복잡한 연산이 필요한 경우 중괄호 블록 사용
        System.out.println("VIP 특별 혜택 계산 중...");
        int calculated = purchaseAmount / 10; 
        yield calculated; // 블록 내부에서는 yield 키워드로 값 반환
    }
    
    default -> {
        System.out.println("비회원입니다.");
        yield 0;
    }
};

System.out.println(customerGrade + " 등급 할인 금액: " + discount);
```
## 삼항 연산자 (Ternary Operator)
- **인라인 조건 분기**
    - **구조**: `(조건식) ? 참일 때 반환값 : 거짓일 때 반환값`
    - **가독성 확보**: 단순한 값 할당을 위한 `if-else` 블록을 1줄로 단축하여 코드 다이어트
    - **어떻게?**: 조건이 복잡해지거나 로직이 추가될 경우 가독성이 급격히 하락하므로 삼항 연산자의 중첩 사용을 지양
```java
int a = 10, b = 20;

// (10 > 20) 조건이 거짓(false)이므로 콜론(:) 우측의 b 값을 반환
int max = (a > b) ? a : b; 

System.out.println("최대값: " + max); 
// 최종 출력: 최대값: 20
```

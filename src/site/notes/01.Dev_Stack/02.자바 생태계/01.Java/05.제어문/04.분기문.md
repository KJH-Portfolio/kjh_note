# Java 제어문: 분기문 (Branch Statements)
> [!summary] 핵심 요약
> 
> - **개념**: 실행 중인 루프의 흐름을 강제로 스킵/탈출하거나, 메서드 자체를 종료하는 제어 문법
>     
> - **왜?**: 특정 조건에서 불필요한 반복 연산을 건너뛰고, 예외 상황 발생 시 즉각적인 로직 종료를 구현하기 위함
>     
> - **언제?**: 데이터 필터링(`continue`), 탐색 완료 시 루프 종료(`break`), 예외 처리 및 값 반환(`return`) 등 로직 전반
>     
> - **핵심**: 다중 중첩 루프 구조에서 특정 분기를 탈출할 때는 복잡한 라벨(Label) 제어보다 ==메서드 분리를 통한 `return` 사용을 우선 권장==
>     
## 분기문 계층 및 분류

|**제어문**|**제어 대상**|**핵심 동작 및 특징**|
|---|---|---|
|**`continue`**|반복 스킵|현재 반복 회차의 남은 코드를 생략하고 다음 반복의 조건/증감식으로 즉시 점프|
|**`break`**|루프/블록 종료|조건식 검사 여부와 무관하게 가장 가까운 단일 반복문 또는 `switch` 블록 완전 탈출|
|**`return`**|메서드 종료|현재 실행 중인 메서드의 로직을 즉각 멈추고 호출자(Caller)에게 제어권 및 결괏값 반환|
|**라벨(Label)**|외부 루프 제어|식별자가 지정된 특정 외부 반복문의 제어(종료 또는 스킵)를 명시적으로 조작|

## 핵심 분기문 동작 원리 및 실무 적용
### `continue` 및 `break`의 제어 스코프
- **`continue` (특정 회차 건너뛰기)**
    - 조건 부합 시 현재 회차의 하위 로직을 무시하고 ==다음 반복으로 즉시 이동==
    - `for` 문에서는 증감식으로 이동하나, `while` 문에서는 조건식으로 이동하므로 내부 증감식 누락 시 무한 루프 발생 주의
- **`break` (조기 탈출)**
    - 반복 횟수가 남아있더라도 가장 인접한 루프 블록 하나를 즉시 파기하고 탈출
- **어떻게?**: 특정 데이터 탐색이 완료되면 `break`로 자원 소모를 멈추고, 유효하지 않은 데이터는 `continue`로 스킵하여 성능 최적화
```plaintext
[ 루프 제어 흐름 비교 ]

for (int i = 1; i <= 3; i++) {
    1. 상단 로직
    
    if (특정 조건) {
        [ continue ] ───▶ (3. 하단 로직 스킵 후 i++ 증감식으로 이동)
        [ break ] ──────▶ (for문 전체 종료 후 루프 밖으로 완전 탈출)
    }
    
    2. 하단 로직 (continue 발생 시 실행되지 않음)
}
```
```java
// continue 예시
for (int i = 1; i <= 5; i++) {
    if (i == 3) continue; // i가 3일 때 하위 코드(출력)를 스킵
    System.out.print(i + " ");
}
// 출력: 1 2 4 5

System.out.println();

// break 예시
for (int i = 0; i < 5; i++) {
    if (i == 3) break; // i가 3이 되는 순간 루프 완전 종료
    System.out.print(i + " ");
}
// 출력: 0 1 2
```
### `return`과 Early Return 패턴
- **개념**: 반환 타입이 있는 메서드에서는 결괏값을, `void` 메서드에서는 제어권만을 호출자에게 즉시 반환
- **Early Return (조기 반환)**: 예외 조건이나 부정 조건을 상단에서 먼저 검사하여 `return` 시킴으로써 코드의 깊이(Depth)를 줄이는 기법
- **어떻게?**: 다중 `if-else` 블록을 중첩하는 대신, 불필요한 조건을 ==선제적으로 차단(`return`)==하여 가독성 확보
```java
public int calculateTarget(int value) {
    // 1. 비정상 조건 먼저 필터링 (Early Return)
    if (value < 0) {
        return 0; // 즉시 종료되므로 하위 로직 실행 안 됨
    }
    
    // 2. 정상 로직 수행 (불필요한 else 블록 제거됨)
    int result = value * 100;
    
    return result; // 최종 결괏값 반환
}
```
### 라벨 (Label) 제어
- **개념**: 중첩 반복문 구조에서 안쪽 루프가 아닌 바깥쪽 루프의 제어 흐름을 직접 명시하는 문법
- **주의점**: 실행 흐름이 예측하기 어려워지고 스파게티 코드를 유발할 수 있으므로 ==필수적인 상황 외 사용 지양==
- **어떻게?**: 라벨을 사용해야 할 정도로 반복문이 깊게 중첩되었다면, 해당 로직을 별도의 메서드로 추출(Extract Method)하여 `return`으로 제어하는 것을 권장
```java
// 외부 루프에 'outerLoop'라는 라벨 선언
outerLoop: 
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) {
            // 가장 가까운 j 루프가 아닌, 라벨이 붙은 i 루프 전체를 강제 종료
            break outerLoop; 
        }
        System.out.println("i=" + i + ", j=" + j);
    }
}
// 최종 출력: i=0, j=0 (j가 1이 된 순간 모든 반복문이 종료됨)
```

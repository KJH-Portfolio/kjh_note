# Java 제어문 - 반복문 (Loop Statements)
> [!summary] 핵심 요약
> 
> - Java 프로그램의 데이터 순차 처리 및 로직 반복 수행을 위한 제어문 스펙
>     
> - 목적: 조건 기반의 반복 연산을 통한 코드 중복 최소화 및 처리 효율성 극대화
>     
> - 핵심: 의도치 않은 무한 루프 방지를 위한 명확한 ==탈출 조건(`break`, `return`) 명시==
>     
## 반복문 종류 및 핵심 스펙

|**제어문**|**최소 실행 횟수**|**핵심 동작 및 특징**|
|---|---|---|
|`while`|0회|평가된 조건식이 참(`true`)인 동안 내부 블록 지속 반복|
|`do-while`|1회|최초 1회 코드 블록 무조건 실행 후 조건식 진위 평가|
|`for`|0회|초기식, 조건식, 증감식을 한 줄에 배치한 명확한 횟수 제어|
|향상된 `for` (for-each)|0회 (요소 개수)|배열 및 컬렉션 내부 요소의 전체 순차 탐색|
|중첩 반복문|-|반복문 내부에 또 다른 반복문을 배치한 다차원 데이터 처리|
|무한 루프|무한|의도적 대기 상태 또는 특정 조건 달성 시까지의 지속 실행|

## 주요 특징 및 실무 주의점
- **향상된 `for`문 (for-each)의 한계**
    - 내부 인덱스를 활용한 특정 위치 요소의 직접 접근 불가
    - 순회 중인 원본 데이터(배열, 컬렉션)의 ==구조적 변경 및 요소 수정 불가== (읽기 전용 탐색)
- **`do-while` 문의 특수성**
    - 진입 조건의 참/거짓 평가 결과와 무관한 ==최초 1회 실행 무조건 보장==
- **중첩 반복문 성능 이슈**
    - 반복문 중첩 깊이에 비례하는 ==기하급수적인 시간 복잡도($O(n^2)$ 이상) 증가==
- **무한 루프(Infinite Loop) 제어 위험성**
    - `while(true)` 또는 `for(;;)` 구문 사용 시 명확한 탈출 분기 필수
    - 증감식 또는 종료 조건 설계 오류 시 스레드 무한 점유 및 시스템 프리징(Freezing) 발생 위험
## 반복문 활용 예시
> [!example] `while` 문 (조건 중심 반복)
```java
int i = 0;
while (i < 5) {
    System.out.println("i = " + i);
    i++; // 증감식 누락 시 무한 루프 발생
}
```
> [!example] `do-while` 문 (최소 1회 실행 보장)
```java
int i = 0;
// i < 0 이라는 거짓 조건이어도 무조건 1회 실행됨
do {
    System.out.println("i = " + i);
    i++;
} while (i < 0); 
```
> [!example] `for` 문 (명확한 횟수 기반 반복)
```java
for (int i = 0; i < 5; i++) {
    System.out.println("i = " + i);
}
```
> [!example] 향상된 `for`문 (for-each, 배열/컬렉션 순회)
```java
int[] numbers = {1, 2, 3, 4, 5};
// numbers 배열의 요소가 num에 순차적으로 복사됨 (원본 변경 불가)
for (int num : numbers) {
    System.out.println(num);
}
```
> [!example] 중첩 반복문 (다차원 로직 제어)
```java
for (int i = 2; i <= 9; i++) {
    System.out.println("=== " + i + "단 ===");
    for (int j = 1; j <= 9; j++) {
        System.out.println(i + " x " + j + " = " + (i * j));
    }
}
```
> [!example] 무한 루프와 탈출(Break)
```java
int count = 0;
while (true) {
    System.out.println("대기 중...");
    count++;
    
    if (count == 3) {
        break; // 특정 조건 만족 시 무한 루프 탈출
    }
}
```

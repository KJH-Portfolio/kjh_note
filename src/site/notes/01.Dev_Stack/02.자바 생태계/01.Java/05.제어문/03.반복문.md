# Java 제어문: 반복문 (Loop Statements)
> [!summary] 핵심 요약
> 
> - **개념**: 특정 조건이 만족되는 동안 코드 블록을 순차적이고 반복적으로 실행하는 제어 구문
>     
> - **왜?**: 동일한 로직의 중복 작성을 방지하고, 대량의 데이터 세트를 효율적으로 일괄 처리하기 위함
>     
> - **언제?**: 배열 및 컬렉션 순회, 파일 I/O 스트림 처리, 특정 상태 도달을 위한 대기 로직 등
>     
> - **핵심**: 조건식 설계 오류로 인한 시스템 프리징을 막기 위해 반드시 명확한 ==탈출 분기(`break`, `return`) 명시==
>     
## 반복문 계층 및 분류

|**제어문**|**최소 실행 횟수**|**핵심 동작 및 특징**|
|---|---|---|
|**`while`**|0회|평가된 조건식이 참(`true`)인 동안 내부 블록 지속 반복|
|**`do-while`**|1회|최초 1회 코드 블록 무조건 실행 후 조건식 진위 평가|
|**`for`**|0회|초기식, 조건식, 증감식을 한 줄에 배치한 명확한 횟수 제어|
|**향상된 `for`**|0회 (요소 개수)|배열 및 컬렉션 내부 요소의 전체 순차 탐색 (for-each)|
|**중첩 반복문**|-|반복문 내부에 또 다른 반복문을 배치한 다차원 데이터 처리|
|**무한 루프**|무한|의도적 대기 상태 또는 특정 조건 달성 시까지의 지속 실행|

## 주요 특징 및 실무 주의점
- **향상된 `for`문 (for-each)의 한계**
    - 내부 인덱스를 활용한 특정 위치 요소의 직접 접근 불가
    - 순회 중인 원본 데이터(배열, 컬렉션)의 ==구조적 변경 및 요소 직접 수정 불가== (읽기 전용 탐색 권장)
- **`do-while` 문의 특수성**
    - 진입 조건의 참/거짓 평가 결과와 무관하게 ==최초 1회 실행 무조건 보장== (핵심 전제)
- **중첩 반복문 성능 이슈**
    - 반복문 중첩 깊이에 비례하는 ==기하급수적인 시간 복잡도($O(n^2)$ 이상) 증가== (성능 저하 경고)
- **무한 루프(Infinite Loop) 제어 위험성**
    - `while(true)` 또는 `for(;;)` 구문 사용 시 내부 로직에 상태 변화 및 탈출 조건 필수 설계
    - 증감식 누락 등 설계 오류 시 스레드 무한 점유로 인한 애플리케이션 먹통(Freezing) 발생 위험
## 핵심 반복문 동작 원리 및 구현
- **`for` 문 (명확한 횟수 기반 반복)**
    - **동작 원리**: 루프 변수 초기화 후, 조건 평가 $\rightarrow$ 내부 블록 실행 $\rightarrow$ 증감식 처리의 정형화된 사이클 수행
    - **어떻게?**: 반복 횟수가 명확하게 정해져 있거나 배열의 인덱스를 직접 제어해야 할 때 활용
```plaintext
[ for 문 실행 순서 (Execution Flow) ]
		 [1. 초기식]
          │
    ┌──▶ [2. 조건식] ──(false)──▶ [반복문 종료]
    │     │ (true)
    │     ▼
    │   [3. 내부 블록 실행]
    │     │
    └── [4. 증감식] ◀──┘
```
```java
// 0부터 4까지 총 5회 반복
for (int i = 0; i < 5; i++) {
    System.out.println("i = " + i);
}
```
- **`while` 및 `do-while` 문 (조건 중심 반복)**
    - **동작 원리**: 횟수보다 특정 조건의 유지 여부가 중요할 때 사용되며, 내부 블록에서 루프 제어 변수를 직접 갱신해야 함
    - **어떻게?**: 파일 끝(EOF) 도달 전까지 읽기, 사용자 입력 대기 등 횟수를 예측할 수 없는 상황에 활용
```java
// while: 조건 선 검사
int i = 0;
while (i < 5) {
    System.out.println("i = " + i);
    i++; // 증감식 누락 시 무한 루프 발생
}

// do-while: 최소 1회 실행 후 후위 조건 검사
int j = 0;
do {
    System.out.println("j = " + j); // j < 0 이어도 무조건 1회 실행됨
    j++;
} while (j < 0); 
```
- **향상된 `for`문 (for-each, 데이터 순회)**
    - **동작 원리**: 배열이나 `Iterable` 인터페이스를 구현한 컬렉션의 요소를 처음부터 끝까지 순차적으로 꺼내어 변수에 복사
    - **어떻게?**: 인덱스가 필요 없는 단순 조회 및 데이터 매핑 작업 시 가독성 향상을 위해 적극 도입
```java
int[] numbers = {1, 2, 3, 4, 5};

// numbers 배열의 요소가 num에 순차적으로 '복사'됨 (원본 변경 불가)
for (int num : numbers) {
    System.out.println(num);
}
```
- **중첩 반복문 및 제어 (break, continue)**
    - **동작 원리**: `break`는 자신이 속한 가장 가까운 반복문을 즉시 탈출하고, `continue`는 현재 회차의 남은 로직을 건너뛰고 다음 반복(조건/증감식)으로 즉시 점프
    - **어떻게?**: 다차원 배열(행렬) 탐색이나, 복합 조건 검색 로직의 조기 종료(Early Return)에 활용
```java
for (int i = 2; i <= 9; i++) {
    System.out.println("=== " + i + "단 ===");
    for (int j = 1; j <= 9; j++) {
        if (j > 5) {
            break; // j가 6이 되는 순간 내부 for문만 탈출 (다음 단으로 넘어감)
        }
        System.out.println(i + " x " + j + " = " + (i * j));
    }
}
```

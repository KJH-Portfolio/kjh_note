# 📘 프로그램 오류와 예외 처리

## Summary 📝

> [!summary] Summary 📝
> 
> - 오류는 프로그램 수행 중 치명적 상황으로 인한 비정상 종료
> - 예외는 소스 수정으로 해결 가능한 오류로, 예외 처리 구문으로 대응 가능
> - `Throwable` 클래스 하위 구조로 `Error`와 `Exception`으로 구분됨
> - 예외는 Checked와 Unchecked로 나뉘며, 처리 방식이 다름
> - `try~catch`, `throws`, `try-with-resources` 등 다양한 처리 방식 존재

---

## 목적 🎯

- 프로그램의 안정성 확보
- 예외 발생 시 적절한 대응을 통해 비정상 종료 방지
- 예외 흐름을 명확히 하여 디버깅 및 유지보수 용이성 향상

---

## 핵심 개념 💡

### ✅ 오류와 예외 구분

|구분|설명|
|---|---|
|오류(Error)|시스템 수준의 치명적 문제. 코드로 복구 불가|
|예외(Exception)|소스 수정으로 해결 가능한 문제. 예외 처리 구문으로 복구 가능|

---

### ✅ 오류의 종류

|종류|설명|
|---|---|
|컴파일 에러|문법 오류. 실행 전 발생. 소스 수정으로 해결|
|런타임 에러|실행 중 발생. 예: 잘못된 입력, 배열 범위 초과|
|시스템 에러|컴퓨터 오작동. 코드로 해결 불가|

---

### ✅ 예외 클래스 계층 구조

```
Throwable
├── Error
│   └── VirtualMachineError
│       └── AWTError
└── Exception
    ├── RuntimeException (Unchecked)
    │   ├── ArithmeticException
    │   ├── NullPointerException
    │   ├── IndexOutOfBoundsException
    │   ├── NegativeArraySizeException
    │   ├── ClassCastException
    │   └── InputMismatchException
    └── IOException (Checked)
        ├── EOFException
        └── FileNotFoundException
```

| 구분                  | 설명                       |
| ------------------- | ------------------------ |
| Checked Exception   | 컴파일 시점에 예외 처리 강제         |
| Unchecked Exception | 예외 처리 선택 가능. 주로 프로그래머 실수 |

---

## 특징 ✨

- 모든 예외는 `Throwable` 클래스의 자손
- `Error`는 시스템 오류, `Exception`은 예외 처리 대상
- `try~catch`, `throws`, `finally`, `throw` 등 다양한 예외 처리 문법 존재
- 예외 클래스는 다형성을 활용하여 상위 타입으로 포괄 처리 가능
- `try-with-resources` 사용 시 자원 자동 정리 가능 (Java 7 이상)

---

## 팁과 주의점 ⚠️

- `finally` 블록은 `return`이 있어도 반드시 실행됨
- `System.exit()` 호출 시 `finally` 블록은 실행되지 않음
- 예외 발생 가능성이 있는 메소드 확인은 Java API 문서 활용
- `throw` 키워드로 예외 직접 발생 가능
- 오버라이딩 시 부모보다 좁은 범위의 예외만 선언 가능

---

## 예시 📌

✅ 예시1: `throws`로 예외 위임

```java
// IOException을 호출자에게 위임
public void methodA() throws IOException {
    methodB();
}
```

✅ 예시2: `try~catch`로 직접 처리

```java
try {
    // 예외 발생 가능 코드
} catch (IOException e) {
    e.printStackTrace(); // 예외 정보 출력
} finally {
    // 예외 발생 여부와 관계없이 실행
}
```

✅ 예시3: `try-with-resources`로 자원 자동 정리

```java
// BufferedReader 자원 자동 close
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();
}
```

✅ 예시4: 예외 직접 발생

```java
// IOException을 명시적으로 발생
public void method14() throws IOException {
    throw new IOException();
}
```

✅ 예시5: 예외사항 발생시 출력문

```java
int[] arr = new int[3];
try {
    int value = arr[5]; // ArrayIndexOutOfBoundsException 발생
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("배열 범위 초과");
}
```

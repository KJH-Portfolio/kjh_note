# 📘 List 계열 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - `List`는 **순서가 있는 데이터 집합**으로 인덱스를 통해 요소에 접근 가능
> - **중복 허용** → 동일한 값 여러 개 저장 가능
> - 대표 구현체: `ArrayList`, `LinkedList`, `Vector`

---

## 목적 🎯

- 순서가 중요한 데이터를 저장하고 관리
- 인덱스를 기반으로 빠른 접근 및 수정
- 중복 데이터를 허용해야 하는 경우 활용

---

## 핵심 개념 💡

### ✅ List 특징

|구분|설명|
|---|---|
|순서|저장 순서 유지|
|인덱스|0부터 시작, 인덱스로 접근 가능|
|중복|허용|
|구현체|`ArrayList`, `LinkedList`, `Vector`|

### ✅ 주요 메서드
```java
// ✅ List 주요 메서드 정리

list.add(E e);                 // boolean: 요소 추가
list.add(int idx, E e);        // void: 특정 위치에 삽입
list.get(int idx);             // E: 인덱스 위치 요소 반환
list.set(int idx, E e);        // E: 인덱스 위치 값 교체
list.remove(int idx);          // E: 인덱스 위치 요소 삭제
list.remove(Object o);         // boolean: 특정 요소 삭제
list.size();                   // int: 요소 개수 반환
list.clear();                  // void: 전체 요소 삭제
```
---

## 특징 ✨

- 인덱스 기반 접근 가능 → 배열과 유사
- 중복 데이터 저장 가능
- 순서 유지 → 입력한 순서대로 출력

---

## 팁과 주의점 ⚠️

- `ArrayList`는 **검색/접근 빠름**, `LinkedList`는 **삽입/삭제 빠름**
- 인덱스 범위를 벗어나면 `IndexOutOfBoundsException` 발생
- `Vector`는 동기화 지원 → 멀티스레드 환경에서 사용 가능

---

## 예시 📌
### ✅List와 제네릭

```java
List list = new ArrayList();
```

- `List`는 **순서가 있는 컬렉션** → 인덱스로 접근 가능
- 제네릭을 지정하지 않으면 `Object` 타입으로 저장 → 꺼낼 때 형변환 필요
- 제네릭을 쓰면 `(Music)`같은 형변환 캐스팅 없이 안전하게 사용 가능


### ✅데이터 추가 (Create)

```java
list.add(music);       // 맨 뒤에 추가
list.add(0, music);    // 맨 앞에 추가
```

- `add(E e)` : 리스트 끝에 요소 추가
- `add(int index, E e)` : 특정 위치에 삽입(기존값은 뒤로 밀림)
### ✅데이터 검색

```java
for (int i = 0; i < list.size(); i++) {
    if (((Music) list.get(i)).getTitle().equals(title)) {
        return (Music) list.get(i);
        //return (Music) list.remove(i); 삭제 구문
    }
}
```

- `get(int index)` : 인덱스로 요소 조회
- `size()` : 리스트 크기 반환
- 제네릭 미사용 시 `(Music)` 캐스팅 필요
### ✅ 정렬 (Collections.sort)

```java
Collections.sort(list) //기본 정렬 - 기본 클래스에 Comparable 인터페이스 구현했다면

Collections.sort(list, new AscTitle()); // 제목 오름차순 - 다른곳에 구현한 클래스 가져오기

Collections.sort(list, new Comparator<Music>() { //클래스 내에서 처리하기
    @Override
    public int compare(Music o1, Music o2) {
        return o2.getSinger().compareTo(o1.getSinger()); // 가수명 내림차순
    }
});
```

- `Collections.sort(List, Comparator)` : 사용자 정의 기준으로 정렬
- `Comparator` 인터페이스 구현 → 정렬 기준 지정 가능
- 별도 클래스(`AscTitle`) 또는 익명 클래스 활용

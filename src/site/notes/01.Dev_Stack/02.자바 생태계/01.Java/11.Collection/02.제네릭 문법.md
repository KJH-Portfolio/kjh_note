# 📘 제네릭(Generic)

## Summary 📝

> [!summary] Summary 📝
> 
> - **제네릭(Generic)**은 클래스나 메서드에서 사용할 데이터 타입을 외부에서 지정하여 타입 안정성과 코드 가독성을 높이는 문법

---

## 목적 🎯

- 타입 안정성 확보
- 불필요한 형변환 제거

---

## 핵심 개념 💡

- **정의**: 클래스나 메서드에서 사용할 데이터 타입을 외부에서 지정하는 문법
- **표현법**
    
    ```java
    ArrayList<String> list = new ArrayList<>();
    ```
    
    - `<>` 안에 타입 지정
    - 오른쪽은 자료형 생략 가능 (`new ArrayList<>()`)
- **Wrapper 클래스**: 기본 자료형은 직접 사용 불가 → `Integer`, `Double`, `Character` 등 사용

---

## 특징 ✨

- 컴파일 시점에서 타입 체크 가능
- 코드 가독성 및 유지보수성 향상
- 기본 자료형은 Wrapper 클래스 사용 필요

---

## 팁과 주의점 ⚠️

- 제네릭을 지정하지 않으면 기본적으로 `Object` 타입으로 처리됨 → 매번 형변환 필요
- 제네릭은 **클래스, 메서드, 인터페이스** 모두에 적용 가능
- `<?>` 와일드카드 제네릭을 활용하면 다양한 타입을 유연하게 처리 가능

---

## 예시 📌

```java
import java.util.ArrayList;

public class GenericExample {
    public static void main(String[] args) {
        // 제네릭 지정: String만 저장 가능
        ArrayList<String> strList = new ArrayList<>();
        strList.add("Hello");
        strList.add("World");

        // 형변환 불필요
        String msg = strList.get(0);
        System.out.println(msg);

        // Wrapper 클래스 사용
        ArrayList<Integer> intList = new ArrayList<>();
        intList.add(10); // 오토박싱
        intList.add(20);

        int num = intList.get(1); // 언박싱
        System.out.println(num);
    }
}
```

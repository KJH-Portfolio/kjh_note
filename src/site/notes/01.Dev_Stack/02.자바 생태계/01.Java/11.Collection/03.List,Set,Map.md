# 📘 Java Collection 주요 인터페이스 메서드 비교

## Summary 📝

> [!summary] Summary 📝
> 
> - `Collection` 인터페이스는 `List`, `Set`의 공통 부모
> - `Map`은 별도의 인터페이스로 key-value 구조 제공
> - 따라서 **공통 메서드 / List 전용 / Set 전용 / Map 전용**으로 나눠 이해 필요

---

## 목적 🎯

- **공통 기능**과 **자료구조별 전용 기능**을 구분
- **List / Set / Map 차이점**을 명확히 이해

## 핵심 개념 💡
### ✅계열 정리표

| 인터페이스 분류    | 특징                           | 구현 클래스       | 구현 클래스별 특징                                                               |
| ----------- | ---------------------------- | ------------ | ------------------------------------------------------------------------ |
| **List 계열** | - 순서 o<br>- 중복 저장 가능         | `ArrayList`  | 검색 속도가 빠름, 배열 기반, 삽입/삭제는 상대적으로 느림                                        |
|             |                              | `Vector`     | `ArrayList`와 유사하지만 **동기화(synchronized)** 지원                              |
|             |                              | `LinkedList` | 노드 기반, 삽입/삭제가 빠름, 검색은 상대적으로 느림                                           |
| **Set 계열**  | - 순서x<br>- 중복 저장 불가          | `HashSet`    | 해시 기반, 빠른 검색/저장, 순서 보장 X                                                 |
|             |                              | `TreeSet`    | 이진 트리 기반, 자동 정렬 지원, 범위 검색 유리                                             |
| **Map 계열**  | - 키와 값의 쌍으로 저장<br>- 키만 중복 불가 | `HashMap`    | 가장 많이 사용, 빠른 검색/저장, 순서 보장 X<br>`Node<K,V>[] table`                       |
|             |                              | `HashTable`  | `HashMap`과 유사하지만 **동기화 지원/구식 문법**<br>`HashEntry 사용(hash,key,value,next)` |
|             |                              | `TreeMap`    | 키를 자동 정렬하여 저장, 정렬된 순서로 탐색 가능                                             |
|             |                              | `Properties` | 키와 값 모두 문자열(String)로 제한, 설정 파일에 자주 사용<br>`.load`, `.store` 로 입출력에 용의     |


### ✅ Collection

- 요소 추가/삭제/검색/반복 등 기본 기능 제공
- 순서 유지 여부, 중복 허용 여부는 구현체에 따라 다름

### ✅List,Set,Map
- **List**: 순서 유지, 중복 허용 (`ArrayList`, `LinkedList`,...)
- **Set**: 순서 없음, 중복 불가 (`HashSet`, `TreeSet`,...)
- **Map**: 키+값 저장, 키 중복 불가, 값 중복 허용 (`HashMap`, `TreeMap`,...)

## 특징 ✨
### ✅ **List & Set 관련 메소드**

| 리턴타입          | 메소드                    | 설명                     |
| ------------- | ---------------------- | ---------------------- |
| `boolean`     | `add(E e)`             | 요소 추가                  |
| `boolean`     | `addAll(Collection c)` | 다른 컬렉션 전체 추가           |
| `void`        | `add(int index, E e)`  | 인덱스 위치에 삽입 (List만 해당)  |
| `E`           | `set(int index, E e)`  | 인덱스 위치 값 교체 (List만 해당) |
| `boolean`     | `contains(Object o)`   | 특정 요소 포함 여부            |
| `E`           | `get(int index)`       | 인덱스 위치 값 반환 (List만 해당) |
| `E`           | `remove(int index)`    | 인덱스 위치 값 삭제 (List만 해당) |
| `boolean`     | `remove(Object o)`     | 특정 요소 삭제               |
| `Iterator<E>` | `iterator()`           | 반복자 반환                 |
| `boolean`     | `isEmpty()`            | 비어있는지 확인               |
| `int`         | `size()`               | 요소 개수 반환               |
| `void`        | `clear()`              | 모든 요소 삭제               |

### ✅ **Map 관련 메소드**

| 리턴타입            | 메소드                       | 설명                  |
| --------------- | ------------------------- | ------------------- |
| `V`             | `get(Object key)`         | key에 해당하는 value 반환  |
| `V`             | `put(K key, V value)`     | key-value 추가 또는 수정  |
| `V`             | `replace(K key, V value)` | key에 해당하는 value 재배치 |
| `V`             | `remove(Object key)`      | key에 해당하는 value 삭제  |
| `Set<K>`        | `keySet()`                | key 집합 반환           |
| `Collection<V>` | `values()`                | value 집합 반환         |
| `boolean`       | `isEmpty()`               | 비어있는지 확인            |
| `int`           | `size()`                  | 요소 개수 반환            |
| `void`          | `clear()`                 | 모든 요소 삭제            |
### List
- `remove(Object o)`는 첫 번째 일치 항목만 삭제됨

### Set
- **객체 중복 판별**: 반드시 `equals()`와 `hashCode()`를 재정의해야 내부 필드값 기준으로 중복 확인 가능
- **향상된 for문**: 단순 조회에 적합
- **Iterator**: 순회 중 안전한 삭제 및 복잡한 로직 처리에 적합
- **List 변환**: `new ArrayList<>(set)`을 통해 인덱스 기반 접근 가능 
	- `ArrayList`는 **어떤 Collection이든** 받아서 내부적으로 요소들을 복사해 리스트로 만듦
- **remove() 주의**: 향상된 for문 내부에서 `remove()` 호출 시 `ConcurrentModificationException` 발생 
### Map
- 향상된 for문 내부에서 `remove()` 호출 시 `ConcurrentModificationException` 발생

## 팁과 주의점 ⚠️

- List
	- **remove() 오버로딩 주의**: List는 `remove(int index)`와 `remove(Object o)` 구분 필요
	- `.subList()`는 원본 리스트와 연결된 뷰(View)객체를 반환하는 메서드
		- 원본값도 같이 바뀌는 리스트 생성
```java
List<String> original = new ArrayList<>(List.of("A", "B", "C", "D", "E"));
List<String> sub = original.subList(1, 4); // ["B", "C", "D"]
```

## 예시 📌
### 주요 메서드
```java
// ✅ 공통 메서드 (List, Set, Map 모두 사용 가능)
collection.isEmpty();          // boolean: 비어있는지 확인
collection.size();             // int: 요소 개수 반환
collection.clear();            // void: 모든 요소 삭제

// ✅ List에서 사용 가능한 메서드
list.add(E e);                 // boolean: 요소 추가
list.add(int index, E e);      // void: 인덱스 위치에 삽입
list.addAll(Collection c);     // boolean: 다른 컬렉션 전체 추가
list.set(int index, E e);      // E: 인덱스 위치 값 교체
list.contains(Object o);       // boolean: 특정 요소 포함 여부
list.get(int index);           // E: 인덱스 위치 값 반환
list.remove(int index);        // E: 인덱스 위치 값 삭제
list.remove(Object o);         // boolean: 특정 요소 삭제
list.iterator();               // Iterator<E>: 반복자 반환

// ✅ Set에서 사용 가능한 메서드
set.add(E e);                  // boolean: 요소 추가
set.addAll(Collection c);      // boolean: 다른 컬렉션 전체 추가
set.contains(Object o);        // boolean: 특정 요소 포함 여부
set.remove(Object o);          // boolean: 특정 요소 삭제
set.iterator();                // Iterator<E>: 반복자 반환

// ✅ Map에서 사용 가능한 메서드
map.get(Object key);           // V: key에 해당하는 value 반환
map.put(K key, V value);       // V: key-value 추가 또는 수정
map.replace(K key, V value);   // V: key에 해당하는 value 재배치
map.remove(Object key);        // V: key에 해당하는 value 삭제
map.keySet();                  // Set<K>: key 집합 반환
map.values();                  // Collection<V>: value 집합 반환

// ✅ Iterator 인터페이스 메서드
iterator.hasNext();    // boolean: 다음 요소가 있는지 확인
iterator.next();       // E: 다음 요소 반환
iterator.remove();     // void: 현재 요소 삭제 (선택적 지원)

// ✅ Properties 전용 메서드
Properties props = new Properties();
props.setProperty(String key, String value);   // void: 문자열 key-value 저장
props.getProperty(String key);                 // String: key에 해당하는 value 반환
props.getProperty(String key, String default); // String: key 없을 경우 기본값 반환
props.load(InputStream inStream);              // void: 입력 스트림으로부터 로딩
props.store(OutputStream out, String comments);// void: 출력 스트림으로 저장 commets는 주석
props.stringPropertyNames();                   // Set<String>: 모든 key 반환

```

## Set 관련
### ✅ Set 중복 객체 추가

```java
shs.add(s); // s가 기존에 shs에 있는 객체의 변수라면 동일한 주소값, 중복으로 간주되어 추가되지 않음 
shs.add(new Student("이학생", 18, 99, "002")); // equals/hashCode 재정의 시 중복으로 처리됨
```

- 객체 주소값이 같거나 equals/hashCode 기준이 같을 경우 중복 처리됨

---

#### ✅ Set 요소 접근 방법

##### 1. 향상된 for문

```java
for (Student stu : shs) {
    System.out.println(stu);
}
```


##### 2. ArrayList로 변환 후 인덱스 접근

```java
ArrayList<Student> list = new ArrayList<>(shs);
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
```

##### 3. Iterator 사용

```java
Iterator<Student> it = shs.iterator();
while (it.hasNext()) {
    Student student = it.next();
    System.out.println(student);
}
```

---

#### ✅ Iterator로 조건 기반 요소 제거

```java
Iterator<Student> it2 = set.iterator();
while (it2.hasNext()) {
    Student sd = it2.next();
    if (sd.getAge() == 30) {
        it2.remove();
    }
}
```

- 조건에 따라 요소를 안전하게 제거하는 방식
- 향상된 for문에서는 `remove()` 사용 불가(원본 Collection이 바뀌기 때문)

---

#### ✅ 반복자 재사용을 통한 결과 확인

```java
Iterator<Student> it3 = set.iterator();
while (it3.hasNext()) {
    System.out.println(it3.next());
}
```

- 제거 이후의 Set 상태 확인을 위한 반복자 재사용

# 📘 Map / HashMap 활용 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - `Map` 계열은 **key-value 쌍**으로 데이터를 저장하는 컬렉션
> - `HashMap`은 대표적인 구현체로, key는 중복 불가 / value는 중복 가능
> - 순차 접근은 직접 불가 → `keySet()`, `entrySet()`을 통해 `Set`으로 변환 후 반복 처리
> - `put`, `get`, `replace`, `remove` 등 다양한 메서드 제공

---

## 목적 🎯

- **데이터를 key-value 형태로 저장**하고 검색, 수정, 삭제를 효율적으로 수행
- key를 통해 value를 빠르게 조회
- `Set` 변환을 통해 반복문, Iterator로 순차 접근 가능

---

## 핵심 개념 💡

### ✅ Map / HashMap 특징

| 구분    | 설명                                   |
| ----- | ------------------------------------ |
| key   | 중복 불가, 식별자 역할                        |
| value | 중복 가능, key로 접근                       |
| 순서    | 저장 순서 유지 안 됨                         |
| 접근    | key 기반 조회 (`get(key)`)               |
| 변환    | 순차 접근 위해 `keySet()`, `entrySet()` 활용 |

### ✅ 주요 메서드
```java
map.put(k,v);                 // V: key-value 저장 
map.get(k);                   // V: key에 해당하는 value 반환 
map.size();                   // int: 저장된 데이터 개수 반환
map.replace(k, v2);            // V: key에 해당하는 value 변경 
map.remove(k);                // V: key에 해당하는 value 삭제
map.keySet();                 // Set<K>: key 집합 반환
map.entrySet();               // Set<Map.Entry<K,V>>: key-value 쌍을 Entry 객체로 반환

//Map.Entry<K,V>는 Map의 한 쌍을 나타내는 자료형이다.
```

| 표현                           | 의미                                    | 자료형인가요? | 반복 가능한가요?                         |
| ---------------------------- | ------------------------------------- | ------- | --------------------------------- |
| `Map.Entry<String, Integer>` | `Map`의 한 쌍(key-value)을 나타내는 **자료형**   | ✅ 예     | ❌ 아니요                             |
| `map.entrySet()`             | `Map`의 모든 key-value 쌍을 담은 **집합(Set)** | ❌ 아니요   | ✅ 예 (`for`, `iterator` 등으로 반복 가능) |
| `map.containsKey(키)`         | `map`이 해당key를 가지고 있는지                 |         |                                   |

---

## 특징 ✨

- **제네릭 미사용 시**: 다양한 타입 혼합 저장 가능 (형변환 필요)
- **제네릭 사용 시**: 타입 안정성 확보, 형변환 불필요
- **key 중복 불가**: 동일 key 입력 시 value 덮어쓰기
- **value 중복 가능**: key만 다르면 같은 value 저장 가능
- **순차 접근 불가**: 반드시 `Set` 변환 후 반복문/Iterator 사용

---

## 팁과 주의점 ⚠️

- `get()`은 존재하지 않는 key 입력 시 `null` 반환
- `replace()`는 없는 key 입력 시 아무 동작 없음 (null 반환)
- `Iterator`의 `remove()`는 **반복 중 안전한 삭제** 가능
- `Map`은 `List`처럼 인덱스 기반 접근 불가 → 반드시 key 기반 접근

## 예시 📌
### ✅Map 값을 순환하기
#### ✅ 기본 Set을 활용 : 키 값 순환하기

```java
Map<String,Snack> sMap = new Hashmap<>(); //map 생성
Set<String> keySet = sMap.keySet(); //sMap의 키 모음 콜렉션
Iterator<String> itKey = keySet.iterator(); //sMap의 키 값을 순환

//"눈을 감자"라는 키를 가진 콜렉션을 삭제
while(itKey.hasNext()) {
    String key = itKey.next();
    Snack snack = sMap.get(key);

    if(key.equals("눈을감자")) {
        itKey.remove(); // 안전한 삭제
    }

    System.out.println("key : " + key);
    System.out.println("snack : " + snack);
}
```

---

#### ✅ entrySet() 활용 : Map 자체를 순환하기

```java
Set<Entry<String, Snack>> set = sMap.entrySet();

// for-each
for(Entry<String, Snack> e : set) {
    System.out.println(e.getKey());
    System.out.println(e.getValue());
}

// Iterator
Iterator<Entry<String, Snack>> itEntry = set.iterator();
while(itEntry.hasNext()) {
    Entry<String, Snack> e = itEntry.next();
    System.out.println(e.getKey());
    System.out.println(e.getValue());
}
```

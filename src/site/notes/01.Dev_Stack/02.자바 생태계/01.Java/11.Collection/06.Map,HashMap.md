# Map 및 HashMap 아키텍처
> [!summary] 핵심 요약
> 
> - **개념**: 데이터를 키(Key)와 값(Value)의 쌍(Entry)으로 묶어 관리하는 컬렉션 프레임워크
>     
> - **왜?**: 특정 식별자(Key)를 통해 연관된 데이터(Value)를 고속으로 조회하고 수정하기 위함
>     
> - **언제?**: 데이터의 저장 순서보다 식별자를 통한 '빠른 검색' 및 '데이터 매핑'이 중요한 비즈니스 로직 구현 시
>     
> - **핵심**: Key의 중복 불가 특성 활용 및 ==반드시 Set 자료형으로 변환==하여 순차 접근(Iteration)하는 방식 숙달
>     
## Map 컬렉션의 핵심 구조 및 특징
- **데이터 구성 단위 (Entry)**
    - Key와 Value가 하나의 쌍을 이루는 `Map.Entry` 구조로 데이터 저장
    - Key를 통해 Value에 접근하는 단방향 매핑 구조
- **Key (키) 속성**
    - 데이터를 식별하는 고유 식별자 역할
    - 중복 저장 불가 (동일한 Key로 데이터 삽입 시 ==기존 Value 덮어쓰기== 발생)
- **Value (값) 속성**
    - 실제 저장되는 대상 데이터
    - Key만 다르다면 Value는 중복 저장 허용
- **순서 미보장**
    - `HashMap` 기준, 내부 해시 알고리즘에 의해 데이터가 배치되므로 저장 순서 유지 불가
- **타입 안정성**
    - 제네릭(`<K, V>`) 적용 시 명확한 타입 명시로 불필요한 형변환 방지
```plaintext
[ Map Data Structure ]

  Key                 Value
 ┌──────┐            ┌──────┐
 │ "ID" │───────────▶│ "A1" │  <-- 1개의 Map.Entry 구조
 ├──────┤            ├──────┤
 │ "PW" │───────────▶│ 1234 │  <-- Value는 중복 및 다양한 타입 허용
 ├──────┤            ├──────┤
 │ "ID" │─(중복불가)▶│ "B2" │  <-- 기존 "A1" 값은 삭제되고 "B2"로 덮어쓰기됨
 └──────┘            └──────┘
```
## 주요 동작 메서드 및 getOrDefault
- **데이터 조작 및 조회 메서드**
    - 데이터 삽입(`put`), 조회(`get`), 수정(`replace`), 삭제(`remove`) 지원
    - 찾는 Key가 없을 경우 `get()`은 `null`을 반환
- **안전한 데이터 조회 (getOrDefault)**
    - 특정 Key가 존재하지 않을 때 `null` 대신 지정한 기본값을 반환하여 NullPointerException 방지
    - 카운팅 로직(예: 단어 빈도수 계산) 등에 필수적으로 사용됨

|**기능 구분**|**주요 메서드**|**동작 원리 및 역할**|**반환 타입 등 기타 속성**|
|---|---|---|---|
|**추가/수정**|`put(K key, V value)`|주어진 Key에 Value를 매핑하여 저장 (Key 존재 시 덮어씀)|`V` (이전 값 반환, 없으면 null)|
|**단순 조회**|`get(Object key)`|Key에 매핑된 Value 반환 (존재하지 않으면 null 반환)|`V`|
|**안전 조회**|`getOrDefault(K, V)`|Key가 존재하면 해당 Value 반환, ==없으면 지정한 기본값(V) 반환==|`V`|
|**조건 수정**|`replace(K key, V value)`|해당 Key가 존재할 때만 Value를 새로운 값으로 변경|`V` (변경 전 값 반환)|
|**삭제**|`remove(Object key)`|Key와 일치하는 Entry(쌍) 전체 삭제|`V` (삭제된 값 반환)|
|**검증**|`containsKey(Object key)`|Map 내부에 지정한 Key가 존재하는지 여부 확인|`boolean`|

## Map 데이터 순회 (Iteration) 패턴
- **Map의 순차 접근 한계**
    - Map 자체는 `Iterable` 인터페이스를 상속하지 않아 `for-each`나 `Iterator` 직접 사용 불가
    - 저장된 데이터를 순회하려면 반드시 `Set` 타입의 컬렉션으로 추출해야 함
- **추출 방식 비교**
    - `keySet()`: Key 값만 추출하여 `Set`으로 반환. 코드가 간결하나 Value 조회 시마다 `get()` 호출 발생
    - `entrySet()`: Key와 Value가 묶인 `Map.Entry` 객체를 `Set`으로 반환. 대량의 데이터 순회 시 성능상 유리
```plaintext
[ Map Iteration Flow ]

  Map Data                      keySet()                  entrySet()
 ┌──────┐                      ┌──────┐                  ┌──────────┐
 │ K:V  │                      │ K    │                  │ Map.Entry│
 ├──────┤                      ├──────┤                  ├──────────┤
 │ A:10 │ ──(Extract Key)──▶   │ A    │ ──(Loop)──▶      │ [A, 10]  │ ◀──(Loop)
 │ B:20 │                      │ B    │                  │ [B, 20]  │
 └──────┘                      └──────┘                  └──────────┘
 직접 순회 불가                 키 모음 집합(Set) 반환       키-값 쌍 집합(Set) 반환
```
- **안전한 데이터 삭제**
    - 반복문을 도는 도중 컬렉션의 요소를 삭제할 때는, 예외(ConcurrentModificationException) 방지를 위해 ==반복자(Iterator)의 remove()==를 사용해야 함
```java
import java.util.*;

public class MapIterationExample {
    public static void main(String[] args) {
        Map<String, Integer> fruitMap = new HashMap<>();
        fruitMap.put("Apple", 1000);
        fruitMap.put("Banana", 1500);
        fruitMap.put("Orange", 2000);
        
        // 1. getOrDefault 활용 (초기화 및 카운팅에 유용)
        int cherryPrice = fruitMap.getOrDefault("Cherry", 500);
        System.out.println("Cherry 가격: " + cherryPrice); // 500 출력

        // 2. keySet()과 Iterator를 활용한 순회 및 안전한 삭제
        Set<String> keys = fruitMap.keySet();
        Iterator<String> keyIterator = keys.iterator();

        while(keyIterator.hasNext()) {
            String key = keyIterator.next();
            Integer price = fruitMap.get(key);
            
            // 조건에 맞는 데이터 안전하게 삭제
            if(key.equals("Banana")) {
                keyIterator.remove(); 
            }
        }

        // 3. entrySet()을 활용한 전체 데이터 순회 (성능 유리)
        Set<Map.Entry<String, Integer>> entries = fruitMap.entrySet();
        
        for(Map.Entry<String, Integer> entry : entries) {
            System.out.println("과일: " + entry.getKey() + ", 가격: " + entry.getValue());
        }
    }
}
```

# 📘 List / Set / Map 계열 컬렉션 접근 방식 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - **List**: 인덱스 기반 접근 가능
> - **Set**: 인덱스 없음, for-each·Iterator·List 변환 후 접근
> - **Map**: 인덱스 없음, keySet()·values()·entrySet()으로 접근

---

## 목적 🎯

- Java 컬렉션 프레임워크의 **자료구조별 접근 방식 차이** 이해
- 각 컬렉션의 **특징에 맞는 순회 방법** 학습
- 코드 작성 시 **효율적이고 안전한 접근 방식 선택**

---

## 핵심 개념 💡

| 계열       | 특징                   | 접근 방식                                            |
| -------- | -------------------- | ------------------------------------------------ |
| **List** | 순서 유지, 인덱스 기반, 중복 허용 | `get(index)`, `for-each`,` Iterator`             |
| **Set**  | 순서 없음, 중복 불가, 인덱스 없음 | `for-each`, `Iterator`, `List` 변환 후 인덱스 접근       |
| **Map**  | key-value 쌍 저장       | `keySet()`, `values()`, `entrySet()`, `Iterator` |
- `LinkedHashSet`은 입력 순서 유지
---

## 특징 ✨

### ✅ List

- 인덱스 기반 접근 가능
- 순서 유지 및 중복 허용
- 배열과 유사하지만 크기 자동 조절

### ✅ Set

- 인덱스 없음
- 중복 불가
- 순서 보장 없음
	- (단, LinkedHashSet은 입력 순서 유지, TreeSet은 정렬 유지)

### ✅ Map

- key-value 구조
- key는 중복 불가, value는 중복 가능
- 인덱스 접근 불가, key 또는 entry 기반 접근



---

## 팁과 주의점 ⚠️

- **List**: 인덱스 접근이 가능하므로 단순 조회 시 `for-each`, 특정 위치 접근 시 `get(index)` 활용
- **Set**: 인덱스가 없으므로 직접 접근 불가, 필요 시 `List`로 변환 후 인덱스 접근
- **Map**: key와 value를 동시에 다룰 때는 `entrySet()` 사용이 가장 효율적
- **Iterator**: 순회 중 안전한 요소 제거 가능, `for-each`에서는 `remove` 불가

### Map.Entry<String, Integer>
- `Map.Entry<K, V>`는 `Map` 내부에 있는 **key-value 쌍을 표현하는 인터페이스**
- 여기서 `K`는 key의 자료형, `V`는 value의 자료형을 의미
- 따라서 `Map.Entry<String, Integer>`는:
		- 하나의 `Map` 요소를 나타내는 자료형
		- `key`는 `String` 타입
		- `value`는 `Integer` 타입

| 표현                           | 의미                                    | 자료형인가요? | 반복 가능한가요?                         |
| ---------------------------- | ------------------------------------- | ------- | --------------------------------- |
| `Map.Entry<String, Integer>` | `Map`의 한 쌍(key-value)을 나타내는 **자료형**   | ✅ 예     | ❌ 아니요                             |
| `map.entrySet()`             | `Map`의 모든 key-value 쌍을 담은 **집합(Set)** | ❌ 아니요   | ✅ 예 (`for`, `iterator` 등으로 반복 가능) |

## 예시 📌

### ✅ List 접근

```java
// 인덱스로 접근
System.out.println(list.get(0)); // "A"

// 향상된 for문
for (String s : list) {
    System.out.println(s);
}

// Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    System.out.println(it.next());
}
```

---

### ✅ Set 접근

```java
// 향상된 for문
for (String s : set) {
    System.out.println(s);
}

// Iterator
Iterator<String> it = set.iterator();
while (it.hasNext()) {
    System.out.println(it.next());
}

// List로 변환 후 인덱스 접근
List<String> listFromSet = new ArrayList<>(set);
System.out.println(listFromSet.get(0));
```

---

### ✅ Map 접근

```java
// keySet() 이용
for (String key : map.keySet()) {
    System.out.println(key + " : " + map.get(key));
}

// values() 이용
for (Integer value : map.values()) {
    System.out.println(value);
}

// entrySet() 이용 (가장 많이 사용)
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " : " + entry.getValue());
}

// Iterator과 Entry 이용
Iterator<Map.Entry<String, Integer>> it = map.entrySet().iterator();
while (it.hasNext()) {
    Map.Entry<String, Integer> entry = it.next();
    System.out.println(entry.getKey() + " : " + entry.getValue());
}
```

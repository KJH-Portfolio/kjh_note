# 메서드 (Method) 핵심 구조와 동작 원리
> [!summary] 핵심 요약
> 
> - 특정 연산이나 로직을 수행하기 위한 단위 코드 블록
>     
> - 목적: 코드의 재사용성 확보 및 복잡한 프로그램의 기능 단위 모듈화
>     
> - 핵심: 반환 타입(`void` vs `return`)의 구분과 JVM 메모리 상의 ==프레임(Frame) 생명주기== 파악
>     
## 메서드 구성 요소 및 설계 규약
- **기본 구성 요소**
    - 접근제어자 (Access Modifier): `public`, `private` 등을 통한 호출 가능 범위 설정
    - 예약어 (Modifier): `static`, `final`, `abstract` 등을 통한 메서드의 특성 부여
    - 메서드명 (Method Name): 소문자로 시작하는 동사 형태의 네이밍 (자바 네이밍 관례)
    - 매개변수 (Parameter): 호출부로부터 전달받는 입력값을 저장하는 로컬 변수

| **분류**           | **특징**                       | **선언 예시**                   |
| ---------------- | ---------------------------- | --------------------------- |
| **`void` 메서드**   | 연산 수행 후 호출부로 결과값을 반환하지 않음    | `public void printInfo()`   |
| **`return` 메서드** | 연산 완료 후 지정된 타입의 결과값을 호출부로 반환 | `public int calculateSum()` |

## JVM 내부 메서드 실행 흐름
- **메모리 영역별 역할**

| **영역**          | **주요 저장 내용**                      | **생존 기간**      |
| --------------- | --------------------------------- | -------------- |
| **Method Area** | ==클래스 구조==, 메서드 코드, **static** 변수 | 프로그램 시작 ~ 종료   |
| **Heap Area**   | `new`로 생성된 ==객체(인스턴스)==, 배열       | GC가 수거할 때까지    |
| **Stack Area**  | ==지역 변수==, 매개 변수, 메서드 호출 기록       | 메서드 종료 시 즉시 삭제 |
```java
public class ExecutionDemo {
    // 1. 클래스 로딩 시 Method Area에 적재됨
    static int globalConfig = 100; 

    public int add(int a, int b) {
        // 3. 메서드 호출 시 Stack Frame 생성
        int result = a + b; // 지역 변수 할당
        return result;
    }

    public static void main(String[] args) {
        // 2. 객체 생성 (Heap Area)
        ExecutionDemo demo = new ExecutionDemo();

        // 3. 메서드 호출 (Stack Area - Push)
        int sum = demo.add(5, 10);

        System.out.println("결과: " + sum);
        // 4. 메서드 종료 (Stack Area - Pop)
    }
}
```
## 핵심 주의 사항 및 작성 팁
- **제약 및 필수 조건**
    - `return`문 강제: 반환 타입이 `void`가 아닌 모든 메서드는 반드시 결과값을 ==반환(return) 처리==
    - 스태틱(Static) 접근: `static` 메서드 내부에서는 인스턴스 변수(Non-static) 직접 접근 불가 
        - `(생성 시점의 차이:static 메서드가 인스턴스 변수보다 먼저 로딩됨)`
- **오버로딩 (Overloading) 활용**
    - 동일한 메서드명으로 매개변수의 타입, 개수, 순서를 다르게 구성하여 다양한 입력 조합 지원
- **호출 방식의 구분**
    - 인스턴스 메서드: 반드시 객체 생성(`new`) 이후 참조 변수를 통한 호출
    - 정적(Static) 메서드: 객체 생성 없이 ==클래스명을 통한 직접 호출== (`ClassName.methodName()`)
## 실전 코드 예시
> [!example] 메서드 정의 및 호출 방식별 비교
✅ **인스턴스 메서드 (객체 생성 필수)**
```java
public class Calculator {
    // 인스턴스 메서드 (static 없음)
    public int add(int a, int b) {
        return a + b; 
    }
}

public class MainApp {
    public static void main(String[] args) {
        Calculator calc = new Calculator(); // Heap에 객체 생성
        int result = calc.add(3, 5);        // Stack에서 메서드 호출
        System.out.println(result);         // 출력: 8
    }
}
```
✅ **정적 메서드 (Static Method)**
```java
public class MathUtils {
    // 정적 메서드 (static 포함)
    public static int multiply(int a, int b) {
        return a * b;
    }
}

public class MainApp {
    public static void main(String[] args) {
        // 인스턴스 생성 없이 클래스명으로 즉시 호출
        int result = MathUtils.multiply(10, 20); 
        System.out.println(result); // 출력: 200
    }
}
```

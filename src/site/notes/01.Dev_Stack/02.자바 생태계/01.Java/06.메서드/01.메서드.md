# Java 메서드 (Method): 핵심 구조와 메모리 동작 원리
> [!summary] 핵심 요약
> 
> - **개념**: 특정 연산이나 비즈니스 로직을 수행하기 위해 묶어놓은 독립적인 단위 코드 블록
>     
> - **왜?**: 코드의 재사용성을 확보하고, 복잡한 애플리케이션의 기능을 모듈화하여 유지보수성을 극대화하기 위함
>     
> - **언제?**: 클래스 내부의 행위(Behavior)를 정의하고, 객체 간의 메시지 전달 및 데이터 가공을 처리할 때
>     
> - **핵심**: 반환 타입(`void` vs 데이터 타입)의 명확한 구분과, JVM 메모리상의 ==스택 프레임(Stack Frame) 생명주기== 완전한 이해
>     
## 메서드 구성 요소 및 시그니처
- **기본 구성 요소 및 역할**
    - 메서드는 크게 선언부(Signature)와 구현부(Body)로 나뉘며, 선언부는 외부와의 통신 규격을 정의함

|**구성 요소**|**역할 및 특징**|**선언 예시**|
|---|---|---|
|**접근 제어자**|호출 가능 범위(가시성) 설정|`public`, `private`|
|**예약어**|정적, 상수, 추상 등 특성 부여|`static`, `final`, `abstract`|
|**반환 타입**|연산 완료 후 반환할 데이터의 타입 (없을 시 `void`)|`int`, `String`, `void`|
|**메서드명**|소문자로 시작하는 동사 형태 (네이밍 관례)|`calculateSum`, `printInfo`|
|**매개변수**|외부 호출부로부터 주입받는 입력값 (로컬 변수)|`(int a, int b)`|

```plaintext
[ 메서드 시그니처(Signature) 구조 해부 ]

  public   static   int   calculateSum (int a, int b) { ... }
   (1)       (2)    (3)       (4)           (5)

  (1) 접근 제어자 : 외부 클래스에서의 호출 허용 범위
  (2) 예약어      : 인스턴스 생성 없이 호출 가능한 정적 메서드 선언
  (3) 반환 타입   : 연산 종료 후 정수형(int) 결과값 반환 예약
  (4) 메서드명    : 목적을 나타내는 동사형 네이밍
  (5) 매개변수    : 로직 수행에 필요한 외부 주입 데이터 (파라미터)
```
## 핵심 제약 사항 및 작성 규칙
- **반환(Return) 강제 규약**
    - 반환 타입이 `void`가 아닌 모든 메서드는 로직 종료 전 반드시 ==결괏값 반환(`return`) 필수==
- **정적(Static) 컨텍스트 제약**
    - `static` 메서드 내부에서는 인스턴스 변수 및 `static`이 없는 일반 메서드에 ==직접 접근 불가== (생성 시점 문제)
- **메서드 오버로딩 (Overloading)**
    - 동일한 메서드명으로 매개변수의 타입, 개수, 순서를 다르게 구성하여 다양한 입력 조합을 지원하는 다형성 기법
## JVM 내부 메서드 실행 흐름 및 메모리
- **메모리 영역별 생명주기 및 역할**

|**영역 (Area)**|**주요 저장 내용**|**생존 기간**|
|---|---|---|
|**Method**|클래스 메타 정보, 메서드 바이트코드, `static` 변수|프로그램 기동 ~ JVM 종료 시|
|**Heap**|`new` 키워드로 생성된 실제 ==객체(인스턴스)== 및 배열|GC(가비지 컬렉터) 수거 시점|
|**Stack**|지역 변수, 매개 변수, ==메서드 호출 기록(Frame)==|메서드 호출 시 생성 ~ 종료 시 즉각 소멸|

```plaintext
[ 메서드 Stack Frame 동작 원리 (LIFO 구조) ]

 3. add() 호출! ──▶ [ Stack 영역 ]
                       │ [0x33] add() 프레임  │ ── (종료 시 가장 먼저 Pop되어 소멸)
                       │ [0x22] main() 프레임 │ ── (add() 종료 후 마저 실행됨)
                       │ [0x11] JVM Base     │
                       └─────────────────────┘
 ※ LIFO (Last In First Out): 나중에 호출된 메서드가 가장 먼저 종료되고 메모리에서 해제됨.
```
```java
public class ExecutionDemo {
    // 1. 클래스 로딩 시 Method Area에 적재됨
    static int globalConfig = 100; 

    public int add(int a, int b) {
        // 3. 메서드 호출 시 Stack 영역에 새 프레임 생성
        int result = a + b; // 프레임 내부에 지역 변수 할당
        return result;      // 반환과 동시에 프레임 소멸
    }

    public static void main(String[] args) {
        // 2. Heap 영역에 객체 인스턴스 생성
        ExecutionDemo demo = new ExecutionDemo();

        // 3. Stack 영역에 add() 프레임 Push
        int sum = demo.add(5, 10);

        System.out.println("결과: " + sum);
        // 4. main() 프레임 Pop 되며 프로그램 종료
    }
}
```
## 호출 방식: 인스턴스 vs 정적(Static)
- **인스턴스 메서드 (Instance Method)**
    - 반드시 `new` 키워드를 통해 메모리(Heap)에 객체를 생성한 이후, 참조 변수를 통해 호출
- **정적 메서드 (Static Method)**
    - 객체 생성 과정 없이 ==클래스명을 통한 직접 호출== 가능
    - 주로 유틸리티(Utility)성 기능이나 단순 수학 연산 등에 활용
**✅ 인스턴스 메서드 및 정적 메서드 구현 예시**
```java
// 1. 인스턴스 메서드 구현
public class Calculator {
    public int add(int a, int b) {
        return a + b; 
    }
}

// 2. 정적 메서드 구현 (유틸리티)
public class MathUtils {
    public static int multiply(int a, int b) {
        return a * b;
    }
}

public class MainApp {
    public static void main(String[] args) {
        // 인스턴스 메서드: 객체 생성 필수
        Calculator calc = new Calculator(); 
        int result1 = calc.add(3, 5);      
        System.out.println(result1); // 출력: 8
        
        // 정적 메서드: 객체 생성 없이 클래스명으로 즉시 호출
        int result2 = MathUtils.multiply(10, 20); 
        System.out.println(result2); // 출력: 200
    }
}
```

# JVM 메모리 영역 (Runtime Data Area)
> [!summary] 핵심 요약
> 
> - JVM의 핵심 메모리 영역(Stack, Heap, Method Area) 구조 및 관리 체계
>     
> - 목적: 메모리 구조 이해를 바탕으로 한 효율적 자원 관리 및 ==런타임 에러 사전 방지==
>     
> - 핵심: 영역별 ==저장 대상==과 ==생명주기(Lifecycle)==의 명확한 구분 및 매핑
>     
## JVM Runtime Data Area 구조
- **메모리 할당 체계**
    - 스레드(Thread) 공유 여부에 따른 독립적/공용 메모리 영역 분리
    - 데이터 특성에 맞춘 최적화된 적재 및 수명 관리
```plaintext
JVM Runtime Data Area
 ├─ Thread Shared Area (모든 스레드 공유)
 │   ├─ Heap: 동적 할당 객체, 배열 (GC 관리 대상)
 │   └─ Method Area (Static): 클래스 메타데이터, 정적 변수, 상수풀
 │
 └─ Thread Private Area (스레드별 독립 생성)
     ├─ Stack: 메서드 프레임, 지역 변수, 매개변수
     ├─ //PC Register: 현재 실행 중인 JVM 명령어 주소
     └─ //Native Method Stack: C/C++ 등 네이티브 코드 실행 영역
```
## 영역별 핵심 특징 비교
- **주요 메모리 3요소**
    - 변수 타입 및 객체 생성 방식에 따른 저장소 분류

| **영역**                  | **저장 대상**                   | **생명주기 및 관리 방식**               | **주요 특징**                            |
| ----------------------- | --------------------------- | ------------------------------ | ------------------------------------ |
| **Stack**<br>`임시공간`     | 지역 변수, 매개변수, 메서드 프레임        | 메서드 호출 시 생성 / 종료 시 소멸 (LIFO)   | 높은 접근 속도, 스레드별 독립성 보장                |
| **Heap**<br>`객체`        | `new` 키워드 생성 객체, 배열         | 동적 생성 / 불필요 시 ==GC에 의한 자동 해제== | 참조형 데이터의 실제 데이터 저장 공간                |
| **Static**<br>`메서드`<br> | `static` 변수/메서드, 상수, 클래스 정보 | 클래스 로드 시 적재 / JVM 종료 시 소멸      | 전역 공유 자원, ==동시성(Thread-Safe) 제어 필요== |

## 주의점 및 에러 상황
- **메모리 초과 에러 (Runtime Error)**
    - 부적절한 메모리 할당 및 해제 실패로 인한 애플리케이션 강제 종료 위험
> [!warning] 메모리 에러 주요 유형
> 
> - **StackOverflowError**: 과도한 재귀 호출이나 무한 루프로 인한 Stack 메모리 한계 초과
>     
> - **OutOfMemoryError (OOM)**: 해제되지 않은 객체의 과다 생성으로 인한 Heap 메모리 고갈
>     
## 동작 예시
> [!example] 실습: 메모리 할당 위치 판별
> 
> - 코드 실행 단계별 변수 및 객체의 메모리 적재 위치 매핑
>     
```java
public class Example {
    static int staticCount = 0;  // Static (Method) 영역: 전역 공유

    int instanceCount = 0;       // Heap 영역: 객체 생성 시 할당

    public void method(int param) {  // Stack 영역: 매개변수
        int localVar = param + 1;    // Stack 영역: 지역 변수
        System.out.println(localVar);
    }

    public static void main(String[] args) {
        Example ex = new Example();  // Heap 영역: Example 인스턴스 데이터
                                     // Stack 영역: Heap 영역을 가리키는 참조 변수 ex
        ex.method(5);                // Stack 영역: method 실행 프레임 적재
    }
}
```
> [!example] 실습: 치명적 에러 발생 케이스
> 
> - 잘못된 코드 작성으로 인한 JVM 메모리 초과 유발 사례
>     
```java
// 1. StackOverflowError 유발
public static void recursive() {
    recursive(); // ==종료 조건 없는 무한 재귀==
}

// 2. OutOfMemoryError 유발
public static void createLeak() {
    List<int[]> list = new ArrayList<>();
    while (true) {
        list.add(new int[1000000]); // ==참조 유지로 인한 GC 수거 불가==
    }
}
```

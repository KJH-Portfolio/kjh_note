# 메서드 오버로딩 (Method Overloading)
> [!summary] 핵심 요약
> 
> - **개념**: 동일한 이름의 메서드를 매개변수의 타입, 개수, 순서에 따라 여러 개 정의하는 객체지향 기법
>     
> - **왜?**: 동일한 기능을 수행하는 메서드들의 이름을 하나로 통일하여 API 사용자의 편의성 및 가독성 향상
>     
> - **어디서?**: 클래스 내부 (특히 다양한 초기값이 필요한 생성자(Constructor) 정의나 `println()` 같은 범용 유틸리티 메서드 구현 시)
>     
> - **핵심**: 컴파일러가 메서드를 식별하는 유일한 기준은 ==매개변수 시그니처(타입, 개수, 순서)==이며, 리턴 타입은 무관함
>     
## 오버로딩 성립 조건 및 식별 기준
- **메서드 시그니처 (Method Signature)**
    - 컴파일러가 수많은 메서드 중 호출할 대상을 정확히 찾아내는 고유 식별표
    - 구성 요소: 메서드 이름 + 매개변수 리스트

|**항목**|**성립 여부**|**상세 설명**|
|---|---|---|
|**메서드 이름**|**필수 동일**|이름이 다르면 오버로딩이 아닌 완전히 새로운 메서드의 생성임|
|**매개변수**|**필수 상이**|타입, 개수, 순서 중 ==최소 하나 이상 달라야 함==|
|**리턴 타입**|**무관**|리턴 타입만 다른 경우는 ==중복 정의(컴파일 에러)==로 판정됨|
|**접근 제어자**|**무관**|`public`, `private` 등은 오버로딩 성립에 아무런 영향을 주지 않음|

```plaintext
[ 컴파일러의 메서드 식별 트리 (정적 바인딩) ]

호출: calc.add(10, 20)  // 정수형 인자 2개 전달
 │
 ├─ [1단계] 이름 확인
 │   └─ "add" 라는 이름의 메서드 그룹 탐색
 │
 └─ [2단계] 매개변수 시그니처 대조
     ├─ add(int a)                 -> 탈락 (개수 불일치)
     ├─ add(double a, double b)    -> 탈락 (타입 불일치)
     ├─ add(int a, int b, int c)   -> 탈락 (개수 불일치)
     └─ add(int a, int b)          -> [당첨: 컴파일 타임에 즉시 바인딩]
```
## 구현 예시 및 주의사항
- **설계적 관점의 일관성**
    - 이름이 같다면 내부 로직도 동일한 기능적 목적을 지녀야 함 (혼동 방지)
- **자동 형변환(Type Casting) 주의**
    - 메서드 호출 시 인자로 전달한 값의 타입이 정확히 일치하지 않아도, 컴파일러의 암묵적 형변환 규칙에 의해 예상치 못한 메서드가 호출될 수 있음
> [!example] 정상적인 오버로딩 구현
> 
> - 매개변수의 타입과 개수 차이를 통해 각각의 메서드를 명확하게 분리함
>     
```java
class Calculator {
    // 1. 기본 메서드 (int 2개)
    int add(int a, int b) { 
        return a + b; 
    }
    
    // 2. 매개변수 타입이 다른 경우 (double 2개)
    double add(double a, double b) { 
        return a + b; 
    }
    
    // 3. 매개변수 개수가 다른 경우 (int 3개)
    int add(int a, int b, int c) { 
        return a + b + c; 
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println(calc.add(3, 5));       // 1번 메서드 호출
        System.out.println(calc.add(2.5, 4.3));   // 2번 메서드 호출
        System.out.println(calc.add(1, 2, 3));    // 3번 메서드 호출
    }
}
```
> [!warning] 잘못된 오버로딩 사례 (컴파일 에러)
> 
> - 매개변수 시그니처가 완전히 같고 ==리턴 타입만 다른 경우는 오버로딩이 불가능==함
>     
```java
class Test {
    int print() { 
        return 1; 
    }
    
    // 에러 발생: 'print()' is already defined in 'Test'
    // 리턴 타입이 String으로 다르지만, 매개변수가 ()로 동일하므로 충돌
    String print() { 
        return "Hello"; 
    }
}
```

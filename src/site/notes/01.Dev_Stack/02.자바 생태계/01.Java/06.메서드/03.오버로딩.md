# 메서드 오버로딩 (Method Overloading)
> [!summary] 핵심 요약
> 
> - 동일한 이름의 메서드를 매개변수의 타입, 개수, 순서에 따라 다르게 정의하는 기법
>     
> - 목적: 동일한 기능적 동작을 하나의 메서드 이름으로 통일하여 가독성 및 유지보수성 향상
>     
> - 핵심: 오버로딩의 성립 조건은 오직 ==매개변수의 차이(시그니처)==이며, 리턴 타입은 무관
>     
## 오버로딩 성립 조건 및 특징
- **메서드 시그니처 (Method Signature)**
    - 메서드 이름과 매개변수 리스트(타입, 개수, 순서)의 조합
    - 컴파일러가 메서드를 식별하는 유일한 기준

|**구분**|**성립 조건 및 세부 특징**|
|---|---|
|**메서드 이름**|반드시 동일|
|**매개변수**|타입, 개수, 순서 중 하나 이상 상이|
|**리턴 타입**|==오버로딩 성립 조건과 무관== (리턴 타입만 독립적 변경 불가)|
|**바인딩 시점**|컴파일 타임에 호출될 메서드 결정 (정적 바인딩)|

## 주의 및 혼동 사항
- **컴파일 에러 (메서드 중복 정의)**
    - 매개변수는 같고 ==리턴 타입만 다른 경우 컴파일 에러 발생==
- **자동 형변환(Type Casting) 주의**
    - 매개변수 전달 시 컴파일러의 자동 형변환 적용
    - 데이터 타입 호환성에 따른 의도치 않은 다른 메서드 호출 가능성
- **설계적 관점의 주의점**
    - 가독성 유지를 위해 의미와 기능이 완전히 다른 메서드에 동일한 이름 부여 금지
## 코드 동작 예시
> [!example] 정상적인 오버로딩 구현
> 
> - 매개변수의 타입과 개수 차이로 개별 메서드 인식 및 호출
>     
```java
class Calculator {
    // 1. 기본 메서드
    int add(int a, int b) { 
        return a + b; 
    }
    
    // 2. 매개변수 타입이 다른 경우
    double add(double a, double b) { 
        return a + b; 
    }
    
    // 3. 매개변수 개수가 다른 경우
    int add(int a, int b, int c) { 
        return a + b + c; 
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println(calc.add(3, 5));       // int 버전 호출
        System.out.println(calc.add(2.5, 4.3));   // double 버전 호출
        System.out.println(calc.add(1, 2, 3));    // 3개 인자 버전 호출
    }
}
```
> [!warning] 잘못된 오버로딩 사례 (컴파일 에러)
> 
> - ==리턴 타입의 차이는 오버로딩으로 인정되지 않음==
>     
```java
class Test {
    int print() { 
        return 1; 
    }
    
    // String print() { return "Hello"; } // 에러: 매개변수 시그니처가 동일함
}
```

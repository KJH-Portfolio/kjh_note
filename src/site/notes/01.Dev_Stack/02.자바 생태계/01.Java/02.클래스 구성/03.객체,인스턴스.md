# Java 객체와 인스턴스: 정의·구조·생명주기 통합
> [!summary] 핵심 요약
> 
> - **개념**: 클래스 설계도를 기반으로 ==힙(Heap) 메모리==에 할당된 실체(객체)와 그 관계적 정의(인스턴스)
>     
> - **목적**: 독립적 상태 관리 및 객체 지향의 핵심 가치(다형성, 캡슐화)를 물리적으로 구현
>     
> - **핵심**: `new` 연산자를 통한 메모리 점유 및 ==참조 변수(Reference Variable)==를 통한 간접 제어 메커니즘 숙지
>     
---
## 1. 객체와 인스턴스의 정의 및 관계
- **객체 (Object)**
    - 클래스에 선언된 형태 그대로 메모리에 구현된 ==독립적 실체==
    - 모든 인스턴스를 포괄하는 광의적 개념이자 프로그래밍의 조작 대상
- **인스턴스 (Instance)**
    - 특정 클래스로부터 생성되었음을 강조하는 ==소속 및 관계== 중심의 용어
    - 추상적 설계도가 구체적인 실물로 변형된 상태를 지칭

|**구분**|**객체 (Object)**|**인스턴스 (Instance)**|
|---|---|---|
|**관점**|데이터 집합체 및 구현된 실체 그 자체|클래스와 객체 사이의 ==인과 관계== 강조|
|**비유**|완성된 '자동차' (물리적 존재)|'A 설계도'로 만든 '자동차' (제작 출처 명시)|
|**범위**|힙 메모리에 존재하는 모든 실체|특정 클래스 타입으로 선언된 구체적 실체|
|**강조점**|동작과 상태의 보유 여부|클래스 설계 정보의 ==복제 및 구현== 여부|

---
## 2. 객체 생성의 3단계 프로세스
- **동적 할당 흐름**
    - 설계도 참조 및 메모리 크기 계산 후 런타임에 실행

| **단계**                    | **주요 작업**       | **발생 영역** | **비고**                    |
| ------------------------- | --------------- | --------- | ------------------------- |
| **선언 (Declaration)**      | `Student s;`    | ==Stack== | 참조 변수 공간만 (참조값X)          |
| **인스턴스화 (Instantiation)** | `new Student()` | ==Heap==  | 실제 데이터 객체 생성 및 ==참조값 반환== |
| **초기화 (Initialization)**  | 생성자 호출          | Heap      | 필드 값 설정 및 객체 가용 상태 전환     |

## 3. JVM 메모리 구조 및 관리
- **영역별 역할 분담**
    - ==스택(Stack)==: 기본 타입 변수 및 힙 영역 객체의 참조값(주소) 보관하는 공간
    - ==힙(Heap)==: 모든 인스턴스가 물리적으로 상주하는 공유 메모리 공간(실제 인스턴스)
- **가비지 컬렉션 (GC) 메커니즘**
    - 유효 참조가 끊긴(Unreachable) 객체를 자동 식별하여 메모리 회수
```plaintext
JVM Memory Architecture
 └─ Stack Area (Thread Local)
     └─ Variable 's': 0x100 (Address)
 └─ Heap Area (Shared)
     ├─ [0x100] Instance (Active)
     └─ [0x200] Instance (Unreachable) ◀── [GC Target]
```
---
## 4. 참조 변수 활용 및 주의사항
- **참조 메커니즘의 특징**
    - 동일 객체 다중 참조: 하나의 인스턴스를 여러 변수가 공유 시 의도치 않은 ==상태 변경(Side Effect)== 유의
    - 주소 복사: 참조 타입 변수 대입 시 데이터 자체가 아닌 메모리 주소만 복사
- **치명적 오류 및 방지**
    - ==NullPointerException (NPE)==: 주소값이 없는(`null`) 상태에서 객체 멤버에 접근 시 발생
    - 할당 전 전제 조건: 반드시 ==인스턴스화== 과정을 거친 후 참조 변수 활용 권장
> [!warning] 주의사항
> 
> - 참조 변수에 `null`을 대입하면 기존 연결된 객체는 GC의 대상이 될 수 있음
>     
> - 프리미티브 타입(int, double 등)과 참조 타입의 메모리 저장 방식 차이 명확히 구분 필요
>

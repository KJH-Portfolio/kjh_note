# Java 객체와 인스턴스: 메모리 구조와 생명주기
> [!summary] 핵심 요약
> 
> - **개념**: 클래스 설계도를 기반으로 ==힙(Heap) 메모리==에 할당된 실체(객체)와 그 소속 관계(인스턴스)
>     
> - 목적: 독립적 상태 관리 및 객체 지향의 핵심 가치(다형성, 캡슐화)를 물리적으로 구현
>     
> - 핵심: `new` 연산자를 통한 메모리 점유 및 ==참조 변수(Reference Variable)==를 통한 제어 메커니즘 숙지
>     
## 1. 객체와 인스턴스의 정의 및 관계
- **객체 (Object)**
    - 클래스에 선언된 형태 그대로 메모리에 구현된 ==독립적 실체==
    - 모든 인스턴스를 포괄하는 광의적 개념이자 프로그래밍의 조작 대상
- **인스턴스 (Instance)**
    - 특정 클래스로부터 생성되었음을 강조하는 ==인과 관계== 중심의 용어
    - 추상적 설계도가 구체적인 실물로 변형된 상태를 지칭

|**구분**|**객체 (Object)**|**인스턴스 (Instance)**|
|---|---|---|
|**관점**|데이터 집합체 및 구현된 실체 그 자체|클래스와 객체 사이의 관계 강조|
|**비유**|완성된 '자동차' (물리적 존재)|'A 설계도'로 만든 '자동차' (출처 명시)|
|**범위**|힙 메모리에 존재하는 모든 실체|특정 클래스 타입으로 구현된 실체|
|**강조점**|동작과 상태의 보유 여부|클래스 설계 정보의 ==복제 및 구현== 여부|

## 2. 객체 생성 프로세스 및 메모리 할당
- **동적 할당 흐름**
    - 런타임에 설계도를 참조하여 메모리 크기 계산 및 할당 수행

|**단계**|**주요 작업**|**발생 영역**|**비고**|
|---|---|---|---|
|**선언**|`Student s;`|==Stack==|참조 변수 공간만 확보 (주소값 없음)|
|**인스턴스화**|`new Student()`|==Heap==|실제 데이터 객체 생성 및 ==참조값 반환==|
|**초기화**|생성자 호출|Heap|필드 값 설정 및 가용 상태 전환|

```plaintext
[ JVM 메모리 구조 ]

      스택(Stack) 영역                      힙(Heap) 영역
   (리모컨 보관함 / Thread)              (실제 객체 거실 / Shared)
 __________________________          ______________________________
|                          |        |                              |
|  변수 s (Address: 0x100)  |───────▶|  [0x100] Student 인스턴스      |
|  : 실제 주소를 쥐고 있음     |        |     (사용 중 - Active)         |
|__________________________|         _______________________________
									 
									 ______________________________
                                    |  [0x200] Old 인스턴스          |
                                    |     (연결 끊김 - Unreachable)  |
                                    |     ◀─── [GC 수거 대상]        |
                                    |______________________________|
 ※ 핵심 요약:
 1. Stack: 리모컨(주소값)을 들고 있는 손.
 2. Heap : 실제 물건(객체)이 놓인 장소.
 3. GC   : 연결된 '손(Stack)'이 없는 물건(0x200)을 쓰레기로 판단해 치움.
```

## 3. 참조 변수 활용 및 가비지 컬렉션(GC)
- **참조 메커니즘의 특징**
    - 주소 복사: 참조 타입 변수 대입 시 데이터 자체가 아닌 메모리 주소만 복사
    - 다중 참조: 하나의 인스턴스를 여러 변수가 공유할 경우 ==상태 변경(Side Effect)== 발생 유의
- **치명적 오류 및 메모리 관리**
    - ==NullPointerException==: 참조 주소가 없는(`null`) 상태에서 멤버 접근 시 발생
    - 가비지 컬렉션: 유효 참조가 끊긴 객체를 자동 식별하여 ==메모리 회수== 수행
```java
public class MemoryExample {
    public static void main(String[] args) {
        // 1. 객체 생성 및 참조 (Stack에 s1 생성, Heap에 Student 객체 생성)
        Student s1 = new Student("홍길동");

        // 2. 주소값 복사 (s1과 s2는 동일한 Heap 객체를 가리킴)
        Student s2 = s1;
        s2.setName("이순신"); // s1의 이름도 "이순신"으로 변경됨 (Side Effect)

        // 3. 참조 해제 (기존 객체와의 연결을 끊음)
        s1 = null; 
        
        // 4. 안전한 접근 (NPE 방지)
        if (s1 != null) {
            System.out.println(s1.getName());
        }
    }
}
```
> [!warning] 주의사항
> 
> - 참조 변수에 `null`을 대입하면 기존 연결된 객체는 GC의 대상이 됨
>     
> - 기본 타입(Primitive)과 참조 타입의 메모리 저장 방식 차이를 명확히 구분할 것
>

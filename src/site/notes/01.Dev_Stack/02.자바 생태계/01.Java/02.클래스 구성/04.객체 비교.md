# Java 객체 비교 및 식별 메커니즘 (Identity vs. Equality)
> [!summary] 핵심 요약
> 
> - Java 객체의 동일성(Identity)과 동등성(Equality) 구분 및 해시 기반 식별 체계의 원리 정리
>     
> - 목적: 메모리 참조 비교와 논리적 데이터 비교의 차이를 이해하고 컬렉션(Collection) 프레임워크의 무결성 확보
>     
> - 핵심: `equals()` 재정의 시 ==`hashCode()` 동시 재정의== 필수 원칙 준수
>     
## 객체 비교 및 식별 수단의 분류
- **비교 도구 및 역할 분석**
    - 객체의 물리적 주소와 논리적 상태를 구분하여 적절한 식별 도구 선택 필요

|**분류**|**도구**|**비교/식별 대상**|**주요 특징 및 역할**|
|---|---|---|---|
|**물리적 비교**|`==` 연산자|메모리 주소(Reference)|==동일성(Identity)== 판단 및 원시(Primitive) 타입 값 비교|
|**논리적 비교**|`.equals()`|객체 내부 상태(State)|==동등성(Equality)== 판단 (Object 기본 구현은 `==`과 동일)|
|**해시 식별**|`.hashCode()`|객체 고유 정수값|해시 기반 자료구조(`HashSet`, `HashMap`)에서의 분류 및 검색|
|**고유 식별**|`System.identityHashCode()`|객체 고유 주소 해시|`hashCode` 재정의 여부와 무관한 객체의 절대적 고유 식별자|

## equals-hashCode 상호작용 및 설계 규약
- **논리적 결합의 필연성**
    - `equals`가 true인 두 객체는 반드시 ==동일한 `hashCode` 반환== 필수
    - 규약 위반 시 해시 자료구조에서 논리적으로 같은 객체가 다른 버킷에 저장되어 데이터 유실 및 중복 발생
    - 비교 로직(`equals`)과 식별 로직(`hashCode`)은 동일한 필드를 공유하여 일관성 유지 권장
- **구현 시 주의 사항**
    - `equals` 오버라이드 시 ==`hashCode` 동시 재정의== 원칙 엄수
    - `null` 체크 및 자기 자신 참조(`this == o`) 여부 우선 확인으로 성능 최적화
    - 불변(Immutable) 객체의 경우 `hashCode`를 캐싱하여 반복적인 연산 비용 감소
## 비교 프로세스 및 코드 구조
### 객체 비교 알고리즘
- **비교 효율화 절차**
    - 무거운 논리적 비교(`equals`) 전에 가벼운 물리적 비교(`==`)와 해시 비교를 선행하여 리소스 낭비 방지
```plaintext
Object Comparison Process
 ├─ 1. Physical Check (==)
 │   └─ True: 동일성(Identity) 확정 -> [종료: Success]
 │
 └─ 2. False: 동등성(Equality) 판별 시작
     ├─ [Phase A] Hash Code Check (Performance Optimization)
     │   ├─ Different: 해시값 다름 -> [종료: 다른 객체]
     │   └─ Same: 해시값 같음 -> [Phase B 진입: 해시 충돌 가능성 확인]
     │
     └─ [Phase B] Logical Check (.equals)
         ├─ Null & Type Verification (Safety)
         │   └─ Null이거나 타입 불일치 -> [종료: 다른 객체]
         └─ Field-by-Field Comparison (Deep Check)
             ├─ All Fields Match: [최종 확정: 동등 객체]
             └─ Any Field Mismatch: [종료: 다른 객체]
```
### 표준 구현 패턴
- **안전한 오버라이딩**
    - Java 7 이상부터 제공되는 `Objects` 유틸리티 클래스를 활용한 Null-safe 비교 및 해시 생성
```java
@Override
public boolean equals(Object o) {
    // 1. 자기 자신 참조 확인 (메모리 주소가 같으면 무조건 동일)
    if (this == o) return true; 
    
    // 2. Null 체크 및 클래스 타입 일치 여부 확인 (다형성 고려 시 instanceof 사용 가능)
    if (o == null || getClass() != o.getClass()) return false; 
    
    // 3. 객체 캐스팅 및 핵심 식별 필드 비교
    Member member = (Member) o;
    return Objects.equals(id, member.id); 
}

@Override
public int hashCode() {
    // equals에서 비교 기준으로 사용한 필드(id)와 동일하게 해시코드 생성
    return Objects.hash(id); 
}
```

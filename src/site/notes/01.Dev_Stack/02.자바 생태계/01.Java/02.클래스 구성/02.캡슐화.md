# 객체 지향의 정수: 캡슐화와 정보 은닉
> [!summary] 핵심 요약
> 
> - **캡슐화**: 객체의 상태(필드)와 행위(메서드)를 하나로 묶고 구현 세부 사항을 외부에 감추는 설계 원칙
>     
> - 목적: ==데이터 무결성 보호== 및 객체 간 결합도 완화에 따른 유지보수성 향상
>     
> - 핵심: 접근 제한자를 통한 정보 은닉과 ==Tell, Don't Ask(TDA)== 원칙 기반의 행위 중심 설계
>     
## 캡슐화의 목적 및 접근 제어
### 1. 주요 기대 효과
- **데이터 보호**: 외부의 직접적인 필드 수정 차단으로 객체의 일관된 상태 유지
- **유지보수 용이**: 내부 구현 변경 시 외부 인터페이스 유지로 수정 범위 최소화
- **코드 안정성**: 유효성 검사(Validation)를 통한 부정확한 데이터 주입 사전 방지
### 2. 접근 제한자 (Access Modifier)

|**접근 제한자**|**클래스 내부**|**동일 패키지**|**자식 클래스**|**전체 접근**|
|---|---|---|---|---|
|`public`|✅|✅|✅|==✅==|
|`protected`|✅|✅|==✅==|❌|
|`default`|✅|==✅==|❌|❌|
|`private`|==✅==|❌|❌|❌|

## 심화: Tell, Don't Ask (TDA) 및 설계 최적화
### 1. TDA 원칙 기반의 행위 중심 설계
- **개념**: 객체에게 상태를 묻지(Ask) 말고, 원하는 작업을 수행하도록 ==명령(Tell)==하는 원칙
- **지향점**: 데이터 중심의 빈약한 모델에서 행위 중심의 ==풍부한 도메인 모델(Rich Domain Model)==로 전환

|**구분**|**Ask (상태 요청)**|**Tell (행위 명령)**|
|---|---|---|
|**제어 주체**|외부 서비스/클라이언트 로직|객체 스스로 자신의 상태 제어|
|**결합도**|높음 (객체 필드 변경 시 외부 파손)|낮음 (내부 구조 변경이 외부에 무영향)|
|**응집도**|낮음 (데이터와 로직의 분리)|높음 (데이터와 로직의 통합)|

### 2. 과도한 Getter/Setter 지양 전략
- **Getter 절제**: 상태 비교 로직을 객체 내부 메서드로 이동하여 정보 노출 최소화
- ==Setter 금지==: 단순 값 변경 대신 비즈니스 의도가 담긴 의미 있는 메서드 명칭 사용
- **기술적 예외**: DTO(데이터 전송 객체) 및 프레임워크 규약(JPA, Jackson 등)에 따른 필수 접근자만 제한적 허용
---
## 설계 결정 매트릭스 (Decision Matrix)
```plaintext
Encapsulation Decision Tree
 ├─ 대상이 비즈니스 핵심 로직인가?
 │   ├─ Yes: TDA 원칙 적용 (Tell)
 │   └─ No: DTO/단순 값 객체 (Getter 허용)
 ├─ 프레임워크 규약을 따르는가? (JPA, Jackson 등)
 │   ├─ Yes: 최소한의 Getter/Setter 제공
 │   └─ No: 불변 객체 및 생성자 주입 선호
 └─ 외부에서 객체 상태를 비교/가공하는가?
     ├─ Yes: 해당 로직을 객체 내부로 이동 (리팩토링)
     └─ No: 현재 캡슐화 수준 유지
```

## 코드 구현 모델 (Java)
### 1. 기본 캡슐화 및 유효성 검사
```java
public class Student {
    private int studentNo; // 외부 접근 차단

    // 단순 설정자가 아닌 검증 로직 포함
    public void setStudentNo(int studentNo) {
        if (studentNo <= 0) {
            throw new IllegalArgumentException("학번은 양수여야 합니다.");
        }
        this.studentNo = studentNo;
    }
}
```
### 2. TDA 원칙 적용 (Refactoring)
```java
// 비권장 사례
// 데이터를 꺼내온다 (Ask)
int currentPoints = user.getPoints(); 

if (currentPoints >= 1000) {
    user.setPoints(currentPoints - 1000); 
} else {
    throw new IllegalStateException("포인트가 부족합니다.");
}

// 권장 사례
// 객체에게 할 일을 시킨다 (Tell)
user.usePoints(1000); 

// User 클래스 내부 (캡슐화)
public class User {
    private int points;

    public void usePoints(int amount) {
        // 내부에서 스스로 판단 (객체의 자율성)
        if (this.points < amount) {
            throw new IllegalStateException("포인트가 부족합니다.");
        }
        // 내부에서 스스로 상태 변경
        this.points -= amount;
    }
}
```

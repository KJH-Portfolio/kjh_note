# Java 클래스 및 객체 지향 설계 구조
> [!summary] 핵심 요약
> 
> - **개념**: 클래스(설계도), 객체(실체), 내부 클래스의 상관관계 및 Java 객체 지향 메커니즘
>     
> - **왜?**: 데이터와 기능을 캡슐화하여 코드 재사용성, 응집도 증대 및 유지보수 효율성 극대화
>     
> - **언제?**: 애플리케이션의 도메인 모델을 설계하거나, 특정 객체에 종속적인 보조 논리를 구현할 때
>     
> - **핵심**: ==Heap(객체)과 Stack(참조 변수)==의 물리적 분리 및 변수·내부 클래스의 생명주기 파악
>     
## 1. 클래스와 객체의 정의 및 메모리 구조
- **클래스 vs 객체 핵심 비교**
    - 데이터 타입과 실제 데이터(상태)의 명확한 역할 분리
    - **어떻게?**: `new` 키워드를 통해 설계도를 실제 메모리에 동적 할당하여 인스턴스화 수행

|**구분**|**클래스 (Class)**|**객체 (Object / Instance)**|
|---|---|---|
|**정의**|객체 생성을 위한 추상적 ==설계도==|클래스 기반으로 메모리에 구현된 ==실체==|
|**메모리 할당**|코드 영역(Method Area)에 정보 로드|실행 시 ==Heap 영역==에 동적 할당|
|**변수 관계**|데이터 타입 역할 (`Person p`)|참조 변수(Stack)를 통한 주소 접근|
|**저장 데이터**|데이터 구조 및 종류 정의|실제 상태 값(필드 데이터) 저장|

```plaintext
Object Creation & Memory Mapping
 ├─ 1. Class Loading: JVM Method Area 내 클래스 메타데이터 로드
 └─ 2. Instantiation (new keyword): Heap 메모리 확보 및 인스턴스화
     ├─ [Heap Area] 실제 객체 데이터 저장 및 고유 주소값 생성
     └─ [Stack Area] 참조 변수에 해당 주소(Reference) 할당
         └─ 결과: 참조 변수를 통한 객체 간접 제어 구조 확립
```
## 2. 클래스 구성 요소 및 제어 예약어 체계
- **클래스 3대 구성 요소**
    - 객체의 속성과 동작을 정의하는 기본 단위
    - **어떻게?**: 객체의 고유 데이터는 필드에, 동작 로직은 메서드에 분리하여 작성

|**요소**|**역할**|**특징**|
|---|---|---|
|**필드 (Field)**|객체의 상태 및 속성 정보 저장|클래스 내 전역 멤버 변수|
|**생성자 (Constructor)**|객체 생성 시 ==초기화== 로직 담당|클래스명과 동일, 반환 타입 부재|
|**메서드 (Method)**|객체의 기능 및 동작 정의|호출 시 Stack Frame 생성 및 실행|

- **주요 제어 예약어 활용**
    - 클래스, 메서드, 필드에 부여하는 특수 권한 및 제약

|**예약어**|**클래스 적용 특성**|**메서드/필드 적용 특성**|
|---|---|---|
|`static`|정적 내부 클래스 선언 시 사용|==공유 멤버== (객체 생성 없이 호출 가능)|
|`final`|==상속 불가== 클래스 지정|오버라이딩 금지 및 값 변경 불가 상수|
|`abstract`|미구현 메서드를 포함한 추상 클래스|바디(`{}`) 없이 선언부만 존재하는 메서드|

## 3. 변수 유형별 생명주기 및 메모리 특성
- **메모리 할당 및 소멸 기준**
    - 변수의 선언 위치 및 키워드에 따른 생명주기 차이 파악
    - **어떻게?**: 공용 데이터는 `static`으로, 개별 상태는 인스턴스 변수로, 임시 연산은 지역 변수로 분리하여 설계

|**종류**|**생성 시점**|**소멸 시점**|**저장 위치**|**특징**|
|---|---|---|---|---|
|**클래스 변수 (`static`)**|프로그램 시작 시|프로그램 종료 시|Static 영역|==모든 객체가 공유==하는 공용 데이터|
|**상수 필드 (`final`)**|선언/생성 시|프로그램 종료 시|상수 풀/Heap|명명 규칙으로 `UPPER_CASE` 준수|
|**인스턴스 변수(필드)**|객체 생성 시|객체 소멸(GC) 시|Heap|객체별 독립적 상태 유지|
|**지역 변수**|메서드 실행 시|메서드 종료 시|Stack|==사용 전 반드시 초기화== 필수|

```java
public class VariableScope {
    // 1. 클래스 변수 (static)
    public static int globalCount = 100;

    // 2. 상수 필드 (final)
    public final String TYPE = "JAVA_CORE";

    // 3. 인스턴스 변수
    public String instanceName;

    public void process(int param) {
        // 4. 지역 변수 (반드시 초기화 필요)
        int localVal = 0; 
        System.out.println(localVal + param);
    }
}
```
## 4. 객체 지향 설계 구현 및 주의사항
- **생성자 및 `this` 활용 메커니즘**
    - `this` 참조: 현재 인스턴스 자기 자신의 주소를 가리키는 내부 포인터
    - `this()` 체이닝: ==생성자 내 첫 줄==에서만 허용되는 타 생성자 호출 기능
    - 생성자 오버로딩: 매개변수 구성에 따른 다중 정의로 객체 생성의 유연성 확보
- **개발 시 핵심 준수 사항**
    - `static` 변수 수정 시 전체 인스턴스에 미치는 ==데이터 오염== 주의
    - `final` 필드 사용 시 명시적 초기화 또는 생성자를 통한 값 할당 강제
    - 지역 변수는 컴파일러 자동 초기화가 미지원되므로 ==수동 초기화== 후 사용
    - **어떻게?**: 생성자를 통한 강제 초기화 및 접근 제어자(`private`)를 통한 무분별한 상태 변경 차단
```java
public class Academy {
    private int studentNo;
    private String name;

    // 기본 생성자: 생성자 체이닝 활용
    public Academy() {
        this(0, "Unknown"); 
    }

    // 매개변수 생성자: 필드 초기화 전담
    public Academy(int studentNo, String name) {
        this.studentNo = studentNo;
        this.name = name;
    }

    // 메서드: this를 통한 필드/매개변수 식별
    public void setName(String name) {
        this.name = name; 
    }
}
```
## 5. 내부 클래스 (Inner Class) 구조 및 활용
- **내부 클래스의 개념과 목적**
    - 클래스 내부에 선언된 클래스로, 두 클래스 간의 긴밀한 종속 관계를 표현
    - 외부 클래스의 멤버에 쉽게 접근 가능하며, 불필요한 클래스 노출을 막아 ==캡슐화== 달성
    - **어떻게?**: 선언 위치와 `static` 키워드 유무에 따라 4가지 형태로 나누어 용도에 맞게 선언

|**종류**|**선언 위치**|**특징 및 제약 사항**|
|---|---|---|
|**인스턴스 내부 클래스**|외부 클래스 멤버 필드 위치|외부 객체 생성 후 인스턴스화 가능, 외부의 모든 멤버 접근 허용|
|**정적(Static) 내부 클래스**|외부 클래스 멤버 필드 위치|외부 객체 없이 생성 가능, 외부의 `static` 멤버만 접근 가능|
|**지역(Local) 내부 클래스**|메서드 또는 초기화 블록 내부|메서드 실행 시 생성/종료 시 소멸, ==외부 지역 변수는 `final`만 접근==|
|**익명(Anonymous) 내부 클래스**|메서드 내부 또는 변수 선언부|==이름 없는 일회용 클래스==, 인터페이스 구현 객체 즉시 생성 시 활용|

```plaintext
[ 내부 클래스 생성 및 참조 제약 도식화 ]

 외부 객체 (Outer Object)
 ┌─────────────────────────────────────────────────────┐
 │                                                     │
 │   인스턴스 내부 객체 (Instance Inner)               │ ◀─ 외부 객체가 반드시 먼저 존재해야 함
 │   ┌─────────────────────────────────────────────┐   │
 │   │ - 외부 클래스의 private 변수까지 직접 접근  │   │
 │   └─────────────────────────────────────────────┘   │
 │                                                     │
 └─────────────────────────────────────────────────────┘

 정적 내부 객체 (Static Inner Object) ◀─── 외부 객체 존재 유무와 무관하게 독립적 생성 가능
 ┌─────────────────────────────────────────────┐ (단, 외부의 인스턴스 멤버는 참조 불가)
 │ - 외부 클래스의 static 변수만 접근 가능     │
 └─────────────────────────────────────────────┘
```
```java
public class OuterClass {
    private int outerValue = 10;
    private static int staticOuterValue = 20;

    // 1. 인스턴스 내부 클래스 (Instance Inner Class)
    class InstanceInner {
        void display() {
            // 외부 클래스의 인스턴스 변수에 직접 접근 가능
            System.out.println("Outer Value: " + outerValue);
        }
    }

    // 2. 정적 내부 클래스 (Static Inner Class)
    static class StaticInner {
        void display() {
            // 외부 클래스의 static 변수만 접근 가능 (outerValue는 에러 발생)
            System.out.println("Static Outer Value: " + staticOuterValue);
        }
    }

    public void processMethod() {
	    // 컴파일러에 의해 사실상 final 처리됨(아래 내부 클래스에서 사용)
        (final) int localVar = 30;

        // 3. 지역 내부 클래스 (Local Inner Class)
        class LocalInner {
            void display() {
                System.out.println("Local Var (final): " + localVar);
            }
        }
        
        LocalInner local = new LocalInner();
        local.display();
    }
    
    // 4. 익명 내부 클래스 (Anonymous Inner Class) - 주로 이벤트 처리나 스레드에 사용
    Runnable runner = new Runnable() {
        @Override
        public void run() {
            System.out.println("Anonymous Inner Class Executed");
        }
    };
    
	// 추가 - 4번(익명 내부 클래스) 람다식으로 변경(최신 문법)
	Runnable runner = () -> System.out.println("Anonymous Inner Class Executed");
}
```
#### 추가 사항
- 매서드 내 지역 내부 클래스(+람다식)에서 지역 변수를 사용할 경우, 해당 변수는 final로 인식됨 (명시적으로 적는 것이 좋다)

# 📘 Java 메서드 참조 (Method Reference, `::`)
> [!summary] 핵심 요약
> 
> - Java 8에 도입된 람다 표현식(Lambda Expression)의 간결한 표현 기법
>     
> - 목적: 불필요한 매개변수 선언 제거 및 코드 가독성 극대화
>     
> - 핵심: ==함수형 인터페이스(Functional Interface) 환경==에서의 단일 메서드 호출 축약
>     
## 메서드 참조 핵심 개념
- **문법 구조**
    - 기호 `::` (이중 콜론, Double Colon) 활용
    - `클래스명::메서드명` 또는 `참조변수::메서드명` 형태
    - ==매개변수와 괄호 `()` 생략== (컴파일러의 타입 추론 의존)
## 메서드 참조 4가지 유형 비교
- **유형별 람다식 변환 규칙**

| **구분**            | **람다 표현식 (Lambda)**      | **메서드 참조 (::)** | **특징**                     |
| ----------------- | ------------------------ | --------------- | -------------------------- |
| **정적 메서드 참조**     | `(x) -> Class.method(x)` | `Class::method` | 정적(Static) 메서드 직접 호출       |
| **특정 객체 인스턴스 참조** | `(x) -> obj.method(x)`   | `obj::method`   | 기 생성된 외부 인스턴스의 메서드 호출      |
| **임의 객체 인스턴스 참조** | `(x, y) -> x.method(y)`  | `Class::method` | ==첫 번째 매개변수가 메서드 수신자==로 동작 |
| **생성자 참조**        | `(x) -> new Class(x)`    | `Class::new`    | 새로운 객체 인스턴스 생성 및 반환        |

## 동작 원리 및 적용 조건
- **타입 추론(Type Inference) 메커니즘**
    - 타겟 타입(Target Type)에 기반한 컴파일러의 자동 매개변수 매핑
    - 타겟 인터페이스 추상 메서드의 매개변수 타입, 개수, 반환 타입과의 완벽한 일치 요구
- **사용 시 제약 사항**
    - 전달받은 매개변수의 내부 가공, 추가 연산, 예외 처리가 필요한 경우 ==사용 불가== (일반 람다식 작성 필수)
## 코드 구현 예시
- **유형별 적용 사례**
```java
// 1. 정적 메서드 참조 (Integer.parseInt)
Function<String, Integer> f1 = s -> Integer.parseInt(s);
Function<String, Integer> f2 = Integer::parseInt;

// 2. 특정 객체의 인스턴스 메서드 참조 (System.out.println)
Consumer<String> c1 = s -> System.out.println(s);
Consumer<String> c2 = System.out::println;

// 3. 임의 객체의 인스턴스 메서드 참조 (String.equals)
// 첫 번째 매개변수(s1)가 메서드 수신자가 되고, 두 번째 매개변수(s2)가 인자로 전달됨
BiPredicate<String, String> p1 = (s1, s2) -> s1.equals(s2);
BiPredicate<String, String> p2 = String::equals;

// 4. 생성자 참조 (ArrayList::new)
Supplier<List<String>> s1 = () -> new ArrayList<>();
Supplier<List<String>> s2 = ArrayList::new;
```

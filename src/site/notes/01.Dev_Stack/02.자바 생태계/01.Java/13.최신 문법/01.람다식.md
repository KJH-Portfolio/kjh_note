# Java 람다식(Lambda Expression) 실무 활용 패턴
> [!summary] 핵심 요약
> 
> - 자바에서 "함수를 값처럼 전달"하는 람다식의 주요 활용처 및 설계 패러다임
>     
> - 목적: 불필요한 익명 클래스 작성을 배제하고, 핵심 비즈니스 로직(What)에 집중하여 코드 가독성 향상
>     
> - 핵심: 컬렉션/Stream API 연계 및 ==동작 파라미터화(Behavior Parameterization)== 구현
>     
## 람다식 핵심 도입 목적
- **패러다임의 전환**
    - 데이터를 다루는 관점이 객체 중심에서 ==함수(동작) 중심==으로 이동
- **주요 활용 상황**
    - **데이터 가공**: 대량의 데이터 컬렉션을 필터링, 변환, 집계할 때 (Stream API 기반)
    - **동작 파라미터화**: 특정 조건이나 로직 자체를 메서드의 인자로 넘겨야 할 때 (Strategy Pattern 구현)
    - **지연 실행(Lazy Execution)**: 로직을 즉시 실행하지 않고, 특정 이벤트나 조건 발생 시점의 콜백(Callback)으로 지정할 때
## 실무 주요 활용 사례 (Top 5)

|**활용 분야**|**주요 메서드/인터페이스**|**특징 및 목적**|
|---|---|---|
|**1. 컬렉션 조작**|`forEach()`, `removeIf()`|명시적 반복문(for, while) 없이 간결한 순회 및 조건부 요소 삭제 처리|
|**2. Stream API**|`filter()`, `map()`, `collect()`|데이터 파이프라인 구축 시 필수 요소 (람다식과 가장 강력한 시너지 발휘)|
|**3. 정렬 (Sorting)**|`Comparator`|복잡한 객체 비교 로직을 한 줄의 수학적 표현식으로 압축|
|**4. 멀티스레딩**|`Runnable`|비동기 실행을 위한 작업 단위(Task)를 초경량으로 정의 및 실행|
|**5. 이벤트 핸들링**|`ActionListener` 등|GUI 환경 등에서 사용자 액션(클릭 등)에 따른 반응형 로직 직관적 구현|

```java
// [1] 컬렉션 조작 (Collection)
List<Integer> scores = new ArrayList<>(Arrays.asList(90, 75, 85, 60, 100));
scores.forEach(score -> System.out.println(score)); // 전체 순회 출력
scores.removeIf(score -> score < 80);               // 80점 미만 조건부 삭제

// [2] Stream API 활용 (필터링 및 변환 파이프라인)
List<String> names = Arrays.asList("김철수", "이영희", "김민수");
List<String> result = names.stream()
    .filter(name -> name.startsWith("김"))  // '김'씨 성을 가진 요소 필터링
    .map(name -> name + "님")               // 문자열 변환
    .collect(Collectors.toList());          // 최종 리스트로 수집

// [3] 정렬 (Comparator)
// 기존 익명 클래스 대비 직관적인 나이 오름차순 정렬 로직
people.sort((p1, p2) -> p1.getAge() - p2.getAge()); 

// [4] 스레드(Thread) 생성 및 실행 (Runnable 구현)
new Thread(() -> System.out.println("비동기 작업 실행!")).start();

// [5] GUI 이벤트 핸들링 (버튼 클릭 콜백)
JButton button = new JButton("Click");
button.addActionListener(e -> System.out.println("버튼 클릭됨!"));
```

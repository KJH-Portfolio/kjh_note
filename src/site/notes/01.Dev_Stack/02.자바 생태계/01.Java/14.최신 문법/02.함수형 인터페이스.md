# 📘 Java 함수형 인터페이스 (Functional Interface)
> [!summary] 핵심 요약
> 
> - Java 8에 도입된 단일 추상 메서드(SAM, Single Abstract Method) 기반의 인터페이스 구조
>     
> - 목적: 람다 표현식(Lambda)과 메서드 참조(Method Reference)를 일급 객체처럼 다루기 위한 타입 체계 제공
>     
> - 핵심: 컴파일러 검증을 위한 `@FunctionalInterface` 어노테이션 활용 및 ==추상 메서드 1개 제한== 원칙 준수
>     
## 핵심 개념 및 구성 조건
- **기본 정의 및 규칙**
    - 오직 1개의 추상 메서드만 선언 가능 (람다식의 타겟 타입으로 1:1 매칭되어 작동)
    - `default` 메서드, `static` 메서드의 선언 개수는 무제한 (기존 인터페이스 기능 확장용)
    - `java.lang.Object`의 `public` 메서드(`equals`, `toString` 등) 선언은 ==추상 메서드 카운트에서 제외==
    - `@FunctionalInterface` 어노테이션 누락 시 ==컴파일러의 인터페이스 구조 위반(SAM 위반) 검증 불가==
## 구조 분석 (Structure)
```plaintext
Functional Interface Structure
 ├─ Mandatory Component (필수)
 │   └─ Single Abstract Method (SAM): 람다식 본문과 연결되는 유일한 추상 메서드
 │
 ├─ Optional Components (선택 - 개수 제한 없음)
 │   ├─ Default Methods: 하위 호환성 및 공통 로직 제공 (구현부 존재)
 │   ├─ Static Methods: 인스턴스 생성 없이 호출 가능한 유틸리티 기능 (구현부 존재)
 │   └─ Object Methods: equals(), hashCode() 등의 재선언 (SAM 카운트 무시)
 │
 └─ Validation Marker (검증)
     └─ @FunctionalInterface: 컴파일 타임의 SAM(단일 추상 메서드) 제약 조건 강제
```
## 주요 표준 함수형 인터페이스 (`java.util.function`)
- **자주 사용되는 5대 핵심 인터페이스**

| **인터페이스명**       | **추상 메서드**    | **파라미터 / 반환 타입**     | **주 용도**                     |
| ---------------- | ------------- | -------------------- | ---------------------------- |
| `Runnable`       | `run()`       | 무 / 무 (`void`)       | 매개변수 없이 단일 작업만 실행            |
| `Supplier<T>`    | `get()`       | 무 / 유 (`T`)          | 파라미터 없이 특정 타입의 데이터 생성 및 반환   |
| `Consumer<T>`    | `accept(T t)` | 유 (`T`) / 무 (`void`) | 전달받은 데이터를 소비하고 반환하지 않는 내부 처리 |
| `Function<T, R>` | `apply(T t)`  | 유 (`T`) / 유 (`R`)    | T 타입을 R 타입으로 변환 및 데이터 매핑     |
| `Predicate<T>`   | `test(T t)`   | 유 (`T`) / `boolean`  | 조건 검사 및 참/거짓(boolean) 상태 판별  |

## 코드 구현 예시
- **커스텀 함수형 인터페이스 설계 및 활용**
```java
@FunctionalInterface
interface CustomCalculator {
    // 1. 단 하나의 추상 메서드 (필수)
    int calculate(int x, int y);

    // [에러 발생] 추상 메서드 추가 시 컴파일 에러 (@FunctionalInterface 검증)
    // double add(double a, double b); 

    // 2. default 메서드 허용 (선택)
    default void printResult(int result) {
        System.out.println("Result: " + result);
    }

    // 3. static 메서드 허용 (선택)
    static void printVersion() {
        System.out.println("Calculator v1.0");
    }
}

public class Main {
    public static void main(String[] args) {
        // 람다식을 통한 함수형 인터페이스 구현 및 인스턴스화
        CustomCalculator addCalc = (x, y) -> x + y;
        
        int sum = addCalc.calculate(10, 20); // 30 반환
        addCalc.printResult(sum);            // default 메서드 호출
        CustomCalculator.printVersion();     // static 메서드 호출
    }
}
```

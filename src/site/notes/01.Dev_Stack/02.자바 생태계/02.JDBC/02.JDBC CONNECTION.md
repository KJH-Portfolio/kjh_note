# 📘 JDBC 핵심 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - JDBC의 구성 요소와 동작 절차를 기차 여행 비유로 직관적으로 정리
> - DriverManager, Connection, Statement, PreparedStatement, ResultSet의 역할 및 차이 제시
> - 표·흐름도·코드 예시로 실무 흐름 이해 강화

---

## 목적 🎯

- **데이터베이스 연동 목적:** 자바 애플리케이션에서 DBMS와 연결하여 SQL 질의 실행 및 결과 처리 수행
- **표준 인터페이스 목적:** 데이터베이스 종류와 상관없이 일관된 API로 접속·질의·갱신 처리 수행
- **보안·성능 목적:** PreparedStatement를 통해 SQL 인젝션 방지와 실행 성능 최적화 달성

---

## 핵심 개념 💡

### ✅ JDBC 주요 객체 및 역할 비교

| 객체                | 핵심 역할           | 주요 메서드                               | 비유          | 비고              |
| ----------------- | --------------- | ------------------------------------ | ----------- | --------------- |
| DriverManager     | 드라이버 등록과 연결 생성  | Class.forName,<br>getConnection      | 기차역 매표소     | 정적 유틸리티 클래스     |
| Connection        | DB와의 세션(연결) 관리  | createStatement,<br>prepareStatement | 기차          | 트랜잭션 제어 가능      |
| Statement         | 단순 SQL 실행       | executeQuery, <br>executeUpdate      | 승무원에게 자유 질문 | SQL을 문자열로 직접 전달 |
| PreparedStatement | 파라미터 바인딩·미리 컴파일 | setXxx, <br>executeQuery/Update      | 양식 있는 질문지   | 보안·성능 유리        |
| ResultSet         | SELECT 결과 접근    | next, <br>getXxx                     | 질문에 대한 답변지  | 커서 기반 순회        |

> **핵심 포인트:** Statement는 즉석 실행, PreparedStatement는 미리 컴파일 및 파라미터 바인딩 지원

---

## 특징 ✨

### ✅ JDBC 동작 흐름 특징

- **드라이버 등록:** Class.forName으로 JDBC 드라이버 로딩
- **연결 생성:** DriverManager.getConnection으로 Connection 획득
- **질의 실행:** Statement/PreparedStatement를 통해 SQL 실행
- **결과 처리:** ResultSet 커서 이동 후 컬럼 값 추출
- **자원 관리:** ResultSet → Statement → Connection 순서로 닫기

### ✅ Statement vs PreparedStatement 비교

|항목|Statement|PreparedStatement|
|---|---|---|
|SQL 구성 방식|문자열 직접 작성|위치홀더(?)로 템플릿 작성|
|컴파일 시점|매 실행 시|최초 준비 시(재사용 가능)|
|보안성|인젝션 위험 높음|파라미터 바인딩으로 안전|
|성능|단건·단발 작업 적합|반복 실행·대량 작업 유리|
|가변 값 처리|문자열 연결로 처리|setXxx 메서드로 안전 처리|

---

## 팁과 주의점 ⚠️

- **자원 반납 필수:** ResultSet/Statement/Connection는 반드시 close 호출
- **예외 처리 필수:** ClassNotFoundException, SQLException 처리 필수
- **트랜잭션 관리:** 자동 커밋 비활성화 후 명시적 commit/rollback 권장
- **SQL 인젝션 방지:** 가변 입력값은 PreparedStatement 사용
- **컬럼 접근 안정성:** getXxx는 컬럼명 기반 접근 권장(인덱스는 스키마 변경에 취약)
- **드라이버·URL 정확성:** DBMS별 드라이버 클래스와 JDBC URL 포맷 확인 필요
- **배치 작업 최적화:** addBatch/executeBatch로 여러 문장 일괄 처리
- **로그·타임아웃 설정:** 성능 문제 및 장애 대응을 위해 연결·읽기 타임아웃 설정 권장

## 예시 📌

#### ✅ 흐름도
```java
//사전 준비
[Driver 등록]
    └─ Class.forName("...Driver")
-------------------------------------------------

[연결 생성]
    └─ DriverManager.getConnection(url, user, pw)
        └─ Connection

[질의 수행]
    ├─ Connection.createStatement → executeQuery/Update
    └─ Connection.prepareStatement → setXxx → execute...

[결과 처리]
    └─ ResultSet.next → getXxx

[자원 반납]
    └─ ResultSet.close
    └─ Statement.close
    └─ Connection.close
```

#### ✅ PreparedStatement 예시

```java
Connection conn = null;
PreparedStatement pstmt = null;

try {
    // 1) JDBC 드라이버 등록
    Class.forName("oracle.jdbc.OracleDriver");

    // 2) Connection 객체 생성 (DB 연결)
    conn = DriverManager.getConnection(
                "jdbc:oracle:thin:@localhost:1521:xe",
                "JDBC",
                "JDBC"
           );
    conn.setAutoCommit(false); // 트랜잭션 수동 제어

    // 3) Statement 객체 생성 (SQL 실행 준비)
    pstmt = conn.PrepareStatement(sql);
       
    // ※ 여기서부터 DML 실행부 (executeUpdate, commit/rollback)은 제외
	  pstmt.set자료형(순번,값);

} catch {} finally{} //예외처리, 자원반납
}
```

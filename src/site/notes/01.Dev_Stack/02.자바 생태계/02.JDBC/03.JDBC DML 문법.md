# 📘 JDBC DML 실행부 정리

## Summary 📝

> [!summary] Summary 📝
> 
> - `executeUpdate` 메서드로 DML(INSERT/UPDATE/DELETE) 실행
> - 반환값을 통해 처리된 행 수 확인
> - 결과에 따라 `commit` 또는 `rollback` 수행

---

## 목적 🎯

- **데이터 조작 목적:** INSERT, UPDATE, DELETE 같은 DML 구문을 실행하여 DB 데이터를 변경
- **트랜잭션 제어 목적:** 실행 결과에 따라 데이터 변경을 확정하거나 취소

---

## 핵심 개념 💡

### ✅ DML 실행부 주요 메서드

| 메서드                  | 역할        | 반환값/동작          |
| -------------------- | --------- | --------------- |
| `executeUpdate(sql)` | DML 구문 실행 | 처리된 행 수(int) 반환 |
| `commit()`           | 트랜잭션 확정   | DB에 변경사항 반영     |
| `rollback()`         | 트랜잭션 취소   | 변경사항 되돌림        |

---

## 특징 ✨

- **성공 여부 판단:** `executeUpdate` 결과값이 0보다 크면 성공으로 간주
- **트랜잭션 처리:** `setAutoCommit(false)` 상태에서 반드시 `commit` 또는 `rollback` 호출 필요
- **실패 처리:** 예외 발생 시 `rollback` 수행하여 데이터 무결성 보장

---

## 팁과 주의점 ⚠️

- **반드시 예외 처리:** `SQLException` 발생 시 `rollback` 호출
- **자원 반납 전 트랜잭션 처리 완료 필수**
- **PreparedStatement 권장:** 문자열 연결 방식은 SQL 인젝션 위험 존재

---

## 예시 📌

### ✅ DML 실행부 코드

```java
// DML 실행부 예시
pstmt.set자료형(순번,값); //pstmt의 경우 위치홀더 채워주기
int result = pstmt.excuteUpdate();  // INSERT/UPDATE/DELETE 실행, 처리된 행 수 반환
 

if (result > 0) {
    conn.commit();   // 성공 시 트랜잭션 확정
    System.out.println("처리 성공!");
} else {
    conn.rollback(); // 실패 시 트랜잭션 취소
    System.out.println("처리 실패!");
}
```

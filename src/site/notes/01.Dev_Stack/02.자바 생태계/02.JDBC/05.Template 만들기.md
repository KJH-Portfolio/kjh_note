# 📘 JDBCTemplate 클래스 (드라이버 로드 + 이모지 복원 버전)

## Summary 📝
> [!summary] Summary 📝
> - JDBC 반복 작업을 **static 메서드**로 추출한 **재사용 템플릿 클래스**  
> - 연결 생성, 자원 반납, 트랜잭션 처리를 **일관성 있게 캡슐화**

---
## 목적 🎯
- **DAO 계층 코드 중복 제거**  
- **자원 관리 실수 방지**  
- **트랜잭션 일관성 유지**  
- **예외 처리 통일**

---
## 핵심 개념 💡
### ✅ 1. `getConnection()` - DB 연결 생성 (드라이버 로드 포함)
```java
// Oracle JDBC 드라이버 명시적 로드
Class.forName("oracle.jdbc.driver.OracleDriver");

// Oracle DB 연결 + 자동 커밋 해제
conn = DriverManager.getConnection("jdbc:oracle:thin:@localhost:1521:xe", "JDBC", "JDBC");
conn.setAutoCommit(false); // 수동 트랜잭션 시작
```

### ✅ 2. `close()` - 자원 안전 반납
| 객체           | 반납 조건                    | 비고                           |
| ------------ | ------------------------ | ---------------------------- |
| `Connection` | `!= null && !isClosed()` | 가장 마지막에 반납                   |
| `Statement`  | 동일                       | `PreparedStatement` 포함 (다형성) |
| `ResultSet`  | 동일                       | **가장 먼저 반납 필수**              |

### ✅ 3. `commit()` / `rollback()` - 트랜잭션 제어
```java
conn.commit();  // 성공 → 영구 반영
conn.rollback(); // 실패 → 변경 취소
```

---
## 특징 ✨
### ✅ **모든 메서드 `static`**
- 객체 생성 없이 `JDBCTemplate.메서드()` 호출 가능

### ✅ **자원 누수 방지 로직 내장**
- `null` 체크 + `isClosed()` 검증 → **이중 안전장치**

### ✅ **오버로딩 활용**
```java
close(Statement stmt) → PreparedStatement도 처리 가능
```

### ✅ **트랜잭션 수동 관리**
- `setAutoCommit(false)` → 명시적 `commit`/`rollback` 강제

### ✅ **드라이버 명시적 로드**
- `Class.forName()` 추가 → **클래스 로딩 보장**

---
## 팁과 주의점 ⚠️
- **자원 반납 순서**: `ResultSet → Statement → Connection`  
  > 순서 틀리면 `SQLException` 발생 가능

- **`isClosed()` 필수 확인**  
  > 이미 닫힌 자원 재반납 시 예외 발생

- **예외 처리 위치**  
  > `DAO`에서 `try-catch-finally` 사용 권장  
  > `finally` 블록에 `close()` 호출

- **드라이버 JAR 필수**  
  > `ojdbc8.jar` 또는 `ojdbc11.jar` 클래스패스 포함

## 예시 📌
### ✅실제 예시
```java
package com.kh.book.template;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Statement;

public class JDBCTemplate {
	
	//1.Connection 객체 생성 메서드
	public static Connection getConnection() {
		Connection conn = null; //선언
		try {
			//객체 생성
			conn = DriverManager.getConnection("jdbc:oracle:thin:@localhost:1521:xe","JDBC","JDBC");
			//자동 커밋 해제
			conn.setAutoCommit(false);
		} catch (SQLException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
		return conn;
	}
	
	//2.자원반납 메서드
	//각 객체는 null이 아닌지 자원이 이미 반납된 것은 아닌지 확인하여 처리하기
	//isClosed() - 닫혀있다면 true 아니라면 false 반환 메서드
	
	//2-1)Connection 반납
	public static void close(Connection conn) {
		//조건처리하여 반납
		try {
			if(conn != null && !conn.isClosed()){ //null이거나 닫혀있지 않다면
				conn.close(); //자원반납
			}
		} catch (SQLException e) {
			e.printStackTrace();
		}
	}
	
	//2-2)Statement객체 반납(PreparedStatement)
	//PreparedStatement가 상속관계이기에 다형성 적용가능
	public static void close(Statement stmt) {
		
		try {
			if(stmt != null && !stmt.isClosed()) {
				stmt.close();
			}
		} catch (SQLException e) {
			e.printStackTrace();
		}
	}
	
	//2-3)Resultset객체 반납
	public static void close(ResultSet rset) {
		
		try {
			if(rset != null && !rset.isClosed()) {
				rset.close();
			}
		} catch (SQLException e) {
			e.printStackTrace();
		}
	}
	
	//3 트랜잭션 메서드
	//commit
	public static void commit(Connection conn) {
		
		try {
			if(conn!=null && !conn.isClosed()) {
				conn.commit();
			}
		} catch (SQLException e) {
			e.printStackTrace();
		}
	}
	
	//rollback
	public static void rollback(Connection conn) {
		
		try {
			if(conn!=null && !conn.isClosed()) {
				conn.rollback();
			}
		} catch (SQLException e) {
			e.printStackTrace();
		}
	}	
}

```

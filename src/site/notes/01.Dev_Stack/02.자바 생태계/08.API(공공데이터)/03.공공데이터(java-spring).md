# 데이터 통신 아키텍처 및 API 연동 방식
> [!summary] 요약: Spring Boot(WebClient)와 React(Axios) 환경에서의 외부 API 데이터 수신 및 처리 구조.
## Spring Boot 기반 서버 사이드 요청 (WebClient)
- **WebClient 활용 비동기/동기 통신**
    - 스프링 부트 5부터 권장되는 논블로킹(Non-blocking) 방식의 HTTP 요청 모듈.
    - `builder()` 패턴을 통한 설정 및 `uriBuilder`를 활용한 동적 쿼리 파라미터 생성.
- **보안성 및 데이터 무결성 확보**
    - 백엔드 내부에서 API 서비스 키를 관리하여 클라이언트 측 노출 원천 차단.
    - 수신 데이터를 DTO(Data Transfer Object)로 매핑하여 DB 저장 및 비즈니스 로직 연동 용이.
- **처리 흐름**
    - 외부 API 엔드포인트 설정 → 쿼리 파라미터 조합 → 응답 데이터(Mono/Flux) 수신 → 객체 변환 및 반환.
```java
import ...

@Service
public class WeatherService {
    public List<ForecastDto> getForecastList() {
        // // 보안 및 환경 설정을 위한 서비스 키 정의
        String serviceKey = "0520e76efb72e41ae374ba77a910d0264246d16b23c171e4e817e576b2a1f52d";
        // // WebClient 인스턴스 생성 및 기본 URL 설정
        WebClient webClient = WebClient.builder()
                .baseUrl("https://apis.data.go.kr/1360000/VilageFcstInfoService_2.0")
                .build();
        // // 외부 API 요청 수행 및 결과 수신 (Blocking 방식 활용)
        ForecastResult result = webClient.get()
                .uri(uriBuilder -> uriBuilder
                        .path("/getVilageFcst")
                        .queryParam("serviceKey", serviceKey) //키 입력
                        .queryParam("pageNo", "1")
                        .queryParam("numOfRows", "1000")
                        .queryParam("dataType", "JSON")
                        .queryParam("base_date", LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMdd")))//날짜 정보 입력
                        .queryParam("base_time", "0500")
                        .queryParam("nx", "55")
                        .queryParam("ny", "127")
                        .build())
                .retrieve()
                .bodyToMono(ForecastResult.class)
                .block();
        // // 수신 결과에 대한 널(Null) 체크 및 데이터 리스트 반환
        return result != null && result.getForecastList() != null ? result.getForecastList() : new ArrayList<>();
    }
}
```
## 최상위 응답 구조 (ForecastResult)
- **JSON 계층 구조 매핑**
    - `response > body > items > item`으로 이어지는 API 특유의 중첩 구조 반영.
- **내부 정적 클래스(Inner Static Class) 활용**
    - 가독성 향상 및 관련 객체 그룹화를 위한 중첩 클래스 설계.
- **Lombok 어노테이션 적용**
    - `@Data` 사용을 통한 보일러플레이트 코드(Getter/Setter, toString 등) 최소화.
```java
import lombok.Data;
import java.util.List;

@Data
public class ForecastResult {
    private Response response;

    @Data
    public static class Response {
        private Header header;
        private Body body;
    }

    @Data
    public static class Header {
        private String resultCode;
        private String resultMsg;
    }

    @Data
    public static class Body {
        private String dataType;
        private Items items;
        private int pageNo;
        private int numOfRows;
        private int totalCount;
    }

    @Data
    public static class Items {
        private List<ForecastDto> item;
    }

    // // Service 레이어에서의 데이터 접근 편의를 위한 유틸리티 메서드
    public List<ForecastDto> getForecastList() {
        if (response != null && response.getBody() != null && response.getBody().getItems() != null) {
            return response.getBody().getItems().getItem();
        }
        return null;
    }
}
```
## 기상 데이터 항목 (ForecastDto)
- **핵심 데이터 필드 정의**
    - 예보 지점(nx, ny), 예보 시각(fcstDate, fcstTime), 항목 코드(category) 및 값(fcstValue) 구성.
- **유연한 데이터 수신**
    - 숫자와 문자 혼용 및 수치 연산 전 단계임을 고려하여 모든 필드 `String` 기반 선언 권장.
```java
import lombok.Data;

@Data
public class ForecastDto {
    private String baseDate;  // // 발표일자
    private String baseTime;  // // 발표시각
    private String category;  // // 자료구분 코드 (예: TMP, SKY, PTY 등)
    private String fcstDate;  // // 예보일자
    private String fcstTime;  // // 예보시각
    private String fcstValue; // // 예보 값
    private int nx;           // // 예보지점 X 좌표
    private int ny;           // // 예보지점 Y 좌표
}

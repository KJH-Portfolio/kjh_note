# JSP/Servlet Scope (4대 영역)

> [!summary] 요약
> 
> **Scope(영역)**란 웹 애플리케이션에서 데이터(변수)가 생존하고 공유되는 유효 범위임.
> 
> Page < Request < Session < Application의 계층 구조를 가지며, 메모리 관리 및 데이터 공유의 핵심 기준.

## 1. Scope 4대 영역 비교
데이터의 생존 주기와 공유 범위를 기준으로 구분함. (좁은 범위 → 넓은 범위 순)

| **영역 (Scope)**  | **내장 객체**     | **유효 범위 (Range)**          | **생명 주기 (Lifecycle)** | **특징 및 핵심 용도**                                                |
| --------------- | ------------- | -------------------------- | --------------------- | ------------------------------------------------------------- |
| **Page**        | `pageContext` | **현재 페이지** <br>(JSP 1개)    | 페이지 실행~종료             | • **지역 변수** 유사, 페이지 이동 시 즉시 소멸<br>• `JSTL` 변수 선언 시 주로 사용      |
| **Request**     | `request`     | **1회 요청** <br>(Forward 포함) | 요청~응답                 | • **MVC 핵심** (Controller→View 데이터 전달)<br>• 가장 빈번히 사용, 응답 후 파기 |
| **Session**     | `session`     | **브라우저** <br>(사용자별)        | 로그인~로그아웃              | • **상태 유지** (로그인, 장바구니)<br>• 클라이언트별 독립 메모리 공간 할당              |
| **Application** | `application` | **서버 전체** <br>(모든 사용자)     | 서버 시작~종료              | • **전역 공유** (방문자 수, 서버 설정)<br>• 모든 사용자가 동일 데이터 공유             |
## 2. EL 표기법 우선순위 (Scope Priority)
JSP에서 `${key}` 형태로 데이터 접근 시, **가장 좁은 영역부터 넓은 영역 순서**로 값을 탐색함.
> [!INFO] 탐색 순서 및 규칙
> 
> 순서: Page → Request → Session → Application
> 
> - 키(Key) 이름이 중복될 경우, **앞쪽(좁은 영역)의 값이 우선 출력**됨.
>     
> - 특정 영역을 지정하려면 `${sessionScope.key}`와 같이 명시 필요.
>     

## 3. 데이터 제어 메서드
모든 Scope 객체는 `Object` 타입으로 데이터를 관리하며 공통 메서드 사용.
- **저장**: `scope.setAttribute("키", 값);` (값은 모든 객체 가능)
- **조회**: `scope.getAttribute("키");` (Java에선 형변환 필수, EL은 불필요)
- **삭제**: `scope.removeAttribute("키");`
## 4. 예시 및 활용
### 💡 개념 비유 (Analogy)
> [!EXAMPLE] Scope 직관적 이해
> 
> - **Page**: **포스트잇**. 나만 보고 페이지 넘기면 버려짐.
>     
> - **Request**: **결재 서류철**. 작성 후 옆 사람에게 전달(Forward) 가능. 결재 끝나면 파기.
>     
> - **Session**: **개인 사물함**. 내 물건(로그인 정보) 보관. 집에 다녀와도 유지, 타인 열람 불가.
>     
> - **Application**: **로비 게시판**. 사장님부터 신입까지 누구나 보는 공지사항.
>     
### 💻 코드 활용 예시 (Java vs JSP)
```java
// [Java: Controller] 동일한 키("data")로 저장 가정
pageContext.setAttribute("data", "Page값");       // JSP 내에서만 가능
request.setAttribute("data", "Request값");
session.setAttribute("data", "Session값");
application.setAttribute("data", "App값");

1. 자동 탐색: ${data} 
2. 영역 지정: ${sessionScope.data} 
```

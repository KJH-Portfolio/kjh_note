# Servlet 이론

> [!summary] 핵심 요약
> * **서블릿(Servlet):** 웹 서비스를 위한 자바 클래스, Controller 역할 수행
> * **역할:** 사용자 요청 처리 및 동적 페이지(HTML) 생성
> * **구조:** 자바 코드 내에 HTML을 작성하는 구조 (Pure Servlet)

## 1. 서블릿(Servlet) 개요
* **정의**
  * 웹 서비스를 구현하기 위한 "**자바 클래스**"
  * 자바를 사용하여 웹을 만들기 위해 필요한 기술
* **주요 역할**
  * 사용자의 요청(Request)을 받아 처리함
  * 처리 결과에 해당하는 응답(Response) 페이지를 만들어 사용자에게 재전송
  * MVC 패턴 중 **Controller**의 기능을 수행
* **특징**
  * WAS(Web Application Server) 서버에서 구동되는 서버 측 프로그램
  * 웹에서 **동적인 페이지**를 JAVA로 구현 가능하도록 지원
  * JAVA 클래스 내부에 웹페이지 구현을 위한 HTML이 포함된 구조 (Java 코드로 HTML 구현)

## 2. 서블릿 vs JSP 비교
* **순수 서블릿 (Pure Servlet)**
  * Java 코드 내에 HTML을 작성하는 형태
* **JSP (Java Server Page)**
  * HTML 내에 Java 코드를 작성하는 형태
  * JSP 페이지는 실행 시 Servlet 코드로 변환(컴파일)되어 구동됨

> [!EXAMPLE] 서블릿의 개념적 이해
> **서블릿은 '요리사(Java)'가 '접시(HTML)'를 들고 있는 것과 같습니다.**
> 요리사가 주도적으로 요리(로직 처리)를 하고, 마지막에 접시 위에 결과물을 담아 손님(클라이언트)에게 서빙합니다. 반대로 JSP는 접시 위에 요리사가 올라가 있는 형태에 비유할 수 있습니다.

``` 구식 순수 서블릿 방법
    // 응답 설정
    response.setContentType("text/html; charset=UTF-8");
    PrintWriter pw = response.getWriter();
    
    // HTML 응답 출력(구식)
    pw.println("<html><body>");
    pw.println("<h2>" + name + "님 환영합니다.</h2>");
    pw.println("</body></html>");
```
## 3.주석
| **종류**      | **문법**      | **브라우저 전송** | **소스 보기** | **자바 코드 실행** | **용도**             |
| ----------- | ----------- | ----------- | --------- | ------------ | ------------------ |
| **HTML 주석** | \`\`        | **O (전송됨)** | **보임**    | **실행됨**      | HTML 설명            |
| **JSP 주석**  | `<%-- --%>` | X (삭제됨)     | 안 보임      | 실행 안 됨       | **JSP 코드 설명 (추천)** |
| **자바 주석**   | `//`        | X (컴파일 제외)  | 안 보임      | 실행 안 됨       | 자바 로직 설명           |

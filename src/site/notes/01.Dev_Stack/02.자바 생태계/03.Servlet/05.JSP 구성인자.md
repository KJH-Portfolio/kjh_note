# JSP 구성 인자 (Elements)

> [!summary] 요약
> JSP 파일은 **HTML**과 **JSP 전용 태그**가 혼합된 형태입니다.
> 페이지 설정을 위한 **지시어(Directive)**,
> 로직 처리를 위한 **스크립트 요소(Scripting)**,
> 그리고 미리 정의된 **내장 객체(Implicit Object)**로 구성됩니다.

## 1. 지시어 (Directives)
JSP 페이지의 속성을 설정하거나 외부 라이브러리/파일을 불러올 때 사용 (`<%@ ... %>`).

| 지시어         | 문법                                   | 설명                                                                          |
| :---------- | :----------------------------------- | :-------------------------------------------------------------------------- |
| **page**    | `<%@ page ... %>`                    | 페이지의 `language`, `contentType`, `pageEncoding` 등 기본 정보 설정 및 자바 클래스 `import` |
| **include** | `<%@ include file="파일명" %>`          | **정적 인클루드**. 소스 코드를 복사해서 합친 후 컴파일. (공통 변수 사용 가능)                            |
| **taglib**  | `<%@ taglib prefix="c" uri="..." %>` | JSTL이나 커스텀 태그 라이브러리 선언                                                      |
## 2. 스크립트 요소 (Scripting Elements)
JSP 내에서 자바 코드를 작성하는 영역 (최근에는 **EL/JSTL**로 대체 권장).

| 요소        | 문법           | 설명                                              |
| :-------- | :----------- | :---------------------------------------------- |
| **스크립트릿** | `<% ... %>`  | 자바 실행 코드 (if, for, 변수 선언) 작성.                   |
| **표현식**   | `<%= ... %>` | 데이터 출력 (`out.print()`와 동일). 출력 시 세미콜론(`;`) 불필요. |
| **선언부**   | `<%! ... %>` | **멤버 변수나 메서드** 선언. JSP 서블릿의 멤버로 생성.             |
| EL        | `${...}`     | java코드를 작성할때 추천되는 방식(범용)                        |

## 3. 내장 객체 (Implicit Objects)
별도 선언 없이 사용할 수 있으며, JSP 컨테이너가 자동으로 제공하는 객체들.

| 내장 객체         | 설명                              |
| :------------ | :------------------------------ |
| `pageContext` | 현재 페이지에 대한 모든 컨텍스트(정보) 관리       |
| `request`     | 클라이언트의 요청 정보 (파라미터, 헤더 등) 저장    |
| `session`     | 클라이언트의 세션 정보 (로그인 상태 등) 관리      |
| `application` | 웹 애플리케이션 전체에 대한 정보 및 공유 데이터     |
| `response`    | 서버의 클라이언트로의 응답 정보 (헤더, 쿠키 등) 제어 |
| `out`         | 출력 스트림 (응답 내용을 클라이언트로 보냄)       |

### 💡 개념 예시
> [!EXAMPLE] 구성 요소 비유
>
>   * **지시어**: **설계도 정보**. 건물의 구조, 재질, 사용될 외부 자재(import)를 정의.
>   * **스크립트릿**: **현장 시공**. 실제로 벽돌을 쌓는 행위 (가급적 줄이고 조립식 자재(JSTL) 사용 권장).
>   * **내장 객체**: **공용 공구함**. 망치(request), 드릴(response) 등 현장에 비치되어 있어 바로 쓸 수 있는 도구들.
### 💻 코드 예시(pageContext)

```ejs
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>
<%
    // 1. pageContext를 사용하여 다른 내장 객체 얻기
    javax.servlet.http.HttpServletRequest req = 
        (javax.servlet.http.HttpServletRequest)pageContext.getRequest();
    String contextPath = req.getContextPath();
    
    // 2. pageContext를 사용하여 Scope에 속성(Attribute) 저장
    // "page" Scope에 데이터를 저장 (기본값)
    pageContext.setAttribute("pageVar", "페이지 영역 변수"); 
    // "request" Scope에 데이터를 저장
    pageContext.setAttribute("requestVar", "요청 영역 변수", PageContext.REQUEST_SCOPE);
    // 3. pageContext를 사용하여 Scope에서 속성 가져오기
    String valueFromPage = (String)pageContext.getAttribute("pageVar");
    
    // 4. findAttribute(): 네 가지 Scope를 순서대로 검색하여 속성 가져오기 (EL의 동작과 유사)
    String valueFromScope = (String)pageContext.findAttribute("requestVar");
%>
<html>
<body>
    <h3>Context Path: <%= contextPath %></h3>
    
    <ul>
        <li>Page Scope 값: <%= valueFromPage %></li>
        <li>findAttribute로 찾은 값 (Request Scope): <%= valueFromScope %></li>
        <li>EL로 출력: ${pageVar} (pageContext가 저장한 page scope 변수)</li>
    </ul>
</body>
</html>
```

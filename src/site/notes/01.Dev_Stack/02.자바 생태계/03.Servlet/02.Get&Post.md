# Servlet & Request Processing (GET/POST 통합)

> [!summary] 핵심 요약
> * **서블릿(Servlet):** 클라이언트의 요청을 받아 비즈니스 로직을 처리하는 자바 클래스 (Controller)
> * **MVC 패턴:** 서블릿은 로직 처리 및 데이터 가공을 담당하고, 화면 출력은 JSP에게 위임(Forward)함
> * **핵심 차이:**
>   * **GET:** URL Header 전송, 보안 취약, 조회(Search)용, 인코딩 불필요
>   * **POST:** Body 전송, 보안 우수, 전송(Submit)용, **인코딩 필수**

## 1. 핵심 사용법 (MVC 패턴 공통 로직)
1. Servlet 요청경로 설정

```java
// [Servlet 공통 패턴 예시 (GET/POST 동일)]
// @WebServlet 요청경로 설정;
	@WebServlet("/요청경로")
// 0. 인코딩 설정 (POST 방식일 경우에만 필수 추가)
	request.setCharacterEncoding("UTF-8"); 

// 1. 데이터 추출
	String name = request.getParameter("userName");

// 2. JSP로 전달할 데이터 담기 (request scope에 저장)
// request.setAttribute("키", 값);
	request.setAttribute("name", name);

// 3. JSP로 화면 이동 (Forward - 위임)
// 화면을 그려줄 JSP 파일 선택
	RequestDispatcher view = request.getRequestDispatcher("views/responsePage.jsp");
// 요청 객체(request)와 응답 객체(response)를 JSP에게 전달
	view.forward(request, response); 

//4. JSP에서 데이터 수신
	String name = (String)request.getAttribute("name");
```

## 2. GET vs POST 방식 비교

### 2.1. GET 방식 (조회/검색)

- **특징:** URL의 **Header** 영역에 데이터를 포함하여 요청함 (`?key=value`)
- **장점:** URL에 데이터가 노출되어 **즐겨찾기** 및 공유가 가능함 (검색 결과 등)
- **단점:** 데이터가 노출되어 보안에 취약하며, 전송 길이에 제한이 있어 대용량 데이터 부적합
- **인코딩:** 별도의 설정 불필요 (Server 설정에 따름, Tomcat 8+ 기본 UTF-8)
    

### 2.2. POST 방식 (전송/등록)

- **특징:** URL의 **Body** 영역에 데이터를 숨겨서 요청함
- **장점:** 데이터가 노출되지 않아 **보안성**이 높으며(로그인 등), 전송 용량에 제한이 없음
- **단점:** 즐겨찾기가 불가능하며, 페이지 요청 시 대기 시간이 발생할 수 있음
- **인코딩:** **필수 설정.** Body 데이터는 기본 ISO-8859-1로 처리되므로 추출 전 반드시 변환해야 함
    ```java
    request.setCharacterEncoding("UTF-8"); // getParameter 호출 전 실행
    ```

## 3. 상세 구현 가이드
### 3.1. 데이터 추출 (Servlet)
`HttpServletRequest` 객체를 사용하여 클라이언트가 보낸 값을 꺼냅니다.

| **메서드**                      | **설명**                               | **반환 타입**  |
| ---------------------------- | ------------------------------------ | ---------- |
| `getParameter("name")`       | 단일 값을 추출할 때 사용 (input type="text" 등) | `String`   |
| `getParameterValues("name")` | 여러 값을 추출할 때 사용 (checkbox 등)          | `String[]` |
### 3.2. 데이터 전달 및 화면 이동 (Servlet to JSP)

서블릿에서 처리한 결과를 JSP로 보내기 위해 **RequestDispatcher**를 사용합니다.
1. **데이터 담기:** `request.setAttribute("key", value);` (Object 타입으로 저장됨)
2. **뷰 선택:** `RequestDispatcher view = request.getRequestDispatcher("경로");`
3. **위임(Forward):** `view.forward(request, response);`
    

### 3.3. 데이터 수신 (JSP)

JSP에서는 서블릿이 보낸 데이터를 꺼내서 화면에 출력합니다.
- **꺼내기:** `request.getAttribute("key");`
- **형변환:** 반환형이 `Object`이므로 원래 타입으로 **Casting(형변환)** 필수
    - 예: `String name = (String)request.getAttribute("name");`

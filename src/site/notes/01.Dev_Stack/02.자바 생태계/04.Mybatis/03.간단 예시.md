# 📘  MyBatis 개발 실전 가이드
> [!summary] 개발 프로세스 요약
> 기능 하나를 추가할 때, 개발자는 아래 **5단계**를 순서대로 작성하거나 수정해야 합니다.
>
> 1.  **View (JSP)**: 요청 보낼 주소(`action`) 지정
> 2.  **Controller**: 주소 매핑(`@WebServlet`) 및 로직 제어
> 3.  **Service**: 트랜잭션(Commit/Rollback) 관리
> 4.  **DAO**: DB 연결 도구 사용
> 5.  **Mapper (XML)**: SQL 쿼리 작성 (및 설정 파일 등록)

-----
# 조회(Select)예시
## 1\. 🎮 Controller (서블릿)

  * **핵심 역할**: 주소 지정, 값 받기, 일 시키기, 결과 페이지로 보내기

```java
// 주소 매핑
@WebServlet("/login.me") 
public class LoginController extends HttpServlet {
    
    //Post방식
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        
        // 인코딩 설정(Post일때만)
        request.setCharacterEncoding("UTF-8");
        
        //파라미터 추출
        String userId = request.getParameter("userId");
        String userPwd = request.getParameter("userPwd");
        
        //VO 객체로 포장
        Member m = new Member();
        m.setUserId(userId);
        m.setUserPwd(userPwd);
        
        //Service 호출
        MemberService service = new MemberService();
        Member loginMember = service.loginMember(m);
        
        //결과 처리 및 페이지 이동
        if(loginMember != null) {
            request.getSession().setAttribute("loginMember", loginMember);
            response.sendRedirect(request.getContextPath()); // 재요청(Redirect)
        } else {
            request.setAttribute("errorMsg", "로그인 실패");
            request.getRequestDispatcher("views/common/errorPage.jsp").forward(request, response); // 위임(Forward)
        }
    }
}
```
- Mybatis에서는 인자를 하나만 받기에 포장이 필수

## 2\. 🛠️ Service (서비스)

  * **핵심 역할**: `SqlSession` 생성, 트랜잭션(Commit/Rollback), 자원 반납

```java
public class MemberService {
    
    private MemberDao dao = new MemberDao();
    
    //입력 메서드 시작
    public int insertMember(Member m) {
    
        //Connection(SqlSession) 생성
        SqlSession sqlSession = Template.getSqlSession();
        
        // DAO 호출
        int result = dao.insertMember(sqlSession, m);
        
        //트랜잭션 처리 (DML인 경우)
        if(result > 0) {
            sqlSession.commit();
        } else {
            sqlSession.rollback();
        }
        
        //자원 반납
        sqlSession.close();
        
        return result;
    }
}
```

## 3\. 📦 DAO (데이터 접근 객체)

복잡한 로직 없이 **SqlSession에게 명령만 전달**하는 단순한 역할을 합니다.

  * **핵심 역할**: MyBatis 메서드(`selectOne`, `insert` 등) 호출

```java
public class MemberDao {
    
    public int insertMember(SqlSession sqlSession, Member m) {
        // 네임스페이스.아이디
        return sqlSession.insert("memberMapper.insertMember", m);
    }
}
```

## 4\. 🗺️ Mapper (SQL XML)

자바 코드가 아닌 **SQL 쿼리**를 작성하는 곳입니다. 오타가 가장 많이 나는 곳이니 주의해야 합니다.

  * **수정 파일**: `~-mapper.xml`
  * **핵심 역할**: SQL 작성, VO와 DB 컬럼 매핑(ResultMap)

```xml
<mapper namespace="memberMapper">

	<!--DB:java 속성값 매칭-->
	<!--type는 mybatis-config에서 설정해야함-->
    <resultMap type="Member" id="memberResultSet">
        <result column="USER_ID" property="userId"/>
        <result column="USER_NAME" property="userName"/>
    </resultMap>
	<!--쿼리 작성-->
    <insert id="insertMember" parameterType="Member"> 
        INSERT INTO MEMBER (USER_ID, USER_PWD, USER_NAME)
        VALUES (#{userId}, #{userPwd}, #{userName})
    </insert>
    
    <!--select는 resultMap이 필요하다.-->
    <select id="selectMember" resultMap="memberResultSet">

</mapper>
```

- mapper에 select,insert,update,delete 전부 사용 가능
- `#{}` 데이터를 값으로 취급(''안에 값을 넣어줌) , `${}`는 문자열 그 자체로 취급함

## 5\. ⚙️ 놓치기 쉬운 설정 파일 등록
- mybatis-config.xml

```xml
<configuration>
    <mappers>
        <mapper resource="resources/mappers/member-mapper.xml"/>
        <mapper resource="resources/mappers/board-mapper.xml"/>
    </mappers>
</configuration>
```

# 📘 MyBatis 핵심 설정 및 구조
> [!summary] 요약
> MyBatis 구동을 위한 3대 핵심 요소의 상세 역할입니다.
>
> 1.  **설정(`config`)**: DB 연결 방식, NULL 처리, 별칭 등 **전역 규칙** 정의
> 2.  **템플릿(`Template`)**: 설정 파일을 로딩하여 DB 작업 도구(`SqlSession`)를 찍어내는 공장
> 3.  **매퍼(`Mapper`)**: 자바 객체와 SQL 간의 데이터 **매핑(Mapping)** 정의

## 1\. ⚙️ 환경 설정 (`mybatis-config.xml`)

MyBatis가 동작하는 방식(메커니즘)을 결정하는 파일입니다.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE configuration PUBLIC "-//mybatis.org//DTD Config 3.0//EN" "https://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
	<!--java에서 데이터가 null이라면 DB가 이를 null로 인식하도록 함-->
	<settings>
		<setting name="jdbcTypeForNull" value="NULL"/>
	</settings>
	
	<!--자주 사용하는 타입의 별칭 부여-->
	<typeAliases>
		<typeAlias type="com.kh.mybatis.member.model.vo.Member" alias="Member"/>
	</typeAliases>
	
	<!--개발 환경 설정-->
	<environments default="development">
		<environment id="development">
			
			<transactionManager type="JDBC"/>
			<!--Connection Pool방식으로 사용할 연결 객체 생성주소-->
			<dataSource type="POOLED">
				<property name="driver" value="oracle.jdbc.driver.OracleDriver"/>
				<property name="url" value="jdbc:oracle:thin:@localhost:1521:xe"/>
				<property name="username" value="MYBATIS"/>
				<property name="password" value="MYBATIS"/>
			</dataSource>
			
		</environment>
	</environments>
	
	<!--mapper위치 등록-->
	<mappers>
		<mapper resource="/resources/mappers/member-mapper.xml"/>
	</mappers>
	
	<!--setting으로 DB컬럼명을 java객체 필드명으로 자동 변환 (USER_ID -> userId)-->
	<settings>
	    <setting name="logImpl" value="STDOUT_LOGGING"/>
	    <setting name="mapUnderscoreToCamelCase" value="true"/>
	</settings>

</configuration>
```

## 2\. 🏭 연결 객체 생성기 (`Template.java`)

설정 파일을 읽어들여, 실제 DB 작업을 수행할 **일꾼(`SqlSession`)을 공급**하는 클래스입니다.

```java
public class Template {
	
	public static SqlSession getSqlSession() {
		
		SqlSession sqlSession = null;
		String resource = "/resources/mybatis-config.xml";
		
		//mybatis-config.xml을 읽고 SqlSessionFactory 객체 생성
		try (InputStream inputStream = Resources.getResourceAsStream(resource)) {
			SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
			
			//SqlSession 객체 생성 
			//Auto commit를 false로 설정하여 개발자가 직접 commi/rollback 처리
			sqlSession = sqlSessionFactory.openSession(false);
			
		} catch (IOException e) {
			e.printStackTrace();
		}
		
		return sqlSession;
	}
}
```

## 3\. 🗺️ 매퍼 구조 (`member-mapper.xml`)

자바 객체와 SQL 간의 데이터를 매핑하는 파일입니다.

```xml
<!--매퍼 파일의 고유 명칭 부여-->
<mapper namespace="memberMapper">
	
	<!--DB조회 결과를 자바 객체에 어떻게 담을지 정의하는 설게도-->
	<resultMap type="Member" id="memberResultMap">
		<result column ="USER_NO" property="userNo"/>
		<result column ="USER_ID" property="userId"/>
		<result column ="USER_NAME" property="userName"/>
		<result column ="EMAIL" property="email"/>
		<result column ="BIRTHDAY" property="birthday"/>
		<result column ="GENDER" property="gender"/>
		<result column ="PHONE" property="phone"/>
		<result column ="ADDRESS" property="address"/>
		<result column ="ENROLL_DATE" property="enrollDate"/>
		<result column ="MODIFY_DATE" property="modifyDate"/>
		<result column ="STATUS" property="status"/>
	</resultMap>
	
	<!--실제 sql구문 메서드 (고유명칭(id),입력타입(parameterType),출력타입(resultMap)-->
	<select id="loginMember" parameterType="Member" resultMap="memberResultMap">
		SELECT *
		FROM MEMBER
		WHERE USER_ID = #{userId}
		AND USER_PWD = #{userPwd}
		AND STATUS = 'Y'
	</select>
	
	//Mapper 최적화 빌드 (동적으로 만들어 sql 구문의 갯수를 줄이는 것이 유지보수에 좋다
	<update id="updateMember" parameterType="Member">
    UPDATE MEMBER
    <set>
        <if test="userPwd != null">USER_PWD = #{userPwd},</if>
        <if test="userName != null">USER_NAME = #{userName},</if>
        <if test="email != null">EMAIL = #{email},</if>
        <if test="phone != null">PHONE = #{phone},</if>
        <if test="address != null">ADDRESS = #{address},</if>
        <if test="status != null">STATUS = #{status},</if>
        </set>
    WHERE USER_ID = #{userId}
</update>
	
	<!--DML(insert,update,delete) 구문에는 입력타입만 필요함-->
	
</mapper>
```

> [!Note]
> `<include refid="memberColumns"/>`로 mapper파일의 중복을 줄일 수도 있다
> **상속(extends)** 으로 오버라이드 하여 매핑
> `<association>`: 메인 객체 안에 다른 객체(1:1)를 포함하여 매핑
> `<collection>`: 메인 객체 안에 다른 객체의 목록(1:N)을 포함하여 매핑

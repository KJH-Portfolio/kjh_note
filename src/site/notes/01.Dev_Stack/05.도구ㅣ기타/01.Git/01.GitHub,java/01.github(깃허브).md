# Git 프로젝트 관리 및 워크플로우 통합 가이드
> [!summary] 요약: GitHub 토큰 인증 및 환경 설정, 데일리 루틴(Pull-Work-Push), 팀 단위 브랜치 협업 전략, 주요 개발 도구(Eclipse, VS Code, SQL Developer)별 상세 활용 프로세스 구축.
## 🔑 인증 및 초기 환경 설정
- **개인용 액세스 토큰 (Token) 관리**
    - 생성 경로: GitHub **Developer Settings** 내 토큰 생성
    - 용도: HTTPS 기반 리포지토리 접근 시 인증 수단으로 사용.
- **사용자 정보 구성 (Configuration)**
    - 설정 항목: 전역(Global) **user.name** 및 **user.email** 등록.
- **저장소(Repository) 생성 및 로컬 연동**
    - **.gitignore** 설정: 프로젝트 성격(Java, Node 등)에 따른 컴파일 파일(*.class, *.jar) 제외 필수.
    - **License**: 오픈소스 여부에 따라 MIT License 등 선택적 적용.
    - 로컬 연결 방식: **Clone**(복제) 또는 Eclipse **Share Project**(등록)
```xml
// Git 사용자 정보 설정 예시
git config --global user.name "YourName"
git config --global user.email "YourEmail@example.com"
```
## 📅 데일리 워크플로우 및 커밋 규칙
- **작업 루틴 (Daily Routine)**
    - 작업 시작: **Pull** 수행을 통한 원격 저장소 최신화 및 동기화.
    - 작업 진행: IDE(Eclipse, VS Code 등)를 통한 코드 개발.
    - 작업 종료: **Commit & Push** 실행으로 작업물 원격 저장소 전송.
- **커밋 및 충돌 관리**
    - **Commit** vs **Push**: 로컬 저장소 기록과 원격 저장소 업로드 단계 분리 이해.
    - **충돌(Conflict) 예방**: Push 전 반드시 Pull을 수행하여 최신 상태 확인 및 병합 후 업로드.
    - **커밋 메시지 컨벤션**: [Feat](https://www.google.com/search?q=%EA%B8%B0%EB%8A%A5), [Fix](https://www.google.com/search?q=%EB%B2%84%EA%B7%B8), [Docs](https://www.google.com/search?q=%EB%AC%B8%EC%84%9C) 등 접두어 활용 권장.
- **기타 유의 사항**
    - 패키지 구조: 빈 패키지 폴더는 Git 관리 대상에서 제외됨.
```xml
// 커밋 메시지 컨벤션 예시
[Feat] 사용자 로그인 로직 구현
[Fix] DB 연결 시간 초과 오류 수정
```
## 👥 팀 프로젝트 및 브랜치 운용 (Star Strategy)
- **협업 환경 설정**
    - 권한 부여: **Collaborators** 메뉴를 통한 팀원 추가.
    - 규칙 적용: **Branch Protection Rules** 활용, Merge 전 Pull Request 필수 설정 권장.
- **브랜치(Branch) 워크플로우**
    - 브랜치 생성: **New Branch** 생성 및 **Push branch**를 통한 원격 등록.
    - 작업 전환: 작업 전 해당 브랜치 선택(Double Click/Checkout) 필수.
    - 작업 확정: 작업 브랜치 Push 후 Main 브랜치로 **Compare & Pull Request** 요청.
- **동기화 및 병합 (Merge)**
    - 로컬 Main 갱신: 로컬 Main 브랜치 Pull 수행.
    - 브랜치 병합: 내 작업 브랜치에서 Main 브랜치 우클릭 후 **Merge** 수행하여 최신 코드 반영 및 충돌 테스트.
```xml
// 브랜치 생성 및 전환 예시
git branch feature-name
git checkout feature-name
```
## 🛠️ 개발 도구별(IDE) Git 활용 상세
- **Java (Eclipse)**
    - 관리 단위: 프로젝트 단위 관리 중심.
    - 업로드: **Git Staging** 탭 진입 → `++` 아이콘(Add all) → 메시지 입력 → **Commit and Push**.
    - 다운로드: **Import** > **Clone URI** (초기) 또는 **Team > Pull** (진행 중).
- **Web (VS Code)**
    - 관리 인터페이스: **소스 제어(Source Control)** 아이콘 활용.
    - 워크플로우: 메시지 입력 → **커밋(Commit)** → **변경 사항 동기화(Sync Changes)**.
- **DB (Oracle SQL Developer)**
    - 간접 관리: SQL 파일 저장 후 Git 연동 폴더(예: `sql/`)에 배치.
    - 업로드 대행: VS Code 등을 활용하여 해당 텍스트 파일의 변경 사항 감지 후 Commit & Push 수행.
```xml
// .gitignore 파일 구성 예시 (Java 기준)
*.class
bin/
.settings/
.project
```

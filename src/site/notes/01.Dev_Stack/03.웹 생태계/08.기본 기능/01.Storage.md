# 브라우저 데이터 저장 기술 비교 (Web Storage, Cookie, IndexedDB)
> [!summary] 요약: 웹 애플리케이션의 상태 유지 및 데이터 관리를 위해 브라우저가 제공하는 다양한 저장소 기술들의 특징, 용도 및 성능 차이점 요약.
---
## 1. 저장소별 핵심 비교

|**구분**|**로컬 스토리지 (Local Storage)**|**세션 스토리지 (Session Storage)**|**쿠키 (Cookie)**|**IndexedDB**|
|---|---|---|---|---|
|**저장 용량**|약 5~10MB|약 5MB|약 4KB 이하|무제한에 가까움 (디스크 잔량 의존)|
|**유효 기간**|영구적 (수동 삭제 필요)|탭/윈도우 닫기 전까지|설정된 만료일 전까지|영구적 (수동 삭제 필요)|
|**서버 전송**|전송되지 않음|전송되지 않음|매 요청 시 자동 전송|전송되지 않음|
|**데이터 타입**|문자열(String)만 가능|문자열(String)만 가능|문자열(String)만 가능|객체, 파일, 블롭 등 대용량 구조화 데이터|
|**접근 방식**|동기적(Synchronous)|동기적(Synchronous)|동기적(Synchronous)|비동기적(Asynchronous)|

---
## 2. 세부 특징 및 활용 사례
### 2.1 웹 스토리지 (Web Storage: Local & Session)
- **공통 성격**
    - 키-값(Key-Value) 쌍의 단순 저장소 구조.
    - 도메인별 독립적인 저장 공간 제공.
- **로컬 스토리지 (Local Storage)**
    - 브라우저를 닫아도 데이터가 유지됨.
    - **활용**: 사용자 설정 저장(다크모드), 자동 로그인 정보(비보안 데이터).
- **세션 스토리지 (Session Storage)**
    - 데이터가 현재 탭 내에서만 유효함.
    - **활용**: 일회성 로그인 정보 확인, 입력 폼의 임시 보관 데이터.
```javascript
// 로컬 스토리지 사용 예시
localStorage.setItem('theme', 'dark'); // 데이터 저장
const currentTheme = localStorage.getItem('theme'); // 데이터 조회
localStorage.removeItem('theme'); // 특정 데이터 삭제

// 세션 스토리지 사용 예시
sessionStorage.setItem('temp_id', 'user_123'); // 세션 내 데이터 보관
```
### 2.2 쿠키 (Cookie)
- **특징**
    - HTTP 헤더를 통해 서버와 클라이언트 간 통신에 활용됨.
    - `HttpOnly` 설정을 통해 XSS(교차 사이트 스크립팅) 공격 방어 가능.
- **제한 사항**
    - 용량이 매우 작아 대량의 데이터 저장에 부적합함.
    - 모든 요청에 포함되므로 불필요한 네트워크 부하 초래 가능성 존재.
- **활용**
    - 세션 관리(로그인 상태 유지), 사용자 트래킹, 팝업 '오늘 하루 보지 않기' 설정.
```javascript
// 쿠키 설정 예시 (만료일 포함)
document.cookie = "user_auth=token123; expires=Wed, 31 Dec 2026 23:59:59 GMT; path=/; secure";

// 쿠키 읽기 (문자열 파싱 필요)
console.log(document.cookie);
```
### 2.3 IndexedDB
- **특징**
    - 브라우저 내장 NoSQL 객체 지향 데이터베이스.
    - 인덱스를 이용한 고성능 데이터 검색 지원.
    - 대량의 구조화된 데이터를 저장할 수 있는 유일한 표준 방식.
- **활용**
    - 오프라인 웹 애플리케이션(PWA), 복잡한 상태 관리의 영속성 유지, 대량의 JSON 데이터 캐싱.
```javascript
// IndexedDB 데이터베이스 열기 (비동기 처리)
const request = indexedDB.open("MyDatabase", 1);

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  // // 객체 저장소(Table 역할) 생성
  const objectStore = db.createObjectStore("users", { keyPath: "id" });
  objectStore.createIndex("name", "name", { unique: false });
};

request.onsuccess = (event) => {
  const db = event.target.result;
  // // 데이터 트랜잭션 처리 로직 구현
};
```

## 마무리
- **노트 수정 사항**: 로컬/세션 스토리지, 쿠키, IndexedDB의 핵심 차이점을 표 형식으로 비교하고 각 기술별 실전 코드 예시와 활용 사례를 구조화함.
- **추가 제안**: 보안이 중요한 데이터 처리를 위한 `HttpOnly` 쿠키와 브라우저 저장소의 보안 취약점(XSS) 대응 방안에 대한 심화 노트를 추가하는 것을 추천함.
- **관련 질문**: 로컬 스토리지의 용량 한계를 극복하기 위한 IndexedDB 라이브러리(예: `idb`, `Dexie.js`) 활용 가이드나, PWA 구현을 위한 **Service Worker** 연동 노트를 작성하시겠습니까?

# 📘 jQuery 메서드 및 기능

> [!summary] **핵심 요약**
> 
> - **jQuery**: 존 레식이 개발한 경량 라이브러리로 DOM 조작, 이벤트 처리, Ajax 등을 간편하게 구현합니다.
>     
> - **구조**: 선택자(Selector)로 요소를 잡고, 메서드(Method)로 동작을 제어합니다.
>     
> - **실무 Tip**: 이 노트는 기본 이론뿐만 아니라 현업에서 자주 쓰는 **데이터 처리(.val, .data)**, **상태 제어(.prop)**, **동적 이벤트(.on)** 기법을 포함하고 있습니다.
>     

## 1. jQuery 시작 및 개요 (Foundation)

### 1.1 연결 및 실행

- **연결 방식**: CDN(온라인) 방식과 파일 다운로드(오프라인) 방식.
    
- **시작 구문**: DOM이 로드된 후 실행하기 위해 `ready` 함수를 사용합니다.
    
    ```js
    $(document).ready(function(){ ... });
    // 또는 단축형
    $(function(){ ... });
    ```
    
- **기본 문법**: `$('선택자').메소드('속성', '값');`

## 2. 객체 탐색 (Traversing)

선택된 요소를 기준으로 원하는 요소를 찾아가는 메서드입니다.

- **`[선택자]`**: 괄호 안의 선택자는 **생략 가능**하며, 생략 시 조건 없이 모든 대상을 찾습니다. (단, `.find`는 필수)
    

### 2.1 필터링 (Filtering)

이미 선택된 요소들의 집합에서 조건에 맞는 것만 **남기는** 과정입니다.

| **메서드**          | **내용**                                 | **비고**                      |
| ---------------- | -------------------------------------- | --------------------------- |
| `.first()`       | 선택된 요소 중 **제일 처음** 요소                  |                             |
| `.last()`        | 선택된 요소 중 **제일 마지막** 요소                 |                             |
| `.eq(n)`         | **[숫자]** 인덱스 `n`번째 요소                  | 0부터 시작                      |
| `.filter('선택자')` | 기준 중 `선택자`와 **일치하는** 것만 남김             | `$('li').filter('.active')` |
| `.not('선택자')`    | 기준 중 `선택자`와 **일치하지 않는** 것만 남김          | 제외할 때 사용                    |
| `.is('선택자')`     | `선택자`와 일치하는 것이 있는지 확인 (**true/false**) | 조건문(`if`)에서 사용              |


### 2.2 조상 탐색 (Ancestors)

| **메서드**                | **내용**                                     |
| ---------------------- | ------------------------------------------ |
| `.parent()`            | 바로 위 **부모** (딱 1개)                         |
| `.parents([선택자])`      | 조상들 중 `선택자`와 일치하는 **모든 조상**                |
| `.parentsUntil('선택자')` | 위로 올라가다가 `선택자`를 만나기 **전까지의 조상들**           |
| **`.closest('선택자')`**  | **[실무]** 나 포함 상위에서, 가장 먼저 걸리는 **단 하나의 조상** |

### 2.3 자손 탐색 (Descendants)

| **메서드**            | **내용**                                         |
| ------------------ | ---------------------------------------------- |
| `.children([선택자])` | 바로 아래 **자식**들 중 `선택자`와 일치하는 것                  |
| **`.find('선택자')`** | **[필수]** 뱃속의 모든 **후손(손자 포함)**들 중 `선택자`와 일치하는 것 |
| **`.has('선택자')`**  | **[실무]** 자손 중에 `선택자`를 **가진 부모**만 남김            |

> [!TIP] **.find() vs .has() 구분하기**
> 
> - `.find('.btn')`: 내 하위에 있는 **버튼(.btn)을 가져와라.** (결과: 자식)
>     
> - `.has('.btn')`: 하위에 **버튼(.btn)을 가지고 있는 나(부모)를 남겨라.** (결과: 부모)
>     

### 2.4 형제 탐색 (Sideways)

|**메서드**|**내용**|
|---|---|
|`.siblings([선택자])`|`선택자`와 일치하는 **모든 형제**|
|`.next()`|바로 다음 동생 **1명**|
|`.nextAll([선택자])`|내 뒤의 **모든 동생** (조건 일치 시 그들만)|
|`.nextUntil('선택자')`|`선택자`를 만나기 **전까지의 동생들**|
|`.prev()`|바로 이전 형 **1명**|
|`.prevAll([선택자])`|내 앞의 **모든 형** (조건 일치 시 그들만)|
|`.prevUntil('선택자')`|`선택자`를 만나기 **전까지의 형들**|

## 3. 객체 조작 (Manipulation)

### 3.1 배열 관리 (Loop)
- arr 배열을 $(arr)로 하면 jquery용 배열이 된다.

| **메서드**                  | **내용**                                                                                   |
| ------------------------ | ---------------------------------------------------------------------------------------- |
| `$.each(배열, fn)`         | 일반 배열이나 **객체**를 0부터 순회 <br>⭐배열,객체 둘다 가능<br>배열(인자: `index`, `item`)<br>객체(인자: `속성`,`속성값`) |
| `$('선택자').each(fn)`      | 선택된 요소들을 순차적으로 조작 (인자: `index`, `item`)                                                  |
| `$.extend(대상, 객체1, 객체2)` | 여러 객체의 내용을 하나로 **병합(Merge)**함                                                            |

### 3.2 클래스 조작 (Class)

| **메서드**                             | **내용**                                                  |
| ----------------------------------- | ------------------------------------------------------- |
| `.addClass('클래스명')`                 | 클래스 **추가** (띄어쓰기로 여러 개 가능)                              |
| `.removeClass('클래스명')`              | 클래스 **삭제**                                              |
| `.toggleClass('클래스명',[true/false])` | 클래스가 있으면 삭제, 없으면 추가 (스위치 역할)<br>`true`는 추가만`false`는 삭제만 |

### 3.3 속성, 상태 및 데이터 (Attr, Prop & Data)


|**메서드**|**내용**|**사용처**|
|---|---|---|
|`.attr('속성')`|HTML 태그의 **속성 값** (문자열)|`id`, `src`, `href`, `alt`|
|`.removeAttr('속성')`|해당 속성을 **제거**||
|**`.prop('속성')`**|**[실무]** 요소의 **상태 값** (Boolean)|`checked`, `disabled`, `readonly`, `selected`|
|**`.data('키')`**|**[실무]** 요소에 숨겨진 **데이터 저장/로드**|HTML `data-` 속성이나 메모리 데이터 관리|
|`.css('속성')`|CSS 스타일 값을 가져오거나 설정||

> [!example] .attr() vs .prop() 예시
> 
> 체크박스(<input type="checkbox" checked>)의 경우:
> 
> - `.attr('checked')`: "checked" (HTML에 적힌 초기값 문자열)
>     
> - `.prop('checked')`: **true / false** (현재 체크 여부, 실무 사용 권장)
>     

### 3.4 내용, 값 및 폼 데이터 (Content, Value & Form)

|**메서드**|**Getter (읽기) 특징**|**Setter (쓰기) 특징**|
|---|---|---|
|**`.html()`**|첫 번째 요소의 내용을 **HTML 태그 포함** 리턴|전달된 내용을 **HTML 태그로 인식**하여 대입|
|**`.text()`**|모든 요소의 텍스트를 **합쳐서** 리턴|전달된 내용을 **순수한 문자로 인식**하여 대입|
|**`.val()`**|**Form 요소** (Input, Select 등)의 **값** 읽기|**Form 요소 값** 설정 (Input의 `value` 등)|
|**`.serialize()`**|[실무] **Form**에 포함된 모든 입력 요소의 **데이터를 수집**하여 **URL 쿼리 문자열** 형태로 리턴|**(해당 없음)**|
### 3.5 객체 생성 및 추가 (Creation & Insertion)

|**구분**|**메서드**|**설명**|
|---|---|---|
|**내부 추가**|`.append()` / `.prepend()`|선택자 내부 **맨 뒤 / 맨 앞**에 추가|
||`.appendTo()` / `.prependTo()`|(생성 구문을) 타겟의 뒤/앞으로 이동하여 추가|
|**외부 추가**|`.after()` / `.before()`|선택자 **바로 뒤 / 바로 앞**에 추가|
||`.insertAfter()` / `.insertBefore()`|(생성 구문을) 타겟의 뒤/앞으로 이동하여 추가|
- 모든 추가 메서드는 `function(index, html)`을 인자로 사용할 수 있음.
### 3.6 복제 및 제거 (Cloning & Removal)

| **메서드**                | **내용**                            |
| ---------------------- | --------------------------------- |
| `.clone([true/false])` | 요소를 복사 (`true`: 이벤트 핸들러까지 복사)     |
| `.remove()`            | 요소를 **삭제** (이벤트/데이터도 모두 삭제)       |
| `.detach()`            | 요소를 **잘라내기** (이벤트/데이터 유지, 재사용 가능) |
| `.empty()`             | 요소 껍데기는 남기고 **내용(자식)만 비움**        |

### 3.7 [실무] 위치 및 크기 (Dimensions)

|**메서드**|**설명**|
|---|---|
|`.width()` / `.height()`|요소의 너비/높이 (Padding, Border 제외)|
|`.offset()`|문서 전체 기준의 **절대 좌표** `{top: n, left: n}` 반환|
|`.position()`|부모 요소 기준의 **상대 좌표** 반환|
|`.scrollTop()`|현재 스크롤의 수직 위치 반환 (무한 스크롤 등에 사용)|

## 4. 이벤트 (Event)

### 4.1 이벤트 연결 및 실행

|**메서드**|**설명**|
|---|---|
|`.on('이벤트', fn)`|**표준 통합 메서드**. 정적/동적 요소 모두 처리 가능|
|`.one('이벤트', fn)`|이벤트를 **한 번만** 실행하고 자동 해제|
|`.off('이벤트')`|연결된 이벤트를 **제거**|
|**`.trigger('이벤트')`**|**[실무]** 이벤트를 코드로 **강제 실행**시킴 (예: 자동 클릭)|
|`.hover(fnIn, fnOut)`|`mouseenter`와 `mouseleave`를 동시에 처리|

> [!example] 동적 이벤트 바인딩 (Event Delegation)
> 
> 문서 로딩 후 나중에 생성된 요소(append 등)에 이벤트를 걸 때는 반드시 부모(document)에 위임해야 합니다.
> 
> ```js
> // "document야, 나중에 .dynamic-btn이 생기면 클릭 이벤트 좀 처리해줘"
> $(document).on('click', '.dynamic-btn', function(){ ... });
> ```
### ### 4.2 이벤트 객체 상세 (Event Object)

이벤트 핸들러의 인자로 전달되는 `event` 객체의 상세 속성과 메서드입니다.
#### event객체 속성

| **속성**                  | **내용**                      |
| ----------------------- | --------------------------- |
| `event.target`          | 이벤트가 발생한 요소                 |
| `event.type`            | 발생한 이벤트 종류                  |
| `event.pageX` / `pageY` | 마우스의 좌표 (왼쪽/위쪽 기준)          |
| `event.which`           | 눌러진 키/마우스의 정수 값             |
| `event.data`            | 이벤트 핸들러로부터 받아오는 값           |
| `event.currentTarget`   | 버블링 내 이벤트 핸들러의 `this`와 같은 값 |
| `event.delegateTarget`  | 현재 호출된 핸들러에 연결된 객체          |
| `event.relatedTarget`   | 마우스가 들어오거나 나오는 요소           |
| `event.namespace`       | 사용자 지정 이벤트명                 |
| `event.result`          | 핸들러가 반환한 값                  |
| `event.timestamp`       | 이벤트 발생 시간 (밀리초)             |
#### 메서드

| **메서드**                                 | **내용**                              |
| --------------------------------------- | ----------------------------------- |
| `event.preventDefault()`                | 기본 이벤트 발생 방지                        |
| `event.stopPropagation()`               | 이벤트 버블링 중단                          |
| `event.isDefaultPrevented()`            | `preventDefault` 호출 여부 확인           |
| `event.isPropagationStopped()`          | `stopPropagation` 호출 여부 확인          |
| `event.stopImmediatePropagation()`      | 동일 요소의 다른 핸들러까지 실행 중단               |
| `event.isImmediatePropagationStopped()` | `stopImmediatePropagation` 호출 여부 확인 |
### 4.2 간편 이벤트 메서드

| **분류**     | **메서드**      | **설명**                                    |
| ---------- | ------------ | ----------------------------------------- |
| **마우스**    | `click`      | 요소를 마우스로 클릭했을 때 발생                        |
|            | `mouseenter` | 마우스 포인터가 요소 영역 안으로 진입할 때 발생               |
|            | `mouseleave` | 마우스 포인터가 요소 영역 밖으로 나갈 때 발생                |
|            | `mousemove`  | 요소 영역 내에서 마우스 포인터가 움직일 때 발생               |
| **폼 / 입력** | `change`     | 입력 요소(`input` 등)의 값이 바뀌고 포커스를 잃을 때 발생     |
|            | `submit`     | `<form>` 양식의 데이터 전송(제출)이 시도될 때 발생         |
|            | `focus`      | 요소가 포커스(입력 대기 상태)를 얻었을 때 발생               |
|            | `blur`       | 요소가 포커스를 잃었을 때(다른 곳 클릭 등) 발생              |
|            | `select`     | `input`이나 `textarea`의 텍스트를 선택(드래그)했을 때 발생 |
| **키보드**    | `keydown`    | 키보드의 키를 누르는 순간 발생                         |
|            | `keyup`      | 누르고 있던 키보드의 키를 뗄 때 발생                     |
| **윈도우**    | `resize`     | 브라우저 창(Window)의 크기가 변경될 때 발생              |
|            | `scroll`     | 스크롤바가 움직이거나 페이지가 스크롤될 때 발생                |

---



### 4.3 이벤트 객체 (Event Object)

이벤트 핸들러(`function(event){...}`)의 매개변수로 전달됩니다.

- **속성**: `event.target`(발생 요소), `event.pageX/Y`(좌표), `event.which`(키 값)
    
- **메서드**:
    
    - `event.preventDefault()`: 태그의 기본 동작(페이지 이동/폼 전송) 중단
        
    - `event.stopPropagation()`: 이벤트 버블링(상위 전파) 중단

## 5. 이펙트 (Effect)

| **종류**        | **메서드**               | **설명**              |
| ------------- | --------------------- | ------------------- |
| **Show/Hide** | `.show()` / `.hide()` | 보이거나 숨김             |
|               | `.toggle()`           | show/hide 번갈아 적용    |
| **Slide**     | `.slideDown()`        | 아래로 펼쳐지며 보임         |
|               | `.slideUp()`          | 위로 접히며 숨겨짐          |
|               | `.slideToggle()`      | slideDown/Up 번갈아 적용 |
| **Fade**      | `.fadeIn()`           | 점점 진해지며 보임          |
|               | `.fadeOut()`          | 점점 희미해지며 사라짐        |
|               | `.fadeToggle()`       | fadeIn/Out 번갈아 적용   |
|               | `.fadeTo()`           | 특정 투명도까지만 조절        |
- 모든 이펙트 메서드는 `(시간, [easing], [callback])` 형태의 인자를 가짐

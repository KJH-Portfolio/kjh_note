## 📝 노트 1. jQuery 개요 및 환경설정

> [!summary] **핵심 요약**
> 
> - **jQuery:** 자바스크립트의 생산성을 높여주는 경량 라이브러리 (Write less, Do more).
>     
> - **연결 방식:** 다운로드(오프라인) 방식과 CDN(온라인) 방식 존재.
>     
> - **실행 시점:** `$(document).ready()` 사용. DOM 트리 생성 직후 실행되어 JS의 `window.onload`보다 빠르고 중복 작성 가능.
>     

### 1. jQuery 개요

#### 1.1 정의 및 목적

- **정의:** 자바스크립트 기반의 라이브러리로, 유용한 함수와 도구들의 모음.
    
- **목적:** 복잡한 DOM 조작 구문을 간소화하고, 적은 코드로 풍부한 기능을 구현함.
    
- **주요 장점:**
    
    - **편의성:** DOM 요소 선택 및 스크립트 구현이 매우 쉬움.
        
    - **기능성:** Ajax(비동기 통신), 이벤트 처리, 애니메이션 등 지원.
        
    - **확장성:** 다양한 플러그인 및 오픈 API 활용 가능.
        

### 2. 라이브러리 연결 방법

HTML의 `head` 태그 내에 `script` 태그로 연결함.

#### 2.1 오프라인 방식 (다운로드)

- 공식 사이트에서 `.js` 파일을 다운로드하여 프로젝트 경로에 저장 및 연결.
    
- **파일 구분:**
    
    - **Uncompressed (일반 JS):** 개발자용. 주석과 들여쓰기가 있어 가독성이 좋음.
        
    - **Compressed (min JS):** 배포용. 용량 최소화를 위해 공백과 주석 제거.
        

#### 2.2 온라인 방식 (CDN)

- **CDN(Content Delivery Network):** 제공된 URL을 통해 웹상에서 직접 라이브러리를 로드.
    
- 인터넷 연결 필수.
    

### 3. 실행 구문 (진입점)

문서의 요소가 로드된 후 코드가 실행되도록 보장하는 구문.

#### 3.1 window.onload (JS) vs $(document).ready (jQuery)

|**구분**|**window.onload (JS)**|**$(document).ready (jQuery)**|
|---|---|---|
|**실행 시점**|모든 요소(이미지 등 외부 리소스 포함) 로드 후|**DOM 트리(HTML 구조)** 완성 직후 (속도 빠름)|
|**작성 횟수**|1회 (중복 시 덮어쓰기 됨)|**N회** (순서대로 모두 실행됨)|

#### 3.2 작성 패턴 (3가지)

1. `jQuery(document).ready(function(){ ... });` (정석)
    
2. `$(document).ready(function(){ ... });` (일반적 사용)
    
3. `$(function(){ ... });` (축약형)
    

---

> [!EXAMPLE] 환경 설정 코드 예시
> 
> jQuery를 CDN으로 연결하고, 문서가 준비되었을 때 로그를 출력하는 기본 구조입니다.

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script>
    <script>
        // 2. 실행 구문 (여러 번 작성 가능, 순차 실행)
        $(document).ready(function(){
            console.log("첫 번째 준비 완료");
        });

        // 3. 축약형 실행 구문
        $(function(){
            console.log("두 번째 준비 완료");
        });
    </script>
</head>
<body>
</body>
</html>
```

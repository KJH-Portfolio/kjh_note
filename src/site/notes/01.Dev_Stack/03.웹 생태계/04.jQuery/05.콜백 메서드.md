# 📘 jQuery 콜백 함수 패턴

> [!summary] **핵심 요약**
> 
> - **콜백 함수(Callback)**: 메서드의 인자로 전달되어, 특정 시점(이벤트 발생, 완료, 반복 등)에 **"나중에 호출되는"** 함수.
>     
> - **jQuery의 힘**: jQuery는 거의 모든 비동기 처리와 제어 흐름을 이 콜백 함수 패턴으로 처리합니다.
>     
> - **주요 패턴**: ①값 계산(Setter), ②이벤트(Event), ③완료 시점(Effect), ④반복(Loop), ⑤통신(Ajax).
>     

---
## 1. 동적 값 설정 (Functional Setter)

값을 설정하는 메서드(`.html`, `.val` 등)에서 **고정된 값 대신 함수를 사용**하여, 요소마다 다른 값을 계산해 넣는 방식입니다.

- **실행 시점**: 메서드 실행 즉시 (요소 개수만큼 반복 실행)
    
- **매개변수**: `index` (순서), `oldValue` (기존 값)
    
- **반환값(`return`)**: **새로 적용할 값**
    

```js
// 예: 모든 리스트 아이템에 번호 매기기
$('li').text(function(index, oldText) {
    return (index + 1) + ". " + oldText;
});
```

## 2. 이벤트 핸들러 (Event Handler)

사용자가 클릭하거나 키보드를 누르는 등 **특정 사건이 터졌을 때** 실행됩니다.

- **실행 시점**: 사용자가 행동을 취했을 때 (비동기)
    
- **매개변수**: `event` (발생한 이벤트 정보 객체)

1. `this`는 DOM 요소다 (순수 JS 객체)

이벤트 핸들러 내에서 `this`를 사용하면, 이는 `document.getElementById('id')` 등으로 얻는 것과 같은 **JavaScript DOM 요소** 그 자체입니다.

- **DOM 객체로서 사용**: `this.style.backgroundColor = 'red';` (JS 방식)
    

2. `$(this)`는 jQuery 객체다 (메서드 사용)

순수한 DOM 객체인 `this`에 jQuery 메서드(예: `.hide()`, `.css()`, `.text()`)를 사용하고 싶다면, `this`를 **jQuery 객체로 감싸주어야** 합니다.

- **jQuery 객체로 변환**: `$(this)`
    
- **jQuery 메서드 사용**: `$(this).text('클릭됨');`
    

```js
$('#btn').on('click', function(event) {
    console.log('클릭 좌표:', event.pageX, event.pageY);
    alert('버튼이 클릭되었습니다!');
});
```

## 3. 이펙트/애니메이션 완료 (Effect Callback)

애니메이션은 시간이 걸리기 때문에, **"모든 움직임이 끝난 직후"**에 무언가를 해야 할 때 사용합니다.

- **실행 시점**: 애니메이션 종료 직후
    
- **주의**: 콜백 없이 다음 줄에 코드를 쓰면, 애니메이션 도중에 실행되어 버립니다.
    

```js
// 1. 박스를 2초간 숨김
$('#box').slideUp(2000, function() {
    // 2. (중요) 이 함수는 2초 뒤, 완전히 사라진 후에 실행됨
    alert('애니메이션 끝! 이제 박스를 삭제합니다.');
    $(this).remove(); 
});
```

## 4. 반복 처리 (Iteration)

`each` 메서드를 사용해 배열이나 요소 집합을 **하나씩 순서대로 꺼내며** 처리합니다.

- **실행 시점**: `each` 메서드 실행 시 (동기)
    
- **매개변수**: `index` (인덱스), `item` (현재 요소 - DOM 객체)
    

```js
$('li').each(function(index, item) {
    // item은 순수 JS 객체이므로 jQuery 메서드를 쓰려면 $(item)으로 감싸야 함
    if (index % 2 === 0) {
        $(item).css('color', 'red'); // 짝수 번째만 빨간색
    }
});
```
---
## 5. 비동기 통신 (Ajax Success)

서버에 데이터를 요청하고, **응답이 무사히 도착했을 때** 실행됩니다.

- **실행 시점**: 서버 응답 도착 시 (비동기 - 언제 끝날지 모름)
    
- **매개변수**: `result` (서버에서 받은 데이터)
    

```js
$.ajax({
    url: '/api/get-user',
    success: function(result) {
        // 서버가 응답을 주면 실행됨
        $('#userName').text(result.name);
    }
});
```

### 📊 패턴 한눈에 비교

|**패턴 종류**|**역할**|**실행 타이밍**|**주요 매개변수**|
|---|---|---|---|
|**Setter**|동적 값 계산|즉시 (Loop)|`index`, `oldValue`|
|**Event**|상호작용 대응|사건 발생 시|`event`|
|**Effect**|후속 작업|애니메이션 종료 후|(없음)|
|**Iteration**|순차 반복|즉시 (Loop)|`index`, `item`|
|**Ajax**|데이터 처리|서버 응답 도착 시|`result` (데이터)|

특히 **3번(이펙트 완료)**과 **5번(Ajax)**은 "코드는 위에서 아래로 실행된다"는 고정관념을 깨야 이해할 수 있는 부분이라, 초보 개발자가 가장 많이 실수하는 구간입니다. 이 두 가지 패턴은 꼭 숙지해 두시길 추천합니다.

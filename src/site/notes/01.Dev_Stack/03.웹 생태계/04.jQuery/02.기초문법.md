# 📘 jQuery 기초 문법

> [!summary] **핵심 요약**
> 
> - **기본 문법**: `$("선택자").메서드("값");` (주어 + 서술어 구조).
>     
> - **jQuery 특징**: 묵시적 반복(루프 없음), 메서드 체이닝(꼬리 물기), 강력한 탐색(가족 찾기).
>     
> - **실무 필수**: `.on()`(이벤트), `.val()`(입력값), `.find()`(자손 찾기), `.closest()`(가장 가까운 조상 찾기).
>     
> - **심화 개념**: **이벤트 위임(Delegation)** - 나중에 생길 요소까지 제어하기.
>     

---

## 1. 기본 문법 (Basic Syntax)

jQuery의 모든 코드는 **"찾아서(Select) -> 실행한다(Do)"**의 구조를 가집니다.

### 1.1 기본 공식

```js
// 문서가 준비되면 실행 (Safe Zone)
$(function() {
    // $("누구를").어떻게한다("값을");
    $("#target").css("color", "red");
});
```

### 1.2 Getter vs Setter (읽기/쓰기)

메서드의 괄호 `()` 안에 값이 있는지에 따라 역할이 바뀝니다.

- **Getter (값 가져오기)**: `$(".txt").text();` (인자 없음)
    
- **Setter (값 변경하기)**: `$(".txt").text("변경");` (인자 있음)

## 2. jQuery만의 고유 특징 (Unique Features)

순수 자바스크립트(Vanilla JS)와 구별되는 가장 강력한 편의 기능 3가지입니다.

### 2.1 묵시적 반복 (Implicit Iteration)

- **특징**: 선택된 요소가 100개라도 반복문(`for`) 없이 한 번에 적용됩니다.
    
- **예시**: `$("li").css("color", "blue");` (모든 li 태그가 파란색이 됨)
    

### 2.2 메서드 체이닝 (Method Chaining)

- **특징**: 한 요소에 여러 명령을 줄바꿈 없이 기차처럼 연결합니다.
    
- **예시**: `$("#box").show().css("color","red").fadeOut();`
    

### 2.3 강력한 DOM 탐색 (Easy Traversing)

- **특징**: JS의 복잡한 `parentNode`, `childNodes` 대신 직관적인 메서드를 사용합니다.
    
- **예시**: `$(this).parent().next();` (내 부모의 다음 형제)

## 3. 실무 빈출 메서드 & 선택자 모음 (Cheat Sheet)

실제 프로젝트에서 90% 이상 사용하는 핵심 기능 리스트입니다.

### 3.1 자주 쓰는 선택자 (Selectors)

|**구분**|**문법**|**설명**|
|---|---|---|
|**기본**|`$("#id")`, `$(".class")`|아이디, 클래스 선택|
|**속성**|`$("input[name='user']")`|특정 속성을 가진 요소 (폼 제어 시 필수)|
|**상태**|`$("input:checked")`|체크된 체크박스/라디오 버튼만 선택|
|**가시성**|`$("div:visible")`|현재 눈에 보이는 요소만 선택|
|**순서**|`$("li:eq(0)")`|첫 번째 요소 (인덱스 0번)|
|**자식**|`$("ul > li")`|직계 자식만 선택|

### 3.2 필수 메서드 (Methods)

#### A. 값/속성 제어

- `.val()`: **(가장 중요)** Input, Select, Textarea의 입력값 읽기/쓰기.
    
- `.text()`: 태그 안의 텍스트만 제어.
    
- `.html()`: 태그를 포함한 HTML 구조 제어.
    
- `.attr("src", "img.jpg")`: 이미지 경로, 링크 등 속성 변경.
    
- `.data("id")`: `data-id="..."` 속성 값을 쉽게 가져옴 (실무 데이터 연동 시 필수).
    

#### B. 스타일 제어

- `.css("property", "value")`: 스타일 직접 지정.
    
- `.addClass("active")`: 클래스 추가 (활성화 상태 만들 때 사용).
    
- `.removeClass("active")`: 클래스 제거.
    
- `.toggleClass("active")`: 클래스 켰다 껐다 반복.
    

#### C. 탐색 (Traversing) - 별표 다섯 개(⭐⭐⭐⭐⭐)

- `.find(".child")`: 내 안에 있는 자손 중 특정 요소 찾기.
    
- `.closest("선언자")`: **(실무 1순위)** 나를 감싸는 조상 중 가장 가까운 놈 찾기.
    
- `.siblings()`: 나 뺴고 나머지 형제들.
    

#### D. 이벤트

- `.on("click", function(){...})`: 클릭 이벤트 연결.
    
- `.trigger("click")`: 강제로 클릭 이벤트 발생시키기.

## 4. 어려운 예시: 동적 요소 제어와 이벤트 위임

초보자가 가장 많이 실패하는 **"나중에 생긴 요소에 이벤트 걸기(Event Delegation)"** 패턴입니다.

> [!EXAMPLE] **시나리오: 동적 할 일 목록(To-Do List)**
> 
> 1. '추가' 버튼을 누르면 리스트(`li`)가 새로 생깁니다.
>     
> 2. 새로 생긴 리스트의 '삭제' 버튼을 누르면 지워져야 합니다.
>     
> 3. **문제점**: 처음에 없던 요소에는 `$(".btn").on("click")`이 먹히지 않습니다.
>     

```js
$(function() {
    // 1. [추가 기능]
    $("#addBtn").on("click", function() {
        // 새로운 HTML을 문자열로 생성
        var newItem = '<li>' +
                      '  <span>새로운 할 일</span>' +
                      '  <button class="delBtn">삭제</button>' +
                      '</li>';
        
        // ul 태그 끝에 추가
        $("#list").append(newItem);
    });

    /* [잘못된 방법] 
       $(".delBtn").on("click", function() {...}); 
       => 페이지 로딩 시점에 있던 버튼에만 작동하고, 
          방금 추가된 버튼은 작동 안 함.
    */

    // 2. [삭제 기능 - 심화: 이벤트 위임]
    // "부모(#list)야, 네 안에 '.delBtn'이 눌리면 대신 처리해줘"
    $("#list").on("click", ".delBtn", function() {
        // this는 클릭된 바로 그 삭제 버튼
        
        $(this)
            .closest("li")  // 1. 버튼의 조상 중 가장 가까운 li(한 줄 전체)를 찾아서
            .slideUp(300, function() { // 2. 부드럽게 올리고
                $(this).remove();      // 3. 애니메이션 끝나면 아예 삭제해라
            });
    });
});
```

**💡 포인트 해석:**

1. **`.closest("li")`**: `parent().parent()` 처럼 복잡하게 안 쓰고, "나를 감싼 `li` 찾아줘"라고 명확하게 지시합니다.
    
2. **이벤트 위임 (`on`의 두 번째 인자)**: `$("#list").on("click", ".delBtn", ...)` 형태를 사용해야 나중에 추가된 버튼도 클릭을 인식합니다. 이 패턴은 실무 게시판, 댓글 시스템 등에서 필수입니다.

# Props 심화 패턴: 함수 전달과 children 활용

> [!summary]
> 전개 연산자를 통한 효율적인 데이터 전파, 함수 전달을 통한 상태 제어, 그리고 **children**을 활용한 레이아웃 컨테이너 구현 패턴 정리.

## 1. 데이터 전파 심화 기법

### 1.1 객체 전달 및 전개 연산자(`...`)

* **객체 단위 관리**: 연관된 데이터를 하나의 묶음으로 처리하여 데이터 응집도 향상.
* **자동 매핑**: 전개 연산자를 사용하여 객체의 모든 속성을 자식의 Props로 일괄 전달.

> [!EXAMPLE] 리스트 렌더링 및 전개 연산자 활용

```jsx
const students = [
    { name: "김지환", year: 1, subject: ['자바'], isPresent: true },
    { name: "박사원", year: 2, subject: ['JS'], isPresent: false }
];

{students.map((student, index) => (
    <StudentCard key={index} {...student} /> // 개별 속성으로 분산 주입
))}

```

## 2. 함수형 Props (부모-자식 상호작용)

부모의 상태 변경 로직을 자식에게 위임하여 하위 이벤트를 상위로 전달.

> [!EXAMPLE] 카운터 제어를 통한 함수 Props 전달

```jsx
//2,3,4번째 전달값은 함수
function Counter({ counter, onIncrement, onDecrement, onReset }) {
    return (
        <div>
            <h2>카운터 : {counter}</h2>
            <button onClick={onIncrement}>증가(+)</button>
            <button onClick={onDecrement}>감소(-)</button>
            <button onClick={onReset}>초기화(0)</button>
        </div>
    );
}
```

## 3. 특수 속성: children 및 레이아웃 관리

### 3.1 children (Container Pattern)

* **정의**: 컴포넌트 여는 태그와 닫는 태그 사이의 모든 내용을 수신하는 예약된 키워드.
* **용도**: 레이아웃 틀은 유지한 채 내부 알맹이만 교체하는 컨테이너 컴포넌트에 활용.

### 3.2 조건부 렌더링 (Early Return)
- 심화 : [[조건부 렌더링 패턴\|조건부 렌더링 패턴]]
* 논리값에 따라 UI의 렌더링 여부를 최상단에서 결정하여 불필요한 연산 방지.

> [!EXAMPLE] 모달 컴포넌트 실전 구조

```jsx
function Modal({ isOpen, title, onClose, children }) {
    if (!isOpen) return null; // Early Return 패턴

    return (
        <div className="modal-overlay"> {/*주변을 반 투명하게 css*/}
            <div className="modal-content"> {/*화면 중앙 하얀 박스 css*/}
                <header>
                    <h2>{title}</h2> {/*전달값*/}
                    <button onClick={onClose}>X</button> {/*전달값*/}
                </header>
                <div>{children}</div>{/*전달값*/}
            </div>
        </div>
    );
}
```

# 리액트 조건부 렌더링 패턴: 일반 방식부터 Enum 매핑까지

> [!summary] 요약
> JSX 내 UI 분기 처리를 위한 기초 문법(삼항 연산자, `&&`)부터 가독성 및 유지보수성을 극대화하는 **객체 매핑**, **함수 분리**, **Enum 상수화** 전략의 통합 정리.

---

## 1. 일반적인 조건부 렌더링 (General Methods)

가장 기본적인 분기 처리 방식으로, 단순한 이분법적 선택이나 예외 처리에 적합.

* **삼항 연산자 (`? :`)**: 조건에 따른 이중 UI 선택.
* **논리 연산자 (`&&`)**: 특정 조건 충족 시에만 UI 노출.
* **Early Return**: 함수 상단에서 조건 미충족 시 즉시 반환을 통한 로직 조기 종료.

```jsx
// 예시: 기본적인 분기 처리 패턴
function UserProfile({ user, isLoading }) {
  // 1. Early Return (로딩 처리)
  if (isLoading) return <p>로딩 중...</p>;

  return (
    <div>
      {/* 2. 삼항 연산자 (데이터 유무) */}
      <h1>{user ? user.name : "로그인 필요"}</h1>
      
      {/* 3. && 연산자 (권한 확인) */}
      {user?.isAdmin && <button>관리자 설정</button>
	      : user?.isSubAdmin && <button>서브 관리자 설정</button>}
    </div>
  );
}
```

## 2. 렌더링 함수 분리 (Function Extraction)

JSX 본문의 비대화를 방지하고 복잡한 분기 로직을 외부로 추출하여 가독성 확보.

* **로직 분리**: UI 선언부와 제어 로직의 역할 명확화.
* **동적 제어**: 매개변수 주입을 통한 상황별 UI 생성 유연성 증대.

```javascript
// 예시: 상태에 따른 액션 버튼 분리 (외부 헬퍼 함수)
const renderActionButton = (status) => {
  if (status === 'pending') return <button>취소</button>;
  if (status === 'error') return <button>재시도</button>;
  return null;
};
```

## 3. 객체 매핑 및 Enum 방식 (Advanced Patterns)

다중 조건 발생 시 선언적 구조를 통해 코드 정합성을 높이는 고도화 기법.

* **객체 매핑**: 상태(Key)와 UI(Value)의 1:1 대응 테이블 구축 및 즉시 조회.
* **Enum (상수화)**: 하드코딩 문자열 배제를 통한 오타 및 런타임 에러 원천 차단.

```javascript
// 예시: Enum과 객체 매핑의 결합
const TAB_TYPE = { HOME: 'HOME', MY: 'MY' };

const TAB_UI = {
  [TAB_TYPE.HOME]: <HomeView />,
  [TAB_TYPE.MY]: <MyView />
};

	**본문 사용방식 : {TAB_UI[currentTab]}**
	{/*currentTab는 변수로 받는다.*/}
```

## 4. 실전 종합 예시 (Example)

> [!EXAMPLE] Enum 상수와 객체 매핑을 활용한 상태별 대시보드

```jsx
import React, { useState } from 'react';

// 1. Enum 정의 (상수화)
const STEP = {
  START: 'START',
  PROCESS: 'PROCESS',
  END: 'END'
};

// 2. 객체 매핑 (UI 매칭)
const STEP_UI = {
[STEP.START]: <p>학습을 시작하세요.</p>,
[STEP.PROCESS]: <p>진행 중입니다...</p>,
[STEP.END]: <p>모든 과정을 마쳤습니다.</p>
};

//상태값 생성
function StepperApp() {
	const [currentStep, setCurrentStep] = useState(STEP.START);
  
	// 3. 함수 분리 (추가 로직 제어)
	const renderActionButtons = () => {
	    if (currentStep === STEP.END){
		    return <button onClick={() => setCurrentStep(STEP.START)}>다시 시작</button>;}
	    
	    return <button onClick={() => setCurrentStep(STEP.PROCESS)}>다음 단계</button>;
	};

	return (
	    <div className="stepper-container">
	      {/* 객체 매핑 결과 출력 */}
	      <div className="view-area">
	        {STEP_UI[currentStep]}
	      </div>
	
	      {/* 함수 분리 결과 출력 */}
	      <div className="action-area">
	        {renderActionButtons()}
	      </div>
	    </div>
	  );
}

```

## 5. 패턴별 비교 및 적용 가이드

| 패턴 | 주요 특징 | 권장 상황 |
| --- | --- | --- |
| **삼항 연산자** | 간결한 문법 | 조건 2개 이하의 단순 분기 |
| **Early Return** | 로직 흐름 단순화 | 예외 케이스 조기 차단 필요 시 |
| **함수 분리** | 파라미터 기반 동적 제어 | 연산 로직이 포함된 복잡한 UI 생성 |
| **객체 매핑** | 선언적 구조, 확장성 우수 | 3개 이상의 다중 조건 및 탭 전환 |
| **Enum** | 데이터 안정성 확보 | 상태 명칭의 엄격한 관리 필요 시 |

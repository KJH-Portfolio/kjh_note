# 리액트의 심장: JSX와 컴포넌트 시스템 구조화

> [!summary] 요약
> 자바스크립트 확장 문법인 **JSX**의 동작 원리, 기존 기술(**JSP**)과의 차이점, 그리고 **Fragment**와 **단일 루트**를 포함한 엄격한 문법 규칙 총정리

## 1. JSX (JavaScript XML) 개념 및 동작 원리

### JSX의 본질

* 자바스크립트 내 HTML 마크업 작성을 위한 **확장 문법**
* 로직(JS)과 구조(HTML)를 **컴포넌트** 단위로 통합 관리
* **선언적 UI** 지향: 상태(What)에 따른 화면 결과 기술 중심

### 동작 메커니즘

* 브라우저 직독 불가: **Babel** 등 트랜스파일러 필수
* 변환 경로: JSX 코드 ➡️ `React.createElement()` 호출 ➡️ **Virtual DOM** (JS 객체) ➡️ 실제 DOM 반영

---

## 2. 기술 비교: JSX vs JSP

| 구분         | JSP (Java Server Pages)        | JSX (React)                       |
| ---------- | ------------------------------ | --------------------------------- |
| **렌더링 주체** | **서버** (Server-Side Rendering) | **클라이언트** (Client-Side Rendering) |
| **결합 방식**  | HTML 내 Java 코드 삽입 (`<% %>`)    | JS 내 HTML 구조 편입                   |
| **관리 단위**  | 페이지 (Page) 중심                  | **컴포넌트 (Component)** 중심           |
| **업데이트**   | 페이지 전체 새로고침 발생                 | **Virtual DOM** 기반 부분 업데이트        |

---

## 3. JSX의 엄격한 문법 규칙

### 단일 루트 요소 (Single Root Element)

* 모든 JSX 요소는 반드시 **하나의 부모 요소**로 감싸기 필수
* 이유: 컴포넌트 반환 값은 단일 자바스크립트 객체여야 함

### React Fragment (`<> </>` / `<React.Fragment>`)

- **불필요한 DOM 생성 방지**: DOM 노드를 추가하지 않고 자식 요소 그룹화(의미 없는 `<div>`제거)
- **단축 문법**: 주로 `<> ... </>`를 사용하여 간결하게 표현
- **key 속성 전달**: 배열 데이터를 이용해 리스트를 렌더링(`map`)할 때, Fragment에 **key 속성을 부여**해야 한다면 단축 문법 대신 명시적 문법인 **`<React.Fragment key={item.id}>`**를 사용해야 함 (단축 문법은 속성을 가질 수 없음)

### 태그의 폐쇄 및 명칭 규약

* **Closing Tags**: 모든 태그의 명시적 닫기 필수 (예: `<br />`, `<img />`)
* **CamelCase 속성**: HTML 속성명 변경 (예: `class` ➡️ `className`, `onclick` ➡️ `onClick`)
* **대문자 시작**: 사용자 정의 컴포넌트는 반드시 대문자로 명명 (소문자 시 HTML 태그 인식)

---

## 4. 모듈 시스템: Import & Export

### Export (내보내기)

* `export default 함수명`: 파일당 1회, 호출 시 자유로운 명칭 지정 가능
* `export {함수명,함수명} `: 다수 내보내기, 중괄호 `{ }` 내 정확한 명칭 사용 필수, 하나도 가능

### Import (불러오기)

* 외부 컴포넌트, 이미지, CSS 자원 로딩
* 컴포넌트 사용 형태: `<ComponentName />`

## 5. 종합 예시 코드

```jsx
import React from 'react';
import Step1_JSX from './01_JSX';

function App() {
  return (
    // 1. Fragment 사용: 단일 루트 규칙 준수 및 불필요 div 제거
    <>
      <header>
        <h1 className="main-title">React 학습</h1>
      </header>
      
      <main>
        {/* 2. 대문자로 시작하는 컴포넌트 호출 */}
        <Step1_JSX />
        
        {/* 3. 셀프 클로징 태그 */}
        <input type="text" placeholder="메모 입력" />
        <br />
      </main>
    </>
  );
}

export default App;

```

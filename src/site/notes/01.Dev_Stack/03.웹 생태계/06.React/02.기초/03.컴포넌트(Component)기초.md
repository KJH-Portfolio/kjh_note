# 리액트 컴포넌트 선언 및 Props 기초

> [!summary]
> UI를 구성하는 **컴포넌트**의 정의 및 선언 방식과 부모 컴포넌트로부터 데이터를 수신하는 **Props**의 기본적인 취득 및 사용 규칙 정리.

## 1. 컴포넌트(Component) 선언 방식

독립적이고 재사용 가능한 UI 단위의 정의 방법.

* **일반 함수형**: `function Welcome() { ... }` 기반의 표준 선언.
* **화살표 함수형**: `const Welcome = () => { ... }` 기반의 간결한 선언 및 스코프 관리.

## 2. Props 수신 및 초기 설정

부모로부터 전달된 데이터를 자식 내부에서 처리하는 기법.

### 2.1 데이터 주입 및 접근

* **부모**: HTML 속성 형태의 데이터 주입 (`<Card name="홍길동" />`).
* **자식**: 함수의 기본 매개변수(`props`)를 통한 객체 수신 및 점 표기법 접근.

### 2.2 구조 분해 할당 (Destructuring) - **권장**

* 매개변수 위치에서 필요한 속성을 즉시 추출하여 코드 가독성 증대.
* 예: `function StudentCard({ name, year }) { ... }`

### 2.3 기본값 설정 (Default Parameters)

* 전달 데이터 부재 시 적용할 예비값 지정으로 렌더링 안정성 확보.
* 문법: `function Component({ text = '기본값' })`

## 3. 컴포넌트 및 Props 작성 규칙 요약

| 항목          | 핵심 규칙 및 특징                                |
| ----------- | ----------------------------------------- |
| **명명 규칙**   | 반드시 **대문자**로 시작 (PascalCase 준수 필수)        |
| **데이터 흐름**  | 부모에서 자식으로만 흐르는 **단방향** 바인딩 원칙             |
| **불변성**     | **읽기 전용(Read-only)** 데이터 (자식 내부 직접 수정 불가) |
| **값 주입 문법** | 문자열: `" "`, 그 외(숫자, 배열, 함수, 객체 등): `{ }`  |

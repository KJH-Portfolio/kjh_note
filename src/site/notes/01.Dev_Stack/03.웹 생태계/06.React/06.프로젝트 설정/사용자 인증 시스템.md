# React Context API 기반 사용자 인증 시스템 구축

>[!summary]
React의 **Context API**와 **LocalStorage**를 결합한 전역 인증 상태 관리 체계. 사용자 정보, 로딩 상태, 인증 여부의 실시간 동기화 및 로그인·로그아웃·권한 확인 로직의 캡슐화 구현.

---

## 인증 컨텍스트 및 프로바이더 설정

### **AuthContext**

* 인증 데이터 전송을 위한 기본 컨텍스트 객체.
* 초기값 `null` 설정 및 데이터 공유의 근간.

### **AuthProvider**

* 하위 컴포넌트 트리 전체에 인증 상태 및 메서드 공급.
* `value` 객체를 통한 데이터 번들링 및 전달.

```javascript
import { createContext, useCallback, useContext, useEffect, useState } from "react";
import authApi from "../api/authApi";

const AuthContext = createContext(null);

export function AuthProvider({children}) {
    return (
        <AuthContext.Provider value={value}>
            {children}
        </AuthContext.Provider>
    );
}

```

## 사용자 인증 상태 및 초기화

### **State** 정의

* **user**: 현재 접속 중인 사용자 정보 객체.
* **isLoading**: 인증 정보 복원 과정 중 UI 차단을 위한 로딩 상태.
* **isAuthenticated**: 불리언 값 기반의 서비스 접근 권한 여부.

### **initAuth** (초기화 로직)

* `useEffect`를 이용한 컴포넌트 마운트 시 1회 실행.
* **LocalStorage** 내 `token` 및 `user` 데이터 존재 여부 검증.
* 저장된 JSON 문자열의 객체 역직렬화 및 상태 반영.
* 파싱 오류 발생 시 저장소 데이터 강제 삭제 및 초기화 처리.

```javascript
const [user,setUser] = useState(null);
const [isLoading,setIsLoading] = useState(true);
const [isAuthenticated,setIsAuthenticated] = useState(false);

useEffect(()=>{
    const initAuth = async () =>{
        const token = localStorage.getItem('token');
        const savedUser = localStorage.getItem('user');

        if(token && savedUser){
            try{
                const userData = JSON.parse(savedUser);
                setUser(userData);
                setIsAuthenticated(true);
            }catch(error){
                console.error('사용자 정보 파싱오류',error);
                localStorage.removeItem('token');
                localStorage.removeItem('user');
            }
        }
        setIsLoading(false);
    }
    initAuth();
},[]);

```

## 인증 핵심 기능 구현

### **Login** 기능

* `useCallback` 기반의 메모이제이션 적용 및 API 호출.
* 서버 응답 데이터 내 토큰 존재 시 **LocalStorage** 영구 저장.
* 사용자 정보의 문자열 직렬화 및 전역 상태 갱신.
* 성공/실패 여부에 따른 결과 객체 반환.

### **Logout** 기능

* 서버 로그아웃 API 호출 시도.
* 성공 여부와 무관하게 클라이언트 내 인증 자격 증명(토큰, 사용자 정보) 삭제.
* 전역 인증 상태 초기화(`null`, `false`).

```javascript
const login = useCallback(async (credentials) =>{
    try{
        const data = await authApi.login(credentials);
        if(data.token){
            localStorage.setItem("token",data.token);
            localStorage.setItem('user',JSON.stringify(data.user||data));
            setUser(data.user||data);
            setIsAuthenticated(true);
            return {success : true,data};
        }
        return {success : false,message: '로그인 실패'};
    }catch(error){
        return {
            success: false,
            message : error.response?.data?.message || '로그인 실패'
        }
    }
},[]);

const logout = useCallback(async ()=>{
    try { 
        await authApi.logout();
    }catch(error){
        console.error('로그아웃 API 오류');
    }finally{
        localStorage.removeItem('token');
        localStorage.removeItem('user');
        setUser(null);
        setIsAuthenticated(false); 
    }
},[]);

```

## 정보 관리 및 접근 최적화

### **User Update** 및 **Role Check**

* **updateUser**: 프로필 변경 등 사용자 정보 갱신 시 저장소 및 상태 동기화.
* **isAdmin**: `user` 객체 내 `ROLE_ADMIN` 포함 여부 판별.
* 사용자 정보 변경 시점에 따른 의존성 관리.

### **useAuth** (커스텀 훅)

* `useContext` 직접 호출의 번거로움 해소.
* **AuthProvider** 외부 사용 시 예외 발생 처리를 통한 안정성 확보.

```javascript
const updateUser = useCallback((userData)=>{
    setUser(userData);
    localStorage.setItem('user',JSON.stringify(userData));
},[]);

const isAdmin = useCallback(()=>{
    return user?.role ==='ROLE_ADMIN';
},[user]);

export function useAuth(){
    const context = useContext(AuthContext);
    if(!context){
        throw new Error('useContext 오류 발생! AuthProvider 내부에서 사용 권장');
    }
    return context;
}
```

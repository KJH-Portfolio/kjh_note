# 리액트 훅(Hooks) 시스템 및 설계 원칙

> [!summary]
> * 클래스 컴포넌트의 복잡성을 해결하고 함수형 컴포넌트에서 상태와 생명주기 기능을 관리하기 위해 도입된 **Hooks**의 정의, 등장 배경 및 엄격한 사용 규칙 정리.
> * 리액트의 **렌더링 단계(Render Phase)**와 **커밋 단계(Commit Phase)** 중 특정 시점에 실행됨. 상태 관리, DOM 조작, 성능 최적화 등 목적에 따른 실행 시점과 기본 문법 정리.
> * **useContext**를 통해 컴포넌트 트리 깊이에 상관없이 데이터를 전역적으로 공유하고 **Props Drilling** 문제를 해결함.
> 
>

# 기본 설명

## 리액트 훅(Hooks)의 정의 및 특징

* 함수형 컴포넌트에서 상태(State) 관리와 생명주기(Lifecycle) 기능을 연동(Hook into)하는 인터페이스.
* 기존 클래스 기반 개발의 복잡성 해소 및 로직의 선언적 관리 가능.

## 도입 배경

* **로직 재사용 용이성**: 고차 컴포넌트(HOC)나 Render Props 패턴의 복잡성 없이 로직 공유 가능.
* **컴포넌트 단순화**: 분산된 생명주기 메서드(`componentDidMount`, `componentDidUpdate` 등)를 단일 API로 통합.
* **유지보수 향상**: 기능 단위의 로직 분리를 통한 가독성 및 개발 효율성 증대.

## 훅 사용 규칙 (Rules of Hooks)

* **최상위 호출 원칙**: 반복문, 조건문, 중첩 함수 내부에서 호출 전면 금지 (동일한 호출 순서 보장 필수).
* **호출 주체 제한**: 오직 리액트 함수형 컴포넌트 또는 커스텀 훅(`use` 접두사 함수) 내부에서만 호출 가능.
* **명명 규칙**: 사용자 정의 훅 생성 시 반드시 `use` 접두사 사용 (예: `useAuth`, `useInput`).

# 내장 Hooks

## 훅별 실행 시점 및 작용 요약표

| 구분 | 훅(Hook) | 실행 타이밍 | 주요 작용 |
| --- | --- | --- | --- |
| **상태** | **useState** | 렌더링 시 | 데이터 보존 및 리렌더링 유발 |
| **전역** | **useContext** | 렌더링 시 | 전역 데이터 즉시 호출 (Props Drilling 해결) |
| **최적화** | **useMemo** | 렌더링 도중 | 연산 결과값 캐싱 |
| **최적화** | **useCallback** | 렌더링 도중 | 함수 참조 유지 (자식 리렌더링 방지) |
| **참조** | **useRef** | 렌더링 도중 | DOM 접근 및 리렌더링 없는 데이터 저장 |
| **효과** | **useLayoutEffect** | 화면 노출 전 (동기) | 레이아웃 측정 및 동기적 UI 수정 |
| **효과** | **useEffect** | 화면 노출 후 (비동기) | 외부 데이터 통신, 부수 효과 처리 |

## 상태 및 전역 데이터 관련 훅

### useState

* **실행 타이밍**: 렌더링 시 현재 상태 반환 및 상태 변경 시 리렌더링 트리거.
* **작용**: 동적 데이터 보존 및 UI 갱신.
* **문법**:

```javascript
const [state, setState] = useState(초기값);
```

### useContext

* **실행 타이밍**: 렌더링 시점에 컨텍스트의 현재 값을 조회.
* **작용**: 중간 단계 컴포넌트를 거치지 않고 전역 저장소(Context)에서 데이터를 직접 수신.
* **문법**:

```javascript
const value = useContext(MyContext);
```

## 생명주기 및 효과 관련 훅

### useEffect

* **실행 타이밍**: 브라우저가 화면을 **그린 후(After Paint)** 비동기적으로 실행.
* **작용**: API 요청, 타이머 등 외부 시스템과의 동기화.
* **문법**:

```javascript
useEffect(() => {
  // 실행 로직
  return () => { /* 클린업 로직 */ };
}, [의존성배열]);
```

### useLayoutEffect

* **실행 타이밍**: DOM 변경 후, 브라우저가 화면을 **그리기 전(Before Paint)** 동기적으로 실행.
* **작용**: DOM 수치 측정 및 레이아웃 수정 (화면 깜빡임 방지).
* **문법**:

```javascript
useLayoutEffect(() => {
  // DOM 측정 및 수정 로직
}, [의존성배열]);
```

## 성능 최적화 및 참조 관련 훅

### useMemo

* **실행 타이밍**: **렌더링 과정 중(During Render)** 동기적으로 실행.
* **작용**: 고비용 연산 결과값을 메모리에 저장하여 재사용.
* **문법**:

```javascript
const memoizedValue = useMemo(() => 고비용연산함수(), [의존성배열]);
```

### useCallback

* **실행 타이밍**: **렌더링 과정 중(During Render)** 동기적으로 실행.
* **작용**: 함수 인스턴스 자체를 메모리에 저장하여 참조 동일성 유지.
* **문법**:

```javascript
const memoizedCallback = useCallback(() => { /* 로직 */ }, [의존성배열]);
```

### useRef

* **실행 타이밍**: 렌더링 과정 중 초기화 및 컴포넌트 생애주기 동안 참조 유지.
* **작용**: DOM 직접 접근 또는 리렌더링을 유발하지 않는 변수 관리.
* **문법**:

```javascript
const ref = useRef(초기값);
// 접근 시: ref.current
```

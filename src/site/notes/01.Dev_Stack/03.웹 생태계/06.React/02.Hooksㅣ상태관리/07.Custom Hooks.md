# 리액트 커스텀 훅(Custom Hooks) 설계 및 구현

> [!summary] 요약
> 리액트 내장 훅들을 조합하여 특정 로직을 추상화하고, 컴포넌트 간 재사용 가능한 형태로 분리한 **사용자 정의 함수** 설계법 정리.

---

## 1. 작성 규칙 및 특징

코드의 재사용성과 가독성 향상을 위한 설계 원칙.

* **명명 규칙**: 반드시 **`use`** 접두사 사용 (예: `useFetch`, `useAuth`, `useInput`).
* **로직 공유**: 컴포넌트 간의 상태(Data) 자체를 공유하는 것이 아닌, **로직 구성 방식(Logic)**의 공유.
* **독립성**: 커스텀 훅을 사용하는 각 컴포넌트는 완전히 독립된 상태 유지.
* **Hooks 제약 준수**: 내장 훅과 동일하게 최상위(Top-level)에서만 호출 가능.

---

## 2. 실전 구현 예시 (POST 방식 데이터 송수신)

비동기 통신 로직을 캡슐화한 `usePost` 훅 정의.

```jsx
import { useState, useEffect } from 'react';

function usePost(url, payload) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    // 전송할 데이터(payload)가 없으면 실행 방지
    if (!payload) return;

    const fetchData = async () => {
      setLoading(true);
      setError(null);
      
      try {
        const response = await fetch(url, {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
          },
          body: JSON.stringify(payload),
        });

        if (!response.ok) throw new Error("데이터 전송 및 수신 실패");

        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url, payload]); // URL 또는 데이터 변경 시 재요청

  return { data, loading, error };
}

```

## 3. 컴포넌트 적용 사례

사용자 등록 폼에서의 커스텀 훅 활용.

```jsx
function UserRegistration() {
  const [requestData, setRequestData] = useState(null);
  
  // payload(requestData)가 설정되는 순간 훅 내부의 useEffect 트리거
  const { data, loading, error } = usePost('https://api.example.com/users', requestData);

  const handleRegister = () => {
    // 버튼 클릭 시 송신할 데이터 설정 및 요청 개시
    setRequestData({
      name: '홍길동',
      email: 'test@example.com'
    });
  };

  return (
    <div>
      <button onClick={handleRegister} disabled={loading}>
        {loading ? '등록 중...' : '사용자 등록'}
      </button>

      {error && <p style={{ color: 'red' }}>에러: {error}</p>}
      {data && (
        <div>
          <h3>등록 완료 결과:</h3>
          <pre>{JSON.stringify(data, null, 2)}</pre>
        </div>
      )}
    </div>
  );
}

```

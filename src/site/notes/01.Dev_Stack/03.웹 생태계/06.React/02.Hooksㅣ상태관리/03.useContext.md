# 리액트 컨텍스트 API (useContext)

> [!summary] 요약
> **useContext**는 컴포넌트 트리의 깊이에 상관없이 데이터를 전역적으로 공유하기 위한 **내장 도구(API)**임. 부모의 데이터를 여러 단계 아래의 자식에게 전달하는 **Props Drilling** 문제를 해결하고, 필요한 곳에서 즉시 데이터를 꺼내 쓸 수 있게 함.

---

## 1. 핵심 구성 요소

* **createContext**: 데이터를 담을 저장소(Context 객체)를 생성함.
	* & [[3. Resource/01.Dev_Stack/03.웹 생태계/06.React/03.API/01.Context\|01.Context]]
* **Provider**: 하위 컴포넌트들에게 데이터를 공급하고 공유 범위를 지정함.
* **useContext**: 생성된 저장소에서 데이터를 읽어오는 Hook임.

---

## 2. 사용법 및 문법

```javascript
// 1. 부모에서 Context 생성 (저장소 구축)
const MyContext = createContext(null);

// 2. 데이터 공급 (부모 쪽에서 Provider로 감싸기)
<MyContext.Provider value={공유할데이터}>
  <ChildComponent />
</MyContext.Provider>

// 3. 데이터 사용 (필요한 자손 쪽에서 직접 호출)
const [value,setValue] = useContext(MyContext);
```

---

## 3. 특징 및 주의사항

* **효율적인 전달**: 중간 단계 컴포넌트들이 전달자 역할을 하지 않아도 되므로 코드가 간결해짐.
* **리렌더링 주의**: 컨텍스트의 `value`가 변경되면 이를 구독(`useContext`)하는 **모든 하위 컴포넌트가 리렌더링**됨.
* **적정 기술**: 단순 1~2단계 전달은 일반적인 **Props**가 유리하며, 로그인 정보나 테마 설정 등 앱 전반의 공통 데이터 관리에 최적화됨.

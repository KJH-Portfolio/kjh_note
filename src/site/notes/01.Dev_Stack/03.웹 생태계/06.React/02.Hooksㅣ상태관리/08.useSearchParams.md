# React useSearchParams hook
> [!summary]
> 
> React Router에서 URL의 **쿼리 스트링(Query String)**을 읽고 수정하기 위해 사용하는 hook. `useState`와 유사한 인터페이스를 통해 URL 상태 관리 및 동적 라우팅 구현 지원.
---
## 주요 개념 및 특징
- **상태 관리 메커니즘**
    - URL의 질의 매개변수(`?key=value`)를 컴포넌트 내 상태처럼 취급.
    - `searchParams`와 `setSearchParams`를 포함한 배열 반환.
- **URLSearchParams 객체 활용**
    - 브라우저 표준 API인 `URLSearchParams`의 인스턴스 사용.
    - 데이터 읽기 시 전용 메서드(`get`, `getAll`, `has` 등) 활용 필수.
---
## searchParams (읽기 전용 객체)
- **get(key)**
    - 특정 키에 해당하는 첫 번째 값을 문자열로 반환.
- **getAll(key)**
    - 동일한 키를 가진 모든 값을 배열 형태로 반환.
- **has(key)**
    - 특정 키의 존재 여부를 불리언 값으로 반환.
- **toString()**
    - 전체 쿼리 스트링을 문자열 형태로 직렬화.(URL문자열)
```javascript
import { useSearchParams } from 'react-router-dom';

const SearchComponent = () => {
  const [searchParams] = useSearchParams();
  const query = searchParams.get('q'); // 'keyword' 추출
  const page = searchParams.get('page'); // '1' 추출

  return (
    <div>
      <p>검색어: {query}</p>
      <p>페이지: {page}</p>
    </div>
  );
};
```
---
## setSearchParams (업데이트 함수)
- **URL 갱신**
    - 인자로 전달된 객체나 `URLSearchParams`를 바탕으로 브라우저 주소창 업데이트.
- **탐색 옵션 (navigate options)**
    - `replace: true`: 히스토리에 기록을 남기지 않고 현재 주소 대체.
    - `state`: 이동 시 특정 상태 데이터 전달 가능.
- **기존 값 유지 주의점**
    - 새로운 값을 설정할 때 기존 파라미터가 덮어씌워지므로, 유지 필요 시 기존 객체 복사 필수.
```javascript
import { useSearchParams } from 'react-router-dom';

const UpdateParams = () => {
  const [searchParams, setSearchParams] = useSearchParams();

  const updateSearch = () => {
    // 특정 파라미터만 변경하고 나머지는 유지하는 예시
    searchParams.set('sort', 'desc');
    setSearchParams(searchParams);
    
    // 객체 형태로 전체 교체하는 예시
    // setSearchParams({ q: 'new-item', page: '2' }, { replace: true });
  };

  return <button onClick={updateSearch}>필터 적용</button>;
};
```
---
## 활용 시 유의 사항
- **문자열 타입 고정**
    - 모든 쿼리 데이터는 문자열로 취급되므로 숫자형 연산 시 형 변환 작업 동반.
- **컴포넌트 리렌더링**
    - `setSearchParams` 호출 시 URL 변경과 함께 관련 컴포넌트 재호출 발생.
- **React Router 의존성**
    - 해당 훅 사용을 위해 `react-router-dom` 라이브러리 설치 및 `BrowserRouter` 환경 필수.

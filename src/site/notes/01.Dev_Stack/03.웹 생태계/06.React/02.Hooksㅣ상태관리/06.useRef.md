# 리액트 useRef 활용 가이드: 이론부터 실전 패턴까지
> [!summary]
> 
> **렌더링 유발 없이** 컴포넌트 생애주기 동안 유지되는 **변경 가능한 참조 객체**의 이론적 배경과 세 가지 주요 활용 패턴(DOM 접근, 변수 보존, 참조 전달).
---
## 1. useRef 이론 및 핵심 메커니즘
- **참조 객체 정의**
    - `current` 프로퍼티를 가진 순수 자바스크립트 객체.
    - `useRef(초기값)` 호출 시 `{ current: 초기값 }` 반환.
- **동작 원리 및 메모리**
    - 힙(Heap) 영역에 저장되어 컴포넌트 리렌더링 시에도 동일한 메모리 주소 유지.
    - **동기적 값 업데이트**: State와 달리 비동기 처리가 아닌 즉각적인 값 반영.
- **State와의 차이점**
    - **State**: 값 변경 시 리액트 트리 재연산 및 UI 갱신(리렌더링) 발생.
    - **Ref**: 값 변경 시 UI 갱신 미발생, 렌더링 로직과 무관한 데이터 처리에 최적.
---
## 2. 활용 패턴 1: DOM 요소 직접 접근
- **필요성**
    - 선언적 프로그래밍(Props/State)으로 해결 불가능한 명령형 API 호출 시 사용.
- **주요 사례**
    - 입력창 자동 포커스 처리.
    - 스크롤 위치 조작 및 애니메이션 실행.
    - 비디오/오디오 플레이어 재생 제어.
```javascript
import { useRef } from 'react';

function DomAccess() {
  const inputRef = useRef(null); // DOM 노드를 담을 참조 변수 선언
  //inputRef = {current : null}

  const handleClick = () => {
    //current를 통해 실제 DOM 노드에 직접 접근 및 포커스 명령 실행
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} type="text" />
      //inputRef= {current : 렌더링된 input 엘리먼트}
      <button onClick={handleClick}>입력창 포커스</button>
    </>
  );
}
```

## 3. 활용 패턴 2: 렌더링 독립적 데이터 보존
- **필요성**
    - 화면 출력에 영향을 주지 않으면서 컴포넌트 내부에서 지속적으로 추적해야 하는 값 관리.
- **주요 사례**
    - `setInterval` 등의 타이머 ID 저장 및 정리.
    - 이전 Props 또는 State 값 저장(이전 데이터 비교).
    - 최초 마운트 여부 확인용 플래그 변수.
```javascript
import { useRef, useState } from 'react';

function StopWatch() {
  const [seconds, setSeconds] = useState(0);
  const timerId = useRef(null); // 리렌더링과 관계없이 ID 보존

  const start = () => {
    timerId.current = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);
  };
  //timerId = {current : setInterval 함수의 고유 타이머 ID(함수 그 자체}

  const stop = () => {
    //저장된 타이머 ID로 인터벌 중지
    clearInterval(timerId.current);
  };

  return (
    <div>
      <p>경과 시간: {seconds}초</p>
      <button onClick={start}>시작</button>
      <button onClick={stop}>정지</button>
    </div>
  );
}
```

## 4. 활용 패턴 3: 컴포넌트 간 참조 전달 (forwardRef)
- **필요성**
    - 캡슐화된 하위 컴포넌트 내부의 특정 DOM 요소에 부모가 접근해야 할 경우 사용.
    - 기본적으로 `ref`는 일반 `props`로 전달되지 않는 특수 속성임에 기인.
- **구현 방식**
    - `forwardRef` 고차 컴포넌트로 자식을 감싸 `ref` 매개변수 노출.
```javascript
import { forwardRef, useRef } from 'react';

//자식 컴포넌트: forwardRef를 사용하여 부모로부터 ref를 전달받음
const MyInput = forwardRef((props, ref) => (
  <input ref={ref} {...props} style={{ border: '2px solid blue' }} />
));

//부모 컴포넌트
function Parent() {
  const customInputRef = useRef(null);

  const focusChild = () => {
    //자식 내부의 input 엘리먼트에 직접 접근
    customInputRef.current.focus();
  };

  return (
    <div>
      <MyInput ref={customInputRef} />
      <button onClick={focusChild}>자식 인풋 포커스</button>
    </div>
  );
}
```

# useState를 활용한 리액트 상태 관리 기초

> [!summary]
> 컴포넌트 내 동적 데이터인 **상태(State)**의 개념과 함수형 컴포넌트의 표준 상태 관리 도구인 **useState**의 문법, 업데이트 패턴 및 주의 사항 요약.

---

## 1. 상태(State)의 핵심 개념

* **반응성(Reactivity)**: 상태 변경 감지 시 리액트 엔진의 UI 리렌더링 트리거.
* **불변성(Immutability)**: 원본 데이터 직접 수정 금지 및 새로운 객체/배열 교체를 통한 업데이트 원칙.
* **캡슐화(Encapsulation)**: 각 컴포넌트가 독립적으로 소유하며 외부와 격리된 데이터 영역.

## 2. useState Hook 문법 구조

* **선언**: `const [state, setState] = useState(initialValue);`
* **state**: 현재 시점의 저장된 데이터 값.
* **setState**: 상태를 갱신하고 리렌더링을 요청하는 전용 함수.
* **initialValue**: 첫 렌더링 시 부여할 초기값 (모든 데이터 타입 허용).

## 3. 상태 업데이트 패턴

* **값 직접 주입**: 특정 값으로 즉시 교체 (`setState(100)`).
* **함수형 업데이트**: 이전 상태(`prev`)를 참조하여 안전하게 갱신 (`setState(prev => prev + 1)`).
	* prev는 예약어가 아니고, setState에 함수가 들어가면 첫번째 자리에는 최신 상태값을 가지고 온다.
* **참조 타입 업데이트**: 객체나 배열 업데이트 시 전개 연산자(`...`)를 활용한 원본 복사 및 일부 변경 필수.

## 4. 실전 종합 예시

> [!EXAMPLE] 카운터 및 텍스트 입력 동시 관리 모델

```jsx
import { useState } from 'react';

function StateManagementExam() {
    // 숫자형 상태 관리
    const [count, setCount] = useState(0);
    // 문자열 상태 관리
    const [text, setText] = useState("");

    return (
        <div>
            <h2>현재 카운트: {count}</h2>
            <button onClick={() => setCount(prev => prev + 1)}>증가</button>
            <button onClick={() => setCount(prev => prev - 1)}>감소</button>
            
            <hr />
            
            <input 
                type="text" 
                value={text} 
                onChange={(e) => setText(e.target.value)} 
                placeholder="내용 입력"
            />
            <p>입력 데이터: {text}</p>
        </div>
    );
}

```

## 5. 상태 관리 주의 사항

* **직접 할당 금지**: `state = newValue`와 같은 할당은 리렌더링을 유발하지 않음.
* **비동기적 처리**: 상태 변경 함수 호출 즉시 변수 값이 변하지 않으며 다음 렌더링 주기에 반영.
* **일괄 업데이트(Batching)**: 성능 최적화를 위해 여러 상태 변경 요청을 그룹화하여 단일 리렌더링으로 처리하는 특성 존재.

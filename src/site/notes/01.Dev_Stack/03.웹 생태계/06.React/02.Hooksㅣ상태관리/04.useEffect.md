# 리액트 부수 효과 관리(useEffect)
> [!summary] 요약
> 컴포넌트 렌더링 외부에 영향을 주는 **부수 효과(Side Effect)**의 정의와 컴포넌트 생명주기에 맞춘 효율적인 제어 기법 정리.
---
## 1. useEffect 정의 및 주요 사례
컴포넌트의 생명주기(Mount, Update, Unmount)와 동기화하여 외부 시스템과의 연결을 수행하는 도구.
* **부수 효과(Side Effect)**: 함수 수행 시 결과값 반환 외에 외부 상태를 변경하는 행위.
* **주요 사례**:
	* 서버 API 데이터 요청 및 수신.
	* `setTimeout`, `setInterval` 등 타이머 설정.
	* `localStorage`, `sessionStorage` 등 브라우저 저장소 접근.
	* `addEventListener`를 통한 이벤트 리스너 등록.
---
## 2. 기본 문법 및 클린업(Cleanup)
부수 효과의 실행과 자원 해제를 위한 표준 구조.
```jsx
import { useEffect } from 'react';

useEffect(() => {

  // 1. 실행할 부수 효과 로직 (Effect)
  // 마운트 시 또는 의존성 변경 시 실행
  
  return () => {
    // 2. 정리 함수 (Cleanup)
    // 언마운트 시 또는 다음 Effect 실행 직전에 수행하여 메모리 누수 방지
  };
  
}, [dependencies]); // 3. 의존성 배열
```
---
## 3. 의존성 배열 패턴에 따른 동작 방식
배열 구성에 따른 실행 시점의 차별화.
* **배열 생략**: 매 렌더링 시마다 Effect 실행 (성능 저하 주의).
* **빈 배열 (`[]`)**: 컴포넌트 마운트(최초 등장) 시 1회만 실행.
* **특정 값 포함 (`[value]`)**: 컴포넌트 마운트 시 + 해당 값 변경 시마다 실행.
---
## 4. 핵심 요약 및 주의 사항

| 항목 | 상세 내용 |
| --- | --- |
| **목적** | 외부 시스템(API, DOM 등)과의 동기화 |
| **주의점** | 불필요한 리렌더링 유발 방지 및 무한 루프 차단 |
| **최적화** | 효과적인 의존성 배열 관리 및 불필요한 Effect 제거 |

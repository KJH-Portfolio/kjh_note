# 리액트 성능 최적화: React.memo, useMemo, useCallback

> [!summary]
> **메모이제이션(Memoization)** 기법을 활용한 불필요한 리렌더링 방지 및 리소스 최적화 전략. 컴포넌트 단위의 **React.memo**, 연산 결과 캐싱을 위한 **useMemo**, 함수 참조 고정을 위한 **useCallback**의 유기적 결합을 통한 앱 성능 극대화.

---

## useMemo: 값(Value) 메모이제이션

* **정의**: 복잡한 연산의 **결과값**을 메모리에 저장하는 훅.
* **작동 원리**: 의존성 배열 내 값 변경 시에만 함수 재실행 및 결과 갱신.
* **활용 사례**:
* **고비용 연산**: 대규모 루프 처리 및 복잡한 산술 계산.
* **데이터 가공**: 필터링(Filter), 정렬(Sort) 등의 로직 적용.



```javascript
// 검색 필터링 최적화 예시
const filterBoards = useMemo(() => {
    console.log('검색 필터링 진행');
    return boards.filter(board =>
        board.boardTitle.includes(searchKeyword) || board.boardWriter.includes(searchKeyword)
    );
}, [boards, searchKeyword]); // 게시글 목록 또는 검색어 변경 시에만 실행

```

---

## useCallback: 함수(Function) 메모이제이션

* **정의**: 함수 인스턴스의 **참조 동일성** 유지를 위한 훅.
* **작동 원리**: 컴포넌트 재생성 시 내부 함수의 참조 주소값을 고정하여 자식 컴포넌트에 동일 객체 전달.
* **최적화 시너지**: `React.memo`를 적용한 자식 컴포넌트에 Props로 함수를 전달할 때 필수적.

```javascript
// 이벤트 핸들러 참조 고정 예시
const handleLike = useCallback((boardNo) => {
    setBoards(prev =>
        prev.map(board =>
            board.boardNo === boardNo ? { ...board, likes: board.likes + 1 } : board
        )
    );
}, []); // 빈 배열 설정 시 최초 렌더링 시에만 함수 생성

```

## React.memo: 컴포넌트 메모이제이션

* **정의**: 고차 컴포넌트(HOC)를 통한 **컴포넌트** 최적화 도구.
* **작동 원리**: 전달받은 **Props**의 변경 여부 확인 및 변경 부재 시 이전 렌더링 결과 재사용.
* **주요 특징**: 부모 컴포넌트 리렌더링 시 발생하는 자식 컴포넌트의 불필요한 연쇄 반응 차단.

```javascript
// 최적화 적용 예시: Props가 동일할 경우 리렌더링 방지
const ChildWithMemo = React.memo(({ name, age }) => {
    console.log('ChildWithMemo 리렌더링');
    return (
        <div>
            <p>이름: {name}</p>
            <p>나이: {age}</p>
        </div>
    );
});
```

## 핵심 비교 및 기술적 차이

| 구분      | useMemo       | useCallback  | React.memo    |
| ------- | ------------- | ------------ | ------------- |
| **대상**  | 함수의 **반환값**   | **함수 인스턴스**  | **컴포넌트(HOC)** |
| **목적**  | 고비용 연산 결과 재사용 | 함수 참조 무결성 유지 | 불필요한 UI 갱신 방지 |
| **의존성** | 의존성 배열 기준     | 의존성 배열 기준    | Props 변경 비교   |

---

## 실무 성능 최적화 가이드

### 권장 상황 (Optimization Needed)

* **대량 리스트 렌더링**: 게시판, 데이터 테이블 등 다수 아이템 표시 시.
* **참조 기반 의존성**: `useEffect`의 의존성 배열에 함수나 객체가 포함되는 경우.
* **복잡한 UI 계층**: 깊은 컴포넌트 트리를 가진 애플리케이션의 전파 비용 절감.

### 주의 사항 (Overhead Warning)

* **단순 컴포넌트**: 리렌더링 비용보다 메모이제이션 비교 비용이 큰 경우 지양.
* **의존성 관리**: 잘못된 의존성 배열 설정으로 인한 데이터 불일치(Stale Closure) 주의.
* **메모리 점유**: 과도한 캐싱으로 인한 메모리 소비 및 가비지 컬렉션 부담 증가 가능성.

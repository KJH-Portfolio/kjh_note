# React 02. Navigate 컴포넌트 상세 활용 및 단계별 구현 사례

> [!summary]
> 요약: `react-router-dom`의 `<Navigate>` 컴포넌트를 활용한 선언적 경로 이동 제어, 히스토리 관리 및 `state` 객체 기반 데이터 전달 패턴 정리.

---

## Navigate 컴포넌트 정의 및 기본 리다이렉션

* **선언적 라우팅 도구**: 컴포넌트 렌더링 시점의 즉시 경로 이동 수행.
* **접근 제한**: 잘못된 경로 접근 차단 및 초기 진입 경로 강제 설정.
* **기본 리다이렉션 구문**: `<Navigate to="이동경로" />`

```javascript
<Routes>
  <Route path="/" element={<Home />} />
  {/* 정의되지 않은 경로 접근 시 홈으로 즉시 이동 */}
  <Route path="*" element={<Navigate to="/" />} />
</Routes>

```

---

## replace 속성 기반 히스토리 스택 관리

* **히스토리 대체**: 현재 기록을 새 경로로 대체하여 뒤로 가기 버튼 활성화 방지.
* **활용 사례**: 로그인 성공 후 로그인 페이지 재진입 차단 및 결제 완료 후 이전 페이지 복귀 방지.
* **히스토리 대체 구문**: `<Navigate to="경로" replace />`

```javascript
// 결제 완료 후 다시 결제 페이지로 돌아오지 못하도록 기록 대체
const SuccessPage = ({ isOrderComplete }) => {
  if (!isOrderComplete) {
    return <Navigate to="/cart" replace />;
  }
  return <div>주문 완료 페이지</div>;
};
```

## state 속성을 활용한 선언적 데이터 전송

* **컨텍스트 유지**: 이동 대상 페이지에 임의의 객체를 전달하여 정보 상태 유지.
* **비노출 정보 전달**: URL에 노출되지 않는 보안 데이터 및 상태 정보 전송.
* **데이터 포함 전송 구문**: `<Navigate to="경로" state={{ key: value }} />`

```javascript
// 1단계에서 2단계로 이동 시 입력 데이터 전달
const StepOne = () => {
  const [email, setEmail] = useState("");
  const isEmailValid = email.includes("@");

  return (
    <div>
      {isEmailValid && <Navigate to="/step-2" state={{ userEmail: email }} />}
      <input onChange={(e) => setEmail(e.target.value)} placeholder="이메일 입력" />
    </div>
  );
};
```

## 실전 패턴: Protected Route와 Redirect Back

* **인증 기반 접근 제어**: 비인증 사용자의 특정 경로 접근 차단 및 로그인 페이지 강제 이동.
* **원래 경로 복구**: 이동 전 경로를 `state`에 저장하여 로그인 완료 후 이전 목적지 복귀 유도.
* **현재 위치 저장 구문**: `state={{ from: location.pathname }}`

```javascript
const ProtectedRoute = ({ children }) => {
  const token = localStorage.getItem('token');
  const location = useLocation();

  if (!token) {
    // 현재 페이지 경로를 state에 담아 로그인 페이지로 전송
    return <Navigate to="/login" state={{ from: location.pathname }} replace />;
  }

  return children;
};

const LoginPage = () => {
  const location = useLocation();
  const navigate = useNavigate();
  
  // 전달받은 state.from 값 참조 또는 기본 경로 설정
  const from = location.state?.from || "/";

  const handleLogin = () => {
    localStorage.setItem('token', 'auth_key');
    // 로그인 완료 후 원래 목적지로 이동
    navigate(from, { replace: true });
  };

  return <button onClick={handleLogin}>로그인</button>;
};

```

## 조건부 접근 제어 및 중복 진입 방지

* **권한별 분기**: 사용자 등급(Role)에 따른 페이지 접근 권한 차등 적용.
* **중복 진입 차단**: 이미 로그인된 상태의 사용자 대상 로그인/회원가입 페이지 접근 제한.
* **권한 확인 및 리다이렉션**: `if(조건) return <Navigate ... />` 패턴 활용.

```javascript
const AdminSection = ({ userRole }) => {
  // 관리자 권한 미달 시 권한 부족 페이지로 이동 및 사유 전달
  if (userRole !== 'ADMIN') {
    return <Navigate to="/unauthorized" state={{ reason: '권한 부족' }} replace />;
  }
  return <AdminPanel />;
};

```

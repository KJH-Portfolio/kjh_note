# 리액트(React) 핵심 아키텍처 및 입문 가이드

> [!summary]
> **리액트(React)**의 근간인 가상 DOM과 단방향 데이터 흐름, 컴포넌트 조립 방식의 UI 구축, **합성(Composition) 패턴** 및 **Hooks**를 활용한 현대적 프론트엔드 개발 체계 요약.

---

## 1. 리액트 동작 원리 및 철학

효율적인 UI 업데이트와 예측 가능한 데이터 관리를 위한 설계 원칙.

* **가상 DOM (Virtual DOM)**: 실제 DOM의 가벼운 복사본을 메모리에 유지하여 변경 사항만 선별적으로 실제 DOM에 반영(Reconciliation)하는 성능 최적화 기법.
* **단방향 데이터 흐름 (Unidirectional Data Flow)**: 데이터가 항상 부모에서 자식 방향으로만 흐르는 구조를 통한 데이터 추적성 및 디버깅 용이성 확보.

---

## 2. 컴포넌트와 JSX 규칙

UI의 최소 단위인 컴포넌트 정의 및 자바스크립트 확장 문법(JSX) 활용법.

### 2.1 컴포넌트 (Component)

* **정의**: 화면을 구성하는 독립적인 UI 블록 (자바스크립트 함수 형태).
* **역할**: UI 구조 결정 및 반환(return).

### 2.2 JSX 및 작성 규칙

* **개념**: 자바스크립트 내 HTML 유사 문법 사용.
* **Fragment (<> </>)**: 불필요한 `<div>` 태그 없이 여러 요소를 그룹화.
* **CamelCase 속성**: HTML 속성명을 자바스크립트 명명 규칙(`className`, `onClick` 등)으로 작성.
* **태그 폐쇄**: 모든 태그의 명시적 닫기(`<br />`, `<img />`) 필수.

---

## 3. 데이터 관리 (Props & State)

컴포넌트 간 통신 및 내부 동적 데이터 처리 전략.

| 항목 | Props (프롭스) | State (스테이트) |
| --- | --- | --- |
| **정의** | 부모로부터 전달받은 데이터 | 컴포넌트 내부에서 보유한 동적 데이터 |
| **성격** | 읽기 전용 (수정 불가) | 변경 가능 (변경 시 리렌더링 유발) |
| **비유** | 외부에서 주입된 설정값 | 컴포넌트의 기억(메모리) |

### 3.1 상태 끌어올리기 (Lifting State Up)

* **목적**: 동일 데이터를 공유하는 하위 컴포넌트 간의 데이터 동기화.
* **방법**: 상태를 공통 부모 컴포넌트로 이동시킨 후 Props를 통해 하향 전달.

---

## 4. 컴포넌트 합성(Composition) 패턴

상속 대신 조합을 통해 유연한 UI를 구축하는 리액트의 핵심 권장 패턴.

* **담기 (Containment)**: 하위 요소 내용을 미리 알 수 없는 범용 틀(Layout) 제작 방식.
* **children 활용**: 컴포넌트 태그 사이의 콘텐츠를 자동으로 전달받아 특정 위치에 배치.
* **특수화 (Specialization)**: 범용 컴포넌트에 특정 데이터를 주입하여 구체적 목적의 컴포넌트로 변주.

---

## 5. UI 로직 및 제어

상태에 따른 화면 분기 및 대량 데이터 렌더링 기법.

### 5.1 조건부 렌더링

* **논리 연산자 (&&)**: 참일 때만 요소 표시. (0 출력 부작용 주의 및 불리언 변환 권장)
* **삼항 연산자 (? :)**: 조건에 따른 상이한 요소 선택 출력.
* **Early Return**: 특정 조건 충족 시 함수 상단에서 즉시 `null` 반환.

### 5.2 리스트 렌더링 (map)

* **동작**: 배열 데이터를 순회하여 JSX 엘리먼트로 변환.
* **key 속성**: 각 항목의 고유성 식별을 위한 필수 속성 지정.

---

## 6. 훅(Hooks)과 생명주기

함수형 컴포넌트에 상태와 생명주기 기능을 부여하는 내장 함수.

* **생명주기 (Lifecycle)**: 마운트(등장), 업데이트(변경), 언마운트(제거)의 과정.
* **useState**: 컴포넌트 내 상태 선언 및 변경 함수 제공.
* **useEffect**: 부수 효과(API 호출, 타이머 등) 처리 및 생명주기 동기화.

---

## 7. 종합 실전 예시

> [!EXAMPLE] 합성 패턴과 상태 관리가 결합된 통합 모델

```jsx
import { useState } from 'react';

// 1. 합성 패턴: 레이아웃 프레임 (틀)
function PageLayout({ children, title }) {
  return (
    <div className="layout-container">
      <header><h1>{title}</h1></header>
      <main>{children}</main>
    </div>
  );
}

// 2. 메인 앱 컴포넌트 (조립)
function CounterApp() {
  const [count, setCount] = useState(0);
  const items = ['React', 'Composition', 'Hooks'];

  return (
    <PageLayout title="나의 리액트 학습">
      {/* 3. 상태 제어 및 이벤트 */}
      <section>
        <p>현재 클릭 횟수: {count}</p>
        <button onClick={() => setCount(count + 1)}>증가</button>
      </section>

      {/* 4. 조건부 렌더링 */}
      {count >= 5 && <p>축하합니다! 학습 중반 도달.</p>}

      {/* 5. 리스트 렌더링 */}
      <ul>
        {items.map((item, idx) => (
          <li key={idx}>{item}</li>
        ))}
      </ul>
    </PageLayout>
  );
}

```

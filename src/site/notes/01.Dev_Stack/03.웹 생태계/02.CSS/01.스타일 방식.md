# 📘 CSS 스타일 방식 정리

## Summary 📝
> [!summary] Summary 📝
> - CSS 스타일은 **인라인**, **내부**, **외부** 방식으로 적용, 각각 적용 범위와 유지보수성 상이.  
> - 우선순위: 인라인 > 내부/외부 (특이성 및 선언 순서 따라 결정).

---

## 핵심 개념 💡
- **인라인 스타일**: HTML 요소의 `style` 속성에 직접 작성.  
- **내부 스타일**: `<style>` 태그로 HTML 문서 내 정의.  
- **외부 스타일**: 별도 `.css` 파일로 작성, `<link>`로 연결.  

#### 비교표: CSS 스타일 방식
| **방식**  | **설명**                      | **적용 범위**  | **특이성 점수**   |
| ------- | --------------------------- | ---------- | ------------ |
| **인라인** | HTML 태그의 `style` 속성에 작성     | 단일 요소      | 1000         |
| **내부**  | `<head>` 내 `<style>` 태그에 작성 | 단일 HTML 문서 | 0~100 (선택자별) |
| **외부**  | `.css` 파일로 작성, `<link>`로 연결 | 다수 HTML 문서 | 0~100 (선택자별) |

---

## 특징 ✨
- ✅ **인라인**: 즉시 적용, 특정 요소 타겟팅 용이.  
- ✅ **내부**: 단일 문서 내 스타일 관리 간편.  
- ✅ **외부**: 재사용성, 유지보수성 우수, 다수 페이지 적용 가능.  
- ✅ **우선순위**: 인라인 > 내부/외부 (동일 특이성 시 나중 선언 우선).

---

## 팁과 주의점 ⚠️
- ✅ **외부 스타일 권장**: 유지보수와 확장성 위해 `.css` 파일 사용.  
- ⚠️ **인라인 제한**: 특이성 높아 스타일 충돌 가능, 테스트용으로 제한 사용.  
- ✅ **캐시 활용**: 외부 CSS는 브라우저 캐싱으로 로딩 속도 개선.  
- ⚠️ **파일 경로**: 외부 CSS의 `<link>` 경로 오류 주의.

---

## 예시 📌
### ✅ 외부 스타일
```html
<!-- HTML -->
<head>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <p>외부 스타일 문단</p>
</body>
```
```css
/* styles.css */
p { color: red; font-size: 18px; }
```
**실행 흐름**: `styles.css` 파일 연결, 모든 연결된 HTML 문서에 스타일 적용.

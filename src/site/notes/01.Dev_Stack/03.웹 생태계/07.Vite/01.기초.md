# Vite 기초 및 프록시 설정 가이드

> [!summary]
> **Vite**의 기본 구성, **경로 별칭(Alias)**, 그리고 **API 프록시** 설정을 한눈에 확인하기 위한 가이드입니다.

## 1. Vite 기본 구성 요소

> [!info] Vite 핵심 API 및 플러그인
> * **defineConfig**: 설정 객체에 대한 타입 인텔리전스(자동 완성)를 지원합니다.
> * **plugins**: `react()` 플러그인을 통해 JSX 변환 및 HMR(Hot Module Replacement)을 활성화합니다.
> 
> 

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  // ... 기타 설정
})

```

---

## 2. 경로 별칭 설정 (Path Alias)

> [!note] 복잡한 상대 경로를 간결한 별칭으로 대체
> * **resolve.alias**: `../../`와 같은 복잡한 경로 대신 `@`와 같은 별칭을 사용합니다.
> * **path.resolve**: 현재 위치(`__dirname`)를 기준으로 실제 시스템 경로를 계산합니다.
> 
> 

```javascript
import path from 'path'

export default defineConfig({
  resolve: {
    alias: {
      // './src' 를 '@'으로 접근 가능
      '@': path.resolve(__dirname, './src'),
      // 'src/components/Button' 대신 '@components/Button'으로 접근 가능(가독성)
      '@components': path.resolve(__dirname, './src/components')
    }
  }
})
```

## 3. 서버 및 프록시 설정 (Server & Proxy)

> [!tip] 개발 서버 환경 제어 및 CORS 문제 해결
> * **proxy**: 특정 경로(예: `/api`)로 들어오는 요청을 백엔드 서버로 전달합니다.
> * **changeOrigin**: 요청 헤더의 Host를 타겟 서버 주소로 변경하여 보안 정책을 우회합니다.
> * **rewrite**: 요청 경로에서 특정 문구를 추가하거나 삭제합니다. (예: `/api/user` → `/spring/api/user`)
> 
> 

```javascript
export default defineConfig({
  server: {
    proxy: {
      "/api": { // '/api'로 연결되는 요청을
        target: "http://localhost:8080", // 백엔드 서버 주소로 보내기
        changeOrigin: true,
        secure: false,
        // 클라이언트의 /api 요청을 서버의 /spring/api로 변경
        rewrite: (path) => `/spring${path}`
      },
    },
  },
})

```

---

> [!warning] TypeScript 사용 시 필수 체크
> `vite.config.ts` 설정만으로는 VS Code 상의 경로 에러(빨간 줄)가 사라지지 않습니다. 반드시 `tsconfig.json` 파일에도 동일한 경로 설정을 추가해야 에러 없이 자동 완성을 사용할 수 있습니다.

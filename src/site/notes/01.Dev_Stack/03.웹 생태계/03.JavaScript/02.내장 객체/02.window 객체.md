# JS Window 전역 객체 및 DOM/BOM 구조 핵심 정리

> [!summary]
> 브라우저 환경의 최상위 **Window 객체** 제어법, 시간 기반 비동기 처리(**Interval**), 브라우저 정보(**BOM**) 및 문서 객체 모델(**DOM**)의 체계적 탐색과 동적 조작 기법 요약.

---

## 1. Window 객체 및 창 제어

브라우저 창을 대변하는 최상위 전역 객체로서 모든 전역 변수와 함수의 근간.

### 1.1 창 열기 및 닫기

* **`window.open(URL, 창이름, 옵션)`**: 새 창 또는 탭 생성.
* **URL**: 대상 주소 지정 (공백 시 `about:blank`).
* **창이름**: 기존 창 재사용 및 새로고침 기준값.
* **옵션**: `width`, `height`, `left`, `top` 등 팝업 크기 및 위치 제어.


* **`window.close()`**: 현재 창 닫기 (스크립트로 생성된 창에 한함).

---

## 2. 타이머 및 인터벌 (Timer & Interval)

특정 시간 경과 후 로직을 실행하거나 일정 간격으로 반복하는 비동기 제어 기법.

### 2.1 주요 메서드 비교

| 메서드 | 실행 방식 | 중지 메서드 |
| --- | --- | --- |
| **`setTimeout`** | 설정 시간 경과 후 **1회** 실행 | `clearTimeout(id)` |
| **`setInterval`** | 설정 시간 간격으로 **무한 반복** 실행 | `clearInterval(id)` |

### 2.2 인터벌(Interval)의 상세 특징

* **반복성**: 중단 지시 전까지 지정된 밀리초(ms) 단위로 콜백 함수 지속 호출.
* **ID 반환**: 실행 시 고유 식별 번호(ID) 반환 및 이를 이용한 실행 중단 필수.

```javascript
// 디지털 시계 구현 예시
const timerId = setInterval(() => {
    const now = new Date();
    const timeStr = now.toLocaleTimeString();
    document.querySelector("#display").innerText = timeStr;
}, 1000); // 1초 간격 반복

// 중단 시
// clearInterval(timerId);

```

## 3. BOM (Browser Object Model)

브라우저의 환경 및 상태 정보를 관리하는 객체 집합.

* **`location`**: 현재 URL 정보 관리 및 페이지 이동(`href`, `replace`, `reload`).
* **`history`**: 브라우저 방문 기록 탐색(`back`, `forward`, `go`).
* **`navigator`**: 브라우저 엔진, 버전, 운영체제 정보 제공.
* **`screen`**: 사용자 모니터 해상도 및 가용 영역 정보 제공.

## 4. DOM (Document Object Model) 기초 및 탐색

HTML 문서를 노드(Node) 트리 구조로 형상화한 인터페이스.

### 4.1 노드 분류

* **요소 노드(Element Node)**: HTML 태그 그 자체.
* **텍스트 노드(Text Node)**: 태그 내부의 문자열 정보 (공백 및 줄바꿈 포함).

### 4.2 노드 탐색 메서드 비교

* **`firstChild` / `lastChild**`: 모든 타입의 노드(공백 포함) 중 첫 번째/마지막 탐색.
* **`firstElementChild` / `lastElementChild**`: 오직 **요소 노드(태그)** 중 첫 번째/마지막 탐색.
* **`children`**: 자식 요소 노드들만의 컬렉션 반환.

---

## 5. DOM 동적 생성 및 조작 (Create & Append)

### 5.1 요소 생성 및 추가 절차

1. **생성**: `document.createElement("태그명")` 호출.
2. **속성**: `.src`, `.className`, `.id` 등 속성 직접 주입.
3. **연결**: 부모 노드에 삽입 메서드 실행.

### 5.2 삽입 메서드 비교 (`append` vs `appendChild`)

| 구분 | `appendChild()` | `append()` |
| --- | --- | --- |
| **인자 형식** | 오직 **Node 객체** | **Node 객체 + 문자열** |
| **인자 개수** | 단일 노드 제한 | 다수 노드 동시 전달 가능 |
| **반환값** | 추가된 Node 객체 | 반환값 없음 (`undefined`) |

### 5.3 위치별 조작 메서드

* **`prepend()`**: 부모 노드의 첫 번째 자식으로 삽입.
* **`before()` / `after()**`: 기준 노드의 형제 레벨 앞/뒤에 삽입.
* **`remove()`**: 해당 요소 자체를 문서에서 제거.

```javascript
// 동적 리스트 항목 추가 예시
const list = document.querySelector("#myList");
const newItem = document.createElement("li");

newItem.innerText = "새로운 항목";
newItem.className = "list-item";

list.append(newItem, "텍스트 추가 가능"); // 가장 뒤에 추가
list.prepend("공지사항: "); // 가장 앞에 추가

```

---

**수정 사항**:

* **인사말 및 서술형 종결 제거**: 모든 문장을 명사/명사구로 종결하여 구조화 완료.
* **Interval 상세 내용 보강**: `setInterval`의 작동 원리, ID 반환, 중단 방법(`clearInterval`) 및 실무 활용 예시 추가.
* **DOM 조작 기법 체계화**: 생성부터 삽입 위치별 메서드(`prepend`, `before` 등)까지 세분화하여 정리.

**제안 사항**:

* **이벤트 위임(Event Delegation)**: 동적으로 생성된 요소(`append` 등)에 이벤트를 효율적으로 할당하는 기법 학습 추천.
* **Template Tag**: 복잡한 HTML 구조를 생성할 때 `createElement` 대신 활용 가능한 `<template>` 태그 사용법 탐구 권장.

**역질문**:

* `setInterval` 사용 시 발생할 수 있는 **메모리 누수(Memory Leak)** 방지를 위한 `clearInterval` 호출 타이밍에 대한 상세 예시가 필요하신가요?

# 🔄 JS 배열 순회: for...of, forEach, for...in

> [!summary] **핵심 요약**
> * **`for...of`**: 배열 순회의 현대적 표준. **`break` 사용 가능**하며, JSX 리턴문 밖의 로직 처리 시 유용합니다.
> * **`forEach`**: 배열 요소를 단순 반복 실행. 반환값이 없으므로 상태(State) 업데이트나 로그 출력에 사용됩니다.
> * **`for...in`**: **객체 전용**. 배열 순회에는 사용을 엄격히 금지합니다.
> 
> 

## 1. `for...of` 루프 (Modern Standard)

배열의 값을 하나씩 꺼내어 처리하는 가장 직관적인 방식입니다.

* **특징**: ES6 표준 문법으로, `break`, `continue` 등을 사용하여 반복 흐름을 자유롭게 제어할 수 있습니다.
* **JSX 활용**: JSX의 `return` 문 안에서는 사용할 수 없으므로, 컴포넌트 상단에서 데이터를 미리 가공하거나 비동기 처리(async/await) 시 주로 사용됩니다.

```javascript
// 검색 로직 예시 (JSX 외부 로직)
const users = ['Kim', 'Lee', 'Park'];

for (const user of users) {
  if (user === 'Lee') {
    console.log("사용자를 찾았습니다.");
    break; // 원하는 값을 찾으면 즉시 중단 (효율적)
  }
}

```

---

## 2. `forEach` 메서드 (Execute)

배열의 각 요소에 대해 제공된 함수를 한 번씩 실행합니다.

* **특징**: 반환값(`return`)이 항상 `undefined`입니다. 즉, 배열을 변형하지 않고 **부수 효과(Side Effect)**를 일으키는 데 집중합니다.
* **주의**: 반복 중간에 `break`로 멈출 수 없습니다.
* **JSX 활용**: 주로 이벤트 핸들러 내부에서 여러 상태를 동시에 업데이트하거나, API 요청을 보낼 때 사용합니다.

```javascript
// 상태 업데이트 예시
const nums = [1, 2, 3];
nums.forEach((num) => {
  // 예: setCounts(prev => [...prev, num * 2]);
  console.log(`현재 데이터 처리 중: ${num}`);
});

```

---

## 3. `for...in` 루프 (Object Only)

배열이 아닌 **객체의 속성(Key)**을 순회할 때 사용합니다.

* **주의**: 배열에서 사용하면 인덱스 순서가 보장되지 않고 성능상 불리하므로 **배열 순회에는 절대 권장하지 않습니다.**
* **JSX 활용**: 컴포넌트의 `props` 객체를 검사하거나, 서버에서 받아온 설정 객체의 키-값을 추출할 때 유용합니다.

```javascript
const styles = { color: 'red', fontSize: '16px' };

for (const key in styles) {
  console.log(`${key}: ${styles[key]}`); 
  // 출력: color: red, fontSize: 16px
}

```

---

## 💡 JSX 로직 내 사용 비교

이 세 가지 방식은 모두 **"문(Statement)"** 혹은 **"반환값 없음"**의 특징을 가지므로, JSX의 `return` 블록 내부에서 직접 사용할 수 없습니다.

| 구분 | `for...of` | `forEach` | `for...in` |
| --- | --- | --- | --- |
| **순회 대상** | 배열 (Array) | 배열 (Array) | **객체 (Object)** |
| **흐름 제어** | `break`, `continue` 가능 | 불가 | 가능 |
| **JSX 위치** | `return` 키워드 이전 로직 영역 | 이벤트 핸들러 혹은 `useEffect` 내부 | 객체 데이터 처리 로직 |

---

> [!EXAMPLE] **JSX에서의 올바른 배치**

```jsx
function UserList({ users }) {
  // 1. return 밖에서 for...of로 데이터 정제 가능
  const activeUsers = [];
  for (const user of users) {
    if (user.active) activeUsers.push(user);
  }

  const handleClick = () => {
    // 2. 이벤트 핸들러 내부에서 forEach 사용
    users.forEach(u => console.log(`클릭 로그: ${u.name}`));
  };

  return (
    <button onClick={handleClick}>로그 출력</button>
    /* 주의: 이 안에서 for...of나 forEach를 직접 쓸 수 없음 */
  );
}

```

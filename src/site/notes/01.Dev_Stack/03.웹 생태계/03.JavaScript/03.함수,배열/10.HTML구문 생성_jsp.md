# # JS 배열 메서드를 활용한 HTML 생성 (map & join)

> [!summary] **핵심 요약**
> * **현대적 패턴**: `let` 변수를 통한 누적 방식(`forEach`) 대신, 배열을 직접 변환(`map`)하고 합치는(`join`) 선언적 방식을 사용합니다.
> * **성능과 가독성**: 매번 문자열을 새로 만드는 대신 배열에 담아 한 번에 합치므로 성능이 좋고, HTML 구조가 한눈에 들어옵니다.
> * **안정성(방어 코드)**: 데이터가 비어 있을 경우(`list.length === 0`)에 대한 예외 처리를 추가하여 사용자 경험을 개선합니다.
> 
> 

---

## 1. 작동 원리 및 흐름

이 패턴은 **데이터 배열 → 문자열 배열 → 단일 문자열**의 3단계 공정을 거칩니다.

1. **`map()`**: 원본 데이터 객체를 하나하나 HTML 텍스트(문자열)로 변환하여 새로운 배열을 생성합니다.
2. **`join('')`**: 생성된 배열 요소들 사이의 콤마(`,`)를 제거하고 빈 문자(`''`)로 이어 붙여 긴 HTML 문자열을 만듭니다.
3. **DOM 삽입**: 완성된 문자열을 `innerHTML`을 통해 화면에 한 번에 출력합니다.

---

## 2. `map().join('')` vs `forEach` 비교

| 비교 항목 | `forEach` (누적 방식) | `map().join('')` (변환 방식) |
| --- | --- | --- |
| **코드 스타일** | 명령형 (절차 중심) | **선언형 (결과 중심)** |
| **외부 변수** | `let html = ""` (필요함) | **불필요 (즉시 할당 가능)** |
| **성능** | 반복마다 메모리 재할당 | 배열에 모아 한 번에 결합 (효율적) |
| **코드 가독성** | 로직과 문자열이 섞임 | **HTML 구조가 직관적으로 보임** |

---

## 3. 예외 상황 처리: 방어 코드 (Defensive Coding)

서버에서 가져온 데이터가 없거나 배열이 비어 있을 경우, 빈 화면이 출력되는 것을 방지하기 위해 **조건부 렌더링**을 적용합니다.

* **원리**: `list.length`가 0인지 확인하여 처리합니다.
* **삼항 연산자 활용**: `리스트.length === 0 ? "없을 때" : "있을 때"`

```javascript
// 리스트가 비어있을 때 "데이터가 없습니다" 라는 행(tr)을 보여주는 로직
const html = list.length === 0 
    ? "<tr><td colspan='4' class='text-center'>신청 내역이 없습니다.</td></tr>"
    : list.map(app => `<tr>...</tr>`).join('');

```

---

## 4. JSP 환경에서의 주의사항 (중요)

JSP 파일 내의 `<script>` 태그 안에서 백틱(```)을 사용할 경우, 자바스크립트 변수 `${app.id}`와 JSP의 EL 표기법(`${...}`)이 충돌합니다.

* **해결책**: JS 문법인 `${}` 앞에 역슬래시(`\`)를 붙여 서버의 간섭을 막아야 합니다.
* **형식**: `\${app.variableName}`

---

## 5. 실무 적용 예시 (Full Code)

모든 개념(방어 코드 + 백틱 + 이스케이프 + map/join)이 적용된 최종 코드입니다.

```javascript
const listBody = document.querySelector("#listBody");

// [방어 코드 + map + join 적용]
const resultHtml = list.length === 0 
    ? `<tr><td colspan="4" style="text-align:center;">신청 내역이 없습니다.</td></tr>`
    : list.map(app => `
        <tr>
            <td>\${app.adoptionAppId}</td>
            <td>\${app.userId} (\${app.userName || "이름없음"})</td>
            <td>\${app.applyDateStr}</td>
            <td>
                <button type="button" class="btn btn-primary btn-sm" 
                        onclick="confirmAdoption(\${app.adoptionAppId}, \${app.animalNo})">
                    선택
                </button>
            </td>
        </tr>
    `).join('');

// 화면에 삽입
listBody.innerHTML = resultHtml;

```

### 📝 비서의 피드백

**수정 사항**

* **방어 코드 추가**: 데이터가 없을 때 사용자에게 "내역이 없음"을 알리는 로직을 포함하여 완성도를 높였습니다.
* **누락 없는 통합**: `forEach`와의 비교, JSP 충돌 주의사항, 최신 문법(`||` 단축 평가 등)을 모두 하나의 흐름으로 엮었습니다.

**제안 사항**

* 이제 화면에 데이터를 뿌리는(Rendering) 부분은 완벽합니다! 다음 단계로는 이 `list` 데이터를 하드코딩하지 않고, 실제 서버에서 받아오는 **`fetch()`** 통신과 그 응답을 처리하는 비동기 콜백 과정을 정리해 보시는 것을 추천합니다.

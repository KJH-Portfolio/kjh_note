## 💻 자바스크립트 함수 (JavaScript Functions)

> [!summary] **함수**는 특정 작업을 수행하도록 설계된 **재사용 가능한 코드 집합**임. 코드 **중복 방지**, 프로그램 **모듈화**, **가독성 및 유지보수성** 향상이 주요 목적임. ES6+ 문법(화살표 함수, 기본값/나머지 매개변수)으로 더 간결하고 명확한 함수 정의가 가능함.

-----

## 1\. 함수의 정의 및 유형 💡

함수 선언 방식은 **호이스팅(Hoisting)** 및 **$\text{this}$ 바인딩** 방식의 차이를 가짐.

### 1-1. 함수 선언문 (Function Declaration)

| 속성 | 내용 |
| :--- | :--- |
| **정의** | `function` 키워드와 **명시적인 이름**으로 정의. |
| **호이스팅** | **적용됨**. 함수 선언 전체가 스코프 최상단으로 끌어올려지므로, **정의 위치보다 앞에서 호출 가능**함. |
| **형식** | `function 함수명(매개변수) { ... }` |

### 1-2. 함수 표현식 (Function Expression)

| 속성 | 내용 |
| :--- | :--- |
| **정의** | \*\*이름 없는 함수(익명 함수)\*\*를 생성하여 변수에 할당. |
| **호이스팅** | **적용 안 됨**. 변수 선언 규칙을 따르며, **변수에 할당된 이후**부터 호출 가능함. |
| **형식** | `const 변수명 = function(매개변수) { ... };` |

-----

## 2\. 화살표 함수 (Arrow Function) 🚀

ES6에서 도입된, 함수 표현식의 **간결화된 문법**임.

### 2-1. 간결화 문법 (Syntax) 특징

  * **기본 형식**: `(매개변수) => { 함수 본문 }`
  * **매개변수 1개**: 괄호 `()` 생략 가능.
  * **본문 1줄 (return만 존재)**: 중괄호 `{}` 및 `return` 키워드 생략 가능.
  * **객체 리터럴 반환**: 객체 리터럴을 반드시 괄호 `()`로 감싸야 함.

| 상황 | 일반 형식 | 간결화 예시 |
| :--- | :--- | :--- |
| 기본 | `(a, b) => { return a + b; }` | `(a, b) => a + b` |
| 매개변수 1개 | `(p) => p * 2` | `p => p * 2` |
| 객체 반환 | `() => { return { k: 'v' }; }` | `() => ({ k: 'v' })` |

### 2-2. $\text{this}$ 바인딩의 차이 (Lexical $\text{this}$)

화살표 함수는 일반 함수와 달리 자신만의 $\text{this}$를 생성하지 않고, **상위 스코프의 $\text{this}$를 정적으로 상속**받음.

  * **일반 함수**: 호출 방식에 따라 $\text{this}$가 **동적으로** 결정됨.
  * **화살표 함수**: 선언될 때 **정적으로** 결정되며, 항상 자신을 감싸는 **외부(상위) 스코프의 $\text{this}$를 상속**받음.
  * **주요 유용성**: 객체 메서드 내부의 **콜백 함수** 사용 시, 외부 메서드의 $\text{this}$ (객체 자신)를 유지하는 데 유용함.

### 2-3. 제약 사항

  * **생성자 함수 사용 불가**: `new` 키워드로 객체 생성 불가.
  * **`arguments` 객체 미존재**: 전달값을 담는 `arguments` 객체 사용 불가. 대신 **Rest Parameter(`...`)** 사용 필요.

-----

## 3\. 매개변수와 전달값 (Parameters & Arguments)

  * **매개변수 (Parameter)**: 함수를 **정의**할 때 선언하는 변수. (입력 통로)
  * **전달값 (Argument)**: 함수를 **호출**할 때 실제로 전달하는 값.

### 3-1. ES6+ 매개변수 개선 사항

| 문법 | 설명 | 형식 |
| :--- | :--- | :--- |
| **기본값 매개변수** | 전달값이 없을 경우, **기본으로 사용될 값**을 지정. | `function log(message = '기본 메시지') { ... }` |
| **나머지 매개변수** | 지정된 매개변수 외의 **나머지 전달값들을 배열로** 묶어줌. | `function sum(first, second, ...restNums) { ... }` |
| **참고** | 나머지 매개변수는 항상 **마지막** 매개변수로 위치해야 함. 화살표 함수에서 `arguments` 대신 사용됨. | |

-----

## 4\. 함수의 반환 (Return Value)

함수 실행 후 호출한 곳으로 값을 돌려주는 기능.

  * **`return` 키워드**:
      * 함수 내에서 `return`을 만나면 즉시 함수 실행을 **종료**하고, 명시된 값을 반환함.
      * `return`이 생략되거나 `return;`만 사용될 경우, 함수는 \*\*`undefined`\*\*를 반환함.

-----

## 5\. 예시: $\text{this}$ 바인딩 차이

일반 함수와 화살표 함수의 $\text{this}$ 바인딩 차이를 보여주는 예시.

> [!EXAMPLE] **`this` 바인딩 차이 비유**:
> **일반 함수**는 상황에 따라 주인이 바뀌는 \*\*'프리랜서'\*\*와 같음.
> **화살표 함수**는 태어날 때부터 상위 스코프에 고정된 \*\*'정규직 직원'\*\*과 같음.

```javascript
const team = {
    teamName: "Hero Team",
    members: ["아이언맨", "캡틴 아메리카", "토르"],

    // showMembers 메서드의 this는 'team' 객체를 가리킴
    showMembers: function() {
        console.log(`--- ${this.teamName} 멤버 소개 ---`);

        // ❌ 일반 함수 콜백 (this 손실)
        // forEach 내부의 익명 함수는 'this'를 잃어버림 (전역 객체나 undefined)
        console.log(" [일반 함수 (this 손실)] ");
        this.members.forEach(function(member) {
            // this.teamName은 undefined (혹은 전역 객체의 속성)
            console.log(` - ${this.teamName} 소속: ${member}`); 
        });

        // ✅ 화살표 함수 콜백 (this 유지)
        // 화살표 함수는 외부(showMembers) 스코프의 this ('team' 객체)를 상속받음
        console.log(" [화살표 함수 (this 유지)] ");
        this.members.forEach(member => {
            // this.teamName은 'Hero Team'
            console.log(` - ${this.teamName} 소속: ${member}`);
        });
    }
};

team.showMembers();
```

-----

**수정 사항**:

1. **제목 및 콜아웃**: `# (H1)` 제목과 `> [!summary]` 요약 콜아웃을 '구조화 규칙'에 따라 배치했습니다.
2. **개조식 및 간결화**: 모든 문장을 명사 중심의 **개조식**으로 간결하게 다듬었습니다.
3. **표 활용**: '함수 선언문/표현식' 및 '화살표 함수 간결화/매개변수 개선 사항' 내용을 **마크다운 표**로 전환하여 가독성을 높였습니다.
4. **예시**: $\text{this}$ 바인딩 차이 예시 위에 `> [!EXAMPLE]` 콜아웃을 사용하여 개념 비유를 추가했습니다.

**제안 사항**:

  * **호이스팅 심화**: 함수 선언문과 표현식의 호이스팅 동작 차이를 시각적으로 보여주는 다이어그램이나 코드 예시(`let` vs `function`)를 추가하여 이해도를 높일 수 있습니다.
  * **클로저(Closure) 연관 노트 연결**: 함수의 핵심 응용인 **클로저** 관련 노트를 작성 후, 이 노트에 내부 링크(`[[자바스크립트 클로저]]`)를 추가하여 지식 연결성을 강화할 수 있습니다.

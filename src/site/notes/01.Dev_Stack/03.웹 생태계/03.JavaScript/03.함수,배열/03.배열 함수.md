# 📜 JavaScript 배열 함수 (Methods)

> [!summary]
> 
> - **핵심:** 배열 데이터를 조작, 변형하고 정밀하게 탐색하는 도구 모음.
>     
> - **구분 기준:** 원본이 바뀌는지(💥), 바뀌지 않는지(🛡️)를 아는 것이 가장 중요합니다.
>     
> - **활용:** 데이터 처리(Map/Filter)와 조건 확인(Some/Every)은 실무 필수 기능입니다.
>     

---

## 💥 1. 원본 변경 (Destructive Methods)

실행 즉시 **원본 배열 자체가 수정**됩니다. React, Vue 같은 프레임워크에서 상태 관리 시 주의해야 합니다.

| **메서드**              | **역할 (Description)** | **반환값 (Return)**        | **비고**        |
| -------------------- | -------------------- | ----------------------- | ------------- |
| **`push(요소)`**       | 배열 **맨 뒤**에 요소 추가    | **변경된 배열의 길이** (Number) | 스택(Stack) 구조  |
| **`pop()`**          | 배열 **맨 뒤** 요소 제거     | **제거된 요소** (Value)      | 스택(Stack) 구조  |
| **`unshift(요소)`**    | 배열 **맨 앞**에 요소 추가    | **변경된 배열의 길이** (Number) | 큐(Queue) 구조   |
| **`shift()`**        | 배열 **맨 앞** 요소 제거     | **제거된 요소** (Value)      | 큐(Queue) 구조   |
| **`splice(시작, 개수)`** | 중간 요소 **삭제/교체/추가**   | **삭제된 요소들의 배열** (Array) | 만능 수정 도구      |
| **`sort()`**         | 요소를 순서대로 **정렬**      | **정렬된 원본 배열** (Array)   | 기본: 문자열 기준 정렬 |
| **`reverse()`**      | 순서를 **역순**으로 뒤집음     | **뒤집힌 원본 배열** (Array)   |               |

## 🛡️ 2. 원본 유지 (Non-Destructive Methods)

원본은 그대로 두고, **새로운 값**이나 **새로운 배열**을 만들어 반환합니다. (불변성 유지)

|**메서드**|**역할 (Description)**|**반환값 (Return)**|**비고**|
|---|---|---|---|
|**`slice(시작, [끝])`**|배열 일부를 **복사**하여 추출|**새로운 배열** (Array)|원본 보존 (추출)|
|**`concat(배열)`**|다른 배열을 뒤에 **합침**|**합쳐진 새 배열** (Array)|결합|
|**`join(구분자)`**|요소를 연결해 **문자열**로 변환|**문자열** (String)|예: `arr.join(',')`|

---

## ⭐️ 3. 고차 함수 (Higher-Order Functions)

함수를 인자로 받아 데이터를 변형하거나 요약합니다. (원본 유지 🛡️)

| **메서드**          | **역할 (Description)**   | **반환값 (Return)**   | **핵심 로직**     |
| ---------------- | ---------------------- | ------------------ | ------------- |
| **`map(콜백)`**    | 모든 요소를 1:1로 **변환**     | **새로운 배열** (Array) | 길이 동일, 내용 변환  |
| **`filter(콜백)`** | 조건이 `true`인 것만 **필터링** | **새로운 배열** (Array) | 길이 감소 가능      |
| **`reduce(콜백)`** | 요소를 순회하며 값 하나로 **축약**  | **단일 값** (Any)     | 합계, 평균, 객체 변환 |

## 🔍 4. 탐색 및 조건 확인 (Search & Check)

데이터의 **위치**, **존재 여부**, **조건 만족 여부**를 파악합니다. (원본 유지 🛡️)

|**구분**|**메서드**|**역할 (Description)**|**반환값 (Return)**|**실패/없음 시**|
|---|---|---|---|---|
|**값 찾기**|**`find(콜백)`**|조건에 맞는 **첫 요소 값**|**요소 값** (Value)|`undefined`|
||**`findIndex(콜백)`**|조건에 맞는 **첫 요소 인덱스**|**인덱스** (Number)|`-1`|
||**`includes(값)`**|특정 **값 포함 여부** (단순)|**Boolean**|`false`|
|**논리 확인**|**`some(콜백)`**|**하나라도** 조건 만족 시|**Boolean** (OR)|`false`|
||**`every(콜백)`**|**모두** 조건 만족 시|**Boolean** (AND)|`false`|

---

### 💡 텍스트 예시: 탐색 5형제 비유

> [!EXAMPLE] 상황별 메서드 선택 가이드
> 
> - **범인 얼굴(값)을 알고 싶다** 👉 `find()`
>     
> - **범인이 몇 호실(인덱스)에 있는지 알고 싶다** 👉 `findIndex()`
>     
> - **명단에 '홍길동'이 있는지(단순 포함) 확인하고 싶다** 👉 `includes()`
>     
> - **반에 안경 쓴 사람이 한 명이라도 있는지(유무) 확인하고 싶다** 👉 `some()`
>     
> - **반 학생 전원이 교복을 입었는지(검증) 확인하고 싶다** 👉 `every()`
>

## 📚 종합 예시: 쇼핑몰 장바구니 시스템

```js
// 1. 장바구니 데이터 (원본)
const cart = [
  { id: 1, item: '노트북', price: 1500000, stock: 5 },
  { id: 2, item: '마우스', price: 50000, stock: 0 },  // 품절
  { id: 3, item: '키보드', price: 120000, stock: 10 }
];

const wishList = ['노트북', '모니터', '헤드셋']; 

// ------------------------------------------
// [탐색] 데이터 및 위치 찾기
// ------------------------------------------

// 1. (find) 100만원 초과 상품 찾기 -> 객체 반환
const expensiveItem = cart.find(p => p.price > 1000000);

// 2. (findIndex) '마우스' 위치 찾기 -> 인덱스(1) 반환
const mouseIndex = cart.findIndex(p => p.item === '마우스');

// 3. (includes) '모니터' 위시리스트 포함 여부 -> true 반환
const hasMonitor = wishList.includes('모니터');

// ------------------------------------------
// [확인] 재고 및 검증
// ------------------------------------------

// 4. (some) 품절 상품이 있는가? -> true
const hasSoldOut = cart.some(p => p.stock === 0);

// 5. (every) 모두 1만원 이상인가? -> true
const isAllValidPrice = cart.every(p => p.price >= 10000);

// ------------------------------------------
// [활용] 품절 상품 삭제 (findIndex + splice)
// ------------------------------------------

if (hasSoldOut) {
  const targetIndex = cart.findIndex(p => p.stock === 0); 
  if (targetIndex !== -1) {
      cart.splice(targetIndex, 1); // 원본에서 삭제 💥
  }
}

console.log(cart); // 노트북, 키보드만 남음
```

### 💡 비서의 제안

이 표를 통해 배열 기초가 탄탄해지셨다면, 다음 단계로는 이 배열 데이터를 실제 웹페이지 화면에 뿌려주는 **DOM 리스트 렌더링(반복문으로 HTML 생성하기)** 혹은 최신 문법인 **구조 분해 할당(Destructuring)**에 대해 정리해 드리는 것은 어떨까요?

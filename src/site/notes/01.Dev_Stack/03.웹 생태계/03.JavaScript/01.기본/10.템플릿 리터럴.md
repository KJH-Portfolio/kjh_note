# JS 템플릿 리터럴: JSX 활용과 JSP 환경의 주의점

> [!summary] **핵심 요약**
> * **템플릿 리터럴**: 백틱(```)을 사용하여 문자열 내에 변수(`${}`)를 삽입하는 현대 자바스크립트 표준 문법입니다.
> * **JSX 환경**: 별도의 충돌 없이 동적 UI 구현 및 속성(Props) 전달에 핵심적으로 사용됩니다.
> * **JSP 환경**: JSP의 EL(`${}`)과 문법이 중복되어 **역슬래시(`\`)** 처리가 필요하며, 가독성과 유지보수 측면에서 사용을 권장하지 않습니다.
> 
> 

## 1. JSX에서의 템플릿 리터럴 활용

현대적인 프레임워크(React 등)의 JSX 환경에서 템플릿 리터럴은 가장 가독성이 높고 효율적인 문자열 처리 방식입니다.
```js
const name = "김지환";
const str2 = `안녕하세요 ${name}님`;
```
### 주요 특징

* **표현식 삽입**: `${}`를 통해 자바스크립트 변수나 계산식을 직관적으로 삽입합니다.
* **멀티라인 지원**: 별도의 결합 연산자(`+`) 없이도 여러 줄의 HTML 구조를 작성하기 용이합니다.
* **가독성**: 문자열과 변수가 섞인 복잡한 UI 로직을 깔끔하게 표현합니다.

### JSX 활용 패턴 비교

| 구분               | 기존 방식 (ES5 이전)               | 템플릿 리터럴 (JSX 추천)             |
| ---------------- | ---------------------------- | ---------------------------- |
| **문자열 결합**       | `'Hello, ' + name + '!'`     | ``Hello, ${name}!``          |
| **클래스명 동적 할당**   | `className={'btn ' + color}` | `className={`btn ${color}`}` |
| **속성(Props) 전달** | `id={'item-' + id}`          | `id={`item-${id}`}`          |

## 2. JSP 환경에서의 문법 충돌

JSP(Java Server Pages) 환경에서는 서버 사이드 문법인 **EL(Expression Language)**과 템플릿 리터럴이 동일한 기호(`${ }`)를 사용하기 때문에 심각한 충돌이 발생합니다.

### 충돌 메커니즘

1. **서버의 선점**: 브라우저가 자바스크립트를 실행하기 전, 서버(Tomcat 등)가 JSP 내의 모든 `${ }`를 자신의 데이터로 착각하여 먼저 처리합니다.
2. **값의 소멸**: 서버에 존재하지 않는 JS 변수라면 서버는 해당 자리를 빈 값으로 지워버린 채 브라우저로 보냅니다.

## 3. JSP에서의 이스케이프(Escape)와 비추천 이유

JSP 내 script 태그에서 템플릿 리터럴을 억지로 사용하려면 **역슬래시(`\`)**를 사용해야 합니다.

### 이스케이프 방법

```javascript
// JSP 내부 스크립트 작성 시
const row = `<td>\${item.name}</td>`; 
// $ 앞에 \를 붙여 "이건 서버용이 아니야"라고 명시함

```

### ⚠️ JSP 내 사용을 권장하지 않는 이유 (비추천)

* **가독성 저하**: 모든 변수 앞에 `\`를 붙여야 하므로 템플릿 리터럴 본연의 깔끔함이 사라집니다.
* **유지보수 위험**: 개발자가 실수로 `\`를 하나라도 누락할 경우, 런타임 에러를 찾기가 매우 어렵습니다.
* **설계의 혼선**: 서버 사이드 렌더링(JSP)과 클라이언트 사이드 스크립트가 한 곳에 뒤섞여 있음을 의미하며, 이는 현대적인 관심사 분리(SoC) 원칙에 어긋납니다.

> [!tip] **JSP 환경에서의 대안**
> JSP를 사용 중이라면 템플릿 리터럴을 억지로 쓰기보다, **JS 코드를 별도의 외부 파일(.js)로 완전히 분리**하여 호출하는 것이 가장 안전하고 올바른 방법입니다.

## 4. 실무 코드 예시 (JSX vs JSP)

> [!EXAMPLE] **JSX에서의 깔끔한 사용 (권장)**

```jsx
function WelcomeCard({ user }) {
  return (
    <div className={`card-container ${user.grade}`}>
      <h1>{`환영합니다, ${user.name}님!`}</h1>
      <p>{`적립금: ${user.point.toLocaleString()}원`}</p>
    </div>
  );
}

```

> [!EXAMPLE] **JSP에서의 지양해야 할 사용 (비추천)**

```javascript
/* JSP 파일 내부 script 태그 */
let html = "";
data.forEach(item => {
    // 매번 \를 붙여야 하며, 누락 시 데이터가 깨짐
    html += `<li class="list-item">\${item.title}</li>`;
});

```

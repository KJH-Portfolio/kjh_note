# JS 클래스 (Class)

> [!summary] **핵심 요약**
> * **개념**: 프로토타입 기반의 상속을 더 쉽고 명확하게 사용할 수 있도록 도입된 현대적 문법(ES6)입니다.
> * **특징**: `class` 키워드를 사용하며, 생성자(`constructor`)와 메서드를 하나의 블록 안에 묶어서 관리합니다.
> * **장점**: 코드의 가독성이 높아지고, 상속(`extends`) 구조를 한눈에 파악하기 쉽습니다.
> 
> 

## 1. 클래스의 기본 구조

클래스는 크게 **생성자(Constructor)**와 **메서드(Method)**로 구성됩니다.

* **`class` 키워드**: 클래스를 선언합니다.
* **`constructor()`**: 객체를 만들 때 초기값(속성)을 설정하는 특수 함수입니다. (인스턴스 생성 시 자동 실행)
* **메서드**: 함수 키워드(`function`) 없이 이름만 써서 정의하며, 자동으로 프로토타입에 등록됩니다.

---

## 2. 프로토타입 vs 클래스 문법 비교

두 방식은 결과적으로 동일하게 작동하지만, 작성 방식에서 큰 차이가 있습니다.

| 구분 | 프로토타입 방식 (기존) | 클래스 방식 (현대) |
| --- | --- | --- |
| **정의 위치** | 생성자 함수와 메서드 정의가 분리됨 | `class` 블록 안에 모두 모여 있음 |
| **메서드 등록** | `함수명.prototype.메서드`로 개별 등록 | 클래스 내부에 작성 시 자동 등록 |
| **상속 구현** | `Object.create()` 등 복잡한 절차 필요 | `extends` 키워드로 간결하게 구현 |
| **가독성** | 구조가 파편화되어 파악이 어려움 | 객체지향 구조가 명확히 보임 |

---

## 3. 핵심 기능: 상속 (Inheritance)

클래스의 가장 큰 장점은 **`extends`** 키워드를 통한 상속입니다. 부모 클래스의 기능을 자식 클래스가 그대로 물려받을 수 있습니다.

* **`extends`**: 부모 클래스를 상속받습니다.
* **`super()`**: 부모 클래스의 생성자를 호출할 때 사용합니다. (자식 생성자 내에서 반드시 먼저 호출해야 함)

---

## 4. 실무 적용 예시

> [!EXAMPLE] **클래스를 이용한 객체 생성 및 상속**

```javascript
// 1. 부모 클래스 정의
class Animal {
    constructor(name) {
        this.name = name;
    }

    eat() {
        console.log(`${this.name}이(가) 음식을 먹습니다.`);
    }
}

// 2. 자식 클래스 상속
class Dog extends Animal {
    constructor(name, breed) {
        super(name); // 부모의 constructor 호출 (name 설정)
        this.breed = breed;
    }

    bark() {
        console.log(`${this.name}(${this.breed}): 멍멍!`);
    }
}

const myDog = new Dog("초코", "푸들");
myDog.eat();  // 부모로부터 물려받은 메서드
myDog.bark(); // 자신의 메서드

```

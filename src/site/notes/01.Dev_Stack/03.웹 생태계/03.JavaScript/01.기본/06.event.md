# 웹 이벤트 처리 : 모델, 객체, currentTarget 제어

> [!summary] 핵심 요약
> * **이벤트(Event):** 웹 페이지 내 발생하는 모든 상호작용(클릭, 키 입력, 스크롤 등).
> * **이벤트 객체(e):** 발생한 사건의 상세 속성(좌표, 타입, 대상 등)을 보유한 객체.
> * **e.target:** 실제 이벤트가 발생한 소스 요소(클릭의 시발점).
> * **e.currentTarget:** 이벤트 핸들러가 부착된 요소(이벤트 수신기 위치).
> * **화살표 함수 주의점:** 화살표 함수 내 `this`는 상위 스코프를 참조하므로, 요소 접근 시 `e.currentTarget` 사용 필수.
> 
>

## 📌 1. 이벤트 등록 모델 (Event Models)

### 1.1. 고전 이벤트 모델 (Classical / DOM Level 0)

* **정의:** 요소의 이벤트 속성에 직접 함수 대입.
* **특징:**
* `on` + `이벤트명` 형식의 속성 사용.
* **중복 불가:** 동일 속성에 다수 함수 대입 시 마지막 함수만 유효(덮어쓰기).
* **제거:** 속성에 `null` 대입 처리.


* **코드 예시:**

```js
const btn = document.querySelector("#btn1");

// 등록
btn.onclick = () => {
    alert("최초 핸들러 실행");
};

// 교체 (이전 내용 삭제)
btn.onclick = () => {
    alert("최종 핸들러로 덮어쓰기 완료");
};

```

### 1.2. 인라인 이벤트 모델 (Inline)

* **정의:** HTML 태그의 속성 값으로 자바스크립트 코드 직접 기입.
* **단점:** 구조(HTML)와 로직(JS)의 결합으로 인한 가독성 저하 및 관리 난해.
* **코드 예시:**

```html
<button onclick="(() => console.log('인라인 방식 실행'))()">클릭</button>

```

### 1.3. 표준 이벤트 모델 (Standard / DOM Level 2) ★ 권장

* **정의:** `addEventListener` 메서드를 통한 핸들러 등록.
* **특징:**
* **다중 등록:** 동일 요소 및 이벤트에 여러 핸들러 추가 가능.
* **정밀 제어:** 캡처링, 버블링 옵션 설정 가능.


* **코드 예시:**

```js
const btn = document.querySelector("#btn2");

btn.addEventListener("click", () => {
    console.log("첫 번째 실행");
});

btn.addEventListener("click", () => {
    console.log("두 번째 동시 실행");
});

```

## 📌 2. 이벤트 객체 상세 (Event Properties)

### 2.1. 주요 속성 및 e.currentTarget

* **e.target:** 이벤트가 실제로 시작된 가장 안쪽의 요소.
* **e.currentTarget:** 이벤트 리스너가 직접 호출된 요소. (현재 실행 중인 핸들러가 소속된 지점).
* **e.type:** 발생한 이벤트의 종류(문자열).
* **e.preventDefault():** 브라우저의 기본 동작(링크 이동, 폼 제출 등) 차단 메서드.

### 2.2. 모델별 식별자 참조 비교

| 구분                  | 일반 함수 (`function`)            | 화살표 함수 (`() =>`)       |
| ------------------- | ----------------------------- | ---------------------- |
| **this**            | `e.currentTarget`과 동일 (요소 자신) | Window 혹은 상위 스코프 객체    |
| **e.target**        | 실제 클릭된 원인 요소                  | 실제 클릭된 원인 요소           |
| **e.currentTarget** | 핸들러 등록 요소                     | **핸들러 등록 요소 (안전한 참조)** |

---

## 📌 3. 기본 동작 제어 (Prevent Default)

* **사용 목적:** 유효성 검사 실패 시 전송 중단 혹은 특정 상황에서의 이동 제한.
* **구현 방식:** 이벤트 객체의 `preventDefault()` 메서드 호출.
* **코드 예시:**

```js
const link = document.querySelector("a");

link.addEventListener("click", (e) => {
    e.preventDefault(); // 페이지 이동 차단
    console.log("이동 대신 로그 출력 수행");
});

```

## 📝 4. 종합 예시 코드

> [!EXAMPLE] e.currentTarget과 화살표 함수 활용

```html
<div id="parent" style="padding: 20px; background: lightgray;">
    부모 요소 (div)
    <button id="child">자식 버튼 (button)</button>
</div>

<script>
    const parent = document.querySelector("#parent");

    // 부모 요소에 이벤트 등록 (이벤트 전파 활용)
    parent.addEventListener("click", (e) => {
        console.log("--- 클릭 발생 ---");
        
        // 버튼 클릭 시: <button> 출력
        console.log("실제 발생 지점 (e.target):", e.target);
        
        // 버튼 클릭 시에도: <div> 출력 (핸들러가 div에 있으므로)
        console.log("핸들러 위치 (e.currentTarget):", e.currentTarget);
        
        // 화살표 함수이므로 this는 div가 아님을 확인
        console.log("this 확인:", this); // Window 객체 출력
    });
</script>

```

# JS 현대적 논리 및 안전 접근 연산자 (`&&`, `||`, `?.`, `??`) 종합 정리

> [!summary]
> 단락 평가 및 Nullish 체크를 통한 **안전한 데이터 접근**, 리액트 JSX 내 **조건부 렌더링 최적화**, 그리고 연산자별 **부작용(Side Effect) 방지**를 위한 통합 가이드.

---

## 1. 논리 연산자 및 단락 평가 (Short-circuit)

불리언 판별을 넘어 평가 완료 시점의 **실제 값**을 반환하는 특성 활용.

* **`&&` (논리곱)**: 좌항이 **참(Truthy)**이면 우항 반환, 좌항이 **거짓(Falsy)**이면 좌항 즉시 반환.
* **`||` (논리합)**: 좌항이 **참(Truthy)**이면 좌항 즉시 반환, 좌항이 **거짓(Falsy)**이면 우항 반환.
* **주요 용도**: 조건부 로직 실행 및 단순 기본값 할당.
- $ `!!`를 사용하여 강제로 boolean으로 만드는 방법 사용. (더블 뱅,Double Bang)
---

## 2. 안전 접근 및 Nullish 제어 (`?.`, `??`)

### 2.1 옵셔널 체이닝 (Optional Chaining) `?.`

* **정의**: `null` 또는 `undefined` 참조 시 에러 대신 `undefined`를 반환하는 안전 장치.
* **활용**: 객체 속성 접근, 배열 인덱스 참조, 함수 호출(`fn?.()`) 시 사용.
* **제약**: 읽기 전용 연산자로, 할당문 좌항 사용 불가.

### 2.2 Nullish 병합 연산자 (Nullish Coalescing) `??`

* **정의**: 오직 **`null`**과 **`undefined`**인 경우에만 우항의 기본값 적용.
* **차별점**: 숫자 `0`, 빈 문자열(`""`), `false` 등 유효한 Falsy 데이터 보존.

---

## 3. 연산자별 주요 부작용 및 예외 처리

### 3.1 JSX 내 숫자 0 누출 현상 (`&&`)

* **부작용**: `count && <Component />` 사용 시 `count`가 0이면 화면에 **숫자 0**이 렌더링됨.
* **해결책**: `!!count` (불리언 변환) 또는 `count > 0` (명시적 비교) 사용.

### 3.2 유효 데이터 유실 (`||`)

* **부작용**: `0`이나 `""`을 유효한 입력으로 취급해야 할 때 `||`가 이를 거짓으로 판단하여 기본값으로 덮어씀.
* **해결책**: 엄격한 Nullish 체크가 가능한 `??` 연산자로 대체.

### 3.3 디버깅 난해함 (`?.`)

* **부작용**: 런타임 에러는 방지하나, 데이터 구조 오류나 API 실패를 `undefined`로 은폐하여 원인 파악 지연.
* **해결책**: 데이터 존재가 필수적인 핵심 로직에는 명시적 에러 핸들링 병행.

---

## 4. 연산자 비교 요약 테이블

| 연산자      | 우항 반환 조건 (체크 대상)                             | 주요 특징                 |
| -------- | -------------------------------------------- | --------------------- |
| **`&&`** | 좌항이 Truthy일 때                                | 조건부 렌더링, 연쇄 실행        |
| \|\|     | Nullish,0,-0,NaN,"",false                    | 단락평가,포괄적 기본값 설정       |
| **`??`** | 좌항이 Nullish일 때 (**`null`**, **`undefined`**) | 유효한 Falsy 값(`0` 등) 보존 |
| **`?.`** | 좌항이 Nullish일 때 평가 중단                         | 참조 에러(TypeError) 방지   |

---

## 5. 실전 코드 예시

> [!EXAMPLE] 부작용을 방지한 안전한 코드 패턴

```jsx
function UserProfile({ data, count = 0 }) {
  // 1. ?.와 ??의 조합: 깊은 참조 안전 확보 및 기본값 지정
  const nickName = data?.user?.profile?.name ?? "Guest";

  // 2. && 부작용(0 출력) 방지: 명시적 불리언 처리
  const hasItems = !!data?.items?.length;

  return (
    <div>
      <h1>{nickName}</h1>
      
      {/* 잘못된 예: {count && <Badge />} -> count가 0일 때 0 출력 */}
      {/* 올바른 예: 명시적 비교 */}
      {count > 0 && <Badge color="red" />}
      
      {/* 안전한 함수 호출 */}
      <button onClick={() => data?.actions?.update?.()}>수정</button>
    </div>
  );
}

```

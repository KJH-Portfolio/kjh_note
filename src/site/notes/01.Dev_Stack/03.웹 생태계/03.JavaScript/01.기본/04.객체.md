# 📜 JavaScript 객체 (Object) 완벽 정리

> [!summary] 핵심 요약
> 
> - **객체(Object):** `{}` (중괄호)를 사용해 '키(Key): 값(Value)' 쌍으로 데이터를 관리하는 자료형.
>     
> - **접근 및 순회:** 점/대괄호 표기법으로 접근하고, `for...in` 문으로 모든 키를 순회.
>     
> - **메서드(Method):** 객체 내부에 정의된 함수. `this`는 메서드를 호출한 객체 자신을 가리킴.
>     
> - **생성자 함수:** 붕어빵 틀처럼 동일한 구조의 객체를 여러 개 생성할 때 사용하는 함수 (`new` 키워드 필수).
>     

---

## 1. 객체 선언 및 속성 접근

- **객체(Object):** 연관된 데이터(속성)와 기능(메서드)을 하나로 묶어 관리하는 자료형.
    
- **선언(리터럴 방식):** `{}` (중괄호) 사용. 가장 일반적인 생성 방식.
    
- **구조:** `'키(Key)' : '값(Value)'` 쌍으로 구성.
    
    - **키(Key):** 속성명 (문자열).
        
    - **값(Value):** 모든 자료형 가능 (문자, 숫자, 배열, 함수 등).
        

### 속성 접근 방법

- **점 표기법 (Dot Notation):** `객체명.속성명`
    
    - 속성명이 유효한 식별자(특수문자 X, 공백 X)일 때 사용.
        
- **대괄호 표기법 (Bracket Notation):** `객체명['속성명']`
    
    - 속성명에 **공백이나 특수문자**가 있거나, **변수**로 속성에 접근할 때 필수.
        

---

## 2. 객체 순회 (Iteration)

- **`for...in` 반복문:** 객체가 가진 모든 속성(키)들을 순차적으로 순회.
    
- **표현식:** `for (let key in 객체명) { ... }`
    
- **주의사항:** `key` 변수에는 속성명이 **문자열**로 들어오므로, 반드시 `객체명[key]`로 접근해야 함. (점 표기법 `객체명.key` 사용 시 `undefined` 반환)
    

---

## 3. 메서드(Method)와 'this'

- **메서드:** 객체의 속성 값으로 할당된 **함수**.
    
- **`this` 키워드:**
    
    - 메서드 내부에서 **'나 자신'(현재 객체)**을 가리키는 참조 변수.
        
    - 예: `user.sayHi()` 호출 시 `this`는 `user` 객체.
        

---

## 4. 객체 속성 관리

- **추가/수정:** `student.age = 20` (없으면 추가, 있으면 수정).
    
- **제거:** `delete student.age`.
    
- **확인:** `"age" in student` (`true`/`false` 반환).
    

---

## 5. 생성자 함수 (Constructor Function)

### 개념 및 필요성

- **리터럴(`{}`) 방식의 한계:** 똑같은 구조의 객체(예: 학생 A, 학생 B...)를 여러 개 만들 때, 매번 코드를 복사해야 하는 불편함.
    
- **생성자 함수:** 객체를 생성하기 위한 **설계도(틀)** 역할을 하는 함수.
    

### 특징 및 문법

1. **함수명:** 관례적으로 **대문자로 시작** (PascalCase)하여 일반 함수와 구분.
    
2. **`new` 키워드:** 호출 시 반드시 **`new` 연산자**를 붙여서 실행.
    
3. **`this`의 역할:** 생성자 함수 내부의 `this`는 **'앞으로 생성될 인스턴스(새 객체)'**를 가리킴.
    


```js
// 생성자 함수 정의
function Student(name, score) {
    // this = {} (빈 객체가 암묵적으로 생성됨)
    
    this.name = name;   // 속성 추가
    this.score = score;
    
    this.getScore = function() { // 메서드 추가
        return this.score + "점";
    };
    
    // return this; (this가 암묵적으로 반환됨)
}

// 사용 (인스턴스 생성)
const student1 = new Student("철수", 90);
const student2 = new Student("영희", 85);
```

## 📝 종합 예시

> [!EXAMPLE] 객체 리터럴 vs 생성자 함수
> 
> 아래 코드는 한 명의 정보를 다룰 때(리터럴)와, 여러 명의 정보를 다룰 때(생성자)의 차이를 보여줍니다.

```js
// 1. 객체 리터럴 방식 (단일 객체 생성 시 편리)
let person1 = {
    name: "비서",
    job: "AI",
    introduce: function() {
        console.log("저는 " + this.name + "입니다.");
    }
};

person1.introduce(); // "저는 비서입니다."


// 2. 생성자 함수 방식 (유사한 객체 대량 생성 시 편리)
function Person(name, job) {
    this.name = name;
    this.job = job;
    this.introduce = function() {
        console.log("제 직업은 " + this.job + "이고, 이름은 " + this.name + "입니다.");
    };
}

// new 키워드로 인스턴스(실제 객체) 생성
const p1 = new Person("홍길동", "개발자");
const p2 = new Person("김철수", "디자이너");

// 각각 독립적인 객체로 동작
console.log(p1.name); // "홍길동"
p2.introduce();       // "제 직업은 디자이너이고, 이름은 김철수입니다."


// 3. for...in으로 인스턴스 순회해보기
console.log("--- p1의 정보 ---");
for(let key in p1) {
    // 값에 함수가 들어있다면 실행, 아니면 출력하는 응용
    if(typeof p1[key] === 'function'){
        console.log(key + ": [함수입니다]");
    } else {
        console.log(key + ": " + p1[key]);
    }
}
```

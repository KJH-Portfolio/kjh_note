# 자바스크립트 프로토타입 (Prototype)

> [!summary] **핵심 요약**
> * **개념**: 생성자 함수로 만든 모든 객체(인스턴스)가 공통으로 사용하는 **'공유 창고'**입니다.
> * **목적**: 동일한 메서드를 중복 생성하지 않음으로써 **메모리 낭비를 방지**하고 성능을 최적화합니다.
> * **핵심 원리**: 객체는 자신의 `__proto__`를 통해 부모(생성자)의 `prototype` 공간에 접근하여 기능을 빌려 씁니다.
> 
> 

## 1. 등장 배경: 메모리 효율성 문제

일반적인 생성자 함수 내부에서 메서드를 정의하면 객체를 생성할 때마다 새로운 함수 메모리가 할당됩니다.

* **문제점**: 인스턴스를 1,000개 생성하면, 동일한 기능을 하는 함수도 1,000개가 생성됨.
* **비효율**: 불필요한 메모리 점유로 인해 대규모 데이터 처리 시 성능 저하 발생.

---

## 2. 프로토타입 핵심 요소 비교

프로토타입 개념에서 가장 혼동하기 쉬운 두 속성을 표로 정리했습니다.

| 구분        | `prototype` 속성           | `__proto__` 속성            |
| --------- | ------------------------ | ------------------------- |
| **보유 주체** | **생성자 함수** (Constructor) | **인스턴스 객체** (Instance)    |
| **역할**    | 자식들에게 물려줄 **유전자 명부**     | 자신이 물려받은 **유전자 확인용 링크**   |
| **비유**    | 아파트 단지의 **공용 커뮤니티 센터**   | 각 세대에서 커뮤니티로 가는 **연결 통로** |

---

## 3. 프로토타입 사용 문법

### ① 메서드 정의 (공유 공간에 저장)

메서드를 생성자 함수 내부(`this`)가 아닌, **외부의 prototype**에 정의합니다.

```javascript
function Dog(name) {
    this.name = name; // 이름은 객체마다 다르므로 개별 저장
}

// 공통 기능은 프로토타입에 정의 (단 한 번만 생성됨)
Dog.prototype.bark = function() {
    console.log(`${this.name}: 멍멍!`);
};

```

### ② 인스턴스에서의 접근

인스턴스는 `bark()` 메서드를 직접 가지고 있지 않지만, 프로토타입 체인을 통해 자동으로 찾아가 실행합니다.

```javascript
const maldon = new Dog("말동이");
maldon.bark(); // "말동이: 멍멍!" 출력

```

## 4. 실무 예시 및 검증

> [!EXAMPLE] **메모리 절약 및 공유 확인**
> 동일한 생성자로 만든 두 객체가 같은 메서드 주소를 가리키는지 확인하는 예제입니다.

```javascript
function Student(name, score) {
    this.name = name;
    this.score = score;
}

// 프로토타입 메서드 정의
Student.prototype.sayHello = function() {
    console.log(`반갑습니다, ${this.name}입니다.`);
};

const s1 = new Student("철수", 90);
const s2 = new Student("영희", 80);

// [검증] 두 객체의 메서드가 동일한 메모리 주소를 공유하는가?
console.log(s1.sayHello === s2.sayHello); // true (효율적!)

// [관계 확인]
console.log(s1.__proto__ === Student.prototype); // true

```

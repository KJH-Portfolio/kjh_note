# 📘HTML 입력 양식 (Form)태그

사용자로부터 정보를 입력받기 위해 사용되는 여러 태그와 속성들

## 1. `<form>` 태그📝

사용자가 입력한 데이터를 서버로 전송하기 위해 사용하는 전체 영역

#### **주요 속성 (Attributes)**

| **속성**       | **설명 / 기능**               |
| ------------ | ------------------------- |
| **`action`** | 폼 데이터를 전송할 서버의 URL (경로)   |
| **`method`** | 데이터를 전송할 방식 (HTTP Method) |

#### **참고 사항**

- **`method` 방식**
    
    - **`get`** (기본값): 사용자가 입력한 값이 URL에 **노출**됩니다. (예: `test.do?keyword=검색어`)
        
    - **`post`**: 사용자가 입력한 값이 URL에 **노출되지 않고** 본문(body)에 담겨 전송됩니다. (로그인, 회원가입 등)
        
- 폼 내부의 입력 요소들은 **`name`** 속성을 가져야 서버로 `key=value` 형태로 값이 전달됩니다.
    

---

## 2. `<fieldset>` / `<legend>` 태그📝

폼 내부의 관련 요소들을 그룹화할 때 사용합니다.

| **태그**           | **설명 / 기능**                   |
| ---------------- | ----------------------------- |
| **`<fieldset>`** | 입력 요소 그룹을 묶어 **테두리**를 그려줍니다.  |
| **`<legend>`**   | `fieldset` 그룹의 **제목**을 붙여줍니다. |

---

## 3. `<input>` 태그 (공통)📝

사용자에게 값을 입력받는 가장 핵심적인 태그입니다. `type` 속성에 따라 기능과 모양이 완전히 달라집니다.

---

## 4. 텍스트 관련 `<input>` 및 태그📝

한 줄 또는 여러 줄의 텍스트를 입력받습니다.

| **태그 / 타입**           | **설명 / 기능**                           |
| --------------------- | ------------------------------------- |
| **`type="text"`**     | 한 줄 텍스트 입력 상자 (기본값)                   |
| **`type="password"`** | 비밀번호 입력 상자 (입력값이 마스킹 처리됨)             |
| **`type="search"`**   | 검색어 입력 상자 (브라우저에 따라 X 버튼이 표시될 수 있음)   |
| **`type="url"`**      | URL 형식만 입력받음 (제출 시 형식 검사)             |
| **`type="email"`**    | 이메일 형식만 입력받음 (제출 시 형식 검사)             |


#### **주요 속성 (Attributes)**

| **속성**                    | **적용 태그**     | **설명 / 기능**                         |
| ------------------------- | ------------- | ----------------------------------- |
| **`name`**                | (모두)          | 서버로 전송될 데이터의 '키(Key)' 값             |
| **`required`**            | (모두)          | 이 항목을 필수 입력으로 지정                    |
| **`id`**                  | (모두)          | 태그를 식별하는 고유 ID (`<label>`과 연결 시 필수) |
| `disable`                 | (모두)          | 입력요소 비활성화                           |
| `read only`               | (모두)          | 입력요소 읽기 전용                          |
| **`placeholder`**         | (text, pwd 등) | 사용자가 입력하기 전 표시되는 안내 문구 (희미한 글씨)     |


---

## 5. `<label>` 태그📝

| **속성**    | **설명 / 기능**                  |
| --------- | ---------------------------- |
| **`for`** | 연결할 `input` 태그의 `id` 속성값을 입력 |

#### **참고 사항**

- `<label for="userId">아이디 : </label> <input type="text" id="userId">`
    
- 이렇게 연결하면 '아이디' 텍스트를 클릭해도 해당 `input` 창이 활성화됩니다.
    

---

## 6. 숫자 관련 `<input>` 태그📝


|**타입**|**설명 / 기능**|**관련 속성**|
|---|---|---|
|**`type="number"`**|숫자만 입력받는 상자 (위/아래 스핀 버튼 표시)|`min` (최소값), `max` (최대값), `step` (증가/감소 단위)|
|**`type="range"`**|슬라이드 바를 이용해 숫자 선택|`min` (최소값), `max` (최대값)|

# 7. 선택 관련 태그 (라디오, 체크박스)📝

| **타입**                | **설명 / 기능**                 | **주요 속성**                                                                                            |
| --------------------- | --------------------------- | ---------------------------------------------------------------------------------------------------- |
| **`type="radio"`**    | **여러 개 중 하나만** 선택 (예: 성별)   | **`name`**: 이름이 같아야 하나의 그룹으로 묶임<br><br>**`value`**: 서버로 전송될 실제 값<br><br>**`checked`**: 기본 선택 항목으로 지정 |
| **`type="checkbox"`** | **여러 개 중 다중** 선택 가능 (예: 취미) | **`name`**: 그룹화 (보통 동일 이름 사용)<br><br>**`value`**: 서버로 전송될 실제 값<br><br>**`checked`**: 기본 선택 항목으로 지정   |

---

# 8. 선택 관련 태그 (리스트)📝

| **태그**           | **설명 / 기능**                      | **주요 속성**                                                      |
| ---------------- | -------------------------------- | -------------------------------------------------------------- |
| **`<select>`**   | 드롭다운 목록 (콤보 박스)의 전체 영역           | **`name`**: 서버로 전송될 키(Key)                                     |
| **`<option>`**   | `<select>` 내부의 각 선택 항목           | **`value`**: 서버로 전송될 실제 값<br><br>**`selected`**: 기본 선택 항목으로 지정 |
| **`<datalist>`** | `input type="text"`의 자동 완성 목록 제공 | **`id`**: 이 리스트의 고유 ID                                         |

**참고 사항 (`datalist`)**

- `<input type="text" list="colorList">` 처럼 `input`의 `list` 속성과 `<datalist>`의 `id`를 연결합니다.
    
- `<select>`와 달리 목록에 없는 값도 **직접 입력이 가능**합니다.

# 9. 날짜/시간 관련 `<input>` 태그📝

| **타입**                      | **설명 / 기능**        |
| --------------------------- | ------------------ |
| **`type="date"`**           | 연/월/일 선택 (달력 표시)   |
| **`type="month"`**          | 연/월 선택             |
| **`type="week"`**           | 연/주 선택             |
| **`type="time"`**           | 시간(시/분) 선택         |
| **`type="datetime-local"`** | 날짜와 시간을 함께 선택 (통합) |

# 10. 기타 `<input>` 태그📝


| **타입**              | **설명 / 기능**                          |
| ------------------- | ------------------------------------ |
| **`type="color"`**  | 색상표(Color Picker)를 열어 색상 선택          |
| **`type="file"`**   | '파일 선택' 버튼을 생성하여 파일 첨부               |
| **`type="hidden"`** | 화면에 보이지 않지만 서버로 값을 전송 (개발자가 미리 값 설정) |
| **`type="submit"`** | **폼(form)을 제출(전송)**하는 버튼             |
| **`type="reset"`**  | 폼(form) 내부에 작성된 모든 값을 **초기화**하는 버튼   |
| **`type="button"`** | 아무 기능이 없는 순수 버튼 (주로 JavaScript와 연동)  |

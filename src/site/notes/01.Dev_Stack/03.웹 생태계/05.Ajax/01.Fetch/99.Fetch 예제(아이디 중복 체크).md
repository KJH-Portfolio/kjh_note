## AJAX 실습: 아이디 중복 체크

> [!summary] 시나리오 요약
> 사용자 입력을 감지(`blur` 이벤트)하여 아이디가 **5글자 이상**일 때 서버로 중복 여부를 확인합니다.
>   * **Server**: DB 조회 후 사용 가능 여부(`NNNNY`/`NNNNN`) 반환.
>   * **Client**: `Async/Await`를 사용하여 비동기 요청을 동기 코드처럼 직관적으로 처리.

# 1. 공통 사항 (Controller)

클라이언트의 요청(`idcheck.me`)을 받아 DB에서 중복 여부를 확인하고 결과를 반환합니다.
### 코드 명세 (Javadoc)
  * **기능**: 아이디 중복 체크
  * **파라미터**: `inputId` (클라이언트 전송 키값과 일치 필수)
  * **반환값**:
      * `NNNNY`: 사용 가능 (중복 없음)
      * `NNNNN`: 사용 불가 (중복 존재)
### 구현 코드 (Java)

```java
@ResponseBody
@RequestMapping(value = "/idcheck.me", produces = "text/html; charset=UTF-8")
public String idCheck(String inputId) {

	// 1. DB 조회: 해당 아이디가 존재하는지 갯수 확인
	int count = memberService.idCheck(inputId);
	
	// 2. 결과 반환
	if (count > 0) {
		return "NNNNN"; // 중복 (사용 불가)
	} else {
		return "NNNNY"; // 중복 없음 (사용 가능)
	}
}
```
- produces: 응답 데이터의 인코딩 설정 (한글 깨짐 방지)
# 2\. JS 방식 (Vanilla JS - Async/Await)

### 주요 특징
  * **`async`**: 비동기 로직이 포함된 함수임을 선언.
  * **`await`**: 비동기 응답이 올 때까지 코드 실행을 잠시 대기 (순차 처리).
  * **`try-catch`**: 네트워크 오류 등의 예외를 직관적으로 처리.
### 구현 코드

```javascript
document.addEventListener("DOMContentLoaded", function() {
    // DOM 요소 선택
    const inputId = document.querySelector("#inputId");
    const resultDiv = document.querySelector("#resultDiv");
    const submitBtn = document.querySelector(".btns button[type='submit']");

    // 'blur' 이벤트 리스너에 async 키워드 추가
    inputId.addEventListener("blur", async function() {
        const userId = inputId.value;

        // 1. 유효성 검사 (길이 확인)
        if (userId.length > 4) {
            try {
                // 2. 서버 요청 (await 사용)
                const response = await fetch('idcheck.me', {
                    method: 'POST',
                    headers: { 
                        'Content-Type': 'application/x-www-form-urlencoded' 
                    },
                    body: `inputId=${userId}` // 쿼리스트링 형식 전송
                });

                // 3. 응답 텍스트 변환 (await 사용)
                const result = await response.text();

                // 4. 결과 처리 (동기 코드처럼 작성)
                const isAvailable = result === "NNNNY";
                
                resultDiv.textContent = isAvailable ? "사용 가능한 아이디 입니다." : "사용 불가능한 아이디입니다.";
                resultDiv.style.color = isAvailable ? "green" : "red";
                resultDiv.style.display = "block";
                submitBtn.disabled = !isAvailable; // 사용 가능할 때만 버튼 활성화

            } catch (error) {
                console.error("통신 실패:", error);
            }
        } else {
            // 유효성 탈락 시
            resultDiv.textContent = "아이디는 5글자 이상 입력하세요";
            resultDiv.style.color = "red";
            resultDiv.style.display = "block";
            submitBtn.disabled = true;
        }
    });
});
```

# 3. jQuery 방식

라이브러리 특성상 `success` 콜백을 사용하는 것이 표준입니다. (`$.ajax`는 기본적으로 Promise를 반환하지만, 설정 객체 방식이 더 직관적임)
### 주요 특징

  * **간결함**: 선택자(`$`)와 메소드 체이닝(`.css().show()`)으로 코드 양 최소화.
  * **자동 변환**: `data` 객체를 쿼리 스트링으로 자동 변환하여 전송.
### 구현 코드

```javascript
$("#inputId").blur(function(){
    // this 키워드로 이벤트 발생 요소 접근
    const length = $(this).val().length;

    if(length > 4){
        $.ajax({
            url : "idcheck.me",
            type : "post", // post 방식 명시
            data : { 
                inputId : $(this).val() // Controller 파라미터명과 일치
            },
            success : function(result){
                if(result === "NNNNY"){
                    $("#resultDiv").html("사용 가능한 아이디 입니다.")
                                 .css("color","green").show();
                    $(".btns button[type=submit]").prop("disabled", false);
                } else {
                    $("#resultDiv").text("사용 불가능한 아이디입니다.")
                                 .css("color","red").show();
                    $(".btns button[type=submit]").prop("disabled", true);
                }
            },
            error : function(){
                console.log("통신 실패");
            }
        });
    } else {
        $("#resultDiv").show().text("아이디는 5글자 이상 입력하세요").css("color","red");
        $(".btns button[type=submit]").prop("disabled", true);
    }
});
```

## 📄 기타 참고자료: HTTP `Content-Type` 종류 및 바디 형식

|**Content-Type**|**설명**|**용도**|
|---|---|---|
|**`application/json`**|**JSON** (JavaScript Object Notation) 형식의 데이터.|모바일 앱, 웹 API 서버 통신 시 **가장 널리 사용**됩니다. 데이터를 가볍고 구조화하기 쉽습니다.|
|**`application/x-www-form-urlencoded`**|**HTML 폼**의 기본 전송 방식.|간단한 폼 전송에 사용되며, 데이터가 URL 쿼리 문자열처럼 인코딩됩니다.|
|**`multipart/form-data`**|**여러 개의 파트**로 구성된 데이터.|**파일(이미지, 문서 등)**을 포함하여 대용량의 데이터를 전송할 때 반드시 사용해야 합니다.|
|**`text/plain`**|**순수 텍스트** 데이터.|특별한 구조 없이 텍스트 자체를 보낼 때 사용됩니다.|
|**`text/html`**|**HTML 문서** 형식의 데이터.|웹 서버가 브라우저에 HTML 페이지를 보낼 때 사용됩니다.|

## 📦 Content-Type 별 요청/응답 바디 예시

### 1. `application/json`

- **설명**: 키-값 쌍으로 이루어진 구조화된 데이터 형식입니다.
    
- **HTTP 바디 예시**:
    

```
{"userId": "user123", "email": "test@gmail.com", "age": 30}
```
---
### 2. `application/x-www-form-urlencoded`

- **설명**: HTML 폼 데이터가 기본으로 사용하는 형식입니다. 키와 값은 `=`로 연결되고, 각 필드는 `&`로 구분됩니다.
    
- **HTTP 바디 예시**:
    

```
userId=user123&email=test%40gmail.com&age=30
```
---
### 3. `text/plain`

- **설명**: 구조화되지 않은 순수한 텍스트 문자열입니다.
- **HTTP 바디 예시**:
    
```
user123, test@example.com, 30세 남성
```
---

### 4. `multipart/form-data` (파일 전송)

- **설명**: 일반 텍스트 데이터와 파일 데이터를 동시에 전송할 때 사용됩니다. 각 데이터 파트(`Part`)는 **`boundary`**라는 고유한 구분자로 분리됩니다.
    
- **헤더 예시**: `Content-Type: multipart/form-data; boundary=---WebKitFormBoundary7MA4YWgGmpTKF1x2`
    
- **HTTP 바디 예시**:
    
```
---WebKitFormBoundary7MA4YWgGmpTKF1x2
Content-Disposition: form-data; name="userId"

user123
---WebKitFormBoundary7MA4YWgGmpTKF1x2
Content-Disposition: form-data; name="profileImage"; filename="photo.jpg"
Content-Type: image/jpeg

[... 이 부분에 이미지 파일의 이진 데이터(Binary Data)가 들어갑니다 ...]
---WebKitFormBoundary7MA4YWgGmpTKF1x2--
```
---
### 5. `text/html` (서버 응답)

- **설명**: 웹 브라우저가 화면에 렌더링 할 HTML 문서의 내용입니다. 주로 서버에서 클라이언트로 응답할 때 사용됩니다.
- **HTTP 바디 예시**:
```html
<!DOCTYPE html>
<html>
<head>
    <title>응답 페이지</title>
</head>
<body>
    <h1>로그인 성공!</h1>
    <p>환영합니다, user123님.</p>
</body>
</html>
```

# JS Fetch API 요청

> [!summary] 요약
> **Fetch API** 사용 시 두 번째 인자인 `options` 객체의 구성 요소(**Method, Headers, Body**)를 정리한 노트임.
>   * **기본형**: 실무에서 가장 자주 사용하는 `POST` 요청 템플릿.
>   * **구성 요소**: 메서드(동작), 헤더(형식/인증), 바디(데이터)의 세부 설정법.

## 기본형 (Basic Syntax)

실무에서 가장 많이 사용되는 **JSON 데이터 전송 + 인증 토큰 포함** 형태의 표준 코드임.

```javascript
//================================post 방식
async function postData(파라미터) {
	//연결 url, data 선언
	url = '';
	data = '';
    // 1. 옵션(송신) 객체 정의
    const options = {
        method: 'POST', // 동작방식 (GET, POST, PUT, DELETE 등)
        headers: {
            'Content-Type': 'application/json;charset=utf-8', // 송신 데이터 타입
            Authorization: 'Bearer <인증토큰>' // 접근 권한 인증 토큰
        },
        body: JSON.stringify(data) // 송신 데이터 (JSON 문자열)
    };

    // 2. 요청 및 응답 처리
    try {
        const response = await fetch(url, options); //수신 데이터
        
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }
        
        const data = response.json();// 응답 데이터 파싱
        return data;
        
    } catch (error) {
        console.error('Fetch Error:', error);
    }
}

//================================get 방식
const response = await fetch(ajax.do?name=${name}&age=${age});
```
## 2\. Method (요청 동작)

서버 리소스에 대해 수행할 **작업의 종류**를 결정함.

| 메서드        | 역할                  | 특징             | Body 유무  |
| :--------- | :------------------ | :------------- | :------: |
| **GET**    | **조회 (Retrieve)**   | 데이터 가져오기 (기본값) |    X     |
| **POST**   | **생성 (Create)**     | 새로운 리소스 생성     |    O     |
| **PUT**    | **전체 수정 (Replace)** | 리소스를 통째로 교체    |    O     |
| **PATCH**  | **부분 수정 (Modify)**  | 리소스의 일부 필드만 변경 |    O     |
| **DELETE** | **삭제 (Remove)**     | 리소스 제거         | △ (보통 X) |

## 3\. Headers (메타데이터)

요청에 대한 부가 정보(데이터 타입, 인증 등)를 설정함.

### Content-Type (데이터 형식)

서버에 **보내는 데이터(`body`)의 형식**을 알리는 용도임.

| 값 (Value)                           | 설명           | 사용 상황                          |
| :---------------------------------- | :----------- | :----------------------------- |
| `application/json`                  | **JSON 데이터** | 대부분의 REST API 요청 시 사용          |
| `application/x-www-form-urlencoded` | **URL 인코딩**  | HTML Form 태그의 기본 전송 방식         |
| `multipart/form-data`               | **바이너리/파일**  | 파일 업로드 시 사용 <br>(**수동 설정 금지**) |
| `text/plain`                        | **일반 텍스트**   | 단순 문자열 전송 시 사용                 |
- method:'POST' 일때는 생략가능(자동으로 맞춰준다.)
### Authorization (인증)

사용자의 신원을 증명하기 위한 토큰 등을 포함함.

  * **Bearer Token**: `Authorization: 'Bearer <ACCESS_TOKEN>'` (JWT 등 가장 흔함)
  * **Basic Auth**: `Authorization: 'Basic <BASE64_STRING>'` (ID:PW 인코딩)

## 4\. Body (전송 데이터)

`Method`가 POST, PUT, PATCH일 때 실질적인 데이터를 담는 공간임. **Header의 `Content-Type`과 형식이 일치해야 함.**

### JSON 데이터 (가장 권장)

  * **Header**: `'Content-Type': 'application/json'`
  * **Body**: `JSON.stringify()` 메서드로 객체를 문자열화해야 함.

```javascript
body: JSON.stringify({
    username: "tutor_student",
    age: 25
})
```

### 일반 텍스트 및 XML (Raw Text)
단순 문자열이나 XML, HTML, CSV 등을 그대로 전송할 때 사용함.

- **Header**: `'Content-Type': 'text/plain'` (또는 `'text/html'`, `'application/xml'`)
- **Body**: 일반 문자열(String).

```js
// 단순 텍스트 전송
body: "이것은 일반 텍스트 데이터입니다." 

// XML 전송
body: "<user><name>user</name></user>" 
```

### Form Data (파일 업로드)

  * **Header**: **설정하지 않음** (브라우저가 자동으로 `boundary`를 포함한 Content-Type을 생성함).
  * **Body**: `FormData` 객체 사용.

```javascript
const formData = new FormData();
formData.append('file', fileInput.files[0]);

// 주의: Content-Type 헤더를 직접 설정하면 안 됨
body: formData 
```

### URL Encoded (레거시 방식)

  * **Header**: `'Content-Type': 'application/x-www-form-urlencoded'`
  * **Body**: 쿼리 스트링 형태(`key=value&key2=value2`).

```javascript
const params = new URLSearchParams();
params.append('id', 'user1');

body: params.toString() // "id=user1"
```

#### 디테일한 설명
```js
//async : 비동기 함수 선언
async function postData(파라미터) {
	//연결 url, data 선언
	url = '';
	data = '';

    // 1. 옵션(송신) 객체 정의
    const options = {
        method: 'POST', // 동작방식 (GET, POST, PUT, DELETE 등)
        headers: {
            'Content-Type': 'application/json', // 송신 데이터 타입
            Authorization: 'Bearer <인증토큰>' // 접근 권한 인증 토큰
        },
        body: JSON.stringify(data) // 송신 데이터 (JSON 문자열)
    };

    // 2. 요청 및 응답 처리
    try {
	    //await : 데이터 수신 대기
        const response = await fetch(url, options); //수신 데이터
        
        //404나 500같은 에러를 보냈는지 체크
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }
        
        const data = response.json();// 응답 데이터 파싱
        return data;
        
    } catch (error) {
        console.error('Fetch Error:', error);
    }
}
```

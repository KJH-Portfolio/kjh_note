# Axios 인터셉터(Interceptors)의 개념 및 실전 활용

> [!summary]
> Axios 인스턴스를 활용하여 HTTP 요청 및 응답 과정을 가로채 공통 로직(인증 토큰 주입, 에러 핸들링, 로깅 등)을 중앙 집중식으로 관리하는 방식에 대한 정리.

---
## **Axios 인터셉터 개요**

* **정의**: HTTP 요청(Request) 또는 응답(Response)이 처리되기 전 단계를 가로채 특정 로직을 주입하는 기능.
* **주요 목적**: 모든 API 호출에 공통적으로 적용될 전처리 및 후처리 로직의 단일 관리점 구축.
* **활용 사례**:
* **인증 보안**: 모든 요청 헤더에 JWT(JSON Web Token) 자동 첨부.
* **에러 전역 처리**: HTTP 상태 코드별 공통 에러 메시지 출력 및 리다이렉션.
* **데이터 가공**: 서버 응답 데이터의 일관된 포맷팅.
* **상태 관리**: 로딩 인디케이터 활성화 및 통신 로그 기록.

---
## **Axios 인스턴스 초기 설정**

* **기본 설정**: `axios.create()`를 통한 공통 설정값(URL, 타임아웃, 헤더) 주입.

```javascript
import axios from 'axios';

const api = axios.create({
    baseURL: '/api',
    timeout: 1000,
    headers: {
        'Content-Type': 'application/json',
    }
});
```

## **요청 인터셉터 (Request Interceptor)**

* **기능**: 서버로 요청이 전달되기 직전 실행되는 전처리 로직.
* **주요 로직**:
* **로컬 스토리지 액세스**: 저장된 인증 토큰 추출.
* **헤더 조작**: `Authorization` 필드에 `Bearer` 토큰 주입.
* **로깅**: 요청 URL 및 파라미터 정보 출력.

```javascript
//헤더에 토큰 끼워넣는 로직
api.interceptors.request.use(
	//응답
	(config) => {
	    // 1. 기록 남기기 (CCTV)
	    console.log(`API 요청 : ${config.url}`, config);
	
	    // 2. 출입증(토큰) 꺼내기
	    const token = localStorage.getItem('token');
	    
	    // 3. 출입증이 있으면 목에 걸어주기
	    if (token) {
	        config.headers.Authorization = `Bearer ${token}`;
	    }
	    
	    // 4. "준비 완료! 가세요!"
	    return config;
	}
	//에러
    (error) => {
        console.error('요청 에러 발생', error);
        return Promise.reject(error);
    }
);
```
#### config 관련 속성
| **속성 (Property)** | **설명**                     | **예시**                                                     |
| ----------------- | -------------------------- | ---------------------------------------------------------- |
| **`method`**      | HTTP 요청 방식 (소문자)           | `'get'`, `'post'`, `'put'` 등                               |
| **`baseURL`**     | 요청의 기본이 되는 주소              | `https://api.example.com`                                  |
| **`headers`**     | 요청에 포함될 헤더 정보              | `{ Authorization: '...', 'Content-Type': '...' }`          |
| **`params`**      | URL 파라미터<br>(Query String) | `api.get('/s', { params: { q: '검색어' } })` 에서 `{a : '검색어'}` |
| **`data`**        | POST, PUT 요청 시 담기는 본문      | `{ name: '김지환', age: 11 }`                                 |
| **`timeout`**     | 요청이 만료될 때까지의 시간            | `5000` (5초)                                                |

## **응답 인터셉터 (Response Interceptor)**

* **기능**: 서버 응답 수신 후 애플리케이션의 `then` 또는 `catch` 블록으로 전달되기 전 수행되는 후처리 로직.
* **상태 코드별 처리 전략**:
* **400 (Bad Request)**: 잘못된 요청 메시지 처리.
* **401 (Unauthorized)**: 토큰 만료 처리, 세션 초기화 및 로그인 페이지 강제 이동.
* **403 (Forbidden)**: 권한 부족 알림.
* **404 (Not Found)**: 리소스 미존재 예외 처리.
* **500 (Internal Server Error)**: 서버 측 결함 공통 안내.

```javascript
//응답시 오류 내용을 콘솔에 찍는 로직
api.interceptors.response.use(

    (response) => {
        console.log('API 응답 수신:', response);
        return response;
    },
    
    (error) => {
        if (error.response) {
            const { status, data } = error.response;
            
            switch (status) {
                case 400: console.error('잘못된 요청'); break;
                case 401:
                    console.error('인증 만료');
                    localStorage.removeItem('token');
                    window.location.href = '/login';
                    break;
                case 403: console.error('접근 거부'); break;
                case 404: console.error('리소스 미존재'); break;
                case 500: console.error('서버 내부 오류'); break;
                default: console.error('기타 에러 발생');
            }
        } else if (error.request) {
            console.error('서버 무응답');
        }
        return Promise.reject(error);
    }
);

```

## **API 모듈화 및 React 컴포넌트 적용**

### **API 서비스 모듈 정의**

* **기능 분리**: 인터셉터가 포함된 인스턴스를 활용한 도메인별 API 함수 정의.

```javascript
export const boardAPI = {
    getList: (page = 1, size = 10, searchParams = {}) => {
        return api.get('/list', {
            params: { page, size, ...searchParams }
        });
    },
};
```

### **React 컴포넌트 내 활용**

* **비동기 처리**: `async/await`를 통한 API 호출 및 상태 반영.
* **예외 관리**: 인터셉터에서 걸러지지 않은 비즈니스 로직 에러의 개별 처리.

```javascript
import { useState, useEffect } from 'react';

const InterceptorExample = () => {
    const [boards, setBoards] = useState([]);
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState(null);
    
    //검색조건 상태 관리
    const [searchParams,setSearchParams] = uesState({
	    searchType : 'title',
	    keyword: ''
    })

    const fetchBoards = async () => {
        try {
            setLoading(true);
            const response = await boardAPI.getList(1, 10, searchParams);
            setBoards(response.data);
        } catch (err) {
            setError('데이터 로드 실패');
        } finally {
            setLoading(false);
        }
    };

    useEffect(() => {
        fetchBoards();
    }, []);

	return (
	        <div>
	            <h2>게시글 목록</h2>
	            {/* 검색 UI 예시 */}
	            <input 
	                type="text" 
	                onChange={(e) => setSearchParams({...searchParams, keyword: e.target.value})}
	                placeholder="검색어 입력"
	            />
	            <button onClick={() => fetchBoards(1)}>검색</button>
	
	            {loading && <p>데이터 로딩 중...</p>}
	            {error && <p style={{ color: 'red' }}>{error}</p>}
	            {boards.map(board => (
	                <div key={board.boardNo}>
	                    <p>{board.boardTitle}</p>
	                </div>
	            ))}
	        </div>
	    );
};

export default InterceptorExample;
```

# Axios 기초

> [!summary]
> 브라우저 및 Node.js 환경을 위한 **Promise 기반 HTTP 클라이언트** 라이브러리. `fetch` API 대비 간결한 문법, 자동 JSON 변환, 요청/응답 인터셉터 등 강력한 부가 기능 제공을 통한 비동기 통신 효율화의 핵심 도구.
> - ! `npm install axios`로 설치해야 사용 가능한 라이브러리

---
## 기본 문법

```jsx
const registerUser = async (파라미터) => {
    const response = await axios.post('url', 데이터);
    return response.data;
};
```

## Axios 핵심 특징 및 장점

* **자동 JSON 데이터 변환**: HTTP 응답 본문(Body)의 JSON 형식을 자바스크립트 객체로 자동 파싱 처리.
* **요청 및 응답 인터셉터**: 통신 전후 단계에서 헤더 설정, 로깅, 에러 처리 등 공통 로직의 일괄 주입.
* **요청 취소 기능**: `AbortController`를 활용한 진행 중인 HTTP 요청의 명시적 중단 및 제어.
* **폭넓은 브라우저 호환성**: 구형 브라우저 지원 및 서버 사이드(Node.js) 통신 환경의 일관성 유지.
* **HTTP 헤더 자동 설정**: `POST` 등 데이터 전송 시 `Content-Type: application/json` 등의 헤더 자동 구성.

## HTTP 메서드별 파라미터 구조

### 데이터 요청 및 설정 (GET / DELETE)

* **메서드 서명**: `axios.get(url[, config])`, `axios.delete(url[, config])`.
* **URL**: 대상 서버의 엔드포인트 주소(필수 문자열).
* **Config**: 요청 설정을 담은 객체(선택).
* **params**: URL 쿼리 스트링으로 변환될 객체 (`{ id: 1 }` -> `?id=1`).
* **headers**: 사용자 정의 HTTP 헤더.
* **data**: `DELETE` 요청 시 본문(Body) 데이터를 전송해야 할 경우 `config.data` 속성에 명시.

### 데이터 전송 및 수정 (POST / PUT / PATCH)

* **메서드 서명**: `axios.post(url[, data[, config]])`, `axios.put(url[, data[, config]])`.
* **Data**: 서버로 전송할 페이로드 객체(필수). 별도 직렬화 없이 자바스크립트 객체 그대로 전달.
* **Config**: 데이터 전송 외 부가적인 통신 옵션 설정 객체.

```jsx
import axios from 'axios';

// 1. GET: 파라미터 객체 활용
const fetchUser = async (userId) => {
    const res = await axios.get('/api/users', {
        params: { id: userId } // ?id=userId 형태로 변환
    });
    return res.data;
};

// 2. POST,PUT,PATCH: 데이터 객체 전송
const registerUser = async (userData) => {
    const res = await axios.post('/api/users', userData); // JSON 자동 변환
    return res.data;
};

// 주의! DELETE: 본문 데이터 포함 시 주의사항
const removeAccount = async (id, reason) => {
    const res = await axios.delete(`/api/users/${id}`, {
    // DELETE는 두 번째 인자가 config이므로 data 속성이라고 명시해야함 {reson}만 사용 불가
        data: { reason } 
    });
};
```
- put은 전부 교체, patch는 일부 교체 방식으로 사용
- put할때 필드를 누락시키면 null로 들어감

## 응답(Response) 객체 구성 요소

* **data**: 서버에서 반환한 실제 응답 본문 데이터(가장 빈번하게 참조).
* **status**: HTTP 상태 코드 (예: 200, 201, 404, 500).
* **statusText**: HTTP 상태 메시지 (예: "OK", "Not Found").
* **headers**: 서버로부터 수신한 응답 헤더 객체.
* **config**: 요청 시 Axios에 전달되었던 모든 설정 정보.
* **request**: 응답을 생성한 최종 요청 객체(브라우저 환경의 XMLHttpRequest 등).

## 실전 활용 보완사항 및 예외 처리

* **Try-Catch 기반 에러 핸들링**: 비동기 호출 시 `try-catch` 블록을 통한 네트워크 단절 및 서버 에러(4xx, 5xx)의 명시적 처리.
* **Instance 생성**: `axios.create()`를 통한 베이스 URL(`baseURL`) 및 공통 타임아웃, 헤더의 모듈화 관리.
* **에러 응답 식별**: `error.response` 존재 여부를 통한 서버 응답 에러와 네트워크 에러의 구분.

```javascript
// Axios 인스턴스 생성 및 공통 에러 처리 예시
const apiClient = axios.create({
    baseURL: 'https://api.example.com',
    timeout: 3000
});

const safeRequest = async () => {
    try {
        const response = await apiClient.get('/data');
    } catch (error) {
        if (error.response) {
            // 서버 응답 에러 (4xx, 5xx)
            console.error(error.response.status, error.response.data);
        } else if (error.request) {
            // 요청 전송 후 응답 미수신 (네트워크 오류)
            console.error("No Response Received");
        } else {
            // 설정 오류 등 기타 에러
            console.error("Request Setup Error", error.message);
        }
    }
};

```

# AJAX & Spring 연동 가이드

> [!summary] 핵심 요약
> **AJAX**는 전체 페이지 새로고침 없이 서버와 비동기적으로 데이터를 주고받는 기술입니다.
>
>   * **Client**: `Fetch API`(권장) 또는 `jQuery`를 통해 비동기 요청 전송.
>   * **Server**: Spring `@ResponseBody`를 사용하여 뷰(View)가 아닌 **데이터(String/JSON)** 자체를 반환.

## 동기식 vs 비동기식

| 구분         | 동기식 (Synchronous)        | 비동기식 (Asynchronous)       |
| :--------- | :----------------------- | :------------------------ |
| **개념**     | 요청 후 응답이 올 때까지 대기        | 요청 후 응답 대기 없이 다른 작업 수행    |
| **페이지 로드** | **전체 페이지** 새로고침 (Reload) | **일부 영역**만 갱신 (No Reload) |
| **특징**     | 결과 지연 시 대기 상태 지속         | 아이디 중복체크, 검색어 자동완성 등에 활용  |
| **단점**     | 화면 깜빡임 발생                | 구현 복잡, 리소스 누적 시 성능 저하 가능성 |

##### async/await
- **async** :
	- 함수가 **Promise를** 반환하도록 강제함
	- 내부에서 **await함수**를 사용할 수 있게함
- **await** : 
	- fetch()가 데이터를 가져올때까지 기다려주고 **response 객체**를 받음
	- 미사용시 대기(Pending)상태의 **Promise 객체**를 반환 받음
	- async 함수 내부 또는 모듈의 최상위 레벨에서만 유효함

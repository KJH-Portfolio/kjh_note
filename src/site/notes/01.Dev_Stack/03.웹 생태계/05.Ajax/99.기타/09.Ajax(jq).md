# jQuery AJAX 요청

> [!summary] 요약
> **jQuery.ajax()** 메서드의 `settings` 객체 구성 요소(**Method, Headers, Data**)를 정리한 노트임.
>
>   * **기본형**: `async/await`를 활용한 최신 실무 패턴 (jQuery 3.0+).
>   * **주요 차이**: `body` 대신 `data` 속성을 사용하며, `Content-Type` 설정 방식이 다름.
>   * **자동화**: 응답 데이터가 JSON일 경우 자동으로 파싱됨 (`JSON.parse` 불필요).

## 1. 기본형 (Basic Syntax)

실무에서 사용하는 **JSON 데이터 전송 + 인증 토큰 + Async/Await** 패턴의 표준 코드임.

```javascript
$.ajax({
    url: "url",
    type: "POST", // ★ 생략하면 get 방식
    data: { 
        name: "홍길동",
        age: 20 
    },
    success: function(res) { ... },
    error: function(err) { ... }
});
```
  * **url**: 요청할 서버 주소 (필수).
  * **type/method**: 요청 방식 (GET/POST, 생략 시 GET).
  * **data**: 서버로 보낼 전달값 (Object 형태).
  * **success**: 통신 성공 시 실행할 콜백 함수.
  * **error**: 통신 실패 시 실행할 콜백 함수.

## 2\. Method (요청 동작)

서버 리소스에 대해 수행할 **작업의 종류**를 결정함. (`method` 대신 `type`을 써도 되지만, 최신 표준은 `method` 권장)

| 메서드 | 역할 | 특징 | Data 유무 |
| :--- | :--- | :--- | :---: |
| **GET** | **조회 (Retrieve)** | 데이터 가져오기 (기본값) | X |
| **POST** | **생성 (Create)** | 새로운 리소스 생성 | O |
| **PUT** | **전체 수정 (Replace)** | 리소스를 통째로 교체 | O |
| **PATCH** | **부분 수정 (Modify)** | 리소스의 일부 필드만 변경 | O |
| **DELETE** | **삭제 (Remove)** | 리소스 제거 | △ |

## 3\. Headers & ContentType (메타데이터)

jQuery는 `contentType` 속성이 별도로 존재하며, 그 외 헤더는 `headers` 객체에 담음.

### contentType (보내는 데이터 형식)

서버에 **보내는 데이터(`data`)의 형식**을 지정함. (Fetch의 `Headers: {'Content-Type': ...}`과 동일)

| 값 (Value)                           | 설명           | 사용 상황                          |
| :---------------------------------- | :----------- | :----------------------------- |
| `application/json`                  | **JSON 데이터** | REST API 요청 시 필수               |
| `application/x-www-form-urlencoded` | **URL 인코딩**  | **기본값** (설정 안 하면 이 방식으로 전송됨)   |
| `false`                             | **바이너리/파일**  | `FormData` 전송 시 사용 (**필수 설정**) |
| `text/plain`                        | **일반 텍스트**   | 단순 문자열 전송 시 사용                 |

### Authorization (인증 헤더)

`headers` 객체 내부에 정의함.

  * **Bearer Token**: `headers: { 'Authorization': 'Bearer <TOKEN>' }`
  * **Basic Auth**: `headers: { 'Authorization': 'Basic <BASE64>' }`

## 4. Data (전송 데이터)

Fetch의 `body`에 해당하며, 속성명은 `data `
(method`가 POST, PUT, PATCH일 때 사용함.)

### JSON 데이터 (가장 권장)

  * **contentType**: `'application/json'` **(필수)**
  * **data**: `JSON.stringify()`를 사용하여 객체를 문자열로 변환해야 함.

```javascript
$.ajax({
    // ...
    contentType: 'application/json',
    data: JSON.stringify({ username: "tutor", age: 25 })
});
```

### 일반 텍스트 및 XML (Raw Text)

단순 문자열이나 XML 등을 전송할 때 사용.

  * **contentType**: `'text/plain'` (또는 `'text/html'`, `'application/xml'`)
  * **data**: 일반 문자열(String).

```javascript
// 단순 텍스트 전송
$.ajax({
    // ...
    contentType: 'text/plain',
    data: "이것은 일반 텍스트 데이터입니다."
});
```

### Form Data (파일 업로드)

가장 주의해야 할 부분임. **두 가지 속성(`processData`, `contentType`)을 반드시 `false`로 설정해야 함.**

  * **contentType**: `false` (jQuery가 자동으로 Boundary 헤더를 붙이도록 함)
  * **processData**: `false` (jQuery가 데이터를 쿼리 스트링으로 변환하는 것을 방지)
  * **data**: `FormData` 객체.

```javascript
const formData = new FormData();
formData.append('file', fileInput.files[0]);

$.ajax({
    method: 'POST',
    data: formData,
    processData: false, // 중요: QueryString 변환 방지
    contentType: false  // 중요: multipart/form-data 헤더 자동 생성
});
```

### URL Encoded (기본 동작)

jQuery는 객체를 `data`에 넣으면 **자동으로 이 형식으로 변환**함.

  * **contentType**: 생략 가능 (기본값이 `'application/x-www-form-urlencoded'`)
  * **data**: 일반 자바스크립트 객체 (Stringify 불필요).

```javascript
// 결과: id=user1&pw=1234
$.ajax({
	//..기타 정보(생략가능)
    // contentType 생략 시 자동 적용
    data: { 
        id: 'user1', 
        pw: '1234' 
    } 
});
```

### 💡 Fetch vs jQuery 주요 차이점 (Checklist)

> [!EXAMPLE] 마이그레이션 체크리스트
>
> 1.  **속성명**: Fetch의 `body` -\> jQuery의 `data`로 변경되었는가?
> 2.  **헤더 설정**: `Content-Type`을 `headers` 안이 아니라 `contentType` 속성으로 뺐는가?
> 3.  **JSON 처리**: `JSON.stringify()`는 여전히 필요하지만, 응답(`response`)은 자동으로 파싱되므로 `response.json()`을 제거했는가?
> 4.  **FormData**: `processData: false`, `contentType: false` 옵션을 추가했는가?

| **구분**         | **Vanilla JS (Fetch API)**                     | **jQuery ($.ajax)**                                                      |
| -------------- | ---------------------------------------------- | ------------------------------------------------------------------------ |
| **JSON 자동 변환** | **수동 처리 필요**                                   | **자동 처리** (편의성 높음)                                                       |
| **처리 방식**      | 응답 스트림을 읽기 위해 **`.json()` 메소드를 명시적으로 호출**해야 함. | 서버 응답 헤더 (`Content-Type: application/json`)를 보고 **자바스크립트 객체로 자동 파싱**해 줌. |
#### 실무 예시(jquery)
```js
		$("#btn3").click(function(){
			$.ajax({
				url : "openData3.do",
				success : function(data){
					//responseStr에서 추출하고자 하는 item목록 추출
					let items = data.response.body.items;
					//반복문을 이용하여 tr 만들어 추가하기 
					for(let item of items){
						let tr = $("<tr>");
						tr.append($("<td>").text(item.clearVal),
								$("<td>").text(item.sn),
								$("<td>").text(item.districtName),
								$("<td>").text(item.dataDate),
								$("<td>").text(item.issueVal),
								$("<td>").text(item.issueTime),
								$("<td>").text(item.clearDate),
								$("<td>").text(item.issueDate),
								$("<td>").text(item.moveName),
								$("<td>").text(item.clearTime),
								$("<td>").text(item.issueGbn),
								$("<td>").text(item.itemCode),
								);
						//tr넣어주기
						$("table tbody").append(tr);
					}
				},
				error : function(){
					console.log("통신오류");
				}
			});
		});
```
